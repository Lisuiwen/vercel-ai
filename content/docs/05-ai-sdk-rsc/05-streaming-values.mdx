# 流式传输值

<Note type="warning">
  AI SDK RSC 目前处于实验阶段。我们建议在生产环境使用 [AI SDK
  UI](/docs/ai-sdk-ui/overview)。有关从 RSC 迁移到 UI 的指南，请参阅[迁移指南](/docs/ai-sdk-rsc/migrating-to-ui)。
</Note>

RSC API 提供多种实用函数，允许你从服务器向客户端流式传输值。当你需要对流式传输的内容和方式进行更细粒度的控制时，这会很有用。

<Note>
  这些实用函数也可与 [AI SDK Core](/docs/ai-sdk-core) 的函数（如 [`streamText`](/docs/reference/ai-sdk-core/stream-text)）配合使用，轻松将 LLM 生成从服务器流式传输到客户端。
</Note>

RSC API 提供两个函数用于创建可流式传输的值：

- [`createStreamableValue`](/docs/reference/ai-sdk-rsc/create-streamable-value) — 创建可流式传输的（可序列化）值，完全控制如何创建、更新和关闭流。
- [`createStreamableUI`](/docs/reference/ai-sdk-rsc/create-streamable-ui) — 创建可流式传输的 React 组件，完全控制如何创建、更新和关闭流。

## `createStreamableValue`

RSC API 允许你使用 [`createStreamableValue`](/docs/reference/ai-sdk-rsc/create-streamable-value) 将可序列化的 JavaScript 值从服务器流式传输到客户端，例如字符串、数字、对象和数组。

适用于流式传输：

- 语言模型的实时文本生成。
- 多模态模型的图像和音频生成的缓冲值。
- 多步 agent 运行的进度更新。

## 创建可流式传输的值

你可以从 `@ai-sdk/rsc` 导入 `createStreamableValue` 并用于创建可流式传输的值。

```tsx file='app/actions.ts'
'use server';

import { createStreamableValue } from '@ai-sdk/rsc';

export const runThread = async () => {
  const streamableStatus = createStreamableValue('thread.init');

  setTimeout(() => {
    streamableStatus.update('thread.run.create');
    streamableStatus.update('thread.run.update');
    streamableStatus.update('thread.run.end');
    streamableStatus.done('thread.end');
  }, 1000);

  return {
    status: streamableStatus.value,
  };
};
```

## 读取可流式传输的值

你可以在客户端使用 `readStreamableValue` 读取可流式传输的值。它返回一个 async iterator，在值更新时产出流式值：

```tsx file='app/page.tsx'
import { readStreamableValue } from '@ai-sdk/rsc';
import { runThread } from '@/actions';

export default function Page() {
  return (
    <button
      onClick={async () => {
        const { status } = await runThread();

        for await (const value of readStreamableValue(status)) {
          console.log(value);
        }
      }}
    >
      Ask
    </button>
  );
}
```

了解如何在 Next.js App Router 中使用 `createStreamableValue` 流式传输文本生成（配合 `streamText`），请参阅此[示例](/examples/next-app/basics/streaming-text-generation)。

## `createStreamableUI`

`createStreamableUI` 创建持有 React 组件的流。与 AI SDK Core API 不同，此函数不调用大语言模型，而是提供可对 React 组件流式传输进行细粒度控制的 primitive。

## 使用 `createStreamableUI`

让我们看看如何将 `createStreamableUI` 与 Server Action 配合使用。

```tsx filename='app/actions.tsx'
'use server';

import { createStreamableUI } from '@ai-sdk/rsc';

export async function getWeather() {
  const weatherUI = createStreamableUI();

  weatherUI.update(<div style={{ color: 'gray' }}>Loading...</div>);

  setTimeout(() => {
    weatherUI.done(<div>It&apos;s a sunny day!</div>);
  }, 1000);

  return weatherUI.value;
}
```

首先，创建一个空状态的可流式传输 UI，然后用加载消息更新它。1 秒后，传入实际天气信息作为最终值并标记流完成。`.value` 属性包含可发送到客户端的实际 UI。

## 读取可流式传输 UI

在客户端，你可以调用 `getWeather` Server Action 并像渲染任何其他 React 组件一样渲染返回的 UI。

```tsx filename='app/page.tsx'
'use client';

import { useState } from 'react';
import { readStreamableValue } from '@ai-sdk/rsc';
import { getWeather } from '@/actions';

export default function Page() {
  const [weather, setWeather] = useState<React.ReactNode | null>(null);

  return (
    <div>
      <button
        onClick={async () => {
          const weatherUI = await getWeather();
          setWeather(weatherUI);
        }}
      >
        What&apos;s the weather?
      </button>

      {weather}
    </div>
  );
}
```

点击按钮时，会调用 `getWeather` 函数，返回的 UI 会设置到 `weather` 状态并渲染在页面上。用户会先看到加载消息，1 秒后看到实际天气信息。

有关在单个请求中处理多个流的更多信息，请参阅[多个可流式传输对象](/docs/advanced/multiple-streamables)指南。

有关更复杂用例的状态处理，请参阅 [AI/UI 状态](/docs/ai-sdk-rsc/generative-ui-state)。
