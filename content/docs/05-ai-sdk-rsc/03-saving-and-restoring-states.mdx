# 保存和恢复状态

<Note type="warning">
  AI SDK RSC 目前处于实验阶段。我们建议在生产环境使用 [AI SDK
  UI](/docs/ai-sdk-ui/overview)。有关从 RSC 迁移到 UI 的指南，请参阅[迁移指南](/docs/ai-sdk-rsc/migrating-to-ui)。
</Note>

AI SDK RSC 提供保存和恢复 AI 与 UI 状态的便捷方法。这对于在每次模型生成后保存应用状态，并在用户重新访问生成内容时恢复状态很有用。

## AI 状态

### 保存 AI 状态

可以使用 [`onSetAIState`](/docs/reference/ai-sdk-rsc/create-ai#on-set-ai-state) 回调保存 AI 状态，该回调在 AI 状态更新时调用。在以下示例中，每当生成标记为完成时，你将聊天历史保存到数据库。

```tsx filename='app/ai.ts'
export const AI = createAI<ServerMessage[], ClientMessage[]>({
  actions: {
    continueConversation,
  },
  onSetAIState: async ({ state, done }) => {
    'use server';

    if (done) {
      saveChatToDB(state);
    }
  },
});
```

### 恢复 AI 状态

可以使用 [`createAI`](/docs/reference/ai-sdk-rsc/create-ai) 函数创建的上下文提供者所接收的 [`initialAIState`](/docs/reference/ai-sdk-rsc/create-ai#initial-ai-state) prop 恢复 AI 状态。在以下示例中，组件挂载时从数据库恢复聊天历史。

```tsx file='app/layout.tsx'
import { ReactNode } from 'react';
import { AI } from './ai';

export default async function RootLayout({
  children,
}: Readonly<{ children: ReactNode }>) {
  const chat = await loadChatFromDB();

  return (
    <html lang="en">
      <body>
        <AI initialAIState={chat}>{children}</AI>
      </body>
    </html>
  );
}
```

## UI 状态

### 保存 UI 状态

UI 状态无法直接保存，因为其内容尚不可序列化。你可以使用 AI 状态作为代理来存储 UI 状态的详细信息，并在需要时用它恢复 UI 状态。

### 恢复 UI 状态

可以使用 AI 状态作为代理恢复 UI 状态。在以下示例中，组件挂载时从 AI 状态恢复聊天历史。你使用 [`onGetUIState`](/docs/reference/ai-sdk-rsc/create-ai#on-get-ui-state) 回调监听 SSR 事件并恢复 UI 状态。

```tsx filename='app/ai.ts'
export const AI = createAI<ServerMessage[], ClientMessage[]>({
  actions: {
    continueConversation,
  },
  onGetUIState: async () => {
    'use server';

    const historyFromDB: ServerMessage[] = await loadChatFromDB();
    const historyFromApp: ServerMessage[] = getAIState();

    // If the history from the database is different from the
    // history in the app, they're not in sync so return the UIState
    // based on the history from the database

    if (historyFromDB.length !== historyFromApp.length) {
      return historyFromDB.map(({ role, content }) => ({
        id: generateId(),
        role,
        display:
          role === 'function' ? (
            <Component {...JSON.parse(content)} />
          ) : (
            content
          ),
      }));
    }
  },
});
```

要了解更多，请查看此[示例](/examples/next-app/state-management/save-and-restore-states)，了解如何在 Next.js 应用中持久化和恢复状态。
