# AI SDK RSC

<Note type="warning">
  AI SDK RSC 目前处于实验阶段。我们建议在生产环境使用 [AI SDK
  UI](/docs/ai-sdk-ui/overview)。有关从 RSC 迁移到 UI 的指南，请参阅[迁移指南](/docs/ai-sdk-rsc/migrating-to-ui)。
</Note>

<Note>
  `@ai-sdk/rsc` 包兼容支持 React Server Components 的框架。
</Note>

[React Server Components](https://nextjs.org/docs/app/building-your-application/rendering/server-components)（RSC）允许你编写可在服务器上渲染并流式传输到客户端的 UI。RSC 支持 [Server Actions](https://nextjs.org/docs/app/building-your-application/data-fetching/server-actions-and-mutations#with-client-components)，这是一种像调用普通函数一样从客户端直接调用服务端代码的新方式，并具有端到端类型安全。这种组合为构建 AI 应用开辟了新途径，使大语言模型（LLM）能够直接从服务器向客户端生成并流式传输 UI。

## AI SDK RSC 函数

AI SDK RSC 提供多种函数，旨在帮助你使用 React Server Components 构建 AI 原生应用。这些函数：

1. 为构建生成式 UI 应用提供抽象。
   - [`streamUI`](/docs/reference/ai-sdk-rsc/stream-ui)：调用模型并允许其使用 React Server Components 响应。
   - [`useUIState`](/docs/reference/ai-sdk-rsc/use-ui-state)：返回当前 UI 状态及更新 UI 状态的函数（类似 React 的 `useState`）。UI 状态是 AI 状态的可视化表示。
   - [`useAIState`](/docs/reference/ai-sdk-rsc/use-ai-state)：返回当前 AI 状态及更新 AI 状态的函数（类似 React 的 `useState`）。AI 状态用于包含与 AI 模型共享的上下文和信息，例如系统消息、函数响应及其他相关数据。
   - [`useActions`](/docs/reference/ai-sdk-rsc/use-actions)：从客户端访问你的 Server Actions。这对于构建需要与服务器进行用户交互的界面特别有用。
   - [`createAI`](/docs/reference/ai-sdk-rsc/create-ai)：创建客户端-服务端上下文提供者，可用于包裹应用树的部分，轻松管理应用的 UI 和 AI 状态。
2. 简化在服务器与客户端之间处理可流式传输的值。
   - [`createStreamableValue`](/docs/reference/ai-sdk-rsc/create-streamable-value)：创建从服务器向客户端发送值的流。值可以是任何可序列化的数据。
   - [`readStreamableValue`](/docs/reference/ai-sdk-rsc/read-streamable-value)：从客户端读取最初使用 `createStreamableValue` 创建的可流式传输的值。
   - [`createStreamableUI`](/docs/reference/ai-sdk-rsc/create-streamable-ui)：创建从服务器向客户端发送 UI 的流。
   - [`useStreamableValue`](/docs/reference/ai-sdk-rsc/use-streamable-value)：接受使用 `createStreamableValue` 创建的可流式传输的值，并返回当前值、错误和 pending 状态。
