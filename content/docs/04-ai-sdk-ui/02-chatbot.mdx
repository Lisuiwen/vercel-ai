# 聊天机器人

`useChat` hook 可让你轻松为聊天机器人应用创建对话式用户界面。它支持从 AI 提供商流式传输聊天消息、管理聊天状态，并在新消息到达时自动更新 UI。

概括而言，`useChat` hook 提供以下功能：

- **消息流式传输**：来自 AI 提供商的所有消息都会实时流式传输到聊天 UI。
- **托管状态**：hook 为你管理输入、消息、状态、错误等状态。
- **无缝集成**：以最小工作量将聊天 AI 轻松集成到任何设计或布局中。

本指南将介绍如何使用 `useChat` hook 创建具有实时消息流式传输的聊天机器人应用。
查看我们的[带工具的聊天机器人指南](/docs/ai-sdk-ui/chatbot-tool-usage)了解如何在聊天机器人中使用工具。
让我们先从以下示例开始。

## 示例

```tsx filename='app/page.tsx'
'use client';

import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';
import { useState } from 'react';

export default function Page() {
  const { messages, sendMessage, status } = useChat({
    transport: new DefaultChatTransport({
      api: '/api/chat',
    }),
  });
  const [input, setInput] = useState('');

  return (
    <>
      {messages.map(message => (
        <div key={message.id}>
          {message.role === 'user' ? 'User: ' : 'AI: '}
          {message.parts.map((part, index) =>
            part.type === 'text' ? <span key={index}>{part.text}</span> : null,
          )}
        </div>
      ))}

      <form
        onSubmit={e => {
          e.preventDefault();
          if (input.trim()) {
            sendMessage({ text: input });
            setInput('');
          }
        }}
      >
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          disabled={status !== 'ready'}
          placeholder="Say something..."
        />
        <button type="submit" disabled={status !== 'ready'}>
          Submit
        </button>
      </form>
    </>
  );
}
```

```ts filename='app/api/chat/route.ts'
import { convertToModelMessages, streamText, UIMessage } from 'ai';
__PROVIDER_IMPORT__;

// Allow streaming responses up to 30 seconds
export const maxDuration = 30;

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    instructions: 'You are a helpful assistant.',
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

<Note>
  UI 消息具有新的 `parts` 属性，其中包含消息部分。
  我们建议使用 `parts` 属性而非 `content` 属性渲染消息。`parts` 属性支持不同的消息类型，包括文本、工具调用和工具结果，并允许更灵活、更复杂的聊天 UI。
</Note>

在 `Page` 组件中，每当用户通过 `sendMessage` 发送消息时，`useChat` hook 都会向你的 AI 提供商端点发起请求。
消息随后会实时流式返回并显示在聊天 UI 中。

这提供了流畅的聊天体验：用户无需等待完整响应即可看到 AI 回复。

## 自定义 UI

`useChat` 还提供通过代码管理聊天消息状态、显示状态以及在不依赖用户交互的情况下更新消息的方式。

### 状态

`useChat` hook 返回 `status`，可能的值如下：

- `submitted`：消息已发送到 API，正在等待响应流开始。
- `streaming`：正在从 API 主动流式接收响应数据块。
- `ready`：已接收并处理完整响应；可以提交新的用户消息。
- `error`：API 请求期间发生错误，无法成功完成。

你可以将 `status` 用于例如以下用途：

- 在聊天机器人处理用户消息时显示加载动画。
- 显示「Stop」按钮以中止当前消息。
- 禁用提交按钮。

```tsx filename='app/page.tsx' highlight="6,22-29,36"
'use client';

import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';
import { useState } from 'react';

export default function Page() {
  const { messages, sendMessage, status, stop } = useChat({
    transport: new DefaultChatTransport({
      api: '/api/chat',
    }),
  });
  const [input, setInput] = useState('');

  return (
    <>
      {messages.map(message => (
        <div key={message.id}>
          {message.role === 'user' ? 'User: ' : 'AI: '}
          {message.parts.map((part, index) =>
            part.type === 'text' ? <span key={index}>{part.text}</span> : null,
          )}
        </div>
      ))}

      {(status === 'submitted' || status === 'streaming') && (
        <div>
          {status === 'submitted' && <Spinner />}
          <button type="button" onClick={() => stop()}>
            Stop
          </button>
        </div>
      )}

      <form
        onSubmit={e => {
          e.preventDefault();
          if (input.trim()) {
            sendMessage({ text: input });
            setInput('');
          }
        }}
      >
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          disabled={status !== 'ready'}
          placeholder="Say something..."
        />
        <button type="submit" disabled={status !== 'ready'}>
          Submit
        </button>
      </form>
    </>
  );
}
```

### 错误状态

同样，`error` 状态反映 fetch 请求期间抛出的错误对象。
可用于显示错误消息、禁用提交按钮或显示重试按钮：

<Note>
  我们建议向用户显示通用错误消息，例如「出了点问题。」这是避免泄露服务器信息的好习惯。
</Note>

```tsx file="app/page.tsx" highlight="6,20-27,33"
'use client';

import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';
import { useState } from 'react';

export default function Chat() {
  const { messages, sendMessage, error, regenerate } = useChat({
    transport: new DefaultChatTransport({
      api: '/api/chat',
    }),
  });
  const [input, setInput] = useState('');

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.role}:{' '}
          {m.parts.map((part, index) =>
            part.type === 'text' ? <span key={index}>{part.text}</span> : null,
          )}
        </div>
      ))}

      {error && (
        <>
          <div>An error occurred.</div>
          <button type="button" onClick={() => regenerate()}>
            Retry
          </button>
        </>
      )}

      <form
        onSubmit={e => {
          e.preventDefault();
          if (input.trim()) {
            sendMessage({ text: input });
            setInput('');
          }
        }}
      >
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          disabled={error != null}
        />
      </form>
    </div>
  );
}
```

更多信息请参阅[错误处理](/docs/ai-sdk-ui/error-handling)指南。

### 修改消息

有时你可能希望直接修改已有消息。例如，可为每条消息添加删除按钮，让用户从聊天历史中移除它们。

可以使用 `setMessages` 函数完成这些任务：

```tsx
const { messages, setMessages } = useChat()

const handleDelete = (id) => {
  setMessages(messages.filter(message => message.id !== id))
}

return <>
  {messages.map(message => (
    <div key={message.id}>
      {message.role === 'user' ? 'User: ' : 'AI: '}
      {message.parts.map((part, index) => (
        part.type === 'text' ? (
          <span key={index}>{part.text}</span>
        ) : null
      ))}
      <button onClick={() => handleDelete(message.id)}>Delete</button>
    </div>
  ))}
  ...
```

可以将 `messages` 和 `setMessages` 视为 React 中的 `state` 和 `setState` 配对。

### 取消与重新生成

在 AI 提供商仍在流式返回响应时中止消息也是常见需求。可调用 `useChat` hook 返回的 `stop` 函数实现。

```tsx
const { stop, status } = useChat()

return <>
  <button onClick={stop} disabled={!(status === 'streaming' || status === 'submitted')}>Stop</button>
  ...
```

当用户点击「Stop」按钮时，fetch 请求将被中止。这避免消耗不必要的资源并改善聊天机器人应用体验。

同样，你可以调用 `useChat` hook 返回的 `regenerate` 函数，请求 AI 提供商重新处理最后一条消息：

```tsx
const { regenerate, status } = useChat();

return (
  <>
    <button
      onClick={regenerate}
      disabled={!(status === 'ready' || status === 'error')}
    >
      Regenerate
    </button>
    ...
  </>
);
```

当用户点击「Regenerate」按钮时，AI 提供商将重新生成最后一条消息并相应替换当前消息。

### 节流 UI 更新

<Note>此功能目前仅适用于 React。</Note>

默认情况下，`useChat` hook 在每次收到新块时都会触发渲染。
你可以通过 `experimental_throttle` 选项对 UI 更新进行节流。

```tsx filename="page.tsx" highlight="2-3"
const { messages, ... } = useChat({
  // Throttle the messages and data updates to 50ms:
  experimental_throttle: 50
})
```

## 事件回调

`useChat` 提供可选的事件回调，用于处理聊天机器人生命周期的不同阶段：

- `onFinish`：助手响应完成时调用。事件包含响应消息、所有消息，以及中止、断开连接和错误的标志。
- `onError`：fetch 请求发生错误时调用。
- `onData`：每当收到数据部分时调用。

这些回调可用于触发额外操作，例如日志、分析或自定义 UI 更新。

```tsx
import { UIMessage } from 'ai';

const {
  /* ... */
} = useChat({
  onFinish: ({ message, messages, isAbort, isDisconnect, isError }) => {
    // use information to e.g. update other UI states
  },
  onError: error => {
    console.error('An error occurred:', error);
  },
  onData: data => {
    console.log('Received data part from server:', data);
  },
});
```

值得注意的是，你可以在 `onData` 回调中抛出错误以中止处理。这将触发 `onError` 回调并阻止消息追加到聊天 UI。这对于处理 AI 提供商的意外响应很有用。

## 请求配置

### 自定义 headers、body 和 credentials

默认情况下，`useChat` hook 向 `/api/chat` 端点发送 HTTP POST 请求，并将消息列表作为请求体。你可以通过两种方式自定义请求：

#### Hook 级配置（应用于所有请求）

你可以配置传输层选项，这些选项将应用于 hook 发起的所有请求：

```tsx
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/custom-chat',
    headers: {
      Authorization: 'your_token',
    },
    body: {
      user_id: '123',
    },
    credentials: 'same-origin',
  }),
});
```

#### 动态 Hook 级配置

你也可以提供返回配置值的函数。这对于需要刷新的身份验证令牌或依赖运行时条件的配置很有用：

```tsx
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/custom-chat',
    headers: () => ({
      Authorization: `Bearer ${getAuthToken()}`,
      'X-User-ID': getCurrentUserId(),
    }),
    body: () => ({
      sessionId: getCurrentSessionId(),
      preferences: getUserPreferences(),
    }),
    credentials: () => 'include',
  }),
});
```

<Note>
  对于随时间变化的组件状态，使用 `useRef` 存储当前值并在配置函数中引用 `ref.current`，或优先使用请求级选项（见下一节）以获得更好的可靠性。
</Note>

#### 请求级配置（推荐）

<Note>
  **推荐**：使用请求级选项以获得更好的灵活性和控制。请求级选项优先于 hook 级选项，允许你单独自定义每个请求。
</Note>

```tsx
// Pass options as the second parameter to sendMessage
sendMessage(
  { text: input },
  {
    headers: {
      Authorization: 'Bearer token123',
      'X-Custom-Header': 'custom-value',
    },
    body: {
      temperature: 0.7,
      max_tokens: 100,
      user_id: '123',
    },
    metadata: {
      userId: 'user123',
      sessionId: 'session456',
    },
  },
);
```

请求级选项与 hook 级选项合并，请求级选项优先。在服务端，你可以使用这些额外信息处理请求。

### 为每个请求设置自定义 body 字段

你可以使用 `sendMessage` 函数的第二个参数按请求配置自定义 `body` 字段。
若要将消息列表之外的其他信息传给后端，这会很有用。

```tsx filename="app/page.tsx" highlight="20-25"
'use client';

import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const { messages, sendMessage } = useChat();
  const [input, setInput] = useState('');

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.role}:{' '}
          {m.parts.map((part, index) =>
            part.type === 'text' ? <span key={index}>{part.text}</span> : null,
          )}
        </div>
      ))}

      <form
        onSubmit={event => {
          event.preventDefault();
          if (input.trim()) {
            sendMessage(
              { text: input },
              {
                body: {
                  customKey: 'customValue',
                },
              },
            );
            setInput('');
          }
        }}
      >
        <input value={input} onChange={e => setInput(e.target.value)} />
      </form>
    </div>
  );
}
```

你可以在服务端通过解构请求体获取这些自定义字段：

```ts filename="app/api/chat/route.ts" highlight="3,4"
export async function POST(req: Request) {
  // Extract additional information ("customKey") from the body of the request:
  const { messages, customKey }: { messages: UIMessage[]; customKey: string } =
    await req.json();
  //...
}
```

## 消息元数据

你可以为消息附加自定义元数据，用于跟踪时间戳、模型详情和 token 用量等信息。

```ts
// Server: Send metadata about the message
return result.toUIMessageStreamResponse({
  messageMetadata: ({ part }) => {
    if (part.type === 'start') {
      return {
        createdAt: Date.now(),
        model: 'gpt-5.1',
      };
    }

    if (part.type === 'finish') {
      return {
        totalTokens: part.totalUsage.totalTokens,
      };
    }
  },
});
```

```tsx
// Client: Access metadata via message.metadata
{
  messages.map(message => (
    <div key={message.id}>
      {message.role}:{' '}
      {message.metadata?.createdAt &&
        new Date(message.metadata.createdAt).toLocaleTimeString()}

      {message.parts.map((part, index) =>
        part.type === 'text' ? <span key={index}>{part.text}</span> : null,
      )}
      {/* Show token count if available */}
      {message.metadata?.totalTokens && (

    </div>
  ));
}
```

## 传输配置

你可以使用 `transport` 选项配置自定义传输行为，以自定义消息如何发送到 API：

```tsx filename="app/page.tsx"
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

export default function Chat() {
  const { messages, sendMessage } = useChat({
    id: 'my-chat',
    transport: new DefaultChatTransport({
      prepareSendMessagesRequest: ({ id, messages }) => {
        return {
          body: {
            id,
            message: messages[messages.length - 1],
          },
        };
      },
    }),
  });

  // ... rest of your component
}
```

相应的 API 路由接收自定义请求格式：

```ts filename="app/api/chat/route.ts"
export async function POST(req: Request) {
  const { id, message } = await req.json();

  // Load existing messages and add the new one
  const messages = await loadMessages(id);
  messages.push(message);

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

### 高级：基于 trigger 的路由

对于消息重新生成等更复杂的场景，可以使用基于 trigger 的路由：

```tsx filename="app/page.tsx"
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

export default function Chat() {
  const { messages, sendMessage, regenerate } = useChat({
    id: 'my-chat',
    transport: new DefaultChatTransport({
      prepareSendMessagesRequest: ({ id, messages, trigger, messageId }) => {
        if (trigger === 'submit-user-message') {
          return {
            body: {
              trigger: 'submit-user-message',
              id,
              message: messages[messages.length - 1],
              messageId,
            },
          };
        } else if (trigger === 'regenerate-assistant-message') {
          return {
            body: {
              trigger: 'regenerate-assistant-message',
              id,
              messageId,
            },
          };
        }
        throw new Error(`Unsupported trigger: ${trigger}`);
      },
    }),
  });

  // ... rest of your component
}
```

相应的 API 路由将处理不同的 trigger：

```ts filename="app/api/chat/route.ts"
export async function POST(req: Request) {
  const { trigger, id, message, messageId } = await req.json();

  const chat = await readChat(id);
  let messages = chat.messages;

  if (trigger === 'submit-user-message') {
    // Handle new user message
    messages = [...messages, message];
  } else if (trigger === 'regenerate-assistant-message') {
    // Handle message regeneration - remove messages after messageId
    const messageIndex = messages.findIndex(m => m.id === messageId);
    if (messageIndex !== -1) {
      messages = messages.slice(0, messageIndex);
    }
  }

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

要了解更多关于构建自定义传输的信息，请参阅[传输 API 文档](/docs/ai-sdk-ui/transport)。

### 直接 Agent 传输

若希望直接与 Agent 通信而不经过 HTTP，可使用 `DirectChatTransport`。适用于：

- 服务端渲染场景
- 无网络测试
- 单进程应用

```tsx filename="app/page.tsx"
import { useChat } from '@ai-sdk/react';
import { DirectChatTransport, ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
});

export default function Chat() {
  const { messages, sendMessage, status } = useChat({
    transport: new DirectChatTransport({ agent }),
  });

  return (
    <>
      {messages.map(message => (
        <div key={message.id}>
          {message.role === 'user' ? 'User: ' : 'AI: '}
          {message.parts.map((part, index) =>
            part.type === 'text' ? <span key={index}>{part.text}</span> : null,
          )}
        </div>
      ))}

      <button
        onClick={() => sendMessage({ text: 'Hello!' })}
        disabled={status !== 'ready'}
      >
        Send
      </button>
    </>
  );
}
```

`DirectChatTransport` 直接调用 agent 的 `stream()` 方法，将 UI 消息转换为模型消息，并将响应流式返回为 UI 消息块。

更多详情请参阅 [DirectChatTransport 参考](/docs/reference/ai-sdk-ui/direct-chat-transport)。

## 控制响应流

使用 `streamText` 时，你可以控制错误消息和用量信息如何返回给客户端。

### 错误消息

默认情况下，出于安全原因会屏蔽错误消息。
默认错误消息为「An error occurred.」
你可以通过提供 `getErrorMessage` 函数转发错误消息或发送自定义错误消息：

```ts filename="app/api/chat/route.ts" highlight="13-27"
import { convertToModelMessages, streamText, UIMessage } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse({
    onError: error => {
      if (error == null) {
        return 'unknown error';
      }

      if (typeof error === 'string') {
        return error;
      }

      if (error instanceof Error) {
        return error.message;
      }

      return JSON.stringify(error);
    },
  });
}
```

### 用量信息

使用[消息元数据](/docs/ai-sdk-ui/message-metadata)跟踪 token 消耗和资源用量：

1. 定义包含用量字段的自定义元数据类型（可选，用于类型安全）
2. 在响应中使用 `messageMetadata` 附加用量数据
3. 在 UI 组件中显示用量指标

用量数据作为元数据附加到消息，在模型完成响应生成后即可使用。

```ts
import { openai } from '@ai-sdk/openai';
import {
  convertToModelMessages,
  streamText,
  UIMessage,
  type LanguageModelUsage,
} from 'ai';
__PROVIDER_IMPORT__;

// Create a new metadata type (optional for type-safety)
type MyMetadata = {
  totalUsage: LanguageModelUsage;
};

// Create a new custom message type with your own metadata
export type MyUIMessage = UIMessage<MyMetadata>;

export async function POST(req: Request) {
  const { messages }: { messages: MyUIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse({
    originalMessages: messages,
    messageMetadata: ({ part }) => {
      // Send total usage when generation is finished
      if (part.type === 'finish') {
        return { totalUsage: part.totalUsage };
      }
    },
  });
}
```

然后，在客户端可以访问消息级元数据。

```tsx
'use client';

import { useChat } from '@ai-sdk/react';
import type { MyUIMessage } from './api/chat/route';
import { DefaultChatTransport } from 'ai';

export default function Chat() {
  // Use custom message type defined on the server (optional for type-safety)
  const { messages } = useChat<MyUIMessage>({
    transport: new DefaultChatTransport({
      api: '/api/chat',
    }),
  });

  return (
    <div className="flex flex-col w-full max-w-md py-24 mx-auto stretch">
      {messages.map(m => (
        <div key={m.id} className="whitespace-pre-wrap">
          {m.role === 'user' ? 'User: ' : 'AI: '}
          {m.parts.map(part => {
            if (part.type === 'text') {
              return part.text;
            }
          })}
          {/* Render usage via metadata */}
          {m.metadata?.totalUsage && (
            <div>Total usage: {m.metadata?.totalUsage.totalTokens} tokens</div>
          )}
        </div>
      ))}
    </div>
  );
}
```

你也可以从 `useChat` 的 `onFinish` 回调访问元数据：

```tsx
'use client';

import { useChat } from '@ai-sdk/react';
import type { MyUIMessage } from './api/chat/route';
import { DefaultChatTransport } from 'ai';

export default function Chat() {
  // Use custom message type defined on the server (optional for type-safety)
  const { messages } = useChat<MyUIMessage>({
    transport: new DefaultChatTransport({
      api: '/api/chat',
    }),
    onFinish: ({ message }) => {
      // Access message metadata via onFinish callback
      console.log(message.metadata?.totalUsage);
    },
  });
}
```

### 文本流

通过将 `streamProtocol` 选项设为 `text`，`useChat` 可以处理纯文本流：

```tsx filename="app/page.tsx" highlight="7"
'use client';

import { useChat } from '@ai-sdk/react';
import { TextStreamChatTransport } from 'ai';

export default function Chat() {
  const { messages } = useChat({
    transport: new TextStreamChatTransport({
      api: '/api/chat',
    }),
  });

  return <>...</>;
}
```

此配置也适用于流式传输纯文本的其他后端服务器。
更多信息请参阅[流协议指南](/docs/ai-sdk-ui/stream-protocol)。

<Note>
  使用 `TextStreamChatTransport` 时，工具调用、用量信息和完成原因不可用。
</Note>

## 推理

某些模型（如 DeepSeek `deepseek-r1` 和 Anthropic `claude-sonnet-4-5-20250929`）支持推理 token。
这些 token 通常在消息内容之前发送。
你可以使用 `sendReasoning` 选项将它们转发给客户端：

```ts filename="app/api/chat/route.ts" highlight="13"
import { convertToModelMessages, streamText, UIMessage } from 'ai';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: 'deepseek/deepseek-r1',
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse({
    sendReasoning: true,
  });
}
```

在客户端，可以访问消息对象的推理部分。

推理部分具有包含推理内容的 `text` 属性。

```tsx filename="app/page.tsx"
messages.map(message => (
  <div key={message.id}>
    {message.role === 'user' ? 'User: ' : 'AI: '}
    {message.parts.map((part, index) => {
      // text parts:
      if (part.type === 'text') {
        return <div key={index}>{part.text}</div>;
      }

      // reasoning parts:
      if (part.type === 'reasoning') {
        return <pre key={index}>{part.text}</pre>;
      }
    })}
  </div>
));
```

某些模型也可能在推理过程中生成文件（例如图像）。
这些可作为 `reasoning-file` 部分（`ReasoningFileUIPart`）使用，具有与常规文件部分类似的 `mediaType` 和 `url` 属性。

## 来源

某些提供商（如 [Perplexity](/providers/ai-sdk-providers/perplexity#sources) 和 [Google](/providers/ai-sdk-providers/google#sources)）在响应中包含来源。

目前来源仅限于支撑响应的网页。
你可以使用 `sendSources` 选项将它们转发给客户端：

```ts filename="app/api/chat/route.ts" highlight="13"
import { convertToModelMessages, streamText, UIMessage } from 'ai';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: 'perplexity/sonar-pro',
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse({
    sendSources: true,
  });
}
```

在客户端，可以访问消息对象的来源部分。
来源有两种类型：网页的 `source-url` 和文档的 `source-document`。
以下是渲染两种来源的示例：

```tsx filename="app/page.tsx"
messages.map(message => (
  <div key={message.id}>
    {message.role === 'user' ? 'User: ' : 'AI: '}

    {/* Render URL sources */}
    {message.parts
      .filter(part => part.type === 'source-url')
      .map(part => (
        <span key={`source-${part.id}`}>
          [
          <a href={part.url} target="_blank">
            {part.title ?? new URL(part.url).hostname}
          </a>
          ]
        </span>
      ))}

    {/* Render document sources */}
    {message.parts
      .filter(part => part.type === 'source-document')
      .map(part => (
        <span key={`source-${part.id}`}>
          [<span>{part.title ?? `Document ${part.id}`}</span>]
        </span>
      ))}
  </div>
));
```

## 图像生成

某些模型（如 Google `gemini-2.5-flash-image`）支持图像生成。
生成图像时，它们会作为文件暴露给客户端。
在客户端，可以访问消息对象的文件部分并将其渲染为图像。

```tsx filename="app/page.tsx"
messages.map(message => (
  <div key={message.id}>
    {message.role === 'user' ? 'User: ' : 'AI: '}
    {message.parts.map((part, index) => {
      if (part.type === 'text') {
        return <div key={index}>{part.text}</div>;
      } else if (part.type === 'file' && part.mediaType.startsWith('image/')) {
        return <img key={index} src={part.url} alt="Generated image" />;
      }
    })}
  </div>
));
```

## 附件

`useChat` hook 支持随消息发送文件附件并在客户端渲染。这对于构建需要向 AI 提供商发送图像、文件或其他媒体内容的应用很有用。

有两种方式随消息发送文件：使用文件输入的 `FileList` 对象，或使用文件对象数组。

### FileList

使用 `FileList`，你可以通过文件输入元素随消息发送多个文件作为附件。`useChat` hook 会自动将它们转换为 data URL 并发送给 AI 提供商。

<Note>
  目前，只有 `image/*` 和 `text/*` 内容类型会自动转换为[多模态内容部分](/docs/foundations/prompts#multi-modal-messages)。其他内容类型需要手动处理。
</Note>

```tsx filename="app/page.tsx"
'use client';

import { useChat } from '@ai-sdk/react';
import { useRef, useState } from 'react';

export default function Page() {
  const { messages, sendMessage, status } = useChat();

  const [input, setInput] = useState('');
  const [files, setFiles] = useState<FileList | undefined>(undefined);
  const fileInputRef = useRef<HTMLInputElement>(null);

  return (
    <div>
      <div>
        {messages.map(message => (
          <div key={message.id}>
            <div>{`${message.role}: `}</div>

            <div>
              {message.parts.map((part, index) => {
                if (part.type === 'text') {
                  return <span key={index}>{part.text}</span>;
                }

                if (
                  part.type === 'file' &&
                  part.mediaType?.startsWith('image/')
                ) {
                  return <img key={index} src={part.url} alt={part.filename} />;
                }

                return null;
              })}
            </div>
          </div>
        ))}
      </div>

      <form
        onSubmit={event => {
          event.preventDefault();
          if (input.trim()) {
            sendMessage({
              text: input,
              files,
            });
            setInput('');
            setFiles(undefined);

            if (fileInputRef.current) {
              fileInputRef.current.value = '';
            }
          }
        }}
      >
        <input
          type="file"
          onChange={event => {
            if (event.target.files) {
              setFiles(event.target.files);
            }
          }}
          multiple
          ref={fileInputRef}
        />
        <input
          value={input}
          placeholder="Send message..."
          onChange={e => setInput(e.target.value)}
          disabled={status !== 'ready'}
        />
      </form>
    </div>
  );
}
```

### 文件对象

你也可以将文件作为对象随消息发送。这对于发送预上传文件或 data URL 很有用。

```tsx filename="app/page.tsx"
'use client';

import { useChat } from '@ai-sdk/react';
import { useState } from 'react';
import { FileUIPart } from 'ai';

export default function Page() {
  const { messages, sendMessage, status } = useChat();

  const [input, setInput] = useState('');
  const [files] = useState<FileUIPart[]>([
    {
      type: 'file',
      filename: 'earth.png',
      mediaType: 'image/png',
      url: 'https://example.com/earth.png',
    },
    {
      type: 'file',
      filename: 'moon.png',
      mediaType: 'image/png',
      url: 'data:image/png;base64,iVBORw0KGgo...',
    },
  ]);

  return (
    <div>
      <div>
        {messages.map(message => (
          <div key={message.id}>
            <div>{`${message.role}: `}</div>

            <div>
              {message.parts.map((part, index) => {
                if (part.type === 'text') {
                  return <span key={index}>{part.text}</span>;
                }

                if (
                  part.type === 'file' &&
                  part.mediaType?.startsWith('image/')
                ) {
                  return <img key={index} src={part.url} alt={part.filename} />;
                }

                return null;
              })}
            </div>
          </div>
        ))}
      </div>

      <form
        onSubmit={event => {
          event.preventDefault();
          if (input.trim()) {
            sendMessage({
              text: input,
              files,
            });
            setInput('');
          }
        }}
      >
        <input
          value={input}
          placeholder="Send message..."
          onChange={e => setInput(e.target.value)}
          disabled={status !== 'ready'}
        />
      </form>
    </div>
  );
}
```

作为模型推理一部分生成的文件可作为 `reasoning-file` 部分（`ReasoningFileUIPart`）使用，具有相同的 `mediaType` 和 `url` 属性。

## 工具的类型推断

在 TypeScript 中使用工具时，AI SDK UI 提供类型推断辅助函数，以确保工具输入和输出的类型安全。

### InferUITool

`InferUITool` 类型辅助函数推断单个工具在 UI 消息中使用的输入和输出类型：

```tsx
import { InferUITool } from 'ai';
import { z } from 'zod';

const weatherTool = {
  description: 'Get the current weather',
  inputSchema: z.object({
    location: z.string().describe('The city and state'),
  }),
  execute: async ({ location }) => {
    return `The weather in ${location} is sunny.`;
  },
};

// Infer the types from the tool
type WeatherUITool = InferUITool<typeof weatherTool>;
// This creates a type with:
// {
//   input: { location: string };
//   output: string;
// }
```

### InferUITools

`InferUITools` 类型辅助函数推断 `ToolSet` 的输入和输出类型：

```tsx
import { InferUITools, ToolSet } from 'ai';
import { z } from 'zod';

const tools = {
  weather: {
    description: 'Get the current weather',
    inputSchema: z.object({
      location: z.string().describe('The city and state'),
    }),
    execute: async ({ location }) => {
      return `The weather in ${location} is sunny.`;
    },
  },
  calculator: {
    description: 'Perform basic arithmetic',
    inputSchema: z.object({
      operation: z.enum(['add', 'subtract', 'multiply', 'divide']),
      a: z.number(),
      b: z.number(),
    }),
    execute: async ({ operation, a, b }) => {
      switch (operation) {
        case 'add':
          return a + b;
        case 'subtract':
          return a - b;
        case 'multiply':
          return a * b;
        case 'divide':
          return a / b;
      }
    },
  },
} satisfies ToolSet;

// Infer the types from the tool set
type MyUITools = InferUITools<typeof tools>;
// This creates a type with:
// {
//   weather: { input: { location: string }; output: string };
//   calculator: { input: { operation: 'add' | 'subtract' | 'multiply' | 'divide'; a: number; b: number }; output: number };
// }
```

### Using Inferred Types

你可以使用这些推断类型创建自定义 UIMessage 类型，并将其传递给各种 AI SDK UI 函数：

```tsx
import { InferUITools, UIMessage, UIDataTypes } from 'ai';

type MyUITools = InferUITools<typeof tools>;
type MyUIMessage = UIMessage<never, UIDataTypes, MyUITools>;
```

Pass the custom type to `useChat` or `createUIMessageStream`:

```tsx
import { useChat } from '@ai-sdk/react';
import { createUIMessageStream } from 'ai';
import type { MyUIMessage } from './types';

// With useChat
const { messages } = useChat<MyUIMessage>();

// With createUIMessageStream
const stream = createUIMessageStream<MyUIMessage>(/* ... */);
```

这为客户端和服务端的工具输入和输出提供完整的类型安全。
