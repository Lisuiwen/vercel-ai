# 文本补全

`useCompletion` hook 允许你在应用中创建用于处理文本补全的用户界面。它支持从 AI 提供商流式传输文本补全、管理聊天输入状态，并在收到新消息时自动更新 UI。

<Note>
  `useCompletion` hook 现属于 `@ai-sdk/react` 包。
</Note>

本指南将介绍如何在应用中使用 `useCompletion` hook 生成文本补全并实时流式传输给用户。

## 示例

```tsx filename='app/page.tsx'
'use client';

import { useCompletion } from '@ai-sdk/react';

export default function Page() {
  const { completion, input, handleInputChange, handleSubmit } = useCompletion({
    api: '/api/completion',
  });

  return (
    <form onSubmit={handleSubmit}>
      <input
        name="prompt"
        value={input}
        onChange={handleInputChange}
        id="input"
      />
      <button type="submit">Submit</button>
      <div>{completion}</div>
    </form>
  );
}
```

```ts filename='app/api/completion/route.ts'
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

// Allow streaming responses up to 30 seconds
export const maxDuration = 30;

export async function POST(req: Request) {
  const { prompt }: { prompt: string } = await req.json();

  const result = streamText({
    model: __MODEL__,
    prompt,
  });

  return result.toUIMessageStreamResponse();
}
```

在 `Page` 组件中，每当用户提交消息时，`useCompletion` hook 会向你的 AI 提供商端点发起请求。补全结果随后会实时流式返回并显示在 UI 中。

这提供了流畅的文本补全体验：用户无需等待完整响应即可看到 AI 回复。

## 自定义 UI

`useCompletion` 还提供通过代码管理提示、显示加载和错误状态，以及在不依赖用户交互的情况下更新消息的方式。

### 加载与错误状态

要在聊天机器人处理用户消息时显示加载动画，可使用 `useCompletion` hook 返回的 `isLoading` 状态：

```tsx
const { isLoading, ... } = useCompletion()

return(
  <>
    {isLoading ? <Spinner /> : null}
  </>
)
```

同样，`error` 状态反映 fetch 请求期间抛出的错误对象。可用于显示错误消息或 toast 通知：

```tsx
const { error, ... } = useCompletion()

useEffect(() => {
  if (error) {
    toast.error(error.message)
  }
}, [error])

// Or display the error message in the UI:
return (
  <>
    {error ? <div>{error.message}</div> : null}
  </>
)
```

### 受控输入

在初始示例中，我们使用 `handleSubmit` 和 `handleInputChange` 回调管理输入变更和表单提交。这些对常见场景很方便，但你也可以使用非受控 API 处理更高级的场景，例如表单验证或自定义组件。

以下示例演示如何将 `setInput` 等更细粒度的 API 与自定义输入和提交按钮组件配合使用：

```tsx
const { input, setInput } = useCompletion();

return (
  <>
    <MyCustomInput value={input} onChange={value => setInput(value)} />
  </>
);
```

### 取消

在 AI 提供商仍在流式返回响应时中止消息也是常见需求。可调用 `useCompletion` hook 返回的 `stop` 函数实现：

```tsx
const { stop, isLoading, ... } = useCompletion()

return (
  <>
    <button onClick={stop} disabled={!isLoading}>Stop</button>
  </>
)
```

当用户点击「Stop」按钮时，fetch 请求将被中止。这避免消耗不必要的资源并改善应用体验。

### 节流 UI 更新

<Note>此功能目前仅适用于 React。</Note>

默认情况下，`useCompletion` hook 在每次收到新块时都会触发渲染。
你可以通过 `experimental_throttle` 选项对 UI 更新进行节流：

```tsx filename="page.tsx" highlight="2-3"
const { completion, ... } = useCompletion({
  // Throttle the completion and data updates to 50ms:
  experimental_throttle: 50
})
```

## 事件回调

`useCompletion` 还提供可选的事件回调，用于处理聊天机器人生命周期的不同阶段。这些回调可用于触发额外操作，例如日志、分析或自定义 UI 更新。

```tsx
const { ... } = useCompletion({
  onFinish: (prompt: string, completion: string) => {
    console.log('Finished streaming completion:', completion)
  },
  onError: (error: Error) => {
    console.error('An error occurred:', error)
  },
})
```

## 配置请求选项

默认情况下，`useCompletion` hook 会向 `/api/completion` 端点发送 HTTP POST 请求，并将 prompt 作为请求体的一部分。你可以向 `useCompletion` hook 传递额外选项来自定义请求：

```tsx
const { messages, input, handleInputChange, handleSubmit } = useCompletion({
  api: '/api/custom-completion',
  headers: {
    Authorization: 'your_token',
  },
  body: {
    user_id: '123',
  },
  credentials: 'same-origin',
});
```

在此示例中，`useCompletion` hook 向 `/api/completion` 端点发送 POST 请求，并附带指定的 headers、额外 body 字段以及该 fetch 请求的 credentials。在服务端，你可以使用这些额外信息处理请求。
