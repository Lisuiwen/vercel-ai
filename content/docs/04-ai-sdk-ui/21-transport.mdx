# 传输

`useChat` 传输系统可精细控制消息如何发送到 API 端点以及如何处理响应。这对于 WebSocket、自定义身份验证模式或专用后端集成等替代通信协议特别有用。

## 默认传输

默认情况下，`useChat` 使用 HTTP POST 请求将消息发送到 `/api/chat`：

```tsx
import { useChat } from '@ai-sdk/react';

// Uses default HTTP transport
const { messages, sendMessage } = useChat();
```

这等价于：

```tsx
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/chat',
  }),
});
```

## 自定义传输配置

使用自定义选项配置默认传输：

```tsx
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';

const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/custom-chat',
    headers: {
      Authorization: 'Bearer your-token',
      'X-API-Version': '2024-01',
    },
    credentials: 'include',
  }),
});
```

### 动态配置

你也可以提供返回配置值的函数。这对于需要刷新的身份验证令牌或依赖运行时条件的配置很有用：

```tsx
const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/chat',
    headers: () => ({
      Authorization: `Bearer ${getAuthToken()}`,
      'X-User-ID': getCurrentUserId(),
    }),
    body: () => ({
      sessionId: getCurrentSessionId(),
      preferences: getUserPreferences(),
    }),
    credentials: () => 'include',
  }),
});
```

### 请求转换

在发送到 API 之前转换请求：

```tsx
const { messages, sendMessage } = useChat({
  transport: new DefaultChatTransport({
    api: '/api/chat',
    prepareSendMessagesRequest: ({ id, messages, trigger, messageId }) => {
      return {
        headers: {
          'X-Session-ID': id,
        },
        body: {
          messages: messages.slice(-10), // Only send last 10 messages
          trigger,
          messageId,
        },
      };
    },
  }),
});
```

## 直接 Agent 传输

若希望直接与 [Agent](/docs/reference/ai-sdk-core/agent) 通信而不经过 HTTP，可使用 `DirectChatTransport`。该传输在同一进程内直接调用 agent 的 `stream()` 方法。

适用于：

- **服务端渲染**：在服务器上运行 agent 而无需 API 端点
- **测试**：无需网络请求即可测试聊天功能
- **单进程应用**：客户端与 agent 在同一进程运行的桌面或 CLI 应用

```tsx
import { useChat } from '@ai-sdk/react';
import { DirectChatTransport, ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
  tools: {
    weather: weatherTool,
  },
});

const { messages, sendMessage } = useChat({
  transport: new DirectChatTransport({ agent }),
});
```

### 工作原理

与发送 HTTP 请求的 `DefaultChatTransport` 不同：

1. `DirectChatTransport` 验证传入的 UI 消息
2. 使用 `convertToModelMessages` 将其转换为模型消息
3. 直接调用 agent 的 `stream()` 方法
4. 通过 `toUIMessageStream()` 将结果作为 UI 消息流返回

### 配置选项

你可以传递额外选项以自定义流输出：

```tsx
const transport = new DirectChatTransport({
  agent,
  // Pass options to the agent
  options: { customOption: 'value' },
  // Configure what's sent to the client
  sendReasoning: true,
  sendSources: true,
});
```

<Note>
  `DirectChatTransport` 不支持流重新连接，因为没有持久的服务端流。`reconnectToStream()` 方法始终返回 `null`。
</Note>

完整 API 详情请参阅 [DirectChatTransport 参考](/docs/reference/ai-sdk-ui/direct-chat-transport)。

## Workflow 传输

对于基于 [Workflow SDK](/docs/agents/workflow-agent) 构建的聊天应用，`@ai-sdk/workflow` 的 `WorkflowChatTransport` 提供自动流重新连接。它处理工作流函数在流中途超时的常见场景 — 传输会检测缺失的 `finish` 事件并重新连接以从中断处恢复。

```tsx
import { useChat } from '@ai-sdk/react';
import { WorkflowChatTransport } from '@ai-sdk/workflow';
import { useMemo } from 'react';

export default function Chat() {
  const transport = useMemo(
    () =>
      new WorkflowChatTransport({
        api: '/api/chat',
        maxConsecutiveErrors: 5,
        initialStartIndex: -50, // On page refresh, fetch last 50 chunks
        onChatEnd: ({ chatId, chunkIndex }) => {
          console.log(`Chat complete: ${chunkIndex} chunks`);
        },
      }),
    [],
  );

  const { messages, sendMessage } = useChat({ transport });

  // ... render chat UI
}
```

主要特性：

- **自动重新连接**：检测中断的流（无 `finish` 事件）并通过 GET 请求 `{api}/{runId}/stream` 重新连接
- **页面刷新恢复**：`initialStartIndex` 使用负值（例如 `-50`）仅获取流尾部，而非重放全部内容
- **可配置重试**：`maxConsecutiveErrors` 控制容忍的连续重连失败次数
- **生命周期回调**：`onChatSendMessage` 和 `onChatEnd` 用于跟踪聊天状态

完整 API 参考请参阅 [`WorkflowChatTransport`](/docs/reference/ai-sdk-workflow/workflow-chat-transport)。服务端端点设置请参阅 [WorkflowAgent 指南](/docs/agents/workflow-agent#resumable-streaming-with-workflowchattransport)。

## 构建自定义传输

要了解如何构建自己的传输，请参阅默认实现的源代码：

- **[DefaultChatTransport](https://github.com/vercel/ai/blob/main/packages/ai/src/ui/default-chat-transport.ts)** — 完整的默认 HTTP 传输实现
- **[HttpChatTransport](https://github.com/vercel/ai/blob/main/packages/ai/src/ui/http-chat-transport.ts)** — 带请求处理的基础 HTTP 传输
- **[ChatTransport Interface](https://github.com/vercel/ai/blob/main/packages/ai/src/ui/chat-transport.ts)** — 你需要实现的传输接口

这些实现准确展示了如何：

- 处理 `sendMessages` 方法
- 处理 UI 消息流
- 转换请求和响应
- 处理错误和连接管理

传输系统让你完全控制聊天应用的通信方式，可与任何后端协议或服务集成。
