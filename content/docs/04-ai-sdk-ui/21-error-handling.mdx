# 错误处理与警告

## 警告

当某些功能可能无法按预期工作时，AI SDK 会显示警告。
这些警告帮助你在问题引发错误之前加以修复。

### 警告出现时机

在以下情况下，警告会显示在浏览器控制台中：

- **不支持的功能**：你使用了 AI 模型不支持的功能或设置（例如某些选项或参数）。
- **兼容性警告**：某功能以兼容模式使用，其行为可能与预期不同或不够理想。
- **其他警告**：AI 模型报告其他类型的问题，例如一般性问题或建议性消息。

### 警告消息

所有警告均以「AI SDK Warning:」开头，便于识别。例如：

```
AI SDK Warning: The feature "temperature" is not supported by this model
```

### 关闭警告

默认情况下，警告会显示在控制台中。你可以控制此行为：

#### 关闭所有警告

设置全局变量以完全关闭警告：

```ts
globalThis.AI_SDK_LOG_WARNINGS = false;
```

#### 自定义警告处理器

你也可以提供自己的函数来处理警告。
该函数接收 provider id、model id 和警告列表。

```ts
globalThis.AI_SDK_LOG_WARNINGS = ({ warnings, provider, model }) => {
  // Handle warnings your own way
};
```

## 错误处理

### 错误辅助对象

每个 AI SDK UI hook 还会返回一个 [error](/docs/reference/ai-sdk-ui/use-chat#error) 对象，可用于在 UI 中渲染错误。
你可以使用该错误对象显示错误消息、禁用提交按钮或显示重试按钮。

<Note>
  我们建议向用户显示通用错误消息，例如「出了点问题。」这是避免泄露服务器信息的好习惯。
</Note>

```tsx file="app/page.tsx" highlight="7,18-25,31"
'use client';

import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const [input, setInput] = useState('');
  const { messages, sendMessage, error, regenerate } = useChat();

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    sendMessage({ text: input });
    setInput('');
  };

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.role}:{' '}
          {m.parts
            .filter(part => part.type === 'text')
            .map(part => part.text)
            .join('')}
        </div>
      ))}

      {error && (
        <>
          <div>An error occurred.</div>
          <button type="button" onClick={() => regenerate()}>
            Retry
          </button>
        </>
      )}

      <form onSubmit={handleSubmit}>
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          disabled={error != null}
        />
      </form>
    </div>
  );
}
```

#### 替代方案：替换最后一条消息

你也可以编写自定义提交处理程序，在存在错误时替换最后一条消息。

```tsx file="app/page.tsx" highlight="17-23,35"
'use client';

import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const [input, setInput] = useState('');
  const { sendMessage, error, messages, setMessages } = useChat();

  function customSubmit(event: React.FormEvent<HTMLFormElement>) {
    event.preventDefault();

    if (error != null) {
      setMessages(messages.slice(0, -1)); // remove last message
    }

    sendMessage({ text: input });
    setInput('');
  }

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          {m.role}:{' '}
          {m.parts
            .filter(part => part.type === 'text')
            .map(part => part.text)
            .join('')}
        </div>
      ))}

      {error && <div>An error occurred.</div>}

      <form onSubmit={customSubmit}>
        <input value={input} onChange={e => setInput(e.target.value)} />
      </form>
    </div>
  );
}
```

### 错误处理回调

可以通过向 [`useChat`](/docs/reference/ai-sdk-ui/use-chat) 或 [`useCompletion`](/docs/reference/ai-sdk-ui/use-completion) hook 传递 [`onError`](/docs/reference/ai-sdk-ui/use-chat#on-error) 回调函数来处理错误。
该回调函数接收一个 error 对象作为参数。

```tsx file="app/page.tsx" highlight="6-9"
import { useChat } from '@ai-sdk/react';

export default function Page() {
  const {
    /* ... */
  } = useChat({
    // handle error:
    onError: error => {
      console.error(error);
    },
  });
}
```

### 注入错误用于测试

你可能希望创建错误用于测试。
可以在路由处理程序中抛出错误来轻松实现：

```ts file="app/api/chat/route.ts"
export async function POST(req: Request) {
  throw new Error('This is a test error');
}
```
