# 在 Vercel 部署时出现超时

## 问题

在本地开发环境中，使用 AI SDK 流式传输正常。
但部署到 Vercel 后，较长响应在界面中被截断，Vercel 日志中出现超时，或看到错误：`Uncaught (in promise) Error: Connection closed`。

## 解决方案

借助 Vercel 的 [Fluid Compute](https://vercel.com/docs/fluid-compute)，所有套餐的默认函数执行时长现为 **5 分钟（300 秒）**，对多数流式应用已足够。

若需要更长时间运行的流程，可提高 `maxDuration` 设置：

### Next.js（App Router）

在对应路由文件或调用 Server Action 的页面中加入：

```tsx
export const maxDuration = 600;
```

<Note>
  将 `maxDuration` 设为超过 300 秒需要 Pro 或 Enterprise 套餐。
</Note>

### 其他框架

可在 `vercel.json` 中设置超时：

```json
{
  "functions": {
    "api/chat/route.ts": {
      "maxDuration": 600
    }
  }
}
```

<Note>
  将 `maxDuration` 设为超过 300 秒需要 Pro 或 Enterprise 套餐。
</Note>

### 最大时长上限

可设置的最大时长取决于 Vercel 套餐：

- **Hobby**：最长 300 秒（5 分钟）
- **Pro**：最长 800 秒（约 13 分钟）
- **Enterprise**：最长 800 秒（约 13 分钟）
