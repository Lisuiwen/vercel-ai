# AI SDK 5 配合 Zod 时的 TypeScript 性能问题

## 问题

使用 AI SDK 5 与 Zod 时，可能出现：

- TypeScript 语言服务崩溃或卡住
- 导入 AI SDK 的文件类型检查极慢
- 报错类似「Type instantiation is excessively deep and possibly infinite」
- 编辑 AI SDK 相关代码时 IDE 无响应

## 背景

AI SDK 5 对 Zod 版本有特定兼容性要求。使用常规导入路径（`import { z } from 'zod'`）时，TypeScript 的类型推断可能变得过于复杂，导致性能下降或崩溃。

## 解决方案

### 将 Zod 升级到 4.1.8 或更高

主要做法是将 Zod 升级到 4.1.8 及以上，其中包含对该模块解析问题的修复：

```bash
pnpm add zod@^4.1.8
```

该版本修复了在不同模块解析设置下 TypeScript 重复加载同一份 Zod 声明、从而引发昂贵结构比较的问题。

### 备选：调整 TypeScript 配置

若暂时无法升级 Zod，可将 `tsconfig.json` 中的 `moduleResolution` 设为 `"nodenext"`：

```json
{
  "compilerOptions": {
    "moduleResolution": "nodenext"
    // ... other options
  }
}
```

这样在保留标准 Zod 导入的情况下也能缓解 TypeScript 性能问题。
