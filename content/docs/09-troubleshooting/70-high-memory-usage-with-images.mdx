# 处理大量图片时内存占用高

## 问题

在循环或批处理中对很多图片使用 `generateText` 或 `streamText` 时，可能观察到：

- 内存持续增长不回落
- 应用最终内存耗尽
- 即使垃圾回收后内存也未释放

在使用 `experimental_download` 处理 URL 图片，或在提示词中发送 base64 图片时尤为明显。

## 背景

默认情况下，AI SDK 会在步骤结果中包含完整请求消息以及请求/响应体。处理图片时，请求消息与请求体可能含 base64 数据，体积很大（单张图片 base64 后可达 1MB+）。若处理大量图片并长期持有结果引用，这些数据会在内存中累积。

例如处理 100 张各约 500KB 的图片，请求体数据在内存中可达约 50MB 以上。

## 解决方案

使用 `include` 选项，关闭在步骤结果中包含请求消息和/或请求与响应体：

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: openai('gpt-4o'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe this image' },
        { type: 'file', mediaType: 'image', data: imageUrl },
      ],
    },
  ],
  // Request and response bodies are excluded by default.
  include: {
    requestBody: false,
    responseBody: false,
  },
});
```

### 选项说明

`include` 可包含：

- `requestBody`：设为 `true` 时把请求体纳入步骤结果。请求体常存放 base64 图片。默认：`false`。`generateText` 与 `streamText` 均支持。
- `requestMessages`：设为 `true` 时把请求消息纳入步骤结果。消息内容可能含大图与文件。默认：`false`。`generateText` 与 `streamText` 均支持。
- `rawChunks`：设为 `true` 时包含原始 provider 分片。默认：`false`。仅 `streamText` 支持。
- `responseBody`：设为 `true` 时把响应体纳入步骤结果。默认：`false`。仅 `generateText` 支持。

### 何时使用

- **批量处理图片**：在循环中处理大量图片时
- **长时间运行的智能体**：生命周期内可能处理很多图片时
- **内存受限环境**：在内存紧张的环境中运行时

### 权衡

关闭 body 包含后：

- 无法访问 `result.request.body` 或 `result.response.body`
- 除非启用 `requestMessages`，否则无法访问 `result.request.messages`
- 调试变难，因为无法检查原始请求/响应
- 若需要 body 做日志或调试，可在下一轮迭代前先提取所需字段

## 示例：处理多张图片

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const imageUrls = [
  /* array of image URLs */
];
const results = [];

for (const imageUrl of imageUrls) {
  const result = await generateText({
    model: openai('gpt-4o'),
    messages: [
      {
        role: 'user',
        content: [
          { type: 'text', text: 'Describe this image' },
          { type: 'file', mediaType: 'image', data: imageUrl },
        ],
      },
    ],
    include: {
      requestBody: false,
    },
  });

  // Only store the text result, not the full result object
  results.push(result.text);
}
```
