# 使用 OpenAI 时对象生成失败

## 问题

配合 OpenAI 使用结构化输出生成时，可能出现 `NoObjectGeneratedError`，`finish reason` 为 `content-filter`。常见原因是 Zod 模式中包含 OpenAI 结构化输出无法处理的类型。

```typescript
// Problematic code - incompatible schema types
import { generateText, Output } from 'ai';
import { openai } from '@ai-sdk/openai';
import { z } from 'zod';

const result = await generateText({
  model: openai('gpt-4o-2024-08-06'),
  output: Output.object({
    schema: z.object({
      name: z.string().nullish(), // ❌ .nullish() is not supported
      email: z.string().optional(), // ❌ .optional() is not supported
      age: z.number().nullable(), // ✅ .nullable() is supported
    }),
  }),
  prompt: 'Generate a user profile',
});

// Error: NoObjectGeneratedError: No object generated.
// Finish reason: content-filter
```

## 背景

OpenAI 的结构化输出生成底层使用 JSON Schema，对模式兼容性有明确要求。`.nullish()` 与 `.optional()` 生成的 JSON Schema 形态与 OpenAI 实现不兼容，可能导致模型拒绝模式并以 `content-filter` 作为结束原因。

## 解决方案

配合 OpenAI 模型做结构化输出时，将 `.nullish()` 与 `.optional()` 改为 `.nullable()`。

```typescript
import { generateText, Output } from 'ai';
import { openai } from '@ai-sdk/openai';
import { z } from 'zod';

// Correct approach - use .nullable()
const result = await generateText({
  model: openai('gpt-4o-2024-08-06'),
  output: Output.object({
    schema: z.object({
      name: z.string().nullable(), // ✅ Use .nullable() instead of .nullish()
      email: z.string().nullable(), // ✅ Use .nullable() instead of .optional()
      age: z.number().nullable(),
    }),
  }),
  prompt: 'Generate a user profile',
});

console.log(result.output);
// { name: "John Doe", email: "john@example.com", age: 30 }
// or { name: null, email: null, age: 25 }
```

### 模式类型对照

| Zod 类型      | 兼容 | JSON Schema 行为                                   |
| ------------- | -- | ------------------------------------------------ |
| `.nullable()` | ✅  | 允许 `null` 或指定类型                               |
| `.optional()` | ❌  | 字段可省略（不支持）                                  |
| `.nullish()`  | ❌  | 允许 `null`、`undefined` 或省略（不支持）                 |
