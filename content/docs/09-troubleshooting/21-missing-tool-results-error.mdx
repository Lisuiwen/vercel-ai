# 缺少工具结果错误

## 问题

可能遇到 `AI_MissingToolResultsError`，提示类似：

> Tool results are missing for tool calls: ...

## 原因

当你在有上一轮尚未解决的待处理工具调用时，又向大语言模型（LLM）发送新消息，就会触发该错误。

AI SDK 核心逻辑会校验对话历史中所有 `tool-call` 片段在继续前都已解决。「已解决」通常指：

1. 工具已执行并向历史加入 `tool-result`。
2. 或工具调用触发了 `tool-approval-response`（若使用工具审批流程）。

若存在没有对应结果或审批响应的工具调用，会抛出该错误，避免将无效历史发给模型。

## 解决方案

确保对话历史中的每次工具调用都得到妥善处理。

### 1. 提供工具结果

对常规工具调用，确保写入工具执行输出。

```typescript
const messages = [
  { role: 'user', content: 'What is the weather in NY?' },
  {
    role: 'assistant',
    content: [
      {
        type: 'tool-call',
        toolCallId: 'call_123',
        toolName: 'getWeather',
        args: { location: 'New York' },
      },
    ],
  },
  // You MUST include this tool message with the result:
  {
    role: 'tool',
    content: [
      {
        type: 'tool-result',
        toolCallId: 'call_123',
        toolName: 'getWeather',
        result: 'Sunny, 25°C',
      },
    ],
  },
  // Now you can add a new user message
  { role: 'user', content: 'And in London?' },
];
```

### 2. 处理工具审批

若使用工具审批流程，确保包含 `tool-approval-response`。

```typescript
const messages = [
  // ... assistant requests tool execution (needs approval)
  {
    role: 'tool',
    content: [
      {
        type: 'tool-approval-response',
        approvalId: 'approval_123',
        approved: true, // or false
      },
    ],
  },
];
```
