# onToolCall 类型错误

在 TypeScript 中使用 `onToolCall` 回调时，若直接把工具属性传给 `addToolOutput`，可能遇到类型错误。

## 问题

同时存在静态与动态工具时，TypeScript 无法自动缩窄 `toolCall.toolName` 的类型，导致报错：

```tsx
// ❌ This causes a TypeScript error
const { messages, sendMessage, addToolOutput } = useChat({
  async onToolCall({ toolCall }) {
    addToolOutput({
      tool: toolCall.toolName, // Type 'string' is not assignable to type '"yourTool" | "yourOtherTool"'
      toolCallId: toolCall.toolCallId,
      output: someOutput,
    });
  },
});
```

原因包括：

- 静态工具的名称是具体的字面量类型（例如 `"getWeatherInformation"`）
- 动态工具的 `toolName` 为泛化的 `string`
- TypeScript 无法保证 `toolCall.toolName` 一定匹配你的工具名联合类型

## 解决方案

先判断是否为动态工具，以便正确缩窄类型：

```tsx
// ✅ Correct approach with type narrowing
const { messages, sendMessage, addToolOutput } = useChat({
  async onToolCall({ toolCall }) {
    // Check if it's a dynamic tool first
    if (toolCall.dynamic) {
      return;
    }

    // Now TypeScript knows this is a static tool with the correct type
    addToolOutput({
      tool: toolCall.toolName, // No type error!
      toolCallId: toolCall.toolCallId,
      output: someOutput,
    });
  },
});
```

<Note>
  若仍在使用已弃用的 `addToolResult`，同样适用上述做法。建议迁移到 `addToolOutput`，与最新 API 保持一致。
</Note>
