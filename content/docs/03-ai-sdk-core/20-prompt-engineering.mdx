# 提示工程

## 技巧

### 面向工具的提示

创建包含工具的提示时，随着工具数量与复杂度增加，获得良好结果可能变得困难。

以下技巧有助于获得最佳效果：

1. 使用擅长工具调用的模型，例如 `gpt-5` 或 `gpt-4.1`。较弱模型往往难以稳定、无误地调用工具。
1. 保持工具数量较少，例如不超过 5 个。
1. 降低工具参数的复杂度。包含大量嵌套与可选元素、联合类型等的复杂 Zod schema 可能让模型难以处理。
1. 为工具、参数、参数属性等使用语义清晰的名称。向模型传递的信息越多，它就越能理解你的意图。
1. 在 Zod schema 属性上使用 `.describe("...")`，向模型说明各属性的用途。
1. 当工具输出对模型可能不清晰，且工具之间存在依赖时，可在工具的 `description` 字段中说明工具执行的输出信息。
1. 可在提示中包含工具调用的示例输入/输出，帮助模型理解如何使用工具。请注意工具使用 JSON 对象，因此示例应使用 JSON。

总体而言，目标应是以清晰的方式向模型提供其所需的全部信息。

### 工具与结构化数据 Schema

从 Zod schema 到 LLM 输入（通常为 JSON schema）的映射并不总是一一对应。

#### Zod 日期

Zod 期望 JavaScript Date 对象，但模型返回的日期为字符串。
可使用 `z.string().datetime()` 或 `z.string().date()` 指定并校验日期格式，
然后使用 Zod transformer 将字符串转换为 Date 对象。

```ts highlight="8-11"
const result = await generateText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      events: z.array(
        z.object({
          event: z.string(),
          date: z
            .string()
            .date()
            .transform(value => new Date(value)),
        }),
      ),
    }),
  }),
  prompt: 'List 5 important events from the year 2000.',
});
```

#### 可选参数

使用带有可选参数的工具时，可能与采用严格 schema 校验的某些提供商出现兼容性问题。

<Note>
  这对使用结构化输出（严格模式）的 OpenAI 模型尤其相关。
</Note>

为获得最大兼容性，可选参数应使用 `.nullable()` 而非 `.optional()`：

```ts highlight="6,7,16,17"
// This may fail with strict schema validation
const failingTool = tool({
  description: 'Execute a command',
  inputSchema: z.object({
    command: z.string(),
    workdir: z.string().optional(), // This can cause errors
    timeout: z.string().optional(),
  }),
});

// This works with strict schema validation
const workingTool = tool({
  description: 'Execute a command',
  inputSchema: z.object({
    command: z.string(),
    workdir: z.string().nullable(), // Use nullable instead
    timeout: z.string().nullable(),
  }),
});
```

#### 温度设置

对于工具调用与对象生成，建议使用 `temperature: 0` 以确保确定且一致的结果：

```ts highlight="3"
const result = await generateText({
  model: __MODEL__,
  temperature: 0, // Recommended for tool calls
  tools: {
    myTool: tool({
      description: 'Execute a command',
      inputSchema: z.object({
        command: z.string(),
      }),
    }),
  },
  prompt: 'Execute the ls command',
});
```

较低的温度值可降低模型输出的随机性，这在模型需要以下行为时尤为重要：

- 生成具有特定格式的结构化数据
- 以正确参数进行精确的工具调用
- 一致地遵循严格 schema

## 调试

### 检查警告

并非所有提供商都支持全部 AI SDK 功能。
当不支持某功能时，提供商可能抛出异常或返回警告。
要检查提示、工具与设置是否被提供商正确处理，可查看调用警告：

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Hello, world!',
});

console.log(result.warnings);
```

### 请求消息

可使用 `request.messages` 检查发送给模型最后一步的输入消息。
将 `include.requestMessages` 设为 `true`，以在步骤结果中包含这些消息。

```ts highlight="6"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Hello, world!',
  include: { requestMessages: true },
});

console.log(result.finalStep.request.messages);
```

### HTTP 请求体

对于暴露原始 HTTP 请求体的模型（例如 [OpenAI](/providers/ai-sdk-providers/openai)），可进行检查。
这样你可以按提供商特定方式查看发送给模型提供商的确切负载。

请求体可通过响应的 `finalStep.request.body` 属性获取：

```ts highlight="6"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Hello, world!',
});

console.log(result.finalStep.request.body);
```
