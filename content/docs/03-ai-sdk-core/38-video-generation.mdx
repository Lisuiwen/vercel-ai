# 视频生成

<Note>
  视频生成为实验性功能，API 在未来版本中可能变更。
</Note>

AI SDK 提供 [`experimental_generateVideo`](/docs/reference/ai-sdk-core/generate-video)
函数，使用视频模型根据给定提示生成视频。

```tsx
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
});
```

可使用 `base64` 或 `uint8Array` 属性访问视频数据：

```tsx
const base64 = video.base64; // base64 video data
const uint8Array = video.uint8Array; // Uint8Array video data
```

## 设置

### 宽高比

宽高比以 `{width}:{height}` 格式的字符串指定。
模型仅支持少数宽高比，且各模型与提供商支持的宽高比不同。

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
  aspectRatio: '16:9',
});
```

### 分辨率

分辨率以 `{width}x{height}` 格式的字符串指定。
模型仅支持特定分辨率，且各模型与提供商支持的分辨率不同。

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A serene mountain landscape at sunset',
  resolution: '1280x720',
});
```

### 时长

部分视频模型支持以秒为单位指定生成视频的时长。

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A timelapse of clouds moving across the sky',
  duration: 5,
});
```

### 帧率（FPS）

部分视频模型允许指定生成视频的每秒帧数。

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A hummingbird in slow motion',
  fps: 24,
});
```

### 生成多个视频

`experimental_generateVideo` 支持一次生成多个视频：

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { videos } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A rocket launching into space',
  n: 3, // number of videos to generate
});
```

<Note>
  `experimental_generateVideo` 会根据需要自动（并行）多次调用模型，以生成请求数量的视频。
</Note>

每个视频模型对单次 API 调用可生成的视频数量有内部限制。使用 `n` 参数请求多个视频时，AI SDK 会通过适当批处理自动管理。由于计算成本，大多数视频模型每次调用仅支持生成 1 个视频。

若需要，可使用 `maxVideosPerCall` 设置覆盖此行为：

```tsx
const { videos } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A rocket launching into space',
  maxVideosPerCall: 2, // Override the default batch size
  n: 4, // Will make 2 calls of 2 videos each
});
```

### 图生视频

部分视频模型支持从输入图像生成视频。可使用提示对象提供图像：

```tsx highlight={"7-10"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: {
    image: 'https://example.com/my-image.png',
    text: 'Animate this image with gentle motion',
  },
});
```

也可将图像作为 base64 编码字符串或 `Uint8Array` 提供：

```tsx
const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: {
    image: imageBase64String, // or imageUint8Array
    text: 'Animate this image',
  },
});
```

### 提供种子

可向 `experimental_generateVideo` 提供种子以控制视频生成过程的输出。
若模型支持，相同种子将始终产生相同视频。

```tsx highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
  seed: 1234567890,
});
```

### 提供商特定设置

视频模型通常具有提供商甚至模型特定的设置。
可使用 `providerOptions` 参数将这些设置传给 `experimental_generateVideo` 函数。
提供商选项将成为请求体属性。

```tsx highlight={"8-10"}
import { experimental_generateVideo as generateVideo } from 'ai';
import { fal } from '@ai-sdk/fal';

const { video } = await generateVideo({
  model: fal.video('luma-dream-machine/ray-2'),
  prompt: 'A cat walking on a treadmill',
  aspectRatio: '16:9',
  providerOptions: {
    fal: { loop: true, motionStrength: 0.8 },
  },
});
```

### 中止信号与超时

`experimental_generateVideo` 接受可选的 `abortSignal` 参数（类型为 [`AbortSignal`](https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal)），
用于中止视频生成过程或设置超时。

```ts highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
  abortSignal: AbortSignal.timeout(60000), // Abort after 60 seconds
});
```

<Note>
  视频生成通常比图像生成耗时更长。根据模型与视频长度，建议使用更长的超时（60 秒或更长）。
</Note>

### 轮询超时

视频生成为异步过程，可能需要数分钟完成。大多数提供商使用轮询机制，SDK 定期检查视频是否就绪。默认轮询超时通常为 5 分钟，对较长视频或某些模型可能不足。

可使用提供商特定选项配置轮询超时。各提供商导出其选项类型，可与 `satisfies` 一起使用以确保类型安全：

```tsx highlight={"10-12"}
import { experimental_generateVideo as generateVideo } from 'ai';
import { fal, type FalVideoModelOptions } from '@ai-sdk/fal';

const { video } = await generateVideo({
  model: fal.video('luma-dream-machine/ray-2'),
  prompt: 'A cinematic timelapse of a city from dawn to dusk',
  duration: 10,
  providerOptions: {
    fal: {
      pollTimeoutMs: 600000, // 10 minutes
    } satisfies FalVideoModelOptions,
  },
});
```

<Note>
  生产环境建议将 `pollTimeoutMs` 设为至少 10 分钟（600000ms），
  以应对不同模型与视频长度的生成时间差异。
</Note>

### 自定义头

`experimental_generateVideo` 接受可选的 `headers` 参数（类型为 `Record<string, string>`），
用于向视频生成请求添加自定义头。

```ts highlight={"7"}
import { experimental_generateVideo as generateVideo } from 'ai';
__PROVIDER_IMPORT__;

const { video } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
  headers: { 'X-Custom-Header': 'custom-value' },
});
```

### 警告

若模型返回警告（例如不支持的参数），可在响应的 `warnings` 属性中获取。

```tsx
const { video, warnings } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A cat walking on a treadmill',
});
```

### 额外提供商特定元数据

部分提供商为整体结果或每个视频暴露额外元数据。

```tsx
const prompt = 'A cat walking on a treadmill';

const { video, providerMetadata } = await generateVideo({
  model: fal.video('luma-dream-machine/ray-2'),
  prompt,
});

// Access provider-specific metadata
const videoMetadata = providerMetadata.fal?.videos[0];
console.log({
  duration: videoMetadata?.duration,
  fps: videoMetadata?.fps,
  width: videoMetadata?.width,
  height: videoMetadata?.height,
});
```

返回的 `providerMetadata` 的外层键为提供商名称。内层值为元数据。元数据中通常存在 `videos` 键，为与顶层 `videos` 键长度相同的数组。

使用 `n > 1` 生成多个视频时，也可通过 `responses` 数组访问每次调用的元数据：

```tsx
const { videos, responses } = await generateVideo({
  model: __VIDEO_MODEL__,
  prompt: 'A rocket launching into space',
  n: 5, // May require multiple API calls
});

// Access metadata from each individual API call
for (const response of responses) {
  console.log({
    timestamp: response.timestamp,
    modelId: response.modelId,
    // Per-call provider metadata (lossless)
    providerMetadata: response.providerMetadata,
  });
}
```

### 错误处理

当 `experimental_generateVideo` 无法生成有效视频时，会抛出 [`AI_NoVideoGeneratedError`](/docs/reference/ai-sdk-errors/ai-no-video-generated-error)。

当 AI 提供商未能生成视频时会出现此错误。可能原因包括：

- 模型未能生成响应
- 模型生成的响应无法解析

该错误保留以下信息以帮助记录问题：

- `responses`：视频模型响应的元数据，包括时间戳、模型和头。
- `cause`：错误原因。可用于更详细的错误处理

```ts
import {
  experimental_generateVideo as generateVideo,
  NoVideoGeneratedError,
} from 'ai';

try {
  await generateVideo({ model, prompt });
} catch (error) {
  if (NoVideoGeneratedError.isInstance(error)) {
    console.log('NoVideoGeneratedError');
    console.log('Cause:', error.cause);
    console.log('Responses:', error.responses);
  }
}
```
