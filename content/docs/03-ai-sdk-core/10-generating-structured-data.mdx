# 生成结构化数据

文本生成很有用，但许多场景更需要结构化数据。
例如，从文本抽取信息、分类数据或生成合成数据。

许多语言模型能生成结构化数据，常通过「JSON 模式」或「tools」实现。
但你仍需手动提供 schema 并校验生成结果，因为 LLM 可能产生不正确或不完整的结构化数据。

AI SDK 通过 [`generateText`](/docs/reference/ai-sdk-core/generate-text)
与 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 上的 `output` 属性，
在各 model provider 间标准化结构化对象生成。
可使用 [Zod schema](/docs/reference/ai-sdk-core/zod-schema)、[Valibot](/docs/reference/ai-sdk-core/valibot-schema) 或 [JSON schema](/docs/reference/ai-sdk-core/json-schema) 指定所需数据形状，
模型将生成符合该结构的数据。

<Note>
  结构化输出生成是 `generateText` 与 `streamText`
  流程的一部分，因此可在同一请求中与 tool 调用组合使用。
</Note>

## 生成结构化输出

使用 `generateText` 与 `Output.object()` 根据提示生成结构化数据。
schema 也用于校验生成数据，确保类型安全与正确性。

```ts
import { generateText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { output } = await generateText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({ name: z.string(), amount: z.string() }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});
```

<Note>
  结构化输出生成在 AI SDK 的多轮执行模型中计为一步（每次模型调用或 tool 执行为一步）。与 tool 组合使用时，请在 `stopWhen` 配置中考虑这一点。
</Note>

### 访问响应头与 body

有时需要访问模型提供商的完整响应，
例如获取 provider 特定的头或 body 内容。

可通过 `response` 属性访问原始响应头与 body：

```ts
import { generateText, Output } from 'ai';

const result = await generateText({
  // ...
  output: Output.object({ schema }),
});

console.log(JSON.stringify(result.response.headers, null, 2));
console.log(JSON.stringify(result.response.body, null, 2));
```

## 流式结构化输出

返回结构化数据会增加复杂度，模型响应时间可能无法满足交互式场景。
通过 `streamText` 与 `output`，可在生成过程中流式输出模型的结构化响应。

```ts
import { streamText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { partialOutputStream } = streamText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({ name: z.string(), amount: z.string() }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});

// use partialOutputStream as an async iterable
for await (const partialObject of partialOutputStream) {
  console.log(partialObject);
}
```

可在客户端通过 [`useObject`](/docs/reference/ai-sdk-ui/use-object) hook 消费结构化输出。

### 流式错误处理

`streamText` 会立即开始流式输出。流式过程中发生的错误会成为流的一部分，而非抛出异常（以避免流崩溃）。

要处理错误，请提供 `onError` 回调：

```tsx highlight="5-7"
import { streamText, Output } from 'ai';

const result = streamText({
  // ...
  output: Output.object({ schema }),
  onError({ error }) {
    console.error(error); // log to your error tracking service
  },
});
```

关于 `generateText` 的非流式错误处理，见下文[错误处理](#error-handling)一节。

## 输出类型

AI SDK 通过 `Output` 对象支持多种方式指定生成数据的预期结构。可选择不同的结构化/文本生成与校验策略。

### `Output.text()`

使用 `Output.text()` 从模型生成纯文本。该选项不对结果强制任何 schema：你直接收到模型的文本字符串。未指定 `output` 时此为默认行为。

```ts
import { generateText, Output } from 'ai';

const { output } = await generateText({
  // ...
  output: Output.text(),
  prompt: 'Tell me a joke.',
});
// output will be a string (the joke)
```

### `Output.object()`

使用 `Output.object({ schema })` 根据 schema（例如 Zod schema）生成结构化对象。输出会经类型校验，确保返回结果符合 schema。

```ts
import { generateText, Output } from 'ai';
import { z } from 'zod';

const { output } = await generateText({
  // ...
  output: Output.object({
    schema: z.object({
      name: z.string(),
      age: z.number().nullable(),
      labels: z.array(z.string()),
    }),
  }),
  prompt: 'Generate information for a test user.',
});
// output will be an object matching the schema above
```

<Note>
  通过 `streamText` 流式输出的部分结果无法对照你提供的 schema 校验，
  因为不完整的数据可能尚不符合预期结构。
</Note>

### `Output.array()`

使用 `Output.array({ element })` 指定期望模型返回类型化对象数组，其中每个元素应符合 schema（在 `element` 属性中定义）。

```ts
import { generateText, Output } from 'ai';
import { z } from 'zod';

const { output } = await generateText({
  // ...
  output: Output.array({
    element: z.object({
      location: z.string(),
      temperature: z.number(),
      condition: z.string(),
    }),
  }),
  prompt: 'List the weather for San Francisco and Paris.',
});
// output will be an array of objects like:
// [
//   { location: 'San Francisco', temperature: 70, condition: 'Sunny' },
//   { location: 'Paris', temperature: 65, condition: 'Cloudy' },
// ]
```

使用 `streamText` 流式输出数组时，可用 `elementStream` 在生成时接收每个已完成的元素：

```ts
import { streamText, Output } from 'ai';
import { z } from 'zod';

const { elementStream } = streamText({
  // ...
  output: Output.array({
    element: z.object({
      name: z.string(),
      class: z.string(),
      description: z.string(),
    }),
  }),
  prompt: 'Generate 3 hero descriptions for a fantasy role playing game.',
});

for await (const hero of elementStream) {
  console.log(hero); // Each hero is complete and validated
}
```

<Note>
  `elementStream` 发出的每个元素都是完整且经 element schema 校验的。
  这与 `partialOutputStream` 不同，后者会流式输出包含未完成元素的整个部分数组。
</Note>

### `Output.choice()`

当期望模型从特定字符串选项集合中选择时（例如分类或固定枚举答案），使用 `Output.choice({ options })`。

```ts
import { generateText, Output } from 'ai';

const { output } = await generateText({
  // ...
  output: Output.choice({
    options: ['sunny', 'rainy', 'snowy'],
  }),
  prompt: 'Is the weather sunny, rainy, or snowy today?',
});
// output will be one of: 'sunny', 'rainy', or 'snowy'
```

可提供任意字符串选项集合，输出始终是匹配某一指定选项的单个字符串。AI SDK 会校验结果是否匹配你的选项之一；若模型返回无效值则抛出错误。

这对分类式生成或为 API 兼容性强制有效值尤其有用。

### `Output.json()`

当需要生成并解析模型返回的非结构化 JSON、且不强制特定 schema 时，使用 `Output.json()`。适用于捕获任意对象、灵活结构，或依赖模型自然输出而非严格校验的场景。

```ts
import { generateText, Output } from 'ai';

const { output } = await generateText({
  // ...
  output: Output.json(),
  prompt:
    'For each city, return the current temperature and weather condition as a JSON object.',
});

// output could be any valid JSON, for example:
// {
//   "San Francisco": { "temperature": 70, "condition": "Sunny" },
//   "Paris": { "temperature": 65, "condition": "Cloudy" }
// }
```

使用 `Output.json` 时，AI SDK 仅检查响应是否为合法 JSON，不校验值的结构或类型。若需要 schema 校验，请改用 `.object` 或 `.array` 输出。

更高级的校验或不同结构，见 [Output API 参考](/docs/reference/ai-sdk-core/output)。

## 结合 Tool 生成结构化输出

在 `generateText` 与 `streamText` 中使用结构化输出的关键优势之一，是可与 tool 调用组合。

```ts
import { generateText, Output, tool, isStepCount } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { output } = await generateText({
  model: __MODEL__,
  tools: {
    weather: tool({
      description: 'Get the weather for a location',
      inputSchema: z.object({ location: z.string() }),
      execute: async ({ location }) => {
        // fetch weather data
        return { temperature: 72, condition: 'sunny' };
      },
    }),
  },
  output: Output.object({
    schema: z.object({
      summary: z.string(),
      recommendation: z.string(),
    }),
  }),
  stopWhen: isStepCount(5),
  prompt: 'What should I wear in San Francisco today?',
});
```

<Note>
  将 tool 与结构化输出一起使用时，请记住生成结构化输出也计为一步。请配置 `stopWhen`，为 tool 执行与输出生成留出足够步数。
</Note>

## 属性描述

可在各 schema 属性上使用 `.describe("...")`，向模型说明每个属性的用途。这有助于提高生成结构化数据的质量与准确性：

```ts highlight="5,9"
import { generateText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { output } = await generateText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      name: z.string().describe('The name of the recipe'),
      ingredients: z
        .array(
          z.object({
            name: z.string(),
            amount: z
              .string()
              .describe('The amount of the ingredient (grams or ml)'),
          }),
        )
        .describe('List of ingredients with amounts'),
      steps: z.array(z.string()).describe('Step-by-step cooking instructions'),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});
```

属性描述尤其适用于：

- 澄清含糊的属性名
- 指定预期格式或约定
- 为复杂嵌套结构提供上下文

## 输出名称与描述

可为输出可选指定 `name` 与 `description`。部分 provider 会将其用于额外 LLM 引导，例如通过 tool 或 schema 名称。

```ts highlight="6-7"
import { generateText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { output } = await generateText({
  model: __MODEL__,
  output: Output.object({
    name: 'Recipe',
    description: 'A recipe for a dish.',
    schema: z.object({
      name: z.string(),
      ingredients: z.array(z.object({ name: z.string(), amount: z.string() })),
      steps: z.array(z.string()),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});
```

适用于所有支持结构化生成的输出类型：

- `Output.object({ name, description, schema })`
- `Output.array({ name, description, element })`
- `Output.choice({ name, description, options })`
- `Output.json({ name, description })`

## 访问推理内容

可通过结果上的 `reasoning` 属性访问语言模型生成对象时使用的推理。若可用，该属性包含模型思考过程的字符串。

```ts
import { generateText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const result = await generateText({
  model: __MODEL__, // must be a reasoning model
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({
            name: z.string(),
            amount: z.string(),
          }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});

console.log(result.reasoningText);
```

## 错误处理

当带结构化输出的 `generateText` 无法生成有效对象时，会抛出 [`AI_NoObjectGeneratedError`](/docs/reference/ai-sdk-errors/ai-no-object-generated-error)。

当 AI provider 未能生成可解析且符合 schema 的对象时会出现此错误。
可能原因包括：

- 模型未能生成响应。
- 模型生成的响应无法解析。
- 模型生成的响应无法通过 schema 校验。

错误保留以下信息以帮助记录问题：

- `text`：模型生成的文本。根据对象生成模式，可能是原始文本或 tool 调用文本。
- `response`：语言模型响应的元数据，包括响应 id、时间戳与模型。
- `usage`：请求的 token 用量。
- `cause`：错误原因（例如 JSON 解析错误）。可用于更详细的错误处理。

```ts
import { generateText, Output, NoObjectGeneratedError } from 'ai';

try {
  await generateText({
    model,
    output: Output.object({ schema }),
    prompt,
  });
} catch (error) {
  if (NoObjectGeneratedError.isInstance(error)) {
    console.log('NoObjectGeneratedError');
    console.log('Cause:', error.cause);
    console.log('Text:', error.text);
    console.log('Response:', error.response);
    console.log('Usage:', error.usage);
  }
}
```
