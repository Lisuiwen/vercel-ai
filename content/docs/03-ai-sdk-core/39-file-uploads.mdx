# 文件上传

AI SDK 提供 [`uploadFile`](/docs/reference/ai-sdk-core/upload-file)
函数，用于将文件上传到提供商并获取可在后续 API 调用中使用的 `ProviderReference`。

在 AI SDK 中，已上传文件由 `ProviderReference` 标识——
将提供商名称映射到提供商特定标识符的 `Record<string, string>`。
该概念也用于其他提供商特定资产引用，例如已上传的 skill。

```ts
import { uploadFile, generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import fs from 'node:fs';

const { providerReference } = await uploadFile({
  api: openai.files(),
  data: fs.readFileSync('./photo.png'),
  filename: 'photo.png',
});

const { text } = await generateText({
  model: openai.responses('gpt-4o-mini'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe what you see in this image.' },
        { type: 'file', mediaType: 'image', data: providerReference },
      ],
    },
  ],
});
```

作为简写，可将提供商实例直接传给 `api`，而无需显式调用 `.files()`——SDK 会为你调用 `.files()`：

```ts highlight="3"
const { providerReference } = await uploadFile({
  api: openai, // shorthand for openai.files()
  data: fs.readFileSync('./photo.png'),
  filename: 'photo.png',
});
```

## 支持的文件类型

根据提供商不同，可上传图像、PDF、文本文件及其他文档。
未显式指定时，会从文件字节自动检测媒体类型：

```ts highlight="4"
const { providerReference } = await uploadFile({
  api: anthropic.files(),
  data: fs.readFileSync('./document.pdf'),
  mediaType: 'application/pdf', // optional, auto-detected if omitted
  filename: 'document.pdf',
});
```

在文件内容部分中使用 `providerReference` 及其媒体类型：

```ts
{
  role: 'user',
  content: [
    { type: 'text', text: 'Summarize this document.' },
    { type: 'file', data: providerReference, mediaType: 'application/pdf' },
  ],
}
```

## 提供商特定选项

部分提供商通过 `providerOptions` 接受额外选项。
例如，OpenAI 需要 `purpose` 字段：

```ts highlight="5-9"
import { openai, type OpenAIFilesOptions } from '@ai-sdk/openai';

const { providerReference } = await uploadFile({
  api: openai.files(),
  data: fs.readFileSync('./photo.png'),
  providerOptions: {
    openai: {
      purpose: 'assistants',
    } satisfies OpenAIFilesOptions,
  },
});
```

## 提供商引用

`ProviderReference` 是将提供商名称映射到提供商特定文件标识符的 `Record<string, string>`：

```ts
// Example ProviderReference
{
  openai: 'file-abc123',
}
```

将 `ProviderReference` 作为消息内容部分的 `data` 或 `image` 字段传入时，提供商会从引用中查找自己的文件 ID。若引用中不包含当前提供商的条目，将抛出错误。

## 多提供商用法

若在对话中途切换提供商（例如用 Anthropic 继续以 OpenAI 开始的聊天），需要将文件上传到两个提供商并合并引用：

```ts
const openaiResult = await uploadFile({
  api: openai.files(),
  data: imageBytes,
  filename: 'photo.png',
});

const anthropicResult = await uploadFile({
  api: anthropic.files(),
  data: imageBytes,
  filename: 'photo.png',
});

const mergedReference = {
  ...openaiResult.providerReference,
  ...anthropicResult.providerReference,
};

// mergedReference: { openai: 'file-abc123', anthropic: 'file-xyz789' }
```

合并后的引用可在消息中使用，无论由哪个提供商处理请求——各提供商都会找到自己的文件 ID。

## 支持的提供商

以下提供商支持 `files()` 与文件上传：

| 提供商    | 工厂方法            |
| --------- | ------------------- |
| Anthropic | `anthropic.files()` |
| Google    | `google.files()`    |
| OpenAI    | `openai.files()`    |
| xAI       | `xai.files()`       |

不支持文件上传的提供商若在消息中遇到提供商引用，将抛出 `UnsupportedFunctionalityError`。
