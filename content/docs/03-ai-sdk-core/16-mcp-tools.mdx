# Model Context Protocol (MCP)

AI SDK 支持连接 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 服务器，以访问其 tool、resource 与 prompt。
这使你的 AI 应用可通过标准化接口发现并调用各服务的能力。

<Note>
  若使用 OpenAI 的 Responses API，也可使用内置的
  `openai.tools.mcp` tool，无需转换 tool 即可直接集成 MCP 服务器。详见 [OpenAI provider
  文档](/providers/ai-sdk-providers/openai#mcp-tool)。
</Note>

## 初始化 MCP 客户端

生产环境建议使用 HTTP 传输（如 `StreamableHTTPClientTransport`）。stdio 传输仅应用于连接本地服务器，无法部署到生产环境。

可使用以下传输选项创建 MCP 客户端：

- **HTTP 传输（推荐）**：在客户端上通过 `transport: { type: 'http', ... }` 直接配置 HTTP，或使用 MCP 官方 TypeScript SDK 的 `StreamableHTTPClientTransport`
- SSE（Server-Sent Events）：另一种基于 HTTP 的传输
- `stdio`：仅用于本地开发，通过标准输入/输出流连接本地 MCP 服务器

### HTTP 传输（推荐）

生产部署建议使用 HTTP 传输。可在客户端上直接配置：

```typescript
import { createMCPClient } from '@ai-sdk/mcp';

const mcpClient = await createMCPClient({
  transport: {
    type: 'http',
    url: 'https://your-server.com/mcp',

    // optional: configure HTTP headers
    headers: { Authorization: 'Bearer my-api-key' },

    // optional: provide an OAuth client provider for automatic authorization
    authProvider: myOAuthClientProvider,

    // optional: allow redirect responses (default is 'error' to prevent SSRF)
    redirect: 'follow',
  },
});
```

也可使用 MCP 官方 TypeScript SDK 的 `StreamableHTTPClientTransport`：

```typescript
import { createMCPClient } from '@ai-sdk/mcp';
import { StreamableHTTPClientTransport } from '@modelcontextprotocol/sdk/client/streamableHttp.js';

const url = new URL('https://your-server.com/mcp');
const mcpClient = await createMCPClient({
  transport: new StreamableHTTPClientTransport(url, {
    sessionId: 'session_123',
  }),
});
```

### SSE 传输

SSE 提供另一种基于 HTTP 的传输，通过 `type` 与 `url` 配置。也可提供 `authProvider` 以支持 OAuth：

```typescript
import { createMCPClient } from '@ai-sdk/mcp';

const mcpClient = await createMCPClient({
  transport: {
    type: 'sse',
    url: 'https://my-server.com/sse',

    // optional: configure HTTP headers
    headers: { Authorization: 'Bearer my-api-key' },

    // optional: provide an OAuth client provider for automatic authorization
    authProvider: myOAuthClientProvider,

    // optional: allow redirect responses (default is 'error' to prevent SSRF)
    redirect: 'follow',
  },
});
```

### Stdio 传输（本地服务器）

<Note type="warning">
  stdio 传输仅应用于本地服务器。
</Note>

Stdio 传输可从 MCP SDK 或 AI SDK 导入：

```typescript
import { createMCPClient } from '@ai-sdk/mcp';
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js';
// Or use the AI SDK's stdio transport:
// import { Experimental_StdioMCPTransport as StdioClientTransport } from '@ai-sdk/mcp/mcp-stdio';

const mcpClient = await createMCPClient({
  transport: new StdioClientTransport({
    command: 'node',
    args: ['src/stdio/dist/server.js'],
  }),
});
```

### 自定义传输

也可通过实现 `MCPTransport` 接口提供自定义传输，以满足标准传输未覆盖的需求。

<Note>
  `createMCPClient` 返回的客户端为用于 tool 转换的轻量客户端，目前不支持完整 MCP 客户端的全部能力，例如：session
  管理、可恢复流、接收通知。

使用 AI SDK MCP 的 HTTP 或 SSE 传输并提供 `authProvider` 时，支持通过 OAuth 进行授权。

</Note>

### 关闭 MCP 客户端

初始化后，应根据使用模式关闭 MCP 客户端：

- 短生命周期（如单次请求）：在响应结束后关闭客户端
- 长生命周期（如命令行应用）：保持客户端打开，但在应用退出时确保关闭

流式响应时，可在 LLM 响应结束后关闭客户端。例如使用 `streamText` 时，应使用 `onFinish` 回调：

```typescript
const mcpClient = await createMCPClient({
  // ...
});

const tools = await mcpClient.tools();

const result = await streamText({
  model: __MODEL__,
  tools,
  prompt: 'What is the weather in Brooklyn, New York?',
  onFinish: async () => {
    await mcpClient.close();
  },
});
```

非流式生成时，可在框架中使用 try/finally 或清理函数：

```typescript
import { createMCPClient, type MCPClient } from '@ai-sdk/mcp';

let mcpClient: MCPClient | undefined;

try {
  mcpClient = await createMCPClient({
    // ...
  });
} finally {
  await mcpClient?.close();
}
```

## 使用 MCP Tool

客户端的 `tools` 方法在 MCP tool 与 AI SDK tool 之间充当适配器，支持两种处理 tool schema 的方式：

### Schema 发现

通过 schema 发现，自动列出服务器提供的全部 tool，并根据服务器提供的 schema 推断输入参数类型：

```typescript
const tools = await mcpClient.tools();
```

实现简单，并随服务器变更自动同步。但开发时无 TypeScript 类型安全，且会加载服务器上的全部 tool。

### Schema 定义

为获得更好的类型安全与控制，可在客户端代码中显式定义 tool 及其输入 schema：

```typescript
import { z } from 'zod';

const tools = await mcpClient.tools({
  schemas: {
    'get-data': {
      inputSchema: z.object({
        query: z.string().describe('The data query'),
        format: z.enum(['json', 'text']).optional(),
      }),
    },
    // For tools with zero inputs, you should use an empty object:
    'tool-with-no-args': {
      inputSchema: z.object({}),
    },
  },
});
```

可提供完整 TypeScript 类型安全与 IDE 自动补全，在开发阶段发现参数不匹配。定义 `schemas` 时，客户端仅拉取显式定义的 tool，使应用聚焦于所需 tool。

### 类型化的 Tool 输出

当 MCP 服务器返回 `structuredContent`（见 [MCP 规范](https://modelcontextprotocol.io/specification/2025-06-18/server/tools#structured-content)）时，可定义 `outputSchema` 以获得类型化的 tool 结果：

```typescript
import { z } from 'zod';

const tools = await mcpClient.tools({
  schemas: {
    'get-weather': {
      inputSchema: z.object({
        location: z.string(),
      }),
      // Define outputSchema for typed results
      outputSchema: z.object({
        temperature: z.number(),
        conditions: z.string(),
        humidity: z.number(),
      }),
    },
  },
});

const result = await tools['get-weather'].execute(
  { location: 'New York' },
  { messages: [], toolCallId: 'weather-1' },
);

console.log(`Temperature: ${result.temperature}°C`);
```

提供 `outputSchema` 时：

- 客户端从 tool 结果中提取 `structuredContent`
- 输出在运行时按 schema 校验
- 获得完整的 TypeScript 类型安全

若服务器未返回 `structuredContent`，客户端会回退到解析文本内容中的 JSON。若两者皆无或校验失败，则抛出错误。

<Note>
  未提供 `outputSchema` 时，tool 返回原始 `CallToolResult` 对象，
  包含 `content` 及可选的 `isError` 字段。
</Note>

## 使用 MCP Resource

根据 [MCP 规范](https://modelcontextprotocol.io/docs/learn/server-concepts#resources)，resource 是由**应用驱动**的数据源，为模型提供上下文。与 tool（由模型控制）不同，由应用决定何时获取并将 resource 作为上下文传入。

MCP 客户端提供三个与 resource 相关的方法：

### 列出 Resource

列出 MCP 服务器上的全部可用 resource：

```typescript
const resources = await mcpClient.listResources();
```

### 读取 Resource 内容

按 URI 读取指定 resource 的内容：

```typescript
const resourceData = await mcpClient.readResource({
  uri: 'file:///example/document.txt',
});
```

### 列出 Resource 模板

Resource 模板为动态 URI 模式，支持灵活查询。列出全部可用模板：

```typescript
const templates = await mcpClient.listResourceTemplates();
```

## 使用 MCP Prompt

<Note type="warning">
  MCP Prompt 为实验性功能，未来可能变更。
</Note>

根据 MCP 规范，prompt 是服务器暴露的、由用户控制的模板，客户端可列出并按可选参数获取。

### 列出 Prompt

```typescript
const prompts = await mcpClient.experimental_listPrompts();
```

### 获取 Prompt

获取 prompt 消息，可按服务器定义的参数传入：

```typescript
const prompt = await mcpClient.experimental_getPrompt({
  name: 'code_review',
  arguments: { code: 'function add(a, b) { return a + b; }' },
});
```

## 处理征询请求（Elicitation）

征询是 MCP 服务器在 tool 执行期间向客户端请求额外信息的机制。例如服务器可能需要用户输入以完成注册表单，或对敏感操作进行确认。

<Note type="warning">
  由客户端应用负责正确处理征询请求。
  MCP 客户端仅将服务器的这些请求转交给你的应用代码处理。
</Note>

### 启用征询支持

创建 MCP 客户端时需声明支持征询的能力：

```typescript
const mcpClient = await createMCPClient({
  transport: {
    type: 'sse',
    url: 'https://your-server.com/sse',
  },
  capabilities: {
    elicitation: {},
  },
});
```

### 注册征询处理器

使用 `onElicitationRequest` 注册处理器，在服务器请求输入时调用：

```typescript
import { ElicitationRequestSchema } from '@ai-sdk/mcp';

mcpClient.onElicitationRequest(ElicitationRequestSchema, async request => {
  // request.params.message: A message describing what input is needed
  // request.params.requestedSchema: JSON schema defining the expected input structure

  // Get input from the user (implement according to your application's needs)
  const userInput = await getInputFromUser(
    request.params.message,
    request.params.requestedSchema,
  );

  // Return the result with one of three actions:
  return {
    action: 'accept', // or 'decline' or 'cancel'
    content: userInput, // only required when action is 'accept'
  };
});
```

### 征询响应操作

处理器必须返回包含 `action` 字段的对象，可为：

- `'accept'`：用户提供了请求的信息。`action` 为 `'accept'` 时必须包含 `content`。
- `'decline'`：用户选择不提供信息。
- `'cancel'`：用户取消了整个操作。
