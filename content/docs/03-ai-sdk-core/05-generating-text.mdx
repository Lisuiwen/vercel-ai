# 生成与流式输出文本

大语言模型（LLM）可根据提示生成文本，提示可包含待处理的指令与信息。
例如，可让模型撰写食谱、起草邮件或摘要文档。

AI SDK Core 提供两个函数，用于从 LLM 生成并流式输出文本：

- [`generateText`](#generatetext)：根据给定提示与模型生成文本。
- [`streamText`](#streamtext)：根据给定提示与模型流式输出文本。

[tool 调用](./tools-and-tool-calling)与[结构化数据生成](./generating-structured-data)等高级 LLM 能力均构建在文本生成之上。

## `generateText`

你可以使用 [`generateText`](/docs/reference/ai-sdk-core/generate-text) 函数生成文本。该函数适用于需要撰写文本的非交互式场景（例如起草邮件或汇总网页），以及使用工具的 agent。

```tsx
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const { text } = await generateText({
  model: __MODEL__,
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

你也可以使用更[高级的提示](./prompts)，通过更复杂的指令与内容生成文本：

```tsx
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const { text } = await generateText({
  model: __MODEL__,
  instructions:
    'You are a professional writer. ' +
    'You write simple, clear, and concise content.',
  prompt: `Summarize the following article in 3-5 sentences: ${article}`,
});
```

`generateText` 的返回对象包含生成的输出与元数据：

- `result.content`：所有步骤中生成的内容。
- `result.text`：最后一步生成的文本。
- `result.files`：所有步骤中生成的文件。
- `result.sources`：所有步骤中用作参考的来源（仅部分模型可用）。
- `result.toolCalls`：所有步骤中进行的 tool 调用。
- `result.toolResults`：所有步骤中 tool 调用的结果。
- `result.finishReason`：模型结束生成文本的原因。
- `result.rawFinishReason`：生成结束时的原始原因（来自提供商）。
- `result.usage`：所有步骤的总用量（适用于多步生成）。
- `result.warnings`：所有步骤中来自模型提供商的警告（例如不支持的设置）。
- `result.steps`：所有步骤的详细信息，可用于获取中间步骤相关信息，包含每步的 `performance`。
- `result.finalStep`：最后一步的详细信息，包含 `performance`。
- `result.output`：使用 `output` 规范生成的结构化输出。

每一步都包含带有耗时与吞吐量信息的 `performance`：

- `effectiveOutputTokensPerSecond`：有效输出 token 吞吐（token/秒），按 `outputTokens / requestSeconds` 计算。
- `outputTokensPerSecond`：对流式步骤，首个输出 token 之后的输出 token 吞吐（token/秒），按 `outputTokens / outputStreamSeconds` 计算。对 `generateText`，该值为 `undefined`。
- `inputTokensPerSecond`：对流式步骤，首个输出 token 之前的输入 token 吞吐（token/秒），按 `inputTokens / ttftSeconds` 计算。对 `generateText`，该值为 `undefined`。
- `effectiveTotalTokensPerSecond`：有效总 token 吞吐（token/秒），按 `(inputTokens + outputTokens) / requestSeconds` 计算。
- `stepTimeMs`：该步骤花费的总时间（毫秒），包含语言模型响应时间与 tool 执行时间。
- `responseTimeMs`：等待语言模型响应所花费的时间（毫秒）。
- `toolExecutionMs`：该步骤中每个在客户端执行的 tool 调用所花费的时间（毫秒），以 tool 调用 ID 为键。
- `timeToFirstOutputTokenMs`：对流式步骤，直到收到首个文本、推理或 tool 输入增量为止的时间（毫秒）。对 `generateText`，该值为 `undefined`。

### 访问响应标头与正文

有时你需要访问模型提供商的完整响应，
例如读取某些提供商专属的标头或正文内容。

你可以通过 `finalStep.response` 访问原始响应标头与正文：

```ts
import { generateText } from 'ai';

const result = await generateText({
  // ...
});

console.log(JSON.stringify(result.finalStep.response.headers, null, 2));
console.log(JSON.stringify(result.finalStep.response.body, null, 2));
```

### `onFinish` 回调

使用 `generateText` 时，可以提供在最后一个步骤完成后触发的 `onFinish` 回调（参见 [API 参考](/docs/reference/ai-sdk-core/generate-text#on-finish)）。
其中包含文本、用量信息、结束原因、消息、步骤、总用量等：

```tsx highlight="6-8"
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  onFinish({ text, finishReason, usage, responseMessages, steps, totalUsage }) {
    // your own logic, e.g. for saving the chat history or recording usage

    const messages = responseMessages; // messages that were generated
  },
});
```

### 生命周期回调（实验性）

<Note type="warning">
  实验性回调可能在增量发布的包中发生破坏性变更。
</Note>

`generateText` 提供若干实验性生命周期回调，让你可以接入生成流程的不同阶段。
它们适用于记录日志、可观测性、调试以及自定义遥测。
在这些回调中抛出的错误会被静默捕获，不会中断生成流程。

```tsx
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  prompt: 'What is the weather in San Francisco?',
  tools: {
    // ... your tools
  },

  experimental_onStart({ modelId }) {
    console.log('Generation started', { modelId });
  },

  experimental_onStepStart({ stepNumber, modelId, messages }) {
    console.log(`Step ${stepNumber} starting`, { modelId });
  },

  experimental_onLanguageModelCallStart({ modelId, messages }) {
    console.log('Model call starting', {
      modelId,
      messageCount: messages.length,
    });
  },

  experimental_onLanguageModelCallEnd({ modelId, finishReason, content }) {
    console.log('Model call finished', {
      modelId,
      finishReason,
      contentParts: content.length,
    });
  },

  onToolExecutionStart({ toolCall }) {
    console.log(`Tool call starting: ${toolCall.toolName}`, {
      toolCallId: toolCall.toolCallId,
    });
  },

  onToolExecutionEnd({ toolCall, toolExecutionMs, toolOutput }) {
    console.log(
      `Tool call finished: ${toolCall.toolName} (${toolExecutionMs}ms)`,
      {
        success: toolOutput.type === 'tool-result',
      },
    );
  },

  onStepFinish({ stepNumber, finishReason, usage, performance }) {
    console.log(`Step ${stepNumber} finished`, {
      finishReason,
      usage,
      performance,
    });
  },
});
```

可用的生命周期回调如下：

- **`experimental_onStart`**：在 `generateText` 操作开始时调用一次，早于任何 LLM 调用。可接收模型信息、消息、设置与 `runtimeContext`。
- **`experimental_onStepStart`**：在每一步（LLM 调用）之前调用。可接收步骤编号、模型、即将发送的消息、工具以及先前步骤。
- **`experimental_onLanguageModelCallStart`**：在提供商模型调用即将开始前立即调用。若你希望将模型调用与后续的 tool 执行分开观察，这会很有用。
- **`experimental_onLanguageModelCallEnd`**：在模型响应完成规范化与解析之后、任何客户端 tool 执行开始之前调用。可接收模型调用的内容分片、用量与结束原因。
- **`onToolExecutionStart`**：在工具的 `execute` 函数即将运行前调用。可接收 tool 调用对象、消息与 `toolContext`。
- **`onToolExecutionEnd`**：在工具的 `execute` 函数完成或出错后立即调用。可接收 tool 调用对象、`toolExecutionMs`，以及 `toolOutput` 判别联合（`type: 'tool-result'` 且包含 `output`，或 `type: 'tool-error'` 且包含 `error`）。
- **`onStepFinish`**：在每一步完成后调用。包含 `stepNumber`（已完成步骤的从零开始索引）。

## `streamText`

根据你的模型与提示不同，大语言模型（LLM）可能需要长达约一分钟才能完成响应生成。在聊天机器人或实时应用等交互式场景中，这种延迟往往难以接受，用户通常希望尽快看到输出。

AI SDK Core 提供 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 函数，用于简化从 LLM 流式输出文本：

```ts
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

const result = streamText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
});

// example: use textStream as an async iterable
for await (const textPart of result.textStream) {
  console.log(textPart);
}
```

<Note>
  `result.textStream` 既是 `ReadableStream`，也是 `AsyncIterable`。
</Note>

<Note type="warning">
  `streamText` 会立即开始流式输出，并会抑制错误以防服务器崩溃。请使用 `onError` 回调记录错误。
</Note>

你可以单独使用 `streamText`，也可以与 [AI SDK
UI](/examples/next-pages/basics/streaming-text-generation) 及 [AI SDK
RSC](/examples/next-app/basics/streaming-text-generation) 组合使用。
返回对象包含多个辅助函数，便于接入 [AI SDK UI](/docs/ai-sdk-ui)：

- `result.toUIMessageStreamResponse()`：创建 UI Message 流的 HTTP 响应（包含 tool 调用等），可在 Next.js App Router API 路由中使用。
- `result.pipeUIMessageStreamToResponse()`：将 UI Message 流的增量输出写入类 Node.js 的 response 对象。
- `result.toTextStreamResponse()`：创建简单的文本流 HTTP 响应。
- `result.pipeTextStreamToResponse()`：将文本增量输出写入类 Node.js 的 response 对象。

<Note>
  `streamText` 使用背压机制，仅在被请求时生成 token。你需要消费流，生成才会结束。
</Note>

还提供多个在流结束时完成的 Promise：

- `result.content`：所有步骤中生成的内容。
- `result.text`：最后一步生成的文本。
- `result.finalStep`：最后一步的详细信息，包含每步的 `performance`。
- `result.files`：模型在所有步骤中生成的文件。
- `result.sources`：所有步骤中用作参考的来源（仅部分模型可用）。
- `result.toolCalls`：所有步骤中已执行的 tool 调用。
- `result.toolResults`：所有步骤中生成的 tool 结果。
- `result.finishReason`：模型结束生成文本的原因。
- `result.rawFinishReason`：生成结束时的原始原因（来自提供商）。
- `result.usage`：所有步骤的总用量（适用于多步生成）。
- `result.totalUsage`：已弃用。请改用 `result.usage`。
- `result.warnings`：所有步骤中来自模型提供商的警告（例如不支持的设置）。
- `result.steps`：所有步骤的详细信息，可用于获取中间步骤相关信息，包含每步的 `performance`。

对 `streamText`，当某一步收到首个文本、推理或 tool 输入增量时，会设置 `timeToFirstOutputTokenMs`。

### `onError` 回调

`streamText` 会立即开始流式输出，以便在无需等待模型结束的情况下发送数据。
错误会成为流的一部分而不会被抛出，从而避免例如服务器崩溃等问题。

若要记录错误，可以提供在错误发生时触发的 `onError` 回调。

```tsx highlight="6-8"
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

const result = streamText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  onError({ error }) {
    console.error(error); // your error logging logic here
  },
});
```

### `onChunk` 回调

使用 `streamText` 时，可以提供在流的每个分片到达时触发的 `onChunk` 回调。

它可接收以下分片类型：

- `text`
- `reasoning`
- `source`
- `tool-call`
- `tool-input-start`
- `tool-input-delta`
- `tool-result`
- `raw`

```tsx highlight="6-11"
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

const result = streamText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  onChunk({ chunk }) {
    // implement your own logic here, e.g.:
    if (chunk.type === 'text') {
      console.log(chunk.text);
    }
  },
});
```

### `onFinish` 回调

使用 `streamText` 时，可以在流结束时触发 `onFinish` 回调（参见 [API 参考](/docs/reference/ai-sdk-core/stream-text#on-finish)）。
其中包含文本、用量信息、结束原因、消息、步骤、总用量等：

```tsx highlight="6-8"
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

const result = streamText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  onFinish({ text, finishReason, usage, responseMessages, steps, totalUsage }) {
    // your own logic, e.g. for saving the chat history or recording usage

    const messages = responseMessages; // messages that were generated
  },
});
```

### `fullStream` 属性

你可以通过 `fullStream` 读取包含全部事件的流。
若你想实现自定义 UI 或以不同方式处理流，这会很有用。

import { streamText } from 'ai';

const result = streamText({

      inputSchema: z.object({ city: z.string() }),

        attractions: ['attraction1', 'attraction2', 'attraction3'],
      }),
    },
  },
  prompt: 'What are some San Francisco tourist attractions?',

for await (const part of result.fullStream) {
  switch (part.type) {
    case 'start': {
      // handle start of stream
      break;
    }
    case 'start-step': {
      // handle start of step
      break;
    }
    case 'text-start': {
      // handle text start
      break;
    }
    case 'text-delta': {
      // handle text delta here
      break;
    }
    case 'text-end': {
      // handle text end
      break;
    }
    case 'reasoning-start': {
      // handle reasoning start
      break;
    }
    case 'reasoning-delta': {
      // handle reasoning delta here
      break;
    }
    case 'reasoning-end': {
      // handle reasoning end
      break;
    }
    case 'source': {
      // handle source here
      break;
    }
    case 'file': {
      // handle file here
      break;
    }
    case 'tool-call': {
      switch (part.toolName) {
        case 'cityAttractions': {
          // handle tool call here
          break;
        }
      }
      break;
    }
    case 'tool-input-start': {
      // handle tool input start

    }

      // handle tool input delta
      break;

### 流转换

你可以使用 `experimental_transform` 选项转换流。
这适用于例如过滤、修改或平滑文本流等场景。

转换会在回调触发与 Promise 解析之前应用。
例如，若你有一个将所有文本转为大写的转换，`onFinish` 回调将收到转换后的文本。

#### 平滑流

AI SDK Core 提供 [`smoothStream` 函数](/docs/reference/ai-sdk-core/smooth-stream)，
可用于平滑文本与推理内容的流式输出。

```tsx highlight="6"
import { smoothStream, streamText } from 'ai';

const result = streamText({
  model,
  prompt,
  experimental_transform: smoothStream(),
});
```

#### 自定义转换

你也可以实现自己的自定义转换。
转换函数接收模型可用的工具，
并返回用于转换流的函数。
工具可以是泛型，也可以限定为你正在使用的工具集。

下面是一个将所有文本转为大写的自定义转换示例：

```ts
import { streamText, type TextStreamPart, type ToolSet } from 'ai';

const upperCaseTransform =
  <TOOLS extends ToolSet>() =>
  (options: { tools: TOOLS; stopStream: () => void }) =>
    new TransformStream<TextStreamPart<TOOLS>, TextStreamPart<TOOLS>>({
      transform(chunk, controller) {
        controller.enqueue(
          // for text-delta chunks, convert the text to uppercase:
          chunk.type === 'text-delta'
            ? { ...chunk, text: chunk.text.toUpperCase() }
            : chunk,
        );
      },
    });
```

你也可以使用 `stopStream` 停止流。
例如，若你想在违反模型护栏时停止流（例如生成了不当内容），这会很有用。

调用 `stopStream` 时，重要的是模拟 `finish-step` 与 `finish` 事件，以确保返回格式良好的流
并触发所有回调。

```ts
import { streamText, type TextStreamPart, type ToolSet } from 'ai';

const stopWordTransform =
  <TOOLS extends ToolSet>() =>
  ({ stopStream }: { stopStream: () => void }) =>
    new TransformStream<TextStreamPart<TOOLS>, TextStreamPart<TOOLS>>({
      // note: this is a simplified transformation for testing;
      // in a real-world version more there would need to be
      // stream buffering and scanning to correctly emit prior text
      // and to detect all STOP occurrences.
      transform(chunk, controller) {
        if (chunk.type !== 'text-delta') {
          controller.enqueue(chunk);
          return;
        }

        if (chunk.text.includes('STOP')) {
          // stop the stream
          stopStream();

          // simulate the finish-step event
          controller.enqueue({
            type: 'finish-step',
            finishReason: 'stop',
            rawFinishReason: 'stop',
            usage: {
              inputTokens: undefined,
              inputTokenDetails: {
                cacheReadTokens: undefined,
                cacheWriteTokens: undefined,
                noCacheTokens: undefined,
              },
              outputTokens: undefined,
              outputTokenDetails: {
                reasoningTokens: undefined,
                textTokens: undefined,
              },
              totalTokens: undefined,
            },
            performance: {
              effectiveOutputTokensPerSecond: 0,
              outputTokensPerSecond: undefined,
              inputTokensPerSecond: undefined,
              effectiveTotalTokensPerSecond: 0,
              stepTimeMs: 0,
              responseTimeMs: 0,
              toolExecutionMs: {},
              timeToFirstOutputTokenMs: undefined,
            },
            response: {
              id: 'response-id',
              modelId: 'mock-model-id',
              timestamp: new Date(0),
            },
            providerMetadata: undefined,
          });

          // simulate the finish event
          controller.enqueue({
            type: 'finish',
            finishReason: 'stop',
            rawFinishReason: 'stop',
            totalUsage: {
              inputTokens: undefined,
              inputTokenDetails: {
                cacheReadTokens: undefined,
                cacheWriteTokens: undefined,
                noCacheTokens: undefined,
              },
              outputTokens: undefined,
              outputTokenDetails: {
                reasoningTokens: undefined,
                textTokens: undefined,
              },
              totalTokens: undefined,
            },
          });

          return;
        }

        controller.enqueue(chunk);
      },
    });
```

#### 多个转换

你也可以提供多个转换。它们会按提供的顺序依次应用。

```tsx highlight="4"
const result = streamText({
  model,
  prompt,
  experimental_transform: [firstTransform, secondTransform],
});
```

## 来源

部分提供商（例如 [Perplexity](/providers/ai-sdk-providers/perplexity#sources) 与
[Google](/providers/ai-sdk-providers/google#sources)）会在响应中包含来源。

目前来源仅限于用于支撑响应的网页。
你可以通过结果的 `sources` 属性访问它们。

每个 `url` 来源包含以下属性：

- `id`：来源 ID。
- `url`：来源 URL。
- `title`：可选的来源标题。
- `providerMetadata`：来源的提供商元数据。

使用 `generateText` 时，可通过 `sources` 属性访问来源：

```ts
const result = await generateText({
  model: 'google/gemini-2.5-flash',
  tools: {
    google_search: google.tools.googleSearch({}),
  },
  prompt: 'List the top 5 San Francisco news from the past week.',
});

for (const source of result.sources) {
  if (source.sourceType === 'url') {
    console.log('ID:', source.id);
    console.log('Title:', source.title);
    console.log('URL:', source.url);
    console.log('Provider metadata:', source.providerMetadata);
    console.log();
  }
}
```

使用 `streamText` 时，可通过 `fullStream` 属性访问来源：

```tsx
const result = streamText({
  model: 'google/gemini-2.5-flash',
  tools: {
    google_search: google.tools.googleSearch({}),
  },
  prompt: 'List the top 5 San Francisco news from the past week.',
});

for await (const part of result.fullStream) {
  if (part.type === 'source' && part.sourceType === 'url') {
    console.log('ID:', part.id);
    console.log('Title:', part.title);
    console.log('URL:', part.url);
    console.log('Provider metadata:', part.providerMetadata);
    console.log();
  }
}
```

来源也可在 `result.sources` Promise 中获取。
