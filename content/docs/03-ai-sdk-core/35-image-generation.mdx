# 图像生成

AI SDK 提供 [`generateImage`](/docs/reference/ai-sdk-core/generate-image)
函数，使用图像模型根据给定提示生成图像。

```tsx
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
});
```

可使用 `base64` 或 `uint8Array` 属性访问图像数据：

```tsx
const base64 = image.base64; // base64 image data
const uint8Array = image.uint8Array; // Uint8Array image data
```

## 设置

### 尺寸与宽高比

根据模型不同，可指定尺寸或宽高比。

##### 尺寸

尺寸以 `{width}x{height}` 格式的字符串指定。
模型仅支持少数尺寸，且各模型与提供商支持的尺寸不同。

```tsx highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  size: '1024x1024',
});
```

##### 宽高比

宽高比以 `{width}:{height}` 格式的字符串指定。
模型仅支持少数宽高比，且各模型与提供商支持的宽高比不同。

```tsx highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  aspectRatio: '16:9',
});
```

### 生成多张图像

`generateImage` 也支持一次生成多张图像：

```tsx highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { images } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  n: 4, // number of images to generate
});
```

<Note>
  `generateImage` 会根据需要自动（并行）多次调用模型，以生成请求数量的图像。
</Note>

每个图像模型对单次 API 调用可生成的图像数量有内部限制。使用 `n` 参数请求多张图像时，AI SDK 会通过适当批处理自动管理这一点。默认情况下，SDK 使用提供商文档中的限制（例如 DALL-E 3 每次调用只能生成 1 张，而 DALL-E 2 最多支持 10 张）。

若需要，可在生成图像时使用 `maxImagesPerCall` 设置覆盖此行为。这在处理新的或自定义模型且默认批大小可能不是最优时尤其有用：

```tsx
const { images } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  maxImagesPerCall: 5, // Override the default batch size
  n: 10, // Will make 2 calls of 5 images each
});
```

### 提供种子

可向 `generateImage` 提供种子以控制图像生成过程的输出。
若模型支持，相同种子将始终产生相同图像。

```tsx highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  seed: 1234567890,
});
```

### 提供商特定设置

图像模型通常具有提供商甚至模型特定的设置。
可使用 `providerOptions` 参数将这些设置传给 `generateImage` 函数。
提供商选项（下例中的 `openai`）将成为请求体属性。

```tsx highlight={"12"}
import { generateImage } from 'ai';
import { openai, type OpenAIImageModelGenerationOptions } from '@ai-sdk/openai';

const { image } = await generateImage({
  model: openai.image('dall-e-3'),
  prompt: 'Santa Claus driving a Cadillac',
  size: '1024x1024',
  providerOptions: {
    openai: {
      style: 'vivid',
      quality: 'hd',
    } satisfies OpenAIImageModelGenerationOptions,
  },
});
```

### 中止信号与超时

`generateImage` 接受可选的 `abortSignal` 参数（类型为 [`AbortSignal`](https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal)），
用于中止图像生成过程或设置超时。

```ts highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  abortSignal: AbortSignal.timeout(1000), // Abort after 1 second
});
```

### 自定义头

`generateImage` 接受可选的 `headers` 参数（类型为 `Record<string, string>`），
用于向图像生成请求添加自定义头。

```ts highlight={"7"}
import { generateImage } from 'ai';
__PROVIDER_IMPORT__;

const { image } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
  headers: { 'X-Custom-Header': 'custom-value' },
});
```

### 警告

若模型返回警告（例如不支持的参数），可在响应的 `warnings` 属性中获取。

```tsx
const { image, warnings } = await generateImage({
  model: __IMAGE_MODEL__,
  prompt: 'Santa Claus driving a Cadillac',
});
```

### 额外提供商特定元数据

部分提供商为整体结果或每张图像暴露额外元数据。

```tsx
const prompt = 'Santa Claus driving a Cadillac';

const { image, providerMetadata } = await generateImage({
  model: openai.image('dall-e-3'),
  prompt,
});

const revisedPrompt = providerMetadata.openai.images[0]?.revisedPrompt;

console.log({
  prompt,
  revisedPrompt,
});
```

返回的 `providerMetadata` 的外层键为提供商名称。内层值为元数据。元数据中始终存在 `images` 键，为与顶层 `images` 键长度相同的数组。

### 错误处理

当 `generateImage` 无法生成有效图像时，会抛出 [`AI_NoImageGeneratedError`](/docs/reference/ai-sdk-errors/ai-no-image-generated-error)。

可能原因包括：

- 模型未能生成响应
- 模型生成的响应无法解析

该错误保留以下信息以帮助记录问题：

- `responses`：图像模型响应的元数据，包括时间戳、模型和头。
- `cause`：错误原因。可用于更详细的错误处理

```ts
import { generateImage, NoImageGeneratedError } from 'ai';

try {
  await generateImage({ model, prompt });
} catch (error) {
  if (NoImageGeneratedError.isInstance(error)) {
    console.log('NoImageGeneratedError');
    console.log('Cause:', error.cause);
    console.log('Responses:', error.responses);
  }
}
```

## 图像中间件

可使用 `wrapImageModel` 和 `ImageModelV4Middleware` 增强图像模型，例如设置默认值或实现日志记录。

以下示例在未提供尺寸时设置默认尺寸：

```ts
import { generateImage, wrapImageModel } from 'ai';
__PROVIDER_IMPORT__;

const model = wrapImageModel({
  model: __IMAGE_MODEL__,
  middleware: {
    specificationVersion: 'v3',
    transformParams: async ({ params }) => ({
      ...params,
      size: params.size ?? '1024x1024',
    }),
  },
});

const { image } = await generateImage({
  model,
  prompt: 'Santa Claus driving a Cadillac',
});
```

## 使用语言模型生成图像

部分语言模型（如 Google `gemini-2.5-flash-image`）支持包括图像在内的多模态输出。
使用此类模型时，可通过响应的 `files` 属性访问生成的图像。

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google('gemini-2.5-flash-image'),
  prompt: 'Generate an image of a comic cat',
});

for (const file of result.files) {
  if (file.mediaType.startsWith('image/')) {
    // The file object provides multiple data formats:
    // Access images as base64 string, Uint8Array binary data, or check type
    // - file.base64: string (data URL format)
    // - file.uint8Array: Uint8Array (binary data)
    // - file.mediaType: string (e.g. "image/png")
  }
}
```
