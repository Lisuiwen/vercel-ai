# MCP Apps

MCP Apps 在 [Model Context Protocol (MCP)](/docs/ai-sdk-core/mcp-tools) tool 之上扩展交互式 UI resource。模型仍调用普通 MCP tool，但 tool 可指向包含 HTML 的 `ui://` resource，由应用在沙箱 iframe 中渲染。

AI SDK 为构建 MCP Apps 宿主提供两部分：

- [`@ai-sdk/mcp`](/docs/reference/ai-sdk-core/mcp-apps)：声明 MCP Apps 支持、按可见性拆分 model 可见与 app 可见的 tool、读取 `ui://` resource 的辅助函数。
- [`@ai-sdk/react`](/docs/reference/ai-sdk-ui/mcp-app-renderer)：渲染 app iframe 并桥接 MCP Apps JSON-RPC 消息的 React 组件。

## 宿主流程

MCP Apps 宿主通常执行以下步骤：

1. 使用 MCP Apps 客户端能力连接 MCP 服务器。
1. 列出 tool 并按 MCP Apps 可见性拆分。
1. 仅将 model 可见的 tool 传给 `streamText` 或 `generateText`。
1. 当 tool 部分包含 MCP App 元数据时，读取 app 的 `ui://` resource。
1. 在沙箱 iframe 中渲染 HTML resource。
1. 将 iframe 允许的请求（如 app 可见的 `tools/call`）代理回 MCP 服务器。

## 启用 MCP Apps 支持连接

创建 MCP 客户端时使用 `mcpAppClientCapabilities`，声明宿主可渲染 `text/html;profile=mcp-app` resource。

```ts
import { createMCPClient, mcpAppClientCapabilities } from '@ai-sdk/mcp';
import { StreamableHTTPClientTransport } from '@modelcontextprotocol/sdk/client/streamableHttp.js';

export function createMCPAppsClient(origin: string) {
  return createMCPClient({
    transport: new StreamableHTTPClientTransport(new URL('/mcp', origin)),
    clientName: 'my-mcp-apps-host',
    capabilities: mcpAppClientCapabilities,
  });
}
```

仅当你的宿主能安全地获取并渲染 MCP App resource 时，才应声明这些能力。

## 仅暴露 Model 可见的 Tool

MCP Apps tool 可声明 `_meta.ui.visibility`。可见性为 `"model"` 的 tool 可传给模型；仅 `"app"` 的 tool 应保留给 iframe 请求，不要暴露给模型。

```ts filename="app/api/chat/route.ts"
import { splitMCPAppTools } from '@ai-sdk/mcp';
import { convertToModelMessages, streamText } from 'ai';
import { createMCPAppsClient } from './mcp-client';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  const requestUrl = new URL(req.url);
  const client = await createMCPAppsClient(requestUrl.origin);
  const { messages } = await req.json();

  try {
    const definitions = await client.listTools();
    const { modelVisible } = splitMCPAppTools(definitions);
    const tools = client.toolsFromDefinitions(modelVisible);

    const result = streamText({
      model: openai('gpt-4o-mini'),
      tools,
      messages: await convertToModelMessages(messages),
      onFinish: async () => {
        await client.close();
      },
    });

    return result.toUIMessageStreamResponse();
  } catch (error) {
    await client.close();
    throw error;
  }
}
```

当模型调用由 app 支持的 tool 时，MCP 客户端会在 tool UI 部分保留 app 元数据。React 渲染器据此判断 tool 部分是否包含 MCP App。

## 读取 App Resource

使用 `readMCPAppResource` 在发送到浏览器宿主之前读取并规范化 app resource。

```ts filename="app/api/mcp-app-host/route.ts"
import { readMCPAppResource } from '@ai-sdk/mcp';
import { createMCPAppsClient } from '../chat/mcp-client';

export async function POST(req: Request) {
  const requestUrl = new URL(req.url);
  const { uri } = await req.json();
  const client = await createMCPAppsClient(requestUrl.origin);

  try {
    return Response.json(await readMCPAppResource({ client, uri }));
  } finally {
    await client.close();
  }
}
```

`readMCPAppResource` 会校验 resource 使用 `ui://` URI、要求 MCP Apps MIME 类型、解码文本或 base64 resource 内容，并返回 HTML 及 CSP、权限等渲染元数据。

## 代理 App 可见的 Tool 调用

iframe 无法直接连接 MCP 服务器。它向宿主发送 JSON-RPC 消息，由宿主决定允许的操作。

对于 app 发起的 tool 调用，在调用 MCP 服务器前应校验请求的 tool 为 app 可见。

```ts filename="app/api/mcp-app-host/route.ts"
import { splitMCPAppTools } from '@ai-sdk/mcp';
import { createMCPAppsClient } from '../chat/mcp-client';

export async function callAppVisibleTool(req: Request) {
  const requestUrl = new URL(req.url);
  const { name, arguments: toolArguments } = await req.json();
  const client = await createMCPAppsClient(requestUrl.origin);

  try {
    const { appVisible } = splitMCPAppTools(await client.listTools());
    const isAllowed = appVisible.tools.some(tool => tool.name === name);

    if (!isAllowed) {
      return Response.json(
        { error: 'Tool is not app-visible' },
        { status: 403 },
      );
    }

    return Response.json(
      await client.callTool({
        name,
        arguments: toolArguments ?? {},
      }),
    );
  } finally {
    await client.close();
  }
}
```

生产环境中，在转发 iframe 请求前应添加应用所需的策略与用户审批检查。

## 使用 React 渲染

在 React 聊天 UI 中，照常渲染普通消息部分，并将 tool 部分传给 `experimental_MCPAppRenderer`。

<Note type="warning">
  `experimental_MCPAppRenderer` 为实验性功能，未来版本可能变更。
</Note>

```tsx filename="app/page.tsx"
'use client';

import {
  experimental_MCPAppRenderer as MCPAppRenderer,
  useChat,
  type MCPAppBridgeHandlers,
  type MCPAppMetadata,
  type MCPAppResource,
  type MCPAppSandboxConfig,
} from '@ai-sdk/react';
import { DefaultChatTransport, isToolUIPart } from 'ai';

const sandbox = {
  url: '/mcp-app-sandbox',
  className: 'h-80 w-full rounded-lg border',
  style: { border: 0 },
} satisfies MCPAppSandboxConfig;

async function loadResource(app: MCPAppMetadata): Promise<MCPAppResource> {
  const response = await fetch('/api/mcp-app-host/read-resource', {
    method: 'POST',
    body: JSON.stringify({ uri: app.resourceUri }),
  });

  if (!response.ok) {
    throw new Error('Failed to load MCP App resource');
  }

  return response.json();
}

const handlers: MCPAppBridgeHandlers = {
  callTool: params =>
    fetch('/api/mcp-app-host/call-tool', {
      method: 'POST',
      body: JSON.stringify(params),
    }).then(response => response.json()),
  openLink: ({ url }) => {
    window.open(url, '_blank', 'noopener,noreferrer');
    return {};
  },
};

export default function Chat() {
  const { messages, sendMessage } = useChat({
    transport: new DefaultChatTransport({ api: '/api/chat' }),
  });

  return (
    <>
      {messages.map(message =>
        message.parts.map((part, index) => {
          if (part.type === 'text') {
            return <div key={index}>{part.text}</div>;
          }

          if (isToolUIPart(part)) {
            return (
              <MCPAppRenderer
                key={part.toolCallId}
                part={part}
                loadResource={loadResource}
                handlers={handlers}
                sandbox={sandbox}
                fallback={<div>Loading MCP App...</div>}
              />
            );
          }

          return null;
        }),
      )}

      <button onClick={() => sendMessage({ text: 'Show me a dashboard' })}>
        Send
      </button>
    </>
  );
}
```

`experimental_MCPAppRenderer` 对普通 tool 不渲染任何内容。对由 app 支持的 tool，会加载 resource、创建沙箱桥接、向 iframe 发送 tool 输入与结果通知，并通过 handlers 转发支持的 app 请求。

## 最佳实践

- 将 MCP App HTML 视为不可信内容。在沙箱 iframe 中渲染，理想情况下通过独立源上的沙箱代理路由。
- 不要将仅 app 可见的 tool 传给模型。使用 `splitMCPAppTools` 并仅暴露 `modelVisible` tool。
- 在服务器端校验每个 iframe 请求，再调用 `client.callTool`。
- 按 `resourceUri` 缓存 app resource，避免重复 tool 调用时重复拉取相同 HTML。
- 保持 tool 的 `content` 与 `structuredContent` 在无 UI 时仍有意义，以便纯文本宿主仍可工作。
- 在响应或宿主请求结束时关闭短生命周期的 MCP 客户端。
