# 遥测

<Note type="warning">
  AI SDK 遥测为实验性功能，未来可能变更。
</Note>

AI SDK 使用 [OpenTelemetry](https://opentelemetry.io/) 收集遥测数据。
OpenTelemetry 是开源可观测性框架，旨在为收集遥测数据提供标准化插桩。

查看 [AI SDK 可观测性集成](/providers/observability)，
了解为 AI SDK 应用提供监控与追踪的 provider。

## 启用遥测

### 步骤 1：注册 OpenTelemetry 集成

OpenTelemetry span 收集需要 `@ai-sdk/otel` 包。安装并在应用启动时注册一次集成：

```bash
pnpm install @ai-sdk/otel
```

```ts
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(new OpenTelemetry());
```

Next.js 应用请放在 `instrumentation.ts` 中，与 OpenTelemetry provider 设置一并配置。provider 设置详见 [Next.js OpenTelemetry 指南](https://nextjs.org/docs/app/building-your-application/optimizing/open-telemetry)。

非 Next.js 的 Node.js 应用请在入口文件顶层注册该集成。

### 步骤 2：启用遥测

注册遥测集成后，所有 AI SDK 调用默认会发出遥测事件。
仍可通过 `telemetry` 附加元数据（如 `functionId`）或针对单次调用选择退出：

```ts highlight="4-6"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Write a short story about a cat.',
  telemetry: {
    functionId: `story-agent`,
  },
});
```

默认会记录输入与输出。可将 `recordInputs` 与 `recordOutputs` 设为 `false` 以禁用。

出于隐私、数据传输与性能等原因，可禁用输入输出记录。
例如输入含敏感信息时可关闭输入记录。

### 选择退出

遥测为可退出设计。要对单次调用禁用遥测，设置 `isEnabled: false`：

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Write a short story about a cat.',
  telemetry: { isEnabled: false },
});
```

要全局禁用遥测，请勿通过 `registerTelemetry()` 注册任何遥测集成。

## 遥测元数据

可提供 `functionId` 标识遥测数据对应的函数，
并通过 `runtimeContext` 在遥测中包含额外信息。
更广泛的上下文模型见
[运行时与 Tool 上下文](/docs/ai-sdk-core/runtime-and-tool-context)。

```ts highlight="4-10"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Write a short story about a cat.',
  runtimeContext: {
    userId: 'user_123',
    requestId: 'req_abc',
  },
  telemetry: {
    functionId: 'my-awesome-function',
  },
});
```

### 运行时上下文遥测包含

`runtimeContext` 可包含应用内有用但不应发给遥测 provider 的值，例如用户标识、租户 ID 或凭据。使用 `telemetry.includeRuntimeContext` 标记应包含在遥测中的顶层 `runtimeContext` 属性：

```ts highlight="8-10"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Write a short story about a cat.',
  runtimeContext: {
    userId: 'user_123',
    requestId: 'req_abc',
  },
  telemetry: {
    includeRuntimeContext: {
      requestId: true,
    },
  },
});
```

本例中，遥测集成收到的 `runtimeContext` 为 `{ requestId: 'req_abc' }`。设为 `false` 或省略的属性会被排除。若省略 `telemetry.includeRuntimeContext`，则不包含任何运行时上下文属性。`telemetry.includeRuntimeContext` 由 `generateText`、`streamText` 与 `ToolLoopAgent` 支持。

<Note>
  `telemetry.includeRuntimeContext` 仅过滤遥测集成（含 OpenTelemetry 集成）。
  生命周期回调与返回结果仍会收到完整的 `runtimeContext`。
</Note>

### Tool 上下文

Tool 特定上下文也可包含执行时有用的值，但不应发给遥测 provider，例如 API 密钥或访问令牌。
使用 `telemetry.includeToolsContext` 将 tool `context` 中选定的顶层属性包含进遥测：

```ts highlight="18-24"
const weatherTool = tool({
  inputSchema: z.object({
    location: z.string(),
  }),
  contextSchema: z.object({
    weatherApiKey: z.string(),
    defaultUnit: z.enum(['celsius', 'fahrenheit']),
  }),
  execute: async ({ location }, { context }) =>
    fetchWeather(location, context.weatherApiKey, context.defaultUnit),
});

const result = await generateText({
  model: __MODEL__,
  tools: { weather: weatherTool },
  toolsContext: {
    weather: {
      weatherApiKey: 'weather-123',
      defaultUnit: 'fahrenheit',
    },
  },
  prompt: 'What is the weather in San Francisco?',
  telemetry: {
    includeToolsContext: {
      weather: {
        defaultUnit: true,
      },
    },
  },
});
```

本例中，遥测集成收到的 `toolsContext.weather` 与 tool 执行遥测事件的 `toolContext` 为 `{ defaultUnit: 'fahrenheit' }`。设为 `false` 或省略的属性会被排除。若省略 `telemetry.includeToolsContext`，则不包含任何 tool 上下文属性。`telemetry.includeToolsContext` 由 `generateText`、`streamText` 与 `ToolLoopAgent` 支持。

<Note>
  `telemetry.includeToolsContext` 仅过滤遥测集成（含 OpenTelemetry 集成）。
  Tool 执行、生命周期回调与返回结果仍会收到完整 tool 上下文。tool 执行上下文与遥测事件上下文的区别见[运行时与 Tool 上下文](/docs/ai-sdk-core/runtime-and-tool-context)。
</Note>

## 自定义 Tracer

若希望 trace 使用 `@opentelemetry/api` 单例以外的 `TracerProvider`，可向 `OpenTelemetry` 构造函数传入自定义 `Tracer`：

```ts highlight="6-8"
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

const tracerProvider = new NodeTracerProvider();
registerTelemetry(
  new OpenTelemetry({
    tracer: tracerProvider.getTracer('gen_ai'),
  }),
);
```

## 遥测集成

遥测集成可挂接生成生命周期，构建自定义可观测性——日志、分析、DevTools 或其他监控系统。无需在每次调用上单独接线回调，只需实现一次 `Telemetry` 并全局注册或通过 `telemetry.integrations` 传入。

`@ai-sdk/otel` 的 `OpenTelemetry` 与 `LegacyOpenTelemetry` 是收集 OpenTelemetry span 的内置集成（见上文[启用遥测](#enabling-telemetry)）。

### 全局注册集成

使用 `registerTelemetry` 为所有 AI SDK 调用一次性注册集成：

```ts
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(new OpenTelemetry());
```

也可在一次调用中通过额外参数注册多个集成，它们会收到相同的生命周期事件：

```ts
import { registerTelemetryIntegration } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';
import { DevToolsTelemetry } from '@ai-sdk/devtools';

registerTelemetryIntegration(new OpenTelemetry(), DevToolsTelemetry());
```

### 每次调用集成

也可为单次 `generateText` 或 `streamText` 传入一个或多个集成。提供每次调用集成时，会替换该次调用全局注册的集成：

```ts highlight="6-8"
import { streamText } from 'ai';
import { DevToolsTelemetry } from '@ai-sdk/devtools';

const result = streamText({
  model: openai('gpt-4o'),
  prompt: 'Hello!',
  telemetry: {
    integrations: [DevToolsTelemetry()],
  },
});
```

可组合多个集成——它们会收到相同的生命周期事件：

```ts
telemetry: {
  integrations: [DevToolsTelemetry(), customLogger()],
},
```

集成内部错误会被捕获，不会中断生成流程。

### 诊断通道

在 Node.js 中，AI SDK 遥测生命周期事件也会发布到
`ai.telemetry` [诊断通道](https://nodejs.org/api/diagnostics_channel.html)。
可观测性 provider 可据此订阅 AI SDK 遥测事件，而无需用户再注册单独的遥测集成。

```ts
import { channel } from 'node:diagnostics_channel';
import {
  AI_SDK_TELEMETRY_DIAGNOSTIC_CHANNEL,
  type TelemetryDiagnosticChannelMessage,
} from 'ai';

channel(AI_SDK_TELEMETRY_DIAGNOSTIC_CHANNEL).subscribe(message => {
  const telemetryMessage = message as TelemetryDiagnosticChannelMessage;

  if (telemetryMessage.type === 'onStart') {
    // Inspect telemetryMessage.event and forward it to your provider.
  }
});
```

诊断通道事件遵循与其他遥测集成相同的每次调用遥测设置。
设置 `telemetry: { isEnabled: false }` 会同时禁用该次调用的已注册集成与诊断通道发布。

### 构建自定义集成

实现 `ai` 包中的 `Telemetry` 接口。所有方法均为可选——只需实现关心的生命周期事件：

```ts
import type { Telemetry } from 'ai';

class MyIntegration implements Telemetry {
  async onStart(event) {
    console.log('Generation started:', event.modelId);
  }

  async onStepFinish(event) {
    console.log(
      `Step ${event.stepNumber} done:`,
      event.usage.totalTokens,
      'tokens',
    );
  }

  async onToolExecutionEnd(event) {
    if (event.toolOutput.type === 'tool-result') {
      console.log(
        `Tool "${event.toolCall.toolName}" took ${event.toolExecutionMs}ms`,
      );
    } else {
      console.error(
        `Tool "${event.toolCall.toolName}" failed:`,
        event.toolOutput.error,
      );
    }
  }

  async onEnd(event) {
    console.log('Done. Total tokens:', event.totalUsage.totalTokens);
  }
}

export function myIntegration(): Telemetry {
  return new MyIntegration();
}
```

## 收集的数据

`@ai-sdk/otel` 提供两种发出不同 span 格式的集成。
`OpenTelemetry` 遵循 [OpenTelemetry GenAI 语义约定](https://opentelemetry.io/docs/specs/semconv/gen-ai/)，为推荐集成。
`LegacyOpenTelemetry` 集成发出旧版 AI SDK 专用 span。

### GenAI 语义约定

`OpenTelemetry` 发出的 span 遵循 [OpenTelemetry GenAI 语义约定](https://opentelemetry.io/docs/specs/semconv/gen-ai/)。
默认属性使用 `gen_ai.*` 前缀。provider 名称映射为已知值（例如 `openai`、`anthropic`、`gcp.vertex_ai`）。

#### generateText / streamText

对于 `generateText` 与 `streamText`，集成记录 3 类 span：

- **`invoke_agent {modelId}`**（根 span，`INTERNAL`）：覆盖包含所有 step 与 tool 调用的完整操作。

  初始属性：
  - `gen_ai.operation.name`: `"invoke_agent"`
  - `gen_ai.provider.name`: provider（例如 `"openai"`、`"anthropic"`）
  - `gen_ai.request.model`: 请求的模型 ID
  - `gen_ai.agent.name`: 遥测设置中的 `functionId`
  - `gen_ai.system_instructions`: 格式化为 part JSON 数组的系统 instructions（启用 `recordInputs` 时）
  - `gen_ai.input.messages`: [GenAI 语义约定消息格式](#genai-message-format) 的输入 messages（启用 `recordInputs` 时）
  - `gen_ai.request.temperature`: temperature 设置
  - `gen_ai.request.max_tokens`: 最大输出 token 数
  - `gen_ai.request.top_p`: topP 设置
  - `gen_ai.request.top_k`: topK 设置
  - `gen_ai.request.frequency_penalty`: frequency 惩罚
  - `gen_ai.request.presence_penalty`: presence 惩罚
  - `gen_ai.request.stop_sequences`: 停止序列
  - `gen_ai.request.seed`: seed 值

  完成时设置的属性：
  - `gen_ai.response.finish_reasons`: 结束原因数组（例如 `["stop"]`、`["tool_call"]`）
  - `gen_ai.usage.input_tokens`: 使用的输入 token 数
  - `gen_ai.usage.output_tokens`: 使用的输出 token 数
  - `gen_ai.usage.cache_read.input_tokens`: 读取的缓存输入 token
  - `gen_ai.usage.cache_creation.input_tokens`: 创建的缓存输入 token
  - `gen_ai.output.messages`: [GenAI 语义约定消息格式](#genai-message-format) 的输出（启用 `recordOutputs` 时）

- **`chat {modelId}`**（step span，`CLIENT`）：每次 LLM provider 调用一个 span，嵌套在根 span 下。

  初始属性：
  - `gen_ai.operation.name`: `"chat"`
  - `gen_ai.provider.name`: provider
  - `gen_ai.request.model`: 请求的模型 ID
  - `gen_ai.request.temperature`, `gen_ai.request.max_tokens`, `gen_ai.request.top_p`, `gen_ai.request.top_k`, `gen_ai.request.frequency_penalty`, `gen_ai.request.presence_penalty`, `gen_ai.request.stop_sequences`: 请求参数
  - `gen_ai.input.messages`: [GenAI 语义约定消息格式](#genai-message-format) 的提示 messages（启用 `recordInputs` 时）
  - `gen_ai.tool.definitions`: 字符串化 JSON 的 tool 定义（启用 `recordInputs` 时）

  完成时设置的属性：
  - `gen_ai.response.finish_reasons`: array of finish reasons
  - `gen_ai.response.id`: provider 返回的响应 ID
  - `gen_ai.response.model`: 生成响应的模型（可能与请求模型不同）
  - `gen_ai.usage.input_tokens`: 本 step 使用的输入 token
  - `gen_ai.usage.output_tokens`: 本 step 使用的输出 token
  - `gen_ai.usage.cache_read.input_tokens`: 读取的缓存输入 token
  - `gen_ai.usage.cache_creation.input_tokens`: 创建的缓存输入 token
  - `gen_ai.output.messages`: [GenAI 语义约定消息格式](#genai-message-format) 的输出（启用 `recordOutputs` 时）

- **`execute_tool {toolName}`**（tool span，`INTERNAL`）：每次 tool 执行一个 span，嵌套在 step span 下。详见 [GenAI tool 调用 span](#genai-tool-call-spans)。

#### 已弃用对象 API（generateObject / streamObject）

<Note type="warning">
  `generateObject` and `streamObject` are deprecated. Use `generateText` and
  `streamText` with the `output` property instead.
</Note>

已弃用对象 API 发出与 `generateText`/`streamText` 相同的 span 层次，并在根 span 上附加以下属性：

- `gen_ai.output.type`: `"json"`

step span 还包含 `gen_ai.output.type: "json"`，且 `gen_ai.output.messages` 将生成的对象作为 text 部分。

#### embed / embedMany

对于 `embed` 与 `embedMany`，集成记录 `CLIENT` 类型的 span：

- **`embeddings {modelId}`**（根 span）：覆盖完整嵌入操作。

  初始属性：
  - `gen_ai.operation.name`: `"embeddings"`
  - `gen_ai.provider.name`: provider
  - `gen_ai.request.model`: 请求的模型 ID

  完成时设置的属性：
  - `gen_ai.usage.input_tokens`: 使用的 token 数

- **`embeddings {modelId}`**（内部 span，仅 `embedMany`）：每个 provider 批调用一个 span，嵌套在根 span 下。

  初始属性：
  - `gen_ai.operation.name`: `"embeddings"`
  - `gen_ai.provider.name`: provider
  - `gen_ai.request.model`: 模型 ID

  完成时设置的属性：
  - `gen_ai.usage.input_tokens`: 使用的 token 数

#### rerank

对于 `rerank`，集成记录 `CLIENT` 类型的 span：

- **`rerank {modelId}`**（根 span）：覆盖完整重排序操作。

  初始属性：
  - `gen_ai.operation.name`: `"rerank"`
  - `gen_ai.provider.name`: provider
  - `gen_ai.request.model`: 请求的模型 ID

- **`rerank {modelId}`**（内部 span）：每次 provider 重排序调用一个 span，嵌套在根 span 下。

  初始属性：
  - `gen_ai.operation.name`: `"rerank"`
  - `gen_ai.provider.name`: provider
  - `gen_ai.request.model`: 模型 ID

#### GenAI span 详情

##### GenAI 消息格式

`gen_ai.input.messages` 与 `gen_ai.output.messages` 遵循 [OpenTelemetry GenAI 语义约定消息格式](https://opentelemetry.io/docs/specs/semconv/gen-ai/gen-ai-spans/)。
消息为 JSON 对象数组，每个对象包含 `role` 与 `parts` 数组。每个 part 有 `type` 及类型特定字段：

- `text`: `{ type: "text", content: "..." }`
- `reasoning`: `{ type: "reasoning", content: "..." }`
- `tool_call`: `{ type: "tool_call", id: "...", name: "...", arguments: ... }`
- `tool_call_response`: `{ type: "tool_call_response", id: "...", response: ... }`
- `blob`: `{ type: "blob", modality: "image"|"video"|"audio", mime_type: "...", content: "..." }` (base64-encoded)
- `uri`: `{ type: "uri", modality: "image"|"video"|"audio", mime_type: "...", uri: "..." }` (for URL-based files)

输出消息还包含 `finish_reason` 字段（例如 `"stop"`、`"tool_call"`、`"length"`、`"content_filter"`）。

系统 instructions 单独记录在 `gen_ai.system_instructions` 中，为 `{ type: "text", content: "..." }` part 的 JSON 数组。

##### GenAI tool 调用 span

Tool 调用 span（`execute_tool {toolName}`）嵌套在 step span 下，包含：

- `gen_ai.operation.name`: `"execute_tool"`
- `gen_ai.tool.name`: tool 名称
- `gen_ai.tool.call.id`: tool 调用 ID
- `gen_ai.tool.type`: `"function"`
- `gen_ai.tool.call.arguments`: 输入参数（字符串化 JSON，启用 `recordInputs` 时）
- `gen_ai.tool.call.result`: 输出结果（字符串化 JSON，启用 `recordOutputs` 时）。仅在 tool 调用成功时设置。

### 自定义 OpenTelemetry span 属性

使用 `enrichSpan` 为 `OpenTelemetry` 集成创建的 span 添加自定义属性。当可观测性后端需要非 AI SDK 自有语义的厂商特定属性时很有用。

```ts
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(
  new OpenTelemetry({
    enrichSpan: ({ spanType, operationId, callId, runtimeContext }) => {
      return {
        ...getCustomAttributes(runtimeContext, spanType),
        'my_app.operation_id': operationId,
        'my_app.call_id': callId,
      };
    },
  }),
);
```

回调在每个 span 创建时运行，接收：

- `spanType`：正在创建的 span 类型（`operation`、`step`、`languageModel`、`tool`、`embedding` 或 `reranking`）。
- `operationId`：当前调用的 AI SDK 操作 ID，例如 `ai.generateText` 或 `ai.streamText`。
- `callId`：当前 AI SDK 调用的唯一 ID。
- `runtimeContext`：文本生成 span 的遥测过滤后 runtimeContext，包含来自 `prepareStep` 的更新。

Custom attributes are merged with AI SDK attributes on the span. AI SDK-owned
attributes take precedence when a custom attribute uses the same key.

`sensitiveRuntimeContext` is applied before telemetry integrations receive
`runtimeContext`, so custom span enrichment does not receive top-level runtime
context properties marked as sensitive.

### OpenTelemetry span 上的补充 AI SDK 属性

GenAI 语义约定覆盖核心模型、提示、响应、tool 调用与用量数据。对于 GenAI 语义未表示的数据，可选择启用额外 AI SDK 特定属性：

```ts
import { registerTelemetry } from 'ai';
import { OpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(
  new OpenTelemetry({
    usage: true,
    providerMetadata: true,
    runtimeContext: true,
  }),
);
```

这不会创建额外的 AI SDK 专用 span（如 `ai.generateText`、`ai.generateText.doGenerate`、`ai.toolCall`），而仅为 `OpenTelemetry` 发出的 span（如 `invoke_agent {modelId}`、`step {n}`、`chat {modelId}`、`execute_tool {toolName}`）添加所选补充属性。

所有补充属性默认禁用。仅启用需要收集的数据：

```ts
registerTelemetry(
  new OpenTelemetry({
    usage: true,
    providerMetadata: true,
    embedding: true,
    reranking: true,
    runtimeContext: true,
    headers: true,
    toolChoice: true,
    schema: true,
  }),
);
```

可用选项：

- `usage`: GenAI 用量属性未覆盖的详细用量属性，例如未缓存输入 token 与输出 text/reasoning token 详情。
- `providerMetadata`：`ai.response.providerMetadata`。
- `embedding`: 嵌入输入与输出。
- `reranking`: 重排序输入文档与排序输出。
- `runtimeContext`：`ai.settings.context.*`。
- `headers`：`ai.request.headers.*`。
- `toolChoice`：`ai.prompt.toolChoice`。
- `schema`: 对象生成 schema 与输出模式属性。

补充输入输出属性仍遵循 `recordInputs` 与 `recordOutputs` 遥测选项。

### 旧版 AI SDK Span（`LegacyOpenTelemetry`）

`LegacyOpenTelemetry` 集成使用 AI SDK 专用 `ai.*` 前缀属性发出 span。
此为旧版格式。建议迁移到 `OpenTelemetry` 以获得更好的可观测性平台兼容性。

#### generateText 函数

`generateText` records 3 types of spans:

- `ai.generateText`（span）：generateText 调用的完整时长。包含 1 个或多个 `ai.generateText.doGenerate` span。
  包含[基本 LLM span 信息](#basic-llm-span-information)及以下属性：
  - `operation.name`：`ai.generateText` 及通过 `telemetry.functionId` 设置的 functionId
  - `ai.operationId`: `"ai.generateText"`
  - `ai.prompt`：调用 `generateText` 时使用的提示
  - `ai.response.text`：生成的文本
  - `ai.response.toolCalls`：生成过程中进行的 tool 调用（字符串化 JSON）
  - `ai.response.finishReason`：生成结束的原因
  - `ai.settings.maxOutputTokens`：设置的最大输出 token 数

- `ai.generateText.doGenerate`（span）：provider 的 doGenerate 调用。可包含 `ai.toolCall` span。
  包含[调用 LLM span 信息](#call-llm-span-information)及以下属性：
  - `operation.name`: `ai.generateText.doGenerate` and the functionId that was set through `telemetry.functionId`
  - `ai.operationId`: `"ai.generateText.doGenerate"`
  - `ai.prompt.messages`：传入 provider 的 messages
  - `ai.prompt.tools`：字符串化 tool 定义数组。序列化 tool 可为 `function` 或 `provider` 类型。
    函数 tool 包含 `FunctionTool` 与 `DynamicTool` 定义，具有 `name`、`description`（可选）与 `inputSchema`（JSON schema）。
    Provider tool 包含 `ProviderDefinedTool` 与 `ProviderExecutedTool` 定义，具有 `name`、`id` 与 `args`（Record）。
  - `ai.prompt.toolChoice`：字符串化的 tool 选择设置（JSON）。具有 `type` 属性
    （`auto`、`none`、`required`、`tool`），若类型为 `tool`，则有带具体 tool 的 `toolName` 属性。
  - `ai.response.text`：生成的文本
  - `ai.response.toolCalls`：生成过程中进行的 tool 调用（字符串化 JSON）
  - `ai.response.finishReason`：生成结束的原因

- `ai.toolCall`（span）：generateText 调用中进行的 tool 调用。详见[旧版 tool 调用 span](#legacy-tool-call-spans)。

#### streamText 函数

`streamText` records 3 types of spans and 2 types of events:

- `ai.streamText`（span）：streamText 调用的完整时长。包含一个 `ai.streamText.doStream` span。
  包含[基本 LLM span 信息](#basic-llm-span-information)及以下属性：
  - `operation.name`: `ai.streamText` and the functionId that was set through `telemetry.functionId`
  - `ai.operationId`: `"ai.streamText"`
  - `ai.prompt`：调用 `streamText` 时使用的提示
  - `ai.response.text`：生成的文本
  - `ai.response.toolCalls`：生成过程中进行的 tool 调用（字符串化 JSON）
  - `ai.response.finishReason`：生成结束的原因
  - `ai.settings.maxOutputTokens`：设置的最大输出 token 数

- `ai.streamText.doStream`（span）：provider 的 doStream 调用。
  此 span 包含 `ai.stream.firstChunk` 事件与 `ai.toolCall` span。
  包含[调用 LLM span 信息](#call-llm-span-information)及以下属性：
  - `operation.name`: `ai.streamText.doStream` and the functionId that was set through `telemetry.functionId`
  - `ai.operationId`: `"ai.streamText.doStream"`
  - `ai.prompt.messages`：传入 provider 的 messages
  - `ai.prompt.tools`：字符串化 tool 定义数组。序列化 tool 可为 `function` 或 `provider` 类型。
    函数 tool 包含 `FunctionTool` 与 `DynamicTool` 定义，具有 `name`、`description`（可选）与 `inputSchema`（JSON schema）。
    Provider tool 包含 `ProviderDefinedTool` 与 `ProviderExecutedTool` 定义，具有 `name`、`id` 与 `args`（Record）。
  - `ai.prompt.toolChoice`：字符串化的 tool 选择设置（JSON）。具有 `type` 属性
    （`auto`、`none`、`required`、`tool`），若类型为 `tool`，则有带具体 tool 的 `toolName` 属性。
  - `ai.response.text`：生成的文本
  - `ai.response.toolCalls`：生成过程中进行的 tool 调用（字符串化 JSON）
  - `ai.response.msToFirstChunk`：收到第一个分块的耗时（毫秒）
  - `ai.response.msToFinish`：收到 LLM 流 finish 部分的耗时（毫秒）
  - `ai.response.avgCompletionTokensPerSecond`：每秒平均补全 token 数
  - `ai.response.finishReason`：生成结束的原因

- `ai.toolCall`（span）：streamText 调用中进行的 tool 调用。详见[旧版 tool 调用 span](#legacy-tool-call-spans)。

- `ai.stream.firstChunk`（事件）：收到流第一个分块时发出的事件。
  - `ai.response.msToFirstChunk`：收到第一个分块的耗时

- `ai.stream.finish`（事件）：收到 LLM 流 finish 部分时发出的事件。

#### Deprecated object APIs

<Note type="warning">
  `generateObject` and `streamObject` are deprecated. Use `generateText` and
  `streamText` with the `output` property instead.
</Note>

若仍运行已弃用对象 API，会看到旧版 span 名称：

- `generateObject`: `ai.generateObject`, `ai.generateObject.doGenerate`
- `streamObject`: `ai.streamObject`, `ai.streamObject.doStream`, `ai.stream.firstChunk`

Legacy object spans include the same core metadata as other LLM spans, plus
object-specific attributes such as `ai.schema.*`, `ai.response.object`, and
`ai.settings.output`.

#### 旧版 span 详情

##### 基本 LLM span 信息

使用 LLM 的许多 span（`ai.generateText`、`ai.generateText.doGenerate`、`ai.streamText`、`ai.streamText.doStream`）包含以下属性：

- `resource.name`：通过 `telemetry.functionId` 设置的 functionId
- `ai.model.id`：模型 id
- `ai.model.provider`：模型的 provider
- `ai.request.headers.*`：通过 `headers` 传入的请求头
- `ai.response.providerMetadata`：生成响应返回的 provider 特定元数据
- `ai.settings.maxRetries`：设置的最大重试次数
- `ai.telemetry.functionId`：通过 `telemetry.functionId` 设置的 functionId
- `ai.settings.runtimeContext.*`：通过 `runtimeContext` 选项传入的运行时上下文，配置时过滤为 `telemetry.includeRuntimeContext` 标记的顶层属性
- `ai.usage.completionTokens`：使用的补全 token 数
- `ai.usage.promptTokens`：使用的提示 token 数

##### 调用 LLM span 信息

Spans that correspond to individual LLM calls (`ai.generateText.doGenerate`, `ai.streamText.doStream`) contain
[basic LLM span information](#basic-llm-span-information) and the following attributes:

- `ai.response.model`：用于生成响应的模型。若 provider 支持别名，可能与请求的模型不同。
- `ai.response.id`：响应 id。可用时使用 provider 的 ID。
- `ai.response.timestamp`：响应时间戳。可用时使用 provider 的时间戳。
- [Semantic Conventions for GenAI operations](https://opentelemetry.io/docs/specs/semconv/gen-ai/gen-ai-spans/)
  - `gen_ai.system`：使用的 provider
  - `gen_ai.request.model`：请求的模型
  - `gen_ai.request.temperature`：设置的 temperature
  - `gen_ai.request.max_tokens`：设置的最大 token 数
  - `gen_ai.request.frequency_penalty`: frequency 惩罚 that was set
  - `gen_ai.request.presence_penalty`: presence 惩罚 that was set
  - `gen_ai.request.top_k`：设置的 topK 参数值
  - `gen_ai.request.top_p`：设置的 topP 参数值
  - `gen_ai.request.stop_sequences`: 停止序列
  - `gen_ai.response.finish_reasons`：provider 返回的结束原因
  - `gen_ai.response.model`：用于生成响应的模型。若 provider 支持别名，可能与请求的模型不同。
  - `gen_ai.response.id`：响应 id。可用时使用 provider 的 ID。
  - `gen_ai.usage.input_tokens`：使用的提示 token 数
  - `gen_ai.usage.output_tokens`：使用的补全 token 数

##### 基本嵌入 span 信息

使用嵌入模型的许多 span（`ai.embed`、`ai.embed.doEmbed`、`ai.embedMany`、`ai.embedMany.doEmbed`）包含以下属性：

- `ai.model.id`：模型 id
- `ai.model.provider`：模型的 provider
- `ai.request.headers.*`：通过 `headers` 传入的请求头
- `ai.settings.maxRetries`：设置的最大重试次数
- `ai.telemetry.functionId`：通过 `telemetry.functionId` 设置的 functionId
- `ai.settings.runtimeContext.*`：通过 `runtimeContext` 选项传入的运行时上下文
- `ai.usage.tokens`：使用的 token 数
- `resource.name`：通过 `telemetry.functionId` 设置的 functionId

##### 旧版 tool 调用 span

Tool 调用 span（`ai.toolCall`）包含以下属性：

- `operation.name`: `"ai.toolCall"`
- `ai.operationId`: `"ai.toolCall"`
- `ai.toolCall.name`: tool 名称
- `ai.toolCall.id`：tool 调用 id
- `ai.toolCall.args`：tool 调用的输入参数
- `ai.toolCall.result`：tool 调用的输出结果。仅在调用成功且结果可序列化时可用。
