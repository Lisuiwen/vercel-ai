# 推理

许多语言模型在生成响应前支持内部的「推理」阶段（有时也称为「思考」）。AI SDK 在 [`generateText`](/docs/reference/ai-sdk-core/generate-text) 和 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 上提供顶层 `reasoning` 参数，通过单一、可移植的设置跨提供商控制该行为。

## 基本用法

```ts
import { generateText } from 'ai';

const { text, reasoning, reasoningText } = await generateText({
  model: 'anthropic/claude-sonnet-4.6,
  reasoning: 'medium',
  prompt: 'How many people will live in the world in 2040?',
});
```

`reasoning` 参数接受以下值：

| 值                   | 行为                                                                 |
| -------------------- | -------------------------------------------------------------------- |
| `'provider-default'` | 使用提供商默认推理行为（省略时的默认值）                             |
| `'none'`             | 禁用推理                                                             |
| `'minimal'`          | 最低限度推理                                                         |
| `'low'`              | 快速、简洁推理                                                       |
| `'medium'`           | 平衡推理                                                             |
| `'high'`             | 深入推理                                                             |
| `'xhigh'`            | 最大推理                                                             |

## 流式传输

`reasoning` 参数在 `streamText` 中的用法相同：

```ts
import { streamText } from 'ai';

const result = streamText({
  model: 'google/gemini-3-flash-preview',
  reasoning: 'high',
  prompt: 'Explain the Riemann hypothesis in simple terms.',
});

for await (const part of result.fullStream) {
  if (part.type === 'reasoning') {
    process.stdout.write(part.textDelta);
  } else if (part.type === 'text-delta') {
    process.stdout.write(part.textDelta);
  }
}
```

## 优先级规则

顶层 `reasoning` 参数与提供商特定的 `providerOptions` **永不合并**。若在 `providerOptions` 中设置了与推理相关的选项，它们将完全优先，顶层 `reasoning` 参数会被忽略。

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const { text } = await generateText({
  model: openai.responses('gpt-5.4'),
  reasoning: 'low', // ignored because providerOptions.openai.reasoningEffort is set
  providerOptions: {
    openai: {
      reasoningEffort: 'high', // this wins
    },
  },
  prompt: 'Explain quantum entanglement.',
});
```

该设计让你默认使用可移植的 `reasoning` 参数，仅在需要提供商特定功能（如精确 token 预算）时才回退到 `providerOptions`。

## 提供商支持

以下提供商支持 `reasoning` 参数：OpenAI、Anthropic、Google、xAI、Groq、DeepSeek、Fireworks 和 Amazon Bedrock。各提供商将该值转换为其原生推理 API。部分提供商原生支持全部六个级别，其他则强制映射到更少级别（发生强制映射时会发出警告）。部分提供商使用数值 token 预算而非枚举控制推理；此时顶层 `reasoning` 值会按模型最大输出 token 的百分比映射为预算。

不支持推理的提供商（例如 Mistral、Perplexity、Cohere）会发出 `unsupported` 警告并忽略该参数。

## 从 `providerOptions` 迁移

若当前通过 `providerOptions` 控制推理，可迁移到顶层 `reasoning` 参数以实现跨提供商可移植性。

### 之前（Anthropic）

```ts
const { text } = await generateText({
  model: anthropic('claude-opus-4.6'),
  providerOptions: {
    anthropic: {
      thinking: { type: 'adaptive', effort: 'high' },
    },
  },
  prompt: 'How many people will live in the world in 2040?',
});
```

### 之后（Anthropic）

```ts
const { text } = await generateText({
  model: anthropic('claude-opus-4.6'),
  reasoning: 'high',
  prompt: 'How many people will live in the world in 2040?',
});
```

### 之前（较旧的 Anthropic 模型）

```ts
const { text } = await generateText({
  model: anthropic('claude-sonnet-4-20250514'),
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    },
  },
  prompt: 'How many people will live in the world in 2040?',
});
```

### 之后（较旧的 Anthropic 模型）

```ts
const { text } = await generateText({
  model: anthropic('claude-sonnet-4-20250514'),
  reasoning: 'medium',
  prompt: 'How many people will live in the world in 2040?',
});
```

若需要强制精确 token 预算（例如恰好 12000 token），请继续使用 `providerOptions`，而非顶层 `reasoning` 参数。

### 之前（带 `includeThoughts` 的 Google）

```ts
const { text } = await generateText({
  model: google('gemini-3-flash-preview'),
  providerOptions: {
    google: {
      thinkingConfig: { thinkingBudget: 4096, includeThoughts: true },
    },
  },
  prompt: 'Explain the Riemann hypothesis in simple terms.',
});
```

### 之后（带 `includeThoughts` 的 Google）

```ts
const { text } = await generateText({
  model: google('gemini-3-flash-preview'),
  reasoning: 'medium',
  providerOptions: {
    google: { thinkingConfig: { includeThoughts: true } },
  },
  prompt: 'Explain the Riemann hypothesis in simple terms.',
});
```

### 之前（带 `reasoningSummary` 的 OpenAI）

```ts
const { text } = await generateText({
  model: openai.responses('o3'),
  providerOptions: {
    openai: { reasoningEffort: 'high', reasoningSummary: 'auto' },
  },
  prompt: 'Explain quantum entanglement.',
});
```

### 之后（带 `reasoningSummary` 的 OpenAI）

```ts
const { text } = await generateText({
  model: openai.responses('o3'),
  reasoning: 'high',
  providerOptions: {
    openai: { reasoningSummary: 'auto' },
  },
  prompt: 'Explain quantum entanglement.',
});
```

请注意，`providerOptions` 仍可与 `reasoning` 一起用于与推理力度无关的提供商特定功能。但若 `providerOptions` 包含推理力度/预算设置（例如 `reasoningEffort`、`thinking`、`thinkingConfig.thinkingBudget`），这些将完全优先，顶层 `reasoning` 参数会被忽略。
