# 语言模型中间件

语言模型中间件通过拦截并修改对语言模型的调用，
以增强语言模型行为。

可用于以与语言模型无关的方式添加护栏、RAG、缓存和日志等功能。
此类中间件可独立于其所应用的语言模型进行开发与分发。

## 使用语言模型中间件

可使用 `wrapLanguageModel` 函数配合语言模型中间件。
它接收语言模型与语言模型中间件，返回融合了中间件的新语言模型。

```ts
import { wrapLanguageModel, streamText } from 'ai';

const wrappedLanguageModel = wrapLanguageModel({
  model: yourModel,
  middleware: yourLanguageModelMiddleware,
});
```

包装后的语言模型可像其他语言模型一样使用，例如在 `streamText` 中：

```ts highlight="2"
const result = streamText({
  model: wrappedLanguageModel,
  prompt: 'What cities are in the United States?',
});
```

## 多个中间件

可向 `wrapLanguageModel` 传入多个中间件。
中间件按提供顺序依次应用。

```ts
const wrappedLanguageModel = wrapLanguageModel({
  model: yourModel,
  middleware: [firstMiddleware, secondMiddleware],
});

// applied as: firstMiddleware(secondMiddleware(yourModel))
```

## 内置中间件

AI SDK 提供多种内置中间件，用于配置语言模型：

- `extractReasoningMiddleware`：从生成文本中提取推理信息，并在结果上暴露为 `reasoning` 属性。
- `extractJsonMiddleware`：通过剥离 markdown 代码围栏从文本内容中提取 JSON。适用于将 JSON 响应包在代码块中的模型配合 `Output.object()` 使用。
- `simulateStreamingMiddleware`：用非流式语言模型的响应模拟流式行为。
- `defaultSettingsMiddleware`：为语言模型应用默认设置。
- `addToolInputExamplesMiddleware`：为不支持原生 `inputExamples` 属性的 provider 将 tool 输入示例加入 tool 描述。

### 提取推理

部分 provider 与模型在生成文本中使用特殊标签暴露推理信息，
例如 &lt;think&gt; 与 &lt;/think&gt;。

可使用 `extractReasoningMiddleware` 提取该推理信息，并在结果上暴露为 `reasoning` 属性。

```ts
import { wrapLanguageModel, extractReasoningMiddleware } from 'ai';

const model = wrapLanguageModel({
  model: yourModel,
  middleware: extractReasoningMiddleware({ tagName: 'think' }),
});
```

随后可在 `generateText`、`streamText` 等函数中使用增强后的模型。

`extractReasoningMiddleware` 还提供 `startWithReasoning` 选项。
设为 `true` 时，推理标签会前置到生成文本。
适用于响应开头不包含推理标签的模型。
更多细节见 [DeepSeek R1 指南](/cookbook/guides/r1#deepseek-r1-middleware)。

### 提取 JSON

部分模型即使用户请求结构化输出，仍会用 markdown 代码围栏包裹 JSON（例如 ` ```json ... ``` `）。

`extractJsonMiddleware` 会剥离这些围栏，使响应与 `Output.object()` 兼容。

```ts
import {
  wrapLanguageModel,
  extractJsonMiddleware,
  Output,
  generateText,
} from 'ai';
import { z } from 'zod';

const model = wrapLanguageModel({
  model: yourModel,
  middleware: extractJsonMiddleware(),
});

const result = await generateText({
  model,
  output: Output.object({
    schema: z.object({
      name: z.string(),
      ingredients: z.array(z.string()),
    }),
  }),
  prompt: 'Generate a recipe.',
});
```

也可为使用不同格式的模型提供自定义 transform 函数：

```ts
const model = wrapLanguageModel({
  model: yourModel,
  middleware: extractJsonMiddleware({
    transform: text => text.replace(/^PREFIX/, '').replace(/SUFFIX$/, ''),
  }),
});
```

### 模拟流式

`simulateStreamingMiddleware` 可用非流式语言模型的响应模拟流式行为。
适用于希望保持统一流式接口、但模型仅返回完整响应的场景。

```ts
import { wrapLanguageModel, simulateStreamingMiddleware } from 'ai';

const model = wrapLanguageModel({
  model: yourModel,
  middleware: simulateStreamingMiddleware(),
});
```

### 默认设置

`defaultSettingsMiddleware` 可为语言模型应用默认设置。

```ts
import { wrapLanguageModel, defaultSettingsMiddleware } from 'ai';

const model = wrapLanguageModel({
  model: yourModel,
  middleware: defaultSettingsMiddleware({
    settings: {
      temperature: 0.5,
      maxOutputTokens: 800,
      providerOptions: { openai: { store: false } },
    },
  }),
});
```

### 添加 Tool 输入示例

`addToolInputExamplesMiddleware` 将 tool 输入示例加入 tool 描述。
适用于 tool 上不支持原生 `inputExamples` 的 provider。
中间件将示例序列化进 tool 描述文本，使模型仍能参考示例输入。

```ts
import { wrapLanguageModel, addToolInputExamplesMiddleware } from 'ai';

const model = wrapLanguageModel({
  model: yourModel,
  middleware: addToolInputExamplesMiddleware({
    prefix: 'Input Examples:',
  }),
});
```

定义带 `inputExamples` 的 tool 时，中间件会把示例追加到 tool 描述：

```ts
import { generateText, tool } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model, // wrapped model from above
  tools: {
    weather: tool({
      description: 'Get the weather in a location',
      inputSchema: z.object({
        location: z.string(),
      }),
      inputExamples: [
        { input: { location: 'San Francisco' } },
        { input: { location: 'London' } },
      ],
    }),
  },
  prompt: 'What is the weather in Tokyo?',
});
```

tool 描述将变为：

```
Get the weather in a location

Input Examples:
{"location":"San Francisco"}
{"location":"London"}
```

#### 选项

- `prefix`（可选）：示例前的前缀文本。默认：`'Input Examples:'`。
- `format`（可选）：每个示例的自定义格式化函数，接收示例对象与索引。默认：`JSON.stringify(example.input)`。
- `remove`（可选）：加入描述后是否从 tool 移除 `inputExamples`。默认：`true`。

```ts
const model = wrapLanguageModel({
  model: yourModel,
  middleware: addToolInputExamplesMiddleware({
    prefix: 'Input Examples:',
    format: (example, index) =>
      `${index + 1}. ${JSON.stringify(example.input)}`,
    remove: true,
  }),
});
```

## 社区中间件

AI SDK 提供语言模型中间件规范。社区可开发符合该规范的中间件，与 AI SDK 生态兼容。

可探索的社区中间件示例：

### 自定义 tool 调用解析器

[Custom tool call parser](https://github.com/minpeter/ai-sdk-tool-call-middleware) 中间件为不支持 OpenAI 风格 `tools` 参数的模型扩展 tool 调用能力，包括许多缺少原生函数调用能力的自托管与第三方模型。

<Note>
  在支持原生函数调用的模型上使用此中间件可能导致意外性能下降，使用前请确认模型是否支持原生函数调用。
</Note>

该中间件将函数 schema 转为提示指令，并把模型响应解析为结构化函数调用：把 JSON 函数定义转为模型可理解的自然语言指令，再分析生成文本提取函数调用尝试。开发者可在不同 provider 上使用同一函数调用 API，即使模型不支持 OpenAI 风格函数调用格式，也能获得一致的调用体验。

`@ai-sdk-tool/parser` 提供三种中间件变体：

- `createToolMiddleware`：为特定模型创建自定义 tool 调用中间件的灵活函数
- `hermesToolMiddleware`：适用于 Hermes 与 Qwen 格式函数调用的即用中间件
- `gemmaToolMiddleware`：为 Gemma 3 系列函数调用格式预配置的中间件

以下展示如何为原生不支持函数调用的 Gemma 模型启用函数调用：

```ts
import { wrapLanguageModel } from 'ai';
import { gemmaToolMiddleware } from '@ai-sdk-tool/parser';

const model = wrapLanguageModel({
  model: openrouter('google/gemma-3-27b-it'),
  middleware: gemmaToolMiddleware,
});
```

更多示例见此[链接](https://github.com/minpeter/ai-sdk-tool-call-middleware/tree/main/examples/core/src)。

## 实现语言模型中间件

<Note>
  实现语言模型中间件属于高级功能，需要深入理解[语言模型规范](https://github.com/vercel/ai/blob/v5/packages/provider/src/language-model/v2/language-model-v2.ts)。
</Note>

可实现以下三种函数之一以修改语言模型行为：

1. `transformParams`：在传给语言模型前转换参数，适用于 `doGenerate` 与 `doStream`。
2. `wrapGenerate`：包装[语言模型](https://github.com/vercel/ai/blob/v5/packages/provider/src/language-model/v2/language-model-v2.ts)的 `doGenerate` 方法。可修改参数、调用语言模型并修改结果。
3. `wrapStream`：包装[语言模型](https://github.com/vercel/ai/blob/v5/packages/provider/src/language-model/v2/language-model-v2.ts)的 `doStream` 方法。可修改参数、调用语言模型并修改结果。

以下是实现语言模型中间件的示例：

## 示例

<Note>
  这些示例并非用于生产，仅演示如何用中间件增强语言模型行为。
</Note>

### 日志

本示例展示如何记录语言模型调用的参数与生成文本。

```ts
import type {
  LanguageModelV4Middleware,
  LanguageModelV4StreamPart,
} from '@ai-sdk/provider';

export const yourLogMiddleware: LanguageModelV4Middleware = {
  wrapGenerate: async ({ doGenerate, params }) => {
    console.log('doGenerate called');
    console.log(`params: ${JSON.stringify(params, null, 2)}`);

    const result = await doGenerate();

    console.log('doGenerate finished');
    console.log(`generated text: ${result.text}`);

    return result;
  },

  wrapStream: async ({ doStream, params }) => {
    console.log('doStream called');
    console.log(`params: ${JSON.stringify(params, null, 2)}`);

    const { stream, ...rest } = await doStream();

    let generatedText = '';
    const textBlocks = new Map<string, string>();

    const transformStream = new TransformStream<
      LanguageModelV4StreamPart,
      LanguageModelV4StreamPart
    >({
      transform(chunk, controller) {
        switch (chunk.type) {
          case 'text-start': {
            textBlocks.set(chunk.id, '');
            break;
          }
          case 'text-delta': {
            const existing = textBlocks.get(chunk.id) || '';
            textBlocks.set(chunk.id, existing + chunk.delta);
            generatedText += chunk.delta;
            break;
          }
          case 'text-end': {
            console.log(
              `Text block ${chunk.id} completed:`,
              textBlocks.get(chunk.id),
            );
            break;
          }
        }

        controller.enqueue(chunk);
      },

      flush() {
        console.log('doStream finished');
        console.log(`generated text: ${generatedText}`);
      },
    });

    return {
      stream: stream.pipeThrough(transformStream),
      ...rest,
    };
  },
};
```

### 缓存

本示例展示如何为语言模型调用的生成文本构建简单缓存。

```ts
import type { LanguageModelV4Middleware } from '@ai-sdk/provider';

const cache = new Map<string, any>();

export const yourCacheMiddleware: LanguageModelV4Middleware = {
  wrapGenerate: async ({ doGenerate, params }) => {
    const cacheKey = JSON.stringify(params);

    if (cache.has(cacheKey)) {
      return cache.get(cacheKey);
    }

    const result = await doGenerate();

    cache.set(cacheKey, result);

    return result;
  },

  // here you would implement the caching logic for streaming
};
```

### 检索增强生成（RAG）

本示例展示如何将 RAG 用作中间件。

<Note>
  `getLastUserMessageText`、`findSources` 等辅助函数不属于 AI SDK，仅用于说明 RAG 概念。
</Note>

```ts
import type { LanguageModelV4Middleware } from '@ai-sdk/provider';

export const yourRagMiddleware: LanguageModelV4Middleware = {
  transformParams: async ({ params }) => {
    const lastUserMessageText = getLastUserMessageText({
      prompt: params.prompt,
    });

    if (lastUserMessageText == null) {
      return params; // do not use RAG (send unmodified parameters)
    }

    const instruction =
      'Use the following information to answer the question:\n' +
      findSources({ text: lastUserMessageText })
        .map(chunk => JSON.stringify(chunk))
        .join('\n');

    return addToLastUserMessage({ params, text: instruction });
  },
};
```

### 护栏

护栏用于确保语言模型调用的生成文本安全、恰当。本示例展示如何将护栏用作中间件。

```ts
import type { LanguageModelV4Middleware } from '@ai-sdk/provider';

export const yourGuardrailMiddleware: LanguageModelV4Middleware = {
  wrapGenerate: async ({ doGenerate }) => {
    const { text, ...rest } = await doGenerate();

    // filtering approach, e.g. for PII or other sensitive information:
    const cleanedText = text?.replace(/badword/g, '<REDACTED>');

    return { text: cleanedText, ...rest };
  },

  // here you would implement the guardrail logic for streaming
  // Note: streaming guardrails are difficult to implement, because
  // you do not know the full content of the stream until it's finished.
};
```

## 配置每次请求的自定义元数据

在中间件中发送与访问自定义元数据可使用 `providerOptions`。构建日志中间件时，可传入用户 ID、时间戳等有助于追踪与调试的上下文。

```ts
import { generateText, wrapLanguageModel } from 'ai';
__PROVIDER_IMPORT__;
import type { LanguageModelV4Middleware } from '@ai-sdk/provider';

export const yourLogMiddleware: LanguageModelV4Middleware = {
  wrapGenerate: async ({ doGenerate, params }) => {
    console.log('METADATA', params?.providerMetadata?.yourLogMiddleware);
    const result = await doGenerate();
    return result;
  },
};

const { text } = await generateText({
  model: wrapLanguageModel({
    model: __MODEL__,
    middleware: yourLogMiddleware,
  }),
  prompt: 'Invent a new holiday and describe its traditions.',
  providerOptions: {
    yourLogMiddleware: {
      hello: 'world',
    },
  },
});

console.log(text);
```
