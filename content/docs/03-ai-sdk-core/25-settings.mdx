# 设置

大语言模型（LLM）通常提供设置以调整其输出。

除模型、[提示](./prompts)及额外提供商特定设置外，所有 AI SDK 函数均支持以下通用设置：

```ts highlight="3-6"
const result = await generateText({
  model: __MODEL__,
  maxOutputTokens: 512,
  temperature: 0.3,
  maxRetries: 5,
  timeout: 10000,
  prompt: 'Invent a new holiday and describe its traditions.',
});
```

<Note>
  部分提供商不支持全部通用设置。若在不支持某设置的提供商上使用，将生成警告。可检查结果对象的 `warnings` 属性查看是否生成了警告。
</Note>

## 语言模型调用选项

语言模型调用选项（`LanguageModelCallOptions`）是影响语言模型如何生成响应的设置——token 限制、采样行为、惩罚、停止序列、种子与推理。它们会转发给底层模型。

### `maxOutputTokens`

要生成的最大 token 数。

### `temperature`

温度设置。

该值会传递给提供商。范围取决于提供商与模型。
对大多数提供商，`0` 表示几乎确定性的结果，较高值表示更多随机性。

建议只设置 `temperature` 或 `topP` 之一，不要同时设置。

<Note>在 AI SDK 5.0 中，temperature 不再默认为 `0`。</Note>

### `topP`

核采样（Nucleus sampling）。

该值会传递给提供商。范围取决于提供商与模型。
对大多数提供商，核采样为 0 到 1 之间的数。
例如 0.1 表示仅考虑概率质量前 10% 的 token。

建议只设置 `temperature` 或 `topP` 之一，不要同时设置。

### `topK`

仅从前 K 个选项中为每个后续 token 采样。

用于去除低概率的「长尾」响应。
仅建议高级用例使用。通常只需使用 `temperature`。

### `presencePenalty`

存在惩罚影响模型重复提示中已有信息的倾向。

该值会传递给提供商。范围取决于提供商与模型。
对大多数提供商，`0` 表示无惩罚。

### `frequencyPenalty`

频率惩罚影响模型反复使用相同词语或短语的倾向。

该值会传递给提供商。范围取决于提供商与模型。
对大多数提供商，`0` 表示无惩罚。

### `stopSequences`

用于停止文本生成的停止序列。

若设置，当生成某个停止序列时模型将停止生成文本。
提供商可能对停止序列数量有限制。

### `seed`

用于随机采样的种子（整数）。
若设置且模型支持，调用将产生确定性结果。

### `reasoning`

控制模型在生成响应前进行多少推理。

| 值                   | 行为                                                                 |
| -------------------- | -------------------------------------------------------------------- |
| `'provider-default'` | 使用提供商默认推理行为（省略时的默认值）                             |
| `'none'`             | 禁用推理                                                             |
| `'minimal'`          | 最低限度推理                                                         |
| `'low'`              | 快速、简洁推理                                                       |
| `'medium'`           | 平衡推理                                                             |
| `'high'`             | 深入推理                                                             |
| `'xhigh'`            | 最大推理                                                             |

若在 `providerOptions` 中也设置了与推理相关的选项（例如 `openai.reasoningEffort` 或 `anthropic.thinking`），提供商特定选项将优先，顶层 `reasoning` 参数会被忽略。

有关各提供商映射及从 `providerOptions` 迁移的详情，请参阅[推理指南](/docs/ai-sdk-core/reasoning)。

## 请求选项

请求选项（`RequestOptions`）影响传输、重试、取消与超时——而非模型生成行为。它们控制 SDK 如何与提供商 API 通信。

### `maxRetries`

最大重试次数。设为 `0` 可禁用重试。默认：`2`。

### `abortSignal`

可选的中止信号，用于取消调用。

例如可从用户界面转发中止信号以取消调用，
或使用 `AbortSignal.timeout` 定义超时。

#### 示例：AbortSignal.timeout

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  abortSignal: AbortSignal.timeout(5000), // 5 seconds
});
```

### `timeout`

可选的超时（毫秒）。若调用超过指定时长将被中止。

这是便捷参数，会在内部创建中止信号。可与 `abortSignal` 一起使用——若两者都提供，满足任一条件即中止。

可将超时指定为数字（毫秒）或包含以下属性的对象：

- `totalMs`：整个调用（含所有步骤）的总超时。
- `stepMs`：每个单独步骤（LLM 调用）的超时。适用于多步生成，可独立限制每步耗时。
- `chunkMs`：流块之间的超时（仅流式）。若在此时间内未收到新块则中止。适用于检测停滞的流。
- `toolMs`：所有工具执行的默认超时。若工具耗时更长，将中止并返回 tool-error，以便模型响应或重试。
- `tools`：使用 `{toolName}Ms` 键（例如 `weatherMs`、`slowApiMs`）的按工具超时覆盖。优先于 `toolMs`。工具名称支持自动补全类型检查。

#### 示例：5 秒超时（数字格式）

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  timeout: 5000, // 5 seconds
});
```

#### 示例：5 秒总超时（对象格式）

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  timeout: { totalMs: 5000 }, // 5 seconds
});
```

#### 示例：10 秒步骤超时

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  timeout: { stepMs: 10000 }, // 10 seconds per step
});
```

#### 示例：组合总超时与步骤超时

```ts
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  timeout: {
    totalMs: 60000, // 60 seconds total
    stepMs: 10000, // 10 seconds per step
  },
});
```

#### 示例：流式按块超时（仅 streamText）

```ts
const result = streamText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  timeout: { chunkMs: 5000 }, // abort if no chunk received for 5 seconds
});
```

#### 示例：工具执行超时

```ts
const result = await generateText({
  model: __MODEL__,
  tools: { weather: weatherTool, slowApi: slowApiTool },
  timeout: {
    toolMs: 5000, // 5 seconds default for all tools
  },
  prompt: 'What is the weather in San Francisco?',
});
```

#### 示例：按工具超时覆盖

```ts
const result = await generateText({
  model: __MODEL__,
  tools: { weather: weatherTool, slowApi: slowApiTool },
  timeout: {
    toolMs: 5000, // default for all tools
    tools: {
      weatherMs: 3000, // 3 seconds for weather tool
      slowApiMs: 10000, // 10 seconds for slow API tool
    },
  },
  prompt: 'What is the weather in San Francisco?',
});
```

### `headers`

随请求发送的额外 HTTP 头。仅适用于基于 HTTP 的提供商。

可根据提供商支持情况使用请求头向提供商提供额外信息。
例如，部分可观测性提供商支持 `Prompt-Id` 等头。

```ts
import { generateText } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
  headers: {
    'Prompt-Id': 'my-prompt-id',
  },
});
```

<Note>
  `headers` 设置用于请求特定的头。也可在提供商配置中设置 `headers`。这些头将随提供商的每次请求发送。
</Note>
