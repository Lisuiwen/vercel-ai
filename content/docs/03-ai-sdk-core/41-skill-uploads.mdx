# Skill 上传

AI SDK 提供 [`uploadSkill`](/docs/reference/ai-sdk-core/upload-skill)
函数，用于将自定义 skill 上传到提供商并获取可传给后续推理调用的 `ProviderReference`。

**Skill** 是一组文件（例如描述 skill 行为的 `SKILL.md`），
提供商可加载它们，例如在沙箱容器环境中。

在 AI SDK 中，已上传 skill 由 `ProviderReference` 标识——
将提供商名称映射到提供商特定标识符的 `Record<string, string>`。
该概念也用于其他提供商特定资产引用，例如已上传的媒体文件。

```ts
import { uploadSkill, generateText } from 'ai';
import {
  anthropic,
  type AnthropicLanguageModelOptions,
} from '@ai-sdk/anthropic';
import { readFileSync } from 'fs';

const { providerReference } = await uploadSkill({
  api: anthropic.skills(),
  files: [
    {
      path: 'my-skill/SKILL.md',
      content: readFileSync('./SKILL.md'),
    },
  ],
  displayTitle: 'My Skill',
});

const { text } = await generateText({
  model: anthropic('claude-sonnet-4-6'),
  tools: {
    code_execution: anthropic.tools.codeExecution_20260120(),
  },
  prompt: 'Use the skill to complete the task.',
  providerOptions: {
    anthropic: {
      container: {
        skills: [{ type: 'custom', providerReference }],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

作为简写，可将提供商实例直接传给 `api`，而无需显式调用 `.skills()`——SDK 会为你调用 `.skills()`：

```ts highlight="2"
const { providerReference } = await uploadSkill({
  api: anthropic, // shorthand for anthropic.skills()
  files: [{ path: 'my-skill/SKILL.md', content: readFileSync('./SKILL.md') }],
  displayTitle: 'My Skill',
});
```

## Skill 文件

Skill 由一个或多个文件组成，每个文件具有相对 `path` 和 `content`。
文件内容可作为 `Uint8Array`（例如来自 `fs.readFileSync`）或 base64 编码字符串提供：

```ts
const { providerReference } = await uploadSkill({
  api: openai.skills(),
  files: [
    {
      path: 'my-skill/SKILL.md',
      content: readFileSync('./SKILL.md'), // Uint8Array
    },
    {
      path: 'my-skill/helper.py',
      content: readFileSync('./helper.py'),
    },
  ],
});
```

## 上传结果

`uploadSkill` 返回包含以下字段的 `UploadSkillResult`：

| 字段                | 类型                | 说明                                                             |
| ------------------- | ------------------- | ---------------------------------------------------------------- |
| `providerReference` | `ProviderReference` | 将提供商名称映射到提供商特定 skill ID                            |
| `displayTitle`      | `string?`           | 人类可读标题（若支持且已提供）                                   |
| `name`              | `string?`           | 提供商从 skill 文件推断的名称                                    |
| `description`       | `string?`           | 提供商从 skill 文件推断的描述                                    |
| `latestVersion`     | `string?`           | 提供商分配的最新版本标识符                                       |
| `providerMetadata`  | `object?`           | 额外提供商特定元数据（例如时间戳）                               |
| `warnings`          | `Warning[]`         | 不支持选项的警告（例如在 OpenAI 上使用 `displayTitle`）          |

## 提供商引用

`ProviderReference` 是将提供商名称映射到提供商特定 skill 标识符的 `Record<string, string>`：

```ts
// Example ProviderReference
{
  anthropic: 'skill_abc123',
}
```

在推理时引用 skill 时传入 `providerReference`。各提供商从引用中查找自己的 skill ID。若当前提供商没有对应条目，将抛出错误。

## 多提供商用法

若要在多个提供商间使用同一 skill，请分别上传并合并引用：

```ts
const [openaiUpload, anthropicUpload] = await Promise.all([
  uploadSkill({
    api: openai.skills(),
    files: [{ path: 'my-skill/SKILL.md', content: skillSource }],
  }),
  uploadSkill({
    api: anthropic.skills(),
    files: [{ path: 'my-skill/SKILL.md', content: skillSource }],
    displayTitle: 'My Skill',
  }),
]);

const mergedReference = {
  ...openaiUpload.providerReference,
  ...anthropicUpload.providerReference,
};

// mergedReference: { openai: 'sk_...', anthropic: 'sk_...' }
```

合并后的引用可在推理调用中使用，无论由哪个提供商处理请求——各提供商都会找到自己的 skill ID。

## 在推理调用中使用 Skill

如何将 skill 附加到推理调用取决于提供商。

### Anthropic

在 `providerOptions` 的 `container.skills` 数组中传入 `providerReference`：

```ts
await generateText({
  model: anthropic('claude-sonnet-4-6'),
  tools: {
    code_execution: anthropic.tools.codeExecution_20260120(),
  },
  prompt: '...',
  providerOptions: {
    anthropic: {
      container: {
        skills: [{ type: 'custom', providerReference }],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

### OpenAI

在 `shell` 工具的 `environment.skills` 数组中传入 `providerReference`：

```ts
await generateText({
  model: openai.responses('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      environment: {
        type: 'containerAuto',
        skills: [{ type: 'skillReference', providerReference }],
      },
    }),
  },
  prompt: '...',
});
```

## 支持的提供商

以下提供商支持 `skills()` 与 skill 上传：

| 提供商    | 工厂方法             |
| --------- | -------------------- |
| Anthropic | `anthropic.skills()` |
| OpenAI    | `openai.skills()`    |
