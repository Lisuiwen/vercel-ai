# 将 AI SDK 4.1 迁移到 4.2

<Note>
  更多信息请参阅 [AI SDK 4.2 发布公告](https://vercel.com/blog/ai-sdk-4-2)。
</Note>

本指南帮助你升级到 AI SDK 4.2：

## 稳定 API

下列 API 已晋升为稳定 API，不再使用 `experimental_` 前缀：

- `customProvider`
- `providerOptions`（由面向提供商特定输入的 `providerMetadata` 重命名而来）
- `providerMetadata`（面向提供商特定输出）
- `streamText` 的 `toolCallStreaming` 选项

## 依赖版本

AI SDK 需要 **非可选** 的 `zod` 依赖，版本为 `^3.23.8`。

## UI 消息部件（Parts）

在 AI SDK 4.2 中，我们重新设计了 `useChat` 如何通过消息部件与多步流程呈现模型输出。
这是一次重大改进，让你在 UI 中更简单地渲染复杂的多模态 AI 响应。

### 变更内容

带有工具调用的助手消息现在会合并为单条包含多个部件的消息，而不是为每一步单独创建消息。
这一调整回应了 AI 应用中的两个关键趋势：

1. **多样化的输出类型**：模型不再只生成文本，还会产生推理步骤、来源信息与工具调用。
2. **交错输出**：在多步 agent 场景中，这些不同类型的输出常常交错出现。

### 新方案的优势

以前，`useChat` 将不同类型的输出分别存储，这使得在 UI 中保持正确顺序变得困难，
并且在存在工具调用时会出现多条连续的助手消息。例如：

```javascript
message.content = "Final answer: 42";
message.reasoning = "First I'll calculate X, then Y...";
message.toolInvocations = [{toolName: "calculator", args: {...}}];
```

这种结构存在局限。新的消息部件方案用有序数组取代分散字段，从而精确保留顺序：

```javascript
message.parts = [
  { type: "text", text: "Final answer: 42" },
  { type: "reasoning", reasoning: "First I'll calculate X, then Y..." },
  { type: "tool-invocation", toolInvocation: { toolName: "calculator", args: {...} } },
];
```

### 迁移

仍在使用旧消息格式的应用需要更新 UI 组件以支持新的 `parts` 数组。
旧字段仍为向后兼容而保留，但我们建议迁移到新格式，以获得更好的多模态与多步交互支持。

你可以像下面这样在新消息部件上使用 `useChat`：

```javascript
function Chat() {
  const { messages } = useChat();
  return (
    <div>
      {messages.map(message =>
        message.parts.map((part, i) => {
          switch (part.type) {
            case 'text':
              return <p key={i}>{part.text}</p>;
            case 'source':
              return <p key={i}>{part.source.url}</p>;
            case 'reasoning':
              return <div key={i}>{part.reasoning}</div>;
            case 'tool-invocation':
              return <div key={i}>{part.toolInvocation.toolName}</div>;
            case 'file':
              return (
                <img
                  key={i}
                  src={`data:${part.mediaType};base64,${part.data}`}
                />
              );
          }
        }),
      )}
    </div>
  );
}
```
