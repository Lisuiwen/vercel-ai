# 将 AI SDK 5.x 迁移到 6.0

## 建议的迁移流程

1. 备份项目。若使用版本控制系统，请确保历史版本均已提交。
1. 升级到 AI SDK 6.0。
1. 遵循下方的破坏性变更指南。
1. 验证项目按预期工作。
1. 提交你的更改。

## AI SDK 6.0 包版本

你需要将下列包在 `package.json`（或多份 `package.json`）中更新到最新版本：

- `ai`：`^6.0.0`
- `@ai-sdk/provider`：`^3.0.0`
- `@ai-sdk/provider-utils`：`^4.0.0`
- `@ai-sdk/*`：`^3.0.0`

示例升级命令：

```
pnpm install ai@latest @ai-sdk/react@latest @ai-sdk/openai@latest
```

## Codemods

当某项功能被弃用、移除或以其他方式变更时，AI SDK 提供 Codemod 转换来帮助升级代码库。

Codemods 是会在你的代码库上自动运行的转换，让你无需逐文件手动修改即可批量应用变更。

你可以在项目根目录运行以下命令执行全部 v6 codemods（v5 → v6 迁移）：

```sh
npx @ai-sdk/codemod v6
```

<Note>
  也可以使用 `npx @ai-sdk/codemod upgrade`，但它会运行所有版本的 codemods（v4、v5、v6）。
  若从 v5 升级，请使用 `v6`。
</Note>

若要单独运行某个 codemod，请指定其名称：

```sh
npx @ai-sdk/codemod <codemod-name> <path>
```

例如运行某个 v6 codemod：

```sh
npx @ai-sdk/codemod v6/rename-text-embedding-to-embedding src/
```

<Note>
  Codemods 旨在辅助升级流程，未必覆盖你所需的全部变更；你可能仍需手动完成额外修改。
</Note>

## Codemod 对照表

| Codemod 名称                                             | 说明                                                                                        |
| -------------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| `rename-text-embedding-to-embedding`                     | 将各 provider 上的 `textEmbeddingModel` 重命名为 `embeddingModel`，`textEmbedding` 重命名为 `embedding`   |
| `rename-mock-v2-to-v3`                                   | 将 `ai/test` 中的 V2 mock 类重命名为 V3（例如 `MockLanguageModelV2` → `MockLanguageModelV3`） |
| `rename-tool-call-options-to-tool-execution-options`     | 将 `ToolCallOptions` 类型重命名为 `ToolExecutionOptions`                                       |
| `rename-core-message-to-model-message`                   | 将 `CoreMessage` 类型重命名为 `ModelMessage`                                                   |
| `rename-converttocoremessages-to-converttomodelmessages` | 将 `convertToCoreMessages` 函数重命名为 `convertToModelMessages`                               |
| `rename-vertex-provider-metadata-key`                    | 在 Google Vertex 相关文件中，将 `providerMetadata` 与 `providerOptions` 里的 `google` 键改为 `vertex`   |
| `wrap-tomodeloutput-parameter`                           | 将 `toModelOutput` 的参数改为对象解构（`output` → `{ output }`）                  |
| `add-await-converttomodelmessages`                       | 为 `convertToModelMessages` 调用添加 `await`（AI SDK 6 中已为异步）                             |

## AI SDK Core

### `Experimental_Agent` → `ToolLoopAgent` 类

`Experimental_Agent` 已由 `ToolLoopAgent` 取代。关键变化有两处：

1. `system` 参数重命名为 `instructions`
2. 默认 `stopWhen` 由 `isStepCount(1)` 改为 `isStepCount(20)`

```tsx filename="AI SDK 5"
import { Experimental_Agent as Agent, isStepCount } from 'ai';
__PROVIDER_IMPORT__;

const agent = new Agent({
  model: __MODEL__,
  system: 'You are a helpful assistant.',
  tools: {
    // your tools here
  },
  stopWhen: isStepCount(20), // Required for multi-step agent loops
});

const result = await agent.generate({
  prompt: 'What is the weather in San Francisco?',
});
```

```tsx filename="AI SDK 6"
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
  tools: {
    // your tools here
  },
  // stopWhen defaults to isStepCount(20)
});

const result = await agent.generate({
  prompt: 'What is the weather in San Francisco?',
});
```

进一步了解如何 [构建 Agents](/docs/agents/building-agents)。

### 移除 `CoreMessage`

已弃用的 `CoreMessage` 类型及相关函数已移除（[PR #10710](https://github.com/vercel/ai/pull/10710)）。请将 `convertToCoreMessages` 替换为 `convertToModelMessages`。

```tsx filename="AI SDK 5"
import { convertToCoreMessages, type CoreMessage } from 'ai';

const coreMessages = convertToCoreMessages(messages); // CoreMessage[]
```

```tsx filename="AI SDK 6"
import { convertToModelMessages, type ModelMessage } from 'ai';

const modelMessages = await convertToModelMessages(messages); // ModelMessage[]
```

<Note>
  可使用 `rename-core-message-to-model-message` 与
  `rename-converttocoremessages-to-converttomodelmessages` codemods 自动更新代码库。
</Note>

### `generateObject` 与 `streamObject` 弃用

`generateObject` 与 `streamObject` 已弃用（[PR #10754](https://github.com/vercel/ai/pull/10754)），并将在未来版本中移除。
请改用带 `output` 配置的 `generateText` 与 `streamText`。

```tsx filename="AI SDK 5"
import { generateObject } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { object } = await generateObject({
  model: __MODEL__,
  schema: z.object({
    recipe: z.object({
      name: z.string(),
      ingredients: z.array(z.object({ name: z.string(), amount: z.string() })),
      steps: z.array(z.string()),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});
```

```tsx filename="AI SDK 6"
import { generateText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { output } = await generateText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({ name: z.string(), amount: z.string() }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});
```

若要流式生成结构化数据，请将 `streamObject` 替换为 `streamText`：

```tsx filename="AI SDK 5"
import { streamObject } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { partialObjectStream } = streamObject({
  model: __MODEL__,
  schema: z.object({
    recipe: z.object({
      name: z.string(),
      ingredients: z.array(z.object({ name: z.string(), amount: z.string() })),
      steps: z.array(z.string()),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});

for await (const partialObject of partialObjectStream) {
  console.log(partialObject);
}
```

```tsx filename="AI SDK 6"
import { streamText, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const { partialOutputStream } = streamText({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(
          z.object({ name: z.string(), amount: z.string() }),
        ),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});

for await (const partialObject of partialOutputStream) {
  console.log(partialObject);
}
```

进一步了解如何 [生成结构化数据](/docs/ai-sdk-core/generating-structured-data)。

### 异步 `convertToModelMessages`

为支持异步 `Tool.toModelOutput()`，`convertToModelMessages()` 在 AI SDK 6 中为异步函数。

```tsx filename="AI SDK 5"
import { convertToModelMessages } from 'ai';

const modelMessages = convertToModelMessages(uiMessages);
```

```tsx filename="AI SDK 6"
import { convertToModelMessages } from 'ai';

const modelMessages = await convertToModelMessages(uiMessages);
```

<Note>
  使用 `add-await-converttomodelmessages` codemod 可自动更新代码库。
</Note>

### `Tool.toModelOutput` 变更

在 AI SDK 6 中，`toModelOutput()` 接收包含 `output` 属性的参数对象。

在 AI SDK 5 中，参数本身就是 `output`。

```tsx filename="AI SDK 5"
import { tool } from 'ai';

const someTool = tool({
  // ...
  toModelOutput: output => {
    // ...
  },
});
```

```tsx filename="AI SDK 6"
import { tool } from 'ai';

const someTool = tool({
  // ...
  toModelOutput: ({ output }) => {
    // ...
  },
});
```

<Note>
  使用 `wrap-tomodeloutput-parameter` codemod 可自动更新代码库。
</Note>

### `LanguageModelUsage` 中弃用 `cachedInputTokens` 与 `reasoningTokens`

`LanguageModelUsage` 中的 `cachedInputTokens` 与 `reasoningTokens` 已弃用。

可将 `cachedInputTokens` 替换为 `inputTokenDetails.cacheReadTokens`，
将 `reasoningTokens` 替换为 `outputTokenDetails.reasoningTokens`。

### `ToolCallOptions` → `ToolExecutionOptions` 重命名

`ToolCallOptions` 类型已重命名为 `ToolExecutionOptions`，
且本身也已标记为弃用。

<Note>
  使用 `rename-tool-call-options-to-tool-execution-options` codemod 可自动更新代码库。
</Note>

### 按工具的 Strict 模式

工具 strict 模式现在通过在单个工具上设置 `strict` 来控制（[PR #10817](https://github.com/vercel/ai/pull/10817)）。
这样可以精细控制 strict 工具调用，因为 strict 行为取决于具体工具的输入 schema。

```tsx filename="AI SDK 5"
__PROVIDER_IMPORT__;
import { streamText, tool } from 'ai';
import { z } from 'zod';

// Tool strict mode was controlled by strictJsonSchema
const result = streamText({
  model: __MODEL__,
  tools: {
    calculator: tool({
      description: 'A simple calculator',
      inputSchema: z.object({
        expression: z.string(),
      }),
      execute: async ({ expression }) => {
        const result = eval(expression);
        return { result };
      },
    }),
  },
  providerOptions: {
    openai: {
      strictJsonSchema: true, // Applied to all tools
    },
  },
});
```

```tsx filename="AI SDK 6"
__PROVIDER_IMPORT__;
import { streamText, tool } from 'ai';
import { z } from 'zod';

const result = streamText({
  model: __MODEL__,
  tools: {
    calculator: tool({
      description: 'A simple calculator',
      inputSchema: z.object({
        expression: z.string(),
      }),
      execute: async ({ expression }) => {
        const result = eval(expression);
        return { result };
      },
      strict: true, // Control strict mode per tool
    }),
  },
});
```

### 灵活的工具内容

AI SDK 6 引入了更灵活的工具输出与结果内容支持（[PR #9605](https://github.com/vercel/ai/pull/9605)），支持更丰富的工具交互与复杂的工具执行模式。

### `ToolCallRepairFunction` 签名

`ToolCallRepairFunction` 类型中的 `system` 参数除 `string` 外，现在也接受 `SystemModelMessage`（[PR #10635](https://github.com/vercel/ai/pull/10635)）。
这允许更灵活的系统消息配置，包括缓存等提供商特定选项。

```tsx filename="AI SDK 5"
import type { ToolCallRepairFunction } from 'ai';

const repairToolCall: ToolCallRepairFunction<MyTools> = async ({
  system, // type: string | undefined
  messages,
  toolCall,
  tools,
  inputSchema,
  error,
}) => {
  // ...
};
```

```tsx filename="AI SDK 6"
import type { ToolCallRepairFunction, SystemModelMessage } from 'ai';

const repairToolCall: ToolCallRepairFunction<MyTools> = async ({
  system, // type: string | SystemModelMessage | undefined
  messages,
  toolCall,
  tools,
  inputSchema,
  error,
}) => {
  // Handle both string and SystemModelMessage
  const systemText = typeof system === 'string' ? system : system?.content;
  // ...
};
```

### 嵌入模型方法重命名

各 provider 上的 `textEmbeddingModel` 与 `textEmbedding` 已分别重命名为 `embeddingModel` 与 `embedding`。
此外，`EmbeddingModel`、`embed`、`embedMany` 上的泛型也已移除（[PR #10592](https://github.com/vercel/ai/pull/10592)）。

```tsx filename="AI SDK 5"
import { openai } from '@ai-sdk/openai';
import { embed } from 'ai';

// Using the full method name
const model = openai.textEmbeddingModel('text-embedding-3-small');

// Using the shorthand
const model = openai.textEmbedding('text-embedding-3-small');

const { embedding } = await embed({
  model: openai.textEmbedding('text-embedding-3-small'),
  value: 'sunny day at the beach',
});
```

```tsx filename="AI SDK 6"
import { openai } from '@ai-sdk/openai';
import { embed } from 'ai';

// Using the full method name
const model = openai.embeddingModel('text-embedding-3-small');

// Using the shorthand
const model = openai.embedding('text-embedding-3-small');

const { embedding } = await embed({
  model: openai.embedding('text-embedding-3-small'),
  value: 'sunny day at the beach',
});
```

<Note>
  使用 `rename-text-embedding-to-embedding` codemod 可自动更新代码库。
</Note>

### 警告日志记录器

AI SDK 6 引入警告日志记录器，用于输出弃用警告与最佳实践建议（[PR #8343](https://github.com/vercel/ai/pull/8343)）。

若要禁用警告日志，请将环境变量 `AI_SDK_LOG_WARNINGS` 设为 `false`：

```bash
export AI_SDK_LOG_WARNINGS=false
```

### 警告类型统一

各生成函数各自的警告类型已合并为从 `ai` 包导出的单一 `Warning` 类型（[PR #10631](https://github.com/vercel/ai/pull/10631)）。

```tsx filename="AI SDK 5"
// Separate warning types for each generation function
import type {
  CallWarning,
  ImageModelCallWarning,
  SpeechWarning,
  TranscriptionWarning,
} from 'ai';
```

```tsx filename="AI SDK 6"
// Single Warning type for all generation functions
import type { Warning } from 'ai';
```

### 结束原因：`unknown` 并入 `other`

`unknown` 结束原因已移除，现在会返回 `other`。

## AI SDK UI

### 工具 UI 部件辅助函数重命名

工具 UI 部件辅助函数已重命名，以更贴合用途，并同时涵盖静态与动态工具部件（[PR #XXXX](https://github.com/vercel/ai/pull/XXXX)）。

#### `isToolUIPart` → `isStaticToolUIPart`

`isToolUIPart` 已重命名为 `isStaticToolUIPart`，明确表示仅检查静态工具部件。

```tsx filename="AI SDK 5"
import { isToolUIPart } from 'ai';

// Check if a part is a tool UI part
if (isToolUIPart(part)) {
  console.log(part.toolName);
}
```

```tsx filename="AI SDK 6"
import { isStaticToolUIPart } from 'ai';

// Check if a part is a static tool UI part
if (isStaticToolUIPart(part)) {
  console.log(part.toolName);
}
```

#### `isToolOrDynamicToolUIPart` → `isToolUIPart`

`isToolOrDynamicToolUIPart` 已重命名为 `isToolUIPart`。旧名称仍可用但已弃用。

```tsx filename="AI SDK 5"
import { isToolOrDynamicToolUIPart } from 'ai';

// Check if a part is either a static or dynamic tool UI part
if (isToolOrDynamicToolUIPart(part)) {
  console.log('Tool part found');
}
```

```tsx filename="AI SDK 6"
import { isToolUIPart } from 'ai';

// Check if a part is either a static or dynamic tool UI part
if (isToolUIPart(part)) {
  console.log('Tool part found');
}
```

#### `getToolName` → `getStaticToolName`

`getToolName` 已重命名为 `getStaticToolName`，明确表示仅从静态工具部件读取名称。

```tsx filename="AI SDK 5"
import { getToolName } from 'ai';

// Get the tool name from a tool part
const name = getToolName(toolPart);
```

```tsx filename="AI SDK 6"
import { getStaticToolName } from 'ai';

// Get the tool name from a static tool part
const name = getStaticToolName(toolPart);
```

#### `getToolOrDynamicToolName` → `getToolName`

`getToolOrDynamicToolName` 已重命名为 `getToolName`。旧名称仍可用但已弃用。

```tsx filename="AI SDK 5"
import { getToolOrDynamicToolName } from 'ai';

// Get the tool name from either a static or dynamic tool part
const name = getToolOrDynamicToolName(toolPart);
```

```tsx filename="AI SDK 6"
import { getToolName } from 'ai';

// Get the tool name from either a static or dynamic tool part
const name = getToolName(toolPart);
```

## Providers

### OpenAI

#### `strictJsonSchema` 默认为 true

面向 JSON 输出与工具调用的 `strictJsonSchema` 默认开启（[PR #10752](https://github.com/vercel/ai/pull/10752)），可提升稳定性并确保输出 JSON 符合 schema。

但 strict 模式对 schema 要求更严格。若遇到 schema 被拒错误，请调整 schema（例如用 `null` 代替 `undefined`），或关闭 strict 模式。

```tsx filename="AI SDK 5"
import { openai } from '@ai-sdk/openai';
import { generateObject } from 'ai';
import { z } from 'zod';

// strictJsonSchema was false by default
const result = await generateObject({
  model: openai('gpt-5.1'),
  schema: z.object({
    name: z.string(),
  }),
  prompt: 'Generate a person',
});
```

```tsx filename="AI SDK 6"
import { openai } from '@ai-sdk/openai';
import { generateObject } from 'ai';
import { z } from 'zod';

// strictJsonSchema is true by default
const result = await generateObject({
  model: openai('gpt-5.1'),
  schema: z.object({
    name: z.string(),
  }),
  prompt: 'Generate a person',
});

// Disable strict mode if needed
const resultNoStrict = await generateObject({
  model: openai('gpt-5.1'),
  schema: z.object({
    name: z.string(),
  }),
  prompt: 'Generate a person',
  providerOptions: {
    openai: {
      strictJsonSchema: false,
    } satisfies OpenAIResponsesProviderOptions,
  },
});
```

#### 聊天模型移除 `structuredOutputs` 选项

聊天模型的 `structuredOutputs` provider 选项已移除（[PR #10752](https://github.com/vercel/ai/pull/10752)）。请改用 `strictJsonSchema`。

### Azure

#### 默认 Provider 使用 Responses API

调用 `azure()` 时，`@ai-sdk/azure` provider 现在默认使用 Responses API（[PR #9868](https://github.com/vercel/ai/pull/9868)）。
若要沿用此前的 Chat Completions API 行为，请使用 `azure.chat()`。

```tsx filename="AI SDK 5"
import { azure } from '@ai-sdk/azure';

// Used Chat Completions API
const model = azure('gpt-4o');
```

```tsx filename="AI SDK 6"
import { azure } from '@ai-sdk/azure';

// Now uses Responses API by default
const model = azure('gpt-4o');

// Use azure.chat() for Chat Completions API
const chatModel = azure.chat('gpt-4o');

// Use azure.responses() explicitly for Responses API
const responsesModel = azure.responses('gpt-4o');
```

<Note>
  Responses API 与 Chat Completions API 的行为与默认值不同。
  如果你依赖 Chat Completions API，请将模型实例切换到 `azure.chat()`，并审查相关配置。
</Note>

#### Responses API 下的 `providerMetadata` 与 `providerOptions` 键名

对 **Responses API** 而言，`@ai-sdk/azure` provider 现在在 `providerMetadata` 与 `providerOptions` 中使用 `azure` 键，而不再是 `openai`。输入侧的 `providerOptions` 仍可使用 `openai` 键，但输出的 `providerMetadata` 会使用 `azure`。

```tsx filename="AI SDK 5"
import { azure } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure.responses('gpt-5-mini'), // use your own deployment
  prompt: 'Hello',
  providerOptions: {
    openai: {
      // AI SDK 5: use `openai` key for Responses API options
      reasoningSummary: 'auto',
    },
  },
});

// Accessed metadata via 'openai' key
console.log(result.providerMetadata?.openai?.responseId);
```

```tsx filename="AI SDK 6"
import { azure } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  // azure() now uses the Responses API by default
  model: azure('gpt-5-mini'), // use your own deployment
  prompt: 'Hello',
  providerOptions: {
    azure: {
      // AI SDK 6: use `azure` key for Responses API options
      reasoningSummary: 'auto',
    },
  },
});

// Access metadata via 'azure' key
console.log(result.providerMetadata?.azure?.responseId);
```

### Anthropic

#### Structured Outputs 模式

Anthropic 已为 [ Claude Sonnet 4.5 及更高版本模型引入原生结构化输出 ](https://www.claude.com/blog/structured-outputs-on-the-claude-developer-platform)。`@ai-sdk/anthropic` provider 现已包含 `structuredOutputMode` 选项，用于控制结构化输出的生成方式（[PR #10502](https://github.com/vercel/ai/pull/10502)）。

可用模式如下：

- `'outputFormat'`：使用 Anthropic 原生的 `output_format` 参数
- `'jsonTool'`：使用特殊的 JSON 工具来指定结构化输出格式
- `'auto'`（默认）：模型支持时使用 `'outputFormat'`，否则回退到 `'jsonTool'`

```tsx filename="AI SDK 6"
import { anthropic } from '@ai-sdk/anthropic';
import { generateObject } from 'ai';
import { z } from 'zod';

const result = await generateObject({
  model: anthropic('claude-sonnet-4-5-20250929'),
  schema: z.object({
    name: z.string(),
    age: z.number(),
  }),
  prompt: 'Generate a person',
  providerOptions: {
    anthropic: {
      // Explicitly set the structured output mode (optional)
      structuredOutputMode: 'outputFormat',
    } satisfies AnthropicProviderOptions,
  },
});
```

### Google Vertex

#### `providerMetadata` 与 `providerOptions` 键名

`@ai-sdk/google-vertex` provider 现在在 `providerMetadata` 与 `providerOptions` 中使用 `vertex` 键，而不再是 `google`。输入侧的 `providerOptions` 仍可使用 `google` 键，但输出的 `providerMetadata` 会使用 `vertex`。

```tsx filename="AI SDK 5"
import { vertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: vertex('gemini-2.5-flash'),
  providerOptions: {
    google: {
      safetySettings: [
        /* ... */
      ],
    }, // Used 'google' key
  },
  prompt: 'Hello',
});

// Accessed metadata via 'google' key
console.log(result.providerMetadata?.google?.safetyRatings);
```

```tsx filename="AI SDK 6"
import { vertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: vertex('gemini-2.5-flash'),
  providerOptions: {
    vertex: {
      safetySettings: [
```
