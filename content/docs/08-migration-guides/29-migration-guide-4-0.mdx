# 将 AI SDK 3.4 迁移到 4.0

<Note>
  更多信息请参阅 [AI SDK 4.0 发布公告](https://vercel.com/blog/ai-sdk-4-0)。
</Note>

## 建议的迁移流程

1. 备份项目。若使用版本控制系统，请确保历史版本均已提交。
1. [迁移到 AI SDK 3.4](/docs/migration-guides/migration-guide-3-4)。
1. 升级到 AI SDK 4.0。
1. 使用 [codemods](#codemods) 自动迁移代码。
   > 若不想使用 codemods，建议在升级到 AI SDK 4.0 前先处理所有弃用警告。
1. 遵循下方的破坏性变更指南。
1. 验证项目按预期工作。
1. 提交你的更改。

## AI SDK 4.0 包版本

你需要将下列包在 `package.json`（或多份 `package.json`）中更新到指定版本：

- `ai`：`4.0.*`
- `ai-sdk@provider-utils`：`2.0.*`
- `ai-sdk/*`：`1.0.*`（其他 `@ai-sdk` 包）

## Codemods

当某项功能被弃用、移除或以其他方式变更时，AI SDK 提供 Codemod 转换来帮助升级代码库。

Codemods 会以程序化方式在你的代码库上运行，
让你无需逐文件手动修改即可批量应用变更。

<Note>
  Codemods 旨在辅助升级流程，未必覆盖你所需的全部变更；你可能仍需手动完成额外修改。
</Note>

你可以在项目根目录运行以下命令，执行 4.0 升级相关的全部 codemods：

```sh
npx @ai-sdk/codemod upgrade
```

仅运行 v4 codemods：

```sh
npx @ai-sdk/codemod v4
```

若要单独运行某个 codemod，请指定其名称：

```sh
npx @ai-sdk/codemod <codemod-name> <path>
```

例如运行某个 v4 codemod：

```sh
npx @ai-sdk/codemod v4/replace-baseurl src/
```

另请参阅 [codemods 对照表](#codemod-table)。此外，最新的 codemods 集合位于 [`@ai-sdk/codemod`](https://github.com/vercel/ai/tree/main/packages/codemod/src/codemods) 仓库。

## Provider 变更

### 移除 `baseUrl` 选项

所有 providers 已移除 `baseUrl` 选项。请改用 `baseURL`。

```ts filename="AI SDK 3.4"
const perplexity = createOpenAI({
  // ...
  baseUrl: 'https://api.perplexity.ai/',
});
```

```ts filename="AI SDK 4.0"
const perplexity = createOpenAI({
  // ...
  baseURL: 'https://api.perplexity.ai/',
});
```

### Anthropic 提供商

#### 移除 `Anthropic` facade

Anthropic provider 已移除 `Anthropic` facade。
请改用 `anthropic` 对象或 `createAnthropic` 函数。

```ts filename="AI SDK 3.4"
const anthropic = new Anthropic({
  // ...
});
```

```ts filename="AI SDK 4.0"
const anthropic = createAnthropic({
```

<Note type="warning">
  此项变更没有可用的 codemod。请自行审查并手动更新代码。

```ts filename="AI SDK 3.4"
const result = await generateText({
```

## AI SDK Core 变更

### `streamText` 立即返回

`streamText` 不再返回 Promise，而是立即返回结果。
无需再 `await streamText` 的返回值。

```ts filename="AI SDK 3.4"
const result = await streamText({
  // ...
});
```

```ts filename="AI SDK 4.0"
const result = streamText({
  // ...
});
```

### `streamObject` 立即返回

`streamObject` 不再返回 Promise，而是立即返回结果。
无需再 `await streamObject` 的返回值。

```ts filename="AI SDK 3.4"
const result = await streamObject({
  // ...
});
```

```ts filename="AI SDK 4.0"
const result = streamObject({
  // ...
});
```

### 移除 roundtrips（往返）

`generateText` 与 `streamText` 已移除 `maxToolRoundtrips` 与 `maxAutomaticRoundtrips` 选项。
请改用 `maxSteps`。

`GenerateTextResult` 类型已移除 `roundtrips` 属性。
请改用 `steps`。

```ts filename="AI SDK 3.4"
const { text, roundtrips } = await generateText({
  maxToolRoundtrips: 1, // or maxAutomaticRoundtrips
  // ...
});
```

```ts filename="AI SDK 4.0"
const { text, steps } = await generateText({
  maxSteps: 2,
  // ...
});
```

### 移除 `nanoid` 导出

`nanoid` 导出已移除。请改用 [`generateId`](/docs/reference/ai-sdk-core/generate-id)。

```ts filename="AI SDK 3.4"
import { nanoid } from 'ai';
```

```ts filename="AI SDK 4.0"
import { generateId } from 'ai';
```

### 提高生成 ID 的默认长度

<Note type="warning">
  此项变更没有可用的 codemod。请自行审查并手动更新代码。
</Note>

[`generateId`](/docs/reference/ai-sdk-core/generate-id) 现在默认生成 16 字符 ID；此前默认为 7 字符。

例如当你限制了 ID 长度时，可能需要同步更新数据库 schema。

```ts filename="AI SDK 4.0"
import { generateId } from 'ai';

const id = generateId(); // now 16 characters
```


## AI SDK 2.x Legacy Changes

<Note type="warning">
  本节变更没有可用的 codemods。请自行审查并手动更新代码。
</Note>

### 移除 2.x 遗留 providers

AI SDK 2.x 的遗留 providers 已移除。请改用新的 [AI SDK provider 架构](/docs/foundations/providers-and-models)。

#### 移除 2.x 遗留 function 与 tools 调用

已从 `useChat` 与 `Message` 移除遗留的 `function_call` 与 `tools` 选项。
`Message` 类型已移除 `name` 属性。
请改用 [AI SDK Core 的工具调用](/docs/ai-sdk-core/tools-and-tool-calling)。

### 移除 2.x prompt helpers

在 AI SDK provider 架构下不再需要用于构造消息提示的 prompt helpers，相关代码已移除。

### 移除 2.x `AIStream`

已移除 `AIStream` 函数及相关导出。
请改用 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 及其 `toDataStream()`。

### 移除 2.x `StreamingTextResponse`

已移除 `StreamingTextResponse` 函数。
请改用 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 及其 `toDataStreamResponse()`。

### 移除 2.x `streamToResponse`

已移除 `streamToResponse` 函数。
请改用 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 及其 `pipeDataStreamToResponse()`。

### 移除 2.x RSC `Tokens` 流式传输

已移除 2.x 遗留的 `Tokens` RSC 流式传输。
`Tokens` 早于 AI SDK RSC 实现，现已不再需要。
