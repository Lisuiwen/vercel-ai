# 版本策略

版本号遵循：`MAJOR.MINOR.PATCH`

- **主版本（Major）**：破坏性 API 变更，需要修改代码。
- **次版本（Minor）**：通过博文汇总新功能与改进，并公开发布以突出收益。
- **修订版本（Patch）**：新功能与缺陷修复。

## API 稳定性

我们通过以下方式说明 API 的稳定性：

### 稳定 API

凡是没有特殊前缀的 API 均视为稳定，可用于生产环境。我们对稳定功能保持向后兼容，仅在主版本中引入破坏性变更。

### 实验性 API

带有 `experimental_` 或 `Experimental_` 前缀的 API（例如 `experimental_generateImage()`）仍在演进，可能在任意版本中发生变化。若要安全使用实验性 API：

1. 先在开发环境验证，而不是直接用于生产
2. 升级前查阅发行说明
3. 为可能的代码调整做好准备

<Note type="warning">
  如果使用实验性 API，请将 AI SDK 的版本号精确固定（避免使用 ^ 或 ~ 这类版本范围），以防出现意外的破坏性变更。
</Note>

### 已弃用 API

标记为 `deprecated` 的 API 将在未来的主版本中移除。你可以等到主版本发布后再更新代码。处理弃用项时可以：

1. 切换到推荐的替代 API
2. 遵循迁移指南（随主版本一同发布）

<Note>
  对于主版本升级，我们会在可行的情况下提供自动化 codemod，帮助你迁移到新版本。
</Note>
