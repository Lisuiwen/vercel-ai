# Svelte 快速入门

AI SDK 是一个强大的 TypeScript 库，旨在帮助开发者构建由 AI 驱动的应用。

在本快速入门中，你将构建一个带有流式聊天界面的简易 agent。在此过程中，你还会学到在自己项目中使用 SDK 所需的核心概念与方法。

如果你对 [提示词工程（Prompt Engineering）](/docs/advanced/prompt-engineering) 与 [HTTP 流式传输](/docs/foundations/streaming) 还不熟悉，可以先阅读这些文档（可选）。

## 前置条件

要完成本快速入门，你需要：

- 在本地开发环境中安装 Node.js 18+ 与 pnpm。
- 一枚 [ Vercel AI Gateway ](https://vercel.com/ai-gateway) API 密钥。

若尚未获取 Vercel AI Gateway API 密钥，可在 Vercel 网站[注册](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai&title=Go+to+AI+Gateway)后创建。

## 创建 API 路由

新建 SvelteKit 端点 `src/routes/api/chat/+server.ts`，并加入以下代码：

```tsx filename="src/routes/api/chat/+server.ts"
import {
  streamText,
  type UIMessage,
  convertToModelMessages,
  createGateway,
} from 'ai';

import { AI_GATEWAY_API_KEY } from '$env/static/private';

const gateway = createGateway({
  apiKey: AI_GATEWAY_API_KEY,
});

export async function POST({ request }) {
  const { messages }: { messages: UIMessage[] } = await request.json();

  const result = streamText({
    model: gateway('anthropic/claude-sonnet-4.5'),
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

<Note>
  若遇到与 `AI_GATEWAY_API_KEY` 或 `POST`
  函数有关的类型报错，可先启动开发服务器再试。
</Note>

这段代码在做的事如下：

1. 使用 `ai` 包中的 `createGateway` 创建 Gateway Provider 实例。
2. 定义 `POST` 请求处理函数，从请求体取出 `messages`。`messages` 保存你与聊天机器人之间的对话历史，并为下一次生成提供必要上下文；其类型为 `UIMessage`，面向应用界面使用——包含完整消息历史及时间戳等元数据。
3. 调用从 `ai` 包导入的 [`streamText`](/docs/reference/ai-sdk-core/stream-text)。该函数接收一个配置对象，其中包含第 1 步的 `model` Provider 与第 2 步的 `messages`。你还可以传入更多 [settings](/docs/ai-sdk-core/settings) 以进一步调节模型行为。`messages` 字段需要 `ModelMessage[]`，与 `UIMessage` 不同在于不包含时间戳或发送者等元数据。两者之间的转换可使用 `convertToModelMessages`，它会去掉仅供 UI 使用的元数据，把 `UIMessage[]` 转成模型所需的 `ModelMessage[]`。
4. `streamText` 返回 [`StreamTextResult`](/docs/reference/ai-sdk-core/stream-text#result-object)。该结果对象上的 [ `toUIMessageStreamResponse` ](/docs/reference/ai-sdk-core/stream-text#to-ui-message-stream-response) 用于把生成结果转成可流式返回的响应对象。
5. 将返回值发送给客户端，以流式输出响应。

```ts
// Option 2: Install and import from '@ai-sdk/gateway' package
import { gateway } from '@ai-sdk/gateway';
model: gateway('anthropic/claude-sonnet-4.5');
```

## 接入界面

有了可调用 LLM 的 API 路由后，接下来搭建前端。AI SDK 的 [UI](/docs/ai-sdk-ui) 包把聊天界面的复杂度收敛到一个类 `Chat` 上；其属性与 API 与 React 的 [`useChat`](/docs/reference/ai-sdk-ui/use-chat) 大体一致。

用下面代码更新根页面（`src/routes/+page.svelte`），展示聊天消息列表并提供用户输入框：

```svelte filename="src/routes/+page.svelte"
<script lang="ts">
  import { Chat } from '@ai-sdk/svelte';

  let input = '';
  const chat = new Chat({});

  function handleSubmit(event: SubmitEvent) {
    event.preventDefault();
    chat.sendMessage({ text: input });
    input = '';
  }
</script>

<main>
  <ul>
    {#each chat.messages as message, messageIndex (messageIndex)}
      <li>
        <div>{message.role}</div>
        <div>
          {#each message.parts as part, partIndex (partIndex)}
            {#if part.type === 'text'}
              <div>{part.text}</div>
            {/if}
          {/each}
        </div>
      </li>
    {/each}
  </ul>
  <form onsubmit={handleSubmit}>
    <input bind:value={input} />
    <button type="submit">Send</button>
  </form>
</main>
```

该页面使用 `Chat` 类；默认会使用你先前创建的 `POST` 路由处理函数。该类提供处理用户输入与表单提交的函数和状态，还包含多种工具函数与状态变量，例如：

- `messages`：当前聊天消息（对象数组，含 `id`、`role`、`parts` 等字段）。
- `sendMessage`：向聊天 API 发送消息的函数。

组件用本地状态保存输入框内容，提交表单时调用 `sendMessage` 传入文本并清空输入。

LLM 的回复通过消息的 `parts` 数组访问。每条消息包含有序的 `parts`，表示模型在该轮输出中的全部片段，可包含纯文本、推理 token 等（后文还会遇到）。`parts` 保留模型输出的顺序，便于你按生成顺序展示或处理每一部分。

## 用 Tool 增强聊天机器人

大语言模型（LLM）虽然生成能力很强，但在离散任务（例如数学）或与外部世界交互（例如查天气）上往往吃力。这时就需要 [tools](/docs/ai-sdk-core/tools-and-tool-calling)。

Tool 是 LLM 可以调用的动作；这些动作的结果可以再回传给 LLM，供其在后续回复中参考。

例如用户问当前天气，没有 tool 时模型只能根据训练数据给出笼统信息；有了天气 tool，就能拉取与地点相关的最新天气数据。

下面通过添加一个简单的天气 tool 来增强聊天机器人。

### 更新 API 路由

修改 `src/routes/api/chat/+server.ts`，加入新的天气 tool：

```tsx filename="src/routes/api/chat/+server.ts" highlight="2,3,17-31"
import {
  createGateway,
  streamText,
  type UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
import { z } from 'zod';

import { AI_GATEWAY_API_KEY } from '$env/static/private';

const gateway = createGateway({
  apiKey: AI_GATEWAY_API_KEY,
});

export async function POST({ request }) {
  const { messages }: { messages: UIMessage[] } = await request.json();

  const result = streamText({
    model: gateway('anthropic/claude-sonnet-4.5'),
    messages: await convertToModelMessages(messages),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

更新后的代码要点：

1. 从 `ai` 包导入 `tool`，从 `zod` 导入 `z` 做 schema 校验。
2. 在 `tools` 中定义名为 `weather` 的 tool：
   - 通过 `description` 帮助模型理解何时应调用它。
   - 用 Zod 定义 `inputSchema`，要求提供 `location` 字符串；模型会尝试从对话上下文中抽取该参数，若无法提取则会向用户追问缺失信息。
   - 定义 `execute` 函数，这里用随机温度模拟天气数据（异步函数在服务端运行，因此可以对接真实外部 API）。

现在聊天机器人可以为用户询问的任意地点「获取」天气。当模型判断需要调用天气 tool 时，会生成带必要入参的 tool 调用；`execute` 会自动运行，工具输出会作为 `tool` 类型的消息加入 `messages`。

试试提问类似 「What's the weather in New York?」，观察模型如何调用新 tool。

你是否注意到界面里有一处空白回复？那是因为模型本轮没有产出文本，而是生成了 tool 调用。你可以在客户端通过 `message.parts` 中类型为 `tool-weather` 的 part 查看 tool 调用及后续工具结果。

<Note>
  Tool 对应的 part 始终命名为 `tool-{toolName}`，其中 `{toolName}` 是你在定义 tool
  时使用的键名。此处 tool 名为 `weather`，因此 part 类型为 `tool-weather`。
</Note>

## 启用多步 Tool 调用

你可能已经注意到：虽然 tool 在界面里可见，但模型似乎没有利用这些信息回答最初的问题。这是因为一旦模型生成了 tool 调用，按定义它在本轮就完成了「生成」。

要解决这一点，可使用 `stopWhen` 启用多步 tool 调用。默认 `stopWhen` 为 `isStepCount(1)`，即存在 tool 结果时也在第一步之后就结束生成。通过调整该条件，可以让模型自动把工具结果再次喂给自己，在满足你设定的停止条件前持续多轮生成。此处我们希望模型在拿到天气工具结果后继续生成，从而回答用户的原始问题。

### 更新 API 路由

修改 `src/routes/api/chat/+server.ts`，加入 `stopWhen` 条件：

```ts filename="src/routes/api/chat/+server.ts" highlight="15"
import {
  createGateway,
  streamText,
  type UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
import { z } from 'zod';

import { AI_GATEWAY_API_KEY } from '$env/static/private';

const gateway = createGateway({
  apiKey: AI_GATEWAY_API_KEY,
});

export async function POST({ request }) {
  const { messages }: { messages: UIMessage[] } = await request.json();

  const result = streamText({
    model: gateway('anthropic/claude-sonnet-4.5'),
    messages: await convertToModelMessages(messages),
    stopWhen: isStepCount(5),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

回到浏览器询问某地天气。你现在应能看到模型利用天气工具的结果来作答。

设置 `stopWhen: isStepCount(5)` 表示允许单次请求中最多经历 5 个「步骤」，从而支持更复杂的多轮交互：模型可按需跨多步收集并处理信息。若再添加一个把华氏度转为摄氏度的 tool，可以更直观地观察这一行为。

### 再添加一个 tool

更新 `src/routes/api/chat/+server.ts`，加入将华氏温度转为摄氏温度的新 tool：

```tsx filename="src/routes/api/chat/+server.ts" highlight="32-45"
import {
  createGateway,
  streamText,
  type UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
import { z } from 'zod';

import { AI_GATEWAY_API_KEY } from '$env/static/private';

const gateway = createGateway({
  apiKey: AI_GATEWAY_API_KEY,
});

export async function POST({ request }) {
  const { messages }: { messages: UIMessage[] } = await request.json();

  const result = streamText({
    model: gateway('anthropic/claude-sonnet-4.5'),
    messages: await convertToModelMessages(messages),
    stopWhen: isStepCount(5),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
      convertFahrenheitToCelsius: tool({
        description: 'Convert a temperature in fahrenheit to celsius',
        inputSchema: z.object({
          temperature: z
            .number()
            .describe('The temperature in fahrenheit to convert'),
        }),
        execute: async ({ temperature }) => {
          const celsius = Math.round((temperature - 32) * (5 / 9));
          return {
            celsius,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

## `@ai-sdk/svelte` 与 `@ai-sdk/react` 有何不同？

表面上的区别是：Svelte 用类管理状态，而 React 用 hook，因此 React 里的 `useChat` 对应 Svelte 的 `Chat`。除此之外还有几点需要注意：

### 1. 传给类的参数默认不具备响应式

与 React 中 hook 会在组件失效时重复执行不同，Svelte 组件 `script` 中的代码仅在组件创建时执行一次。
这意味着若希望类的入参具备响应式，需要向类传入 _引用_ 而不是瞬时拷贝的值：

```svelte
<script>
  import { Chat } from '@ai-sdk/svelte';

  let { id } = $props();

  // won't work; the class instance will be created once, `id` will be copied by value, and won't update when $props.id changes
  let chat = new Chat({ id });

  // will work; passes `id` by reference, so `Chat` always has the latest value
  let chat = new Chat({
    get id() {
      return id;
    },
  });
</script>
```

多数情况下这并不是问题：传入 `Chat` 的参数大多为静态配置（例如你通常不会假定 `onError` 处理器会频繁变化）。

### 2. 不要对类实例属性做解构

在原生 JavaScript 中，解构类实例上的属性是按值拷贝，会与其类实例「脱钩」：

```js
const classInstance = new Whatever();
classInstance.foo = 'bar';
const { foo } = classInstance;
classInstance.foo = 'baz';

console.log(foo); // 'bar'
```

Svelte 中的类也是一样：

```svelte
<script>
  import { Chat } from '@ai-sdk/svelte';

  const chat = new Chat({});
  let { messages } = chat;

  chat.sendMessage({ text: 'Hello, world!' }).then(() => {
    console.log(messages); // []
    console.log(chat.messages); // [{ id: '...', role: 'user', parts: [{ type: 'text', text: 'Hello, world!' }] }]
  });
</script>
```

### 3. 实例同步需要上下文

在 React 中，`id` 相同的 hook 实例会保持同步——因此两个 `useChat` 若 `id` 相同，`messages`、`status` 等也会一致。
大多数场景并不需要这种行为；若有需要，可以在根布局里用 `createAIContext` 创建上下文：

```svelte
<script>
  import { createAIContext } from '@ai-sdk/svelte';

  let { children } = $props();

  createAIContext();
  // all hooks created after this or in components that are children of this component
  // will have synchronized state
</script>

{@render children()}
```
