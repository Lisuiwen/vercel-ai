# Node.js 快速入门

AI SDK 是一款强大的 TypeScript 库，旨在帮助开发者构建 AI 驱动的应用。

在本快速入门教程中，你将搭建一个简单的 agent，并配有流式聊天的用户界面。过程中将掌握在自己的项目中使用 SDK 所需的核心概念与常用做法。

如果你还不熟悉 [Prompt Engineering](/docs/advanced/prompt-engineering) 与 [HTTP Streaming](/docs/foundations/streaming) 的概念，可以先阅读这些文档（可选）。

## 前置条件

要完成本快速入门，你需要：

- 在本地开发机上安装 Node.js 18+ 与 pnpm。
- 一个 [ Vercel AI Gateway ](https://vercel.com/ai-gateway) API key。

如果你还没有 Vercel AI Gateway 的 API key，可以在 Vercel 网站[注册](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai&title=Go+to+AI+Gateway)后获取。

## 搭建应用

先用 `mkdir` 命令创建一个新目录，进入该目录后运行 `pnpm init`。这会在新目录中生成 `package.json`。

```bash
mkdir my-ai-app
cd my-ai-app
pnpm init
```

### 安装依赖

安装 `ai`（AI SDK）以及其他必要依赖。

<Note>
  AI SDK 被设计为与任意大语言模型交互的统一接口。
  这意味着你只需改一行代码就能切换 model 与 provider！在 [providers](/providers) 章节了解[可用 provider](/providers)与[构建自定义 provider](/providers/community-providers/custom-providers)。
</Note>

```bash
pnpm add ai zod dotenv
pnpm add -D @types/node tsx typescript
```

`ai` 包包含 AI SDK。你将用 `zod` 定义传给大语言模型（LLM）的类型安全 schema，并用 `dotenv` 在应用中读取环境变量（你的 Vercel AI Gateway key）。另外还会用 `-D` 安装三个开发依赖，用于运行 TypeScript 代码。

### 配置 Vercel AI Gateway API key

在项目根目录创建 `.env` 文件并填入 Vercel AI Gateway API Key，用于向 Vercel AI Gateway 服务认证你的应用。

<Snippet text="touch .env" />

编辑 `.env` 文件：

```env filename=".env"
AI_GATEWAY_API_KEY=xxxxxxxxx
```

将 `xxxxxxxxx` 替换为你的 Vercel AI Gateway API key。

<Note className="mb-4">
  AI SDK 会使用 `AI_GATEWAY_API_KEY` 环境变量向 Vercel AI Gateway 认证。
</Note>

## 创建应用

在项目根目录创建 `index.ts` 并加入以下代码：

```ts filename="index.ts"
import { ModelMessage, streamText } from 'ai';
__PROVIDER_IMPORT__;
import 'dotenv/config';
import * as readline from 'node:readline/promises';

const terminal = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

const messages: ModelMessage[] = [];

async function main() {
  while (true) {
    const userInput = await terminal.question('You: ');

    messages.push({ role: 'user', content: userInput });

    const result = streamText({
      model: __MODEL__,
      messages,
    });

    let fullResponse = '';
    process.stdout.write('\nAssistant: ');
    for await (const delta of result.textStream) {
      fullResponse += delta;
      process.stdout.write(delta);
    }
    process.stdout.write('\n\n');

    messages.push({ role: 'assistant', content: fullResponse });
  }
}

main().catch(console.error);
```

下面说明这段代码在做什么：

1. 设置 readline 接口，从终端读取输入，以便在命令行中直接进行交互。
2. 初始化名为 `messages` 的数组，保存对话历史，让 agent 在持续对话中保持上下文。
3. 在 `main` 函数中：

- 提示并获取用户输入，存入 `userInput`。
- 将用户输入作为 user 消息加入 `messages` 数组。
- 调用从 `ai` 包导入的 [`streamText`](/docs/reference/ai-sdk-core/stream-text)。该函数接受包含 `model` provider 与 `messages` 的配置对象。
- 遍历 `streamText` 返回的文本流（`result.textStream`），把流内容打印到终端。
- 将 assistant 的回复加入 `messages` 数组。

## 运行应用

至此，搭建 agent 所需的代码已经齐备。用以下命令启动应用：

<Snippet text="pnpm tsx index.ts" />

终端中会出现提示符。输入一条消息试试，你会看到 AI agent 实时回复！借助 AI SDK，在 Node.js 中构建 AI 聊天界面既快又简单。

## 选择 Provider

AI SDK 通过 [官方](/providers/ai-sdk-providers)、[OpenAI 兼容](/providers/openai-compatible-providers)与[社区](/providers/community-providers)包支持数十家 model provider。

本快速入门使用 [Vercel AI Gateway](https://vercel.com/ai-gateway) provider，它也是默认的 [global provider](/docs/ai-sdk-core/provider-management#global-provider-configuration)。因此你可以在 model 配置里用简单字符串引用模型：

```ts
model: __MODEL__;
```

你也可以用另外两种等价方式显式导入并使用 gateway provider：

```ts
// Option 1: Import from 'ai' package (included by default)
import { gateway } from 'ai';
model: gateway('anthropic/claude-sonnet-4.5');

// Option 2: Install and import from '@ai-sdk/gateway' package
import { gateway } from '@ai-sdk/gateway';
model: gateway('anthropic/claude-sonnet-4.5');
```

## 用 tool 增强 agent

大语言模型（LLM）生成能力很强，但在离散任务（例如数学）以及与外部世界交互（例如查天气）方面较弱，这时就需要 [tool](/docs/ai-sdk-core/tools-and-tool-calling)。

tool 是 LLM 可以调用的动作，执行结果会回传给 LLM，供下一次回复参考。

例如用户问当前天气：没有 tool 时，agent 只能根据训练数据给出笼统信息；有了天气 tool，就能拉取并返回最新的、与地点相关的天气。

下面通过添加一个简单的天气 tool 来增强 agent。

### 更新应用

修改 `index.ts`，加入新的天气 tool：

```ts filename="index.ts" highlight="2,4,24-37"
import { ModelMessage, streamText, tool } from 'ai';
__PROVIDER_IMPORT__;
import 'dotenv/config';
import { z } from 'zod';
import * as readline from 'node:readline/promises';

const terminal = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

const messages: ModelMessage[] = [];

async function main() {
  while (true) {
    const userInput = await terminal.question('You: ');

    messages.push({ role: 'user', content: userInput });

    const result = streamText({
      model: __MODEL__,
      messages,
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
      },
    });

    let fullResponse = '';
    process.stdout.write('\nAssistant: ');
    for await (const delta of result.textStream) {
      fullResponse += delta;
      process.stdout.write(delta);
    }
    process.stdout.write('\n\n');

    messages.push({ role: 'assistant', content: fullResponse });
  }
}

main().catch(console.error);
```

更新后的代码中：

1. 从 `ai` 包导入 `tool` 函数。
2. 在 `tools` 对象中定义 `weather` tool。该 tool：
   - 带有 description，帮助 agent 判断何时使用。
   - 用 Zod schema 定义 `inputSchema`，要求提供 `location` 字符串才能执行。agent 会尝试从对话上下文中提取该输入；若无法提取，会向用户询问缺失信息。
   - 定义 `execute` 函数模拟获取天气（此处返回随机温度）。这是运行在服务端上的异步函数，你可以改为从外部 API 拉取真实数据。

现在 agent 可以为用户询问的任意地点「获取」天气。当 agent 判断需要天气 tool 时，会生成带必要参数的 tool 调用，`execute` 会自动运行，结果会用于 agent 的后续回复。

可以试试问「What's the weather in New York?」，观察 agent 如何使用新 tool。

注意到 assistant 回复为空？这是因为 agent 生成了 tool 调用而非文本。你可以在结果对象的 `toolCall` 与 `toolResult` 字段中查看 tool 调用及结果。

```typescript highlight="46-47"

import 'dotenv/config';
import { z } from 'zod';
import * as readline from 'node:readline/promises';

const terminal = readline.createInterface({

    const result = streamText({
      model: __MODEL__,
      messages,
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
      },
    });

    let fullResponse = '';
    process.stdout.write('\nAssistant: ');
    for await (const delta of result.textStream) {
      fullResponse += delta;
      process.stdout.write(delta);
    }
    process.stdout.write('\n\n');

    console.log(await result.toolCalls);
    console.log(await result.toolResults);

  }
}

```

询问天气时，你会在聊天界面中看到 tool 调用及其结果。

## 启用多步 tool 调用

你可能已经注意到：虽然 tool 结果可见，agent 却没用这些信息回答最初的问题。这是因为 agent 一旦生成 tool 调用，从技术上讲该轮生成已经结束。

可通过 `stopWhen` 启用多步 tool 调用：该功能会把 tool 结果自动送回 agent，触发新一轮生成，直到满足你定义的停止条件。此处你希望 agent 利用天气 tool 的结果来回答问题。

### 更新应用

修改 `index.ts`，用 `stopWhen` 配置停止条件：

```ts filename="index.ts" highlight="38-41"
import { ModelMessage, streamText, tool, isStepCount } from 'ai';
__PROVIDER_IMPORT__;
import 'dotenv/config';
import { z } from 'zod';
import * as readline from 'node:readline/promises';

const terminal = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

const messages: ModelMessage[] = [];

async function main() {
  while (true) {
    const userInput = await terminal.question('You: ');

    messages.push({ role: 'user', content: userInput });

    const result = streamText({
      model: __MODEL__,
      messages,
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
      },
      stopWhen: isStepCount(5),
      onStepFinish: async ({ toolResults }) => {
        if (toolResults.length) {
          console.log(JSON.stringify(toolResults, null, 2));
        }
      },
    });

    let fullResponse = '';
    process.stdout.write('\nAssistant: ');
    for await (const delta of result.textStream) {
      fullResponse += delta;
      process.stdout.write(delta);
    }
    process.stdout.write('\n\n');

    messages.push({ role: 'assistant', content: fullResponse });
  }
}

main().catch(console.error);
```

更新后的代码中：

1. 将 `stopWhen` 设为 `isStepCount(5)`，允许 agent 在单次生成中最多使用 5 个「步骤」。
2. 添加 `onStepFinish` 回调，记录每一步的 `toolResults`，便于理解 agent 的 tool 使用情况。因此也可以删除上一示例中的 `toolCall` 与 `toolResult` 的 `console.log`。

询问某地天气时，agent 应会利用天气 tool 的结果作答。

设置 `stopWhen: isStepCount(5)` 后，agent 在单次生成中最多可走 5 步，从而支持更复杂的交互，并在多步中收集与处理信息。可以再添加一个把摄氏温度转为华氏温度的 tool 来体验这一点。

### 添加第二个 tool

更新 `index.ts`，新增将华氏温度转为摄氏温度的 tool：

```ts filename="index.ts" highlight="37-48"
import { ModelMessage, streamText, tool, isStepCount } from 'ai';
__PROVIDER_IMPORT__;
import 'dotenv/config';
import { z } from 'zod';
import * as readline from 'node:readline/promises';

const terminal = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

const messages: ModelMessage[] = [];

async function main() {
  while (true) {
    const userInput = await terminal.question('You: ');

    messages.push({ role: 'user', content: userInput });

    const result = streamText({
      model: __MODEL__,
      messages,
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
        convertFahrenheitToCelsius: tool({
          description: 'Convert a temperature in fahrenheit to celsius',
          inputSchema: z.object({
            temperature: z
              .number()
              .describe('The temperature in fahrenheit to convert'),
          }),
          execute: async ({ temperature }) => {
            const celsius = Math.round((temperature - 32) * (5 / 9));
            return {
              celsius,
            };
          },
        }),
      },
      stopWhen: isStepCount(5),
      onStepFinish: async ({ toolResults }) => {
        if (toolResults.length) {
          console.log(JSON.stringify(toolResults, null, 2));
        }
      },
    });

    let fullResponse = '';
    process.stdout.write('\nAssistant: ');
    for await (const delta of result.textStream) {
      fullResponse += delta;
      process.stdout.write(delta);
    }
    process.stdout.write('\n\n');

    messages.push({ role: 'assistant', content: fullResponse });
  }
}

main().catch(console.error);
```

当你问「What's the weather in New York in celsius?」时，应能看到更完整的一条链路：

1. agent 会先为纽约调用天气 tool。
2. 你会在日志中看到 tool 结果。
3. 随后调用温度转换 tool，把华氏转为摄氏。
4. agent 再综合这些信息，用自然语言说明纽约的天气。

这种多步方式让 agent 能先收集信息再作答，回复更准确、更贴合上下文，实用性也会高很多。

本示例展示了 tool 如何扩展 agent 能力。你还可以接入真实 API、数据库或其他外部系统，构建更复杂的 tool，让 agent 实时访问并处理真实数据，与外部世界交互。tool 弥合了 agent 知识截止与当前信息之间的鸿沟，并使其能采取有意义的行动，而不仅是生成文本。
