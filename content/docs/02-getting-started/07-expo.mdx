# Expo 快速入门

在本快速入门教程中，你将使用 [Expo](https://expo.dev/) 搭建一个简单的 agent，并配有流式聊天的用户界面。过程中将掌握在自己的项目中使用 SDK 所需的核心概念与常用做法。

如果你还不熟悉 [Prompt Engineering](/docs/advanced/prompt-engineering) 与 [HTTP Streaming](/docs/foundations/streaming) 的概念，可以先阅读这些文档（可选）。

## 前置条件

要完成本快速入门，你需要：

- 在本地开发机上安装 Node.js 18+ 与 pnpm。
- 一个 [ Vercel AI Gateway ](https://vercel.com/ai-gateway) API key。

如果你还没有 Vercel AI Gateway 的 API key，可以在 Vercel 网站[注册](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai&title=Go+to+AI+Gateway)后获取。

## 创建 API 路由

创建 route handler：`app/api/chat+api.ts`，并加入以下代码：

```tsx filename="app/api/chat+api.ts"
import { streamText, UIMessage, convertToModelMessages } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse({
    headers: {
      'Content-Type': 'application/octet-stream',
      'Content-Encoding': 'none',
    },
  });
}
```

这段代码在做什么：

1. 定义异步 `POST` 请求处理函数，从请求体中解构出 `messages`。`messages` 保存了你与聊天机器人之间的对话历史，为下一轮生成提供上下文。
2. 调用从 `ai` 包导入的 [`streamText`](/docs/reference/ai-sdk-core/stream-text)。该函数接收包含 `model` provider（从 `ai` 导入）与 `messages`（第 1 步）的配置对象。你还可以传入更多 [settings](/docs/ai-sdk-core/settings) 定制模型行为。
3. `streamText` 返回 [`StreamTextResult`](/docs/reference/ai-sdk-core/stream-text#result-object)。该结果对象包含 [ `toUIMessageStreamResponse` ](/docs/reference/ai-sdk-core/stream-text#to-ui-message-stream-response)，用于把结果转成可流式返回的响应对象。
4. 最后把结果返回给客户端，以流式输出响应。

该 API 路由在 `/api/chat` 上提供了一个 POST 端点。

## 选择 Provider

AI SDK 通过 [官方](/providers/ai-sdk-providers)、[OpenAI 兼容](/providers/openai-compatible-providers)与[社区](/providers/community-providers)包支持数十家 model provider。

```ts

import { gateway } from '@ai-sdk/gateway';
model: gateway('anthropic/claude-sonnet-4.5');
```

## 连接前端

既然已经有了能查询 LLM 的 API 路由，接下来配置前端。AI SDK 的 [ UI ](/docs/ai-sdk-ui) 包把聊天界面的复杂度收拢到一个 hook：[`useChat`](/docs/reference/ai-sdk-ui/use-chat)。

用下列代码更新根页面（`app/(tabs)/index.tsx`），以展示聊天消息列表并提供用户输入：

```tsx filename="app/(tabs)/index.tsx"
import { generateAPIUrl } from '@/utils';
import { useChat } from '@ai-sdk/react';
import { DefaultChatTransport } from 'ai';
import { fetch as expoFetch } from 'expo/fetch';
import { useState } from 'react';
import { View, TextInput, ScrollView, Text, SafeAreaView } from 'react-native';

export default function App() {
  const [input, setInput] = useState('');
  const { messages, error, sendMessage } = useChat({
    transport: new DefaultChatTransport({
      fetch: expoFetch as unknown as typeof globalThis.fetch,
      api: generateAPIUrl('/api/chat'),
    }),
    onError: error => console.error(error, 'ERROR'),
  });

  if (error) return <Text>{error.message}</Text>;

  return (
    <SafeAreaView style={{ height: '100%' }}>
      <View
        style={{
          height: '95%',
          display: 'flex',
          flexDirection: 'column',
          paddingHorizontal: 8,
        }}
      >
        <ScrollView style={{ flex: 1 }}>
          {messages.map(m => (
            <View key={m.id} style={{ marginVertical: 8 }}>
              <View>
                <Text style={{ fontWeight: 700 }}>{m.role}</Text>
                {m.parts.map((part, i) => {
                  switch (part.type) {
                    case 'text':
                      return <Text key={`${m.id}-${i}`}>{part.text}</Text>;
                  }
                })}
              </View>
            </View>
          ))}
        </ScrollView>

        <View style={{ marginTop: 8 }}>
          <TextInput
            style={{ backgroundColor: 'white', padding: 8 }}
            placeholder="Say something..."
            value={input}
            onChange={e => setInput(e.nativeEvent.text)}
            onSubmitEditing={e => {
              e.preventDefault();
              sendMessage({ text: input });
              setInput('');
            }}
            autoFocus={true}
          />
        </View>
      </View>
    </SafeAreaView>
  );
}
```

该页面使用 `useChat` hook，默认会调用你先前创建的 `POST` API 路由（`/api/chat`）。hook 提供处理用户输入与表单提交所需的函数与状态。`useChat` 还提供多种工具函数与状态变量：

- `messages`：当前聊天消息（对象数组，含 `id`、`role`、`parts` 属性）。
- `sendMessage`：向聊天 API 发送消息的函数。

组件用本地状态（`useState`）管理输入框，提交时调用 `sendMessage` 发送文本并清空输入。

LLM 的回复通过消息的 `parts` 数组访问。每条消息包含有序的 `parts`，表示模型在该轮生成的一切内容，可包括纯文本、推理 token 等（后文会涉及）。`parts` 保留模型输出的顺序，便于按生成顺序展示或处理。

<Note>
  需使用 expo/fetch 而非原生 node fetch 才能流式接收聊天回复。这要求 Expo 52 或更高版本。
</Note>

### 创建 API URL 生成器

由于使用 expo/fetch 做流式响应而非原生 fetch，需要 API URL 生成器，以便按客户端环境（如 Web 或移动端）使用正确的基础 URL 与格式。在项目根目录新建 `utils.ts`，加入以下代码：

```ts filename="utils.ts"
import Constants from 'expo-constants';

export const generateAPIUrl = (relativePath: string) => {
  const origin = Constants.experienceUrl.replace('exp://', 'http://');

  const path = relativePath.startsWith('/') ? relativePath : `/${relativePath}`;

  if (process.env.NODE_ENV === 'development') {
    return origin.concat(path);
  }

  if (!process.env.EXPO_PUBLIC_API_BASE_URL) {
    throw new Error(
      'EXPO_PUBLIC_API_BASE_URL environment variable is not defined',
    );
  }

  return process.env.EXPO_PUBLIC_API_BASE_URL.concat(path);
};
```

该工具函数会在开发与生产环境中生成 URL，确保 API 调用在不同设备与配置下都能正常工作。

<Note>
  部署到生产环境前，必须在生产环境中设置 `EXPO_PUBLIC_API_BASE_URL`
  环境变量，指向 API 服务器的基础 URL。
</Note>

## 用 tool 增强聊天机器人

大语言模型（LLM）有很强的生成能力，但在离散任务（例如数学）以及与外部世界交互（例如查天气）方面往往吃力。这时就需要 [tools](/docs/ai-sdk-core/tools-and-tool-calling)。

tool 是 LLM 可以触发的操作；这些操作的结果可以回传给 LLM，供其在下一轮回复中参考。

例如用户问当前天气，没有 tool 时，模型只能根据训练数据给出泛泛信息；有了天气 tool，它就能拉取与地点相关的实时天气。

下面通过一个简单的天气 tool 来增强聊天机器人。

### 更新 API 路由

修改 `app/api/chat+api.ts`，加入新的天气 tool：

```tsx filename="app/api/chat+api.ts" highlight="2,11-25"
import { streamText, UIMessage, convertToModelMessages, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse({
    headers: {
      'Content-Type': 'application/octet-stream',
      'Content-Encoding': 'none',
    },
  });
}
```

本次更新的要点：

1. 从 `ai` 包导入 `tool`，从 `zod` 导入 `z`，用于 schema 校验。
2. 在 `tools` 里定义 `weather` tool，该 tool：
   - 带有 `description`，帮助模型判断何时调用。
   - 用 Zod schema 定义 `inputSchema`，要求提供字符串 `location` 才能执行。模型会尝试从对话上下文中抽取该参数；若无法抽取，会向用户追问缺失信息。
   - 定义 `execute`，这里用随机温度模拟天气数据（异步函数跑在服务端，你也可以改为请求真实外部 API）。

现在聊天机器人可以为用户询问的任意地点「获取」天气。当模型判断需要调用天气 tool 时，会生成带参数的 tool call；`execute` 会自动运行，tool 输出会以 `tool` 消息形式加入 `messages`。

<Note>
  修改后可能需要重启开发服务器才能生效。
</Note>

试着问类似「What's the weather in New York?」，观察模型如何使用新 tool。

注意到 UI 里有时像是空白回复？那是因为模型没有生成普通文本，而是生成了 tool call。你可以在客户端通过 `message.parts` 里类型为 `tool-weather` 的 part 访问该 tool call 以及后续的 tool 结果。

<Note>
  Tool part 始终命名为 `tool-{toolName}`，其中 `{toolName}` 是你在定义 tool 时使用的键名。
  这里我们定义的 key 是 `weather`，因此 part 类型为 `tool-weather`。
</Note>

## 启用多步 tool 调用

你可能已经发现：tool 结果虽在界面中可见，模型却没用这些信息回答最初的问题。这是因为模型一旦生成 tool call，从技术上讲该轮生成已经结束。

可通过 `stopWhen` 启用多步 tool 调用。默认 `stopWhen` 为 `isStepCount(1)`，即在出现 tool 结果后第一步就停止生成。修改该条件后，模型可以把 tool 结果自动发回给自己，触发更多轮生成，直到满足你设定的停止条件。此处你希望模型继续生成，以便用天气 tool 的结果回答原问题。

### 更新 API 路由

修改 `app/api/chat+api.ts`，加入 `stopWhen` 条件：

```tsx filename="app/api/chat+api.ts" highlight="10"
import {
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    stopWhen: isStepCount(5),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse({
    headers: {
      'Content-Type': 'application/octet-stream',
      'Content-Encoding': 'none',
    },
  });
}
```

<Note>
  修改后可能需要重启开发服务器才能生效。
</Note>

回到 Expo 应用，询问某地天气，此时模型应会利用天气 tool 的结果作答。

设置 `stopWhen: isStepCount(5)` 后，模型在单次生成中最多可走 5 步，从而支持更复杂的交互，并在多步中收集与处理信息。可以再添加一个把华氏温度转为摄氏温度的 tool 来体验这一点。

### 添加更多 tool

```tsx filename="app/api/chat+api.ts" highlight="28-41"
  tool,

          };
        },

    },

      'Content-Type': 'application/octet-stream',
```

```tsx filename="app/(tabs)/index.tsx" highlight="37-42"
                    case 'tool-convertFahrenheitToCelsius':
                      return (
                        <Text key={`${m.id}-${i}`}>
                          {JSON.stringify(part, null, 2)}
                        </Text>
                      );
                  }
                })}
              </View>
            </View>
          ))}
        </ScrollView>

        <View style={{ marginTop: 8 }}>
          <TextInput
            style={{ backgroundColor: 'white', padding: 8 }}
            placeholder="Say something..."
            value={input}
            onChange={e => setInput(e.nativeEvent.text)}
            onSubmitEditing={e => {
              e.preventDefault();
              sendMessage({ text: input });
              setInput('');
            }}
            autoFocus={true}
          />
        </View>
      </View>
    </SafeAreaView>
  );
}
```

<Note>
  修改后可能需要重启开发服务器才能生效。
</Note>

当你问「What's the weather in New York in celsius?」时，应能看到更完整的一条链路：

1. 模型会先为纽约调用天气 tool。
2. 你会看到 tool 结果展示在界面上。
3. 随后它会调用温度转换 tool，把华氏温度转为摄氏温度。
4. 最后模型会综合这些信息，用自然语言说明纽约的天气。

这种多步方式让模型能先收集信息再作答，回复更准确、更贴合上下文，聊天机器人也会实用得多。

这个简单示例展示了 tool 如何扩展模型能力。你还可以接入真实 API、数据库或其他外部系统，构建更复杂的 tool，让模型实时访问并处理真实世界的数据，弥合模型知识截止与当前信息之间的鸿沟。

```ts filename="polyfills.js"
import { Platform } from 'react-native';
import structuredClone from '@ungap/structured-clone';

if (Platform.OS !== 'web') {
  const setupPolyfills = async () => {
    const { polyfillGlobal } =
      await import('react-native/Libraries/Utilities/PolyfillFunctions');

      await import('@stardazed/streams-text-encoding');
```
