# Next.js Pages Router 快速入门

AI SDK 是一个强大的 TypeScript 库，旨在帮助开发者构建由 AI 驱动的应用。

在本快速入门中，你将构建一个带有流式聊天界面的简易 agent，并在此过程中掌握在自己的项目中使用 AI SDK 所需的核心概念与技巧。

如果你对 [Prompt Engineering](/docs/advanced/prompt-engineering) 与 [HTTP 流式传输](/docs/foundations/streaming) 还不熟悉，可以先阅读这两篇文档。

## 前置条件

要完成本快速入门，你需要：

- 在本地开发环境中安装 Node.js 18+ 与 pnpm。
- 一枚 [ Vercel AI Gateway ](https://vercel.com/ai-gateway) API key。

若尚未获取 Vercel AI Gateway API key，可在 Vercel 网站[注册](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai&title=Go+to+AI+Gateway)后申请。

## 创建 Route Handler

<Note>
  在 Next.js 13+ 中，可以在使用 Pages Router 的同时使用 App Router 的 Route
  Handler。建议这样做，以便使用 Web API 的接口/签名，并更好地支持流式传输。
</Note>

创建 Route Handler（`app/api/chat/route.ts`），并加入以下代码：

```tsx filename="app/api/chat/route.ts"
import { streamText, UIMessage, convertToModelMessages } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

下面说明这段代码在做什么：

1. 定义异步 `POST` 处理函数，并从请求体中取出 `messages`。`messages` 保存你与聊天机器人之间的对话历史，为下一次生成提供上下文。其类型为 `UIMessage`，面向应用 UI：包含完整消息历史及时间戳等元数据。
2. 调用从 `ai` 包导入的 [`streamText`](/docs/reference/ai-sdk-core/stream-text)。该函数接收配置对象，其中包含 `model` provider 与第 1 步的 `messages`。还可传入更多 [设置](/docs/ai-sdk-core/settings) 以定制模型行为。`messages` 字段期望 `ModelMessage[]`，与 `UIMessage` 不同，不含时间戳或发送者等元数据。类型转换使用 `convertToModelMessages`，它会去掉 UI 专用元数据，将 `UIMessage[]` 转为模型所需的 `ModelMessage[]`。
3. `streamText` 返回 [`StreamTextResult`](/docs/reference/ai-sdk-core/stream-text#result-object)。该结果对象包含 [ `toUIMessageStreamResponse` ](/docs/reference/ai-sdk-core/stream-text#to-ui-message-stream-response)，用于把结果转为流式响应对象。
4. 最后将结果返回给客户端以流式输出。

该 Route Handler 在 `/api/chat` 提供 POST 端点。

## 选择 Provider

AI SDK 通过 [官方](/providers/ai-sdk-providers)、[OpenAI 兼容](/providers/openai-compatible-providers)与[社区](/providers/community-providers)包支持大量 model provider。

本快速入门使用 [Vercel AI Gateway](https://vercel.com/ai-gateway) provider，即默认的 [全局 provider](/docs/ai-sdk-core/provider-management#global-provider-configuration)。因此可在 model 配置中用简单字符串引用模型：

```ts

```

也可以显式导入并使用 gateway provider，以下两种方式等价：

```ts
// Option 1: Import from 'ai' package (included by default)

// Option 2: Install and import from '@ai-sdk/gateway' package
import { gateway } from '@ai-sdk/gateway';
model: gateway('anthropic/claude-sonnet-4.5');
```

## 接入 UI

有了可查询 LLM 的 API 路由后，接下来搭建前端。AI SDK 的 [ UI ](/docs/ai-sdk-ui) 包将聊天界面的复杂度抽象为一个 hook：[`useChat`](/docs/reference/ai-sdk-ui/use-chat)。

用以下代码更新根页面（`pages/index.tsx`），展示消息列表并提供用户输入：

```tsx filename="pages/index.tsx"
import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const [input, setInput] = useState('');
  const { messages, sendMessage } = useChat();
  return (
    <div className="flex flex-col w-full max-w-md py-24 mx-auto stretch">
      {messages.map(message => (
        <div key={message.id} className="whitespace-pre-wrap">
          {message.role === 'user' ? 'User: ' : 'AI: '}
          {message.parts.map((part, i) => {
            switch (part.type) {
              case 'text':
                return <div key={`${message.id}-${i}`}>{part.text}</div>;
            }
          })}
        </div>
      ))}

      <form
        onSubmit={e => {
          e.preventDefault();
          sendMessage({ text: input });
          setInput('');
        }}
      >
        <input
          className="fixed dark:bg-zinc-900 bottom-0 w-full max-w-md p-2 mb-8 border border-zinc-300 dark:border-zinc-800 rounded shadow-xl"
          value={input}
          placeholder="Say something..."
          onChange={e => setInput(e.currentTarget.value)}
        />
      </form>
    </div>
  );
}
```

该页面使用 `useChat` hook，默认会请求你先前创建的 `POST` API 路由（`/api/chat`）。hook 提供处理用户输入与表单提交的函数和状态。`useChat` 还提供多种工具函数与状态变量：

- `messages`：当前聊天消息（对象数组，含 `id`、`role`、`parts`）。
- `sendMessage`：向聊天 API 发送消息的函数。

组件用本地状态（`useState`）管理输入框，提交表单时调用 `sendMessage` 传入文本并清空输入。

LLM 的回复通过消息的 `parts` 数组访问。每条消息包含有序的 `parts`，表示模型在该轮生成的全部内容，可含纯文本、reasoning token 等，后续还会见到更多类型。`parts` 保持模型输出的顺序，便于按生成顺序展示或处理。

## 用 Tool 增强聊天机器人

大语言模型（LLM）生成能力很强，但在离散任务（如数学）或与外界交互（如查天气）上往往吃力，这时就需要 [tools](/docs/ai-sdk-core/tools-and-tool-calling)。

Tool 是 LLM 可调用的动作；执行结果可回传给模型，供下一轮回复参考。

例如用户问当前天气，没有 tool 时模型只能根据训练数据泛泛而谈；有了天气 tool，就能拉取与地点相关的实时天气。

### 更新 Route Handler

先为聊天机器人添加天气 tool。更新 Route Handler（`app/api/chat/route.ts`）：

```tsx filename="app/api/chat/route.ts" highlight="1,11-25"
import { streamText, UIMessage, convertToModelMessages, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

本次更新要点：

1. 从 `ai` 导入 `tool`，从 `zod` 导入 `z` 做 schema 校验。
2. 在 `tools` 中定义 `weather` tool：
   - `description` 帮助模型判断何时调用。
   - 用 Zod 定义 `inputSchema`，要求 `location` 字符串；模型会从对话上下文中抽取；若无法抽取会请用户补充。
   - `execute` 模拟获取天气（此处返回随机温度）。该异步函数在服务端运行，可改为请求真实外部 API。

现在聊天机器人可以为用户询问的任意地点「获取」天气。当模型决定调用天气 tool 时，会生成带输入的 tool call；`execute` 会自动执行，输出会以 `tool` 消息加入 `messages`。

可尝试问「纽约天气怎么样？」之类问题，观察模型如何使用新 tool。

界面里是否出现空白回复？那是因为模型本轮生成的是 tool call 而非文本。你可在客户端通过 `message.parts` 中的 `tool-weather` 部分访问 tool 调用及后续结果。

<Note>
  Tool 部分的类型名恒为 `tool-{toolName}`，其中 `{toolName}` 是你在定义 tool
  时使用的键名。本例中键为 `weather`，故 part 类型为 `tool-weather`。
</Note>

## 启用多步 Tool 调用

你可能已经发现：虽然 tool 在界面里可见，模型却没有用这些信息回答最初的问题。这是因为一旦模型生成了 tool call，从技术上讲本轮生成即已结束。

可通过 `stopWhen` 启用多步 tool 调用。默认 `stopWhen` 为 `isStepCount(1)`，即有 tool 结果时会在第一步后停止生成。调整该条件后，模型可自动把 tool 结果带回自身，触发后续生成，直到满足你设定的停止条件。这里我们希望模型在拿到天气结果后继续生成，从而回答原问题。

### 更新 Route Handler

修改 `app/api/chat/route.ts`，加入 `stopWhen`：

```tsx filename="app/api/chat/route.ts"  highlight="16"
import {
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    stopWhen: isStepCount(5),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

回到浏览器，询问某地天气。此时模型应会利用天气 tool 的结果来作答。

设置 `stopWhen: isStepCount(5)` 表示单次生成最多允许 5 个「步骤」，便于更复杂的交互、分多步收集与处理信息。可再添加一个将摄氏温度转为华氏温度的 tool，观察多步流程。

### 再添加一个 tool

更新 `app/api/chat/route.ts`，新增将华氏温度转为摄氏温度的 tool：

```tsx filename="app/api/chat/route.ts" highlight="31-46"
import {
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    stopWhen: isStepCount(5),
    tools: {
      weather: tool({
        description: 'Get the weather in a location (fahrenheit)',
        inputSchema: z.object({
          location: z.string().describe('The location to get the weather for'),
        }),
        execute: async ({ location }) => {
          const temperature = Math.round(Math.random() * (90 - 32) + 32);
          return {
            location,
            temperature,
          };
        },
      }),
      convertFahrenheitToCelsius: tool({
        description: 'Convert a temperature in fahrenheit to celsius',
        inputSchema: z.object({
          temperature: z
            .number()
            .describe('The temperature in fahrenheit to convert'),
        }),
        execute: async ({ temperature }) => {
          const celsius = Math.round((temperature - 32) * (5 / 9));
          return {
            celsius,
          };
        },
      }),
    },
  });

  return result.toUIMessageStreamResponse();
}
```

当你问「纽约天气多少摄氏度？」之类问题时，应能看到更完整的流程：

1. 模型调用纽约的天气 tool。
2. 界面显示 tool 输出。
3. 接着调用温度换算 tool，把华氏转为摄氏。
4. 模型再用这些信息生成关于纽约天气的自然语言回答。

多步流程让模型能汇总信息并给出更准确、更贴合上下文的回复，聊天机器人会实用得多。

这个简单示例说明 tool 如何扩展模型能力。你还可以构建更复杂的 tool，对接真实 API、数据库或其他外部系统，让模型实时访问并处理真实数据。Tool 弥合了模型知识截止与当前信息之间的鸿沟。
