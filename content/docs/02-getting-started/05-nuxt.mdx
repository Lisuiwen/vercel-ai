# Vue.js (Nuxt) 快速入门

AI SDK 是一个强大的 TypeScript 库，旨在帮助开发者构建 AI 驱动的应用。

在本快速入门教程中，你将构建一个带流式聊天界面的简单智能体；在此过程中，你将了解在自己的项目中使用 SDK 所需的核心概念与方法。

如果你还不熟悉 [提示工程（Prompt Engineering）](/docs/advanced/prompt-engineering) 与 [HTTP 流式传输](/docs/foundations/streaming) 等概念，可以先选读这些文档。

## 先决条件

要完成本快速入门，你需要：

- 在本地开发机上安装 Node.js 18+ 与 pnpm。
- 一个 [Vercel AI Gateway](https://vercel.com/ai-gateway) API 密钥。

若尚未获取 Vercel AI Gateway API 密钥，可在 Vercel 网站[注册](https://vercel.com/d?to=%2F%5Bteam%5D%2F%7E%2Fai&title=Go+to+AI+Gateway)后获取。

### 配置 Vercel AI Gateway API 密钥

在项目根目录创建 `.env` 文件并写入你的 Vercel AI Gateway API 密钥。该密钥用于让你的应用向 Vercel AI Gateway 服务认证。

<Snippet text="touch .env" />

编辑 `.env` 文件：

```env filename=".env"
NUXT_AI_GATEWAY_API_KEY=xxxxxxxxx
```

将 `xxxxxxxxx` 替换为你的真实 Vercel AI Gateway API 密钥，并在 `nuxt.config.ts` 中配置该环境变量：

```ts filename="nuxt.config.ts"
export default defineNuxtConfig({
  // rest of your nuxt config
  runtimeConfig: {
    aiGatewayApiKey: '',
  },
});
```

<Note className="mb-4">
  本指南使用 Nuxt 的 runtime config 管理 API 密钥。环境变量中的
  `NUXT_` 前缀会让 Nuxt 自动将其载入 runtime config。虽然 AI Gateway Provider
  也支持默认的 `AI_GATEWAY_API_KEY` 环境变量，但采用本方式能更好地与 Nuxt
  的配置体系集成。
</Note>

## 创建 API 路由

新建 API 路由 `server/api/chat.ts`，并加入以下代码：

```typescript filename="server/api/chat.ts"
import {
  streamText,
  UIMessage,
  convertToModelMessages,
  createGateway,
} from 'ai';

export default defineLazyEventHandler(async () => {
  const apiKey = useRuntimeConfig().aiGatewayApiKey;
  if (!apiKey) throw new Error('Missing AI Gateway API key');
  const gateway = createGateway({
    apiKey: apiKey,
  });

  return defineEventHandler(async (event: any) => {
    const { messages }: { messages: UIMessage[] } = await readBody(event);

    const result = streamText({
      model: gateway('anthropic/claude-sonnet-4.5'),
      messages: await convertToModelMessages(messages),
    });

    return result.toUIMessageStreamResponse();
  });
});
```

下面简要说明这段代码在做什么：

1. 使用 `ai` 包中的 `createGateway` 创建 gateway provider 实例。
2. 定义 Event Handler，并从请求体中取出 `messages`。`messages` 保存你与聊天机器人之间的对话历史，为下一轮生成提供必要上下文。这些 `messages` 的类型为 `UIMessage`，面向应用 UI 设计——包含完整消息历史及时间戳等元数据。
3. 调用从 `ai` 包导入的 [`streamText`](/docs/reference/ai-sdk-core/stream-text)。该函数接收配置对象，其中包含 `model` provider（步骤 1）与 `messages`（步骤 2）。你还可以传入更多 [设置](/docs/ai-sdk-core/settings) 以进一步调整模型行为。`messages` 字段需要 `ModelMessage[]`，与 `UIMessage` 的区别在于不包含时间戳或发送者等元数据。类型转换使用 `convertToModelMessages`，它会去掉 UI 相关元数据，把 `UIMessage[]` 转成模型期望的 `ModelMessage[]`。
4. `streamText` 返回 [`StreamTextResult`](/docs/reference/ai-sdk-core/stream-text#result)。该结果对象上的 [`toUIMessageStreamResponse`](/docs/reference/ai-sdk-core/stream-text#to-ui-message-stream-response) 会把结果转为可流式输出的响应对象。
5. 将结果返回给客户端以流式返回响应。

## 选择 Provider

AI SDK 通过 [官方](/providers/ai-sdk-providers)、[OpenAI 兼容](/providers/openai-compatible-providers) 及 [社区](/providers/community-providers) 包支持大量模型 provider。

本快速入门使用 [Vercel AI Gateway](https://vercel.com/ai-gateway) provider，它也是默认的[全局 provider](/docs/ai-sdk-core/provider-management#global-provider-configuration)。因此在模型配置里只需写一个简单的字符串即可选用模型：

```ts
model: __MODEL__;
```

你也可以用另外两种等价方式显式导入并使用 gateway provider：

```ts
// Option 1: Import from 'ai' package (included by default)
import { gateway } from 'ai';
model: gateway('anthropic/claude-sonnet-4.5');

// Option 2: Install and import from '@ai-sdk/gateway' package
import { gateway } from '@ai-sdk/gateway';
model: gateway('anthropic/claude-sonnet-4.5');
```

## 接入 UI

有了可以查询 LLM 的 API 路由之后，接下来搭建前端。AI SDK 的 [UI](/docs/ai-sdk-ui/overview) 包把聊天界面的复杂度封装进一个 hook：[`useChat`](/docs/reference/ai-sdk-ui/use-chat)。

用以下代码更新根页面（`pages/index.vue`），以展示聊天消息列表并提供用户输入框：

```typescript filename="pages/index.vue"
<script setup lang="ts">
import { Chat } from "@ai-sdk/vue";
import { ref } from "vue";

const input = ref("");
const chat = new Chat({});

const handleSubmit = (e: Event) => {
    e.preventDefault();
    chat.sendMessage({ text: input.value });
    input.value = "";
};
</script>

<template>
    <div>
        <div v-for="(m, index) in chat.messages" :key="m.id ? m.id : index">
            {{ m.role === "user" ? "User: " : "AI: " }}
            <div
                v-for="(part, index) in m.parts"
                :key="`${m.id}-${part.type}-${index}`"
            >
                <div v-if="part.type === 'text'">{{ part.text }}</div>
            </div>
        </div>

        <form @submit="handleSubmit">
            <input v-model="input" placeholder="Say something..." />
        </form>
    </div>
</template>
```

<Note>
  若你的项目使用 `app.vue` 而不是 `pages/index.vue`，请删除
  `app.vue`，新建 `pages/index.vue` 并粘贴上述代码。
</Note>

该页面使用 `useChat` hook；默认情况下它会请求你之前创建的 API 路由（`/api/chat`）。该 hook 提供处理用户输入与表单提交的函数和状态。`useChat` 还提供多种工具函数与状态：

- `messages`：当前聊天消息（对象数组，含 `id`、`role`、`parts` 等字段）。
- `sendMessage`：向聊天 API 发送消息的函数。

组件用本地状态（`ref`）管理输入框内容，在提交表单时调用 `sendMessage` 发送文本并清空输入。

LLM 的回复通过消息的 `parts` 数组访问。每条消息包含有序的 `parts`，表示模型在该轮输出中的全部内容。其中可包含纯文本、推理 token，以及你稍后会遇到的其他类型。`parts` 数组保留模型输出的顺序，便于按生成顺序展示或处理每条片段。

## 用 Tools 增强聊天机器人

大语言模型（LLM）虽然生成能力很强，但在离散任务（例如数学运算）以及与外部世界交互（例如查天气）方面往往吃力。这时就需要 [tools](/docs/ai-sdk-core/tools-and-tool-calling)。

Tools 是 LLM 可以调用的动作；执行结果可以回传给模型，供其在下一轮回复中参考。

例如用户问当前天气，没有 tools 时模型只能根据训练数据泛泛回答；有了天气 tool，就能获取与用户位置相关的实时天气信息。

下面通过添加一个简单的天气 tool 来增强聊天机器人。

### 更新 API 路由

修改 `server/api/chat.ts`，加入新的天气 tool：

```typescript filename="server/api/chat.ts" highlight="1,16-32"
import {
  createGateway,
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
} from 'ai';
import { z } from 'zod';

export default defineLazyEventHandler(async () => {
  const apiKey = useRuntimeConfig().aiGatewayApiKey;
  if (!apiKey) throw new Error('Missing AI Gateway API key');
  const gateway = createGateway({
    apiKey: apiKey,
  });

  return defineEventHandler(async (event: any) => {
    const { messages }: { messages: UIMessage[] } = await readBody(event);

    const result = streamText({
      model: gateway('anthropic/claude-sonnet-4.5'),
      messages: await convertToModelMessages(messages),
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
      },
    });

    return result.toUIMessageStreamResponse();
  });
});
```

更新后的代码要点：

1. 从 `ai` 导入 `tool`，从 `zod` 导入 `z`，用于 schema 校验。
2. 在 `tools` 中定义名为 `weather` 的 tool，该 tool：
   - 带有描述，帮助模型判断何时调用。
   - 使用 Zod schema 定义 `inputSchema`，要求提供 `location` 字符串才会执行。模型会尝试从对话上下文中抽取该入参；若无法抽取，会向用户追问缺失信息。
   - 定义 `execute` 函数模拟获取天气数据（本例返回随机温度）。该函数在服务器上异步运行，可改为请求真实外部 API。

现在聊天机器人可以为用户询问的任意地点「拉取」天气信息。当模型认为需要调用天气 tool 时，会生成带必要入参的 tool 调用；随后会自动运行 `execute`，并将 tool 输出作为 `tool` 类型的消息加入 `messages`。

可以试试问「纽约天气怎么样？」，观察模型如何使用新 tool。

注意到界面上有一段空白回复？这是因为模型本轮生成的是 tool 调用而非文本。你可以在客户端通过 `message.parts` 里类型为 `tool-weather` 的 part 访问该 tool 调用及后续 tool 结果。

<Note>
  Tool 相关 part 始终命名为 `tool-{toolName}`，其中 `{toolName}` 是你在定义
  tool 时使用的键名。本例中 tool 键为 `weather`，因此 part 类型为
  `tool-weather`。
</Note>

## 启用多步 Tool 调用

你可能已经发现：虽然 tool 已在聊天界面中可见，模型却没有用这些信息回答最初的问题。这是因为一旦模型生成了 tool 调用，从协议意义上说它已完成本轮生成。

要解决这一点，可使用 `stopWhen` 启用多步 tool 调用。默认情况下 `stopWhen` 为 `isStepCount(1)`，即存在 tool 结果时，第一步之后就会停止生成。修改该条件后，可以让模型自动把 tool 结果送回自身，触发后续生成，直到满足你设定的停止条件。这里我们希望模型在拿到天气 tool 结果后继续生成，从而回答最初的问题。

### 更新 API 路由

修改 `server/api/chat.ts`，加入 `stopWhen` 条件：

```typescript filename="server/api/chat.ts" highlight="22"
import {
  createGateway,
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
import { z } from 'zod';

export default defineLazyEventHandler(async () => {
  const apiKey = useRuntimeConfig().aiGatewayApiKey;
  if (!apiKey) throw new Error('Missing AI Gateway API key');
  const gateway = createGateway({
    apiKey: apiKey,
  });

  return defineEventHandler(async (event: any) => {
    const { messages }: { messages: UIMessage[] } = await readBody(event);

    const result = streamText({
      model: gateway('anthropic/claude-sonnet-4.5'),
      messages: await convertToModelMessages(messages),
      stopWhen: isStepCount(5),
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
      },
    });

    return result.toUIMessageStreamResponse();
  });
});
```

回到浏览器，询问某地天气。此时模型应会使用天气 tool 的结果来回答你的问题。

设置 `stopWhen: isStepCount(5)` 表示允许模型在单次生成中最多使用 5 个「步骤」，从而支持更复杂的交互，并在需要时分多步收集与处理信息。若再增加一个将华氏度转为摄氏度的 tool，可以更直观地看到这一过程。

### 再添加一个 tool

更新 `server/api/chat.ts`，新增将华氏度转为摄氏度的 tool：

```typescript filename="server/api/chat.ts" highlight="32-45"
import {
  createGateway,
  streamText,
  UIMessage,
  convertToModelMessages,
  tool,
  isStepCount,
} from 'ai';
import { z } from 'zod';

export default defineLazyEventHandler(async () => {
  const apiKey = useRuntimeConfig().aiGatewayApiKey;
  if (!apiKey) throw new Error('Missing AI Gateway API key');
  const gateway = createGateway({
    apiKey: apiKey,
  });

  return defineEventHandler(async (event: any) => {
    const { messages }: { messages: UIMessage[] } = await readBody(event);

    const result = streamText({
      model: gateway('anthropic/claude-sonnet-4.5'),
      messages: await convertToModelMessages(messages),
      stopWhen: isStepCount(5),
      tools: {
        weather: tool({
          description: 'Get the weather in a location (fahrenheit)',
          inputSchema: z.object({
            location: z
              .string()
              .describe('The location to get the weather for'),
          }),
          execute: async ({ location }) => {
            const temperature = Math.round(Math.random() * (90 - 32) + 32);
            return {
              location,
              temperature,
            };
          },
        }),
        convertFahrenheitToCelsius: tool({
          description: 'Convert a temperature in fahrenheit to celsius',
          inputSchema: z.object({
            temperature: z
              .number()
              .describe('The temperature in fahrenheit to convert'),
          }),
          execute: async ({ temperature }) => {
            const celsius = Math.round((temperature - 32) * (5 / 9));
            return {
              celsius,
            };
          },
        }),
      },
    });

    return result.toUIMessageStreamResponse();
  });
});
```

当你问「纽约天气是多少摄氏度？」时，应能看到更完整的一串交互：

1. 模型为纽约调用天气 tool。
2. 你会看到 tool 输出。
3. 随后模型调用温度换算 tool，把华氏度转为摄氏度。
4. 最后模型用这些信息生成关于纽约天气的自然语言回答。

这种多步方式让模型能先收集信息再作答，回答更准确、更有上下文，聊天机器人也会实用得多。

这个简单示例展示了 tools 如何扩展模型能力。你还可以构建更复杂的 tools，对接真实 API、数据库或其他外部系统，使模型能够实时访问并处理真实世界数据。Tools 弥合了模型知识截止日期与当前信息之间的鸿沟。
