# 浏览库

AI SDK 是构建 AI 应用的强大工具包。本页将帮助你根据需求选择合适的工具。

先从 AI SDK 的快速概览开始，它由三部分组成：

- **[AI SDK Core](/docs/ai-sdk-core/overview)：** 统一的、与 provider 无关的 API，用于通过 LLM 生成文本、结构化对象和 tool 调用。
- **[AI SDK UI](/docs/ai-sdk-ui/overview)：** 一组与框架无关的 hooks，用于构建聊天和生成式用户界面。
- [AI SDK RSC](/docs/ai-sdk-rsc/overview)：使用 React Server Components (RSC) 流式传输生成式用户界面。目前处于实验阶段，我们建议使用 [AI SDK UI](/docs/ai-sdk-ui/overview)。

## 根据环境选择合适工具

在决定使用 AI SDK 的哪一部分时，首先应考虑你所处的环境和现有技术栈。SDK 的不同组件针对特定框架和环境进行了优化。

| Library                                   | Purpose                                                                                                                                                                                                  | Environment Compatibility                                          |
| ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| [AI SDK Core](/docs/ai-sdk-core/overview) | 使用统一 API 调用任意 LLM（例如 [generateText](/docs/reference/ai-sdk-core/generate-text) 和 [streamText](/docs/reference/ai-sdk-core/stream-text)）                                                 | 任意 JS 环境（如 Node.js、Deno、Browser）                   |
| [AI SDK UI](/docs/ai-sdk-ui/overview)     | 构建流式聊天和生成式 UI（例如 [useChat](/docs/reference/ai-sdk-ui/use-chat)）                                                                                                             | React & Next.js、Vue & Nuxt、Svelte & SvelteKit                    |
| [AI SDK RSC](/docs/ai-sdk-rsc/overview)   | 从服务端向客户端流式传输生成式 UI（例如 [streamUI](/docs/reference/ai-sdk-rsc/stream-ui)）。目前处于实验阶段，我们建议使用 [AI SDK UI](/docs/ai-sdk-ui/overview)。 | 支持 React Server Components 的任意框架（如 Next.js） |

## 何时使用 AI SDK UI

AI SDK UI 提供一组与框架无关的 hooks，用于快速构建**生产级 AI 原生应用**。它提供：

- 对流式聊天和客户端生成式 UI 的完整支持
- 处理常见 AI 交互模式（如聊天、补全、助手）的工具
- 经过生产验证的可靠性与性能
- 跨主流框架的兼容性

## 何时使用 AI SDK RSC

<Note type="warning">
  AI SDK RSC 目前处于实验阶段。我们建议在生产环境使用 [AI SDK
  UI](/docs/ai-sdk-ui/overview)。有关从 RSC 迁移到 UI 的指南，请参阅我们的[迁移指南](/docs/ai-sdk-rsc/migrating-to-ui)。
</Note>

[React Server Components](https://nextjs.org/docs/app/building-your-application/rendering/server-components)
(RSC) 为构建 React 应用提供了一种新方法：组件可在服务端渲染、直接获取数据并将结果流式传输到客户端，从而减小 bundle 体积并提升性能。它还引入了一种称为 [Server Actions](https://nextjs.org/docs/app/building-your-application/data-fetching/server-actions-and-mutations) 的新方式，可在应用任意位置调用服务端函数。

AI SDK RSC 提供多种实用工具，允许你将值和 UI 直接从服务端流式传输到客户端。但**需要注意当前的限制**：

- **取消**：目前无法使用 Server Actions 中止流。React 和 Next.js 的未来版本将改进这一点。
- **数据传输增加**：使用 [`createStreamableUI`](/docs/reference/ai-sdk-rsc/create-streamable-ui) 可能导致二次方级别的数据传输（与生成文本长度呈二次方关系）。你可以改用 [ `createStreamableValue` ](/docs/reference/ai-sdk-rsc/create-streamable-value)，并在客户端渲染组件以避免此问题。
- **流式传输期间重新挂载**：使用 `createStreamableUI` 时，组件会在 `.done()` 时重新挂载，导致[闪烁](https://github.com/vercel/ai/issues/2232)。

鉴于这些限制，**我们建议在生产应用中使用 [AI SDK UI](/docs/ai-sdk-ui/overview)**。
