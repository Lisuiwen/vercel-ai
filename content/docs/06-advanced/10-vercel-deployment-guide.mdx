# Vercel 部署指南

本指南将教你如何使用 [Next.js](https://nextjs.org)（App Router）将 AI 应用部署到 [Vercel](https://vercel.com)。

Vercel 是面向开发者的平台，提供更快构建和部署 Web 应用所需的工具、工作流和基础设施，无需额外配置。

Vercel 支持在每次推送到分支以及合并到生产分支时自动部署。对于部署 AI 应用来说，这是一个很好的选择。

## 开始之前

要跟随本指南，你需要：

- Vercel 账户
- Git 提供商账户（本教程使用 [Github](https://github.com)）
- OpenAI API 密钥

本指南将教你如何把 Next.js（App Router）快速入门教程中构建的应用部署到 Vercel。若尚未完成快速入门，可从[此仓库](https://github.com/vercel-labs/ai-sdk-deployment-guide)开始。

## 提交更改

Vercel 提供强大的以 Git 为中心的工作流，每次推送到仓库主分支时都会自动部署到生产环境。

在提交本地更改之前，请确保有 `.gitignore`。在 `.gitignore` 中，应排除环境变量（`.env`）和 node modules（`node_modules`）。

若有本地更改，可运行以下命令提交：

```bash
git add .
git commit -m "init"
```

## 创建 Git 仓库

你可以在终端或 [github.com](https://github.com/) 上创建 GitHub 仓库。本教程使用 GitHub CLI（[更多信息](https://cli.github.com/)）。

创建 GitHub 仓库：

1. 访问 [github.com](http://github.com/)
2. 在右上角点击「加号」图标并选择「New repository」
3. 为仓库选择一个名称（可任意）
4. 点击「Create repository」

创建仓库后，GitHub 会重定向到新仓库页面。

1. 向下滚动，复制标题「...or push an existing repository from the command line」下的命令
2. 回到终端，粘贴并运行这些命令

注意：若遇到错误「error: remote origin already exists.」，说明本地仓库仍链接到克隆的仓库。要「取消链接」，可运行：

```bash
rm -rf .git
git init
git add .
git commit -m "init"
```

然后重新运行上一步的代码片段。

## 在 Vercel 中导入项目

在 [New Project](https://vercel.com/new) 页面的 **Import Git Repository** 部分，选择要导入项目的 Git 提供商，并按提示登录 GitHub 账户。

登录后，你应能在「Import Git Repository」部分看到上一步创建的仓库。点击该项目旁的「Import」按钮。

### 添加环境变量

应用使用环境密钥通过 `.env.local` 在本地开发中存储 OpenAI API 密钥。要将此 API 密钥添加到生产部署，展开「Environment Variables」部分并粘贴 `.env.local` 文件内容。Vercel 会自动解析变量并以适当的 `key:value` 格式填入。

### 部署

点击 **Deploy** 按钮。Vercel 将根据所选配置创建项目并部署。

### 庆祝一下！

要查看部署，在仪表板中选择项目，然后选择 **Domain**。拥有该 URL 的任何人都可以访问此页面。

## 注意事项

部署 AI 应用时，需要注意一些与基础设施相关的事项。

### 函数持续时间

在大多数情况下，你会在服务器上调用大语言模型（LLM）。默认情况下，Vercel 无服务器函数在 Hobby 套餐上的最长持续时间为 10 秒。根据你的提示，LLM 可能需要超过此限制才能完成响应。若在此限制内未完成，服务器将抛出错误。

你可以使用 [route segment config](https://nextjs.org/docs/app/api-reference/file-conventions/route-segment-config) 指定 Vercel 函数的最长持续时间。要更新最长持续时间，在路由处理程序或调用 Server Action 的页面顶部添加以下 route segment config：

```ts
export const maxDuration = 30;
```

在 Hobby 套餐上可将最长持续时间增至 60 秒。其他套餐的限额请参阅[文档](https://vercel.com/docs/functions/runtimes#max-duration)。

## 安全注意事项

鉴于调用 LLM 成本较高，采取保护措施防止应用被滥用很重要。

### 速率限制

速率限制通过定义客户端在给定时间窗口内可向服务器发送的最大请求数来调节网络流量。

请遵循[此指南](https://vercel.com/guides/securing-ai-app-rate-limiting)为应用添加速率限制。

### 防火墙

防火墙有助于保护应用和网站免受 DDoS 攻击和未授权访问。

[Vercel Firewall](https://vercel.com/docs/security/vercel-firewall) 是专为安全设计的工具与基础设施套件。它自动缓解 DDoS 攻击，Enterprise 团队可进一步自定义站点，包括专属支持和 IP 封禁的自定义规则。
