# 停止流

经常需要取消正在进行的流。
例如，当用户意识到响应不是他们想要的时，可能希望停止流。

AI SDK 的不同部分以不同方式支持取消流。

## AI SDK Core

AI SDK 函数提供 `abortSignal` 参数，可用于取消流。
若要从服务端向 LLM API 取消流（例如转发请求中的 `abortSignal`），可使用此参数。

```tsx highlight="10,11,12-16"
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { prompt } = await req.json();

  const result = streamText({
    model: __MODEL__,
    prompt,
    // forward the abort signal:
    abortSignal: req.signal,
    onAbort: ({ steps }) => {
      // Handle cleanup when stream is aborted
      console.log('Stream aborted after', steps.length, 'steps');
      // Persist partial results to database
    },
  });

  return result.toTextStreamResponse();
}
```

## AI SDK UI

hook（例如 `useChat` 或 `useCompletion`）提供 `stop` 辅助函数，可用于取消流。
这将取消从客户端到服务端的流。

<Note type="warning">
  流中止功能与流恢复不兼容。若在 `useChat` 中使用 `resume: true`，中止功能会破坏恢复机制。请在中止或恢复功能中二选一，不要同时使用。
</Note>

```tsx file="app/page.tsx" highlight="9,18-20"
'use client';

import { useCompletion } from '@ai-sdk/react';

export default function Chat() {
  const { input, completion, stop, status, handleSubmit, handleInputChange } =
    useCompletion();

  return (
    <div>
      {(status === 'submitted' || status === 'streaming') && (
        <button type="button" onClick={() => stop()}>
          Stop
        </button>
      )}
      {completion}
      <form onSubmit={handleSubmit}>
        <input value={input} onChange={handleInputChange} />
      </form>
    </div>
  );
}
```

## 处理流中止清理

流被中止时，你可能需要执行清理操作，例如持久化部分结果或释放资源。`onAbort` 回调提供在服务端处理这些场景的方式。

与流正常完成时调用的 `onFinish` 不同，`onAbort` 专门在通过 `AbortSignal` 中止流时调用。这一区别使你能够分别处理正常完成和中止的流。

<Note>
  对于 UI 消息流（`toUIMessageStreamResponse`），`onFinish` 回调还会接收 `isAborted` 参数，表示流是否被中止。这样你可以在单个回调中同时处理完成和中止场景。
</Note>

```tsx highlight="8-12"
import { streamText } from 'ai';
__PROVIDER_IMPORT__;

const result = streamText({
  model: __MODEL__,
  prompt: 'Write a long story...',
  abortSignal: controller.signal,
  onAbort: ({ steps }) => {
    // Called when stream is aborted - persist partial results
    await savePartialResults(steps);
    await logAbortEvent(steps.length);
  },
  onFinish: ({ steps, totalUsage }) => {
    // Called when stream completes normally
    await saveFinalResults(steps, totalUsage);
  },
});
```

`onAbort` 回调接收：

- `steps`：中止前所有已完成步骤的数组

这对以下场景特别有用：

- 将部分对话历史持久化到数据库
- 保存部分进度以便后续继续
- 清理服务端资源或连接
- 记录中止事件用于分析

你也可以在流中通过 `abort` 流部分直接处理中止事件：

```tsx highlight="8-12"
for await (const part of result.fullStream) {
  switch (part.type) {
    case 'text-delta':
      // Handle text delta content
      break;
    case 'abort':
      // Handle abort event directly in stream
      console.log('Stream was aborted');
      break;
    // ... other cases
  }
}
```

## UI 消息流

使用 `toUIMessageStreamResponse` 时，需要以略有不同的方式处理流中止。`onFinish` 回调接收 `isAborted` 参数，你应传入 `consumeStream` 函数以确保正确处理中止：

```tsx highlight="5,19,20-24,26"
import { openai } from '@ai-sdk/openai';
import {
  consumeStream,
  convertToModelMessages,
  streamText,
  UIMessage,
} from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages }: { messages: UIMessage[] } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
    abortSignal: req.signal,
  });

  return result.toUIMessageStreamResponse({
    onFinish: async ({ isAborted }) => {
      if (isAborted) {
        console.log('Stream was aborted');
        // Handle abort-specific cleanup
      } else {
        console.log('Stream completed normally');
        // Handle normal completion
      }
    },
    consumeSseStream: consumeStream,
  });
}
```

`consumeStream` 函数对于 UI 消息流中的正确中止处理是必需的。它确保即使在中止时也能正确消费流，防止潜在的内存泄漏或挂起连接。

## AI SDK RSC

<Note type="warning">
  AI SDK RSC 目前不支持停止流。
</Note>
