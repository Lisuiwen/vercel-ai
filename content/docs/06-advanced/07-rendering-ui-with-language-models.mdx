# 使用语言模型渲染用户界面

语言模型生成文本，因此起初你可能认为应用中只需渲染文本。

```tsx highlight="16" filename="app/actions.tsx"
const text = generateText({
  model: __MODEL__,
  instructions: 'You are a friendly assistant',
  prompt: 'What is the weather in SF?',
  tools: {
    getWeather: {
      description: 'Get the weather for a location',
      inputSchema: z.object({
        city: z.string().describe('The city to get the weather for'),
        unit: z
          .enum(['C', 'F'])
          .describe('The unit to display the temperature in'),
      }),
      execute: async ({ city, unit }) => {
        const weather = getWeather({ city, unit });
        return `It is currently ${weather.value}°${unit} and ${weather.description} in ${city}!`;
      },
    },
  },
});
```

如上，语言模型被传入名为 `getWeather` 的[工具](/docs/ai-sdk-core/tools-and-tool-calling)，以文本形式返回天气信息。然而，若不返回文本而返回表示天气信息的 JSON 对象，即可渲染 React 组件。

```tsx highlight="18-23" filename="app/action.ts"
const text = generateText({
  model: __MODEL__,
  instructions: 'You are a friendly assistant',
  prompt: 'What is the weather in SF?',
  tools: {
    getWeather: {
      description: 'Get the weather for a location',
      inputSchema: z.object({
        city: z.string().describe('The city to get the weather for'),
        unit: z
          .enum(['C', 'F'])
          .describe('The unit to display the temperature in'),
      }),
      execute: async ({ city, unit }) => {
        const weather = getWeather({ city, unit });
        const { temperature, unit, description, forecast } = weather;

        return {
          temperature,
          unit,
          description,
          forecast,
        };
      },
    },
  },
});
```

现在可以使用 `getWeather` 函数返回的对象，条件渲染 React 组件 `<WeatherCard/>`，并将对象作为 props 传入。

```tsx filename="app/page.tsx"
return (
  <div>
    {messages.map(message => {
      // Check assistant message parts for tool results
      if (message.role === 'assistant') {
        return message.parts.map(part => {
          if (
            part.type === 'tool-weather' &&
            part.state === 'output-available'
          ) {
            const { temperature, unit, description, forecast } = part.output;

            return (
              <WeatherCard
                weather={{
                  temperature,
                  unit,
                  description,
                  forecast,
                }}
              />
            );
          }
        });
      }
    })}
  </div>
);
```

以下是可能呈现效果的预览。

<div className="not-prose flex flex-col">
  <CardPlayer
    type="weather"
    title="Weather"
    description="An example of an assistant that renders the weather information in a streamed component."
  />
</div>

在语言模型生成过程中渲染界面可提升应用用户体验，让用户超越纯文本与语言模型交互。

它们还有助于理解以多步进行的[顺序工具调用](/docs/ai-sdk-rsc/multistep-interfaces)，并帮助识别和调试模型推理错误的位置。

## 渲染多个用户界面

回顾一下，应用要在模型生成过程中渲染用户界面，需完成以下步骤：

1. 用户向语言模型发出提示。
2. 语言模型生成包含工具调用的响应。
3. 工具调用返回表示用户界面的 JSON 对象。
4. 响应发送到客户端。
5. 客户端接收响应并检查最新消息是否为工具调用。
6. 若是工具调用，客户端根据工具调用返回的 JSON 对象渲染用户界面。

大多数应用有多个由语言模型调用的工具，每个工具可返回不同的用户界面。

例如，搜索课程的工具可返回课程列表，搜索人员的工具可返回人员列表。随着列表增长，应用复杂度也会增加，管理这些用户界面可能越来越困难。

```tsx filename='app/page.tsx'
{
  message.parts.map(part => {
    if (part.state !== 'output-available') return null;

    switch (part.type) {
      case 'tool-api-search-course':
        return <Courses courses={part.output} />;
      case 'tool-api-search-profile':
        return <People people={part.output} />;
      case 'tool-api-meetings':
        return <Meetings meetings={part.output} />;
      case 'tool-api-search-building':
        return <Buildings buildings={part.output} />;
      case 'tool-api-events':
        return <Events events={part.output} />;
      case 'tool-api-meals':
        return <Meals meals={part.output} />;
      case 'text':
        return <div>{part.text}</div>;
      default:
        return null;
    }
  });
}
```

## 在服务端渲染用户界面

**AI SDK RSC（`@ai-sdk/rsc`）** 利用 RSC 解决在客户端管理所有 React 组件的问题，允许在服务端渲染 React 组件并流式传输到客户端。

你无需根据语言模型返回的数据在客户端条件渲染用户界面，而可在模型生成过程中直接从服务端流式传输。

```tsx highlight="3,22-31,38" filename="app/action.ts"
import { createStreamableUI } from '@ai-sdk/rsc'

const uiStream = createStreamableUI();

const text = generateText({
  model: __MODEL__,
  instructions: 'you are a friendly assistant'
  prompt: 'what is the weather in SF?'
  tools: {
    getWeather: {
      description: 'Get the weather for a location',
      inputSchema: z.object({
        city: z.string().describe('The city to get the weather for'),
        unit: z
          .enum(['C', 'F'])
          .describe('The unit to display the temperature in')
      }),
      execute: async ({ city, unit }) => {
        const weather = getWeather({ city, unit })
        const { temperature, unit, description, forecast } = weather

        uiStream.done(
          <WeatherCard
            weather={{
              temperature: 47,
              unit: 'F',
              description: 'sunny'
              forecast,
            }}
          />
        )
      }
    }
  }
})

return {
  display: uiStream.value
}
```

[`createStreamableUI`](/docs/reference/ai-sdk-rsc/create-streamable-ui) 函数属于 `@ai-sdk/rsc` 模块，创建可向客户端发送 React 组件的流。

在服务端，你用传入的 props 渲染 `<WeatherCard/>` 组件，然后流式传输到客户端。在客户端，只需渲染从服务端流式传输的 UI。

```tsx filename="app/page.tsx" highlight="4"
return (
  <div>
    {messages.map(message => (
      <div>{message.display}</div>
    ))}
  </div>
);
```

现在步骤简化为：

1. 用户向语言模型发出提示。
2. 语言模型生成包含工具调用的响应。
3. 工具调用渲染 React 组件及相关 props，表示用户界面。
4. 响应流式传输到客户端并直接渲染。

> **注意：** 你也可以使用 React Server Components 在服务端渲染文本并流式传输到客户端。这样，从语言模型生成到 UI 渲染的所有操作都可在服务端完成，客户端只需渲染从服务端流式传输的 UI。

完整示例请参阅此[示例](/examples/next-app/interface/stream-component-updates)，了解如何在 Next.js App Router 中使用 React Server Components 流式传输组件更新。
