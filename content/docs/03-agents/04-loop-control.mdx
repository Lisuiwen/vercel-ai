# 循环控制

你可以控制 Agent 循环的执行流程以及每一步的设置。循环持续直到：

- 返回的 finish reasoning 不是 tool-calls，或
- 被调用的工具没有 execute 函数，或
- 工具调用需要审批，或
- 满足停止条件

AI SDK 通过两个参数提供内置循环控制：`stopWhen` 用于定义停止条件，`prepareStep` 用于在步骤之间修改设置（模型、工具、消息等）。

## 停止条件

`stopWhen` 参数控制当最后一步存在工具结果时何时停止执行。默认情况下，Agent 使用 `isStepCount(20)` 在 20 步后停止。此默认值是安全措施，可防止失控循环导致过多的 API 调用和成本。

当你提供 `stopWhen` 时，Agent 在工具调用后继续执行，直到满足停止条件。当条件是数组时，满足任一条件即停止执行。

### 使用内置条件

AI SDK 提供多种内置停止条件：

- `isStepCount(count)` — 在指定步数后停止
- `hasToolCall(...toolNames)` — 在调用任一指定工具时停止
- `isLoopFinished()` — 永不触发，让循环运行直到 Agent 自然完成

### 运行至多指定步数

```ts
import { ToolLoopAgent, isStepCount } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  stopWhen: isStepCount(50), // Increasing the default of 20 to 50.
});

const result = await agent.generate({
  prompt: 'Analyze this dataset and create a summary report',
});
```

### 运行直至完成

如果你希望 Agent 运行直到模型自然停止调用工具，请使用 `isLoopFinished()`。这会移除默认步数限制：

```ts
import { ToolLoopAgent, isLoopFinished } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  stopWhen: isLoopFinished(), // No maximum step limit.
});

const result = await agent.generate({
  prompt: 'Analyze this dataset and create a summary report',
});
```

<Note>
  请谨慎使用 `isLoopFinished()`。没有步数限制时，如果模型持续调用工具，Agent 可能无限运行或产生显著成本。
</Note>

### 组合多个条件

组合多个停止条件。满足任一条件时循环停止：

```ts
import { ToolLoopAgent, isStepCount, hasToolCall } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  stopWhen: [
    isStepCount(20), // Maximum 20 steps
    hasToolCall('someTool', 'done'), // Stop after calling either tool
  ],
});

const result = await agent.generate({
  prompt: 'Research and analyze the topic',
});
```

### 创建自定义条件

为特定需求构建自定义停止条件：

```ts
import { ToolLoopAgent, StopCondition, ToolSet } from 'ai';
__PROVIDER_IMPORT__;

const tools = {
  // your tools
} satisfies ToolSet;

const hasAnswer: StopCondition<typeof tools> = ({ steps }) => {
  // Stop when the model generates text containing "ANSWER:"
  return steps.some(step => step.text?.includes('ANSWER:')) ?? false;
};

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools,
  stopWhen: hasAnswer,
});

const result = await agent.generate({
  prompt: 'Find the answer and respond with "ANSWER: [your answer]"',
});
```

自定义条件接收所有步骤的详细信息：

```ts
const budgetExceeded: StopCondition<typeof tools> = ({ steps }) => {
  const totalUsage = steps.reduce(
    (acc, step) => ({
      inputTokens: acc.inputTokens + (step.usage?.inputTokens ?? 0),
      outputTokens: acc.outputTokens + (step.usage?.outputTokens ?? 0),
    }),
    { inputTokens: 0, outputTokens: 0 },
  );

  const costEstimate =
    (totalUsage.inputTokens * 0.01 + totalUsage.outputTokens * 0.03) / 1000;
  return costEstimate > 0.5; // Stop if cost exceeds $0.50
};
```

## Prepare Step

`prepareStep` 回调在循环中每一步之前运行；如果你不返回任何更改，则默认为初始设置。使用它修改设置、管理上下文，或根据执行历史实现动态行为。

它接收当前步骤的 `messages`，以及 `initialMessages` 和 `responseMessages`（当你需要区分原始输入与先前步骤累积的 assistant/tool 消息时）。将 `messages` 视为循环的当前消息状态：如果你返回 `messages` 覆盖，该覆盖会与每个已完成步骤的响应消息一起，作为后续步骤的基础持久存在。

### 动态模型选择

根据步骤需求切换模型：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: 'openai/gpt-4o-mini', // Default model
  tools: {
    // your tools
  },
  prepareStep: async ({ stepNumber, messages }) => {
    // Use a stronger model for complex reasoning after initial steps
    if (stepNumber > 2 && messages.length > 10) {
      return {
        model: __MODEL__,
      };
    }
    // Continue with default settings
    return {};
  },
});

const result = await agent.generate({
  prompt: '...',
});
```

### 上下文管理

长时间运行的 Agent 可能累积大量工具结果、推理部分和 assistant 消息。使用 `prepareStep` 修改后续步骤将使用的消息状态。这对压缩很有用，由你决定何时进行压缩。

`messages` 值包含将发送给当前步骤的消息。当你从 `prepareStep` 返回 `messages` 覆盖时，更改后的列表会成为后续步骤的基础。随着循环继续，新的 assistant 和 tool 响应消息会追加到其上。如果你需要从零散片段而非持久消息状态重建步骤，请对原始输入使用 `initialMessages`，对迄今累积的 model/tool 响应消息使用 `responseMessages`。

`pruneMessages` 辅助函数提供内置方式移除选定消息。当你想要简单的压缩策略时，可在 `prepareStep` 内使用它。

```ts
import { ToolLoopAgent, pruneMessages, type ModelMessage } from 'ai';
__PROVIDER_IMPORT__;

const COMPACTION_THRESHOLD = 100_000;

const estimateTokens = (messages: ModelMessage[]) => {
  return JSON.stringify(messages).length / 4;
};

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  prepareStep: async ({ messages }) => {
    if (estimateTokens(messages) > COMPACTION_THRESHOLD) {
      return {
        messages: pruneMessages({
          messages,
          reasoning: 'all',
          toolCalls: 'before-last-3-messages',
          emptyMessages: 'remove',
        }),
      };
    }
  },
});

const result = await agent.generate({
  prompt: '...',
});
```

上述 token 估算器故意保持简单，仅演示决定何时压缩的一种方式。要点是 `prepareStep` 可以返回新的 `messages` 数组，该数组会成为后续步骤的消息状态。相同模式也适用于 `generateText` 和 `streamText`。

### 工具选择

控制每一步可用的工具：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    search: searchTool,
    analyze: analyzeTool,
    summarize: summarizeTool,
  },
  prepareStep: async ({ stepNumber, steps }) => {
    // Search phase (steps 0-2)
    if (stepNumber <= 2) {
      return {
        activeTools: ['search'],
        toolChoice: 'required',
      };
    }

    // Analysis phase (steps 3-5)
    if (stepNumber <= 5) {
      return {
        activeTools: ['analyze'],
      };
    }

    // Summary phase (step 6+)
    return {
      activeTools: ['summarize'],
      toolChoice: 'required',
    };
  },
});

const result = await agent.generate({
  prompt: '...',
});
```

你也可以强制使用特定工具：

```ts
prepareStep: async ({ stepNumber }) => {
  if (stepNumber === 0) {
    // Force the search tool to be used first
    return {
      toolChoice: { type: 'tool', toolName: 'search' },
    };
  }

  if (stepNumber === 5) {
    // Force the summarize tool after analysis
    return {
      toolChoice: { type: 'tool', toolName: 'summarize' },
    };
  }

  return {};
};
```

### 消息修改

在将消息发送给模型之前进行转换。返回的消息会传递到后续步骤，因此后续的 `messages` 值包含你转换后的消息以及已完成步骤的 assistant/tool 响应消息：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  prepareStep: async ({ messages, stepNumber }) => {
    // Summarize tool results to reduce token usage
    const processedMessages = messages.map(msg => {
      if (msg.role === 'tool' && msg.content.length > 1000) {
        return {
          ...msg,
          content: summarizeToolResult(msg.content),
        };
      }
      return msg;
    });

    return { messages: processedMessages };
  },
});

const result = await agent.generate({
  prompt: '...',
});
```

### 实验性 Sandbox 选择

切换单步工具执行使用的 experimental sandbox：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    runCommand,
  },
  experimental_sandbox: defaultSandbox,
  prepareStep: async ({ stepNumber }) => {
    if (stepNumber === 0) {
      return {
        experimental_sandbox: setupSandbox,
      };
    }

    return {};
  },
});

const result = await agent.generate({
  prompt: '...',
});
```

与 `runtimeContext` 和 `toolsContext` 不同，从 `prepareStep` 返回的 experimental sandbox 仅适用于该步的工具执行。后续步骤使用顶层 `experimental_sandbox`，除非它们返回自己的 experimental sandbox 覆盖。

## 访问步骤信息

`stopWhen` 和 `prepareStep` 都会接收有关当前执行的详细信息：

```ts
prepareStep: async ({
  model, // Current model configuration
  stepNumber, // Current step number (0-indexed)
  steps, // All previous steps with their results
  messages, // Messages to be sent to the model
}) => {
  // Access previous tool calls and results
  const previousToolCalls = steps.flatMap(step => step.toolCalls);
  const previousResults = steps.flatMap(step => step.toolResults);

  // Make decisions based on execution history
  if (previousToolCalls.some(call => call.toolName === 'dataAnalysis')) {
    return {
      toolChoice: { type: 'tool', toolName: 'reportGenerator' },
    };
  }

  return {};
},
```

## 强制工具调用

你可以通过将 `toolChoice: 'required'` 与没有 `execute` 函数的 `done` 工具结合，强制 Agent 始终使用工具。此模式确保 Agent 在每一步都使用工具，并仅在显式发出完成信号时停止。

```ts
import { ToolLoopAgent, tool } from 'ai';
import { z } from 'zod';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    search: searchTool,
    analyze: analyzeTool,
    done: tool({
      description: 'Signal that you have finished your work',
      inputSchema: z.object({
        answer: z.string().describe('The final answer'),
      }),
      // No execute function - stops the agent when called
    }),
  },
  toolChoice: 'required', // Force tool calls at every step
});

const result = await agent.generate({
  prompt: 'Research and analyze this topic, then provide your answer.',
});

// extract answer from done tool call
const toolCall = result.staticToolCalls[0]; // tool call from final step
if (toolCall?.toolName === 'done') {
  console.log(toolCall.input.answer);
}
```

此模式的关键方面：

- **`toolChoice: 'required'`**：强制模型在每一步调用工具，而不是直接生成文本。这确保 Agent 遵循结构化工作流。
- **没有 `execute` 的 `done` 工具**：没有 `execute` 函数的工具充当终止信号。当 Agent 调用此工具时，循环停止，因为没有可执行的函数。
- **访问结果**：最终答案在 `result.staticToolCalls` 中可用，其中包含未执行的工具调用。

当你希望 Agent 始终使用特定工具进行操作（如代码执行或数据检索），而不是直接回答时，此模式很有用。

## 手动循环控制

对于需要完全控制 Agent 循环的场景，你可以使用 AI SDK Core 函数（`generateText` 和 `streamText`）实现自己的循环管理，而不是使用 `stopWhen` 和 `prepareStep`。此方法为复杂工作流提供最大灵活性。

### 实现手动循环

当你需要对执行进行完全控制时，构建自己的 Agent 循环：

```ts
import { generateText, ModelMessage } from 'ai';
__PROVIDER_IMPORT__;

const messages: ModelMessage[] = [{ role: 'user', content: '...' }];

let step = 0;
const maxSteps = 10;

while (step < maxSteps) {
  const result = await generateText({
    model: __MODEL__,
    messages,
    tools: {
      // your tools here
    },
  });

  messages.push(...result.responseMessages);

  if (result.text) {
    break; // Stop when model generates text
  }

  step++;
}
```

此手动方法让你完全控制：

- 消息历史管理
- 逐步决策
- 自定义停止条件
- 动态工具和模型选择
- 错误处理与恢复
