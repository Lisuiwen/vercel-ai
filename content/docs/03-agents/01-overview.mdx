# Agents

Agent 是在**循环**中使用**工具**来完成任务的**大语言模型（LLM）**。

这些组件协同工作：

- **LLM** 处理输入并决定下一步操作
- **工具** 将能力扩展到文本生成之外（读取文件、调用 API、写入数据库）
- **循环** 通过以下方式编排执行：
  - **上下文管理** - 维护对话历史，并决定每一步模型看到什么（输入）
  - **停止条件** - 判断循环（任务）何时完成

## Agent 状态与上下文

Agent 通常需要不应直接放入提示词的服务端状态，例如租户设置、请求 ID、功能开关、凭据或任务进度。

使用 `runtimeContext` 作为 Agent 的共享运行时状态。它流经 Agent 循环，在 `prepareStep` 和生命周期回调中可用，并可在步骤之间更新。使用 `toolsContext` 存放每个工具的值，例如 API 密钥或范围权限；每个工具仅根据其 `contextSchema` 接收自己类型化的 `context`。

了解更多请参阅[运行时与工具上下文](/docs/ai-sdk-core/runtime-and-tool-context)。

## ToolLoopAgent 类

ToolLoopAgent 类处理上述三个组件。下面是一个在循环中使用多个工具完成任务的 Agent：

```ts
import { ToolLoopAgent, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const weatherAgent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    weather: tool({
      description: 'Get the weather in a location (in Fahrenheit)',
      inputSchema: z.object({
        location: z.string().describe('The location to get the weather for'),
      }),
      execute: async ({ location }) => ({
        location,
        temperature: 72 + Math.floor(Math.random() * 21) - 10,
      }),
    }),
    convertFahrenheitToCelsius: tool({
      description: 'Convert temperature from Fahrenheit to Celsius',
      inputSchema: z.object({
        temperature: z.number().describe('Temperature in Fahrenheit'),
      }),
      execute: async ({ temperature }) => {
        const celsius = Math.round((temperature - 32) * (5 / 9));
        return { celsius };
      },
    }),
  },
});

const result = await weatherAgent.generate({
  prompt: 'What is the weather in San Francisco in celsius?',
});

console.log(result.text); // agent's final answer
console.log(result.steps); // steps taken by the agent
```

Agent 会自动：

1. 调用 `weather` 工具获取华氏温度
2. 调用 `convertFahrenheitToCelsius` 进行转换
3. 生成包含最终结果的文本响应

ToolLoopAgent 负责处理循环、上下文管理和停止条件。

## 为何使用 ToolLoopAgent？

ToolLoopAgent 是使用 AI SDK 构建 Agent 的推荐方式，因为它可以：

- **减少样板代码** - 管理循环和消息数组
- **提高可复用性** - 定义一次，在应用中随处使用
- **简化维护** - 在一处更新 Agent 配置

对于大多数用例，请从 ToolLoopAgent 开始。当你需要对复杂结构化工作流的每一步进行显式控制时，再使用核心函数（`generateText`、`streamText`）。

## 结构化工作流

Agent 灵活且强大，但具有非确定性。当你需要可靠、可重复的结果和显式控制流时，请使用核心函数配合结构化工作流模式，结合：

- 用于显式分支的条件语句
- 用于可复用逻辑的标准函数
- 用于健壮性的错误处理
- 用于可预测性的显式控制流

[探索工作流模式](/docs/agents/workflows)，了解更多关于构建结构化、可靠系统的知识。
