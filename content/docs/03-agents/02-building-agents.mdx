# 构建 Agent

ToolLoopAgent 提供了一种结构化方式，将 LLM 配置、工具和行为封装为可复用组件。它为你处理 Agent 循环，使 LLM 能够连续多次调用工具以完成复杂任务。定义一次 Agent，即可在应用中随处使用。

## 为何使用 ToolLoopAgent 类？

构建 AI 应用时，你通常需要：

- **复用配置** - 在应用不同部分使用相同的模型设置、工具和提示词
- **保持一致性** - 确保整个代码库中的行为和能力一致
- **简化 API 路由** - 减少端点中的样板代码
- **类型安全** - 为 Agent 的工具和输出获得完整的 TypeScript 支持

ToolLoopAgent 类提供单一位置来定义 Agent 的行为。

## 创建 Agent

通过使用所需配置实例化 ToolLoopAgent 类来定义 Agent：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const myAgent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
  tools: {
    // Your tools here
  },
});
```

## 配置选项

ToolLoopAgent 接受与 `generateText` 和 `streamText` 相同的所有设置。可配置：

### 模型与系统指令

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are an expert software engineer.',
});
```

### 工具

提供 Agent 用于完成任务的 tools：

```ts
import { ToolLoopAgent, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const codeAgent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    runCode: tool({
      description: 'Execute Python code',
      inputSchema: z.object({
        code: z.string(),
      }),
      execute: async ({ code }) => {
        // Execute code and return result
        return { output: 'Code executed successfully' };
      },
    }),
  },
});
```

### 上下文与 Agent 状态

使用 `runtimeContext` 作为 Agent 的共享运行时状态。它流经 Agent 循环，在 `prepareStep`、生命周期回调和最终结果中可用。如果工具需要凭据、范围权限或默认设置等服务端值，请通过 `toolsContext` 传递，并使用工具的 `contextSchema` 声明它们。

```ts highlight="8-15,21-31,37-49"
import { ToolLoopAgent, tool } from 'ai';
import { z } from 'zod';

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    searchTickets: tool({
      description: 'Search support tickets',
      inputSchema: z.object({
        query: z.string(),
      }),
      contextSchema: z.object({
        apiKey: z.string(),
        accountId: z.string(),
      }),
      execute: async ({ query }, { context }) =>
        searchTickets(query, context.accountId, context.apiKey),
    }),
  },
  prepareStep: async ({ runtimeContext }) => {
    if (runtimeContext.escalated) {
      return { temperature: 0.1 };
    }

    return {};
  },
});

const result = await agent.generate({
  prompt: 'Find open billing tickets for this account.',
  runtimeContext: {
    requestId: 'req_abc',
    escalated: false,
  },
  toolsContext: {
    searchTickets: {
      apiKey: process.env.SUPPORT_API_KEY!,
      accountId: 'acct_123',
    },
  },
});
```

关于完整模型（包括敏感上下文过滤以及每个上下文值的可用位置），请参阅[运行时与工具上下文](/docs/ai-sdk-core/runtime-and-tool-context)。

### 使用实验性 Sandbox 的工具

当 Agent 工具需要命令或代码执行环境时，传递 `experimental_sandbox`。实验性 sandbox 是每次调用的值，因此请在 `generate()`、`stream()` 或调用 Agent 的 UI 流辅助函数时提供。

```ts highlight="8-16,25"
const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a coding assistant. Use the shell tool when needed.',
  tools: {
    shell: tool({
      description: 'Execute shell commands in the experimental sandbox.',
      inputSchema: z.object({
        command: z.string(),
        workingDirectory: z.string().optional(),
      }),
      execute: async (
        { command, workingDirectory },
        { abortSignal, experimental_sandbox },
      ) => {
        if (!experimental_sandbox) {
          throw new Error('Experimental sandbox is not available');
        }

        return experimental_sandbox.runCommand({
          command,
          workingDirectory,
          abortSignal,
        });
      },
    }),
  },
});

const result = await agent.generate({
  prompt: `Run the tests.\n\nSandbox:\n${experimental_sandbox.description}`,
  experimental_sandbox,
});
```

实验性 sandbox 描述不会自动添加到模型提示词中。当模型需要了解环境详情时，请将其包含在 prompt 或 instructions 中。传递 experimental sandbox 并不会对工具本身进行沙箱化；工具必须显式将操作委托给 experimental sandbox。更多详情请参阅[工具调用中的实验性 Sandbox 部分](/docs/ai-sdk-core/tools-and-tool-calling#experimental-sandbox)。

你也可以在工具执行前要求审批。在 `ToolLoopAgent` 上使用 `toolApproval` 进行配置：

```ts
const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    runCode: tool({
      description: 'Execute Python code',
      inputSchema: z.object({
        code: z.string(),
      }),
      execute: async ({ code }) => ({ output: code }),
    }),
  },
  toolApproval: {
    runCode: 'user-approval',
  },
});
```

关于手动审批、自动批准与拒绝、动态策略函数以及 `useChat` 集成，请参阅[工具审批](/docs/agents/tool-approvals)。

### 循环控制

默认情况下，Agent 运行 20 步（`stopWhen: isStepCount(20)`）。在每一步中，模型要么生成文本，要么调用工具。如果生成文本，Agent 完成。如果调用工具，AI SDK 会执行该工具。

你可以不同地配置 `stopWhen` 以允许更多步骤。每次工具执行后，Agent 会触发新的生成，模型可以再次调用工具或生成文本：

```ts
import { ToolLoopAgent, isStepCount } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  stopWhen: isStepCount(50), // Increase default from 20 to 50.
});
```

每一步代表一次生成（结果为文本或工具调用）。循环持续直到：

- 返回的 finish reasoning 不是 tool-calls，或
- 被调用的工具没有 execute 函数，或
- 工具调用需要审批，或
- 满足停止条件

你可以组合多个条件：

```ts
import { ToolLoopAgent, isStepCount } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  stopWhen: [
    isStepCount(20), // Maximum 20 steps
    yourCustomCondition(), // Custom logic for when to stop
  ],
});
```

了解更多请参阅[循环控制与停止条件](/docs/agents/loop-control)。

### 工具选择

控制 Agent 如何使用工具：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools here
  },
  toolChoice: 'required', // Force tool use
  // or toolChoice: 'none' to disable tools
  // or toolChoice: 'auto' (default) to let the model decide
});
```

你也可以强制使用特定工具：

```ts
import { ToolLoopAgent } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    weather: weatherTool,
    cityAttractions: attractionsTool,
  },
  toolChoice: {
    type: 'tool',
    toolName: 'weather', // Force the weather tool to be used
  },
});
```

### 结构化输出

定义结构化输出 schema：

```ts
import { ToolLoopAgent, Output } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const analysisAgent = new ToolLoopAgent({
  model: __MODEL__,
  output: Output.object({
    schema: z.object({
      sentiment: z.enum(['positive', 'neutral', 'negative']),
      summary: z.string(),
      keyPoints: z.array(z.string()),
    }),
  }),
});

const { output } = await analysisAgent.generate({
  prompt: 'Analyze customer feedback from the last quarter',
});
```

## 使用系统指令定义 Agent 行为

系统指令定义 Agent 的行为、个性和约束。它们为所有交互设置上下文，并指导 Agent 如何响应用户查询和使用工具。

### 基本系统指令

设置 Agent 的角色和专长：

```ts
const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions:
    'You are an expert data analyst. You provide clear insights from complex data.',
});
```

### 详细行为指令

为 Agent 行为提供具体指南：

```ts
const codeReviewAgent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: `You are a senior software engineer conducting code reviews.

  Your approach:
  - Focus on security vulnerabilities first
  - Identify performance bottlenecks
  - Suggest improvements for readability and maintainability
  - Be constructive and educational in your feedback
  - Always explain why something is an issue and how to fix it`,
});
```

### 约束 Agent 行为

设置边界并确保一致的行为：

```ts
const customerSupportAgent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: `You are a customer support specialist for an e-commerce platform.

  Rules:
  - Never make promises about refunds without checking the policy
  - Always be empathetic and professional
  - If you don't know something, say so and offer to escalate
  - Keep responses concise and actionable
  - Never share internal company information`,
  tools: {
    checkOrderStatus,
    lookupPolicy,
    createTicket,
  },
});
```

### 工具使用指令

指导 Agent 如何使用可用工具：

```ts
const researchAgent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: `You are a research assistant with access to search and document tools.

  When researching:
  1. Always start with a broad search to understand the topic
  2. Use document analysis for detailed information
  3. Cross-reference multiple sources before drawing conclusions
  4. Cite your sources when presenting information
  5. If information conflicts, present both viewpoints`,
  tools: {
    webSearch,
    analyzeDocument,
    extractQuotes,
  },
});
```

### 格式与风格指令

控制输出格式和沟通风格：

```ts
const technicalWriterAgent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: `You are a technical documentation writer.

  Writing style:
  - Use clear, simple language
  - Avoid jargon unless necessary
  - Structure information with headers and bullet points
  - Include code examples where relevant
  - Write in second person ("you" instead of "the user")

  Always format responses in Markdown.`,
});
```

## 使用 Agent

定义完成后，你可以通过三种方式使用 Agent：

### 生成文本

使用 `generate()` 进行一次性文本生成：

```ts
const result = await myAgent.generate({
  prompt: 'What is the weather like?',
});

console.log(result.text);
```

### 流式文本

使用 `stream()` 进行流式响应：

```ts
const result = await myAgent.stream({
  prompt: 'Tell me a story',
});

for await (const chunk of result.textStream) {
  console.log(chunk);
}
```

### 响应 UI 消息

使用 `createAgentUIStreamResponse()` 为客户端应用创建 API 响应：

```ts
// In your API route (e.g., app/api/chat/route.ts)
import { createAgentUIStreamResponse } from 'ai';

export async function POST(request: Request) {
  const { messages } = await request.json();

  return createAgentUIStreamResponse({
    agent: myAgent,
    uiMessages: messages,
  });
}
```

### 生命周期回调

<Note type="warning">
  实验性回调在增量包发布中可能发生破坏性变更。
</Note>

Agent 提供生命周期回调，让你可以挂接到 Agent 执行的不同阶段。
这些回调适用于日志、可观测性、调试和自定义遥测。

```ts
const result = await myAgent.generate({
  prompt: 'Research and summarize the latest AI trends',

  experimental_onStart({ modelId }) {
    console.log('Agent started', { modelId });
  },

  experimental_onStepStart({ stepNumber, modelId }) {
    console.log(`Step ${stepNumber} starting`, { modelId });
  },

  onToolExecutionStart({ toolCall }) {
    console.log(`Tool call starting: ${toolCall.toolName}`);
  },

  onToolExecutionEnd({ toolCall, toolExecutionMs, toolOutput }) {
    console.log(
      `Tool call finished: ${toolCall.toolName} (${toolExecutionMs}ms)`,
      {
        success: toolOutput.type === 'tool-result',
      },
    );
  },

  onStepFinish({ stepNumber, usage, performance, finishReason, toolCalls }) {
    console.log(`Step ${stepNumber} completed:`, {
      inputTokens: usage.inputTokens,
      outputTokens: usage.outputTokens,
      outputTokensPerSecond: performance.effectiveOutputTokensPerSecond,
      stepTimeMs: performance.stepTimeMs,
      finishReason,
      toolsUsed: toolCalls?.map(tc => tc.toolName),
    });
  },

  onFinish({ totalUsage, steps }) {
    console.log('Agent finished:', {
      totalSteps: steps.length,
      totalTokens: totalUsage.totalTokens,
    });
  },
});
```

可用的生命周期回调包括：

- **`experimental_onStart`**：在 Agent 操作开始时调用一次，在任何 LLM 调用之前。接收模型信息、消息、设置和 `runtimeContext`。
- **`experimental_onStepStart`**：在每一步（LLM 调用）之前调用。接收步骤编号、模型、发送的消息、工具和先前步骤。
- **`onToolExecutionStart`**：在工具的 `execute` 函数运行之前立即调用。接收工具调用对象、消息和 `toolContext`。
- **`onToolExecutionEnd`**：在工具的 `execute` 函数完成或出错之后立即调用。接收工具调用、`toolExecutionMs` 以及判别联合 `toolOutput`（`type: 'tool-result'` 含 `output`，或 `type: 'tool-error'` 含 `error`）。
- **`onStepFinish`**：在每一步完成后调用。接收包括用量、性能、finish reason 和工具调用在内的步骤结果。
- **`onFinish`**：在所有步骤完成且响应就绪时调用。接收所有步骤结果、总用量和 `runtimeContext`。

#### 构造函数 vs. 方法回调

所有生命周期回调都可以在构造函数中定义（用于 Agent 范围的跟踪），在 `generate()`/`stream()` 调用中定义（用于每次调用的跟踪），或两者兼有。当两者都提供时，两者都会被调用（先构造函数，后方法回调）：

```ts
const agent = new ToolLoopAgent({
  model: __MODEL__,
  onStepFinish: async ({ stepNumber, usage }) => {
    // Agent-wide logging
    console.log(`Agent step ${stepNumber}:`, usage.totalTokens);
  },
});

// Method-level callback runs after constructor callback
const result = await agent.generate({
  prompt: 'Hello',
  onStepFinish: async ({ stepNumber, usage }) => {
    // Per-call tracking (e.g., for billing)
    await trackUsage(stepNumber, usage);
  },
});
```

## 端到端类型安全

你可以为 Agent 的 `UIMessage` 推断类型：

```ts
import { ToolLoopAgent, InferAgentUIMessage } from 'ai';

const myAgent = new ToolLoopAgent({
  // ... configuration
});

// Infer the UIMessage type for UI components or persistence
export type MyAgentUIMessage = InferAgentUIMessage<typeof myAgent>;
```

在客户端组件中与 `useChat` 一起使用此类型：

```tsx filename="components/chat.tsx"
'use client';

import { useChat } from '@ai-sdk/react';
import type { MyAgentUIMessage } from '@/agent/my-agent';

export function Chat() {
  const { messages } = useChat<MyAgentUIMessage>();
  // Full type safety for your messages and tools
}
```
