# 工具审批

默认情况下，带有 `execute` 函数的工具在模型调用时会自动运行。在 `ToolLoopAgent` 上使用 `toolApproval` 可在执行前审查、批准或拒绝选定的工具调用。

`toolApproval` 适用于可能修改数据、花费资金、执行代码、发送消息、访问私有数据或执行任何其他敏感操作的工具。

<Note>
  `toolApproval` 适用于由 AI SDK 执行的工具。提供商执行的工具在提供商侧运行，不使用 AI SDK 工具审批。
</Note>

## 状态

每个审批规则返回以下状态之一，可以是字符串或带有 `type` 字段的对象：

- `'not-applicable'`：正常执行工具，无审批元数据。这是默认值。
- `'approved'`：记录自动批准，然后执行工具。
- `'denied'`：记录自动拒绝并返回被拒绝的工具输出。
- `'user-approval'`：发出审批请求并等待显式响应。

对于自动批准和拒绝，当你想包含原因时使用对象形式：

```ts
toolApproval: {
  deleteFile: {
    type: 'denied',
    reason: 'Deleting files is disabled in this workspace',
  },
}
```

审批函数也可以返回 `undefined`，与 `'not-applicable'` 同等对待。

## 要求对工具进行审批

当每个工具有简单策略时使用按工具映射。

```ts highlight="13-15"
import { ToolLoopAgent, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    runCommand: tool({
      inputSchema: z.object({ command: z.string() }),
      execute: async ({ command }) => runCommand(command),
    }),
  },
  toolApproval: {
    runCommand: 'user-approval',
  },
});
```

当调用 `runCommand` 时，Agent 返回 `tool-approval-request` 而不是执行工具。

## 根据工具输入决定

当决策取决于解析后的工具输入时，使用按工具审批函数。该函数接收类型化输入以及 `toolCallId`、`messages`、`toolContext` 和 `runtimeContext`。

```ts highlight="22-25"
import { ToolLoopAgent, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    processPayment: tool({
      inputSchema: z.object({
        amount: z.number(),
        recipient: z.string(),
      }),
      execute: async ({ amount, recipient }) =>
        processPayment({ amount, recipient }),
    }),
  },
  toolApproval: {
    processPayment: async ({ amount }, { runtimeContext }) => {
      if (runtimeContext.role !== 'admin') {
        return { type: 'denied', reason: 'Only admins can send payments' };
      }
      return amount > 1000 ? 'user-approval' : undefined;
    },
  },
});
```

在此示例中，非管理员用户被自动拒绝，大额支付需要手动审批，较小的管理员支付正常执行。

## 对所有工具使用单一策略

当审批取决于完整工具调用、跨工具共享状态或完整工具集时，直接将函数作为 `toolApproval` 传递。这称为 `GenericToolApprovalFunction`。

```ts highlight="17-28"
const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    readFile: tool({
      inputSchema: z.object({ path: z.string() }),
      execute: async ({ path }) => readFile(path),
    }),
    deleteFile: tool({
      inputSchema: z.object({ path: z.string() }),
      execute: async ({ path }) => deleteFile(path),
    }),
  },
  toolApproval: ({ toolCall }) => {
    if (toolCall.dynamic) {
      return 'user-approval';
    }

    if (toolCall.toolName === 'deleteFile') {
      return 'user-approval';
    }

    return undefined;
  },
});
```

通用函数接收：

- `toolCall`：完整工具调用，包括 `toolName`、`toolCallId`、`input` 以及是否为 dynamic。
- `tools`：模型可用的所有工具。
- `toolsContext`：所有工具的上下文。
- `messages`：产生工具调用的步骤发送给模型的消息。
- `runtimeContext`：调用的共享运行时上下文。

## 按请求配置审批

因为 `toolApproval` 是 Agent 设置，你也可以从 `prepareCall` 返回它。当审批策略取决于调用选项、租户策略或用户权限时很有用。

```ts highlight="8-10,11-18"
import { ToolLoopAgent, tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const agent = new ToolLoopAgent({
  model: __MODEL__,
  callOptionsSchema: z.object({
    canRunCommands: z.boolean(),
  }),
  prepareCall: ({ options, ...settings }) => ({
    ...settings,
    toolApproval: {
      runCommand: options.canRunCommands
        ? 'user-approval'
        : { type: 'denied', reason: 'Command access is disabled' },
    },
  }),
  tools: {
    runCommand: tool({
      inputSchema: z.object({ command: z.string() }),
      execute: async ({ command }) => runCommand(command),
    }),
  },
});
```

## 处理手动审批

手动审批需要两次调用：

1. 使用 `toolApproval` 调用 `agent.generate()` 或 `agent.stream()`。
2. 从结果或 UI 流中读取 `tool-approval-request`。
3. 向用户或审批系统询问决定。
4. 向消息添加 `tool-approval-response`。
5. 使用更新后的消息再次调用 Agent。

```ts highlight="11-18,21-28"
import { type ModelMessage, type ToolApprovalResponse } from 'ai';

const messages: ModelMessage[] = [{ role: 'user', content: 'Delete temp.txt' }];

const result = await agent.generate({ messages });
messages.push(...result.responseMessages);

const approvalResponses: ToolApprovalResponse[] = [];

for (const part of result.content) {
  if (part.type === 'tool-approval-request' && !part.isAutomatic) {
    approvalResponses.push({
      type: 'tool-approval-response',
      approvalId: part.approvalId,
      approved: true,
      reason: 'User confirmed the file can be deleted',
    });
  }
}

messages.push({
  role: 'tool',
  content: approvalResponses,
});

const finalResult = await agent.generate({ messages });
```

如果批准，工具在第二次调用时运行。如果拒绝，模型会收到拒绝信息，可以在没有工具结果的情况下响应。

<Note>
  当工具执行被拒绝时，考虑添加类似「当工具执行未获批准时，不要重试」的指令，以防止对同一操作重复请求审批。
</Note>

## 与 `useChat` 一起使用

当将 Agent 流式传输到聊天 UI 时，审批请求会显示为 `state: 'approval-requested'` 的工具部分。使用 `addToolApprovalResponse` 响应。

```tsx highlight="6,14-28"
'use client';

import { useChat } from '@ai-sdk/react';
import { lastAssistantMessageIsCompleteWithApprovalResponses } from 'ai';

export default function Chat() {
  const { messages, addToolApprovalResponse } = useChat({
    sendAutomaticallyWhen: lastAssistantMessageIsCompleteWithApprovalResponses,
  });

  return messages.map(message =>
    message.parts.map(part => {
      if (part.type !== 'tool-runCommand') {
        return null;
      }

      if (part.state === 'approval-requested' && !part.approval.isAutomatic) {
        return (
          <div key={part.toolCallId}>
            <button
              onClick={() =>
                addToolApprovalResponse({
                  id: part.approval.id,
                  approved: true,
                })
              }
            >
              Approve
            </button>
            <button
              onClick={() =>
                addToolApprovalResponse({
                  id: part.approval.id,
                  approved: false,
                })
              }
            >
              Deny
            </button>
          </div>
        );
      }
    }),
  );
}
```

仅对手动审批调用 `addToolApprovalResponse`。自动批准和拒绝已在流中包含审批状态。

## 相关 API

- 将 `toolApproval` 与 `ToolLoopAgent`、`generateText` 和 `streamText` 一起使用。
- 仅将 `needsApproval` 与 [`WorkflowAgent`](/docs/agents/workflow-agent) 一起使用，其中审批会暂停并恢复持久工作流执行。
- 子 Agent 工具不能使用 `toolApproval`；请参阅[子 Agent](/docs/agents/subagents#no-tool-approvals-in-subagents)。
