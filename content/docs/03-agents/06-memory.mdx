# 记忆

Memory 让你的 Agent 能够保存信息并在之后回忆。没有 memory 时，每次对话都从零开始。有了 memory，Agent 可以随时间构建上下文、回忆先前的交互并适应用户。

## 三种方式

你可以通过 AI SDK 以三种方式为 Agent 添加 memory，各有不同的权衡：

| 方式                                          | 工作量 | 灵活性 | 提供商锁定           |
| --------------------------------------------- | ------ | ----------- | -------------------------- |
| [提供商定义的工具](#provider-defined-tools) | 低    | 中等      | 是                        |
| [记忆提供商](#memory-providers)             | 低    | 低         | 取决于记忆提供商 |
| [自定义工具](#custom-tool)                       | 高   | 高        | 否                         |

## 提供商定义的工具

[提供商定义的工具](/docs/foundations/tools#types-of-tools) 是提供商指定工具的 `inputSchema` 和 `description`，但你提供 `execute` 函数的工具。模型经过训练以使用这些工具，与自定义工具相比可能获得更好的性能。

### Anthropic Memory Tool

[Anthropic Memory Tool](https://platform.claude.com/docs/en/agents-and-tools/tool-use/memory-tool) 为 Claude 提供管理 `/memories` 目录的结构化接口。Claude 在开始任务前读取记忆，在工作时创建和更新文件，并在未来对话中引用它们。

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { ToolLoopAgent } from 'ai';

const memory = anthropic.tools.memory_20250818({
  execute: async action => {
    // `action` contains `command`, `path`, and other fields
    // depending on the command (view, create, str_replace,
    // insert, delete, rename).
    // Implement your storage backend here.
    // Return the result as a string.
  },
});

const agent = new ToolLoopAgent({
  model: 'anthropic/claude-haiku-4.5',
  tools: { memory },
});

const result = await agent.generate({
  prompt: 'Remember that my favorite editor is Neovim',
});
```

该工具接收结构化命令（`view`、`create`、`str_replace`、`insert`、`delete`、`rename`），每个命令的 `path` 限定在 `/memories` 范围内。你的 `execute` 函数将这些映射到你的存储后端（文件系统、数据库或任何其他持久层）。

**何时使用**：你希望以最少实现工作量获得 memory，并且已在使用 Anthropic 模型。权衡是提供商锁定，因为此工具仅适用于 Claude。

## 记忆提供商

另一种方式是使用内置 memory 的提供商。这些提供商包装外部记忆服务，并通过 AI SDK 的标准接口暴露。记忆存储、检索和注入透明进行，你无需自己定义任何工具。

### Letta

[Letta](https://letta.com) 为 Agent 提供持久长期记忆。你在 Letta 平台（云或自托管）上创建 Agent，在那里配置其 memory，并使用 AI SDK 提供商与之交互。Letta 的 Agent 运行时处理 memory 管理（核心记忆、归档记忆、回忆）。

```bash
pnpm add @letta-ai/vercel-ai-sdk-provider
```

```ts
import { lettaCloud } from '@letta-ai/vercel-ai-sdk-provider';
import { ToolLoopAgent } from 'ai';

const agent = new ToolLoopAgent({
  model: lettaCloud(),
  providerOptions: {
    letta: {
      agent: { id: 'your-agent-id' },
    },
  },
});

const result = await agent.generate({
  prompt: 'Remember that my favorite editor is Neovim',
});
```

你也可以将 Letta 的内置 memory 工具与自定义工具一起使用：

```ts
import { lettaCloud } from '@letta-ai/vercel-ai-sdk-provider';
import { ToolLoopAgent } from 'ai';

const agent = new ToolLoopAgent({
  model: lettaCloud(),
  tools: {
    core_memory_append: lettaCloud.tool('core_memory_append'),
    memory_insert: lettaCloud.tool('memory_insert'),
    memory_replace: lettaCloud.tool('memory_replace'),
  },
  providerOptions: {
    letta: {
      agent: { id: 'your-agent-id' },
    },
  },
});

const stream = agent.stream({
  prompt: 'What do you remember about me?',
});
```

完整设置与配置请参阅 [Letta 提供商文档](/providers/community-providers/letta)。

### Mem0

[Mem0](https://mem0.ai) 在任何支持的 LLM 提供商之上添加记忆层。它自动从对话中提取记忆、存储它们，并为未来提示检索相关记忆。

```bash
pnpm add @mem0/vercel-ai-provider
```

```ts
import { createMem0 } from '@mem0/vercel-ai-provider';
import { ToolLoopAgent } from 'ai';

const mem0 = createMem0({
  provider: 'openai',
  mem0ApiKey: process.env.MEM0_API_KEY,
  apiKey: process.env.OPENAI_API_KEY,
});

const agent = new ToolLoopAgent({
  model: mem0('gpt-4.1', { user_id: 'user-123' }),
});

const { text } = await agent.generate({
  prompt: 'Remember that my favorite editor is Neovim',
});
```

Mem0 支持多个 LLM 提供商（OpenAI、Anthropic、Google、Groq、Cohere）。你也可以显式管理记忆：

```ts
import { addMemories, retrieveMemories } from '@mem0/vercel-ai-provider';

await addMemories(messages, { user_id: 'user-123' });
const context = await retrieveMemories(prompt, { user_id: 'user-123' });
```

完整设置与配置请参阅 [Mem0 提供商文档](/providers/community-providers/mem0)。

### Supermemory

[Supermemory](https://supermemory.ai) 是一个长期记忆平台，为你的 AI 应用添加持久、自增长的 memory。它提供通过语义搜索自动处理记忆保存和检索的工具。

```bash
pnpm add @supermemory/tools
```

```ts
__PROVIDER_IMPORT__;
import { supermemoryTools } from '@supermemory/tools/ai-sdk';
import { ToolLoopAgent } from 'ai';

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: supermemoryTools(process.env.SUPERMEMORY_API_KEY!),
});

const result = await agent.generate({
  prompt: 'Remember that my favorite editor is Neovim',
});
```

Supermemory 可与任何 AI SDK 提供商配合使用。这些工具为模型提供 `addMemory` 和 `searchMemories` 操作以处理存储和检索。

完整设置与配置请参阅 [Supermemory 提供商文档](/providers/community-providers/supermemory)。

### Hindsight

[Hindsight](/providers/community-providers/hindsight) 通过五个工具为 Agent 提供持久记忆：`retain`、`recall`、`reflect`、`getMentalModel` 和 `getDocument`。可使用 Docker 自托管或作为云服务使用。

```bash
pnpm add @vectorize-io/hindsight-ai-sdk @vectorize-io/hindsight-client
```

```ts
__PROVIDER_IMPORT__;
import { HindsightClient } from '@vectorize-io/hindsight-client';
import { createHindsightTools } from '@vectorize-io/hindsight-ai-sdk';
import { ToolLoopAgent } from 'ai';
import { openai } from '@ai-sdk/openai';

const client = new HindsightClient({ baseUrl: process.env.HINDSIGHT_API_URL });

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: createHindsightTools({ client, bankId: 'user-123' }),
  instructions: 'You are a helpful assistant with long-term memory.',
});

const result = await agent.generate({
  prompt: 'Remember that my favorite editor is Neovim',
});
```

`bankId` 标识记忆存储，通常为用户 ID。在多用户应用中，在请求处理程序内调用 `createHindsightTools`，以便每个请求获得正确的 bank。Hindsight 可与任何 AI SDK 提供商配合使用。

完整设置与配置请参阅 [Hindsight 提供商文档](/providers/community-providers/hindsight)。

**何时使用记忆提供商**：当你希望获得 memory 而无需构建任何存储基础设施时，这些提供商很合适。权衡是提供商控制 memory 行为，因此你对存储内容和检索方式的可见性较低。你还会依赖外部服务。

## 自定义工具

从头构建自己的 memory 工具是最灵活的方式。你控制存储格式、接口和检索逻辑。这需要最多的前期工作，但让你完全掌控 memory 的工作方式，无提供商锁定且无外部依赖。

有两种常见模式：

- **结构化操作**：你定义显式操作（`view`、`create`、`update`、`search`）并自行处理结构化输入。由于你控制每个操作，设计上更安全。
- **Bash 支持**：你为模型提供沙箱化的 bash 环境，通过组合 shell 命令（`cat`、`grep`、`sed`、`echo`）灵活访问 memory。更强大，但需要命令验证以确保安全。

关于实现带有 bash 支持接口、基于 AST 的命令验证和文件系统持久化的自定义 memory 工具的完整演练，请参阅 **[构建自定义 Memory 工具](/cookbook/guides/custom-memory-tool)** 食谱。
