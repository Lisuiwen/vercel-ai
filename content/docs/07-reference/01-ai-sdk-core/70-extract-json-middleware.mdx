# `extractJsonMiddleware()`

`extractJsonMiddleware` 通过去除 Markdown 代码围栏等格式，从文本内容中提取 JSON。在配合 `Output.object()` 使用、且模型将 JSON 包在 Markdown 代码块（例如 ` ```json ... ``` `）中时很有用。

```ts
import { extractJsonMiddleware } from 'ai';

const middleware = extractJsonMiddleware();
```

### 返回值

返回中间件对象，其行为如下：

- 同时处理流式与非流式响应
- 从文本内容中去除 Markdown 代码围栏（` ```json ` 与 ` ``` `）
- 提供 `transform` 时应用自定义转换
- 通过高效缓冲保持正确的流式行为

## 使用示例

### 基本用法

在使用结构化输出时去除模型响应中的 Markdown 代码围栏：

```ts
import {
  generateText,
  wrapLanguageModel,
  extractJsonMiddleware,
  Output,
} from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: wrapLanguageModel({
    model: yourModel,
    middleware: extractJsonMiddleware(),
  }),
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a lasagna recipe.',
});

console.log(result.output);
```

### 与流式配合

该中间件同样适用于流式响应：

```ts
import {
  streamText,
  wrapLanguageModel,
  extractJsonMiddleware,
  Output,
} from 'ai';
import { z } from 'zod';

const { partialOutputStream } = streamText({
  model: wrapLanguageModel({
    model: yourModel,
    middleware: extractJsonMiddleware(),
  }),
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        ingredients: z.array(z.string()),
        steps: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a detailed recipe.',
});

for await (const partialObject of partialOutputStream) {
  console.log(partialObject);
}
```

### 自定义转换函数

针对使用不同格式的模型，可提供自定义转换：

```ts
import { extractJsonMiddleware } from 'ai';

const middleware = extractJsonMiddleware({
  transform: text =>
    text
      .replace(/^PREFIX/, '')
      .replace(/SUFFIX$/, '')
      .trim(),
});
```

## 工作原理

该中间件以两种方式处理文本内容：

### 非流式（generateText）

1. 接收模型的完整响应
2. 应用转换函数去除 Markdown 围栏（或自定义格式）
3. 返回清理后的文本内容

### 流式（streamText）

1. 缓冲初始内容以检测 Markdown 围栏前缀（` ```json\n `）
2. 若检测到围栏，去除前缀并切换到流式模式
3. 维护较小后缀缓冲区以处理结束围栏（` \n``` `）
4. 流结束时从缓冲区去除尾部围栏
5. 自定义转换时缓冲全部内容并在结束时应用转换

这样可在分块可能跨片段拆分代码围栏的情况下，仍保持高效流式处理。
