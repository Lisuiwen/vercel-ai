# `simulateStreamingMiddleware()`

`simulateStreamingMiddleware` 为非流式语言模型的完整响应模拟流式行为。在仅提供完整响应的模型上仍需统一流式接口时很有用。

```ts
import { simulateStreamingMiddleware } from 'ai';

const middleware = simulateStreamingMiddleware();
```

## 导入

<Snippet
  text={`import { simulateStreamingMiddleware } from "ai"`}
  prompt={false}
/>

## API 签名

### 参数

该中间件不接受任何参数。

### 返回值

返回中间件对象，其行为如下：

- 接收语言模型的完整响应
- 将其转换为模拟的分块流
- 正确处理多种响应组成部分，包括：
  - 文本内容
  - 推理（字符串或对象数组）
  - 工具调用
  - 元数据与用量信息
  - 警告

### 使用示例

```ts
import { streamText } from 'ai';
import { wrapLanguageModel } from 'ai';
import { simulateStreamingMiddleware } from 'ai';

// 与非流式模型一起使用的示例
const result = streamText({
  model: wrapLanguageModel({
    model: nonStreamingModel,
    middleware: simulateStreamingMiddleware(),
  }),
  prompt: 'Your prompt here',
});

// 现在可使用流式接口
for await (const chunk of result.fullStream) {
  // 处理流式分块
}
```

## 工作原理

该中间件会：

1. 等待语言模型的完整响应
2. 创建按正确顺序发出分块的 `ReadableStream`
3. 将响应拆分为适当的分块类型以模拟流式输出
4. 保留所有元数据、推理、工具调用及其他响应属性
