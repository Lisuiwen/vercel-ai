# `tool()`

Tool 是一个辅助函数，用于为其 `execute` 方法和生命周期回调推断工具输入与上下文。

它没有任何运行时行为，但可以帮助 TypeScript 推断工具回调的类型。

如果没有这个辅助函数，TypeScript 就无法将 `inputSchema` 和 `contextSchema` 属性关联到工具回调，
也无法推断回调参数类型。

有关 `runtimeContext`、`toolsContext`、工具 `context` 以及敏感上下文过滤的完整模型，请参阅[运行时和工具上下文](/docs/ai-sdk-core/runtime-and-tool-context)。

`Tool` 类型是四种工具类型的联合类型：

- `FunctionTool`：用户定义的函数式工具，具有已知的输入和输出类型。
- `DynamicTool`：运行时定义的函数式工具，其输入和输出类型为 `unknown`。
- `ProviderDefinedTool`：由 provider 定义、由你的代码执行的工具。
- `ProviderExecutedTool`：由 provider 定义、由 provider 执行的工具。

```ts highlight={"1,4,9,10"}
import { tool } from 'ai';
import { z } from 'zod';

export const weatherTool = tool({
  description: 'Get the weather in a location',
  inputSchema: z.object({
    location: z.string().describe('The location to get the weather for'),
  }),
  // location below is inferred to be a string:
  execute: async ({ location }) => ({
    location,
    temperature: 72 + Math.floor(Math.random() * 21) - 10,
  }),
});
```
