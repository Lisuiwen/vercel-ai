# `zodSchema()`

`zodSchema` 将 Zod schema 转换为与 AI SDK 兼容的 JSON schema 对象。
接受 Zod schema 与可选配置，返回带类型的 schema。

可用于[生成结构化数据](/docs/ai-sdk-core/generating-structured-data)与[工具](/docs/ai-sdk-core/tools-and-tool-calling)。

<Note>
  也可直接向 AI SDK 函数传入 Zod 对象。内部会通过 `zodSchema()` 将 Zod schema 转为 JSON schema。
  若需指定 `useReferences` 等选项，可改用 `zodSchema()` 辅助函数。
</Note>

<Note type="warning">
  使用 `.meta()` 或 `.describe()` 为 Zod schema 添加元数据时，请将这些方法放在**链式调用的末尾**。
  
  元数据附着在**特定 schema 实例**上，多数 schema 方法（`.min()`、`.optional()`、`.extend()` 等）
  会返回不继承前序元数据的新实例。
  由于 Zod 的不可变性，仅当 `.meta()` 或 `.describe()` 为链中最后一个方法时，元数据才会出现在 JSON schema 输出中。

```ts
// ❌ 元数据会丢失 — .min() 返回不带元数据的新实例
z.string().meta({ describe: 'first name' }).min(1);

// ✅ 元数据保留 — .meta() 为最后一步
z.string().min(1).meta({ describe: 'first name' });
```

</Note>

## 递归 schema 示例

```ts
import { zodSchema } from 'ai';
import { z } from 'zod';

// 基础分类 schema
const baseCategorySchema = z.object({
  name: z.string(),
});

// 递归 Category 类型
type Category = z.infer<typeof baseCategorySchema> & {
  subcategories: Category[];
};

// 使用 z.lazy 创建递归 schema
const categorySchema: z.ZodType<Category> = baseCategorySchema.extend({
  subcategories: z.lazy(() => categorySchema.array()),
});

// 启用 useReferences 以支持递归
const mySchema = zodSchema(
  z.object({
    category: categorySchema,
  }),
  { useReferences: true },
);
```
