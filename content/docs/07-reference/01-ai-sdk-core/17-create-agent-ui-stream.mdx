# `createAgentUIStream`

`createAgentUIStream` 函数执行一个 [Agent](/docs/reference/ai-sdk-core/agent)，消费 UI message 数组，并通过异步 iterable 将 agent 的输出作为 UI message chunk 流式传输。这使你能在自己的运行时中实时、增量渲染 AI assistant 输出，并完整访问工具使用、中间推理和交互式 UI 功能，非常适合构建由 agent 驱动的聊天 API、仪表盘或机器人。

## 导入

<Snippet text={`import { createAgentUIStream } from "ai"`} prompt={false} />

## 用法

```ts
import { ToolLoopAgent, createAgentUIStream } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
  tools: { weather: weatherTool, calculator: calculatorTool },
});

export async function* streamAgent(
  uiMessages: unknown[],
  abortSignal?: AbortSignal,
) {
  const stream = await createAgentUIStream({
    agent,
    uiMessages,
    abortSignal,
    // experimental_sandbox, // optional: pass an experimental sandbox through to tool execution
    // ...other options (see below)
  });

  for await (const chunk of stream) {
    yield chunk; // Each chunk is a UI message output from the agent.
  }
}
```

## 返回值

一个 `Promise<AsyncIterableStream<UIMessageChunk>>`，其中每个 yield 的 chunk 都是来自 agent 的 UI message 输出（参见 [`UIMessage`](/docs/reference/ai-sdk-core/ui-message)）。它可由任意异步迭代器循环消费，也可传输到流式 HTTP 响应、socket 或任何其他 sink。

## 示例

```ts
import { createAgentUIStream } from 'ai';

const controller = new AbortController();

const stream = await createAgentUIStream({
  agent,
  uiMessages: [{ role: 'user', content: 'What is the weather in SF today?' }],
  abortSignal: controller.signal,
  // experimental_sandbox, // optional
  sendStart: true,
  // ...other UIMessageStreamOptions
});

for await (const chunk of stream) {
  // Each chunk is a UI message update — stream it to your client, dashboard, logs, etc.
  console.log(chunk);
}

// Call controller.abort() to cancel the agent operation early.
```

## 工作原理

1. **UI Message 验证：** 使用 agent 的 `tools` 定义验证并规范化输入 `uiMessages` 数组。任何无效消息都会导致错误。
2. **转换为模型消息：** 根据 agent 要求，将已验证的 UI message 转换为模型专属消息格式。
3. **Agent 流式传输：** 使用转换后的模型消息、可选调用选项、中止信号、experimental_sandbox 和任何实验性转换调用 agent 的 `.stream({ prompt, ... })` 方法。
4. **UI Message Stream 构建：** 结果流会被转换并暴露为 UI message chunk 的流式异步 iterable，供你消费。

## 备注

- agent **必须**实现 `.stream({ prompt, ... })` 方法，并定义其支持的 `tools` 属性。
- 该工具返回异步 iterable，以提供最大的流式传输灵活性。对于 HTTP 响应，请参阅 [`createAgentUIStreamResponse`](/docs/reference/ai-sdk-core/create-agent-ui-stream-response)（Web）或 [`pipeAgentUIStreamToResponse`](/docs/reference/ai-sdk-core/pipe-agent-ui-stream-to-response)（Node.js）。
- `uiMessages` 参数名是 `uiMessages`，**不是**简单的 `messages`。
- 你可以通过 `UIMessageStreamOptions` 提供高级选项（例如包含来源或用量）。
- 若要取消流，请通过 `abortSignal` 参数传入 [`AbortSignal`](https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal)。
- 当你的 agent 工具在执行期间需要实验性沙箱环境时，请传入 `experimental_sandbox`。
