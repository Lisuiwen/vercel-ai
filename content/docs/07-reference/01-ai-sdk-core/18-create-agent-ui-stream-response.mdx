# `createAgentUIStreamResponse`

`createAgentUIStreamResponse` 函数执行一个 [Agent](/docs/reference/ai-sdk-core/agent)，将其流式输出作为 UI message stream 运行，并返回一个 HTTP [Response](https://developer.mozilla.org/en-US/docs/Web/API/Response) 对象，其 body 是实时流式 UI message 输出。它专为交付实时 agent 结果的 API 路由设计，例如聊天端点或流式工具使用操作。

## 导入

<Snippet
  text={`import { createAgentUIStreamResponse } from "ai"`}
  prompt={false}
/>

## 用法

```ts
import { ToolLoopAgent, createAgentUIStreamResponse } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  instructions: 'You are a helpful assistant.',
  tools: { weather: weatherTool, calculator: calculatorTool },
});

export async function POST(request: Request) {
  const { messages } = await request.json();

  // Optional: support cancellation (aborts on disconnect, etc.)
  const abortController = new AbortController();

  return createAgentUIStreamResponse({
    agent,
    uiMessages: messages,
    abortSignal: abortController.signal, // optional
    // experimental_sandbox, // optional: passed through to tool execution
    // ...other UIMessageStreamOptions like sendSources, experimental_transform, etc.
  });
}
```

## 返回值

一个 `Promise<Response>`，其 `body` 是来自 agent 的流式 UI message 输出。在 serverless、Next.js、Express、Hono 或 edge runtime 上下文中，可将它作为 API/server handler 的返回值。

## 示例: Next.js API Route Handler

```ts
import { createAgentUIStreamResponse } from 'ai';
import { MyCustomAgent } from '@/agent/my-custom-agent';

export async function POST(request: Request) {
  const { messages } = await request.json();

  return createAgentUIStreamResponse({
    agent: MyCustomAgent,
    uiMessages: messages,
    // experimental_sandbox, // optional
    sendSources: true, // (optional)
    // headers, status, abortSignal, and other UIMessageStreamOptions also supported
  });
}
```

## 工作原理

- 1. **UI Message 验证：** 根据 agent 指定的工具和要求验证传入的 `uiMessages` 数组。
- 2. **模型消息转换：** 将验证后的 UI message 转换为 agent 使用的内部模型消息格式。
- 3. **流式传输 Agent 输出：** 调用 agent 的 `.stream({ prompt, ... })` 获取 chunk 流（step/UI message），并透传 `experimental_sandbox` 等选项。
- 4. **创建 HTTP Response：** 将输出流包装为可读 HTTP `Response` 对象，用于向客户端流式传输 UI message chunk。

## 备注

- 你的 agent **必须**实现 `.stream({ prompt, ... })` 并定义 `tools` 属性（即使只是 `{}`），才能与此函数配合使用。
- **仅服务端：** 此 API 只能在后端/服务端上下文中调用（API 路由、edge/serverless/server route handler 等）。不适用于浏览器。
- 当你的 agent 工具在执行期间需要实验性沙箱环境时，请传入 `experimental_sandbox`。
- 高级场景可使用额外选项（`headers`、`status`、UI stream 选项、转换等）。
- 它使用 [ReadableStream](https://developer.mozilla.org/en-US/docs/Web/API/ReadableStream)，因此你的平台/客户端必须支持 HTTP 流式消费。
