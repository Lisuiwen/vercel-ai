# `dynamicTool()`

`dynamicTool` 函数用于创建输入和输出类型在编译时未知的工具。这适用于以下场景：

- 没有 schema 的 MCP（Model Context Protocol）工具
- 运行时加载的用户定义函数
- 从外部来源或数据库加载的工具
- 基于用户输入动态生成工具

与常规 `tool` 函数不同，`dynamicTool` 接受并返回 `unknown` 类型，使你可以处理 schema 在运行时确定的工具。

```ts highlight={"1,4,9,10,11"}
import { dynamicTool } from 'ai';
import { z } from 'zod';

export const customTool = dynamicTool({
  description: 'Execute a custom user-defined function',
  inputSchema: z.object({}),
  // input is typed as 'unknown'
  execute: async input => {
    const { action, parameters } = input as any;

    // Execute your dynamic logic
    return {
      result: `Executed ${action} with ${JSON.stringify(parameters)}`,
    };
  },
});
```

### 返回值

一个带有 `type: 'dynamic'` 的 `Tool<unknown, unknown>`，可与 `generateText`、`streamText` 和其他 AI SDK 函数一起使用。

## 类型安全用法

将动态工具与静态工具一起使用时，需要检查 `dynamic` 标志以正确缩小类型：

```ts
const result = await generateText({
  model: __MODEL__,
  tools: {
    // Static tool with known types
    weather: weatherTool,
    // Dynamic tool with unknown types
    custom: dynamicTool({
      /* ... */
    }),
  },
  onStepFinish: ({ toolCalls, toolResults }) => {
    for (const toolCall of toolCalls) {
      if (toolCall.dynamic) {
        // Dynamic tool: input/output are 'unknown'
        console.log('Dynamic tool:', toolCall.toolName);
        console.log('Input:', toolCall.input);
        continue;
      }

      // Static tools have full type inference
      switch (toolCall.toolName) {
        case 'weather':
          // TypeScript knows the exact types
          console.log(toolCall.input.location); // string
          break;
      }
    }
  },
});
```

## 与 `useChat` 一起使用

与 useChat（`UIMessage` 格式）一起使用时，动态工具会显示为 `dynamic-tool` part：

```tsx
{
  message.parts.map(part => {
    switch (part.type) {
      case 'dynamic-tool':
        return (
          <div>
            <h4>Tool: {part.toolName}</h4>
            <pre>{JSON.stringify(part.input, null, 2)}</pre>
          </div>
        );
      // ... handle other part types
    }
  });
}
```
