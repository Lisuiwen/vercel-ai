# `pipeAgentUIStreamToResponse`

`pipeAgentUIStreamToResponse` 函数运行一个 [Agent](/docs/reference/ai-sdk-core/agent)，并将生成的 UI message 输出直接流式传输到 Node.js [`ServerResponse`](https://nodejs.org/api/http.html#class-httpserverresponse) 对象。它非常适合在 Express、Hono 或自定义 Node 服务器等基于 Node.js 的框架中构建实时流式 API 端点（用于聊天、工具使用等）。

## 导入

<Snippet
  text={`import { pipeAgentUIStreamToResponse } from "ai"`}
  prompt={false}
/>

## 用法

```ts
import { pipeAgentUIStreamToResponse } from 'ai';
import { MyAgent } from './agent';

export async function handler(req, res) {
  const { messages } = JSON.parse(req.body);

  await pipeAgentUIStreamToResponse({
    response: res, // Node.js ServerResponse
    agent: MyAgent,
    uiMessages: messages, // Required: array of input UI messages
    // abortSignal: optional AbortSignal for cancellation
    // experimental_sandbox: optional experimental sandbox passed through to tool execution
    // status: 200,
    // headers: { ... },
    // ...other optional UI message stream options
  });
}
```

## 返回值

一个 `Promise<void>`。当 UI message stream 已完整发送到提供的 ServerResponse 时，该函数完成。

## 示例: Express Route Handler

```ts
import { pipeAgentUIStreamToResponse } from 'ai';
import { openaiWebSearchAgent } from './openai-web-search-agent';

app.post('/chat', async (req, res) => {
  // Use req.body.messages as input UI messages
  await pipeAgentUIStreamToResponse({
    response: res,
    agent: openaiWebSearchAgent,
    uiMessages: req.body.messages,
    // experimental_sandbox, // optional
    // abortSignal: yourController.signal
    // status: 200,
    // headers: { ... },
    // ...more options
  });
});
```

## 工作原理

1. **运行 Agent：** 使用提供的 UI message 和选项调用 agent 的 `.stream` 方法，按需将它们转换为模型消息，并透传 `experimental_sandbox` 等选项。
2. **流式传输 UI Message 输出：** 将 agent 输出作为 UI message stream 传输到 `ServerResponse`，通过流式 HTTP 响应发送数据（包括适当标头）。
3. **中止信号处理：** 如果提供了 `abortSignal`，信号触发时（例如客户端断开连接）会立即取消流式传输。
4. **不返回 Response：** 与返回 `Response` 的 Edge/serverless API 不同，此函数会直接向 ServerResponse 写入字节，不返回响应对象。

## 备注

- **中止处理：** 为获得最佳稳健性，请使用 `AbortSignal`（例如连接到 Express/Hono 客户端断开事件），以确保快速取消 agent 计算和流式传输。
- **仅 Node.js：** 仅适用于 Node.js [ServerResponse](https://nodejs.org/api/http.html#class-httpserverresponse) 对象（例如 Express、Hono 的 node adapter 等），不适用于 Edge/serverless/web Response API。
- **流式支持：** 确保你的客户端（以及任何代理）正确支持流式 HTTP 响应，以获得完整效果。
- **参数名称：** 输入消息属性是 `uiMessages`（不是 `messages`），以保持与 SDK agent 工具一致。
- 当你的 agent 工具在执行期间需要实验性沙箱环境时，请传入 `experimental_sandbox`。
