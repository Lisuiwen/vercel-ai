# `ModelMessage`

`ModelMessage` 表示与 AI SDK Core 函数配合使用的基础消息结构。
它包含可用于任何 AI SDK Core 函数的 `messages` 字段的各种消息类型。

你可以通过 `modelMessageSchema` 导出访问 `ModelMessage` 的 Zod 模式。

## `ModelMessage` 类型

### `SystemModelMessage`

可包含系统信息的系统消息。

```typescript
type SystemModelMessage = {
  role: 'system';
  content: string;
};
```

你可以通过 `systemModelMessageSchema` 导出访问 `SystemModelMessage` 的 Zod 模式。

<Note>
  对于系统指令，请使用顶层 `instructions` 属性，而不是系统消息。AI SDK 函数默认会拒绝
  `prompt` 或 `messages` 中的系统消息，除非将 `allowSystemInMessages` 设置为
  `true`。如果用户可以注入系统消息，选择启用此行为可能会带来提示注入风险。
</Note>

### `UserModelMessage`

可包含文本，或文本、图像和文件组合的用户消息。

```typescript
type UserModelMessage = {
  role: 'user';
  content: UserContent;
};

type UserContent = string | Array<TextPart | ImagePart | FilePart>;
```

你可以通过 `userModelMessageSchema` 导出访问 `UserModelMessage` 的 Zod 模式。

### `AssistantModelMessage`

可包含文本、工具调用或两者组合的助手消息。

```typescript
type AssistantModelMessage = {
  role: 'assistant';
  content: AssistantContent;
};

type AssistantContent = string | Array<TextPart | CustomPart | ToolCallPart>;
```

你可以通过 `assistantModelMessageSchema` 导出访问 `AssistantModelMessage` 的 Zod 模式。

### `ToolModelMessage`

包含一个或多个工具调用结果的工具消息。

```typescript
type ToolModelMessage = {
  role: 'tool';
  content: ToolContent;
};

type ToolContent = Array<ToolResultPart>;
```

你可以通过 `toolModelMessageSchema` 导出访问 `ToolModelMessage` 的 Zod 模式。

## `ModelMessage` 部分

### `TextPart`

表示提示中的文本内容部分。它包含一个文本字符串。

```typescript
export interface TextPart {
  type: 'text';
  /**
   * The text content.
   */
  text: string;
}
```

### `ImagePart` <Note type="warning">已弃用</Note>

<Note type="warning">
  `ImagePart` 已弃用。请改用 [`FilePart`](#filepart)，并将 `mediaType: 'image'`
  （或更具体的 `image/*` 子类型）与其配合使用。
</Note>

表示用户消息中的图像部分。

```typescript
/**
 * @deprecated Use `FilePart` with `mediaType: 'image'` instead.
 */
export interface ImagePart {
  type: 'image';

  /**
   * Image data. Can either be:
   * - data: a base64-encoded string, a Uint8Array, an ArrayBuffer, or a Buffer
   * - URL: a URL that points to the image
   * - ProviderReference: a provider reference from `uploadFile`
   */
  image: DataContent | URL | ProviderReference;

  /**
   * Optional IANA media type of the image.
   * We recommend leaving this out as it will be detected automatically.
   */
  mediaType?: string;
}
```

### `FilePart`

表示用户消息中的文件部分。

```typescript
export interface FilePart {
  type: 'file';

  /**
   * File data. Can either be:
   * - data: a base64-encoded string, a Uint8Array, an ArrayBuffer, or a Buffer
   * - URL: a URL that points to the file
   * - ProviderReference: a provider reference from `uploadFile`
   */
  data: DataContent | URL | ProviderReference;

  /**
   * Optional filename of the file.
   */
  filename?: string;

  /**
   * IANA media type of the file.
   */
  mediaType: string;
}
```

### `CustomPart`

表示特定于 provider 的自定义内容部分。`kind` 字段以 `{provider}.{provider-type}` 格式标识内容类型。

```typescript
export interface CustomPart {
  type: 'custom';

  /**
   * The kind of custom content, in the format `{provider}.{provider-type}`.
   */
  kind: `${string}.${string}`;

  /**
   * Additional provider-specific metadata.
   */
  providerOptions?: ProviderOptions;
}
```

### `ToolCallPart`

表示提示中的工具调用内容部分，通常由 AI 模型生成。

```typescript
export interface ToolCallPart {
  type: 'tool-call';

  /**
   * ID of the tool call. This ID is used to match the tool call with the tool result.
   */
  toolCallId: string;

  /**
   * Name of the tool that is being called.
   */
  toolName: string;

  /**
   * Arguments of the tool call. This is a JSON-serializable object that matches the tool's input schema.
   */
  args: unknown;
}
```

### `ToolResultPart`

表示工具消息中的工具调用结果。

```typescript
export interface ToolResultPart {
  type: 'tool-result';

  /**
   * ID of the tool call that this result is associated with.
   */
  toolCallId: string;

  /**
   * Name of the tool that generated this result.
   */
  toolName: string;

  /**
   * Result of the tool call. This is a JSON-serializable object.
   */
  output: LanguageModelV4ToolResultOutput;

  /**
  Additional provider-specific metadata. They are passed through
  to the provider from the AI SDK and enable provider-specific
  functionality that can be fully encapsulated in the provider.
  */
  providerOptions?: ProviderOptions;
}
```
