# `isLoopFinished()`

创建永不触发的停止条件，让智能体循环自然结束（即模型不再发起工具调用）。

默认情况下，`ToolLoopAgent` 使用 `isStepCount(20)` 作为安全措施，防止失控循环导致过多 API 调用与费用。若你确信智能体会自然结束或对费用不敏感，`isLoopFinished()` 可移除该上限，让智能体在模型真正结束之前持续运行。

```ts
import { ToolLoopAgent, isLoopFinished } from 'ai';
__PROVIDER_IMPORT__;

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    // your tools
  },
  stopWhen: isLoopFinished(),
});

const result = await agent.generate({
  prompt: 'Analyze this dataset and create a summary report',
});
```

## API 签名

### 参数

该函数不接受参数。

### 返回值

`StopCondition` 函数，**始终**返回 `false`，即永不触发停止。循环仅通过自然终止条件结束：

- 模型不再发起工具调用，或
- 调用了没有 `execute` 函数的工具，或
- 工具调用需要审批

## 示例

### 基本用法

让智能体运行直至自然结束：

```ts
import { ToolLoopAgent, isLoopFinished } from 'ai';

const agent = new ToolLoopAgent({
  model: yourModel,
  tools: yourTools,
  stopWhen: isLoopFinished(),
});
```

### 与其他条件组合

可将 `isLoopFinished()` 与其他条件组合。由于 `isLoopFinished()` 永不触发，其他条件仍会生效：

```ts
import { ToolLoopAgent, isLoopFinished, hasToolCall } from 'ai';

const agent = new ToolLoopAgent({
  model: yourModel,
  tools: yourTools,
  stopWhen: [isLoopFinished(), hasToolCall('finalAnswer')],
});
```

实际上在此场景下意义不大，因为可以直接省略 `isLoopFinished()`。
