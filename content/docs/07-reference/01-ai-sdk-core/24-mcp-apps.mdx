# MCP Apps

`@ai-sdk/mcp` 中的 MCP Apps 辅助函数帮助 MCP 宿主声明 UI 支持、区分模型可见与应用可见的工具，并读取 `ui://` HTML 资源以供渲染。

## 导入

<Snippet
  text={`import {
  MCP_APP_MIME_TYPE,
  mcpAppClientCapabilities,
  readMCPAppResource,
  splitMCPAppTools,
} from "@ai-sdk/mcp"`}
  prompt={false}
/>

## `MCP_APP_MIME_TYPE`

应作为 MCP App 渲染的 HTML 资源的 MIME 类型。

```ts
const MCP_APP_MIME_TYPE = 'text/html;profile=mcp-app';
```

## `mcpAppClientCapabilities`

当宿主支持 MCP Apps 时，传给 [`createMCPClient`](/docs/reference/ai-sdk-core/create-mcp-client) 的客户端能力。

```ts
import { createMCPClient, mcpAppClientCapabilities } from '@ai-sdk/mcp';

const client = await createMCPClient({
  transport: {
    type: 'http',
    url: 'https://example.com/mcp',
  },
  capabilities: mcpAppClientCapabilities,
});
```

宣告的能力为：

```json
{
  "extensions": {
    "io.modelcontextprotocol/ui": {
      "mimeTypes": ["text/html;profile=mcp-app"]
    }
  }
}
```

## `splitMCPAppTools()`

将 MCP 工具定义拆分为模型可见与应用可见两部分。

无 MCP Apps 可见性元数据的工具仍对模型可见。`_meta.ui.visibility` 包含 `"app"` 的工具归入 `appVisible`。

```ts
const definitions = await client.listTools();
const { modelVisible, appVisible } = splitMCPAppTools(definitions);

const tools = client.toolsFromDefinitions(modelVisible);
```

## `readMCPAppResource()`

从 MCP 服务器读取 `ui://` 资源并规范化为 HTML 与渲染元数据。

```ts
const resource = await readMCPAppResource({
  client,
  uri: 'ui://example/dashboard',
});
```

该辅助函数会校验 URI 以 `ui://` 开头，要求 `text/html;profile=mcp-app` MIME 类型，并支持以文本或 base64 blob 返回的资源内容。
