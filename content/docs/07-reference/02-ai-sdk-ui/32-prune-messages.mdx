# `pruneMessages()`

`pruneMessages` 函数用于裁剪或过滤 `ModelMessage` 对象数组。这对于在发送给 LLM 之前减少消息上下文（以节省 token）、移除中间 reasoning，或裁剪 tool calls 和空消息很有用。

```ts filename="app/api/chat/route.ts"
import { pruneMessages, streamText } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages } = await req.json();

  const prunedMessages = pruneMessages({
    messages,
    reasoning: 'before-last-message',
    toolCalls: 'before-last-2-messages',
    emptyMessages: 'remove',
  });

  const result = streamText({
    model: __MODEL__,
    messages: prunedMessages,
  });

  return result.toUIMessageStreamResponse();
}
```

## 示例用法

```ts
import { pruneMessages } from 'ai';

const pruned = pruneMessages({
  messages,
  reasoning: 'all', // Remove all reasoning parts
  toolCalls: 'before-last-message', // Remove tool calls except those in the last message
});
```

## 裁剪选项

- **reasoning:** 从 assistant messages 中移除 reasoning parts。使用 `'all'` 移除全部，使用 `'before-last-message'` 保留最后一条消息中的 reasoning，或使用 `'none'` 保留全部 reasoning。
- **toolCalls:** 从 assistant/tool messages 中裁剪 tool-call、tool-result 和 tool-approval chunks。默认值为空数组（不裁剪）。选项包括：
  - `'all'`: 裁剪所有此类内容。
  - `'before-last-message'`: 裁剪除最后一条消息以外的内容。
  - `'before-last-N-messages'`: 裁剪除最后 N 条消息以外的内容。
  - `'none'`: 不裁剪。
  - 或提供一个数组，对每个 tool 进行精细控制，例如 `[{ type: 'before-last-message', tools: ['search', 'calculator'] }]`，以便只裁剪特定 tools。
- **emptyMessages:** 设置为 `'remove'`（默认）以排除裁剪后没有内容的消息。

> **提示**：`pruneMessages` 通常用于在向 LLM 发送上下文窗口之前减少消息/token 数量，尤其是在一系列 tool-calls 和 approvals 之后。

有关高级用法和所有可能 message parts 的完整列表，请参阅 [`ModelMessage`](/docs/reference/ai-sdk-core/model-message) 和 [`pruneMessages` 实现](https://github.com/vercel/ai/blob/main/packages/ai/src/generate-text/prune-messages.ts)。
