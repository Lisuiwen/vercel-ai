# `convertToModelMessages()`

`convertToModelMessages` 函数用于将来自 `useChat` hook 的 UI message 数组转换为 `ModelMessage` 对象数组。这些 `ModelMessage` 对象可与 `streamText` 等 AI core 函数兼容。

```ts filename="app/api/chat/route.ts"
import { convertToModelMessages, streamText } from 'ai';
__PROVIDER_IMPORT__;

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

## 工具批准状态

将 UI message 转换回 `ModelMessage` 以用于后续 `generateText` 或 `streamText` 调用时，`convertToModelMessages` 会保留其中的工具批准状态。

- 处于 `approval-requested` 状态的工具部分会变为 `tool-approval-request` 内容部分。
- 处于 `approval-responded` 状态的工具部分会变为 `tool-approval-response` 内容部分，并在存在时包含 `reason`。
- 自动批准元数据会通过将 `approval.isAutomatic` 转发到 `tool-approval-request` 部分来保留。
- 被拒绝的工具批准还会生成一个合成的 `tool-result`，其中包含 `output: { type: 'execution-denied', reason?: string }`，因此模型会收到完整的工具生命周期，并能在下一步响应拒绝。

## 多模态工具响应

`convertToModelMessages` 函数支持可返回多模态内容的工具。当工具需要返回图像等非文本内容时，这很有用。

```ts
import { tool } from 'ai';
__PROVIDER_IMPORT__;
import { z } from 'zod';

const screenshotTool = tool({
  inputSchema: z.object({}),
  execute: async () => 'imgbase64',
  toModelOutput: ({ output }) => [
    { type: 'file-data', data: output, mediaType: 'image/png' },
  ],
});

const result = streamText({
  model: __MODEL__,
  messages: convertToModelMessages(messages, {
    tools: {
      screenshot: screenshotTool,
    },
  }),
});
```

工具可以实现可选的 `toModelOutput` 方法，将其结果转换为多模态内容。内容是 content part 数组，其中每个 part 都有一个 `type`（例如 'text'、'image'）和相应数据。

## 自定义 Data Part 转换

`convertToModelMessages` 函数支持转换附加到用户消息的自定义 data part。当用户需要在消息中包含额外上下文（URL、代码文件、JSON 配置）时，这很有用。

### 基本用法

默认情况下，转换期间会过滤掉用户消息中的 data part。若要包含它们，请提供 `convertDataPart` 回调，将 data part 转换为模型可以理解的文本或文件 part：

```ts filename="app/api/chat/route.ts"
import { convertToModelMessages, streamText } from 'ai';

type CustomUIMessage = UIMessage<
  never,
  {
    url: { url: string; title: string; content: string };
    'code-file': { filename: string; code: string; language: string };
  }
>;

export async function POST(req: Request) {
  const { messages } = await req.json();

  const result = streamText({
    model: __MODEL__,
    messages: convertToModelMessages<CustomUIMessage>(messages, {
      convertDataPart: part => {
        // Convert URL attachments to text
        if (part.type === 'data-url') {
          return {
            type: 'text',
            text: `[Reference: ${part.data.title}](${part.data.url})\n\n${part.data.content}`,
          };
        }

        // Convert code file attachments
        if (part.type === 'data-code-file') {
          return {
            type: 'text',
            text: `\`\`\`${part.data.language}\n// ${part.data.filename}\n${part.data.code}\n\`\`\``,
          };
        }

        // Other data parts are ignored
      },
    }),
  });

  return result.toUIMessageStreamResponse();
}
```

### 使用场景

**附加 URL 内容**
允许用户将 URL 附加到消息中，并为模型获取和格式化这些内容：

```ts
// Client side
sendMessage({
  parts: [
    { type: 'text', text: 'Analyze this article' },
    {
      type: 'data-url',
      data: {
        url: 'https://example.com/article',
        title: 'Important Article',
        content: '...',
      },
    },
  ],
});
```

**将代码文件作为上下文包含**
允许用户在对话中引用代码文件：

```ts
convertDataPart: part => {
  if (part.type === 'data-code-file') {
    return {
      type: 'text',
      text: `\`\`\`${part.data.language}\n${part.data.code}\n\`\`\``,
    };
  }
};
```

**选择性包含**
只有返回文本或文件模型消息 part 的 data part 会被包含，
所有其他 data part 都会被忽略。

```ts
const result = convertToModelMessages<
  UIMessage<
    unknown,
    {
      url: { url: string; title: string };
      code: { code: string; language: string };
      note: { text: string };
    }
  >
>(messages, {
  convertDataPart: part => {
    if (part.type === 'data-url') {
      return {
        type: 'text',
        text: `[${part.data.title}](${part.data.url})`,
      };
    }

    // data-code and data-node are ignored
  },
});
```

### 类型安全

泛型参数可确保自定义 data part 的完整类型安全：

```ts
type MyUIMessage = UIMessage<
  unknown,
  {
    url: { url: string; content: string };
    config: { key: string; value: string };
  }
>;

// TypeScript knows the exact shape of part.data
convertToModelMessages<MyUIMessage>(messages, {
  convertDataPart: part => {
    if (part.type === 'data-url') {
      // part.data is typed as { url: string; content: string }
      return { type: 'text', text: part.data.url };
    }
    // Return undefined to skip this part
  },
});
```
