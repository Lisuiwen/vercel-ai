# AI_UIMessageStreamError

此错误在 UI 消息流包含无效或顺序错误的分片时发生。

常见原因：

- 在未收到前置 `text-start` 分片的情况下收到 `text-delta` 分片
- 在未收到前置 `text-start` 分片的情况下收到 `text-end` 分片
- 在未收到前置 `reasoning-start` 分片的情况下收到 `reasoning-delta` 分片
- 在未收到前置 `reasoning-start` 分片的情况下收到 `reasoning-end` 分片
- 在未收到前置 `tool-input-start` 分片的情况下收到 `tool-input-delta` 分片
- 尝试访问不存在的工具调用

此错误经常在**尚未流式输出任何 token**之前上游请求失败，且自定义传输层尝试在未发送正确起始分片的情况下向 UI 流写入内联错误消息时出现。

## 属性

- `chunkType`：导致错误的分片类型（例如 `text-delta`、`reasoning-end`、`tool-input-delta`）
- `chunkId`：与失败分片关联的 ID（片段 ID 或 toolCallId）
- `message`：包含问题详情的错误消息

## 如何检查此错误

你可以通过以下方式检查错误是否为 `AI_UIMessageStreamError` 的实例：

```typescript
import { UIMessageStreamError } from 'ai';

if (UIMessageStreamError.isInstance(error)) {
  console.log('Chunk type:', error.chunkType);
  console.log('Chunk ID:', error.chunkId);
  // Handle the error
}
```

## 常见解决方案

1. **确保分片顺序正确**：对同一 ID，务必在任意 `*-delta` 或 `*-end` 分片之前发送 `*-start` 分片：

   ```typescript
   // Correct order
   writer.write({ type: 'text-start', id: 'my-text' });
   writer.write({ type: 'text-delta', id: 'my-text', delta: 'Hello' });
   writer.write({ type: 'text-end', id: 'my-text' });
   ```

2. **核对 ID 一致**：确保 `*-delta` 与 `*-end` 分片中使用的 `id` 与对应 `*-start` 分片中的 `id` 一致。

3. **正确处理错误路径**：在自定义传输层中写入错误消息时，确保发出完整的 start/delta/end 序列：

   ```typescript
   // When handling errors in custom transports
   writer.write({ type: 'text-start', id: errorId });
   writer.write({
     type: 'text-delta',
     id: errorId,
     delta: 'Request failed...',
   });
   writer.write({ type: 'text-end', id: errorId });
   ```

4. **检查流式生产者逻辑**：审阅你的流式实现，确保分片按正确顺序发送，尤其在并发操作或合并流的情况下。
