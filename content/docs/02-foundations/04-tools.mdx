# 工具

虽然[大语言模型（LLM）](/docs/foundations/overview#large-language-models)具备强大的生成能力，
它们在离散任务（如数学运算）以及与外部世界交互（如获取天气）方面仍显吃力。

工具是 LLM 可以调用的操作。
这些操作的结果可以回传给 LLM，供其在后续响应中参考。

例如，当你向 LLM 询问「伦敦的天气」，且存在天气工具时，它可以用伦敦作为参数调用该工具。工具会获取天气数据并返回给 LLM，LLM 即可在回答中使用这些信息。

## 什么是工具？

工具是模型可调用的对象，用于执行特定任务。
你可以通过向 `tools` 参数传入一个或多个工具，在 [`generateText`](/docs/reference/ai-sdk-core/generate-text)
与 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 中使用工具。

工具包含三个属性：

- **`description`**：工具的可选描述，可影响模型何时选择该工具。函数工具与动态工具可使用字符串，或使用根据工具上下文与 experimental sandbox 推导描述的函数。
- **`inputSchema`**：定义工具运行所需输入的 [Zod schema](/docs/reference/ai-sdk-core/zod-schema) 或 [JSON schema](/docs/reference/ai-sdk-core/json-schema)。该 schema 供 LLM 消费，也用于校验 LLM 的工具调用。
- **`execute`**：可选的异步函数，使用工具调用中的参数执行。

<Note>
  `streamUI` 使用带有 `generate` 函数的 UI 生成器工具，可返回
  React 组件。
</Note>

若 LLM 决定使用工具，将生成工具调用。
带有 `execute` 函数的工具会在生成这些调用时自动运行。
工具调用的输出通过 tool result 对象返回。

你可以使用 `streamText` 与 `generateText` 的[多步调用](/docs/ai-sdk-core/tools-and-tool-calling#multi-step-calls)，自动将工具结果传回 LLM。

## 工具类型

AI SDK 支持四种工具，各有不同权衡：

### 函数工具

函数工具由你完全自行定义，包括描述、输入 schema 与可选的 execute 函数。它们与提供商无关，可完全掌控。

```ts
import { tool } from 'ai';
import { z } from 'zod';

const weatherTool = tool({
  description: 'Get the weather in a location',
  inputSchema: z.object({
    location: z.string().describe('The location to get the weather for'),
  }),
  execute: async ({ location }) => {
    // Your implementation
    return { temperature: 72, conditions: 'sunny' };
  },
});
```

**适用场景**：需要完全控制、希望提供商可移植，或实现应用特定功能时。

### 动态工具

动态工具是函数式工具，其输入与输出类型在开发时未知。适用于从外部源加载的工具，例如 MCP 服务器、用户定义函数或数据库。

```ts
import { dynamicTool } from 'ai';
import { z } from 'zod';

const runtimeTool = dynamicTool({
  description: 'Execute a tool loaded at runtime',
  inputSchema: z.object({}),
  execute: async input => {
    // input is typed as unknown
    return runRuntimeTool(input);
  },
});
```

**适用场景**：工具在运行时被发现或生成，且编写代码时无法确定其确切的 TypeScript 输入/输出类型。

### 提供商定义工具

提供商定义工具由提供商指定工具的 `inputSchema` 与 `description`，由你提供 `execute` 函数。有时称为「客户端工具」，因为执行发生在你这一侧。

例如 Anthropic 的 `bash` 与 `text_editor` 工具。模型经过专门训练以有效使用这些工具，在支持的任务上可能获得更好表现。

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: anthropic('claude-opus-4-5'),
  tools: {
    bash: anthropic.tools.bash_20250124({
      execute: async ({ command }) => {
        // Your implementation to run the command
        return runCommand(command);
      },
    }),
  },
  prompt: 'List files in the current directory',
});
```

**适用场景**：提供商提供模型已针对其优化的工具，且你希望在该特定任务上获得更好表现。

### 提供商执行工具

提供商执行工具完全在提供商服务器上运行。你进行配置，由提供商负责执行。有时称为「服务端工具」。

例如 OpenAI 的网页搜索与 Anthropic 的代码执行。无需自行搭建基础设施即可获得开箱即用的能力。

```ts
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    web_search: openai.tools.webSearch(),
  },
  prompt: 'What happened in the news today?',
});
```

**适用场景**：希望获得强大能力（如网页搜索或沙箱代码执行）而不自行管理基础设施。

### 对比

| 方面             | 函数工具           | 动态工具                 | 提供商定义工具 | 提供商执行工具 |
| ------------------ | ------------------------ | ----------------------------- | ---------------------- | ----------------------- |
| **执行**      | 你的代码                | 你的代码                     | 你的代码              | 提供商服务器      |
| **Schema**         | 你定义               | 运行时定义               | 提供商定义       | 提供商定义        |
| **可移植性**    | 适用于任意提供商  | 适用于任意提供商       | 提供商特定      | 提供商特定       |
| **模型训练** | 通用工具使用         | 通用工具使用              | 针对该工具优化 | 针对该工具优化  |
| **搭建**          | 你实现一切 | 你加载或生成工具 | 你实现 execute  | 仅配置      |

<Note>
  提供商定义与提供商执行工具在各提供商页面有文档。示例见 [Anthropic Provider](/providers/ai-sdk-providers/anthropic) 与
  [OpenAI Provider](/providers/ai-sdk-providers/openai)。
</Note>

## Schema

Schema 用于定义并校验[工具输入](/docs/ai-sdk-core/tools-and-tool-calling)、工具输出以及结构化输出生成。

AI SDK 支持以下 schema：

- [Zod](https://zod.dev/) v3 与 v4，直接或通过 [`zodSchema()`](/docs/reference/ai-sdk-core/zod-schema)
- [Valibot](https://valibot.dev/)，通过 `@ai-sdk/valibot` 的 [`valibotSchema()`](/docs/reference/ai-sdk-core/valibot-schema)
- 兼容 [Standard JSON Schema](https://standardschema.dev/json-schema) 的 schema
- 通过 [`jsonSchema()`](/docs/reference/ai-sdk-core/json-schema) 的原始 JSON schema

<Note>
  你也可以在 [`generateText`](/docs/reference/ai-sdk-core/generate-text) 与
  [`streamText`](/docs/reference/ai-sdk-core/stream-text) 中使用 `output`
  设置，将 schema 用于结构化输出生成。
</Note>

## 精炼工具输入

不同 LLM 提供商可能对同一工具输入类型生成略有不同的工具输入。例如，某提供商可能对可选字段生成 `null`，而另一提供商生成空字符串。

当你不拥有该工具（例如来自第三方包的工具）时，可能无法修改工具的 `inputSchema` 以同时接受两种形态。此时可使用实验性 `experimental_refineToolInput` 选项，在执行前以及出现在输出、生命周期回调与遥测之前，规范化已解析的工具输入。

```ts
import { generateText, tool } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: 'openai/gpt-5-mini',
  tools: {
    search: tool({
      inputSchema: z.object({
        query: z.string(),
        category: z.string().nullable(),
      }),
      execute: async ({ query, category }) => {
        return search({ query, category });
      },
    }),
  },
  experimental_refineToolInput: {
    search: input => ({
      ...input,
      category: input.category === '' ? null : input.category,
    }),
  },
  prompt: 'Search for ski jackets with no category.',
});
```

精炼函数按工具分别类型化。每个函数接收其工具的已类型化输入，并必须返回相同类型形态的输入。

## 工具包

工具是 JavaScript 对象，可像其他库一样打包并通过 npm 分发，便于在项目与社区间共享可复用工具。

### 使用现成工具包

安装工具包并导入所需工具：

```bash
pnpm add some-tool-package
```

然后直接传入 `generateText`、`streamText` 或 Agent 定义：

```ts highlight="2, 8"
import { generateText, isStepCount } from 'ai';
import { searchTool } from 'some-tool-package';

const { text } = await generateText({
  model: 'anthropic/claude-haiku-4.5',
  prompt: 'When was Vercel Ship AI?',
  tools: {
    webSearch: searchTool,
  },
  stopWhen: isStepCount(10),
});
```

### 发布自己的工具

你可以将工具包发布到 npm 供他人使用。只需从包中导出工具对象：

```ts filename="my-tools/index.ts"
import { tool } from 'ai';
import { z } from 'zod';

export const myTool = tool({
  description: 'A helpful tool',
  inputSchema: z.object({
    query: z.string(),
  }),
  execute: async ({ query }) => {
    // your tool logic
    return result;
  },
});
```

他人可通过安装并导入来使用你的工具。

入门可使用 [AI SDK Tool Package Template](https://github.com/vercel-labs/ai-sdk-tool-as-package-template)，作为发布自有工具的起点。
