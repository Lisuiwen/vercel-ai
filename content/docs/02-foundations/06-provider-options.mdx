# 提供商选项

提供商选项让你传递超出所有提供商共享的[标准设置](/docs/ai-sdk-core/settings)的提供商特定配置。它们通过 `generateText`、`streamText` 等函数上的 `providerOptions` 属性设置。

```ts
const result = await generateText({
  model: openai('gpt-5.2'),
  prompt: 'Explain quantum entanglement.',
  providerOptions: {
    openai: {
      reasoningEffort: 'low',
    },
  },
});
```

提供商选项按提供商名称（如 `openai`、`anthropic`）命名空间化，因此你甚至可在同一次调用中包含多个提供商的选项——仅使用与当前活动提供商匹配的选项。有关在消息与消息部分级别应用选项，请参阅[提示词：提供商选项](/docs/foundations/prompts#provider-options)。

<Note>
  若要控制推理力度，建议使用顶层 [`reasoning`
  参数](/docs/ai-sdk-core/reasoning)，而非提供商特定选项。
  它提供适用于所有支持推理的提供商的可移植设置。
  仅在需要精确 token 预算等功能时使用提供商特定选项。
</Note>

## OpenAI

### 推理力度

对于推理模型（如 `o3`、`o4-mini`、`gpt-5.2`），`reasoningEffort` 控制模型在响应前进行多少内部推理。较低值更快、更便宜；较高值产生更详尽答案。

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5.2'),
  prompt: 'Invent a new holiday and describe its traditions.',
  providerOptions: {
    openai: {
      reasoningEffort: 'low', // 'none' | 'minimal' | 'low' | 'medium' | 'high' | 'xhigh'
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

console.log('Text:', result.text);
console.log('Usage:', result.usage);
console.log(
  'Reasoning tokens:',
  result.finalStep.providerMetadata?.openai?.reasoningTokens,
);
```

| 值       | 行为                                   |
| ----------- | ------------------------------------------ |
| `'none'`    | 无推理（仅 GPT-5.1 模型）         |
| `'minimal'` | 最少推理                     |
| `'low'`     | 快速、简洁推理                    |
| `'medium'`  | 平衡（默认）                         |
| `'high'`    | 深入推理                         |
| `'xhigh'`   | 最大推理（仅 GPT-5.1-Codex-Max） |

<Note>
  `'none'` 与 `'xhigh'` 仅部分模型支持。在不支持的模型上使用将导致错误。
</Note>

### 推理摘要

使用推理模型时，你可能希望了解模型**如何**得出答案。`reasoningSummary` 选项可展示模型的思考过程。

#### 流式

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { streamText } from 'ai';

const result = streamText({
  model: openai('gpt-5.2'),
  prompt: 'Tell me about the Mission burrito debate in San Francisco.',
  providerOptions: {
    openai: {
      reasoningSummary: 'detailed', // 'auto' | 'detailed'
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

for await (const part of result.fullStream) {
  if (part.type === 'reasoning') {
    console.log(`Reasoning: ${part.textDelta}`);
  } else if (part.type === 'text-delta') {
    process.stdout.write(part.textDelta);
  }
}
```

#### 非流式

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5.2'),
  prompt: 'Tell me about the Mission burrito debate in San Francisco.',
  providerOptions: {
    openai: {
      reasoningSummary: 'auto',
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

console.log('Reasoning:', result.finalStep.reasoning);
```

| 值        | 行为                       |
| ------------ | ------------------------------ |
| `'auto'`     | 推理的精简摘要 |
| `'detailed'` | 完整推理输出 |

### 文本详细程度

独立于推理控制模型文本响应的长度与细节：

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Write a poem about a boy and his first pet dog.',
  providerOptions: {
    openai: {
      textVerbosity: 'low', // 'low' | 'medium' | 'high'
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

| 值      | 行为                         |
| ---------- | -------------------------------- |
| `'low'`    | 简短、极简响应         |
| `'medium'` | 平衡细节（默认）        |
| `'high'`   | 详尽、全面响应 |

## Anthropic

### Thinking（扩展推理）

Anthropic 的 thinking 功能为 Claude 模型提供专用的「思考」阶段后再响应。通过提供带 token 预算的 `thinking` 对象启用。

```ts
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text, reasoning, reasoningText } = await generateText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    } satisfies AnthropicLanguageModelOptions,
  },
});

console.log('Reasoning:', reasoningText);
console.log('Answer:', text);
```

`budgetTokens` 设置模型可用于内部推理的 token 上限。更高预算允许更深推理，但会增加延迟与成本。

<Note>
  thinking 支持 `claude-opus-4-20250514`、`claude-sonnet-4-20250514`
  与 `claude-sonnet-4-5-20250929` 模型。
</Note>

### Effort

`effort` 选项提供更简单的方式控制推理深度，无需指定 token 预算。它影响 thinking、文本响应与函数调用。

```ts
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text, usage } = await generateText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      effort: 'low', // 'low' | 'medium' | 'high'
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

| 值      | 行为                             |
| ---------- | ------------------------------------ |
| `'low'`    | 最少推理，最快响应 |
| `'medium'` | 平衡推理                   |
| `'high'`   | 深入推理（默认）         |

### 快速模式

对于 `claude-opus-4-6`，`speed` 选项可将输出 token 速度提升约 2.5 倍：

```ts
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: anthropic('claude-opus-4-6'),
  prompt: 'Write a short poem about the sea.',
  providerOptions: {
    anthropic: {
      speed: 'fast', // 'fast' | 'standard'
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

## 组合选项

你可在单次调用中组合多个提供商选项。例如，在 OpenAI 上同时使用推理力度与推理摘要：

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5.2'),
  prompt: 'What are the implications of quantum computing for cryptography?',
  providerOptions: {
    openai: {
      reasoningEffort: 'high',
      reasoningSummary: 'detailed',
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

或为 Anthropic 启用 thinking 并设置较低 effort：

```ts
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'Explain the Riemann hypothesis in simple terms.',
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 8000 },
      effort: 'medium',
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

## 在 AI Gateway 中使用提供商选项

使用 [Vercel AI Gateway](/providers/ai-sdk-providers/ai-gateway) 时，提供商选项用法相同。使用底层提供商名称（如 `openai`、`anthropic`）作为键——而非 `gateway`。AI Gateway 会自动将这些选项转发给目标提供商。

```ts
import type { OpenAILanguageModelResponsesOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: 'openai/gpt-5.2', // AI Gateway model string
  prompt: 'What are the implications of quantum computing for cryptography?',
  providerOptions: {
    openai: {
      reasoningEffort: 'high',
      reasoningSummary: 'detailed',
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

你也可在同一次调用中组合网关特定选项（如路由与回退）与提供商特定选项：

```ts
import type { AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const result = await generateText({
  model: 'anthropic/claude-sonnet-4',
  prompt: 'Explain quantum computing',
  providerOptions: {
    // Gateway-specific: control routing
    gateway: {
      order: ['vertex', 'anthropic'],
    } satisfies GatewayProviderOptions,
    // Provider-specific: enable reasoning
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

有关网关路由、回退及其他网关特定选项，请参阅 [AI Gateway 提供商文档](/providers/ai-sdk-providers/ai-gateway#provider-options)。

## 类型安全

各提供商导出其选项类型，可与 `satisfies` 配合使用以获得自动补全并在构建时发现拼写错误：

```ts
import { type OpenAILanguageModelResponsesOptions } from '@ai-sdk/openai';
import { type AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
```

完整可用选项列表请参阅提供商文档：

- [OpenAI Provider](/providers/ai-sdk-providers/openai)
- [Anthropic Provider](/providers/ai-sdk-providers/anthropic)
