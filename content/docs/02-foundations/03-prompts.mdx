# 提示词

提示词是你给[大语言模型（LLM）](/docs/foundations/overview#large-language-models)的指令，用来告诉它该做什么。
这就像你向某人问路：问题越清楚，得到的指引就越好。

许多 LLM 提供商提供复杂的接口来指定提示词，涉及不同角色和消息类型。
虽然这些接口功能强大，但可能难以使用和理解。

为简化提示词用法，AI SDK 支持文本提示词、消息提示词和系统提示词。

## 文本提示词

文本提示词是字符串。
它们适合简单的生成场景，
例如对同一提示词文本的变体反复生成内容。

你可以通过 AI SDK 函数（如 [`streamText`](/docs/reference/ai-sdk-core/stream-text) 或 [`generateText`](/docs/reference/ai-sdk-core/generate-text)）提供的 `prompt` 属性设置文本提示词。
你可以按任意方式组织文本并注入变量，例如使用模板字面量。

```ts highlight="3"
const result = await generateText({
  model: __MODEL__,
  prompt: 'Invent a new holiday and describe its traditions.',
});
```

你也可以使用模板字面量为提示词提供动态数据。

```ts highlight="3-5"
const result = await generateText({
  model: __MODEL__,
  prompt:
    `I am planning a trip to ${destination} for ${lengthOfStay} days. ` +
    `Please suggest the best tourist activities for me to do.`,
});
```

## 系统提示词

系统提示词是提供给模型的初始指令集，用于引导和约束模型的行为与回复。
你可以通过 `instructions` 属性设置系统提示词。
系统提示词可与 `prompt` 和 `messages` 属性配合使用。
`prompt` 或 `messages` 中的 system 消息默认会被拒绝；请使用 `instructions` 属性设置系统指令，或在需要发送包含 system 消息的既有消息历史时设置 `allowSystemInMessages: true`。

<Note type="warning">
  使用 `allowSystemInMessages` 可能带来提示词注入风险：
  用户可通过注入 system 消息覆盖或设置系统提示词。
  在大多数情况下，只有受信任的服务端代码应通过 `instructions` 属性设置系统指令。
</Note>

```ts highlight="3-6"
const result = await generateText({
  model: __MODEL__,
  instructions:
    `You help planning travel itineraries. ` +
    `Respond to the users' request with a list ` +
    `of the best stops to make in their destination.`,
  prompt:
    `I am planning a trip to ${destination} for ${lengthOfStay} days. ` +
    `Please suggest the best tourist activities for me to do.`,
});
```

## 消息提示词

消息提示词是由 user、assistant 和 tool 消息组成的数组。
它们适合聊天界面以及更复杂的多模态提示词。
你可以使用 `messages` 属性设置消息提示词。

每条消息都有 `role` 和 `content` 属性。`content` 可以是文本（用于 user 和 assistant 消息），也可以是与该消息类型相关的部分（数据）组成的数组。

```ts highlight="3-7"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'user', content: 'Hi!' },
    { role: 'assistant', content: 'Hello, how can I help?' },
    { role: 'user', content: 'Where can I buy the best Currywurst in Berlin?' },
  ],
});
```

除了在 `content` 属性中发送文本，你还可以发送由文本与其他内容部分混合组成的数组。

<Note type="warning">
  并非所有语言模型都支持所有消息与内容类型。例如，
  部分模型可能无法处理多模态输入或 tool
  消息。[了解部分所选
  模型的能力](./providers-and-models#model-capabilities)。
</Note>

### 提供商选项

你可以传入额外的提供商特定元数据，在 3 个层级启用提供商特定功能。

#### 函数调用层级

像 [`streamText`](/docs/reference/ai-sdk-core/stream-text#provider-options) 或 [`generateText`](/docs/reference/ai-sdk-core/generate-text#provider-options) 这样的函数接受 `providerOptions` 属性。

在函数调用层级添加提供商选项适用于你不需要精细控制提供商选项应用位置的场景。

```ts
const { text } = await generateText({
  model: azure('your-deployment-name'),
  providerOptions: {
    openai: {
      reasoningEffort: 'low',
    },
  },
});
```

#### 消息层级

若要在消息层级精细控制提供商选项的应用，可以将 `providerOptions` 传给消息对象：

```ts
const result = await generateText({
  model: __MODEL__,
  instructions: {
    role: 'system',
    content: 'Cached system message',
    providerOptions: {
      // Sets a cache control breakpoint on the system message
      anthropic: { cacheControl: { type: 'ephemeral' } },
    },
  },
  prompt: 'Invent a new holiday and describe its traditions.',
});
```

#### 消息部分层级

某些提供商特定选项需要在消息部分层级配置：

```ts
import { ModelMessage } from 'ai';

const messages: ModelMessage[] = [
  {
    role: 'user',
    content: [
      {
        type: 'text',
        text: 'Describe the image in detail.',
        providerOptions: {
          openai: { imageDetail: 'low' },
        },
      },
      {
        type: 'file',
        mediaType: 'image',
        data: 'https://github.com/vercel/ai/blob/main/examples/ai-functions/data/comic-cat.png?raw=true',
        // Sets image detail configuration for image part:
        providerOptions: {
          openai: { imageDetail: 'low' },
        },
      },
    ],
  },
];
```

<Note type="warning">
  像 [`useChat`](/docs/reference/ai-sdk-ui/use-chat) 这样的 AI SDK UI hook 会返回
  `UIMessage` 对象数组，这些对象不支持 provider options。我们建议在使用
  [`convertToModelMessages`](/docs/reference/ai-sdk-ui/convert-to-model-messages)
  将 `UIMessage` 对象转换为
  [`ModelMessage`](/docs/reference/ai-sdk-core/model-message) 对象之后，
  再为消息或消息部分添加或附加带有 `providerOptions` 的内容。
</Note>

### User 消息

#### 文本部分

文本内容是最常见的内容类型，是作为字符串传给模型的内容。

如果一条消息只需发送文本内容，`content` 属性可以是字符串，
但你也可以用它发送多个内容部分。

```ts highlight="7-10"
const result = await generateText({
  model: __MODEL__,
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'Where can I buy the best Currywurst in Berlin?',
        },
      ],
    },
  ],
});
```

#### 图像部分

User 消息可以包含图像部分。图像可以是以下形式之一：

- base64 编码的图像：
  - 带有 base-64 编码内容的 `string`
  - data URL `string`，例如 `data:image/png;base64,...`
- 二进制图像：
  - `ArrayBuffer`
  - `Uint8Array`
  - `Buffer`
- URL：
  - http(s) URL `string`，例如 `https://example.com/image.png`
  - `URL` 对象，例如 `new URL('https://example.com/image.png')`

##### 示例：二进制图像（Buffer）

```ts highlight="8-11"
const result = await generateText({
  model,
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe the image in detail.' },
        {
          type: 'file',
          mediaType: 'image',
          data: fs.readFileSync('./data/comic-cat.png'),
        },
      ],
    },
  ],
});
```

##### 示例：Base-64 编码图像（string）

```ts highlight="8-11"
const result = await generateText({
  model: __MODEL__,
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe the image in detail.' },
        {
          type: 'file',
          mediaType: 'image',
          data: fs.readFileSync('./data/comic-cat.png').toString('base64'),
        },
      ],
    },
  ],
});
```

##### 示例：图像 URL（string）

```ts highlight="8-12"
const result = await generateText({
  model: __MODEL__,
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe the image in detail.' },
        {
          type: 'file',
          mediaType: 'image',
          data: 'https://github.com/vercel/ai/blob/main/examples/ai-functions/data/comic-cat.png?raw=true',
        },
      ],
    },
  ],
});
```

#### 文件部分

<Note type="warning">
  目前只有少数提供商和模型支持文件部分：[Google
  Generative AI](/providers/ai-sdk-providers/google)、[Google Vertex
  AI](/providers/ai-sdk-providers/google-vertex)、
  [OpenAI](/providers/ai-sdk-providers/openai)（`gpt-4o-audio-preview` 支持 `wav` 和 `mp3` 音频）、[Anthropic](/providers/ai-sdk-providers/anthropic)、
  [OpenAI](/providers/ai-sdk-providers/openai)（支持 `pdf`）。
</Note>

User 消息可以包含文件部分。文件可以是以下形式之一：

- base64 编码的文件：
  - 带有 base-64 编码内容的 `string`
  - data URL `string`，例如 `data:image/png;base64,...`
- 二进制数据：
  - `ArrayBuffer`
  - `Uint8Array`
  - `Buffer`
- URL：
  - http(s) URL `string`，例如 `https://example.com/some.pdf`
  - `URL` 对象，例如 `new URL('https://example.com/some.pdf')`

你需要指定所发送文件的 MIME 类型。

##### 示例：来自 Buffer 的 PDF 文件

```ts highlight="12-15"
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google('gemini-2.5-flash'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'What is the file about?' },
        {
          type: 'file',
          mediaType: 'application/pdf',
          data: fs.readFileSync('./data/example.pdf'),
          filename: 'example.pdf', // optional, not used by all providers
        },
      ],
    },
  ],
});
```

##### 示例：来自 Buffer 的 mp3 音频文件

```ts highlight="12-14"
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-4o-audio-preview'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'What is the audio saying?' },
        {
          type: 'file',
          mediaType: 'audio/mpeg',
          data: fs.readFileSync('./data/galileo.mp3'),
        },
      ],
    },
  ],
});
```

#### 自定义下载函数（实验性）

你可以使用自定义下载函数实现限流、重试、认证、缓存等。

当模型不支持文件时，默认下载实现会自动并行下载文件。

自定义下载函数可通过 `experimental_download` 属性传入：

```ts
const result = await generateText({
  model: __MODEL__,
  experimental_download: async (
    requestedDownloads: Array<{
      url: URL;
      isUrlSupportedByModel: boolean;
    }>,
  ): PromiseLike<
    Array<{
      data: Uint8Array;
      mediaType: string | undefined;
    } | null>
  > => {
    // ... download the files and return an array with similar order
  },
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'file',
          data: new URL('https://api.company.com/private/document.pdf'),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
});
```

<Note>
  `experimental_download` 选项为实验性功能，未来版本可能变更。
</Note>

### Assistant 消息

Assistant 消息是 `role` 为 `assistant` 的消息。
它们通常是助手先前的回复，
可包含文本、推理和 tool call 部分。

#### 示例：带文本内容的 Assistant 消息

```ts highlight="5"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'user', content: 'Hi!' },
    { role: 'assistant', content: 'Hello, how can I help?' },
  ],
});
```

#### 示例：数组形式文本内容的 Assistant 消息

```ts highlight="7"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'user', content: 'Hi!' },
    {
      role: 'assistant',
      content: [{ type: 'text', text: 'Hello, how can I help?' }],
    },
  ],
});
```

#### 示例：带 tool call 内容的 Assistant 消息

```ts highlight="7-14"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'user', content: 'How many calories are in this block of cheese?' },
    {
      role: 'assistant',
      content: [
        {
          type: 'tool-call',
          toolCallId: '12345',
          toolName: 'get-nutrition-data',
          input: { cheese: 'Roquefort' },
        },
      ],
    },
  ],
});
```

#### 示例：带文件内容的 Assistant 消息

<Note>
  该内容部分用于模型生成的文件。只有少数模型支持
  此功能，且仅支持它们能生成的文件类型。
</Note>

```ts highlight="9-11"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'user', content: 'Generate an image of a roquefort cheese!' },
    {
      role: 'assistant',
      content: [
        {
          type: 'file',
          mediaType: 'image/png',
          data: fs.readFileSync('./data/roquefort.jpg'),
        },
      ],
    },
  ],
});
```

### Tool 消息

<Note>
  [工具](/docs/foundations/tools)（也称为 function calling）是你可以提供给 LLM 以扩展其内置能力的程序。
  这可以是从调用外部 API 到调用 UI 内函数的任何操作。
  在[下一节](/docs/foundations/tools)了解更多关于工具的内容。
</Note>

对于支持 [tool](/docs/foundations/tools) 调用的模型，assistant 消息可包含 tool call 部分，tool 消息可包含 tool output 部分。
一条 assistant 消息可以调用多个工具，一条 tool 消息可以包含多个 tool 结果。

```ts highlight="14-42"
const result = await generateText({
  model: __MODEL__,
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'How many calories are in this block of cheese?',
        },
        {
          type: 'file',
          mediaType: 'image',
          data: fs.readFileSync('./data/roquefort.jpg'),
        },
      ],
    },
    {
      role: 'assistant',
      content: [
        {
          type: 'tool-call',
          toolCallId: '12345',
          toolName: 'get-nutrition-data',
          input: { cheese: 'Roquefort' },
        },
        // there could be more tool calls here (parallel calling)
      ],
    },
    {
      role: 'tool',
      content: [
        {
          type: 'tool-result',
          toolCallId: '12345', // needs to match the tool call id
          toolName: 'get-nutrition-data',
          output: {
            type: 'json',
            value: {
              name: 'Cheese, roquefort',
              calories: 369,
              fat: 31,
              protein: 22,
            },
          },
        },
        // there could be more tool results here (parallel calling)
      ],
    },
  ],
});
```

#### 多模态 Tool 结果

Tool 结果可以是多部分、多模态的，例如文本与图像。
你可以使用 `output: { type: 'content', value: [...] }` 指定多部分 tool 结果。

```ts highlight="14-27"
const result = await generateText({
  model: __MODEL__,
  messages: [
    // ...
    {
      role: 'tool',
      content: [
        {
          type: 'tool-result',
          toolCallId: '12345', // needs to match the tool call id
          toolName: 'get-nutrition-data',
          // for models that do not support multi-part tool results,
          // you can include a regular output part:
          output: {
            type: 'json',
            value: {
              name: 'Cheese, roquefort',
              calories: 369,
              fat: 31,
              protein: 22,
            },
          },
        },
        {
          type: 'tool-result',
          toolCallId: '12345', // needs to match the tool call id
          toolName: 'get-nutrition-data',
          // for models that support multi-part tool results,
          // you can include a multi-part content part:
          output: {
            type: 'content',
            value: [
              {
                type: 'text',
                text: 'Here is the nutrition data for the cheese:',
              },
              {
                type: 'file-data',
                data: fs
                  .readFileSync('./data/roquefort-nutrition-data.png')
                  .toString('base64'),
                mediaType: 'image/png',
              },
            ],
          },
        },
      ],
    },
  ],
});
```

### System 消息

System 消息是在 user 消息之前发送给模型、用于引导助手行为的消息。
你也可以改用 `instructions` 属性。

```ts highlight="4"
const result = await generateText({
  model: __MODEL__,
  messages: [
    { role: 'system', content: 'You help planning travel itineraries.' },
    {
      role: 'user',
      content:
        'I am planning a trip to Berlin for 3 days. Please suggest the best tourist activities for me to do.',
    },
  ],
});
```
