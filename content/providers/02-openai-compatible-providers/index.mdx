# OpenAI 兼容 Provider

你可以使用 [OpenAI Compatible Provider](https://www.npmjs.com/package/@ai-sdk/openai-compatible) 包来使用实现了 OpenAI API 的语言模型 provider。

下文主要介绍通用安装与 provider 实例的创建。你也可以[基于 OpenAI Compatible 包编写自定义 provider 包](/providers/openai-compatible-providers/custom-providers)。

我们为以下 OpenAI 兼容 provider 提供详细文档：

- [LM Studio](/providers/openai-compatible-providers/lmstudio)
- [NIM](/providers/openai-compatible-providers/nim)
- [Heroku](/providers/openai-compatible-providers/heroku)
- [Clarifai](/providers/openai-compatible-providers/clarifai)

这些 provider 的通用安装与实例创建方式相同。

## Provider 实例

要使用 OpenAI 兼容 provider，可使用 `@ai-sdk/openai-compatible` 中的 `createOpenAICompatible` 创建自定义 provider 实例：

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
  includeUsage: true, // Include usage information in streaming responses
});
```

可使用以下可选设置自定义 provider 实例：

- **baseURL** _string_

  设置 API 调用的 URL 前缀。

- **apiKey** _string_

  用于鉴权的 API 密钥。若指定，会在请求头中加入 `Authorization`，值为 `Bearer <apiKey>`。该头会加在 `headers` 选项可能指定的任何头之前。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的可选自定义头。这些头会加在使用 `apiKey` 选项可能添加的头之后。

- **queryParams** _Record&lt;string,string&gt;_

  请求 URL 中要包含的可选自定义查询参数。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。
  可用作拦截请求的中间件，
  或为测试等场景提供自定义 fetch 实现。

- **includeUsage** _boolean_

  在流式响应中包含用量信息。启用后，流式请求的响应元数据中会包含用量数据。默认为 `undefined`（即 `false`）。

- **supportsStructuredOutputs** _boolean_

  若 provider 支持结构化输出则设为 true。仅与 `provider()`、`provider.chatModel()` 和 `provider.languageModel()` 相关。

- **transformRequestBody** _(args: Record&lt;string, any&gt;) =&gt; Record&lt;string, any&gt;_

  在发送到 API 之前转换请求体的可选函数。
  对可能需要与官方 OpenAI API 不同请求格式的代理 provider 很有用。

- **metadataExtractor** _MetadataExtractor_

  用于从 API 响应中提取 provider 特定元数据的可选元数据提取器。
  详见[自定义元数据提取](#custom-metadata-extraction)。

## 语言模型

你可以使用 provider 实例创建 provider 模型。
第一个参数为模型 id，例如 `model-id`。

```ts
const model = provider('model-id');
```

也可使用以下工厂方法：

- `provider.languageModel('model-id')` — 创建聊天语言模型（与 `provider('model-id')` 相同）
- `provider.chatModel('model-id')` — 创建聊天语言模型

### 支持的能力

使用本 provider 创建的聊天模型支持以下能力：

- **文本生成** — 生成文本补全
- **流式输出** — 实时流式传输文本响应
- **工具调用** — 调用 tools/functions，并支持流式
- **结构化输出** — 通过 schema 校验生成 JSON（在启用 `supportsStructuredOutputs` 时）
- **推理内容** — 支持返回推理/思考 token 的模型（例如 DeepSeek R1）
- **系统消息** — 支持 system 提示
- **多模态输入** — 支持图像等内容类型（取决于 provider）

### 示例

可使用 provider 语言模型配合 `generateText` 生成文本：

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateText } from 'ai';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const { text } = await generateText({
  model: provider('model-id'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

### 包含模型 id 以支持自动补全

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateText } from 'ai';

type ExampleChatModelIds =
  | 'meta-llama/Llama-3-70b-chat-hf'
  | 'meta-llama/Meta-Llama-3.1-8B-Instruct-Turbo'
  | (string & {});

type ExampleCompletionModelIds =
  | 'codellama/CodeLlama-34b-Instruct-hf'
  | 'Qwen/Qwen2.5-Coder-32B-Instruct'
  | (string & {});

type ExampleEmbeddingModelIds =
  | 'BAAI/bge-large-en-v1.5'
  | 'bert-base-uncased'
  | (string & {});

type ExampleImageModelIds = 'dall-e-3' | 'stable-diffusion-xl' | (string & {});

const model = createOpenAICompatible<
  ExampleChatModelIds,
  ExampleCompletionModelIds,
  ExampleEmbeddingModelIds,
  ExampleImageModelIds
>({
  name: 'example',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.example.com/v1',
});

// Subsequent calls to e.g. `model.chatModel` will auto-complete the model id
// from the list of `ExampleChatModelIds` while still allowing free-form
// strings as well.

const { text } = await generateText({
  model: model.chatModel('meta-llama/Llama-3-70b-chat-hf'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

### 自定义查询参数

部分 provider 可能需要自定义查询参数。例如 [Azure AI
Model Inference
API](https://learn.microsoft.com/en-us/azure/machine-learning/reference-model-inference-chat-completions?view=azureml-api-2)
需要 `api-version` 查询参数。

可通过可选的 `queryParams` provider 设置来配置。这些参数会添加到该 provider 发起的所有请求中。

```ts highlight="7-9"
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
  queryParams: {
    'api-version': '1.0.0',
  },
});
```

例如，使用上述配置时，API 请求 URL 中会包含查询参数：
`https://api.provider.com/v1/chat/completions?api-version=1.0.0`。

## 图像模型

可使用 `.imageModel()` 工厂方法创建图像模型：

```ts
const model = provider.imageModel('model-id');
```

### 基础图像生成

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateImage } from 'ai';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const { images } = await generateImage({
  model: provider.imageModel('model-id'),
  prompt: 'A futuristic cityscape at sunset',
  size: '1024x1024',
});
```

### 图像编辑

OpenAI Compatible provider 通过 `/images/edits` 端点支持图像编辑。通过 `prompt.images` 传入输入图像以变换或编辑现有图像。

#### 基础图像编辑

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateImage } from 'ai';
import fs from 'fs';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const imageBuffer = fs.readFileSync('./input-image.png');

const { images } = await generateImage({
  model: provider.imageModel('model-id'),
  prompt: {
    text: 'Turn the cat into a dog but retain the style of the original image',
    images: [imageBuffer],
  },
});
```

#### 使用遮罩的局部重绘（Inpainting）

使用遮罩编辑图像的指定区域：

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateImage } from 'ai';
import fs from 'fs';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const image = fs.readFileSync('./input-image.png');
const mask = fs.readFileSync('./mask.png');

const { images } = await generateImage({
  model: provider.imageModel('model-id'),
  prompt: {
    text: 'A sunlit indoor lounge area with a pool containing a flamingo',
    images: [image],
    mask,
  },
});
```

<Note>
  输入图像可提供为 `Buffer`、`ArrayBuffer`、`Uint8Array`、
  base64 字符串或 URL。provider 会自动下载
  基于 URL 的图像并转换为合适格式。
</Note>

## 嵌入模型

可使用 `.embeddingModel()` 工厂方法创建嵌入模型：

```ts
const model = provider.embeddingModel('model-id');
```

### 示例

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { embed } from 'ai';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const { embedding } = await embed({
  model: provider.embeddingModel('text-embedding-model'),
  value: 'The quick brown fox jumps over the lazy dog',
});
```

### 嵌入模型选项

以下 provider 选项可通过 `providerOptions` 用于嵌入模型：

- **dimensions** _number_

  输出嵌入应有的维度数量。
  仅在允许配置维度的模型中支持。

- **user** _string_

  表示终端用户的唯一标识符，可帮助 provider
  监控滥用行为。

```ts
const { embedding } = await embed({
  model: provider.embeddingModel('text-embedding-model'),
  value: 'The quick brown fox jumps over the lazy dog',
  providerOptions: {
    providerName: {
      dimensions: 512,
      user: 'user-123',
    },
  },
});
```

## 补全模型

可使用 `.completionModel()` 工厂方法创建补全模型（用于文本补全，而非聊天）：

```ts
const model = provider.completionModel('model-id');
```

### 示例

```ts
import { createOpenAICompatible } from '@ai-sdk/openai-compatible';
import { generateText } from 'ai';

const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const { text } = await generateText({
  model: provider.completionModel('completion-model-id'),
  prompt: 'The quick brown fox',
});
```

### 补全模型选项

以下 provider 选项可通过 `providerOptions` 用于补全模型：

- **echo** _boolean_

  除补全外，同时回显 prompt。

- **logitBias** _Record&lt;string, number&gt;_

  调整指定 token 出现在补全中的相对概率。
  接受将 token（按其 token ID 指定）映射到
  -100 到 100 之间偏置值的 JSON 对象。

- **suffix** _string_

  插入文本补全之后的后缀。

- **user** _string_

  表示终端用户的唯一标识符，可帮助 provider
  监控滥用行为。

```ts
const { text } = await generateText({
  model: provider.completionModel('completion-model-id'),
  prompt: 'The quick brown fox',
  providerOptions: {
    providerName: {
      echo: true,
      suffix: ' The end.',
      user: 'user-123',
    },
  },
});
```

## 聊天模型选项

以下 provider 选项可通过 `providerOptions` 用于聊天模型：

- **user** _string_

  表示终端用户的唯一标识符，可帮助 provider
  监控滥用行为。

- **reasoningEffort** _string_

  推理模型的推理强度。具体取值取决于 provider。

- **textVerbosity** _string_

  控制生成文本的详略程度。具体取值取决于 provider。

- **strictJsonSchema** _boolean_

  是否使用严格的 JSON schema 校验。为 true 时，模型使用约束解码以保证符合 schema。仅在 provider 支持
  结构化输出且提供了 schema 时使用。默认为 `true`。

```ts
const { text } = await generateText({
  model: provider('model-id'),
  prompt: 'Solve this step by step: What is 15 * 23?',
  providerOptions: {
    providerName: {
      user: 'user-123',
      reasoningEffort: 'high',
    },
  },
});
```

## Provider 专属选项

OpenAI Compatible provider 支持在请求体中加入 provider 专属选项。这些选项通过请求体中的 `providerOptions` 字段指定。

例如，若创建的 provider 实例名称为 `providerName`，可向请求体添加 `customOption` 字段，如下所示：

```ts
const provider = createOpenAICompatible({
  name: 'providerName',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
});

const { text } = await generateText({
  model: provider('model-id'),
  prompt: 'Hello',
  providerOptions: {
    providerName: { customOption: 'magic-value' },
  },
});
```

注意：`providerOptions` 的键使用 camelCase。若将 provider 名称设为 `provider-name`，选项仍须设置在 `providerOptions.providerName` 上。

发往 provider 的请求体会包含值为 `magic-value` 的 `customOption` 字段。这样无需修改 provider 或 AI SDK 代码，即可便捷地向请求添加 provider 专属选项。

## 自定义元数据提取

OpenAI Compatible provider 支持通过元数据提取器从 API 响应中提取 provider 专属元数据。
这些提取器可捕获 provider 在标准响应格式之外返回的附加信息。

元数据提取器接收来自 provider 的原始、未处理响应数据，因此可灵活提取
provider 可能包含的任意自定义字段或实验特性。
在以下情况特别有用：

- 与包含非标准响应字段的 provider 协作
- 试验 beta 或预览功能
- 捕获 provider 专属指标或调试信息
- 在无需改动 SDK 的情况下快速跟进 provider API 变化

元数据提取器同时适用于流式与非流式聊天补全，主要包含两个部分：

1. 从完整响应中提取元数据的函数
2. 可在流式响应的各个分块间累积元数据的流式提取器

下面是一个同时捕获标准与自定义 provider 数据的元数据提取器示例：

```typescript
import { MetadataExtractor } from '@ai-sdk/openai-compatible';

const myMetadataExtractor: MetadataExtractor = {
  // Process complete, non-streaming responses
  extractMetadata: ({ parsedBody }) => {
    // You have access to the complete raw response
    // Extract any fields the provider includes
    return {
      myProvider: {
        standardUsage: parsedBody.usage,
        experimentalFeatures: parsedBody.beta_features,
        customMetrics: {
          processingTime: parsedBody.server_timing?.total_ms,
          modelVersion: parsedBody.model_version,
          // ... any other provider-specific data
        },
      },
    };
  },

  // Process streaming responses
  createStreamExtractor: () => {
    let accumulatedData = {
      timing: [],
      customFields: {},
    };

    return {
      // Process each chunk's raw data
      processChunk: parsedChunk => {
        if (parsedChunk.server_timing) {
          accumulatedData.timing.push(parsedChunk.server_timing);
        }
        if (parsedChunk.custom_data) {
          Object.assign(accumulatedData.customFields, parsedChunk.custom_data);
        }
      },
      // Build final metadata from accumulated data
      buildMetadata: () => ({
        myProvider: {
          streamTiming: accumulatedData.timing,
          customData: accumulatedData.customFields,
        },
      }),
    };
  },
};
```

创建 provider 实例时可提供元数据提取器：

```typescript
const provider = createOpenAICompatible({
  name: 'my-provider',
  apiKey: process.env.PROVIDER_API_KEY,
  baseURL: 'https://api.provider.com/v1',
  metadataExtractor: myMetadataExtractor,
});
```

提取的元数据会出现在响应的 `providerMetadata` 字段下：

```typescript
const { text, providerMetadata } = await generateText({
  model: provider('model-id'),
  prompt: 'Hello',
});

console.log(providerMetadata.myProvider.customMetric);
```

这样即可访问 provider 专属信息，同时在不同 provider 之间保持一致的接口。
