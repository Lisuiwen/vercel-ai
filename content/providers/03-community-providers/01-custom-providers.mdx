# 编写自定义提供商

AI SDK 提供[语言模型规范](https://github.com/vercel/ai/tree/main/packages/provider/src/language-model/v4)，用于创建与 AI SDK 兼容的自定义提供商。该规范在不同提供商之间保持一致性。

## 发布你的提供商

请将自定义提供商发布到你自己的 GitHub 仓库与 NPM 包，并自行负责托管与维护。发布后，可向 AI SDK 仓库提交 PR，将提供商加入[社区提供商](/providers/community-providers)文档。编写文档时可参考 [OpenRouter 提供商文档](https://github.com/vercel/ai/blob/main/content/providers/03-community-providers/13-openrouter.mdx) 模板。

## 为何要采用语言模型规范？

语言模型规范 V4 是与语言模型交互的标准化规范，为所有 AI 提供商提供统一的抽象层。开发者可用相同的模式与方法对接任意提供商。它支持：

## 理解 V4 规范

语言模型规范 V4 在现有与未来 AI 提供商之上建立稳健抽象；通过统一接口，既能支持新兴 LLM 能力，又能让应用代码保持与具体提供商解耦、面向未来。

### 架构

V4 规范核心是三个主要接口：

1. **ProviderV4**：顶层工厂，用于创建各类模型实例
2. **LanguageModelV4**：文本生成模型的主接口
3. **EmbeddingModelV4** 与 **ImageModelV4**：向量嵌入与图像生成接口

### `ProviderV4`

`ProviderV4` 作为入口：

```ts
interface ProviderV4 {
  languageModel(modelId: string): LanguageModelV4;
  .embeddingModel(modelId: string): EmbeddingModelV4<string>;
  imageModel(modelId: string): ImageModelV4;
}
```

### `LanguageModelV4`

`LanguageModelV4` 规定了提供商必须实现的方法：

```ts
interface LanguageModelV4 {
  specificationVersion: 'V4';
  provider: string;
  modelId: string;
  supportedUrls: Record<string, RegExp[]>;

  doGenerate(options: LanguageModelV4CallOptions): Promise<GenerateResult>;
  doStream(options: LanguageModelV4CallOptions): Promise<StreamResult>;
}
```

要点：

- **specificationVersion**：必须为 `'V4'`
- **supportedUrls**：声明提供商可原生处理的文件片段 URL（模式）
- **doGenerate / doStream**：非流式与流式生成

### 理解输入与输出

在深入细节前，需要区分规范中的两类概念：

1. **LanguageModelV4Content**：模型生成内容的规格
2. **LanguageModelV4Prompt**：发送给模型的提示（prompt）规格

### `LanguageModelV4Content`

V4 支持模型可生成的五类内容，各自面向不同场景：

#### 文本内容

文本生成的基本单元：

```ts
type LanguageModelV4Text = {
  type: 'text';
  text: string;
};
```

用于标准模型响应、系统消息及纯文本输出。

#### 工具调用

使模型能够以结构化参数调用函数：

```ts
type LanguageModelV4ToolCall = {
  type: 'tool-call';
  toolCallType: 'function';
  toolCallId: string;
  toolName: string;
  args: string;
};
```

`toolCallId` 用于将工具结果与调用对应起来，流式场景中尤为重要。

#### 文件生成

支持多模态输出：

```ts
type LanguageModelV4File = {
  type: 'file';
  mediaType: string; // IANA media type (e.g., 'image/png', 'audio/mp3')
  data: string | Uint8Array; // Generated file data as base64 encoded strings or binary data
};
```

使模型可直接生成图像、音频、文档等。

#### 推理（Reasoning）

为链式思维推理提供专门支持（对 OpenAI o1 等模型很重要）：

```ts
type LanguageModelV4Reasoning = {
  type: 'reasoning';
  text: string;

  /**
   * Optional provider-specific metadata for the reasoning part.
   */
  providerMetadata?: SharedV4ProviderMetadata;
};
```

推理内容与常规文本分开追踪，便于 token 计量与界面展示。

#### 来源（Sources）

```ts
type LanguageModelV4Source = {
  type: 'source';
  sourceType: 'url';
  id: string;
  url: string;
  title?: string;
  providerMetadata?: SharedV4ProviderMetadata;
};
```

### `LanguageModelV4Prompt`

V4 的提示格式 `LanguageModelV4Prompt` 为灵活的消息数组，支持多模态输入：

#### 消息角色

每条消息具有角色及允许的内容类型：

- **System**：模型指令（仅文本）

  ```ts
  { role: 'system', content: string }
  ```

- **User**：人类输入，可含文本与文件

  ```ts
  { role: 'user', content: Array<LanguageModelV4TextPart | LanguageModelV4FilePart> }
  ```

- **Assistant**：助手输出，支持完整内容类型

  ```ts
  { role: 'assistant', content: Array<LanguageModelV4TextPart | LanguageModelV4FilePart | LanguageModelV4ReasoningPart | LanguageModelV4ToolCallPart> }
  ```

- **Tool**：工具执行结果
  ```ts
  { role: 'tool', content: Array<LanguageModelV4ToolResultPart> }
  ```

#### 提示片段（Prompt Parts）

提示片段用于构造输入消息。`LanguageModelV4Content` 表示模型输出，而提示片段专用于拼装输入消息。不同角色允许的片段类型不同：

- **System**：仅文本
- **User**：文本与文件片段
- **Assistant**：文本、文件、推理与工具调用片段
- **Tool**：仅工具结果片段

下面介绍各片段类型：

##### 文本片段

最基础的片段，包含纯文本：

```ts
interface LanguageModelV4TextPart {
  type: 'text';
  text: string;
  providerOptions?: SharedV4ProviderOptions;
}
```

##### 推理片段

用于助手消息中记录模型的推理过程：

```ts
interface LanguageModelV4ReasoningPart {
  type: 'reasoning';
  text: string;
  providerOptions?: SharedV4ProviderOptions;
}
```

##### 文件片段

在提示中包含文件以实现多模态输入：

```ts
interface LanguageModelV4FilePart {
  type: 'file';
  filename?: string;
  data: LanguageModelV4DataContent;
  mediaType: string;
  providerOptions?: SharedV4ProviderOptions;
}
```

`data` 字段支持：

- **Uint8Array**：原始二进制
- **string**：Base64 编码数据
- **URL**：外部内容引用（若提供商通过 `supportedUrls` 支持）

##### 工具调用片段

表示助手发起的工具调用：

```ts
interface LanguageModelV4ToolCallPart {
  type: 'tool-call';
  toolCallId: string;
  toolName: string;
  args: unknown;
  providerOptions?: SharedV4ProviderOptions;
}
```

##### 工具结果片段

承载已执行工具调用的返回：

```ts
interface LanguageModelV4ToolResultPart {
  type: 'tool-result';
  toolCallId: string;
  toolName: string;
  result: unknown;
  isError?: boolean;
  content?: Array<{
    type: 'text' | 'image';
    text?: string;
    data?: string; // base64 encoded image data
    mediaType?: string;
  }>;
  providerOptions?: SharedV4ProviderOptions;
}
```

可选字段 `content` 可表达含图像等更丰富结果，相较单纯 `result` 更灵活。

### 流式（Streaming）

#### 流式片段

流式系统使用类型化事件表示各阶段：

1. **流生命周期事件**：
   - `stream-start`：起始事件，可含不受支持特性的告警
   - `response-metadata`：模型信息与响应头
   - `finish`：结束事件，含用量统计与停止原因
   - `error`：任意时点可能发生的错误

2. **内容事件**：
   - 各类内容（`text`、`file`、`reasoning`、`source`、`tool-call`）可直接流式输出
   - `tool-call-delta`：工具调用参数的增量更新
   - `reasoning-part-finish`：推理段结束的显式标记

示例序列：

```ts
{ type: 'stream-start', warnings: [] }
{ type: 'text', text: 'Hello' }
{ type: 'text', text: ' world' }
{ type: 'tool-call', toolCallId: '1', toolName: 'search', args: {...} }
{ type: 'response-metadata', modelId: 'gpt-4.1', ... }
{ type: 'finish', usage: { inputTokens: 10, outputTokens: 20 }, finishReason: 'stop' }
```

#### 用量追踪

扩展的用量信息：

```ts
type LanguageModelV4Usage = {
  inputTokens: number | undefined;
  outputTokens: number | undefined;
  totalTokens: number | undefined;
  reasoningTokens?: number | undefined;
  cachedInputTokens?: number | undefined;
};
```

### 工具（Tools）

V4 规范支持两类工具：

#### 函数工具

使用 JSON Schema 校验的标准自定义函数：

```ts
type LanguageModelV4FunctionTool = {
  type: 'function';
  name: string;
  description?: string;
  parameters: JSONSchema7; // Full JSON Schema support
};
```

#### 提供商定义的客户端工具

将提供商原生能力暴露为工具：

```ts
export type LanguageModelV4ProviderClientDefinedTool = {
  type: 'provider-defined-client';
  id: string; // e.g., 'anthropic.computer-use'
  name: string; // Human-readable name
  args: Record<string, unknown>;
};
```

可通过以下方式控制工具选择：

```ts
toolChoice: 'auto' | 'none' | 'required' | { type: 'tool', toolName: string };
```

### 原生 URL 支持

提供商可声明可直接访问的 URL 模式：

```ts
supportedUrls: {
  'image/*': [/^https:\/\/cdn\.example\.com\/.*/],
  'application/pdf': [/^https:\/\/docs\.example\.com\/.*/],
  'audio/*': [/^https:\/\/media\.example\.com\/.*/]
}
```

AI SDK 在下载基于 URL 的内容前会校验这些模式。

### 提供商选项（Provider Options）

规范提供灵活机制承载各提供商专有特性，且不破坏通用接口：

```ts
providerOptions: {
  anthropic: {
    cacheControl: true,
    maxTokens: 4096
  },
  openai: {
    parallelToolCalls: false,
    responseFormat: { type: 'json_object' }
  }
}
```

可在多层指定：

- **调用层**：在 `LanguageModelV4CallOptions` 中
- **消息层**：单条消息上
- **片段层**：具体文本 / 文件等片段上

分层设计便于细粒度控制并保持兼容。

### 错误处理

V4 规范强调稳健错误处理：

1. **流式错误**：任意时刻可通过 `{ type: 'error', error: unknown }` 发出
2. **告警**：非致命问题在 `stream-start` 与响应对象中报告
3. **结束原因**：明确生成停止缘由：
   - `'stop'`：自然结束
   - `'length'`：达到最大 token
   - `'content-filter'`：安全过滤
   - `'tool-calls'`：为执行工具而暂停
   - `'error'`：生成失败
   - `'other'`：提供商特有原因

## 提供商实现指南

实现自定义语言模型提供商需安装：

```bash
npm install @ai-sdk/provider @ai-sdk/provider-utils
```

主要步骤包括：

- 创建入口
- 实现语言模型
- 映射输入（prompt、tools、settings）
- 处理结果（generate、streaming、tool calls）
- 支持结构化对象生成

<Note>
  入门可参考 [Mistral
  provider](https://github.com/vercel/ai/tree/main/packages/mistral) 实现。
</Note>

### 步骤 1：创建提供商入口

创建 `provider.ts`，导出工厂函数与默认实例：

```ts filename="provider.ts"
import {
  generateId,
  loadApiKey,
  withoutTrailingSlash,
} from '@ai-sdk/provider-utils';
import { ProviderV4 } from '@ai-sdk/provider';
import { CustomChatLanguageModel } from './custom-chat-language-model';

// Define your provider interface extending ProviderV4
interface CustomProvider extends ProviderV4 {
  (modelId: string, settings?: CustomChatSettings): CustomChatLanguageModel;

  // Add specific methods for different model types
  languageModel(
    modelId: string,
    settings?: CustomChatSettings,
  ): CustomChatLanguageModel;
}

// Provider settings
interface CustomProviderSettings {
  /**
   * Base URL for API calls
   */
  baseURL?: string;

  /**
   * API key for authentication
   */
  apiKey?: string;

  /**
   * Custom headers for requests
   */
  headers?: Record<string, string>;
}

// Factory function to create provider instance
function createCustom(options: CustomProviderSettings = {}): CustomProvider {
  const createChatModel = (
    modelId: string,
    settings: CustomChatSettings = {},
  ) =>
    new CustomChatLanguageModel(modelId, settings, {
      provider: 'custom',
      baseURL:
        withoutTrailingSlash(options.baseURL) ?? 'https://api.custom.ai/v1',
      headers: () => ({
        Authorization: `Bearer ${loadApiKey({
          apiKey: options.apiKey,
          environmentVariableName: 'CUSTOM_API_KEY',
          description: 'Custom Provider',
        })}`,
        ...options.headers,
      }),
      generateId: options.generateId ?? generateId,
    });

  const provider = function (modelId: string, settings?: CustomChatSettings) {
    if (new.target) {
      throw new Error(
        'The model factory function cannot be called with the new keyword.',
      );
    }

    return createChatModel(modelId, settings);
  };

  provider.languageModel = createChatModel;

  return provider as CustomProvider;
}

// Export default provider instance
const custom = createCustom();
```

### 步骤 2：实现语言模型

创建 `custom-chat-language-model.ts`，实现 `LanguageModelV4`：

```ts filename="custom-chat-language-model.ts"
import { LanguageModelV4, LanguageModelV4CallOptions } from '@ai-sdk/provider';
import { postJsonToApi } from '@ai-sdk/provider-utils';

class CustomChatLanguageModel implements LanguageModelV4 {
  readonly specificationVersion = 'V4';
  readonly provider: string;
  readonly modelId: string;

  constructor(
    modelId: string,
    settings: CustomChatSettings,
    config: CustomChatConfig,
  ) {
    this.provider = config.provider;
    this.modelId = modelId;
    // Initialize with settings and config
  }

  // Convert AI SDK prompt to provider format
  private getArgs(options: LanguageModelV4CallOptions) {
    const warnings: SharedV4Warning[] = [];

    // Map messages to provider format
    const messages = this.convertToProviderMessages(options.prompt);

    // Handle tools if provided
    const tools = options.tools
      ? this.prepareTools(options.tools, options.toolChoice)
      : undefined;

    // Build request body
    const body = {
      model: this.modelId,
      messages,
      temperature: options.temperature,
      max_tokens: options.maxOutputTokens,
      stop: options.stopSequences,
      tools,
      // ... other parameters
    };

    return { args: body, warnings };
  }

  async doGenerate(options: LanguageModelV4CallOptions) {
    const { args, warnings } = this.getArgs(options);

    // Make API call
    const response = await postJsonToApi({
      url: `${this.config.baseURL}/chat/completions`,
      headers: this.config.headers(),
      body: args,
      abortSignal: options.abortSignal,
    });

    // Convert provider response to AI SDK format
    const content: LanguageModelV4Content[] = [];

    // Extract text content
    if (response.choices[0].message.content) {
      content.push({
        type: 'text',
        text: response.choices[0].message.content,
      });
    }

    // Extract tool calls
    if (response.choices[0].message.tool_calls) {
      for (const toolCall of response.choices[0].message.tool_calls) {
        content.push({
          type: 'tool-call',
          toolCallType: 'function',
          toolCallId: toolCall.id,
          toolName: toolCall.function.name,
          args: JSON.stringify(toolCall.function.arguments),
        });
      }
    }

    return {
      content,
      finishReason: this.mapFinishReason(response.choices[0].finish_reason),
      usage: {
        inputTokens: response.usage?.prompt_tokens,
        outputTokens: response.usage?.completion_tokens,
        totalTokens: response.usage?.total_tokens,
      },
      request: { body: args },
      response: { body: response },
      warnings,
    };
  }

  async doStream(options: LanguageModelV4CallOptions) {
    const { args, warnings } = this.getArgs(options);

    // Create streaming response
    const response = await fetch(`${this.config.baseURL}/chat/completions`, {
      method: 'POST',
      headers: {
        ...this.config.headers(),
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ ...args, stream: true }),
      signal: options.abortSignal,
    });

    // Transform stream to AI SDK format
    const stream = response
      .body!.pipeThrough(new TextDecoderStream())
      .pipeThrough(this.createParser())
      .pipeThrough(this.createTransformer(warnings));

    return { stream, warnings };
  }

  // Supported URL patterns for native file handling
  get supportedUrls() {
    return {
      'image/*': [/^https:\/\/example\.com\/images\/.*/],
    };
  }
}
```

### 步骤 3：实现消息转换

将 AI SDK 消息映射为提供商格式：

```ts filename="custom-chat-language-model.ts#L50-100"
private convertToProviderMessages(prompt: LanguageModelV4Prompt) {
  return prompt.map((message) => {
    switch (message.role) {
      case 'system':
        return { role: 'system', content: message.content };

      case 'user':
        return {
          role: 'user',
          content: message.content.map((part) => {
            switch (part.type) {
              case 'text':
                return { type: 'text', text: part.text };
              case 'file':
                return {
                  type: 'image_url',
                  image_url: {
                    url: this.convertFileToUrl(part.data),
                  },
                };
              default:
                throw new Error(`Unsupported part type: ${part.type}`);
            }
          }),
        };

      case 'assistant':
        // Handle assistant messages with text, tool calls, etc.
        return this.convertAssistantMessage(message);

      case 'tool':
        // Handle tool results
        return this.convertToolMessage(message);

      default:
        throw new Error(`Unsupported message role: ${message.role}`);
    }
  });
}
```

### 步骤 4：实现流式

编写将提供商分片转换为 AI SDK 流式片段的 transformer：

```ts filename="custom-chat-language-model.ts#L150-200"
private createTransformer(warnings: SharedV4Warning[]) {
  let isFirstChunk = true;

  return new TransformStream<ParsedChunk, LanguageModelV4StreamPart>({
    async transform(chunk, controller) {
      // Send warnings with first chunk
      if (isFirstChunk) {
        controller.enqueue({ type: 'stream-start', warnings });
        isFirstChunk = false;
      }

      // Handle different chunk types
      if (chunk.choices?.[0]?.delta?.content) {
        controller.enqueue({
          type: 'text',
          text: chunk.choices[0].delta.content,
        });
      }

      if (chunk.choices?.[0]?.delta?.tool_calls) {
        for (const toolCall of chunk.choices[0].delta.tool_calls) {
          controller.enqueue({
            type: 'tool-call-delta',
            toolCallType: 'function',
            toolCallId: toolCall.id,
            toolName: toolCall.function.name,
            argsTextDelta: toolCall.function.arguments,
          });
        }
      }

      // Handle finish reason
      if (chunk.choices?.[0]?.finish_reason) {
        controller.enqueue({
          type: 'finish',
          finishReason: this.mapFinishReason(chunk.choices[0].finish_reason),
          usage: {
            inputTokens: chunk.usage?.prompt_tokens,
            outputTokens: chunk.usage?.completion_tokens,
            totalTokens: chunk.usage?.total_tokens,
          },
        });
      }
    },
  });
}
```

### 步骤 5：处理错误

使用 AI SDK 标准错误类以保持一致行为：

```ts filename="custom-chat-language-model.ts#L250-280"
import {
  APICallError,
  InvalidResponseDataError,
  TooManyRequestsError,
} from '@ai-sdk/provider';

private handleError(error: unknown): never {
  if (error instanceof Response) {
    const status = error.status;

    if (status === 429) {
      throw new TooManyRequestsError({
        cause: error,
        retryAfter: this.getRetryAfter(error),
      });
    }

    throw new APICallError({
      statusCode: status,
      statusText: error.statusText,
      cause: error,
      isRetryable: status >= 500 && status < 600,
    });
  }

  throw error;
}
```

### 步骤 6：支持 Workflow 序列化

若需与 [Workflow DevKit](https://useworkflow.dev) 兼容，模型实例在跨越 workflow 步骤边界时需可序列化。AI SDK 提供辅助方法简化该过程。

首先在配置类型中将 `headers` 设为可选，以便反序列化后的模型在无预配置认证时仍可用：

```ts filename="custom-chat-language-model.ts"
type CustomChatConfig = {
  provider: string;
  baseURL: string;
  headers?: () => Record<string, string | undefined>; // must be optional
  fetch?: FetchFunction;
  generateId?: () => string;
};
```

再在模型类中加入静态序列化方法：

```ts filename="custom-chat-language-model.ts"
import {
  serializeModel,
  deserializeModel,
  WORKFLOW_SERIALIZE,
  WORKFLOW_DESERIALIZE,
} from '@ai-sdk/provider-utils';

class CustomChatLanguageModel implements LanguageModelV4 {
  // Note: classId is generated automatically by the workflow SWC compiler at build time

  static [WORKFLOW_SERIALIZE](model: CustomChatLanguageModel) {
    return serializeModel(model);
  }

  static [WORKFLOW_DESERIALIZE](options: {
    modelId: string;
    config: CustomChatConfig;
  }) {
    return deserializeModel(CustomChatLanguageModel, options);
  }

  // ... rest of class
}
```

`serializeModel()` 会自动提取可序列化配置（字符串、数字、布尔值），并过滤 `headers`、`fetch`、`generateId` 等函数。

由于标头不会序列化，调用处请改用可选链：

```ts
// Before
headers: combineHeaders(this.config.headers(), options.headers),

// After
headers: combineHeaders(this.config.headers?.(), options.headers),
```

<Note>
  反序列化后的模型不会包含认证标头。在 workflow 场景中，认证通常由环境变量或步骤执行环境中的请求级选项处理。
</Note>
