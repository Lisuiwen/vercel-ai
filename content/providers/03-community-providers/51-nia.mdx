# Nia

由 Nozomio Labs 出品的 [Nia](https://www.trynia.ai/) 提供面向技术场景的实时上下文 API。可对文档、仓库、PDF 与数据集建立索引，再用 AI 检索，使智能体获得真实答案而非臆测。[`@nozomioai/nia-ai-sdk`](https://www.npmjs.com/package/@nozomioai/nia-ai-sdk) 通过工具、中间件与直连流式辅助函数将 Nia 接入 AI SDK。

适用于你希望：

- 将 Nia Tracer 作为工具，用于公开 GitHub 与文档调研
- 将 Nia Oracle 作为工具，对已索引仓库、文档、PDF 与数据集做有据检索
- 将 Nia Document Agent 作为工具，进行深度文档分析与结构化抽取
- 用 Nia 支持的中间件增强现有 AI SDK 模型
- 在应用中直接流式输出 Tracer、Oracle 或 Document Agent 事件

更多说明见 [Nia 文档](https://docs.trynia.ai/)。

## 安装与配置

Nia 适配器位于 `@nozomioai/nia-ai-sdk` 包。可使用以下命令安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @nozomioai/nia-ai-sdk" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @nozomioai/nia-ai-sdk" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @nozomioai/nia-ai-sdk" dark />
  </Tab>
  <Tab>
    <Snippet text="bun add @nozomioai/nia-ai-sdk" dark />
  </Tab>
</Tabs>

若要将 Nia 中间件与模型提供商配合使用，还需安装对应提供商，例如：

```bash
pnpm add @ai-sdk/openai
```

设置 Nia API 密钥：

```bash
export NIA_API_KEY=nia_your_api_key
```

可在 [Nia 控制台](https://www.trynia.ai/) 获取密钥。

## 工具用法

Nia 适合作为一组 AI SDK 工具传给 `generateText` 或 `streamText`。

### `generateText`

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { createNiaResearchTools } from '@nozomioai/nia-ai-sdk';

const tools = createNiaResearchTools({
  apiKey: process.env.NIA_API_KEY!,
  tracer: {
    defaultRequest: {
      mode: 'tracer-deep',
    },
  },
  oracle: {
    defaultRequest: {
      repositories: ['vercel/ai'],
      dataSources: ['Vercel AI SDK'],
    },
  },
  documentAgent: {
    defaultRequest: {
      sourceId: 'src_abc123',
    },
  },
});

const result = await generateText({
  model: openai('gpt-4.1'),
  prompt:
    'Research how AI SDK middleware works, then summarize the best integration pattern.',
  tools,
});

console.log(result.text);
```

可将任意服务键设为 `false` 以完全禁用，例如仅需 Tracer 时设置 `oracle: false`。各服务的传输选项（如独立 `apiKey`）在提供时会覆盖顶层配置。

SDK 还导出 `createTracerTool()`、`createOracleTool()`、`createDocumentAgentTool()`，便于仅使用单一工具。

## 中间件用法

若希望在基础模型运行前由 Nia 增强最后一条用户消息，请使用中间件。

### `withOracleContext`

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';
import { withOracleContext } from '@nozomioai/nia-ai-sdk';

const model = withOracleContext(openai('gpt-4.1'), {
  apiKey: process.env.NIA_API_KEY!,
  defaultRequest: {
    repositories: ['vercel/ai'],
    dataSources: ['Vercel AI SDK'],
  },
});

const result = await generateText({
  model,
  prompt: 'How should I think about AI SDK middleware for retrieval?',
});

console.log(result.text);
```

公开 GitHub 调研可使用同样的方式调用 `withTracerContext()`；处理已索引 PDF 或文档时使用 `withDocumentAgentContext()`。

如需更细粒度控制，低阶导出 `createTracerMiddleware()`、`createOracleMiddleware()`、`createDocumentAgentMiddleware()` 返回原始 `LanguageModelMiddleware`，可自行组合。

## 直连流式辅助函数

若需要原始 Nia 任务事件，可使用直连流式辅助函数：

- `streamTracer()`
- `streamOracle()`
- `streamOracleSessionChat()`
- `streamDocumentAgent()`

```ts
import { streamTracer } from '@nozomioai/nia-ai-sdk';

const session = await streamTracer(
  {
    apiKey: process.env.NIA_API_KEY!,
  },
  {
    query: 'How does generateText stream responses?',
    repositories: ['vercel/ai'],
    mode: 'tracer-fast',
  },
);

for await (const event of session.events) {
  console.log(event.event, event.data);
}
```
