# Jina AI 提供商

[patelvivekdev/jina-ai-provider](https://github.com/patelvivekdev/jina-ai-provider) 是社区提供商，使用 [Jina AI](https://jina.ai) 为 AI SDK 提供文本与多模态嵌入支持。

## 提供商实例

从 `jina-ai-provider` 导入默认实例 `jina`：

```ts
import { jina } from 'jina-ai-provider';
```

如需自定义，可导入 `createJina`：

```ts
import { createJina } from 'jina-ai-provider';

const customJina = createJina({
  // custom settings
});
```

可选设置：

- **baseURL** _string_

  Jina API 基础 URL。默认前缀为 `https://api.jina.ai/v1`。

- **apiKey** _string_

  通过 `Authorization` 标头发送的 API key。默认读取环境变量 `JINA_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求附加的自定义标头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch)。默认使用全局 `fetch`，可用于拦截请求或测试。

## 文本嵌入模型

使用 `.embeddingModel()` 工厂方法创建调用 Jina 文本嵌入 API 的模型。

```ts
import { jina } from 'jina-ai-provider';

const .embeddingModel = jina.embeddingModel('jina-embeddings-v3');
```

可与 `embed` 或 `embedMany` 配合：

```ts
import { jina } from 'jina-ai-provider';
import { embedMany } from 'ai';

const .embeddingModel = jina.embeddingModel('jina-embeddings-v3');

export const generateEmbeddings = async (
  value: string,
): Promise<Array<{ embedding: number[]; content: string }>> => {
  const chunks = value.split('\n');

  const { embeddings } = await embedMany({
    model: .embeddingModel,
    values: chunks,
    providerOptions: {
      jina: {
        inputType: 'retrieval.passage',
      },
    },
  });

  return embeddings.map((embedding, index) => ({
    content: chunks[index]!,
    embedding,
  }));
};
```

## 多模态嵌入

使用 `.multiModalEmbeddingModel()` 创建调用 Jina 多模态（文本 + 图像）嵌入 API 的模型。

```ts
import { jina, type MultimodalEmbeddingInput } from 'jina-ai-provider';
import { embedMany } from 'ai';

const multimodalModel = jina.multiModalEmbeddingModel('jina-clip-v2');

export const generateMultimodalEmbeddings = async () => {
  const values: MultimodalEmbeddingInput[] = [
    { text: 'A beautiful sunset over the beach' },
    { image: 'https://i.ibb.co/r5w8hG8/beach2.jpg' },
  ];

  const { embeddings } = await embedMany<MultimodalEmbeddingInput>({
    model: multimodalModel,
    values,
  });

  return embeddings.map((embedding, index) => ({
    content: values[index]!,
    embedding,
  }));
};
```

<Note type="tip">
  使用 `MultimodalEmbeddingInput` 类型以保证多模态嵌入的类型安全。`image` 属性可传入 Base64 图像，格式为 Data URL：`data:[mediatype];base64,<data>`。
</Note>

## 提供商选项

通过 `providerOptions.jina` 传入 Jina 嵌入选项，支持：

- **inputType** _'text-matching' | 'retrieval.query' | 'retrieval.passage' | 'separation' | 'classification'_

  下游用途，帮助模型生成更合适的嵌入。默认 `'retrieval.passage'`。
  - `'retrieval.query'`：输入为检索查询。
  - `'retrieval.passage'`：输入为文档/段落。
  - `'text-matching'`：语义文本相似度任务。
  - `'classification'`：分类任务。
  - `'separation'`：聚类任务。

- **outputDimension** _number_

  输出嵌入维度。有效范围见各模型文档。
  - `jina-embeddings-v3`：最小 32，最大 1024。
  - `jina-clip-v2`：最小 64，最大 1024。
  - `jina-clip-v1`：固定 768。

- **embeddingType** _'float' | 'binary' | 'ubinary' | 'base64'_

  返回嵌入的数据类型。

- **normalized** _boolean_

  是否 L2 归一化。默认 `true`。

- **truncate** _boolean_

  超出上下文是否截断而非报错。默认 `false`。

- **lateChunking** _boolean_

  将长输入自动拆为 1024 token 块。仅适用于文本嵌入模型。

## 模型能力

| Model                | Context Length (tokens) | Embedding Dimension | Modalities    |
| -------------------- | ----------------------- | ------------------- | ------------- |
| `jina-embeddings-v3` | 8,192                   | 1024                | Text          |
| `jina-clip-v2`       | 8,192                   | 1024                | Text + Images |
| `jina-clip-v1`       | 8,192                   | 768                 | Text + Images |

## 支持的输入格式

### 文本嵌入

- 字符串数组，例如：`const strings = ['text1', 'text2']`

### 多模态嵌入

- 文本对象：`const text = [{ text: 'Your text here' }]`
- 图像对象：`const image = [{ image: 'https://example.com/image.jpg' }]` 或 Base64 Data URL
- 混合数组：`const mixed = [{ text: 'object text' }, { image: 'image-url' }, { image: 'data:image/jpeg;base64,...' }]`
