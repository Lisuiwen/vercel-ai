# Hume 提供商

[Hume](https://hume.ai/) 提供商为 Hume 文本转语音（TTS）API 提供支持。

## 安装

Hume 提供商位于 `@ai-sdk/hume` 模块中，可使用以下命令安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/hume" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @ai-sdk/hume" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/hume" dark />
  </Tab>

  <Tab>
    <Snippet text="bun add @ai-sdk/hume" dark />
  </Tab>
</Tabs>

## Provider 实例

可从 `@ai-sdk/hume` 导入默认的 `hume` 提供商实例：

```ts
import { hume } from '@ai-sdk/hume';
```

如需自定义配置，可导入 `createHume` 并创建实例：

```ts
import { createHume } from '@ai-sdk/hume';

const hume = createHume({
  // custom settings, e.g.
  fetch: customFetch,
});
```

可使用以下可选设置自定义 Hume 提供商实例：

- **apiKey** _string_

  通过 `X-Hume-Api-Key` 请求头发送的 API Key。
  默认为环境变量 `HUME_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作拦截请求的中间件，
  或在测试中提供自定义 `fetch`。

## 语音模型

可使用 `.speech()` 工厂方法创建调用 [Hume speech API](https://dev.hume.ai/docs/text-to-speech-tts/overview) 的模型。

```ts
const model = hume.speech();
```

可传入常规的语音生成选项，例如 `voice`、`speed`、`instructions` 与 `outputFormat`：

```ts
import { experimental_generateSpeech as generateSpeech } from 'ai';
import { hume } from '@ai-sdk/hume';

const result = await generateSpeech({
  model: hume.speech(),
  text: 'Hello, world!',
  voice: 'd8ab67c6-953d-4bd8-9370-8fa53a0f1453',
  speed: 1.0,
  instructions: 'Speak in a friendly, conversational tone.',
  outputFormat: 'mp3',
});
```

### 支持的参数

- **text** _string_（必填）

  要转换为语音的文本。

- **voice** _string_

  生成音频所使用的音色 ID。
  默认为 `'d8ab67c6-953d-4bd8-9370-8fa53a0f1453'`。

- **speed** _number_

  语速倍率。

- **instructions** _string_

  描述或以指令说明文本应如何被朗读。

- **outputFormat** _string_

  要生成的音频格式。可选：`'mp3'`、`'pcm'`、`'wav'`。
  默认为 `'mp3'`。

<Note>
  Hume 语音模型不支持 `language` 参数，传入时将忽略并发出警告。
</Note>

### Provider 选项

可通过 `providerOptions` 传入更多提供商专有选项：

```ts
import { experimental_generateSpeech as generateSpeech } from 'ai';
import { hume } from '@ai-sdk/hume';
import { type HumeSpeechModelOptions } from '@ai-sdk/hume';

const result = await generateSpeech({
  model: hume.speech(),
  text: 'Hello, world!',
  providerOptions: {
    hume: {
      context: {
        generationId: 'previous-generation-id',
      },
    } satisfies HumeSpeechModelOptions,
  },
});
```

可用的提供商选项如下：

- **context** _object_

  语音合成请求的上下文。可为：
  - `{ generationId: string }`：使用此前一次语音合成的 ID 作为上下文。
  - `{ utterances: Utterance[] }`：上下文话语数组，其中每条 utterance 包含：
    - `text` _string_（必填）：文本内容。
    - `description` _string_：如何朗读的说明。
    - `speed` _number_：语速倍率。
    - `trailingSilence` _number_：该 utterance 结束后静默时长（秒）。
    - `voice` _object_：音色配置，为 `{ id: string, provider?: 'HUME_AI' | 'CUSTOM_VOICE' }` 或 `{ name: string, provider?: 'HUME_AI' | 'CUSTOM_VOICE' }`。
