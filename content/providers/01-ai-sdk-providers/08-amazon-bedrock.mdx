# Amazon Bedrock 提供程序

面向 [AI SDK](/docs) 的 Amazon Bedrock 提供程序可为 [Amazon Bedrock](https://aws.amazon.com/bedrock) API 提供语言模型支持。

## 设置

Bedrock 提供程序位于 `@ai-sdk/amazon-bedrock` 模块中。可按以下方式安装：

  </Tab>

### 先决条件

默认情况下并不会授予 Amazon Bedrock 基础模型的访问权限。要使用某个基础模型，具有足够权限的 IAM 用户需通过控制台申请访问权限。模型的访问获批后，该账户中的所有用户都可使用该模型。

更多信息请参阅 [模型访问文档](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access.html)。

### 身份验证

Amazon Bedrock 提供程序支持两种可自动降级身份验证的方法：API 密钥身份验证（推荐），以及使用 IAM 凭证或 AWS SDK 凭证链进行的 AWS SigV4 身份验证。

#### 使用 API 密钥（推荐）

相比 AWS SigV4 身份验证，API 密钥方式配置更简单。提供 API 密钥时，它将优先于任何 SigV4 凭证生效。

请在 [AWS 控制台](https://docs.aws.amazon.com/bedrock/latest/userguide/api-keys.html)生成 Bedrock API 密钥，然后将其设为环境变量 `AWS_BEARER_TOKEN_BEDROCK`：

```bash
AWS_BEARER_TOKEN_BEDROCK=your-api-key-here
```

或直接传给 `createAmazonBedrock`：

```ts
import { createAmazonBedrock } from '@ai-sdk/amazon-bedrock';

const amazonBedrock = createAmazonBedrock({
  apiKey: 'your-api-key-here',
  region: 'us-east-1',
});
```

当省略 `apiKey` 时，提供程序会回退使用 `AWS_BEARER_TOKEN_BEDROCK`；若两者都未配置，则会回退到 SigV4 身份验证。

#### 使用 IAM 访问密钥与秘密访问密钥

**步骤 1：创建 AWS 访问密钥与秘密访问密钥**

开始使用前，需要先创建 AWS 访问密钥与秘密访问密钥：

**登录 AWS 管理控制台**

- 打开 [AWS 管理控制台](https://console.aws.amazon.com/)，并使用你的 AWS 账户凭据登录。

**创建 IAM 用户**

- 打开 [IAM 控制台](https://console.aws.amazon.com/iam/home)，在左侧导航中点击「Users（用户）」。
- 点击「Create user（创建用户）」，填写必填信息完成创建。
- 确保访问类型选择「程序化访问（Programmatic access）」。
- 为该用户挂载 `AmazonBedrockFullAccess` 策略。

**创建访问密钥**

- 打开「Security credentials（安全凭证）」标签页，点击「Create access key（创建访问密钥）」。
- 确认创建后可生成一组新的访问密钥。
- 下载包含访问密钥 ID 与秘密访问密钥的 `.csv` 文件。

**步骤 2：配置访问密钥与秘密访问密钥**

若项目中还没有 `.env` 文件，请新增一个；该文件用来将密钥配置为环境变量。在 `.env` 中添加如下内容：

```makefile
AWS_ACCESS_KEY_ID=YOUR_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY=YOUR_SECRET_ACCESS_KEY
AWS_REGION=YOUR_REGION
```

<Note>
  许多框架（例如 [Next.js](https://nextjs.org/)）会自动加载 `.env` 文件。
  若使用其他框架，可能需要借助
  [`dotenv`](https://github.com/motdotla/dotenv) 等包手动加载 `.env`。
</Note>

请记得将 `YOUR_ACCESS_KEY_ID`、`YOUR_SECRET_ACCESS_KEY` 与 `YOUR_REGION` 替换为你 AWS 账户中的实际值。

#### 使用 AWS SDK 凭证链（实例配置文件、实例角色、ECS 角色、EKS Service Account 等）

使用 AWS SDK 时，SDK 会自动通过凭证链解析要使用的凭证，包括实例配置文件、实例角色、ECS 角色、EKS Service Account 等。在 AI SDK 中也可实现类似行为：不在提供程序设置中指定 `accessKeyId`、`secretAccessKey` 与 `sessionToken`，而是传入 `credentialProvider`。

_用法：_

`@aws-sdk/credential-providers` 包提供一组凭证提供程序，可用于构建凭证提供程序链。

## 提供程序实例

可从 `@ai-sdk/amazon-bedrock` 导入默认提供程序实例 `amazonBedrock`：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
```

若需要自定义配置，可从 `@ai-sdk/amazon-bedrock` 导入 `createAmazonBedrock`，并用你的设置创建提供程序实例：

```ts
import { createAmazonBedrock } from '@ai-sdk/amazon-bedrock';

const amazonBedrock = createAmazonBedrock({
  region: 'us-east-1',
  accessKeyId: 'xxxxxxxxx',
  secretAccessKey: 'xxxxxxxxx',
  sessionToken: 'xxxxxxxxx',
});
```

<Note>
  省略的选项会使用下方环境变量。当 **同时** 以字符串形式提供 `accessKeyId` 与 `secretAccessKey` 时，SigV4 **仅在你显式传入 `sessionToken` 时使用该 token**——不会读取环境中的 `AWS_SESSION_TOKEN`——从而避免将静态密钥与工作负载 token（例如 EKS IRSA）混用。
</Note>

可使用以下可选设置自定义 Amazon Bedrock 提供程序实例：

- **region** _string_

  API 调用要使用的 AWS 区域。
  默认读取 `AWS_REGION` 环境变量。

- **accessKeyId** _string_

  API 调用要使用的 AWS 访问密钥 ID。
  默认读取 `AWS_ACCESS_KEY_ID` 环境变量。

- **secretAccessKey** _string_

  API 调用要使用的 AWS 秘密访问密钥。
  默认读取 `AWS_SECRET_ACCESS_KEY` 环境变量。

- **sessionToken** _string_

  可选。用于临时凭证。当 **两份** 访问密钥均以字符串传入时，
  需要的话在此传入 token；不会使用环境中的 `AWS_SESSION_TOKEN`。若任一密钥来源于环境，
  省略 `sessionToken` 时才会使用环境中的 `AWS_SESSION_TOKEN`。

- **credentialProvider** _() =&gt; Promise&lt;&#123; accessKeyId: string; secretAccessKey: string; sessionToken?: string; &#125;&gt;_

  可选。API 调用要使用的 AWS 凭证提供程序链。
  默认使用所指定的凭证。

- **apiKey** _string_

  可选。用于 Bearer Token 认证的 API 密钥。
  传入后将替代 AWS SigV4 认证。
  默认读取 `AWS_BEARER_TOKEN_BEDROCK` 环境变量。

- **baseURL** _string_

  可选。Bedrock API 调用的基础 URL。
  可用于自定义终端节点或代理。

- **headers** _Record&lt;string, string&gt;_

  可选。请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) =&gt; Promise&lt;Response&gt;_

  可选。自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  可用作中间件拦截请求，
  或在测试中提供自定义 fetch。

## 语言模型

可使用提供程序实例创建调用 Bedrock API 的模型；
第一个参数是模型 ID，例如 `meta.llama3-70b-instruct-v1:0`。

```ts
const model = amazonBedrock('meta.llama3-70b-instruct-v1:0');
```

Amazon Bedrock 模型还支持若干模型特定的提供程序选项；它们不包含在[通用调用设置](/docs/ai-sdk-core/settings)中。
可通过 `providerOptions` 传入：

```ts
const model = amazonBedrock('anthropic.claude-3-sonnet-20240229-v1:0');

await generateText({
  model,
  providerOptions: {
    anthropic: {
      additionalModelRequestFields: { top_k: 350 },
    },
  },
});
```

根据所选模型的更多设置请参阅 [Amazon Bedrock 推理参数文档](https://docs.aws.amazon.com/bedrock/latest/userguide/model-parameters.html)。

可使用 Amazon Bedrock 语言模型配合 `generateText` 生成文本：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

const { text } = await generateText({
  model: amazonBedrock('meta.llama3-70b-instruct-v1:0'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

也可以在 `streamText` 中使用 Amazon Bedrock 语言模型
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

### 文件输入

<Note type="warning">
  Amazon Bedrock 仅在部分模型组合下支持文件输入，例如
  `anthropic.claude-3-haiku-20240307-v1:0`。
</Note>

Amazon Bedrock 提供程序支持文件输入，例如 PDF。

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

const result = await generateText({
  model: amazonBedrock('anthropic.claude-3-haiku-20240307-v1:0'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'Describe the pdf in detail.' },
        {
          type: 'file',
          data: readFileSync('./data/ai.pdf'),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
});
```

### 护栏（Guardrails）

可通过 `bedrock` 提供程序选项使用 [Amazon Bedrock Guardrails](https://aws.amazon.com/bedrock/guardrails/)：

```ts
import { type AmazonBedrockLanguageModelChatOptions } from '@ai-sdk/amazon-bedrock';

const result = await generateText({
  model: amazonBedrock('anthropic.claude-3-sonnet-20240229-v1:0'),
  prompt: 'Write a story about space exploration.',
  providerOptions: {
    bedrock: {
      guardrailConfig: {
        guardrailIdentifier: '1abcd2ef34gh',
        guardrailVersion: '1',
        trace: 'enabled' as const,
        streamProcessingMode: 'async',
      },
    } satisfies AmazonBedrockLanguageModelChatOptions,
  },
});
```

若启用追踪（trace），追踪信息会通过提供程序元数据返回。

```ts
if (result.providerMetadata?.bedrock.trace) {
  // ...
}
```

更多说明请参阅 [Amazon Bedrock Guardrails 文档](https://docs.aws.amazon.com/bedrock/latest/userguide/guardrails.html)。

### 引用（Citations）

对于兼容模型的基于文档输入，Amazon Bedrock 支持 citations。启用后：

- 部分模型可进行视觉层面的文档理解，而不仅是抽取文本
- 模型可以引用你所提供文档的特定片段，便于追溯信息来源（尚不支持）

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateText, Output } from 'ai';
import { z } from 'zod';
import fs from 'fs';

const result = await generateText({
  model: amazonBedrock('apac.anthropic.claude-sonnet-4-20250514-v1:0'),
  output: Output.object({
    schema: z.object({
      summary: z.string().describe('Summary of the PDF document'),
      keyPoints: z.array(z.string()).describe('Key points from the PDF'),
    }),
  }),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'Summarize this PDF and provide key points.',
        },
        {
          type: 'file',
          data: readFileSync('./document.pdf'),
          mediaType: 'application/pdf',
          providerOptions: {
            bedrock: {
              citations: { enabled: true },
            },
          },
        },
      ],
    },
  ],
});

console.log('Response:', result.output);
```

### 缓存点（Cache Points）

<Note>
  Amazon Bedrock 提示词缓存当前为预览版本。若要申请权限，请访问 [Amazon Bedrock 提示缓存页面](https://aws.amazon.com/bedrock/prompt-caching/)。
</Note>

在消息中可使用 `providerOptions` 配置缓存点。将 `providerOptions` 对象中的 `bedrock` 设为 `{ cachePoint: { type: 'default' } }` 即可创建缓存点。

也可通过 `ttl` 为缓存点指定存活时间（TTL）。支持 `'5m'`（5 分钟，默认）与 `'1h'`（1 小时）。其中 1 小时 TTL 仅 Claude Opus 4.5、Claude Haiku 4.5 与 Claude Sonnet 4.5 支持。

```ts
providerOptions: {
  bedrock: { cachePoint: { type: 'default', ttl: '1h' } },
}
```

<Note>
  使用多个不同 TTL 的缓存点时，较长 TTL 的缓存条目必须出现在较短 TTL 之前（即：1 小时缓存应在 5 分钟缓存之前）。
</Note>

缓存使用量信息会通过 `providerMetadata` 返回。示例如下。

<Note>
  缓存点对不同模型有不同 token 下限与上限。例如
  Claude 3.5 Sonnet v2 要求缓存点至少需要 1,024 个 token，且最多可有 4 个缓存点。详见 [Amazon Bedrock 提示缓存文档](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html)中支持的模型、区域与限额。
</Note>

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

const cyberpunkAnalysis =
  '... literary analysis of cyberpunk themes and concepts ...';

const result = await generateText({
  model: amazonBedrock('anthropic.claude-3-5-sonnet-20241022-v2:0'),
  messages: [
    {
      role: 'system',
      content: `You are an expert on William Gibson's cyberpunk literature and themes. You have access to the following academic analysis: ${cyberpunkAnalysis}`,
      providerOptions: {
        bedrock: { cachePoint: { type: 'default' } },
      },
    },
    {
      role: 'user',
      content:
        'What are the key cyberpunk themes that Gibson explores in Neuromancer?',
    },
  ],
});

console.log(result.text);
console.log(result.providerMetadata?.bedrock?.usage);
// Shows cache read/write token usage, e.g.:
// {
//   cacheReadInputTokens: 1337,
//   cacheWriteInputTokens: 42,
// }
```

缓存点同样适用于流式响应：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { streamText } from 'ai';

const cyberpunkAnalysis =
  '... literary analysis of cyberpunk themes and concepts ...';

const result = streamText({
  model: amazonBedrock('anthropic.claude-3-5-sonnet-20241022-v2:0'),
  messages: [
    {
      role: 'assistant',
      content: [
        { type: 'text', text: 'You are an expert on cyberpunk literature.' },
        { type: 'text', text: `Academic analysis: ${cyberpunkAnalysis}` },
      ],
      providerOptions: { bedrock: { cachePoint: { type: 'default' } } },
    },
    {
      role: 'user',
      content:
        'How does Gibson explore the relationship between humanity and technology?',
    },
  ],
});

for await (const textPart of result.textStream) {
  process.stdout.write(textPart);
}

console.log(
  'Cache token usage:',
  (await result.providerMetadata)?.bedrock?.usage,
);
// Shows cache read/write token usage, e.g.:
// {
//   cacheReadInputTokens: 1337,
//   cacheWriteInputTokens: 42,
// }
```

### 提供程序元数据

以下为可能出现在 `providerMetadata.bedrock` 中的 Bedrock 专属元数据：

- **trace** _(optional)_
  Guardrail 追踪信息（启用追踪时）。
- **performanceConfig** _(optional)_
  性能配置，例如 `{ latency: 'optimized' }`。
- **serviceTier** _(optional)_
  服务层级信息，例如 `{ type: 'on-demand' }`。
- **usage** _(optional)_
  缓存 token 用量详情，包含 `cacheWriteInputTokens` 与 `cacheDetails`。
- **stopSequence** _string | null_
  触发停止时使用的停止序列（如有）。

## 推理（Reasoning）

Amazon Bedrock 支持模型创建方专属的推理能力：

- Anthropic（例如 `claude-sonnet-4-5-20250929`）：通过 `reasoningConfig` 提供程序选项启用，并以 token 指定思考预算（最小：`1024`，最大：`64000`）。
- Amazon（例如 `us.amazon.nova-2-lite-v1:0`）：通过 `reasoningConfig` 启用，并指定最大推理力度（`'low' | 'medium' | 'high'`）。

```ts
import {
  amazonBedrock,
  type AmazonBedrockLanguageModelChatOptions,
} from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

// Anthropic example
const anthropicResult = await generateText({
  model: amazonBedrock('us.anthropic.claude-sonnet-4-5-20250929-v1:0'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    bedrock: {
      reasoningConfig: { type: 'enabled', budgetTokens: 1024 },
    } satisfies AmazonBedrockLanguageModelChatOptions,
  },
});

console.log(anthropicResult.reasoningText); // reasoning text
console.log(anthropicResult.text); // text response

// Nova 2 example
const amazonResult = await generateText({
  model: amazonBedrock('us.amazon.nova-2-lite-v1:0'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    bedrock: {
      reasoningConfig: { type: 'enabled', maxReasoningEffort: 'medium' },
    } satisfies AmazonBedrockLanguageModelChatOptions,
  },
});

console.log(amazonResult.reasoningText); // reasoning text
console.log(amazonResult.text); // text response
```

关于如何将推理能力集成到聊天机器人中，详见 [AI SDK UI：Chatbot](/docs/ai-sdk-ui/chatbot#reasoning)。

## 服务层级（Service Tiers）

Amazon Bedrock 支持在每个请求上通过 `serviceTier` 提供程序选项选择推理服务层级。

```ts
import {
  amazonBedrock,
  type AmazonBedrockLanguageModelChatOptions,
} from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

const result = await generateText({
  model: amazonBedrock('us.anthropic.claude-sonnet-4-20250514-v1:0'),
  prompt: 'Summarize this support ticket backlog.',
  providerOptions: {
    bedrock: {
      serviceTier: 'priority',
    } satisfies AmazonBedrockLanguageModelChatOptions,
  },
});
```

可取值为：

- `reserved`
- `priority`
- `default`
- `flex`

模型支持与行为请参阅 [Amazon Bedrock 服务层级文档](https://docs.aws.amazon.com/bedrock/latest/userguide/service-tiers-inference.html)。

## 扩展上下文窗口

在使用 `context-1m-2025-08-07` Beta 特性时，Amazon Bedrock 上的 Claude Sonnet 4 模型支持最长约 100 万个 token 的上下文窗口。

```ts
import {
  amazonBedrock,
  type AmazonBedrockLanguageModelChatOptions,
} from '@ai-sdk/amazon-bedrock';
import { generateText } from 'ai';

const result = await generateText({
  model: amazonBedrock('us.anthropic.claude-sonnet-4-20250514-v1:0'),
  prompt: 'analyze this large document...',
  providerOptions: {
    bedrock: {
      anthropicBeta: ['context-1m-2025-08-07'],
    } satisfies AmazonBedrockLanguageModelChatOptions,
  },
});
```

## Computer Use（计算机操控）

借助 Anthropic，Amazon Bedrock 提供三种由提供程序定义的工具，可与外部系统交互：

1. **Bash Tool**：允许执行 bash 命令。
2. **Text Editor Tool**：查看与编辑文本文件的能力。
3. **Computer Tool**：模拟键盘与鼠标操控计算机。

这些工具可通过提供程序实例的 `tools` 访问。

<Note>
  Amazon Bedrock 不支持工具定义上的 strict mode。设置
  `strict: true` 将被忽略并产生警告。
</Note>

### Bash 工具（Bash Tool）

Bash Tool 可用于运行 bash 命令；创建与使用方式如下：

```ts
const bashTool = amazonBedrock.tools.bash_20241022({
  execute: async ({ command, restart }) => {
    // Implement your bash command execution logic here
    // Return the result of the command execution
  },
});
```

参数：

- `command`（string）：要执行的 bash 命令；除非正在重启工具，否则必填。
- `restart`（boolean，可选）：为 `true` 时会重启该工具。

### Text Editor 工具（Text Editor Tool）

Text Editor Tool 可用于查看并编辑文本文件。

**适用于 Claude 4 模型（Opus 与 Sonnet）：**

```ts
const textEditorTool = amazonBedrock.tools.textEditor_20250429({
  execute: async ({
    command,
    path,
    file_text,
    insert_line,
    new_str,
    insert_text,
    old_str,
    view_range,
  }) => {
    // Implement your text editing logic here
    // Return the result of the text editing operation
  },
});
```

**适用于 Claude 3.5 Sonnet 及更早模型：**

```ts
const textEditorTool = amazonBedrock.tools.textEditor_20241022({
  execute: async ({
    command,
    path,
    file_text,
    insert_line,
    new_str,
    insert_text,
    old_str,
    view_range,
  }) => {
    // Implement your text editing logic here
    // Return the result of the text editing operation
  },
});
```

参数：

- `command`（`'view' | 'create' | 'str_replace' | 'insert' | 'undo_edit'`）：要执行的命令。说明：`undo_edit` 仅在 Claude 3.5 Sonnet 及更早模型可用。
- `path`（string）：文件或目录的绝对路径，例如 `/repo/file.py` 或 `/repo`。
- `file_text`（string，可选）：用于 `create` 命令时为必填；为要创建的文件内容。
- `insert_line`（number，可选）：用于 `insert` 命令时为必填；为新字符串插入位置之后的行号。
- `new_str`（string，可选）：`str_replace` 命令的新字符串。
- `insert_text`（string，可选）：用于 `insert` 命令时为必填；为要插入的文本。
- `old_str`（string，可选）：用于 `str_replace` 命令时为必填；为要被替换的字符串。
- `view_range`（number[]，可选）：用于 `view` 命令时可选；指定要显示的行范围。

使用 Text Editor Tool 时，请确保 `tools` 对象中的键名正确：

- **Claude 4 模型**：使用 `str_replace_based_edit_tool`
- **Claude 3.5 Sonnet 及更早**：使用 `str_replace_editor`

```ts
// Claude 4 模型
const response = await generateText({
  model: amazonBedrock('us.anthropic.claude-sonnet-4-20250514-v1:0'),
  prompt:
    "Create a new file called example.txt, write 'Hello World' to it, and run 'cat example.txt' in the terminal",
  tools: {
    str_replace_based_edit_tool: textEditorTool, // Claude 4 tool name
  },
});

// Claude 3.5 Sonnet 及更早模型
const response = await generateText({
  model: amazonBedrock('anthropic.claude-3-5-sonnet-20241022-v2:0'),
  prompt:
    "Create a new file called example.txt, write 'Hello World' to it, and run 'cat example.txt' in the terminal",
  tools: {
    str_replace_editor: textEditorTool, // Earlier models tool name
  },
});
```

## 嵌入模型

可通过 `.embedding()` 工厂方法创建调用 Bedrock [嵌入模型 API](https://docs.aws.amazon.com/bedrock/latest/userguide/titan-embedding-models.html)的模型实例。

```ts
const model = amazonBedrock.embedding('amazon.titan-embed-text-v1');
```

Bedrock Titan 嵌入模型 `amazon.titan-embed-text-v2:0` 还支持若干额外设置；
可作为选项参数传入：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { type AmazonBedrockEmbeddingModelOptions } from '@ai-sdk/amazon-bedrock';
import { embed } from 'ai';

const model = amazonBedrock.embedding('amazon.titan-embed-text-v2:0');

const { embedding } = await embed({
  model,
  value: 'sunny day at the beach',
  providerOptions: {
    bedrock: {
      dimensions: 512, // optional, number of dimensions for the embedding
      normalize: true, // optional, normalize the output embeddings
    } satisfies AmazonBedrockEmbeddingModelOptions,
  },
});
```

以下为 Bedrock Titan 嵌入模型可用的可选提供程序选项：

- **dimensions**：_number_

  输出向量应有的维度。可取：1024（默认）、512、256。

- **normalize** _boolean_

  是否对输出向量进行归一化；默认 `true`。

### Nova 嵌入模型

import { amazonBedrock } from '@ai-sdk/amazon-bedrock';

## 图像模型

可通过 `.image()` 工厂方法创建调用 Bedrock [图像生成 API](https://docs.aws.amazon.com/nova/latest/userguide/image-generation.html)的模型实例。

关于 Amazon Nova Canvas 图像模型，请参阅 [Nova Canvas 概览](https://docs.aws.amazon.com/ai/responsible-ai/nova-canvas/overview.html)。

<Note>
  `amazon.nova-canvas-v1:0` 模型在 `us-east-1`、`eu-west-1`、`ap-northeast-1`
  区域可用。
</Note>

```ts
const model = amazonBedrock.image('amazon.nova-canvas-v1:0');
```

随后可使用 `generateImage` 生成图像：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: 'A beautiful sunset over a calm ocean',
  size: '512x512',
  seed: 42,
});
```

也可向 `generateImage` 传入 `providerOptions`，定制生成行为：

```ts
import { amazonBedrock } from '@ai-sdk/amazon-bedrock';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: 'A beautiful sunset over a calm ocean',
  size: '512x512',
  seed: 42,
  providerOptions: {
    bedrock: {
      quality: 'premium',
      negativeText: 'blurry, low quality',
      cfgScale: 7.5,
      style: 'PHOTOREALISM',
    },
  },
});
```

Amazon Nova Canvas 支持的可选提供程序选项如下：

- **quality** _string_

  图像生成质量档位；接受 `'standard'` 或 `'premium'`。

- **negativeText** _string_

  描述你不希望出现在图像中的元素。

- **cfgScale** _number_

  控制生成图像与提示词的贴合程度；数值越高越贴近提示。

- **style** _string_

  预定义的图像视觉风格；
  可从以下任选其一：
  `3D_ANIMATED_FAMILY_FILM` · `DESIGN_SKETCH` · `FLAT_VECTOR_ILLUSTRATION` ·
  `GRAPHIC_NOVEL_ILLUSTRATION` · `MAXIMALISM` · `MIDCENTURY_RETRO` ·
  `PHOTOREALISM` · `SOFT_DIGITAL_PAINTING`。

更多信息见 [Amazon Bedrock 面向 Amazon Nova 的用户指南——请求/响应结构](https://docs.aws.amazon.com/nova/latest/userguide/image-gen-req-resp-structure.html)。

### 图像编辑

Amazon Nova Canvas 支持多种图像编辑任务类型。当你通过 `prompt.images` 提供输入图像时，模型会自动选择合适的编辑模式，也可在提供程序选项中显式指定 `taskType`。

#### 图像变体（Image Variation）

在保留核心视觉特征的前提下，对现有图像生成变体：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: {
    text: 'Modernize the style, photo-realistic, 8k, hdr',
    images: [imageBuffer],
  },
  providerOptions: {
    bedrock: {
      taskType: 'IMAGE_VARIATION',
      similarityStrength: 0.7, // 0-1, higher = closer to original
      negativeText: 'bad quality, low resolution',
    },
  },
});
```

- **similarityStrength** _number_

  控制输出与输入图的相似程度；取值 `0`-`1`，越大越贴近原图。

#### 涂抹修复（Inpainting）

可对图像特定区域编辑；可使用蒙版图像或文本提示来描述要修改的范围：

**使用掩码提示（文本选择）：**

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: {
    text: 'a cute corgi dog in the same style',
    images: [imageBuffer],
  },
  providerOptions: {
    bedrock: {
      maskPrompt: 'cat', // Describe what to replace
    },
  },
  seed: 42,
});
```

**使用蒙版图像：**

```ts
const image = readFileSync('./input-image.png');
const mask = readFileSync('./mask.png'); // White pixels = area to change

const { images } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: {
    text: 'A sunlit indoor lounge area with a pool containing a flamingo',
    images: [image],
    mask: mask,
  },
});
```

- **maskPrompt** _string_

  用文字描述待修改区域的文本；模型会自动定位并遮盖对应区域。

#### 外延绘制（Outpainting）

在原图边界之外扩展画布：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: {
    text: 'A beautiful sunset landscape with mountains',
    images: [imageBuffer],
  },
  providerOptions: {
    bedrock: {
      taskType: 'OUTPAINTING',
      maskPrompt: 'background',
      outPaintingMode: 'DEFAULT', // or 'PRECISE'
    },
  },
});
```

- **outPaintingMode** _string_

  外延绘制的渲染方式；接受 `'DEFAULT'` 或 `'PRECISE'`。

#### 背景移除

移除图像背景：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: {
    images: [imageBuffer],
  },
  providerOptions: {
    bedrock: {
      taskType: 'BACKGROUND_REMOVAL',
    },
  },
});
```

<Note>
  背景移除无需文本提示，仅需提供输入图像。
</Note>

#### 图像编辑的提供程序选项

图像编辑还可使用以下额外提供程序选项：

- **taskType** _string_

  显式指定编辑任务类型；接受 `'TEXT_IMAGE'`（仅文本时为默认）、`'IMAGE_VARIATION'`、`'INPAINTING'`、`'OUTPAINTING'`、`'BACKGROUND_REMOVAL'`。提供图像但未指定 `taskType` 时，模型默认 `'IMAGE_VARIATION'`（若提供蒙版则为 `'INPAINTING'`）。

- **maskPrompt** _string_

  用文字描述修改区域（用于 inpainting/outpainting）；可提供蒙版图像的替代写法。

- **similarityStrength** _number_

  适用于 `IMAGE_VARIATION`：控制与原图的相似程度（`0`-`1`）。

- **outPaintingMode** _string_

  适用于 `OUTPAINTING`：外延绘制行为（`'DEFAULT'` 或 `'PRECISE'`）。

### 图像模型的通用设置

可通过可选选项进一步定制生成行为：

```ts
await generateImage({
  model: amazonBedrock.image('amazon.nova-canvas-v1:0'),
  prompt: 'A beautiful sunset over a calm ocean',
  size: '512x512',
  seed: 42,
  maxImagesPerCall: 1, // Maximum number of images to generate per API call
});
```

- **maxImagesPerCall** _number_

  覆盖单次 API 调用生成的最大图像数；默认值因模型而异，常见默认值为 5。

### 模型能力与尺寸约束

Amazon Nova Canvas 模型的自定义尺寸需满足以下条件：

- 每条边长度在 **320–4096** 像素（含端点）。
- 每条边必须是 **16** 的倍数。
- 宽高比须在 **1:4** 到 **4:1** 之间（任一边不能超过另一边的 4 倍）。
- 总像素须小于 **4,194,304**。

更多请参阅 [图像生成访问与配额](https://docs.aws.amazon.com/nova/latest/userguide/image-gen-access.html)。

| 模型                         | 尺寸                                                                                                  |
| ------------------------- | ----------------------------------------------------------------------------------------------------- |
| `amazon.nova-canvas-v1:0` | 自定义尺寸：边长 320–4096px（须整除 16），宽高比 1:4 至 4:1，像素上限约 4.2M |

## Bedrock Anthropic 提供程序

Bedrock Anthropic 提供程序通过 Amazon Bedrock 原生 InvokeModel API 支持 Anthropic Claude 模型，功能与 [Anthropic API](https://platform.claude.com/docs/en/build-with-claude/overview)对等，包含 Converse API 未必提供的特性（例如流式响应中的 `stop_sequence`）。

如需了解 Amazon Bedrock 上可用的 Claude 模型，请参阅 [Claude on Amazon Bedrock](https://platform.claude.com/docs/en/build-with-claude/claude-on-amazon-bedrock)。

### 提供程序实例

可从 `@ai-sdk/amazon-bedrock/anthropic` 导入默认提供程序实例 `bedrockAnthropic`：

```typescript
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
```

如需自定义配置，可从 `@ai-sdk/amazon-bedrock/anthropic` 导入 `createBedrockAnthropic`，并用你的设置创建提供程序实例：

```typescript
import { createBedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';

const bedrockAnthropic = createBedrockAnthropic({
  region: 'us-east-1', // optional
  accessKeyId: 'xxxxxxxxx', // optional
  secretAccessKey: 'xxxxxxxxx', // optional
  sessionToken: 'xxxxxxxxx', // optional
});
```

#### 提供程序设置

可使用下列可选配置自定义 Bedrock Anthropic 提供程序实例：

- **region** _string_

  API 调用使用的 AWS 区域；
  默认读取 `AWS_REGION`。

- **accessKeyId** _string_

  API 调用使用的 AWS 访问密钥 ID；
  默认读取 `AWS_ACCESS_KEY_ID`。

- **secretAccessKey** _string_

  API 调用使用的 AWS 秘密访问密钥；
  默认读取 `AWS_SECRET_ACCESS_KEY`。

- **sessionToken** _string_

  可选。临时凭证。**两份**访问密钥均以字符串传入时，
  需要的话在此传入 token；不会读取环境中的 `AWS_SESSION_TOKEN`。若任一密钥来源于环境，
  省略 `sessionToken` 时才会使用 `AWS_SESSION_TOKEN`。

- **apiKey** _string_

  Bearer Token 认证的 API 密钥；
  传入后将替代 SigV4；
  默认读取 `AWS_BEARER_TOKEN_BEDROCK`。

- **baseURL** _string_

  Bedrock API 的基础 URL；
  可用于自定义终端节点或代理。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  请求中要包含的头字段。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch)实现；
  可作为请求中间件，
  或用于测试等场景的自定义 fetch。

- **credentialProvider** _() => PromiseLike&lt;BedrockCredentials&gt;_

  类似于 AWS SDK 的动态凭证来源；设置后，
  其返回值将替代 `accessKeyId`、`secretAccessKey` 与 `sessionToken`。

### 语言模型

可使用提供程序实例调用 [Anthropic Messages API](https://docs.anthropic.com/claude/reference/messages_post)；第一个参数为模型 ID，例如 `us.anthropic.claude-3-5-sonnet-20241022-v2:0`。

```ts
const model = bedrockAnthropic('us.anthropic.claude-3-5-sonnet-20241022-v2:0');
```

也可以使用 Bedrock Anthropic 模型配合 `generateText` 生成文本：

```ts
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: bedrockAnthropic('us.anthropic.claude-3-5-sonnet-20241022-v2:0'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

### 提供程序选项

Bedrock Anthropic 模型支持下列可选提供程序选项：

- `metadata` _object_

  可选。随请求发送的元数据；详见 [Anthropic API 文档](https://platform.claude.com/docs/en/api/messages/create)。
  - `userId` _string_：终端用户的外部标识符。

### 缓存控制（Cache Control）

在消息与消息片段中，可通过 `providerOptions` 设置缓存断点。
将 `providerOptions` 中的 `anthropic` 设为 `{ cacheControl: { type: 'ephemeral' } }` 即可添加缓存断点。

```ts
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: bedrockAnthropic('us.anthropic.claude-sonnet-4-5-20250929-v1:0'),
  messages: [
    {
      role: 'system',
      content: 'You are an expert assistant.',
      providerOptions: {
        anthropic: { cacheControl: { type: 'ephemeral' } },
      },
    },
    {
      role: 'user',
      content: 'Explain quantum computing.',
    },
  ],
});
```

<Note>
  缓存控制要求断点前至少 1,024 个 token。支持的模型与限制详见 [Amazon Bedrock 提示缓存文档](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html)。
</Note>

### Computer Use（计算机操控）

Bedrock Anthropic 提供程序支持 Anthropic 的 computer use 工具：

1. **Bash Tool**：执行 bash 命令。
2. **Text Editor Tool**：查看与编辑文本文件。
3. **Computer Tool**：模拟键盘与鼠标操作。

通过提供程序实例的 `tools` 访问。

<Note>
  Computer use 工具需要 Claude 3.7 Sonnet 或更新模型。Claude 3.5 Sonnet v2 不支持这些工具。
</Note>

#### Bash 工具（Bash Tool）

```ts
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
import { generateText, isStepCount } from 'ai';

const result = await generateText({
  model: bedrockAnthropic('us.anthropic.claude-sonnet-4-5-20250929-v1:0'),
  tools: {
    bash: bedrockAnthropic.tools.bash_20241022({
      execute: async ({ command }) => {
        // Implement your bash command execution logic here
        return [{ type: 'text', text: `Executed: ${command}` }];
      },
    }),
  },
  prompt: 'List the files in my directory.',
  stopWhen: isStepCount(2),
});
```

#### Text Editor 工具（Text Editor Tool）

```ts
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
import { generateText, isStepCount } from 'ai';

const result = await generateText({
  model: bedrockAnthropic('us.anthropic.claude-sonnet-4-5-20250929-v1:0'),
  tools: {
    str_replace_editor: bedrockAnthropic.tools.textEditor_20241022({
      execute: async ({ command, path, old_str, new_str, insert_text }) => {
        // Implement your text editing logic here
        return 'File updated successfully';
      },
    }),
  },
  prompt: 'Update my README file.',
  stopWhen: isStepCount(5),
});
```

#### Computer 工具（Computer Tool）

```ts
import { bedrockAnthropic } from '@ai-sdk/amazon-bedrock/anthropic';
import { generateText, isStepCount } from 'ai';
import fs from 'fs';

const result = await generateText({
  model: bedrockAnthropic('us.anthropic.claude-sonnet-4-5-20250929-v1:0'),
  tools: {
    computer: bedrockAnthropic.tools.computer_20241022({
      displayWidthPx: 1024,
      displayHeightPx: 768,
      execute: async ({ action, coordinate, text }) => {
        if (action === 'screenshot') {
          return {
            type: 'image',
            data: fs.readFileSync('./screenshot.png').toString('base64'),
          };
        }
        return `executed ${action}`;
      },
      toModelOutput({ output }) {
        return {
          type: 'content',
          value: [
            typeof output === 'string'
              ? { type: 'text', text: output }
              : {
                  type: 'image-data',
                  data: output.data,
                  mediaType: 'image/png',
                },
          ],
        };
      },
    }),
  },
  prompt: 'Take a screenshot.',
  stopWhen: isStepCount(3),
});
```

## 迁移到 `@ai-sdk/amazon-bedrock` 2.x

Amazon Bedrock 提供程序在 2.x 版本重写，移除对 `@aws-sdk/client-bedrock-runtime` 的依赖。

原先可用的 `bedrockOptions` 提供程序设置已被删除。若先前使用 `bedrockOptions`，请改用顶层的 `region`、`accessKeyId`、`secretAccessKey` 与 `sessionToken`。

静态 IAM 用户密钥不需要 `sessionToken`。
