# Vercel 提供商

[Vercel](https://vercel.com) 提供商可访问专为构建现代 Web 应用设计的 [v0 API](https://v0.app/docs/api/model)。v0 模型支持文本与图像输入，并提供低延迟的流式输出。

你可以在 [v0.dev](https://v0.dev/chat/settings/keys) 创建 Vercel API Key。

<Note>
  v0 API 当前处于 Beta，需要 Premium 或 Team 套餐并启用按量计费。详情请见[定价页面](https://v0.dev/pricing)。如需更高配额，请联系 Vercel：`support@v0.dev`。
</Note>

## 功能特性

- **框架感知补全**：针对 Next.js、Vercel 等现代栈优化
- **自动修复**：在生成过程中识别并纠正常见编码问题
- **快速编辑**：随生成进度流式输出内联修改
- **多模态**：同时支持文本与图像输入

## Provider 实例

可从 `@ai-sdk/vercel` 导入默认的 `vercel` 提供商实例：

```ts
import { vercel } from '@ai-sdk/vercel';
```

如需自定义配置，可导入 `createVercel` 并按需创建实例：

```ts
import { createVercel } from '@ai-sdk/vercel';

const vercel = createVercel({
  apiKey: process.env.VERCEL_API_KEY ?? '',
});
```

可使用以下可选设置自定义 Vercel 提供商实例：

- **baseURL** _string_

  为 API 请求使用不同的 URL 前缀。默认前缀为 `https://api.v0.dev/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。默认为环境变量 `VERCEL_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中额外加入的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认使用全局 `fetch`。
  可用作拦截请求的中间件，
  或在测试中提供自定义 `fetch`。

## 语言模型

可通过提供商实例创建语言模型，首个参数为模型 ID，例如：

```ts
import { vercel } from '@ai-sdk/vercel';
import { generateText } from 'ai';

const { text } = await generateText({
  model: vercel('v0-1.5-md'),
  prompt: 'Create a Next.js AI chatbot',
});
```

Vercel 语言模型也可用于 `streamText`（参见 [AI SDK Core](/docs/ai-sdk-core)）。

## 模型

### v0-1.5-md

`v0-1.5-md` 适用于日常任务与界面生成。

### v0-1.5-lg

`v0-1.5-lg` 适用于需要更强推理或思考的任务。

### v0-1.0-md（遗留）

`v0-1.0-md` 是由 v0 API 提供的遗留模型。

所有 v0 模型具备以下能力：

- 支持文本与图像输入（多模态）
- 支持函数/工具调用
- 低延迟流式响应
- 面向前端与全栈 Web 开发优化
