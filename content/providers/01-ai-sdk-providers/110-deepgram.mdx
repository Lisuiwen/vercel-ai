# Deepgram 提供商

[Deepgram](https://deepgram.com/) 提供商为 Deepgram 的转写与语音合成 API 提供语言模型支持。

## 安装

Deepgram 提供商位于 `@ai-sdk/deepgram` 模块中，可使用以下命令安装：

  </Tab>

  </Tab>

  <Tab>

  </Tab>
</Tabs>

## Provider 实例

可从 `@ai-sdk/deepgram` 导入默认的 `deepgram` 提供商实例：

```ts
import { deepgram } from '@ai-sdk/deepgram';
```

如需自定义配置，可导入 `createDeepgram` 并创建实例：

```ts
import { createDeepgram } from '@ai-sdk/deepgram';

const deepgram = createDeepgram({
  // custom settings, e.g.
  fetch: customFetch,
});
```

可使用以下可选设置自定义 Deepgram 提供商实例：

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。
  默认为环境变量 `DEEPGRAM_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作拦截请求的中间件，
  或在测试中提供自定义 `fetch`。

## 语音模型

可使用 `.speech()` 工厂方法创建调用 [Deepgram TTS API](https://developers.deepgram.com/docs/text-to-speech) 的模型。

首个参数为模型 ID，其中已包含音色：Deepgram 将音色直接编码在模型 ID 中（例如 `aura-2-helena-en`）。

```ts
const model = deepgram.speech('aura-2-helena-en');
```

可与 `generateSpeech` 搭配使用：

```ts
import { experimental_generateSpeech as generateSpeech } from 'ai';
import { deepgram } from '@ai-sdk/deepgram';

const result = await generateSpeech({
  model: deepgram.speech('aura-2-helena-en'),
  text: 'Hello, world!',
});
```

也可通过 `providerOptions` 传入额外选项：

```ts highlight="7-11"
import { experimental_generateSpeech as generateSpeech } from 'ai';
import { deepgram, type DeepgramSpeechModelOptions } from '@ai-sdk/deepgram';

const result = await generateSpeech({
  model: deepgram.speech('aura-2-helena-en'),
  text: 'Hello, world!',
  providerOptions: {
    deepgram: {
      encoding: 'linear16',
      sampleRate: 24000,
    } satisfies DeepgramSpeechModelOptions,
  },
});
```

可用的提供商选项：

- **encoding** _string_

  输出音频编码。
  支持：`linear16`、`mulaw`、`alaw`、`mp3`、`opus`、`flac`、`aac`。
  可选。

- **container** _string_

  输出容器格式。
  支持：`wav`、`ogg`、`none`。
  可选。

- **sampleRate** _number_

  输出采样率（Hz）。
  取值因编码而异：`8000`、`16000`、`24000`、`32000`、`48000`。
  可选。

- **bitRate** _number | string_

  音频比特率（bps）。
  `mp3`：`32000` 或 `48000`；
  `opus`：4000–650000；
  `aac`：4000–192000。
  可选。

- **callback** _string_

  Deepgram 在完成音频后回调的 URL。
  可选。

- **callbackMethod** _enum_

  回调 HTTP 方法。
  允许：`'POST'`、`'PUT'`。
  可选。

- **mipOptOut** _boolean_

  是否退出 Deepgram 模型改进计划（Model Improvement Program）。
  可选。

- **tag** _string | array of strings_

  为用量报表标记请求的标签。
  可选。

## 转写模型

可使用 `.transcription()` 工厂方法创建调用 [Deepgram transcription API](https://developers.deepgram.com/docs/pre-recorded-audio) 的模型。

首个参数为模型 ID，例如 `nova-3`。

```ts
const model = deepgram.transcription('nova-3');
```

也可通过 `providerOptions` 传入额外选项；例如传入 `summarize` 可为对应段落启用摘要。

```ts highlight="6"
import { experimental_transcribe as transcribe } from 'ai';
import {
  deepgram,
  type DeepgramTranscriptionModelOptions,
} from '@ai-sdk/deepgram';
import { readFile } from 'fs/promises';

const result = await transcribe({
  model: deepgram.transcription('nova-3'),
  audio: await readFile('audio.mp3'),
  providerOptions: {
    deepgram: {
      summarize: true,
    } satisfies DeepgramTranscriptionModelOptions,
  },
});
```

可用的提供商选项：

- **language** _string_

  音频语言代码。
  支持多种 ISO 639-1 / ISO 639-3。
  可选。

- **detectLanguage** _boolean_

  是否自动检测语言。
  为 true 时 Deepgram 将识别音频语种。
  可选。

- **smartFormat** _boolean_

  是否对转写应用智能格式化。
  可选。

- **punctuate** _boolean_

  是否添加标点。
  可选。

- **summarize** _enum | boolean_

  是否生成摘要。
  允许：`v2`、`false`。
  可选。

- **topics** _boolean_

  是否检测主题。
  可选。

- **detectEntities** _boolean_

  是否检测实体。
  可选。

- **redact** _string | array of strings_

  需要从转写中遮盖的内容类型。
  可选。

- **search** _string_

  在转写中检索的关键词。
  可选。

- **diarize** _boolean_

  是否标注不同说话人。
  默认为 `true`。
  可选。

- **utterances** _boolean_

  是否将转写切段为语句（utterance）。
  可选。

- **uttSplit** _number_

  语句拆分的阈值。
  可选。

- **fillerWords** _boolean_

  是否保留填充词（um、uh 等）。
  可选。
