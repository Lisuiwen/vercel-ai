# xAI Grok 提供方

[xAI Grok](https://x.ai) 提供方为 [xAI API](https://x.ai/api) 提供语言模型支持。

## 安装

xAI Grok 提供方通过 `@ai-sdk/xai` 模块提供。可使用下列命令安装：

## 提供方实例

可从 `@ai-sdk/xai` 导入默认提供方实例 `xai`：

```ts
import { xai } from '@ai-sdk/xai';
```

若需要自定义配置，可从 `@ai-sdk/xai` 导入 `createXai`，并用你的设置创建提供方实例：

```ts
import { createXai } from '@ai-sdk/xai';

const xai = createXai({
  apiKey: 'your-api-key',
});
```

可通过下列可选设置自定义 xAI 提供方实例：

- **baseURL** _string_

  为 API 请求使用不同的 URL 前缀，例如经由代理服务器。
  默认前缀为 `https://api.x.ai/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API 密钥。默认读取环境变量 `XAI_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要附加的自定义标头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作中间件拦截请求，或为测试等场景提供自定义 fetch。

## 语言模型

可使用提供方实例创建 [xAI 模型](https://console.x.ai)。第一个参数为模型 ID，例如 `grok-4.20-non-reasoning`。

```ts
const model = xai('grok-4.20-non-reasoning');
```

默认情况下，`xai(modelId)` 使用 Responses API。要使用 [Chat Completions API](https://docs.x.ai/docs/api-reference#chat-completions)（旧版），请使用 `xai.chat(modelId)`。

### 示例

可使用 xAI 语言模型配合 `generateText` 生成文本：

```ts
import { xai } from '@ai-sdk/xai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: xai('grok-4.20-non-reasoning'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

xAI 语言模型也可用于 `streamText`，并支持与 [`Output`](/docs/reference/ai-sdk-core/output) 的结构化数据生成（见 [AI SDK Core](/docs/ai-sdk-core)）。

<h3 id="reasoning-effort">推理强度</h3>

对于具备推理能力的模型，可通过 `providerOptions.xai.reasoningEffort` 控制模型在作答前的思考深度。该选项对 Responses API（默认）与 Chat Completions API（`xai.chat()`）均适用。

```ts
import { xai } from '@ai-sdk/xai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: xai('grok-4.3'),
  prompt: 'Explain quantum entanglement.',
  providerOptions: {
    xai: { reasoningEffort: 'medium' },
  },
});
```

可选取值：

- `'none'` — 完全关闭推理；不使用思考型 token。适合需要近似即时响应的简单场景。`grok-4.3` 及更新的推理模型支持。
- `'low'`（默认）— 使用部分推理 token，仍较快。适合通用智能体与工具调用。
- `'medium'` — 更强推理，适合对延迟不那么敏感的应用，如复杂数据分析与长上下文推理。
- `'high'` — 更多推理 token，更深思考。适合高难度问题、复杂数学、多步逻辑与竞赛级任务。

<Note>
  并非每个 Grok 模型都接受所有取值。请参阅 xAI 的 [推理文档](https://docs.x.ai/docs/guides/reasoning) 了解所选模型支持的取值。`'none'` 需要 `grok-4.3` 或更新版本。
</Note>

## Responses API（智能体工具）

使用 `xai(modelId)` 时默认走 xAI Responses API。也可显式使用 `xai.responses(modelId)`，从而在 xAI 服务器侧自主编排工具调用与研究流程。

```ts
const model = xai.responses('grok-4.20-non-reasoning');
```

Responses API 提供可由模型在推理过程中自主执行的服务器端工具：

- **web_search**：实时网页搜索与浏览
- **x_search**：搜索 X（Twitter）帖子、用户与话题串
- **code_execution**：执行 Python 代码用于计算与数据分析
- **view_image**：查看并分析图像
- **view_x_video**：查看并分析 X 帖子中的视频
- **mcp_server**：连接远程 MCP 服务器并使用其工具
- **file_search**：在向量存储（集合）中检索文档

### 视觉（Vision）

Responses API 在使用视觉模型时支持图像输入：

```ts
import { xai } from '@ai-sdk/xai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: xai.responses('grok-3'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'What do you see in this image?' },
        {
          type: 'file',
          mediaType: 'image',
          data: fs.readFileSync('./image.png'),
        },
      ],
    },
  ],
});
```

### 网页搜索工具

网页搜索工具支持可选域名过滤与图像理解的自主网页研究：

```ts
import { xai } from '@ai-sdk/xai';
import { generateText } from 'ai';

const { text, sources } = await generateText({
  model: xai.responses('grok-4.20-non-reasoning'),
  prompt: 'What are the latest developments in AI?',
  tools: {
    web_search: xai.tools.webSearch({
      allowedDomains: ['arxiv.org', 'openai.com'],
      enableImageUnderstanding: true,
    }),
  },
});

console.log(text);
console.log('Citations:', sources);
```

#### 网页搜索参数

- **allowedDomains** _string[]_

  仅在指定域名内搜索（最多 5 个）。不可与 `excludedDomains` 同时使用。

- **excludedDomains** _string[]_

  从搜索中排除指定域名（最多 5 个）。不可与 `allowedDomains` 同时使用。

- **enableImageUnderstanding** _boolean_

  允许模型查看并分析搜索过程中发现的图像。会增加 token 用量。

### X 搜索工具

X 搜索工具可在 X（Twitter）上搜索帖子，并支持按账号与时间范围过滤：

```ts
const { text, sources } = await generateText({
  model: xai.responses('grok-4.20-non-reasoning'),
  prompt: 'What are people saying about AI on X this week?',
  tools: {
    x_search: xai.tools.xSearch({
      allowedXHandles: ['elonmusk', 'xai'],
      fromDate: '2025-10-23',
      toDate: '2025-10-30',
      enableImageUnderstanding: true,
      enableVideoUnderstanding: true,
    }),
  },
});
```

#### X 搜索参数

- **allowedXHandles** _string[]_

  仅从指定 X 账号发帖中搜索（最多 10 个）。不可与 `excludedXHandles` 同时使用。

- **excludedXHandles** _string[]_

  排除指定 X 账号的发帖（最多 10 个）。不可与 `allowedXHandles` 同时使用。

- **fromDate** _string_

  发帖起始日期，ISO8601 格式（`YYYY-MM-DD`）。

- **toDate** _string_

  发帖结束日期，ISO8601 格式（`YYYY-MM-DD`）。

- **enableImageUnderstanding** _boolean_

  允许模型查看并分析 X 帖子中的图像。

- **enableVideoUnderstanding** _boolean_

  允许模型查看并分析 X 帖子中的视频。

### 代码执行工具

const { text } = await generateText({

### MCP 服务器工具

MCP 服务器工具允许模型连接远程 [模型上下文协议（MCP）](https://modelcontextprotocol.io/) 服务器并使用其工具：

```ts
const { text } = await generateText({
  model: xai.responses('grok-4.20-non-reasoning'),
  prompt: 'Use the weather tool to check conditions in San Francisco',
  tools: {
    weather_server: xai.tools.mcpServer({
      serverUrl: 'https://example.com/mcp',
      serverLabel: 'weather-service',
      serverDescription: 'Weather data provider',
      allowedTools: ['get_weather', 'get_forecast'],
    }),
  },
});
```

#### MCP 服务器参数

- **serverUrl** _string_（必填）

  远程 MCP 服务器的 URL。

- **serverLabel** _string_

  用于标识 MCP 服务器的标签。

- **serverDescription** _string_

  MCP 服务器提供能力的说明。

- **allowedTools** _string[]_

  允许模型从该 MCP 服务器使用的工具名称列表。未指定则允许全部工具。

- **headers** _Record&lt;string, string&gt;_

  连接 MCP 服务器时要附加的自定义标头。

- **authorization** _string_

  连接 MCP 服务器时的 Authorization 标头值（例如 `'Bearer token123'`）。

### 文件搜索工具

文件搜索工具可在 xAI 向量存储（集合）中对文档进行检索：

```ts
import { xai, type XaiLanguageModelResponsesOptions } from '@ai-sdk/xai';
import { streamText } from 'ai';

const result = streamText({
  model: xai.responses('grok-4.20-reasoning'),
  prompt: 'What documents do you have access to?',
  tools: {
    file_search: xai.tools.fileSearch({
      vectorStoreIds: ['collection_your-collection-id'],
      maxNumResults: 10,
    }),
  },
  providerOptions: {
    xai: {
      include: ['file_search_call.results'],
    } satisfies XaiLanguageModelResponsesOptions,
  },
});
```

#### 文件搜索参数

- **vectorStoreIds** _string[]_（必填）

  要检索的向量存储（集合）ID。

- **maxNumResults** _number_

  搜索返回的最大结果条数。

#### 文件搜索的提供方选项

- **include** _Array&lt;'file_search_call.results'&gt;_

  在响应中包含文件搜索结果。设为 `['file_search_call.results']` 时，响应将包含含文件内容与分数的实际搜索结果。

<Note>
  文件搜索需要 grok-4 系列模型（含 grok-4.20）与 Responses API。向量存储可通过 [xAI API](https://docs.x.ai/docs/guides/using-collections/api) 创建。
</Note>

### 多工具组合

可组合多个服务器端工具以进行综合研究：

```ts
import { xai } from '@ai-sdk/xai';
import { streamText } from 'ai';

const { fullStream } = streamText({
  model: xai.responses('grok-4.20-non-reasoning'),
  prompt: 'Research AI safety developments and calculate risk metrics',
  tools: {
    web_search: xai.tools.webSearch(),
    x_search: xai.tools.xSearch(),
    code_execution: xai.tools.codeExecution(),
    file_search: xai.tools.fileSearch({
      vectorStoreIds: ['collection_your-documents'],
    }),
    data_service: xai.tools.mcpServer({
      serverUrl: 'https://data.example.com/mcp',
      serverLabel: 'data-service',
    }),
  },
});

for await (const part of fullStream) {
  if (part.type === 'text-delta') {
    process.stdout.write(part.text);
  } else if (part.type === 'source' && part.sourceType === 'url') {
    console.log('\nSource:', part.url);
  }
}
```

### 提供方选项

Responses API 支持下列提供方选项：

```ts
import { xai, type XaiLanguageModelResponsesOptions } from '@ai-sdk/xai';
import { generateText } from 'ai';

const result = await generateText({
  model: xai.responses('grok-4.20-non-reasoning'),
  providerOptions: {
    xai: {
      reasoningEffort: 'high',
    } satisfies XaiLanguageModelResponsesOptions,
  },
  // ...
});
```

可用提供方选项如下：

- **reasoningEffort** _'none' | 'low' | 'medium' | 'high'_

  控制模型的推理强度。各取值说明见 [推理强度](#reasoning-effort)。`'none'` 完全关闭推理（需要 `grok-4.3` 或更新）。

- **logprobs** _boolean_

  是否返回输出 token 的对数概率。

- **topLogprobs** _number_

  每个 token 位置返回概率最高的若干候选 token 数量（0–8）。设置此项时会自动启用 `logprobs`。

- **include** _Array&lt;'file_search_call.results'&gt;_

  指定要在模型响应中包含的额外输出数据。使用 `['file_search_call.results']` 可包含含分数与内容的文件搜索结果。

- **store** _boolean_

  是否存储输入消息与模型响应以供后续检索。默认为 `true`。

- **previousResponseId** _string_

  模型上一份响应的 ID，可用于延续对话。

<Note>
  Responses API 仅支持服务器端工具。同一请求中无法混用服务器端工具与客户端函数工具。
</Note>

## 图像模型

可使用 `.image()` 工厂方法创建 xAI 图像模型。关于使用 AI SDK 生成图像，参见 [generateImage()](/docs/reference/ai-sdk-core/generate-image)。

```ts
import { xai } from '@ai-sdk/xai';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: xai.image('grok-imagine-image'),
  prompt: 'A futuristic cityscape at sunset',
});
```

<Note>
  xAI 图像模型不支持 `size` 参数，请改用 `aspectRatio`。支持的宽高比：`1:1`、`16:9`、`9:16`、`4:3`、`3:4`、`3:2`、`2:3`、`2:1`、`1:2`、`19.5:9`、`9:19.5`、`20:9`、`9:20` 与 `auto`。
</Note>

### 图像编辑

xAI 通过 `grok-imagine-image` 模型支持图像编辑。通过 `prompt.images` 传入输入图像，即可变换或编辑现有图像。

<Note>
  xAI 图像编辑不支持遮罩。编辑由提示词驱动——请在文本提示中描述希望如何修改。
</Note>

#### 基础图像编辑

使用文本提示变换现有图像：

```ts
import { xai } from '@ai-sdk/xai';
import { generateImage } from 'ai';
import { readFileSync } from 'fs';

const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: xai.image('grok-imagine-image'),
  prompt: {
    text: 'Turn the cat into a golden retriever dog',
    images: [imageBuffer],
  },
});
```

#### 多图编辑

在提示中合并或引用多张输入图像：

```ts
import { xai } from '@ai-sdk/xai';
import { generateImage } from 'ai';
import { readFileSync } from 'fs';

const cat = readFileSync('./cat.png');
const dog = readFileSync('./dog.png');

const { images } = await generateImage({
  model: xai.image('grok-imagine-image'),
  prompt: {
    text: 'Combine these two animals into a group photo',
    images: [cat, dog],
  },
});
```

#### 风格迁移

对图像应用艺术风格：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: xai.image('grok-imagine-image'),
  prompt: {
    text: 'Transform this into a watercolor painting style',
    images: [imageBuffer],
  },
  aspectRatio: '1:1',
});
```

<Note>
  输入图像可为 `Buffer`、`ArrayBuffer`、`Uint8Array` 或 base64 编码字符串。
</Note>

### 图像提供方选项

可通过 `providerOptions.xai` 使用提供方专属设置自定义图像生成行为：

```ts
import { xai, type XaiImageModelOptions } from '@ai-sdk/xai';
import { generateImage } from 'ai';

const { images } = await generateImage({
  model: xai.image('grok-imagine-image-pro'),
  prompt: 'A futuristic cityscape at sunset',
  aspectRatio: '16:9',
  providerOptions: {
    xai: {
      resolution: '2k',
      quality: 'high',
    } satisfies XaiImageModelOptions,
  },
});
```

- **resolution** _'1k' | '2k'_

  输出分辨率。`1k` 约生成 1024×1024 图像，`2k` 约生成 2048×2048（实际尺寸随宽高比变化）。适用于 `grok-imagine-image-pro`。

- **quality** _'low' | 'medium' | 'high'_

  图像质量等级。更高质量可能增加生成耗时。

## 视频模型

可使用 `.video()` 工厂方法创建 xAI 视频模型。
关于使用 AI SDK 生成视频，参见 [generateVideo()](/docs/reference/ai-sdk-core/generate-video)。

本提供方支持由文本提示或图像输入进行标准视频生成，以及显式的视频编辑、视频延展与参考图转视频（R2V）操作。

### 文生视频

由文本提示生成视频：

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { video } = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: 'A chicken flying into the sunset in the style of 90s anime.',
  aspectRatio: '16:9',
  duration: 5,
  providerOptions: {
    xai: {
      pollTimeoutMs: 600000, // 10 minutes
    } satisfies XaiVideoModelOptions,
  },
});
```

### 带图像输入的生成

以图像作为起始帧生成视频，并可附带文本提示。走的是标准生成路径，而非单独的提供方模式：

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { video } = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: {
    image: 'https://example.com/start-frame.png',
    text: 'The cat slowly turns its head and blinks',
  },
  duration: 5,
  providerOptions: {
    xai: {
      pollTimeoutMs: 600000, // 10 minutes
    } satisfies XaiVideoModelOptions,
  },
});
```

### 视频编辑

通过提供方选项传入源视频 URL，并用文本提示编辑现有视频：

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { video } = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: 'Give the person sunglasses and a hat',
  providerOptions: {
    xai: {
      mode: 'edit-video',
      videoUrl: 'https://example.com/source-video.mp4',
      pollTimeoutMs: 600000, // 10 minutes
    } satisfies XaiVideoModelOptions,
  },
});
```

<Note>
  视频编辑接受的输入视频最长约 8.7 秒。编辑不支持自定义 `duration`、`aspectRatio` 与 `resolution`——输出与输入视频属性一致（分辨率上限 720p）。
</Note>

### 链式与并发编辑

xAI 托管的视频 URL 位于 `providerMetadata.xai.videoUrl`。
可用它串联多次编辑，或通过 `Promise.all` 分支进行并发编辑：

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

const providerOptions = {
  xai: {
    mode: 'edit-video',
    videoUrl: 'https://example.com/source-video.mp4',
    pollTimeoutMs: 600000,
  } satisfies XaiVideoModelOptions,
};

// Step 1: Apply an initial edit
const step1 = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: 'Add a party hat to the person',
  providerOptions,
});

// Get the xAI-hosted URL from provider metadata
const step1VideoUrl = step1.providerMetadata?.xai?.videoUrl as string;

// Step 2: Apply two more edits concurrently, building on step 1
const [withSunglasses, withScarf] = await Promise.all([
  generateVideo({
    model: xai.video('grok-imagine-video'),
    prompt: 'Add sunglasses',
    providerOptions: {
      xai: {
        mode: 'edit-video',
        videoUrl: step1VideoUrl,
        pollTimeoutMs: 600000,
      },
    },
  }),
  generateVideo({
    model: xai.video('grok-imagine-video'),
    prompt: 'Add a scarf',
    providerOptions: {
      xai: {
        mode: 'edit-video',
        videoUrl: step1VideoUrl,
        pollTimeoutMs: 600000,
      },
    },
  }),
]);
```

### 视频延展

从现有视频的末帧继续延展。`duration` 仅控制延展片段长度，而非总输出时长。输出的 `aspectRatio` 与 `resolution` 继承自源视频。

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

// Step 1: Generate a source video
const source = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: 'A cat sitting on a sunlit windowsill, tail gently swishing.',
  duration: 5,
  aspectRatio: '16:9',
  providerOptions: {
    xai: {
      pollTimeoutMs: 600000,
    } satisfies XaiVideoModelOptions,
  },
});

const sourceUrl = source.providerMetadata?.xai?.videoUrl as string;

// Step 2: Extend the video with a new scene
const extended = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt: 'The cat turns its head, notices a butterfly, and leaps off.',
  duration: 6,
  providerOptions: {
    xai: {
      mode: 'extend-video',
      videoUrl: sourceUrl,
      pollTimeoutMs: 600000,
    } satisfies XaiVideoModelOptions,
  },
});
```

<Note>
  视频延展不支持自定义 `aspectRatio` 或 `resolution`——二者继承自源视频。支持 `duration`，且仅表示延展长度（不是整段视频总长）。
</Note>

### 参考图转视频（R2V）

提供参考图像以引导视频的风格与内容。与图生视频不同，参考图不作为首帧——模型将其视觉元素融入生成视频。每张参考图可为公开 HTTPS URL 或 base64 data URI。

```ts
import { xai, type XaiVideoModelOptions } from '@ai-sdk/xai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { video } = await generateVideo({
  model: xai.video('grok-imagine-video'),
  prompt:
    'The comic cat from <IMAGE_1> and the comic dog from <IMAGE_2> ' +
    'are having a playful chase through a sunlit park. ' +
    'Cinematic slow-motion, warm afternoon light.',
  duration: 8,
  aspectRatio: '16:9',
  providerOptions: {
    xai: {
      mode: 'reference-to-video',
      referenceImageUrls: [
        'https://example.com/comic-cat.png',
        'https://example.com/comic-dog.png',
      ],
      pollTimeoutMs: 600000,
    } satisfies XaiVideoModelOptions,
  },
});
```

在提示中使用 `<IMAGE_1>`、`<IMAGE_2>` 等引用对应图像。单次请求最多支持 7 张参考图。

<Note>
  参考图转视频支持 `duration`、`aspectRatio` 与 `resolution`。通过 `mode` 选择操作——各模式互斥。
</Note>

### 视频提供方选项

下列选项通过 `providerOptions.xai` 传入。
可使用 `XaiVideoModelOptions` 类型校验提供方选项。

- **pollIntervalMs** _number_

  轮询任务状态的间隔（毫秒）。默认 5000。

- **pollTimeoutMs** _number_

  视频生成的最长等待时间（毫秒）。默认 600000（10 分钟）。

- **resolution** _'480p' | '720p'_

  视频分辨率。使用 SDK 标准 `resolution` 参数时，`1280x720` 映射为 `720p`，`854x480` 映射为 `480p`。
  也可通过本提供方选项直接传入原生格式。

- **mode** _'edit-video' | 'extend-video' | 'reference-to-video'_

  选择显式视频操作。各模式互斥：
  - `'edit-video'` — 编辑现有视频（需要 `videoUrl`）
  - `'extend-video'` — 从末帧延展视频（需要 `videoUrl`）
  - `'reference-to-video'` — 基于参考图生成（需要 `referenceImageUrls`）

  省略时使用标准生成。为兼容旧用法，仍会根据字段自动识别旧版输入。

- **videoUrl** _string_

  源视频 URL。与 `mode: 'edit-video'` 配合用于编辑，与 `mode: 'extend-video'` 配合用于延展。

- **referenceImageUrls** _string[]_

  参考图转视频（R2V）用的参考图 URL 数组（1–7 张）或 base64 data URI。模型融合这些图像的视觉元素，但不将其作为首帧。在提示中用 `<IMAGE_1>`、`<IMAGE_2>` 等引用具体图像。与 `mode: 'reference-to-video'` 配合使用。

<Note>
  视频生成是异步过程，可能耗时数分钟。
  建议将 `pollTimeoutMs` 至少设为 10 分钟（600000ms）以保证稳定。
  生成的视频 URL 为临时地址，请及时下载保存。
</Note>

### 宽高比与分辨率

**文生视频**可同时指定 `aspectRatio` 与 `resolution`。
默认宽高比为 `16:9`，默认分辨率为 `480p`。

**图生视频**的输出默认与输入图像宽高比一致。
若另行指定 `aspectRatio`，将覆盖默认并将图像拉伸至目标比例。

**视频编辑**的输出与输入视频的宽高比和分辨率一致。
不支持自定义 `duration`、`aspectRatio` 与 `resolution`——输出分辨率上限为 720p（例如 1080p 输入会下采样至 720p）。

**视频延展**的输出继承源视频的 `aspectRatio` 与 `resolution`。
支持 `duration`，且仅控制延展片段长度。

**参考图转视频（R2V）**可像文生视频一样指定 `duration`、`aspectRatio` 与 `resolution`。
