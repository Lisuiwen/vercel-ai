# Open Responses 提供商

[Open Responses](https://www.openresponses.org/) 提供商为兼容 Open Responses 的 API 提供语言模型支持。

## 安装

Open Responses 提供商位于 `@ai-sdk/open-responses` 模块中，可使用以下命令安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/open-responses" dark />
  </Tab>
  <Tab>

  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/open-responses" dark />
  </Tab>
  <Tab>
    <Snippet text="bun add @ai-sdk/open-responses" dark />
  </Tab>
</Tabs>

## Provider 实例

使用 `createOpenResponses` 创建 Open Responses 提供商实例：

```ts
import { createOpenResponses } from '@ai-sdk/open-responses';

const openResponses = createOpenResponses({
  name: 'aProvider',
  url: 'http://localhost:1234/v1/responses',
});
```

`name` 与 `url` 为必填项：

- **name** _string_

  提供商名称。用作 provider options 与元数据的键。

- **url** _string_

  Open Responses API 的 POST 端点 URL。

可使用以下可选设置自定义 Open Responses 提供商实例：

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。

## 语言模型

Open Responses 提供商实例可作为函数调用以创建语言模型：

```ts
const model = openResponses('mistralai/ministral-3-14b-reasoning');
```

可与 `generateText`、`streamText` 搭配使用，
并支持通过 [`Output`](/docs/reference/ai-sdk-core/output) 生成结构化数据
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

### 示例

```ts
import { createOpenResponses } from '@ai-sdk/open-responses';
import { generateText } from 'ai';

const openResponses = createOpenResponses({
  name: 'aProvider',
  url: 'http://localhost:1234/v1/responses',
});

const { text } = await generateText({
  model: openResponses('mistralai/ministral-3-14b-reasoning'),
  prompt: 'Invent a new holiday and describe its traditions.',
});
```

## 说明

- 停止序列、`topK` 与 `seed` 不被支持，传入时会被忽略并发出警告。
- 支持在用户消息中使用 `file` 部分且 media type 为图像时传入图像输入。
