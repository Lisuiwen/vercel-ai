# Anthropic 提供方

[Anthropic](https://www.anthropic.com/) 提供方为 [Anthropic Messages API](https://docs.anthropic.com/claude/reference/messages_post) 提供语言模型支持。

## 安装

Anthropic 提供方位于 `@ai-sdk/anthropic` 模块中。可通过以下方式安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/anthropic" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @ai-sdk/anthropic" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/anthropic" dark />
  </Tab>

  <Tab>
    <Snippet text="bun add @ai-sdk/anthropic" dark />
  </Tab>
</Tabs>

## 提供方实例

你可以从 `@ai-sdk/anthropic` 导入默认提供方实例 `anthropic`：

```ts
import { anthropic } from '@ai-sdk/anthropic';
```

若需要自定义配置，可从 `@ai-sdk/anthropic` 导入 `createAnthropic`，并使用你的设置创建提供方实例：

```ts
import { createAnthropic } from '@ai-sdk/anthropic';

const anthropic = createAnthropic({
  // custom settings
});
```

可使用以下可选设置自定义 Anthropic 提供方实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀，例如经由代理服务器。
  默认前缀为 `https://api.anthropic.com/v1`。

- **apiKey** _string_

  通过 `x-api-key` 请求头发送的 API 密钥。
  默认读取 `ANTHROPIC_API_KEY` 环境变量。
  `apiKey` 与 `authToken` 只需配置其一。

- **authToken** _string_

  通过 `Authorization: Bearer` 请求头发送的身份令牌。
  默认读取 `ANTHROPIC_AUTH_TOKEN` 环境变量。
  `apiKey` 与 `authToken` 只需配置其一。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。
  可用作中间件拦截请求，
  或在例如测试中提供自定义 fetch 实现。

## 语言模型

你可以使用提供方实例创建调用 [Anthropic Messages API](https://docs.anthropic.com/claude/reference/messages_post) 的模型。
第一个参数是模型 id，例如 `claude-3-haiku-20240307`。
部分模型支持多模态。

```ts
const model = anthropic('claude-3-haiku-20240307');
```

创建模型时还可使用下列别名：

- `anthropic.languageModel('claude-3-haiku-20240307')` - 创建语言模型
- `anthropic.chat('claude-3-haiku-20240307')` - `languageModel` 的别名
- `anthropic.messages('claude-3-haiku-20240307')` - `languageModel` 的别名

可使用 Anthropic 语言模型配合 `generateText` 函数生成文本：

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: anthropic('claude-3-haiku-20240307'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Anthropic 语言模型也可用于 `streamText` 函数，
并支持使用 [`Output`](/docs/reference/ai-sdk-core/output) 进行结构化数据生成
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

Anthropic 模型可使用下列可选提供方选项：

- `disableParallelToolUse` _boolean_

  可选。禁用并行工具调用。默认为 `false`。

  设为 `true` 时，模型每次只会调用一个工具，而不会并行调用多个。

- `sendReasoning` _boolean_

  可选。在发往模型的请求中包含推理内容。默认为 `true`。

  若模型在处理涉及推理内容的请求时出现问题，
  可将其设为 `false` 以从请求中省略这些内容。

- `effort` _"low" | "medium" | "high" | "xhigh" | "max"_

  可选。更多说明见 [努力度](#努力度)。

- `taskBudget` _object_

  可选。更多说明见 [任务预算小节](#task-budgets)。

- `speed` _"fast" | "standard"_

  可选。更多说明见 [快速模式小节](#fast-mode)。

- `inferenceGeo` _"us" | "global"_

  可选。更多说明见 [数据驻留小节](#data-residency)。

- `thinking` _object_

  可选。更多说明见 [推理小节](#reasoning)。

- `toolStreaming` _boolean_

  是否启用工具流式传输（以及结构化输出流式传输）。默认为 `true`。

- `structuredOutputMode` _"outputFormat" | "jsonTool" | "auto"_

  决定如何生成结构化输出。可选。
  - `"outputFormat"`：使用 `output_format` 参数指定结构化输出格式。
  - `"jsonTool"`：使用特殊的 `"json"` 工具指定结构化输出格式。
  - `"auto"`：在支持时使用 `"outputFormat"`，否则回退到 `"jsonTool"`（默认）。

- `metadata` _object_

  可选。随请求发送的元数据。详情请见 [Anthropic API 文档](https://platform.claude.com/docs/en/api/messages/create)。
  - `userId` _string_ - 终端用户的外部标识符。应为 UUID、哈希或其他不透明标识符。不得包含个人身份信息。

### 结构化输出与工具输入流式传输

默认启用工具调用流式传输。可将 `toolStreaming` 提供方选项设为 `false` 以关闭。

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { streamText, tool } from 'ai';
import { z } from 'zod';

const result = streamText({
  model: anthropic('claude-sonnet-4-20250514'),
  tools: {
    writeFile: tool({
      description: 'Write content to a file',
      inputSchema: z.object({
        path: z.string(),
        content: z.string(),
      }),
      execute: async ({ path, content }) => {
        // Implementation
        return { success: true };
      },
    }),
  },
  prompt: 'Write a short story to story.txt',
});
```

### 努力度

Anthropic 在 `claude-opus-4-5` 中引入了 `effort` 选项，影响思考、文本回复与函数调用。`effort` 默认为 `high`，可设为 `medium` 或 `low` 以节省 token 并降低到最后一个 token 的时间（TTLT）。`claude-opus-4-7` 额外支持 `xhigh` 以获得最大推理强度。

```ts highlight="8-10"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text, usage } = await generateText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      effort: 'low',
    } satisfies AnthropicLanguageModelOptions,
  },
});

console.log(text); // resulting text
console.log(usage); // token usage
```

### 快速模式

Anthropic 为 `claude-opus-4-6` 提供 [`speed` 选项](https://code.claude.com/docs/en/fast-mode)，可启用更快速度，输出 token 速度约快 2.5 倍。

```ts highlight="8-10"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: anthropic('claude-opus-4-6'),
  prompt: 'Write a short poem about the sea.',
  providerOptions: {
    anthropic: {
      speed: 'fast',
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

`speed` 选项接受 `'fast'` 或 `'standard'`（默认行为）。

### 任务预算

`claude-opus-4-7` 支持 `taskBudget` 选项，用于告知模型在智能体一轮中可用 token 总预算。模型会据此优先安排工作、提前规划，并在预算消耗时平稳收尾。

任务预算是建议性的——不会强制硬性的 token 上限。模型会尽量在预算内工作，但实际用量可能有所不同。

```ts highlight="8-13"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: anthropic('claude-opus-4-7'),
  prompt: 'Research the pros and cons of Rust vs Go for building CLI tools.',
  providerOptions: {
    anthropic: {
      taskBudget: {
        type: 'tokens',
        total: 400000,
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

对于会压缩并重启上下文的长时间运行智能体，可使用 `remaining` 字段将剩余预算延续：

```ts
taskBudget: {
  type: 'tokens',
  total: 400000,
  remaining: 215000, // budget left after prior compacted-away contexts
}
```

`taskBudget` 对象接受：

- `type` _"tokens"_ - 预算类型。目前仅支持 `"tokens"`。
- `total` _number_ - 该智能体轮次的任务总预算。最小 20,000。
- `remaining` _number_ - 在先前被压缩掉的上下文之后剩余的预算。须在 0 与 `total` 之间。省略时默认为 `total`。

### 数据驻留

Anthropic 支持 [`inferenceGeo` 选项](https://platform.claude.com/docs/en/build-with-claude/data-residency)，用于控制单次请求的模型推理运行位置。

```ts highlight="8-10"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: anthropic('claude-opus-4-6'),
  prompt: 'Summarize the key points of this document.',
  providerOptions: {
    anthropic: {
      inferenceGeo: 'us',
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

`inferenceGeo` 选项接受 `'us'`（仅限美国基础设施）或 `'global'`（默认，任意可用地域）。

### 推理

Anthropic 模型支持扩展思考，Claude 会在给出最终答案前展示其推理过程。

#### 自适应思考

对于较新模型（`claude-sonnet-4-6`、`claude-opus-4-6` 及更高版本），请使用自适应思考。
Claude 会根据提示的复杂度自动决定推理量。

```ts highlight="4,8-10"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text, reasoningText, reasoning } = await generateText({
  model: anthropic('claude-opus-4-6'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      thinking: { type: 'adaptive' },
    } satisfies AnthropicLanguageModelOptions,
  },
});

console.log(reasoningText); // reasoning text
console.log(reasoning); // reasoning details including redacted reasoning
console.log(text); // text response
```

可将自适应思考与 `effort` 选项结合，以控制 Claude 的推理强度：

```ts highlight="6-8"
const { text } = await generateText({
  model: anthropic('claude-opus-4-6'),
  prompt: 'Invent a new holiday and describe its traditions.',
  providerOptions: {
    anthropic: {
      thinking: { type: 'adaptive' },
      effort: 'max', // 'low' | 'medium' | 'high' | 'max'
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

##### 思考展示（Opus 4.7+）

自 `claude-opus-4-7` 起，思考内容默认从响应中省略——流中仍有思考块但文本为空。若要接收推理输出，请设置 `display: 'summarized'`：

```ts highlight="5"
const { text, reasoningText } = await generateText({
  model: anthropic('claude-opus-4-7'),
  providerOptions: {
    anthropic: {
      thinking: { type: 'adaptive', display: 'summarized' },
    } satisfies AnthropicLanguageModelOptions,
  },
  prompt: 'How many people will live in the world in 2040?',
});

console.log(reasoningText); // reasoning text (empty without display: 'summarized')
console.log(text);
```

<Note>
  若你在向用户流式传输 `claude-opus-4-7` 的推理，默认的
  `"omitted"` 展示会在开始输出前造成长时间停顿。请设置
  `display: "summarized"` 以在思考阶段恢复可见进度。
</Note>

#### 基于预算的思考

对于较早模型（`claude-opus-4-20250514`、`claude-sonnet-4-20250514`、`claude-sonnet-4-5-20250929`），
请使用 `type: 'enabled'` 并指定明确的 token 预算：

```ts highlight="4,8-10"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const { text, reasoningText, reasoning } = await generateText({
  model: anthropic('claude-sonnet-4-5-20250929'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    } satisfies AnthropicLanguageModelOptions,
  },
});

console.log(reasoningText); // reasoning text
console.log(reasoning); // reasoning details including redacted reasoning
console.log(text); // text response
```

关于如何将推理接入聊天机器人，详见 [AI SDK UI：Chatbot](/docs/ai-sdk-ui/chatbot#reasoning)。

### 上下文管理

Anthropic 的上下文管理功能可在满足条件时自动清除工具使用或思考内容，从而管理对话上下文。这有助于优化 token 用量并更高效地处理长对话。

可通过 `contextManagement` 提供方选项配置上下文管理：

```ts highlight="7-20"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5-20250929'),
  prompt: 'Continue our conversation...',
  providerOptions: {
    anthropic: {
      contextManagement: {
        edits: [
          {
            type: 'clear_tool_uses_20250919',
            trigger: { type: 'input_tokens', value: 10000 },
            keep: { type: 'tool_uses', value: 5 },
            clearAtLeast: { type: 'input_tokens', value: 1000 },
            clearToolInputs: true,
            excludeTools: ['important_tool'],
          },
        ],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});

// Check what was cleared
console.log(result.providerMetadata?.anthropic?.contextManagement);
```

#### 上下文编辑

上下文编辑策略会选择性移除对话中较早位置的特定内容类型，以降低 token 用量，同时尽量保持整体对话流程。

##### 清除工具使用

`clear_tool_uses_20250919` 编辑类型会从对话历史中移除较早的工具调用/结果对：

- **trigger** - 触发清除的条件（例如 `{ type: 'input_tokens', value: 10000 }` 或 `{ type: 'tool_uses', value: 10 }`）
- **keep** - 保留多少最近的工具使用（例如 `{ type: 'tool_uses', value: 5 }`）
- **clearAtLeast** - 至少要清除的量（例如 `{ type: 'input_tokens', value: 1000 }`）
- **clearToolInputs** - 是否清除工具输入参数（布尔值）
- **excludeTools** - 永不清除的工具名称数组

##### 清除思考

`clear_thinking_20251015` 编辑类型会移除较早轮次中的思考/推理块，仅保留最近若干：

- **keep** - 保留多少最近的思考轮次（例如 `{ type: 'thinking_turns', value: 2 }`），或 `'all'` 表示全部保留

```ts
const result = await generateText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'Continue reasoning...',
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
      contextManagement: {
        edits: [
          {
            type: 'clear_thinking_20251015',
            keep: { type: 'thinking_turns', value: 2 },
          },
        ],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

#### 压缩

`compact_20260112` 编辑类型会在达到 token 限制时自动摘要较早的对话上下文。适用于希望在 token 限制内保留早期交流要点的长对话。

```ts highlight="7-19"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { streamText } from 'ai';

const result = streamText({
  model: anthropic('claude-opus-4-6'),
  messages: conversationHistory,
  providerOptions: {
    anthropic: {
      contextManagement: {
        edits: [
          {
            type: 'compact_20260112',
            trigger: {
              type: 'input_tokens',
              value: 50000, // trigger compaction when input exceeds 50k tokens
            },
            instructions:
              'Summarize the conversation concisely, preserving key decisions and context.',
            pauseAfterCompaction: false,
          },
        ],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

**配置说明：**

- **trigger** - 触发压缩的条件（例如 `{ type: 'input_tokens', value: 50000 }`）
- **instructions** - 指导模型如何摘要对话的自定义说明。可用于让压缩摘要侧重你希望保留的对话方面。
- **pauseAfterCompaction** - 为 `true` 时，模型在生成压缩摘要后会暂停，便于你在继续前检查或处理。默认为 `false`。

发生压缩时，模型会生成较早上下文的摘要。该摘要以带有特殊提供方元数据的文本块形式出现。

##### 在流中检测压缩

使用 `streamText` 时，可通过检查 `text-start` 事件上的 `providerMetadata` 来检测压缩摘要：

```ts
for await (const part of result.fullStream) {
  switch (part.type) {
    case 'text-start': {
      const isCompaction =
        part.providerMetadata?.anthropic?.type === 'compaction';
      if (isCompaction) {
        console.log('[COMPACTION SUMMARY START]');
      }
      break;
    }
    case 'text-delta': {
      process.stdout.write(part.text);
      break;
    }
  }
}
```

##### 在 UI 应用中的压缩

使用 `useChat` 或其他 UI 钩子时，压缩摘要会以带 `providerMetadata` 的普通文本片段出现。你可以在 UI 中为其设置不同样式：

```tsx
{
  message.parts.map((part, index) => {
    if (part.type === 'text') {
      const isCompaction =
        (part.providerMetadata?.anthropic as { type?: string } | undefined)
          ?.type === 'compaction';

      if (isCompaction) {
        return (
          <div
            key={index}
            className="bg-yellow-100 border-l-4 border-yellow-500 p-2"
          >
            <span className="font-bold">[Compaction Summary]</span>
            <div>{part.text}</div>
          </div>
        );
      }
      return <div key={index}>{part.text}</div>;
    }
  });
}
```

#### 已应用编辑的元数据

生成完成后，可在提供方元数据中查看应用了哪些编辑：

```ts
const metadata = result.providerMetadata?.anthropic?.contextManagement;

if (metadata?.appliedEdits) {
  metadata.appliedEdits.forEach(edit => {
    if (edit.type === 'clear_tool_uses_20250919') {
      console.log(`Cleared ${edit.clearedToolUses} tool uses`);
      console.log(`Freed ${edit.clearedInputTokens} tokens`);
    } else if (edit.type === 'clear_thinking_20251015') {
      console.log(`Cleared ${edit.clearedThinkingTurns} thinking turns`);
      console.log(`Freed ${edit.clearedInputTokens} tokens`);
    } else if (edit.type === 'compact_20260112') {
      console.log('Compaction was applied');
    }
  });
}
```

更多说明见 [Anthropic 上下文管理文档](https://docs.anthropic.com/en/docs/build-with-claude/context-management)。

### 缓存控制

在消息与消息片段中，可使用 `providerOptions` 设置缓存断点。
需在 `providerOptions` 中将 `anthropic` 设为 `{ cacheControl: { type: 'ephemeral' } }` 以设置缓存断点。

缓存读取与缓存写入（创建）的 token 计数会在标准 `usage` 对象上返回，适用于 `generateText` 与 `streamText`。可通过
`result.usage.inputTokenDetails.cacheReadTokens` 与
`result.usage.inputTokenDetails.cacheWriteTokens` 访问。

```ts highlight="8,18-20,29-32"
import { anthropic } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const errorMessage = '... long error message ...';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'You are a JavaScript expert.' },
        {
          type: 'text',
          text: `Error message: ${errorMessage}`,
          providerOptions: {
            anthropic: { cacheControl: { type: 'ephemeral' } },
          },
        },
        { type: 'text', text: 'Explain the error message.' },
      ],
    },
  ],
});

console.log(result.text);
console.log(
  'Cache read tokens:',
  result.usage.inputTokenDetails.cacheReadTokens,
);
console.log(
  'Cache write tokens:',
  result.usage.inputTokenDetails.cacheWriteTokens,
);
```

也可在系统消息上使用缓存控制，在 `messages` 数组开头提供多条系统消息：

```ts highlight="3,7-9"
const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  messages: [
    {
      role: 'system',
      content: 'Cached system message part',
      providerOptions: {
        anthropic: { cacheControl: { type: 'ephemeral' } },
      },
    },
    {
      role: 'system',
      content: 'Uncached system message part',
    },
    {
      role: 'user',
      content: 'User prompt',
    },
  ],
});
```

工具的缓存控制：

```ts
const result = await generateText({
  model: anthropic('claude-haiku-4-5'),
  tools: {
    cityAttractions: tool({
      inputSchema: z.object({ city: z.string() }),
      providerOptions: {
        anthropic: {
          cacheControl: { type: 'ephemeral' },
        },
      },
    }),
  },
  messages: [
    {
      role: 'user',
      content: 'User prompt',
    },
  ],
});
```

#### 更长缓存 TTL

Anthropic 还支持 1 小时更长的缓存时长。

示例如下：

```ts
const result = await generateText({
  model: anthropic('claude-haiku-4-5'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'Long cached message',
          providerOptions: {
            anthropic: {
              cacheControl: { type: 'ephemeral', ttl: '1h' },
            },
          },
        },
      ],
    },
  ],
});
```

#### 限制

可缓存提示的最小长度为：

- Claude Opus 4.5：4096 tokens
- Claude Opus 4.1、Claude Opus 4、Claude Sonnet 4.5、Claude Sonnet 4、Claude Sonnet 3.7 与 Claude Opus 3：1024 tokens
- Claude Haiku 4.5：4096 tokens
- Claude Haiku 3.5 与 Claude Haiku 3：2048 tokens

更短的提示即使用 `cacheControl` 标记也无法缓存。若请求的缓存 token 数少于上述阈值，将按无缓存处理。

关于 Anthropic 的提示缓存，详见 [Anthropic 缓存控制文档](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching)。

  `providerOptions`，可先用 `convertToModelMessages`
  再传给 `generateText` 或
  `streamText` 等函数。`providerOptions` 用法的更多说明见
  [此处](/docs/foundations/prompts#provider-options)。
</Note>

import { anthropic } from '@ai-sdk/anthropic';

### Advisor 工具

Anthropic 提供由提供方执行的 advisor 工具，让执行模型在生成过程中可咨询更强的顾问模型。适用于长时间编码、研究与智能体任务：由更强模型提供规划或纠偏，主回复仍由更快或更低成本的模型生成。

可使用 `anthropic.tools.advisor_20260301` 启用 advisor 工具：

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: anthropic('claude-sonnet-4-6'),
  system: 'You have access to an `advisor` tool backed by a stronger reviewer model.'
  prompt:
    'Build a concurrent worker pool in Go with graceful shutdown. Outline the design first.',
  tools: {
    advisor: anthropic.tools.advisor_20260301({
      model: 'claude-opus-4-7',
      maxUses: 3,
    }),
  },
});
```

<Note>
  包含此工具时，AI SDK 会自动发送所需的 `advisor-tool-2026-03-01` beta
  请求头。
</Note>

#### 配置选项

advisor 工具支持下列配置项：

- **model** _string_

  必填。顾问模型 ID，例如 `claude-opus-4-7`。顾问模型能力不得低于执行模型。

- **maxUses** _number_

  可选。单次请求中允许的 advisor 调用次数上限。执行模型达到上限后，进一步调用将返回 `advisor_tool_result_error` 且 `errorCode: 'max_uses_exceeded'`，执行模型将在无更多建议的情况下继续。

- **caching** _object_

  可选。在同一次对话的多次调用之间，为顾问自身 transcript 启用提示缓存。使用 `{ type: 'ephemeral', ttl: '5m' | '1h' }`。在预期至少有三次 advisor 调用的较长智能体循环中最有用。

#### 工作原理

advisor 工具由提供方执行，因此你无需提供 `execute` 函数。执行模型调用 `advisor` 时，Anthropic 会使用顾问模型单独在服务端进行一次推理。顾问看到 transcript 并向执行模型返回指导，后者在同一请求内继续生成。

advisor 工具的输入始终为空。Anthropic 会自动从完整 transcript 构建顾问视图。

多轮对话时，请在后续轮次传回完整的助手内容（含 advisor 工具结果）。若在某次后续请求中移除 advisor 工具，也须从消息历史中移除先前的 advisor 工具结果；否则 Anthropic API 会返回无效请求错误。

#### 流式传输

顾问子推理不会流式输出。使用 `streamText` 时，执行模型的流会在 advisor 运行期间暂停，随后顾问结果以单个工具结果到达，之后执行模型输出继续。

### 工具搜索

Anthropic 提供由提供方定义的工具搜索能力，使 Claude 可动态发现并按需加载成百上千个工具，而无需在上下文窗口中预先装入全部工具定义。Claude 会在你的工具目录中搜索并只加载所需工具。

有两种变体：

- **BM25 搜索** - 使用自然语言查询查找工具
- **正则搜索** - 使用正则模式（Python `re.search()` 语法）查找工具

#### 基本用法

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText, tool } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  prompt: 'What is the weather in San Francisco?',
  tools: {
    toolSearch: anthropic.tools.toolSearchBm25_20251119(),

    get_weather: tool({
      description: 'Get the current weather at a specific location',
      inputSchema: z.object({
        location: z.string().describe('The city and state'),
      }),
      execute: async ({ location }) => ({
        location,
        temperature: 72,
        condition: 'Sunny',
      }),
      // Defer tool here - Claude discovers these via the tool search tool
      providerOptions: {
        anthropic: { deferLoading: true },
      },
    }),
  },
});
```

#### 使用正则搜索

需要更精确匹配工具时，可使用正则变体：

```ts
const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  prompt: 'Get the weather data',
  tools: {
    toolSearch: anthropic.tools.toolSearchRegex_20251119(),
    // ... deferred tools
  },
});
```

Claude 会构造如 `weather|temperature|forecast` 的正则模式以匹配工具。

#### 自定义工具搜索

可通过 `toModelOutput` 返回 `tool-reference` 内容块来实现自己的工具搜索逻辑（例如使用嵌入或语义搜索）：

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText, tool } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  prompt: 'What is the weather in San Francisco?',
  tools: {
    // Custom search tool
    searchTools: tool({
      description: 'Search for tools by keyword',
      inputSchema: z.object({ query: z.string() }),
      execute: async ({ query }) => {
        // Your custom search logic (embeddings, fuzzy match, etc.)
        const allTools = ['get_weather', 'get_forecast', 'get_temperature'];
        return allTools.filter(name => name.includes(query.toLowerCase()));
      },
      toModelOutput: ({ output }) => ({
        type: 'content',
        value: (output as string[]).map(toolName => ({
          type: 'custom' as const,
          providerOptions: {
            anthropic: {
              type: 'tool-reference',
              toolName,
            },
          },
        })),
      }),
    }),

    // Deferred tools
    get_weather: tool({
      description: 'Get the current weather',
      inputSchema: z.object({ location: z.string() }),
      execute: async ({ location }) => ({ location, temperature: 72 }),
      providerOptions: {
        anthropic: { deferLoading: true },
      },
    }),
  },
});
```

这会将 `tool_reference` 块发送给 Anthropic，从而将对应的延迟加载工具 schema 装入 Claude 的上下文。

#### 配置选项

网页抓取工具支持下列配置项：

- **maxUses** _number_

  `maxUses` 参数限制执行的网页抓取次数。

- **allowedDomains** _string[]_

  仅允许从这些域名抓取。

- **blockedDomains** _string[]_

#### 错误处理

网络搜索错误在流式与非流式场景下的处理方式不同：

**非流式（`generateText`）：**
网络搜索错误会抛出异常，可捕获处理：

```ts
try {
  const result = await generateText({
    model: anthropic('claude-opus-4-20250514'),

      web_search: webSearchTool,
    },

} catch (error) {
  if (error.message.includes('Web search failed')) {
    console.log('Search error:', error.message);
    // Handle search error appropriately
  }
}
```

**流式（`streamText`）：**
网络搜索错误会以流中的错误片段送达：

```ts
const result = await streamText({
  model: anthropic('claude-opus-4-20250514'),
  prompt: 'Search for something',
  tools: {
    web_search: webSearchTool,
  },
});

for await (const part of result.textStream) {
  if (part.type === 'error') {
    console.log('Search error:', part.error);
    // Handle search error appropriately
  }
}
```

### 程序化工具调用

[程序化工具调用](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/programmatic-tool-calling) 允许 Claude 在代码执行容器内编写代码来程序化调用你的工具，而无需每次工具调用都往返模型。这可降低多工具工作流的延迟并减少 token 消耗。

要对希望从代码执行内可调用的工具启用程序化工具调用，请在这些工具上使用 `allowedCallers` 提供方选项：

```ts highlight="13-17"
import {
  anthropic,
  forwardAnthropicContainerIdFromLastStep,
} from '@ai-sdk/anthropic';
import { generateText, tool, isStepCount } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  stopWhen: isStepCount(10),
  prompt:
    'Get the weather for Tokyo, Sydney, and London, then calculate the average temperature.',
  tools: {
    code_execution: anthropic.tools.codeExecution_20260120(),

    getWeather: tool({
      description: 'Get current weather data for a city.',
      inputSchema: z.object({
        city: z.string().describe('Name of the city'),
      }),
      execute: async ({ city }) => {
        // Your weather API implementation
        return { temp: 22, condition: 'Sunny' };
      },
      // Enable this tool to be called from within code execution
      providerOptions: {
        anthropic: {
          allowedCallers: ['code_execution_20260120'],
        },
      },
    }),
  },

  // Propagate container ID between steps for code execution continuity
  prepareStep: forwardAnthropicContainerIdFromLastStep,
});
```

在此流程中：

1. Claude 编写 Python 代码，并行多次调用你的 `getWeather` 工具
2. SDK 自动执行你的工具并将结果返回到代码执行容器
3. Claude 在代码中处理结果并生成最终回复

<Note>
  程序化工具调用需要 `claude-sonnet-4-6`、`claude-sonnet-4-5`、
  `claude-opus-4-6` 或 `claude-opus-4-5` 模型，并使用
  `code_execution_20260120` 或 `code_execution_20250825` 工具。
</Note>

#### 容器持久化

跨多步使用程序化工具调用时，需使用 `prepareStep` 在步骤间保留容器 ID。可使用 `forwardAnthropicContainerIdFromLastStep` 辅助函数自动完成。每步完成后，容器 ID 位于 `providerMetadata.anthropic.container.id`。

## 智能体技能（Agent Skills）

[Anthropic Agent Skills](https://docs.claude.com/en/docs/agents-and-tools/agent-skills/overview) 使 Claude 能执行文档处理（PPTX、DOCX、PDF、XLSX）与数据分析等专项任务。Skills 在沙箱容器中运行，且须启用代码执行工具。

### 使用内置 Skills

Anthropic 提供多种内置 skill：

- **pptx** - 创建与编辑 PowerPoint 演示文稿
- **docx** - 创建与编辑 Word 文档
- **pdf** - 处理与分析 PDF 文件
- **xlsx** - 使用 Excel 电子表格

使用 skill 需要：

1. 启用代码执行工具
2. 在 `providerOptions` 中指定带 skills 的容器

```ts highlight="4,9-17,19-23"
import { anthropic, AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import { generateText } from 'ai';

const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  tools: {
    code_execution: anthropic.tools.codeExecution_20260120(),
  },
  prompt: 'Create a presentation about renewable energy with 5 slides',
  providerOptions: {
    anthropic: {
      container: {
        skills: [
          {
            type: 'anthropic',
            skillId: 'pptx',
            version: 'latest', // optional
          },
        ],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

### 自定义 Skills

也可通过指定 `type: 'custom'` 使用自定义 skill：

```ts highlight="9-11"
const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  tools: {
    code_execution: anthropic.tools.codeExecution_20260120(),
  },
  prompt: 'Use my custom skill to process this data',
  providerOptions: {
    anthropic: {
      container: {
        skills: [
          {
            type: 'custom',
            skillId: 'my-custom-skill-id',
            version: '1.0', // optional
          },
        ],
      },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

<Note>
  Skills 采用渐进式上下文加载，并在具备代码执行能力的沙箱容器中执行。
</Note>

### PDF 支持

Anthropic Claude 模型支持读取 PDF 文件。
可通过 `file` 类型将 PDF 作为消息内容的一部分传入：

方式一：基于 URL 的 PDF 文档

```ts
const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model according to this document?',
        },
        {
          type: 'file',
          data: new URL(
            'https://github.com/vercel/ai/blob/main/examples/ai-functions/data/ai.pdf?raw=true',
          ),
          mimeType: 'application/pdf',
        },
      ],
    },
  ],
});
```

方式二：Base64 编码的 PDF 文档

```ts
const result = await generateText({
  model: anthropic('claude-sonnet-4-5'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model according to this document?',
        },
        {
          type: 'file',
          data: fs.readFileSync('./data/ai.pdf'),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
});
```

模型可访问 PDF 文件内容并
据此回答问题。
PDF 应通过 `data` 字段传入，
`mediaType` 应设为 `'application/pdf'`。
