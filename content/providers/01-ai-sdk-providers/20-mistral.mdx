# Mistral AI 提供商

[Mistral AI](https://mistral.ai/) 提供商为 Mistral Chat API 提供语言模型支持。

## 安装

Mistral 提供商位于 `@ai-sdk/mistral` 模块中，可使用以下命令安装：

## Provider 实例

可从 `@ai-sdk/mistral` 导入默认的 `mistral` 提供商实例：

```ts
import { mistral } from '@ai-sdk/mistral';
```

如需自定义配置，可导入 `createMistral` 并以你的设置创建提供商实例：

```ts
import { createMistral } from '@ai-sdk/mistral';

const mistral = createMistral({
  // custom settings
});
```

可使用以下可选设置自定义 Mistral 提供商实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀（例如经由代理）。
  默认前缀为 `https://api.mistral.ai/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。
  默认为环境变量 `MISTRAL_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可作为中间件拦截请求，
  或在测试中提供自定义 `fetch` 实现。

## 语言模型

可使用提供商实例创建调用 [Mistral Chat API](https://docs.mistral.ai/api/#operation/createChatCompletion) 的模型。
第一个参数为模型 ID，例如 `mistral-large-latest`。
部分 Mistral Chat 模型支持工具调用。

```ts
const model = mistral('mistral-large-latest');
```

Mistral Chat 模型还支持不属于 [标准调用设置](/docs/ai-sdk-core/settings) 的额外模型选项。
可将它们作为选项参数传入，并使用 `MistralLanguageModelChatOptions` 进行类型标注：

```ts
import { mistral, type MistralLanguageModelChatOptions } from '@ai-sdk/mistral';
const model = mistral('mistral-large-latest');

await generateText({
  model,
  providerOptions: {
    mistral: {
      safePrompt: true, // optional safety prompt injection
      parallelToolCalls: false, // disable parallel tool calls (one tool per response)
    } satisfies MistralLanguageModelChatOptions,
  },
});
```

面向 Mistral 模型，可使用以下可选提供商选项：

- **safePrompt** _boolean_

  是否在所有对话前注入安全提示。

  默认为 `false`。

- **documentImageLimit** _number_

  文档中可处理的最大图像数量。

- **documentPageLimit** _number_

  文档中可处理的最大页数。

- **strictJsonSchema** _boolean_

  结构化输出是否使用严格的 JSON Schema 校验。仅在提供 schema 时生效；除启用 [自定义结构化输出（Custom Structured Outputs）](https://docs.mistral.ai/capabilities/structured-output/custom_structured_output/)（提供 schema 时默认使用）外，还会设置 [`strict` 标记](https://docs.mistral.ai/api/#tag/chat/operation/chat_completion_v1_chat_completions_post)。

  默认为 `false`。

- **structuredOutputs** _boolean_

  是否使用 [结构化输出](#structured-outputs)。启用后，工具调用与对象生成将严格要求并遵循所提供的 schema。

  默认为 `true`。

- **parallelToolCalls** _boolean_

  是否在工具使用过程中启用并行函数调用。为 `false` 时，每条回复最多调用一个工具。

  默认为 `true`。

### 文档 OCR

Mistral Chat 模型支持对 PDF 进行文档 OCR。
可通过提供商选项设置图像与页数上限。

```ts
import { mistral, type MistralLanguageModelChatOptions } from '@ai-sdk/mistral';
import { generateText } from 'ai';

const result = await generateText({
  model: mistral('mistral-small-latest'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model according to this document?',
        },
        {
          type: 'file',
          data: new URL(
            'https://github.com/vercel/ai/blob/main/examples/ai-functions/data/ai.pdf?raw=true',
          ),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
  // optional settings:
  providerOptions: {
    mistral: {
      documentImageLimit: 8,
      documentPageLimit: 64,
    } satisfies MistralLanguageModelChatOptions,
  },
});
```

### 推理模型

Mistral 提供具备逐步推理能力的推理模型：

- **magistral-small-2507**：较小的推理模型，适合高效逐步推理
- **magistral-medium-2507**：更强的推理模型，在性能与成本之间取得平衡

这些模型返回结构化的推理内容，由 AI SDK 自动提取。推理内容可通过结果中的 `reasoningText` 属性访问：

```ts
import { mistral } from '@ai-sdk/mistral';
import { generateText } from 'ai';

const result = await generateText({
  model: mistral('magistral-small-2507'),
  prompt: 'What is 15 * 24?',
});

console.log('REASONING:', result.reasoningText);
// Output: "Let me calculate this step by step..."

console.log('ANSWER:', result.text);
// Output: "360"
```

SDK 会自动解析 Mistral 原生的推理格式，并在结果中分别提供 `reasoningText` 与 `text`，无需中间件。

#### 可配置推理

部分 Mistral 模型支持可配置推理，可通过 `reasoning` 参数控制。
可使用 AI SDK 顶层的 `reasoning` 设置来调节推理强度：

```ts
import { mistral } from '@ai-sdk/mistral';
import { generateText } from 'ai';

const result = await generateText({
  model: mistral('mistral-small-latest'),
  reasoning: 'high',
  prompt: 'What is 15 * 24?',
});

console.log('REASONING:', result.reasoningText);
console.log('ANSWER:', result.text);
```

目前 Mistral 仅支持 `'high'` 与 `'none'` 作为推理强度档位。

### 示例

可使用 Mistral 语言模型与 `generateText` 生成文本：

```ts
import { mistral } from '@ai-sdk/mistral';
import { generateText } from 'ai';

const { text } = await generateText({
  model: mistral('mistral-large-latest'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Mistral 语言模型也可用于 `streamText`，
并支持通过 [`Output`](/docs/reference/ai-sdk-core/output) 生成结构化数据
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

#### Structured Outputs（结构化输出）

Mistral Chat 模型支持使用 JSON Schema 的结构化输出。可将 `generateText` 或 `streamText` 与 [`Output`](/docs/reference/ai-sdk-core/output)
以及 Zod、Valibot 或原始 JSON Schema 一起使用。SDK 会通过 Mistral 的 `response_format: { type: 'json_schema' }` 发送你的 schema。

```ts
import { mistral } from '@ai-sdk/mistral';
import { generateText, Output } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: mistral('mistral-large-latest'),
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(z.string()),
        instructions: z.array(z.string()),
      }),
    }),
  }),
  prompt: 'Generate a simple pasta recipe.',
});

console.log(JSON.stringify(result.output, null, 2));
```

可通过提供商选项启用严格的 JSON Schema 校验：

```ts highlight="7-11"
import { mistral, type MistralLanguageModelChatOptions } from '@ai-sdk/mistral';
import { generateText, Output } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: mistral('mistral-large-latest'),
  providerOptions: {
    mistral: {
      strictJsonSchema: true,
    } satisfies MistralLanguageModelChatOptions,
  },
  output: Output.object({
    schema: z.object({
      title: z.string(),
      items: z.array(
        z.object({ id: z.string(), qty: z.number().int().min(1) }),
      ),
    }),
  }),
  prompt: 'Generate a small shopping list.',
});
```

<Note>
  使用结构化输出时，SDK 不再额外注入「以 JSON 回答」类指令，
  而是依赖 Mistral 原生的 `json_schema` / `json_object` 响应格式。
  schema 的名称与描述可通过标准的结构化输出 API 自定义。
</Note>

## 嵌入模型

可使用 `.embedding()` 工厂方法创建调用 [Mistral Embeddings API](https://docs.mistral.ai/api/#operation/createEmbedding) 的模型。

```ts
const model = mistral.embedding('mistral-embed');
```

可使用 Mistral 嵌入模型与 `embed` 生成向量：

```ts
import { mistral } from '@ai-sdk/mistral';
import { embed } from 'ai';

const { embedding } = await embed({
  model: mistral.embedding('mistral-embed'),
  value: 'sunny day at the beach',
});
```
