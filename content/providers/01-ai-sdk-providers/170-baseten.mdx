# Baseten 提供商

[Baseten](https://baseten.co/) 是推理平台，可通过其 [API](https://docs.baseten.co/overview) 提供前沿、企业级开源 AI 模型服务。

## 安装

Baseten 提供商通过 `@ai-sdk/baseten` 模块提供，可使用以下命令安装：

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/baseten" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @ai-sdk/baseten" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/baseten" dark />
  </Tab>
</Tabs>

## Provider 实例

可从 `@ai-sdk/baseten` 导入默认的 `baseten` 提供商实例：

```ts
import { baseten } from '@ai-sdk/baseten';
```

如需自定义配置，可导入 `createBaseten` 并创建实例：

```ts
import { createBaseten } from '@ai-sdk/baseten';

const baseten = createBaseten({
  apiKey: process.env.BASETEN_API_KEY ?? '',
});
```

可使用以下可选设置自定义 Baseten 提供商实例：

- **baseURL** _string_

  为 API 请求使用不同的 URL 前缀，例如经由代理。
  默认前缀为 `https://inference.baseten.co/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。默认为环境变量 `BASETEN_API_KEY`。建议使用 `export` 设置环境变量以免每次手动传入。
  Baseten API Key 获取见[此处](https://app.baseten.co/settings/api_keys)。

- **modelURL** _string_

  面向特定模型（聊天或嵌入）的自定义模型 URL。若未提供，则使用默认 Model APIs。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。

## 托管模型接口（Model APIs）

可使用提供商实例选择 [Baseten 模型](https://www.baseten.co/products/model-apis/)；首个参数为模型 ID，例如 `'moonshotai/Kimi-K2-Instruct-0905'`。Model APIs 下支持的完整模型列表见[此处](https://docs.baseten.co/development/model-apis/overview#supported-models)。

```ts
const model = baseten('moonshotai/Kimi-K2-Instruct-0905');
```

### 示例

可使用 Baseten 语言模型与 `generateText` 生成文本：

```ts
import { baseten } from '@ai-sdk/baseten';
import { generateText } from 'ai';

const { text } = await generateText({
  model: baseten('moonshotai/Kimi-K2-Instruct-0905'),
  prompt: 'What is the meaning of life? Answer in one sentence.',
});
```

Baseten 语言模型也可用于 `streamText`
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

## 专用模型（Dedicated Models）

Baseten 支持为聊天与嵌入模型使用专用 URL。创建提供商时必须指定 `modelURL`：

### OpenAI 兼容端点（`/sync/v1`）

针对使用 Baseten OpenAI 兼容端点部署的模型：

```ts
import { createBaseten } from '@ai-sdk/baseten';

const baseten = createBaseten({
  modelURL: 'https://model-{MODEL_ID}.api.baseten.co/sync/v1',
});
// No modelId is needed because we specified modelURL
const model = baseten();
const { text } = await generateText({
  model: model,
  prompt: 'Say hello from a Baseten chat model!',
});
```

### `/predict` 端点

`/predict` 端点**目前不支持**聊天模型。聊天功能必须使用 `/sync/v1` 端点。

## 嵌入模型

可使用 `.embeddingModel()` 工厂方法创建调用 Baseten embeddings API 的模型。Baseten 提供商使用高性能的 `@basetenlabs/performance-client` 以优化嵌入性能。

<Note>
  **重要：** 嵌入模型需要带自定义 `modelURL` 的专用部署。与聊天模型不同，嵌入不能仅使用 Baseten 默认 Model APIs，必须指定专用模型端点。
</Note>

```ts
import { createBaseten } from '@ai-sdk/baseten';
import { embed, embedMany } from 'ai';

const baseten = createBaseten({
  modelURL: 'https://model-{MODEL_ID}.api.baseten.co/sync',
});

const embeddingModel = baseten.embeddingModel();

// Single embedding
const { embedding } = await embed({
  model: embeddingModel,
  value: 'sunny day at the beach',
});

// Batch embeddings
const { embeddings } = await embedMany({
  model: embeddingModel,
  values: [
    'sunny day at the beach',
    'rainy afternoon in the city',
    'snowy mountain peak',
  ],
});
```

### 嵌入端点支持

**支持：**

- `/sync` 端点（Performance Client 会自动追加 `/v1/embeddings`）
- `/sync/v1` 端点（在传给 Performance Client 前会自动去除 `/v1`）

**不支持：**

- `/predict` 端点（与 Performance Client 不兼容）

### 性能特性

嵌入实现包含：

- **高性能客户端**：使用 `@basetenlabs/performance-client` 以获得最佳性能
- **自动批处理**：在单次请求中高效处理多条文本
- **连接复用**：Performance Client 只创建一次并在所有请求间复用
- **内置重试**：失败请求自动重试

## 错误处理

Baseten 提供商内置常见 API 错误的处理：

```ts
import { baseten } from '@ai-sdk/baseten';
import { generateText } from 'ai';

try {
  const { text } = await generateText({
    model: baseten('moonshotai/Kimi-K2-Instruct-0905'),
    prompt: 'Hello, world!',
  });
} catch (error) {
  console.error('Baseten API error:', error.message);
}
```

### 常见错误场景

```ts
// Embeddings require a modelURL
try {
  baseten.embeddingModel();
} catch (error) {
  // Error: "No model URL provided for embeddings. Please set modelURL option for embeddings."
}

// /predict endpoints are not supported for chat models
try {
  const baseten = createBaseten({
    modelURL:
      'https://model-{MODEL_ID}.api.baseten.co/environments/production/predict',
  });
  baseten(); // This will throw an error
} catch (error) {
  // Error: "Not supported. You must use a /sync/v1 endpoint for chat models."
}

// /sync/v1 endpoints are now supported for embeddings
const baseten = createBaseten({
  modelURL:
    'https://model-{MODEL_ID}.api.baseten.co/environments/production/sync/v1',
});
const embeddingModel = baseten.embeddingModel(); // This works fine!

// /predict endpoints are not supported for embeddings
try {
  const baseten = createBaseten({
    modelURL:
      'https://model-{MODEL_ID}.api.baseten.co/environments/production/predict',
  });
  baseten.embeddingModel(); // This will throw an error
} catch (error) {
  // Error: "Not supported. You must use a /sync or /sync/v1 endpoint for embeddings."
}

// Image models are not supported
try {
  baseten.imageModel('test-model');
} catch (error) {
  // Error: NoSuchModelError for imageModel
}
```
