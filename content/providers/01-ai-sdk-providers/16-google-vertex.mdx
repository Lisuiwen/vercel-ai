# Google Vertex 提供商

面向 [AI SDK](/docs) 的 Google Vertex 提供商为 [Google Vertex AI](https://cloud.google.com/vertex-ai) API 提供语言模型支持，其中包括 [Google 的 Gemini 模型](https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models)、[Anthropic 的 Claude 合作模型](https://cloud.google.com/vertex-ai/generative-ai/docs/partner-models/use-claude)、[xAI 的 Grok 合作模型](https://cloud.google.com/vertex-ai/generative-ai/docs/partner-models/grok)，以及 [MaaS（模型即服务）开放模型](https://cloud.google.com/vertex-ai/generative-ai/docs/maas/use-open-models)。

<Note>
  Google Vertex 提供商同时兼容 Node.js 与 Edge 运行时。
  Edge 运行时通过 `@ai-sdk/google-vertex/edge`
  子模块提供支持。更多说明见下文 [Google Vertex Edge
  运行时](#google-vertex-edge-runtime)、[Google Vertex Anthropic Edge
  运行时](#google-vertex-anthropic-edge-runtime) 与 [Google Vertex MaaS Edge
  运行时](#google-vertex-maas-edge-runtime) 各节。
</Note>

## Google Vertex 提供商用法

通过 Google Vertex 提供商实例可创建调用 Vertex AI API 的模型实例。该提供商涵盖 [Google 的 Gemini 模型](https://cloud.google.com/vertex-ai/generative-ai/docs/learn/models)。若要使用 [Anthropic 的 Claude 模型](https://cloud.google.com/vertex-ai/generative-ai/docs/partner-models/use-claude)，请参阅下文 [Google Vertex Anthropic 提供商](#google-vertex-anthropic-provider-usage) 一节。

### 提供商实例

可从 `@ai-sdk/google-vertex` 导入默认提供商实例 `googleVertex`：

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
```

若需自定义配置，可从 `@ai-sdk/google-vertex` 导入 `createGoogleVertex`，并按你的设置创建提供商实例：

```ts
import { createGoogleVertex } from '@ai-sdk/google-vertex';

const googleVertex = createGoogleVertex({
  project: 'my-project', // optional
  location: 'us-central1', // optional
});
```

Google Vertex 会根据运行时环境与需求支持多种身份验证方式。

#### Node.js 运行时

Node.js 运行时是 AI SDK 的默认运行时，可通过 [`google-auth-library`](https://github.com/googleapis/google-auth-library-nodejs?tab=readme-ov-file#ways-to-authenticate) 使用所有标准 Google Cloud 身份验证选项。常见做法是在 `GOOGLE_APPLICATION_CREDENTIALS` 环境变量中指定 JSON 凭据文件路径。凭据文件可从 [Google Cloud 控制台](https://console.cloud.google.com/apis/credentials) 获取。

若要自定义 Google 身份验证选项，可将其作为参数传给 `createGoogleVertex`，例如：

```ts
import { createGoogleVertex } from '@ai-sdk/google-vertex';

const googleVertex = createGoogleVertex({
  googleAuthOptions: {
    credentials: {
      client_email: 'my-email',
      private_key: 'my-private-key',
    },
  },
});
```

##### 可选提供商设置

可使用以下可选设置自定义提供商实例：

- **project** _string_

  用于发起 API 调用的 Google Cloud 项目 ID。
  默认读取 `GOOGLE_VERTEX_PROJECT` 环境变量。

- **location** _string_

  用于发起 API 调用的 Google Cloud 区域，例如 `us-central1`。
  默认读取 `GOOGLE_VERTEX_LOCATION` 环境变量。

- **googleAuthOptions** _object_

  可选。供 [Google Auth Library](https://github.com/googleapis/google-auth-library-nodejs/) 使用的身份验证选项。另请参阅 [GoogleAuthOptions](https://github.com/googleapis/google-auth-library-nodejs/blob/08978822e1b7b5961f0e355df51d738e012be392/src/auth/googleauth.ts#L87C18-L87C35) 接口。
  - **authClient** _object_
    要使用的 `AuthClient`。

  - **keyFilename** _string_
    指向 .json、.pem 或 .p12 密钥文件的路径。

  - **keyFile** _string_
    指向 .json、.pem 或 .p12 密钥文件的路径。

  - **credentials** _object_
    包含 client_email 与 private_key 字段的对象，或外部账号客户端选项。

  - **clientOptions** _object_
    传给客户端构造函数的选项对象。

  - **scopes** _string | string[]_
    目标 API 请求所需的权限范围。

  - **projectId** _string_
    你的项目 ID。

  - **universeDomain** _string_
    给定 Cloud 宇宙的默认服务域名。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  要包含在请求中的标头。可用多种形式提供：
  - 标头键值对对象：`Record<string, string | undefined>`
  - 返回标头的函数：`() => Record<string, string | undefined>`
  - 异步返回标头的函数：`async () => Record<string, string | undefined>`
  - 解析为标头的 Promise：`Promise<Record<string, string | undefined>>`

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认使用全局 `fetch`。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch。

- **baseURL** _string_

  可选。Google Vertex API 调用的基础 URL，例如用于代理服务器。默认会基于 location 与 project 拼接：
  `https://${location}-aiplatform.googleapis.com/v1/projects/${project}/locations/${location}/publishers/google`

<a id="google-vertex-edge-runtime"></a>
#### Edge 运行时

Edge 运行时（如 Vercel Edge Functions 与 Cloudflare Workers）是更贴近用户的轻量 JavaScript 环境。
仅提供标准 Node.js API 的子集。
例如无法直接访问文件系统，许多 Node.js 专用库
（包括标准 Google Auth 库）也不兼容。

Google Vertex 提供商的 Edge 版本可通过环境变量使用 Google 的[应用程序默认凭据](https://github.com/googleapis/google-auth-library-nodejs?tab=readme-ov-file#application-default-credentials)。相关取值可从 [Google Cloud 控制台](https://console.cloud.google.com/apis/credentials) 的 JSON 凭据文件获取。

可从 `@ai-sdk/google-vertex/edge` 导入默认提供商实例 `googleVertex`：

```ts
import { googleVertex } from '@ai-sdk/google-vertex/edge';
```

<Note>
  `/edge` 子模块已包含在 `@ai-sdk/google-vertex` 包内，因此
  无需单独安装。必须从
  `@ai-sdk/google-vertex/edge` 导入，以与 Node.js 版提供商区分。
</Note>

若需自定义配置，可从 `@ai-sdk/google-vertex/edge` 导入 `createGoogleVertex`，并按你的设置创建提供商实例：

```ts
import { createGoogleVertex } from '@ai-sdk/google-vertex/edge';

const googleVertex = createGoogleVertex({
  project: 'my-project', // optional
  location: 'us-central1', // optional
});
```

在 Edge 运行时中完成身份验证时，需根据 Google 应用程序默认凭据 JSON 文件设置以下环境变量：

- `GOOGLE_CLIENT_EMAIL`
- `GOOGLE_PRIVATE_KEY`
- `GOOGLE_PRIVATE_KEY_ID`（可选）

这些值可从 [Google Cloud 控制台](https://console.cloud.google.com/apis/credentials) 的服务账号 JSON 文件获取。

##### 可选提供商设置

可使用以下可选设置自定义提供商实例：

- **project** _string_

  用于发起 API 调用的 Google Cloud 项目 ID。
  默认读取 `GOOGLE_VERTEX_PROJECT` 环境变量。

- **location** _string_

  用于发起 API 调用的 Google Cloud 区域，例如 `us-central1`。
  默认读取 `GOOGLE_VERTEX_LOCATION` 环境变量。

- **googleCredentials** _object_

  可选。Edge 提供商用于身份验证的凭据。这些凭据通常通过环境变量设置，并源自服务账号 JSON 文件。
  - **clientEmail** _string_
    服务账号 JSON 文件中的客户端邮箱。默认读取 `GOOGLE_CLIENT_EMAIL` 环境变量。

  - **privateKey** _string_
    服务账号 JSON 文件中的私钥。默认读取 `GOOGLE_PRIVATE_KEY` 环境变量。

  - **privateKeyId** _string_
    服务账号 JSON 文件中的私钥 ID（可选）。默认读取 `GOOGLE_PRIVATE_KEY_ID` 环境变量。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  要包含在请求中的标头。可用多种形式提供：
  - 标头键值对对象：`Record<string, string | undefined>`
  - 返回标头的函数：`() => Record<string, string | undefined>`
  - 异步返回标头的函数：`async () => Record<string, string | undefined>`
  - 解析为标头的 Promise：`Promise<Record<string, string | undefined>>`

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认使用全局 `fetch`。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch。

#### 快捷模式（Express Mode）

快捷模式使用 API 密钥完成身份验证，无需 OAuth 或服务账号凭据。启用快捷模式时，不需要配置 `project` 与 `location`。

```ts
import { createGoogleVertex } from '@ai-sdk/google-vertex';

const googleVertex = createGoogleVertex({
  apiKey: process.env.GOOGLE_VERTEX_API_KEY,
});
```

##### 可选提供商设置

- **apiKey** _string_

  Google Vertex AI 的 API 密钥。提供该值后，提供商将使用 API 密钥的快捷模式，而不再使用 OAuth。
  默认读取 `GOOGLE_VERTEX_API_KEY` 环境变量。

### 语言模型

使用提供商实例可创建调用 Vertex API 的模型。
第一个参数为模型 ID，例如 `gemini-2.5-pro`。

```ts
const model = googleVertex('gemini-2.5-pro');
```

<Note>
  若使用[自有
  模型](https://cloud.google.com/vertex-ai/docs/training-overview)，模型
  名称需以 `projects/` 开头。
</Note>

Google Vertex 模型还支持一些不属于[标准调用设置](/docs/ai-sdk-core/settings)的模型专属配置，可通过
options 参数传入：

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';

const model = googleVertex('gemini-2.5-pro');

await generateText({
  model,
  providerOptions: {
    vertex: {
      safetySettings: [
        {
          category: 'HARM_CATEGORY_UNSPECIFIED',
          threshold: 'BLOCK_LOW_AND_ABOVE',
        },
      ],
    } satisfies GoogleLanguageModelOptions,
  },
});
```

Google Vertex 语言模型可使用以下可选提供商选项：

- **cachedContent** _string_

  可选。用作预测上下文的缓存内容名称。
  格式：projects/\{project\}/locations/\{location\}/cachedContents/\{cachedContent\}

- **structuredOutputs** _boolean_

  可选。启用结构化输出。默认为 true。

  当 JSON Schema 包含
  Google Vertex 所使用的 OpenAPI schema 版本
  不支持的元素时，这会很有用。必要时可用其关闭
  结构化输出。

  更多说明见[故障排除：Schema 限制](#schema-limitations)。

- **safetySettings** _Array\<\{ category: string; threshold: string \}\>_

  可选。模型的安全设置。
  - **category** _string_

    安全设置类别。可为以下之一：
    - `HARM_CATEGORY_UNSPECIFIED`
    - `HARM_CATEGORY_HATE_SPEECH`
    - `HARM_CATEGORY_DANGEROUS_CONTENT`
    - `HARM_CATEGORY_HARASSMENT`
    - `HARM_CATEGORY_SEXUALLY_EXPLICIT`
    - `HARM_CATEGORY_CIVIC_INTEGRITY`

  - **threshold** _string_

    安全阈值。可为以下之一：
    - `HARM_BLOCK_THRESHOLD_UNSPECIFIED`
    - `BLOCK_LOW_AND_ABOVE`
    - `BLOCK_MEDIUM_AND_ABOVE`
    - `BLOCK_ONLY_HIGH`
    - `BLOCK_NONE`

- **audioTimestamp** _boolean_

  可选。为音频文件启用时间戳理解。默认为 false。

  适用于需要带准确时间戳的转写场景。
  用法见 [Google 文档](https://cloud.google.com/vertex-ai/generative-ai/docs/multimodal/audio-understanding)。

- **labels** _object_

  可选。在账单报表中使用的标签。

  用法见 [Google 文档](https://cloud.google.com/vertex-ai/generative-ai/docs/multimodal/add-labels-to-api-calls)。

- **streamFunctionCallArguments** _boolean_

  可选。设为 true 时，函数调用参数会在流式响应中
  逐步输出。这样在模型生成函数参数时可更早收到 `tool-input-delta` 事件，
  降低工具调用的感知延迟。默认为 `false`。仅 Vertex AI API（非 Gemini API）在 Gemini 3+ 模型上支持。

  详情见 [Google 文档](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/multimodal/function-calling#streaming-fc)。

可使用 Google Vertex 语言模型配合 `generateText` 生成文本：

```ts highlight="1,4"
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const { text } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Google Vertex 语言模型也可用于 `streamText`
（见 [AI SDK Core](/docs/ai-sdk-core)）。

#### 代码执行

借助[代码执行](https://cloud.google.com/vertex-ai/generative-ai/docs/multimodal/code-execution)，Vertex AI 上的部分 Gemini 模型可生成并执行 Python 代码，从而进行计算、数据处理等程序化任务以丰富回答。

在请求中加入 `code_execution` 工具即可启用代码执行。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  tools: { code_execution: googleVertex.tools.codeExecution({}) },
  prompt:
    'Use python to calculate 20th fibonacci number. Then find the nearest palindrome to it.',
});
```

响应中会包含已执行代码对应的 `tool-call` 与 `tool-result` 片段。

#### URL Context

URL Context 让 Gemini 模型可从 URL 获取并分析内容。支持模型：Gemini 2.5 Flash-Lite、2.5 Pro、2.5 Flash、2.0 Flash。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  tools: { url_context: googleVertex.tools.urlContext({}) },
  prompt: 'What are the key points from https://example.com/article?',
});
```

#### Google Search

Google Search 让 Gemini 模型可访问实时网页信息。支持模型：Gemini 2.5 Flash-Lite、2.5 Flash、2.0 Flash、2.5 Pro。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  tools: { google_search: googleVertex.tools.googleSearch({}) },
  prompt: 'What are the latest developments in AI?',
});
```

#### 企业级网页搜索

[企业级网页搜索](https://cloud.google.com/vertex-ai/generative-ai/docs/grounding/web-grounding-enterprise)使用面向合规的网页索引进行 grounding，适用于金融、医疗、公共部门等强监管行业。与标准 Google Search grounding 不同，企业级网页搜索不会记录客户数据，并支持 VPC 服务控制。支持模型：Gemini 2.0 及更新版本。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-flash'),
  tools: {
    enterprise_web_search: googleVertex.tools.enterpriseWebSearch({}),
  },
  prompt: 'What are the latest FDA regulations for clinical trials?',
});
```

#### Google Maps

Google Maps grounding 让 Gemini 模型可访问 Google Maps 数据，用于位置相关的回答。支持模型：Gemini 2.5 Flash-Lite、2.5 Flash、2.0 Flash、2.5 Pro、3.0 Pro。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-flash'),
  tools: {
    google_maps: googleVertex.tools.googleMaps({}),
  },
  providerOptions: {
    vertex: {
      retrievalConfig: {
        latLng: { latitude: 34.090199, longitude: -117.881081 },
      },
    } satisfies GoogleLanguageModelOptions,
  },
  prompt: 'What are the best Italian restaurants nearby?',
});
```

可选的 `retrievalConfig.latLng` 提供商选项为“附近地点”类查询提供位置上下文。该配置适用于所有支持位置上下文的 grounding 工具。

#### 流式函数调用参数

在 Vertex AI 上为 Gemini 3 Pro 及更新模型使用时，可将 `streamFunctionCallArguments` 设为
`true`，在模型生成过程中流式输出函数调用
参数。这样在需要调用函数时可降低感知延迟，因为
`tool-input-delta` 会增量到达，而无需等待
完整参数。该选项默认为 `false`。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { streamText } from 'ai';
import { z } from 'zod';

const result = streamText({
  model: googleVertex('gemini-3.1-pro-preview'),
  prompt: 'What is the weather in Boston and San Francisco?',
  tools: {
    getWeather: {
      description: 'Get the current weather in a given location',
      inputSchema: z.object({
        location: z.string().describe('City name'),
      }),
    },
  },
  providerOptions: {
    vertex: {
      streamFunctionCallArguments: true,
    } satisfies GoogleLanguageModelOptions,
  },
});

for await (const part of result.fullStream) {
  switch (part.type) {
    case 'tool-input-start':
      console.log(`Tool call started: ${part.toolName}`);
      break;
    case 'tool-input-delta':
      process.stdout.write(part.delta);
      break;
    case 'tool-call':
      console.log(`Tool call complete: ${part.toolName}`, part.input);
      break;
  }
}
```

<Note>
  该功能仅在 Vertex AI API 上可用，Gemini API 不支持。与 Google 提供商一起使用时，会发出警告并忽略该选项。
</Note>

#### 推理（思考令牌）

Google Vertex AI 通过 Gemini 模型支持输出代表推理过程的「思考」令牌。AI SDK 将其作为推理信息对外暴露。

要为兼容的 Gemini 模型（经 Vertex）启用思考令牌，可在 `thinkingConfig` 提供商选项中设置 `includeThoughts: true`。这些选项通过 `providerOptions.vertex` 传入：

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText, streamText } from 'ai';

// For generateText:
const { text, reasoningText, reasoning } = await generateText({
  model: googleVertex('gemini-2.0-flash-001'), // Or other supported model via Vertex
  providerOptions: {
    vertex: {
      thinkingConfig: {
        includeThoughts: true,
        // thinkingBudget: 2048, // Optional
      },
    } satisfies GoogleLanguageModelOptions,
  },
  prompt: 'Explain quantum computing in simple terms.',
});

console.log('Reasoning:', reasoningText);
console.log('Reasoning Details:', reasoning);
console.log('Final Text:', text);

// For streamText:
const result = streamText({
  model: googleVertex('gemini-2.0-flash-001'), // Or other supported model via Vertex
  providerOptions: {
    vertex: {
      thinkingConfig: {
        includeThoughts: true,
        // thinkingBudget: 2048, // Optional
      },
    } satisfies GoogleLanguageModelOptions,
  },
  prompt: 'Explain quantum computing in simple terms.',
});

for await (const part of result.fullStream) {
  if (part.type === 'reasoning') {
    process.stdout.write(`THOUGHT: ${part.textDelta}\n`);
  } else if (part.type === 'text-delta') {
    process.stdout.write(part.textDelta);
  }
}
```

当 `includeThoughts` 为 true 时，API 响应中标记为 `thought: true` 的部分会作为推理处理。

- 在 `generateText` 中，这些内容会汇总到 `reasoningText`（字符串）与 `reasoning`（数组）字段。
- 在 `streamText` 中，这些内容以 `reasoning` 流式片段发出。

<Note>
  模型兼容性与更多说明见 [Google Vertex AI 关于
  「thinking」的文档](https://cloud.google.com/vertex-ai/generative-ai/docs/thinking)。
</Note>

#### 文件输入

Google Vertex 提供商支持文件输入，例如 PDF。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const { text } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model according to this document?',
        },
        {
          type: 'file',
          data: fs.readFileSync('./data/ai.pdf'),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
});
```

<Note>
  若将 URL 作为 `data` 传入，AI SDK 会自动下载，但
  `gs://` URL 除外。可使用 Google Cloud Storage API 将更大的
  文件上传到该位置。
</Note>

在提示词中使用文件的详情见 [File Parts](/docs/foundations/prompts#file-parts)。

### 缓存内容

Google Vertex AI 同时支持显式与隐式缓存，有助于降低重复内容的成本。

#### 隐式缓存

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

// Structure prompts with consistent content at the beginning
const baseContext =
  'You are a cooking assistant with expertise in Italian cuisine. Here are 1000 lasagna recipes for reference...';

const { text: veggieLasagna } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  prompt: `${baseContext}\n\nWrite a vegetarian lasagna recipe for 4 people.`,
});

// Second request with same prefix - eligible for cache hit
const { text: meatLasagna, providerMetadata } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  prompt: `${baseContext}\n\nWrite a meat lasagna recipe for 12 people.`,
});

// Check cached token count in usage metadata
console.log('Cached tokens:', providerMetadata.vertex);
// e.g.
// {
//   groundingMetadata: null,
//   safetyRatings: null,
//   usageMetadata: {
//     cachedContentTokenCount: 2027,
//     thoughtsTokenCount: 702,
//     promptTokenCount: 2152,
//     candidatesTokenCount: 710,
//     totalTokenCount: 3564
//   }
// }
```

#### 显式缓存

可将显式缓存用于 Gemini 模型。你的模型是否支持缓存，请参阅 [Vertex AI 上下文缓存文档](https://cloud.google.com/vertex-ai/generative-ai/docs/context-cache/context-cache-overview)。

首先，使用启用 Vertex 模式的 Google GenAI SDK 创建缓存：

```ts
import { GoogleGenAI } from '@google/genai';

const ai = new GoogleGenAI({
  vertexai: true,
  project: process.env.GOOGLE_VERTEX_PROJECT,
  location: process.env.GOOGLE_VERTEX_LOCATION,
});

const model = 'gemini-2.5-pro';

// Create a cache with the content you want to reuse
const cache = await ai.caches.create({
  model,
  config: {
    contents: [
      {
        role: 'user',
        parts: [{ text: '1000 Lasagna Recipes...' }],
      },
    ],
    ttl: '300s', // Cache expires after 5 minutes
  },
});

console.log('Cache created:', cache.name);
// e.g. projects/my-project/locations/us-central1/cachedContents/abc123
```

随后在 AI SDK 中使用该缓存：

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text: veggieLasagnaRecipe } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
  providerOptions: {
    vertex: {
      cachedContent: cache.name,
    } satisfies GoogleLanguageModelOptions,
  },
});

const { text: meatLasagnaRecipe } = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  prompt: 'Write a meat lasagna recipe for 12 people.',
  providerOptions: {
    vertex: {
      cachedContent: cache.name,
    } satisfies GoogleLanguageModelOptions,
  },
});
```

#### Schema 限制

Google Vertex API 使用 OpenAPI 3.0 schema 的子集，
不支持诸如联合类型（unions）等特性。
此类错误的形态通常如下：

`GenerateContentRequest.generation_config.response_schema.properties[occupation].type: must be specified`

默认会启用结构化输出（工具调用场景下为必需）。
生成对象时可将结构化输出关闭作为变通方案：

```ts highlight="7,12"
import { googleVertex } from '@ai-sdk/google-vertex';
import { type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText, Output } from 'ai';

const result = await generateText({
  model: googleVertex('gemini-2.5-pro'),
  providerOptions: {
    vertex: {
      structuredOutputs: false,
    } satisfies GoogleLanguageModelOptions,
  },
  output: Output.object({
    schema: z.object({
      name: z.string(),
      age: z.number(),
      contact: z.union([
        z.object({
          type: z.literal('email'),
          value: z.string(),
        }),
        z.object({
          type: z.literal('phone'),
          value: z.string(),
        }),
      ]),
    }),
  }),
  prompt: 'Generate an example person for testing.',
});
```

以下 Zod 特性已知与 Google Vertex 不兼容：

- `z.union`
- `z.record`

### 嵌入模型

可使用 `.embeddingModel()` 工厂方法创建调用 Google Vertex AI 嵌入 API 的模型：

```ts
const model = googleVertex.embeddingModel('text-embedding-005');
```

Google Vertex AI 嵌入模型支持额外设置，可通过 options 参数传入：

```ts
import {
  googleVertex,
  type GoogleVertexEmbeddingModelOptions,
} from '@ai-sdk/google-vertex';
import { embed } from 'ai';

const model = googleVertex.embeddingModel('text-embedding-005');

const { embedding } = await embed({
  model,
  value: 'sunny day at the beach',
  providerOptions: {
    vertex: {
      outputDimensionality: 512, // optional, number of dimensions for the embedding
      taskType: 'SEMANTIC_SIMILARITY', // optional, specifies the task type for generating embeddings
      autoTruncate: false, // optional
    } satisfies GoogleVertexEmbeddingModelOptions,
  },
});
```

Google Vertex AI 嵌入模型可使用以下可选提供商选项：

- **outputDimensionality**：_number_

  可选。输出向量的缩减维度。设置后，输出向量末尾多余维度会被截断。

- **taskType**：_string_

  可选。指定生成嵌入的任务类型。支持的任务类型包括：
  - `SEMANTIC_SIMILARITY`：面向文本相似度优化。
  - `CLASSIFICATION`：面向文本分类优化。
  - `CLUSTERING`：面向基于相似度的文本聚类优化。
  - `RETRIEVAL_DOCUMENT`：面向文档检索优化。
  - `RETRIEVAL_QUERY`：面向查询检索优化。
  - `QUESTION_ANSWERING`：面向问答优化。
  - `FACT_VERIFICATION`：面向事实核验优化。
  - `CODE_RETRIEVAL_QUERY`：面向基于自然语言查询检索代码块优化。

- **title**：_string_

  可选。被嵌入文档的标题。通过额外上下文帮助模型生成更佳嵌入。仅在 `taskType` 为 `'RETRIEVAL_DOCUMENT'` 时有效。

- **autoTruncate**：_boolean_

  可选。为 `true` 时，若输入文本超过最大长度会被截断。为 `false` 时，过长会返回错误。默认为 `true`。

#### 模型能力

| 模型                        | 单次调用最大条数 | 并行调用      | 多模态          |
| ---------------------------- | ------------------- | ------------------- | ------------------- |
| `text-embedding-005`         | 2048                | <Check size={18} /> | <Cross size={18} /> |
| `gemini-embedding-2-preview` | 2048                | <Check size={18} /> | <Check size={18} /> |

<Note>
  上表列出常用模型。如有需要，也可直接传入任意可用的提供商模型 ID 字符串。

### 图像模型

可使用 `.image()` 工厂方法创建图像模型。Google Vertex 提供商同时支持 [Imagen](https://cloud.google.com/vertex-ai/generative-ai/docs/image/overview) 与 [Gemini 图像模型](https://cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/2-5-flash-image)。关于使用 AI SDK 生成图像，见 [generateImage()](/docs/reference/ai-sdk-core/generate-image)。

#### Imagen 模型

[Imagen 模型](https://cloud.google.com/vertex-ai/generative-ai/docs/image/generate-images)使用 Vertex AI 上的 Imagen API 生成图像。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: googleVertex.image('imagen-4.0-generate-001'),
  prompt: 'A futuristic cityscape at sunset',
  aspectRatio: '16:9',
});
```

还可通过 Google Vertex 提供商选项进一步配置。可使用 `GoogleVertexImageModelOptions` 类型校验提供商选项。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { GoogleVertexImageModelOptions } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: googleVertex.image('imagen-4.0-generate-001'),
  providerOptions: {
    vertex: {
      negativePrompt: 'pixelated, blurry, low-quality',
    } satisfies GoogleVertexImageModelOptions,
  },
  // ...
});
```

可用提供商选项如下：

- **negativePrompt** _string_
  描述希望在生成图像中避免出现的内容。

- **personGeneration** `allow_adult` | `allow_all` | `dont_allow`
  是否允许生成人物。默认为 `allow_adult`。

- **safetySetting** `block_low_and_above` | `block_medium_and_above` | `block_only_high` | `block_none`
  是否拦截不安全内容。默认为 `block_medium_and_above`。

- **addWatermark** _boolean_
  是否为生成图像添加不可见水印。默认为 `true`。

- **storageUri** _string_
  用于存储生成图像的 Cloud Storage URI。

<Note>
  Imagen 模型不支持 `size` 参数。请改用 `aspectRatio`
  参数。
</Note>

可使用 Google Vertex 元数据获取关于图像的额外信息。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { GoogleVertexImageModelOptions } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';

const { image, providerMetadata } = await generateImage({
  model: googleVertex.image('imagen-4.0-generate-001'),
  prompt: 'A futuristic cityscape at sunset',
  aspectRatio: '16:9',
});

console.log(
  `Revised prompt: ${providerMetadata.vertex.images[0].revisedPrompt}`,
);
```

##### 图像编辑

Google Vertex Imagen 模型支持通过 inpainting、outpainting 等编辑模式处理图像。可通过 `prompt.images` 传入输入图像，并可选通过 `prompt.mask` 传入蒙版。

<Note>
  `imagen-3.0-capability-001` 支持图像编辑。
  `imagen-4.0-generate-001` 目前不支持编辑操作。
</Note>

###### 内补（插入对象）

使用蒙版在指定区域插入或替换对象：

```ts
import {
  googleVertex,
  GoogleVertexImageModelOptions,
} from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';
import fs from 'fs';

const image = fs.readFileSync('./input-image.png');
const mask = fs.readFileSync('./mask.png'); // White = edit area

const { images } = await generateImage({
  model: googleVertex.image('imagen-3.0-capability-001'),
  prompt: {
    text: 'A sunlit indoor lounge area with a pool containing a flamingo',
    images: [image],
    mask,
  },
  providerOptions: {
    vertex: {
      edit: {
        baseSteps: 50,
        mode: 'EDIT_MODE_INPAINT_INSERTION',
        maskMode: 'MASK_MODE_USER_PROVIDED',
        maskDilation: 0.01,
      },
    } satisfies GoogleVertexImageModelOptions,
  },
});
```

###### 外扩（扩展图像）

将图像扩展到原始边界之外：

```ts
import {
  googleVertex,
  GoogleVertexImageModelOptions,
} from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';
import fs from 'fs';

const image = fs.readFileSync('./input-image.png');
const mask = fs.readFileSync('./outpaint-mask.png'); // White = extend area

const { images } = await generateImage({
  model: googleVertex.image('imagen-3.0-capability-001'),
  prompt: {
    text: 'Extend the scene with more of the forest background',
    images: [image],
    mask,
  },
  providerOptions: {
    vertex: {
      edit: {
        baseSteps: 50,
        mode: 'EDIT_MODE_OUTPAINT',
        maskMode: 'MASK_MODE_USER_PROVIDED',
      },
    } satisfies GoogleVertexImageModelOptions,
  },
});
```

###### 编辑提供商选项

`providerOptions.vertex.edit` 下可用选项：

- **mode** - 使用的编辑模式：
  - `EDIT_MODE_INPAINT_INSERTION` - 向蒙版区域插入对象
  - `EDIT_MODE_INPAINT_REMOVAL` - 从蒙版区域移除对象
  - `EDIT_MODE_OUTPAINT` - 向外扩展图像边界
  - `EDIT_MODE_CONTROLLED_EDITING` - 受控编辑
  - `EDIT_MODE_PRODUCT_IMAGE` - 商品图编辑
  - `EDIT_MODE_BGSWAP` - 背景替换

- **baseSteps** _number_ - 采样步数（35-75）。数值越高质量越好但越慢。

- **maskMode** - 蒙版解释方式：
  - `MASK_MODE_USER_PROVIDED` - 直接使用提供的蒙版
  - `MASK_MODE_DEFAULT` - 默认蒙版模式
  - `MASK_MODE_DETECTION_BOX` - 基于检测框的蒙版
  - `MASK_MODE_CLOTHING_AREA` - 基于服装分割的蒙版
  - `MASK_MODE_PARSED_PERSON` - 基于人体解析的蒙版

- **maskDilation** _number_ - 扩展蒙版的百分比（0-1）。建议：0.01。

<Note>
  输入图像需以 `Buffer`、`ArrayBuffer`、`Uint8Array` 或
  base64 字符串提供。Google Vertex
  图像编辑不支持基于 URL 的图像。
</Note>

#### Gemini 图像模型

[Gemini 图像模型](https://cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/2-5-flash-image)（例如 `gemini-2.5-flash-image`）是可多模态输出的语言模型，可与 `generateImage()` 搭配以获得更简单的图像生成体验。在内部，提供商会以 `responseModalities: ['IMAGE']` 调用语言模型 API。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';

const { image } = await generateImage({

  aspectRatio: '1:1',
```

Gemini 图像模型也可通过提供输入图像进行图像编辑：

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';
import fs from 'node:fs';

const sourceImage = fs.readFileSync('./cat.png');

const { image } = await generateImage({
  model: googleVertex.image('gemini-2.5-flash-image'),
  prompt: {
    text: 'Add a small wizard hat to this cat',
    images: [sourceImage],
  },
});
```

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: googleVertex.image('gemini-2.5-flash-image'),
  prompt: {
    text: 'Add a small wizard hat to this cat',
    images: ['https://example.com/cat.png'],
  },
});
```

<Note>
  Gemini 图像模型不支持 `size` 或 `n` 参数。请使用
  `aspectRatio` 代替 `size`。也不支持基于蒙版的外补。
</Note>

<Note>

还可通过 Google Vertex 提供商选项进一步配置。可使用 `GoogleVertexVideoModelOptions` 类型校验提供商选项。

```ts
import { googleVertex } from '@ai-sdk/google-vertex';
import { GoogleVertexVideoModelOptions } from '@ai-sdk/google-vertex';

const { video } = await generateVideo({
```

可用提供商选项如下：

- **generateAudio** _boolean_

  是否同时生成音频。

- **personGeneration** `'dont_allow'` | `'allow_adult'` | `'allow_all'`

  是否允许视频中生成人物。

- **negativePrompt** _string_

  描述希望在生成视频中避免出现的内容。

- **gcsOutputDirectory** _string_

  用于存储生成视频的 Cloud Storage URI。

- **referenceImages** _Array\<\{ bytesBase64Encoded?: string; gcsUri?: string \}\>_

  用于风格或素材引导的参考图。

- **pollIntervalMs** _number_

  轮询任务状态的间隔（毫秒）。

- **pollTimeoutMs** _number_

  视频生成的最长等待时间（毫秒）。

<Note>
  视频生成是异步流程，可能需要数分钟。对于更长视频或更高分辨率，建议将 `pollTimeoutMs` 至少设为
  10 分钟（600000ms）。
</Note>

## Google Vertex Anthropic 提供商用法

面向 [AI SDK](/docs) 的 Google Vertex Anthropic 提供商通过 Google Vertex AI API 支持 Anthropic 的 Claude 模型。本节说明如何配置与使用该提供商。

### 提供商实例

可从 `@ai-sdk/google-vertex/anthropic` 导入默认提供商实例 `vertexAnthropic`：

```typescript
import { vertexAnthropic } from '@ai-sdk/google-vertex/anthropic';
```

若需自定义配置，可从 `@ai-sdk/google-vertex/anthropic` 导入 `createVertexAnthropic`，并按你的设置创建提供商实例：

```typescript
import { createVertexAnthropic } from '@ai-sdk/google-vertex/anthropic';

const vertexAnthropic = createVertexAnthropic({
  project: 'my-project', // optional
  location: 'us-central1', // optional
});
```

#### Node.js 运行时

在 Node.js 环境中，Google Vertex Anthropic 提供商通过 `google-auth-library` 支持所有标准 Google Cloud 身份验证选项。可将身份验证选项传给 `createVertexAnthropic` 进行自定义：

```typescript
import { createVertexAnthropic } from '@ai-sdk/google-vertex/anthropic';

const vertexAnthropic = createVertexAnthropic({
  googleAuthOptions: {
    credentials: {
      client_email: 'my-email',
      private_key: 'my-private-key',
    },
  },
});
```

##### 可选提供商设置

可使用以下可选设置自定义 Google Vertex Anthropic 提供商实例：

- **project** _string_

  用于发起 API 调用的 Google Cloud 项目 ID。
  默认读取 `GOOGLE_VERTEX_PROJECT` 环境变量。

- **location** _string_

  用于发起 API 调用的 Google Cloud 区域，例如 `us-central1`。
  默认读取 `GOOGLE_VERTEX_LOCATION` 环境变量。

- **googleAuthOptions** _object_

  可选。供 [Google Auth Library](https://github.com/googleapis/google-auth-library-nodejs/) 使用的身份验证选项。另请参阅 [GoogleAuthOptions](https://github.com/googleapis/google-auth-library-nodejs/blob/08978822e1b7b5961f0e355df51d738e012be392/src/auth/googleauth.ts#L87C18-L87C35) 接口。
  - **authClient** _object_
    要使用的 `AuthClient`。

  - **keyFilename** _string_
    指向 .json、.pem 或 .p12 密钥文件的路径。

  - **keyFile** _string_
    指向 .json、.pem 或 .p12 密钥文件的路径。

  - **credentials** _object_
    包含 client_email 与 private_key 字段的对象，或外部账号客户端选项。

  - **clientOptions** _object_
    传给客户端构造函数的选项对象。

  - **scopes** _string | string[]_
    目标 API 请求所需的权限范围。

  - **projectId** _string_
    你的项目 ID。

  - **universeDomain** _string_
    给定 Cloud 宇宙的默认服务域名。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  要包含在请求中的标头。可用多种形式提供：
  - 标头键值对对象：`Record<string, string | undefined>`
  - 返回标头的函数：`() => Record<string, string | undefined>`
  - 异步返回标头的函数：`async () => Record<string, string | undefined>`
  - 解析为标头的 Promise：`Promise<Record<string, string | undefined>>`

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认使用全局 `fetch`。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch。

#### Edge 运行时

Edge 运行时（如 Vercel Edge Functions 与 Cloudflare Workers）是更贴近用户的轻量 JavaScript 环境。
仅提供标准 Node.js API 的子集。
例如无法直接访问文件系统，许多 Node.js 专用库
（包括标准 Google Auth 库）也不兼容。

Google Vertex Anthropic 提供商的 Edge 版本可通过环境变量使用 Google 的[应用程序默认凭据](https://github.com/googleapis/google-auth-library-nodejs?tab=readme-ov-file#application-default-credentials)。相关取值可从 [Google Cloud 控制台](https://console.cloud.google.com/apis/credentials) 的 JSON 凭据文件获取。

在 Edge 运行时中，可从 `@ai-sdk/google-vertex/anthropic/edge` 导入提供商实例：

```typescript
import { vertexAnthropic } from '@ai-sdk/google-vertex/anthropic/edge';
```

要自定义配置，请从同一模块导入 `createVertexAnthropic`：

```typescript
import { createVertexAnthropic } from '@ai-sdk/google-vertex/anthropic/edge';

const vertexAnthropic = createVertexAnthropic({
  project: 'my-project', // optional
  location: 'us-central1', // optional
});
```

在 Edge 运行时中完成身份验证时，需根据 Google 应用程序默认凭据 JSON 文件设置以下环境变量：

- `GOOGLE_CLIENT_EMAIL`
- `GOOGLE_PRIVATE_KEY`
- `GOOGLE_PRIVATE_KEY_ID`（可选）

##### 可选提供商设置

可使用以下可选设置自定义提供商实例：

- **project** _string_

  用于发起 API 调用的 Google Cloud 项目 ID。
  默认读取 `GOOGLE_VERTEX_PROJECT` 环境变量。

- **location** _string_

  用于发起 API 调用的 Google Cloud 区域，例如 `us-central1`。
  默认读取 `GOOGLE_VERTEX_LOCATION` 环境变量。

- **googleCredentials** _object_

  可选。Edge 提供商用于身份验证的凭据。这些凭据通常通过环境变量设置，并源自服务账号 JSON 文件。
  - **clientEmail** _string_
    服务账号 JSON 文件中的客户端邮箱。默认读取 `GOOGLE_CLIENT_EMAIL` 环境变量。

  - **privateKey** _string_
    服务账号 JSON 文件中的私钥。默认读取 `GOOGLE_PRIVATE_KEY` 环境变量。

  - **privateKeyId** _string_
    服务账号 JSON 文件中的私钥 ID（可选）。默认读取 `GOOGLE_PRIVATE_KEY_ID` 环境变量。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  要包含在请求中的标头。可用多种形式提供：
  - 标头键值对对象：`Record<string, string | undefined>`
  - 返回标头的函数：`() => Record<string, string | undefined>`
  - 异步返回标头的函数：`async () => Record<string, string | undefined>`
  - 解析为标头的 Promise：`Promise<Record<string, string | undefined>>`

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认使用全局 `fetch`。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch。

### 语言模型

使用提供商实例可创建调用 [Anthropic Messages API](https://docs.anthropic.com/claude/reference/messages_post) 的模型。
第一个参数为模型 ID，例如 `claude-3-haiku-20240307`。
部分模型具备多模态能力。

```ts
const model = anthropic('claude-3-haiku-20240307');
```

可使用 Anthropic 语言模型配合 `generateText` 生成文本：

```ts
import { vertexAnthropic } from '@ai-sdk/google-vertex/anthropic';
import { generateText } from 'ai';

const { text } = await generateText({
  model: vertexAnthropic('claude-3-haiku-20240307'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Anthropic 语言模型也可用于 `streamText`，
并支持配合 [`Output`](/docs/reference/ai-sdk-core/output) 进行结构化数据生成
（见 [AI SDK Core](/docs/ai-sdk-core)）。

<Note>
  Anthropic API 会在延迟后一次性返回流式工具调用。这会导致
  结构化输出在一段时间后整体完成，而不是
  逐步流式输出。
</Note>

Anthropic 模型还可使用以下可选提供商选项：

- `sendReasoning` _boolean_

  可选。是否在发给模型的请求中包含推理内容。默认为 `true`。

  若处理包含推理内容的请求时遇到问题，
  可将其设为 `false` 以从请求中省略推理内容。

- `thinking` _object_

  可选。更多说明见[推理](#reasoning)一节。

- `metadata` _object_

  可选。随请求发送的元数据。详情见 [Anthropic API 文档](https://platform.claude.com/docs/en/api/messages/create)。
  - `userId` _string_ - 终端用户的外部标识符。

### 推理

`claude-3-7-sonnet@20250219` 模型支持 Anthropic 的推理能力。

可使用 `thinking` 提供商选项启用，
并指定以 token 计的思考预算。

```ts
import { vertexAnthropic } from '@ai-sdk/google-vertex/anthropic';
import { generateText } from 'ai';

const { text, reasoningText, reasoning } = await generateText({
  model: vertexAnthropic('claude-3-7-sonnet@20250219'),
  prompt: 'How many people will live in the world in 2040?',
  providerOptions: {
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    },
  },
});

console.log(reasoningText); // reasoning text
console.log(reasoning); // reasoning details including redacted reasoning
console.log(text); // text response
```

将推理能力集成到聊天机器人的更多说明，见 [AI SDK UI：Chatbot](/docs/ai-sdk-ui/chatbot#reasoning)。

#### 缓存控制

<Note>
  在 Google
  Vertex 上，Anthropic 的缓存控制处于预正式发布（Pre-GA）状态。更多见 [Google Vertex Anthropic 缓存控制
  文档](https://cloud.google.com/vertex-ai/generative-ai/docs/partner-models/claude-prompt-caching)。
</Note>

在消息与消息片段中，可使用 `providerOptions` 设置缓存控制断点。
需要在 `providerOptions` 上将 `anthropic` 设为 `{ cacheControl: { type: 'ephemeral' } }` 来创建缓存断点。

`generateText` 与 `streamText` 均会在标准 `usage` 对象上返回缓存读取与缓存写入（创建）的 token 计数。可通过
`result.usage.inputTokenDetails.cacheReadTokens` 与
`result.usage.inputTokenDetails.cacheWriteTokens` 访问。

```ts highlight="8,16-18,27-31"
import { vertexAnthropic } from '@ai-sdk/google-vertex/anthropic';
import { generateText } from 'ai';

const errorMessage = '... long error message ...';

const result = await generateText({
  model: vertexAnthropic('claude-3-5-sonnet-20240620'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: 'You are a JavaScript expert.' },
        {
          type: 'text',
          text: `Error message: ${errorMessage}`,
          providerOptions: {
            anthropic: { cacheControl: { type: 'ephemeral' } },
          },
        },
        { type: 'text', text: 'Explain the error message.' },
      ],
    },
  ],
});

console.log(result.text);
console.log(
  'Cache read tokens:',
  result.usage.inputTokenDetails.cacheReadTokens,
);
console.log(
  'Cache write tokens:',
  result.usage.inputTokenDetails.cacheWriteTokens,
);
```

也可在系统消息上使用缓存控制：在 `messages` 数组开头提供多条系统消息即可。

```ts highlight="3,7-9"
const result = await generateText({
  model: vertexAnthropic('claude-3-5-sonnet-20240620'),
  messages: [
    {
      role: 'system',
      content: 'Cached system message part',
      providerOptions: {
        anthropic: { cacheControl: { type: 'ephemeral' } },
      },
    },
    {
      role: 'system',
      content: 'Uncached system message part',
    },
    {
      role: 'user',
      content: 'User prompt',
    },
  ],
});
```

关于 Anthropic 的提示缓存，更多见 [Google Vertex AI 的 Claude 提示缓存文档](https://cloud.google.com/vertex-ai/generative-ai/docs/partner-models/claude-prompt-caching) 与 [Anthropic 的缓存控制文档](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching)。

### 工具

Google Vertex Anthropic 支持部分 Anthropic 内置工具。可通过提供商实例的 `tools` 属性使用以下工具：

1. **Bash 工具**：允许运行 bash 命令。
2. **文本编辑器工具**：查看与编辑文本文件。
3. **Computer 工具**：控制计算机的键盘与鼠标操作。
4. **网页搜索工具**：访问实时网页内容。

<Note>
  Google Vertex 上仅支持 Anthropic 工具的子集。代码执行、Memory、Web Fetch 等工具不可用。若需要全部 Anthropic 工具，请使用常规
  `@ai-sdk/anthropic` 提供商。
</Note>

<Note>
  Google Vertex Anthropic 不支持工具定义的严格模式（strict）。
  在工具上设置 `strict: true` 会被忽略并发出警告。
</Note>

Anthropic 工具的背景说明见 [Anthropic 文档](https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview)。

#### Bash 工具

Bash 工具用于运行 bash 命令。创建与用法如下：

```ts
const bashTool = vertexAnthropic.tools.bash_20250124({
  execute: async ({ command, restart }) => {
    // Implement your bash command execution logic here
    // Return the result of the command execution
  },
});
```

参数：

- `command` (string)：要运行的 bash 命令。除非正在重启工具，否则为必填。
- `restart` (boolean，可选)：为 true 时将重启该工具。

#### 文本编辑器工具

文本编辑器工具用于查看与编辑文本文件：

```ts
const textEditorTool = vertexAnthropic.tools.textEditor_20250124({
  execute: async ({
    command,
    path,
    file_text,
    insert_line,
    new_str,
    insert_text,
    old_str,
    view_range,
  }) => {
    // Implement your text editing logic here
    // Return the result of the text editing operation
  },
});
```

参数：

- `command` ('view' | 'create' | 'str_replace' | 'insert' | 'undo_edit')：要执行的命令。注意：`textEditor_20250429` 与 `textEditor_20250728` 不支持 `undo_edit`。
- `path` (string)：文件或目录的绝对路径，例如 `/repo/file.py` 或 `/repo`。
- `file_text` (string，可选)：`create` 命令必填，为要创建文件的内容。
- `insert_line` (number，可选)：`insert` 命令必填，表示在其后插入新字符串的行号。
- `new_str` (string，可选)：`str_replace` 命令的新字符串。
- `insert_text` (string，可选)：`insert` 命令必填，为要插入的文本。
- `old_str` (string，可选)：`str_replace` 命令必填，为要被替换的字符串。
- `view_range` (number[]，可选)：`view` 命令可选，指定要显示的行范围。
- `max_characters` (number，可选)：查看文件时可显示的最大字符数（仅在 `textEditor_20250728` 中可用）。

#### Computer 工具

Computer 工具用于控制计算机的键盘与鼠标操作：

```ts
const computerTool = vertexAnthropic.tools.computer_20241022({
  displayWidthPx: 1920,
  displayHeightPx: 1080,
  displayNumber: 0, // Optional, for X11 environments

  execute: async ({ action, coordinate, text }) => {
    // Implement your computer control logic here
    // Return the result of the action

    // Example code:
    switch (action) {
      case 'screenshot': {
        // multipart result:
        return {
          type: 'image',
          data: fs
            .readFileSync('./data/screenshot-editor.png')
            .toString('base64'),
        };
      }
      default: {
        console.log('Action:', action);
        console.log('Coordinate:', coordinate);
        console.log('Text:', text);
        return `executed ${action}`;
      }
    }
  },

  // map to tool result content for LLM consumption:
  toModelOutput({ output }) {
    return typeof output === 'string'
      ? [{ type: 'text', text: output }]
      : [{ type: 'file-data', data: output.data, mediaType: 'image/png' }];
  },
});
```

参数：

- `action` ('key' | 'type' | 'mouse_move' | 'left_click' | 'left_click_drag' | 'right_click' | 'middle_click' | 'double_click' | 'screenshot' | 'cursor_position')：要执行的动作。
- `coordinate` (number[]，可选)：`mouse_move` 与 `left_click_drag` 动作必填，指定 (x, y) 坐标。
- `text` (string，可选)：`type` 与 `key` 动作必填。

#### 网页搜索工具

网页搜索工具让 Claude 可直接访问实时网页内容：

```ts
const webSearchTool = vertexAnthropic.tools.webSearch_20250305({
  maxUses: 5, // Optional: Maximum number of web searches Claude can perform
  allowedDomains: ['example.com'], // Optional: Only search these domains
  blockedDomains: ['spam.com'], // Optional: Never search these domains
  userLocation: {
    // Optional: Provide location for geographically relevant results
    type: 'approximate',
    city: 'San Francisco',
    region: 'CA',
    country: 'US',
    timezone: 'America/Los_Angeles',
  },
});
```

参数：

- `maxUses` (number，可选)：对话期间 Claude 可执行的网页搜索次数上限。
- `allowedDomains` (string[]，可选)：允许 Claude 搜索的域名列表。
- `blockedDomains` (string[]，可选)：搜索时应避免的域名列表。
- `userLocation` (object，可选)：用户位置信息，用于提供地理位置相关的搜索结果。
  - `type` ('approximate')：位置类型（必须为 approximate）。
  - `city` (string，可选)：城市名称。
  - `region` (string，可选)：地区或州。
  - `country` (string，可选)：国家。
  - `timezone` (string，可选)：IANA 时区 ID。

这些工具可与受支持的 Claude 模型配合，完成更复杂的交互与任务。

### 函数调用

Vertex 上的 Grok 模型支持兼容 OpenAI 的函数调用。可按常见方式使用 AI SDK 工具：

```ts

import { generateText, tool } from 'ai';

const result = await generateText({
```

## Google Vertex MaaS 提供商用法

Google Vertex MaaS（Model as a Service，模型即服务）提供商通过兼容 OpenAI 的 Chat Completions API，访问托管在 Vertex AI 上的合作与开放模型，其中包括 DeepSeek、Qwen、Meta、MiniMax、Moonshot 与 OpenAI 等模型。

更多信息见 [Vertex AI MaaS 文档](https://cloud.google.com/vertex-ai/generative-ai/docs/maas/use-open-models)。

### 提供商实例

可从 `@ai-sdk/google-vertex/maas` 导入默认提供商实例 `vertexMaas`：

```typescript
import { vertexMaas } from '@ai-sdk/google-vertex/maas';
```

若需自定义配置，可从 `@ai-sdk/google-vertex/maas` 导入 `createVertexMaas`，并按你的设置创建提供商实例：

```typescript
import { createVertexMaas } from '@ai-sdk/google-vertex/maas';

const vertexMaas = createVertexMaas({
  project: 'my-project', // optional
  location: 'us-east5', // optional, defaults to 'global'
});
```

#### Node.js 运行时

在 Node.js 环境中，Google Vertex MaaS 提供商通过 `google-auth-library` 支持所有标准 Google Cloud 身份验证选项：

```typescript
import { createVertexMaas } from '@ai-sdk/google-vertex/maas';

const vertexMaas = createVertexMaas({
  googleAuthOptions: {
    credentials: {
      client_email: 'my-email',
      private_key: 'my-private-key',
    },
  },
});
```

##### 可选提供商设置

- **project** _string_

  Google Cloud 项目 ID。默认读取 `GOOGLE_VERTEX_PROJECT` 环境变量。

- **location** _string_

  Google Cloud 区域，例如 `us-east5` 或 `global`。默认读取 `GOOGLE_VERTEX_LOCATION` 环境变量。若未设置，默认为 `global`。

- **googleAuthOptions** _object_

  可选。供 [Google Auth Library](https://github.com/googleapis/google-auth-library-nodejs/) 使用的身份验证选项。

- **headers** _Resolvable&lt;Record&lt;string, string | undefined&gt;&gt;_

  要包含在请求中的标头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
