# ElevenLabs 提供商

[ElevenLabs](https://elevenlabs.io/) 提供商对接 ElevenLabs 的语音转写与语音合成 API，提供对应的语言模型能力。

## 安装

ElevenLabs 提供商位于 `@ai-sdk/elevenlabs` 模块中，可使用以下命令安装：

## Provider 实例

可从 `@ai-sdk/elevenlabs` 导入默认的 `elevenLabs` 提供商实例：

```ts
import { elevenLabs } from '@ai-sdk/elevenlabs';
```

如需自定义配置，可导入 `createElevenLabs` 并创建实例：

```ts
import { createElevenLabs } from '@ai-sdk/elevenlabs';

const elevenLabs = createElevenLabs({
  // custom settings, e.g.
  fetch: customFetch,
});
```

可使用以下可选设置自定义 ElevenLabs 提供商实例：

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。
  默认为环境变量 `ELEVENLABS_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作拦截请求的中间件，
  或在测试中提供自定义 `fetch`。

## 语音模型

可使用 `.speech()` 工厂方法创建调用 [ElevenLabs speech API](https://elevenlabs.io/text-to-speech) 的模型。

首个参数为模型 ID，例如 `eleven_multilingual_v2`。

```ts
const model = elevenLabs.speech('eleven_multilingual_v2');
```

`voice` 可设为 [ElevenLabs Voice Library](https://elevenlabs.io/app/voice-library) 中的音色 ID，
在控制台选中音色后即可复制对应 ID。

```ts highlight="6"
import { experimental_generateSpeech as generateSpeech } from 'ai';
import { elevenLabs } from '@ai-sdk/elevenlabs';

const result = await generateSpeech({
  model: elevenLabs.speech('eleven_multilingual_v2'),
  text: 'Hello, world!',
  voice: '21m00Tcm4TlvDq8ikWAM', // Rachel voice
});
```

也可通过 `providerOptions` 传入提供商专有选项：

```ts highlight="7-9"
import { experimental_generateSpeech as generateSpeech } from 'ai';
import {
  elevenLabs,
  type ElevenLabsSpeechModelOptions,
} from '@ai-sdk/elevenlabs';

const result = await generateSpeech({
  model: elevenLabs.speech('eleven_multilingual_v2'),
  text: 'Hello, world!',
  voice: '21m00Tcm4TlvDq8ikWAM',
  providerOptions: {
    elevenlabs: {
      voiceSettings: {
        stability: 0.5,
        similarityBoost: 0.75,
      },
    } satisfies ElevenLabsSpeechModelOptions,
  },
});
```

- **languageCode** _string or null_
  可选。用于强制模型的语言代码（ISO 639-1）。目前仅 Turbo v2.5 与 Flash v2.5 支持；其他模型若传入将导致错误。

- **voiceSettings** _object or null_
  可选。覆盖所选音色的瞬时设置。
  - **stability** _double or null_
    可选。决定音色稳定性与单次生成之间的随机性；数值越低表现力越宽，越高越平直。
  - **useSpeakerBoost** _boolean or null_
    可选。提升与原始说话人的相似度，会增加计算与延迟。
  - **similarityBoost** _double or null_
    可选。控制在多大程度上贴合原始音色。
  - **style** _double or null_
    可选。放大原始音色风格强度；大于 0 可能增加延迟。

- **pronunciationDictionaryLocators** _array of objects or null_
  可选。按顺序应用的发音词典定位器列表，单次请求最多 3 个。
  每个对象：
  - **pronunciationDictionaryId** _string_（必填）：词典 ID。
  - **versionId** _string or null_（可选）：版本 ID；缺省则用最新版本。

- **seed** _integer or null_
  可选。若设定将尽量确定性采样；范围 0–4294967295；不保证完全可复现。

- **previousText** _string or null_
  可选。当前文本之前的上下文，可改善拼接生成时的连续性。

- **nextText** _string or null_
  可选。当前文本之后的上下文，可改善连续性。

- **previousRequestIds** _array of strings or null_
  可选。此前生成请求的 ID（最多 3 个）；若同时提供 `previousText`，则以本字段为准并忽略前者。

- **nextRequestIds** _array of strings or null_
  可选。后续生成请求的 ID（最多 3 个）；用于重生成时对齐上下文。若同时提供 `nextText`，则以本字段为准。

- **applyTextNormalization** _enum_
  可选。文本规范化策略。
  允许：`'auto'`（默认）、`'on'`、`'off'`。
  - `'auto'`：由系统决定是否规范化（例如数字拼读）。
  - `'on'`：始终规范化。
  - `'off'`：不进行规范化。
  对 `eleven_turbo_v2_5` 与 `eleven_flash_v2_5`，仅企业套餐可启用部分能力。

- **applyLanguageTextNormalization** _boolean_
  可选。默认 `false`。语言级规范化，有助于个别语言的发音（当前主要为日语）；可能明显增加延迟。

- **enableLogging** _boolean_
  可选。是否记录本次请求的日志；默认遵循账户全局设置。

## 转写模型

可使用 `.transcription()` 工厂方法创建调用 [ElevenLabs transcription API](https://elevenlabs.io/speech-to-text) 的模型。

首个参数为模型 ID，例如 `scribe_v1`。

```ts
const model = elevenLabs.transcription('scribe_v1');
```

也可通过 `providerOptions` 传入额外选项；若已知语言，可使用 ISO 639-1（如 `en`）有时可提升表现。

```ts highlight="6"
import { experimental_transcribe as transcribe } from 'ai';
import {
  elevenLabs,
  type ElevenLabsTranscriptionModelOptions,
} from '@ai-sdk/elevenlabs';

const result = await transcribe({
  model: elevenLabs.transcription('scribe_v1'),
  audio: new Uint8Array([1, 2, 3, 4]),
  providerOptions: {
    elevenlabs: {
      languageCode: 'en',
    } satisfies ElevenLabsTranscriptionModelOptions,
  },
});
```

可用的提供商选项：

- **languageCode** _string_

  对应音频语言的 ISO 639-1 或 ISO 639-3。
  若事先可知，有时可提升准确度。
  默认 `null` 表示自动判别。

- **tagAudioEvents** _boolean_

  是否在转写中加入 (laughter)、(footsteps) 等事件标签。
  默认为 `true`。

- **numSpeakers** _integer_

  音频中可能出现的最大说话人数，辅助预测轮转。
  最大可预测人数为 32。
  默认为 `null`（视作模型上限）。

- **timestampsGranularity** _enum_

  时间戳粒度。
  默认 `'word'`。
  取值：`'none'`、`'word'`、`'character'`。

- **diarize** _boolean_

  是否标注谁在说话。
  默认为 `true`。

- **fileFormat** _enum_

  输入音频格式。
  默认 `'other'`。
  取值：`pcm_s16le_16`、`other`。对 `'pcm_s16le_16'`，必须为 16kHz、16-bit PCM、单声道、little-endian，
  相对编码波形可降低延迟。
