# 可灵（Kling AI）提供商

[Kling AI](https://klingai.com/) 提供商支持可灵视频生成模型，包括文生视频、图生视频、动作控制与多分镜视频生成。

## 安装

Kling AI 提供商位于 `@ai-sdk/klingai` 模块中，可使用以下命令安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/klingai" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @ai-sdk/klingai" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/klingai" dark />
  </Tab>

  <Tab>
    <Snippet text="bun add @ai-sdk/klingai" dark />
  </Tab>
</Tabs>

## Provider 实例

可从 `@ai-sdk/klingai` 导入默认的 `klingai` 提供商实例：

```ts
import { klingai } from '@ai-sdk/klingai';
```

如需自定义配置，可导入 `createKlingAI` 并以你的设置创建提供商实例：

```ts
import { createKlingAI } from '@ai-sdk/klingai';

const klingai = createKlingAI({
  accessKey: 'your-access-key',
  secretKey: 'your-secret-key',
});
```

可使用以下可选设置自定义 Kling AI 提供商实例：

- **accessKey** _string_

  Kling AI Access Key。默认为环境变量 `KLINGAI_ACCESS_KEY`。

- **secretKey** _string_

  Kling AI Secret Key。默认为环境变量 `KLINGAI_SECRET_KEY`。

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀（例如经由代理）。
  默认前缀为 `https://api-singapore.klingai.com`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  可作为中间件拦截请求，
  或在测试中提供自定义 `fetch` 实现。

## 视频模型

可使用 `.video()` 工厂方法创建 Kling AI 视频模型。
详见 AI SDK 的 [generateVideo()](/docs/reference/ai-sdk-core/generate-video)。

当前提供商支持三种生成模式：文生视频、图生视频与动作控制。

<Note>
  并非所有选项都适用于每种模型版本与模式组合，详见 [KlingAI 能力对照表](https://app.klingai.com/global/dev/document-api/apiReference/model/skillsMap)。
</Note>

### 文生视频

由文本提示生成视频：

```ts
import { klingai, type KlingAIVideoModelOptions } from '@ai-sdk/klingai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { videos } = await generateVideo({
  model: klingai.video('kling-v2.6-t2v'),
  prompt: 'A chicken flying into the sunset in the style of 90s anime.',
  aspectRatio: '16:9',
  duration: 5,
  providerOptions: {
    klingai: {
      mode: 'std',
    } satisfies KlingAIVideoModelOptions,
  },
});
```

### 图生视频

由起始帧图像与可选文本提示生成视频。常用的「首帧+尾帧」能力通过 `imageTail` 选项提供：

```ts
import { klingai, type KlingAIVideoModelOptions } from '@ai-sdk/klingai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { videos } = await generateVideo({
  model: klingai.video('kling-v2.6-i2v'),
  prompt: {
    image: 'https://example.com/start-frame.png',
    text: 'The cat slowly turns its head and blinks',
  },
  duration: 5,
  providerOptions: {
    klingai: {
      // Pro mode required for start+end frame control
      mode: 'pro',
      // Optional: end frame image
      imageTail: 'https://example.com/end-frame.png',
    } satisfies KlingAIVideoModelOptions,
  },
});
```

### 多分镜视频生成

支持多分镜故事板，每镜可单独设置提示与时长（可灵 v3.0+）：

```ts
import { klingai, type KlingAIVideoModelOptions } from '@ai-sdk/klingai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { videos } = await generateVideo({
  model: klingai.video('kling-v3.0-t2v'),
  prompt: '',
  aspectRatio: '16:9',
  duration: 10,
  providerOptions: {
    klingai: {
      mode: 'pro',
      multiShot: true,
      shotType: 'customize',
      multiPrompt: [
        {
          index: 1,
          prompt: 'A sunrise over a calm ocean, warm golden light.',
          duration: '4',
        },
        {
          index: 2,
          prompt: 'A flock of seagulls take flight from the beach.',
          duration: '3',
        },
        {
          index: 3,
          prompt: 'Waves crash against rocky cliffs at sunset.',
          duration: '3',
        },
      ],
      sound: 'on',
    } satisfies KlingAIVideoModelOptions,
  },
});
```

多分镜也可与图生视频结合：在提供起始帧图像的同时为每镜单独写提示。

### 动作控制

将参考视频中的动作迁移到角色图像上，生成视频：

```ts
import { klingai, type KlingAIVideoModelOptions } from '@ai-sdk/klingai';
import { experimental_generateVideo as generateVideo } from 'ai';

const { videos } = await generateVideo({
  model: klingai.video('kling-v3.0-motion-control'),
  prompt: {
    image: 'https://example.com/character.png',
    text: 'The character performs a smooth dance move',
  },
  providerOptions: {
    klingai: {
      videoUrl: 'https://example.com/reference-motion.mp4',
      characterOrientation: 'image',
      mode: 'std',
      // Optional: reference element from element library (v3.0+, max 1)
      elementList: [{ element_id: 829836802793406551 }],
    } satisfies KlingAIVideoModelOptions,
  },
});
```

### 视频的提供商选项

可通过 `providerOptions.klingai` 使用以下提供商选项（因模式而异，各模型支持项见 [KlingAI 能力对照表](https://app.klingai.com/global/dev/document-api/apiReference/model/skillsMap)）。

#### 通用选项

- **mode** _'std' | 'pro'_

  视频生成模式。`'std'` 成本友好。`'pro'` 画质更高但需要更长时间。

- **pollIntervalMs** _number_

  轮询任务状态的间隔（毫秒）。默认为 5000。

- **pollTimeoutMs** _number_

  视频生成的最长等待时间（毫秒）。默认为 600000（10 分钟）。

- **watermarkEnabled** _boolean_

  是否同时生成含水印的结果。

#### 文生视频与图生视频选项

- **negativePrompt** _string_

  描述生成视频中应避免出现的内容（最多 2500 字符）。

- **sound** _'on' | 'off'_

  是否同步生成音频。仅 V2.6 及之后模型支持，且需要 `mode: 'pro'`。

- **cfgScale** _number_

  提示遵循强度；值越高越贴近提示词。取值范围：[0, 1]。V2.x 不支持。

- **cameraControl** _object_

  镜头运动控制：含预设 `type`（`'simple'`、`'down_back'`、`'forward_up'`、`'right_turn_forward'`、`'left_turn_forward'`）与可选 `config`（`horizontal`、`vertical`、`pan`、`tilt`、`roll`、`zoom`，范围 [-10, 10]）。

- **multiShot** _boolean_

  启用多分镜生成（可灵 v3.0+）。为 `true` 时，视频最多拆分为 6 个分镜，各有独立提示与时长。

- **shotType** _'customize' | 'intelligence'_

  多分镜分镜方式。`'customize'` 使用 `multiPrompt` 自定义分镜。`'intelligence'` 由模型根据主提示自动分段。`multiShot` 为 `true` 时必填。

- **multiPrompt** _Array&lt;\{index, prompt, duration\}&gt;_

  多分镜下各分镜详情：每项含 `index`（数字）、`prompt`（字符串，最多 512 字符）、`duration`（字符串，秒）。各分镜时长之和须等于总时长。`multiShot` 为 `true` 且 `shotType` 为 `'customize'` 时必填。

- **voiceList** _Array&lt;\{voice_id: string\}&gt;_

  音色控制用的音色引用（可灵 v3.0+）。最多 2 个音色。在提示中用 `<<<voice_1>>>` 等模板语法引用。需要 `sound: 'on'`。在 I2V 接口上与 `elementList` 互斥。

#### 仅图生视频选项

- **imageTail** _string_

  首尾帧控制中的尾帧图像。可为图像 URL 或原始 base64 数据。多数模型需要 `mode: 'pro'`。

- **staticMask** _string_

  运动笔刷的静态笔刷遮罩图。可为 URL 或 base64。

- **dynamicMasks** _Array_

  运动笔刷的动态配置。最多 6 组；每组包含 `mask`（URL 或 base64）与 `trajectories`（`{x, y}` 坐标数组）。

#### 图生视频与动作控制共用选项

- **elementList** _Array&lt;\{element_id: number\}&gt;_

  元素绑定用的引用元素（可灵 v3.0+）。支持视频角色元素与多图元素。I2V 最多 3 个（与 `voiceList` 互斥）；动作控制最多 1 个。

#### 仅动作控制选项

- **videoUrl** _string_（必填）

  参考动作视频的 URL。支持 .mp4/.mov，最大 100MB，时长 3–30 秒。

- **characterOrientation** _'image' | 'video'_（必填）

  生成视频中角色的朝向基准。
  `'image'`：与参考图朝向一致（视频最长 10s）。
  `'video'`：与参考视频朝向一致（视频最长 30s）。

- **keepOriginalSound** _'yes' | 'no'_

  是否保留参考视频的原始声音。默认为 `'yes'`。

<Note>
  视频生成是异步过程，可能需要数分钟。
  建议将 `pollTimeoutMs` 至少设为 10 分钟（600000 ms）以保证稳定。
</Note>

### 视频模型能力

#### 文生视频

| 模型                    | 说明                                           |
| ----------------------- | ---------------------------------------------- |
| `kling-v3.0-t2v`        | 最新 v3.0，多分镜、音色控制、音频，3–15s       |
| `kling-v2.6-t2v`        | V2.6，pro 模式下支持音频                       |
| `kling-v2.5-turbo-t2v`  | 侧重速度，支持 std / pro                       |
| `kling-v2.1-master-t2v` | 高画质，仅 pro                                 |
| `kling-v2-master-t2v`   | Master 级画质                                  |
| `kling-v1.6-t2v`        | V1.6，std / pro                                |
| `kling-v1-t2v`          | 初代 V1，std 下支持镜头控制                    |

#### 图生视频

| 模型                    | 说明                                                   |
| ----------------------- | ------------------------------------------------------ |
| `kling-v3.0-i2v`        | 最新 v3.0，多分镜、元素/音色控制、音频，3–15s         |
| `kling-v2.6-i2v`        | V2.6，pro 下支持音频与尾帧                           |
| `kling-v2.5-turbo-i2v`  | 侧重速度，pro 下支持尾帧                             |
| `kling-v2.1-master-i2v` | 高画质，仅 pro                                         |
| `kling-v2.1-i2v`        | V2.1，pro 下支持尾帧                                   |
| `kling-v2-master-i2v`   | Master 级画质                                        |
| `kling-v1.6-i2v`        | V1.6，pro 下支持尾帧                                   |
| `kling-v1.5-i2v`        | V1.5，pro 下支持尾帧与运动笔刷                       |
| `kling-v1-i2v`          | 初代 V1，std/pro 均支持尾帧与运动笔刷                |

#### 动作控制

| 模型                         | 说明                               |
| ---------------------------- | ---------------------------------- |
| `kling-v3.0-motion-control`  | 最新 v3.0，通过元素绑定强化面部一致性 |
| `kling-v2.6-motion-control`  | 将参考视频动作迁移到角色图像       |

<Note>
  如需也可直接传入任意可用的提供商模型 ID 字符串。
</Note>
