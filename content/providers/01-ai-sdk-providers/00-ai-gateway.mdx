# AI Gateway 提供商

[AI Gateway](https://vercel.com/docs/ai-gateway) 提供商通过一个统一接口为你接入来自多家 AI 提供商的模型。无需分别对接每家提供商，即可使用 OpenAI、Anthropic、Google、Meta、xAI 以及其他提供商及其模型。

## 功能特性

- 使用多家提供商的模型，而无需安装额外的提供商模块或依赖
- 在不同 AI 提供商之间保持相同的代码结构
- 轻松切换模型与提供商
- 部署在 Vercel 上时自动完成身份验证
- 查看各提供商的价格信息
- 通过 Vercel 控制台观测 AI 模型使用情况

## 安装配置

Vercel AI Gateway 提供商包含在 AI SDK 中。

## 基本用法

在大多数场景下，可以直接使用模型字符串调用 AI Gateway：

```ts
// use plain model string with global provider
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'openai/gpt-5.4',
  prompt: 'Hello world',
});
```

```ts
// use provider instance (requires version 5.0.36 or later)
import { generateText, gateway } from 'ai';

const { text } = await generateText({
  model: gateway('openai/gpt-5.4'),
  prompt: 'Hello world',
});
```

当你以 `creator/model-name` 格式传入模型字符串时，AI SDK 会自动使用 AI Gateway。

## 提供商实例

<Note>
  `gateway` 提供商实例自 `ai` 包的 5.0.36 版本起提供。
</Note>

你也可以从 `ai` 导入默认提供商实例 `gateway`：

```ts
import { gateway } from 'ai';
```

在以下场景中，你可能需要创建自定义提供商实例：

- 设置自定义配置项（API 密钥、基础 URL、请求头）
- 在 [提供商注册表](/docs/ai-sdk-core/provider-management) 中使用该提供商
- 使用 [中间件](/docs/ai-sdk-core/middleware) 包装提供商
- 在应用的不同部分使用不同设置

若要创建自定义提供商实例，请从 `ai` 导入 `createGateway`：

```ts
import { createGateway } from 'ai';

const gateway = createGateway({
  apiKey: process.env.AI_GATEWAY_API_KEY ?? '',
});
```

可使用下列可选设置自定义 AI Gateway 提供商实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀。默认前缀为 `https://ai-gateway.vercel.sh/v4/ai`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API 密钥。默认读取环境变量 `AI_GATEWAY_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要附加的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作拦截请求的中间件，
  或在测试等场景提供自定义 fetch。

- **metadataCacheRefreshMillis** _number_

  元数据缓存刷新间隔（毫秒）。默认为 5 分钟（300,000ms）。

## 身份验证

Gateway 提供商支持两种身份验证方式：

### API 密钥身份验证

通过环境变量设置 API 密钥：

```bash
AI_GATEWAY_API_KEY=your_api_key_here
```

或直接传给提供商：

```ts
import { createGateway } from 'ai';

const gateway = createGateway({
  apiKey: 'your_api_key_here',
});
```

### OIDC 身份验证（Vercel 部署）

部署到 Vercel 时，AI Gateway 提供商支持使用 [OIDC（OpenID Connect）令牌](https://vercel.com/docs/oidc) 进行身份验证，而无需 API 密钥。

#### OIDC 身份验证如何工作

1. **在生产/预览部署中**：
   - OIDC 身份验证会自动处理
   - 无需手动配置
   - 令牌会自动获取并刷新

2. **在本地开发时**：
   - 首先安装并完成 [Vercel CLI](https://vercel.com/docs/cli) 的身份验证
   - 运行 `vercel env pull` 将项目的 OIDC 令牌下载到本地
   - 若需自动管理令牌：
     - 使用 `vercel dev` 启动开发服务器，它会自动处理令牌刷新
   - 若需手动管理令牌：
     - 若不使用 `vercel dev`，请注意 OIDC 令牌会在 12 小时后过期
     - 需要在过期前再次运行 `vercel env pull` 以刷新令牌

<Note>
  若存在 API 密钥（直接传入或通过环境变量），则
  始终会优先使用该密钥，即便密钥无效也是如此。
</Note>

关于 OIDC 令牌的更多说明，请参阅 [Vercel AI Gateway 文档](https://vercel.com/docs/ai-gateway#using-the-ai-gateway-with-a-vercel-oidc-token)。

## 自带密钥（BYOK）

你可以接入自己的提供商凭证，与 Vercel AI Gateway 一起使用。这样可使用既有提供商账号并访问私有资源。

要完成 BYOK 配置，请在 Vercel 团队的 AI Gateway 设置中添加提供商凭证。配置完成后，AI Gateway 会自动使用你的凭证，无需修改代码。

对于 Azure 等可使用自定义部署名称的提供商，你可以配置模型映射，将网关模型 slug 映射到你的部署名称。详情请参阅 [模型映射](https://vercel.com/docs/ai-gateway/byok#model-mappings)。

更多信息见 [BYOK 文档](https://vercel.com/docs/ai-gateway/byok)。

## 语言模型

可使用提供商实例创建语言模型。第一个参数为 `creator/model-name` 格式的模型 ID：

```ts
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'openai/gpt-5.4',
  prompt: 'Explain quantum computing in simple terms',
});
```

AI Gateway 语言模型也可用于 `streamText`，并支持结合 [`Output`](/docs/reference/ai-sdk-core/output) 进行结构化数据生成（参见 [AI SDK Core](/docs/ai-sdk-core)）。

## 重排序模型

可使用提供商实例上的 `rerankingModel` 方法创建重排序模型：

```ts
import { rerank } from 'ai';
import { gateway } from '@ai-sdk/gateway';

const { ranking } = await rerank({
  model: gateway.rerankingModel('cohere/rerank-v3.5'),
  query: 'What is the capital of France?',
  documents: [
    'Paris is the capital of France.',
    'Berlin is the capital of Germany.',
    'Madrid is the capital of Spain.',
  ],
  topN: 2,
});

console.log(ranking);
// [
//   { originalIndex: 0, score: 0.89, document: 'Paris is the capital of France.' },
//   { originalIndex: 2, score: 0.15, document: 'Madrid is the capital of Spain.' },
// ]
```

重排序模型适用于在检索增强生成（RAG）流程中，在初次检索后对候选文档重新打分，从而改进搜索结果。

## 动态模型发现

可通过编程方式发现可用模型：

```ts
import { gateway, generateText } from 'ai';

const availableModels = await gateway.getAvailableModels();

// List all available models
availableModels.models.forEach(model => {
  console.log(`${model.id}: ${model.name}`);
  if (model.description) {
    console.log(`  Description: ${model.description}`);
  }
  if (model.pricing) {
    console.log(`  Input: $${model.pricing.input}/token`);
    console.log(`  Output: $${model.pricing.output}/token`);
    if (model.pricing.cachedInputTokens) {
      console.log(
        `  Cached input (read): $${model.pricing.cachedInputTokens}/token`,
      );
    }
    if (model.pricing.cacheCreationInputTokens) {
      console.log(
        `  Cache creation (write): $${model.pricing.cacheCreationInputTokens}/token`,
      );
    }
  }
});

// Use any discovered model with plain string
const { text } = await generateText({
  model: availableModels.models[0].id, // e.g., 'openai/gpt-5.4'
  prompt: 'Hello world',
});
```

## 额度使用情况

可查询团队当前的额度余额与使用量：

```ts
import { gateway } from 'ai';

const credits = await gateway.getCredits();

console.log(`Team balance: ${credits.balance} credits`);
console.log(`Team total used: ${credits.total_used} credits`);
```

`getCredits()` 方法根据已认证的 API 密钥或 OIDC 令牌返回团队的额度信息：

- **balance** _number_ — 团队当前可用额度余额
- **total_used** _number_ — 团队已累计消耗的额度

## 生成记录查询

可按生成 ID 查询某次生成的详细信息，包括费用、令牌用量、延迟与提供商详情。在 `generateText` 与 `streamText` 的响应中，生成 ID 位于 `providerMetadata.gateway.generationId`。

流式输出时，生成 ID 会在首个内容分片中注入，因此可在流早期捕获，而无需等待结束。这在网络中断或流中途出错导致无法收到最终响应时尤其有用——由于网关在服务端记录最终状态，你可稍后通过 `getGenerationInfo()` 根据生成 ID 查询结果（包括费用、令牌用量与结束原因）。

```ts
import { gateway, generateText } from 'ai';

// Make a request
const result = await generateText({
  model: gateway('anthropic/claude-sonnet-4'),
  prompt: 'Explain quantum entanglement briefly',
});

// Get the generation ID from provider metadata
const generationId = result.providerMetadata?.gateway?.generationId;

// Look up detailed generation info
const generation = await gateway.getGenerationInfo({ id: generationId });

console.log(`Model: ${generation.model}`);
console.log(`Cost: $${generation.totalCost.toFixed(6)}`);
console.log(`Latency: ${generation.latency}ms`);
console.log(`Prompt tokens: ${generation.promptTokens}`);
console.log(`Completion tokens: ${generation.completionTokens}`);
```

使用 `streamText` 时，可通过 `fullStream` 从第一个分片捕获生成 ID：

```ts
import { gateway, streamText } from 'ai';

const result = streamText({
  model: gateway('anthropic/claude-sonnet-4'),
  prompt: 'Explain quantum entanglement briefly',
});

let generationId: string | undefined;

for await (const part of result.fullStream) {
  if (!generationId && part.providerMetadata?.gateway?.generationId) {
    generationId = part.providerMetadata.gateway.generationId as string;
    console.log(`Generation ID (early): ${generationId}`);
  }
}

// Look up cost and usage after the stream completes
if (generationId) {
  const generation = await gateway.getGenerationInfo({ id: generationId });
  console.log(`Cost: $${generation.totalCost.toFixed(6)}`);
  console.log(`Finish reason: ${generation.finishReason}`);
}
```

`getGenerationInfo()` 方法接受：

- **id** _string_ — 要查询的生成 ID（格式：`gen_<ulid>`，必填）

返回的 `GatewayGenerationInfo` 对象包含下列字段：

- **id** _string_ — 生成 ID
- **totalCost** _number_ — 总费用（美元）
- **upstreamInferenceCost** _number_ — 上游推理费用（美元，与 BYOK 相关）
- **usage** _number_ — 使用费用（美元，与 totalCost 相同）
- **createdAt** _string_ — 生成创建时间的 ISO 8601 时间戳
- **model** _string_ — 使用的模型标识符
- **isByok** _boolean_ — 是否使用了自带密钥（BYOK）凭证
- **providerName** _string_ — 实际提供本次生成的提供商
- **streamed** _boolean_ — 是否使用流式输出
- **finishReason** _string_ — 结束原因（例如 `'stop'`）
- **latency** _number_ — 首 token 时间（毫秒）
- **generationTime** _number_ — 总生成时间（毫秒）
- **promptTokens** _number_ — 提示令牌数
- **completionTokens** _number_ — 补全令牌数
- **reasoningTokens** _number_ — 推理令牌数（若适用）
- **cachedTokens** _number_ — 缓存令牌数（若适用）
- **cacheCreationTokens** _number_ — 缓存创建输入令牌数
- **billableWebSearchCalls** _number_ — 可计费的网页搜索调用次数

### 由提供商执行的工具

部分提供商提供由提供商自身执行的工具，例如 [OpenAI 的网页搜索工具](/providers/ai-sdk-providers/openai#web-search-tool)。若要通过 AI Gateway 使用这些工具，请导入对应提供商以获取工具定义：

```ts
import { generateText, isStepCount } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: 'openai/gpt-5.4-mini',
  prompt: 'What is the Vercel AI Gateway?',
  stopWhen: isStepCount(10),
  tools: {
    web_search: openai.tools.webSearch({}),
  },
});

console.dir(result.text);
```

<Note>
  部分由提供商执行的工具需要账号级配置（例如
  Claude Agent Skills），可能无法通过 AI Gateway 使用。要使用这些工具，
  必须将自带密钥（BYOK）直接配置给对应提供商。
</Note>

### Gateway 工具

AI Gateway 提供商包含由网关自身执行的内置工具。这些工具可与任意通过网关调用的模型配合使用。

#### Perplexity Search

Perplexity Search 工具使模型能够通过 [Perplexity 搜索 API](https://docs.perplexity.ai/guides/search-quickstart) 搜索网页。该工具由 AI Gateway 执行，返回可供模型使用的网页搜索结果，以提供最新信息。

```ts
import { gateway, generateText } from 'ai';

const result = await generateText({
  model: 'openai/gpt-5.4-nano',
  prompt: 'Search for news about AI regulations in January 2025.',
  tools: {
    perplexity_search: gateway.tools.perplexitySearch(),
  },
});

console.log(result.text);
console.log('Tool calls:', JSON.stringify(result.toolCalls, null, 2));
console.log('Tool results:', JSON.stringify(result.toolResults, null, 2));
```

也可以使用可选参数配置搜索：

```ts
import { gateway, generateText } from 'ai';

const result = await generateText({
  model: 'openai/gpt-5.4-nano',
  prompt:
    'Search for news about AI regulations from the first week of January 2025.',
  tools: {
    perplexity_search: gateway.tools.perplexitySearch({
      maxResults: 5,
      searchLanguageFilter: ['en'],
      country: 'US',
      searchDomainFilter: ['reuters.com', 'bbc.com', 'nytimes.com'],
    }),
  },
});

console.log(result.text);
console.log('Tool calls:', JSON.stringify(result.toolCalls, null, 2));
console.log('Tool results:', JSON.stringify(result.toolResults, null, 2));
```

Perplexity Search 工具支持下列可选配置：

- **maxResults** _number_

  返回的最大搜索结果数量（1–20，默认：10）。

- **maxTokensPerPage** _number_

  每个搜索结果页面提取的最大令牌数（256–2048，默认：2048）。

- **maxTokens** _number_

  所有搜索结果合计的最大令牌数（默认：25000，最大：1000000）。

- **searchLanguageFilter** _string[]_

  使用 ISO 639-1 语言代码按语言过滤结果（例如 `['en']` 表示英语，`['en', 'es']` 表示英语与西班牙语）。

- **country** _string_

  使用 ISO 3166-1 alpha-2 国家代码按国家过滤结果（例如 `'US'` 表示美国，`'GB'` 表示英国）。

- **searchDomainFilter** _string[]_

  将搜索结果限定在特定域名（例如 `['reuters.com', 'bbc.com']`）。适用于将结果限制在可信来源。

- **searchRecencyFilter** _'day' | 'week' | 'month' | 'year'_

  按相对时间段过滤搜索结果。适用于始终获取较新结果（例如 `'week'` 表示最近一周）。

该工具同时适用于 `generateText` 与 `streamText`：

```ts
import { gateway, streamText } from 'ai';

const result = streamText({
  model: 'openai/gpt-5.4-nano',
  prompt: 'Search for the latest news about AI regulations.',
  tools: {
    perplexity_search: gateway.tools.perplexitySearch(),
  },
});

for await (const part of result.fullStream) {
  switch (part.type) {
    case 'text-delta':
      process.stdout.write(part.text);
      break;
    case 'tool-call':
      console.log('\nTool call:', JSON.stringify(part, null, 2));
      break;
    case 'tool-result':
      console.log('\nTool result:', JSON.stringify(part, null, 2));
      break;
  }
}
```

#### Parallel Search

Parallel Search 工具使模型能够通过 [Parallel AI 搜索 API](https://docs.parallel.ai/api-reference/search-beta/search) 搜索网页。该工具针对 LLM 消费场景优化，返回网页中的相关摘录，可用单次调用替代多次关键词搜索。

```ts
import { gateway, generateText } from 'ai';

const result = await generateText({
  model: 'openai/gpt-5.4-nano',
  prompt: 'Research the latest developments in quantum computing.',
  tools: {
    parallel_search: gateway.tools.parallelSearch(),
  },
});

console.log(result.text);
console.log('Tool calls:', JSON.stringify(result.toolCalls, null, 2));
console.log('Tool results:', JSON.stringify(result.toolResults, null, 2));
```

也可以使用可选参数配置搜索：

```ts
import { gateway, generateText } from 'ai';

const result = await generateText({
  model: 'openai/gpt-5.4-nano',
  prompt: 'Find detailed information about TypeScript 5.0 features.',
  tools: {
    parallel_search: gateway.tools.parallelSearch({
      mode: 'agentic',
      maxResults: 5,
      sourcePolicy: {
        includeDomains: ['typescriptlang.org', 'github.com'],
      },
      excerpts: {
        maxCharsPerResult: 8000,
      },
    }),
  },
});

console.log(result.text);
console.log('Tool calls:', JSON.stringify(result.toolCalls, null, 2));
console.log('Tool results:', JSON.stringify(result.toolResults, null, 2));
```

Parallel Search 工具支持下列可选配置：

- **mode** _'one-shot' | 'agentic'_

  针对不同用场景的预设模式：
  - `'one-shot'` — 单次回答场景下的全面结果与更长摘录（默认）
  - `'agentic'` — 面向多步智能体工作流的精简、省令牌结果

- **maxResults** _number_

  返回的最大结果数量（1–20）。未指定时默认为 10。

- **sourcePolicy** _object_

  控制包含或排除哪些域名的来源策略：
  - `includeDomains` — 搜索结果中要包含的域名列表
  - `excludeDomains` — 搜索结果中要排除的域名列表
  - `afterDate` — 仅包含该日期之后发布的结果（ISO 8601 格式）

- **excerpts** _object_

  控制结果长度的摘录配置：
  - `maxCharsPerResult` — 每个结果的最大字符数
  - `maxCharsTotal` — 所有结果合计的最大字符数

- **fetchPolicy** _object_

  控制内容新鲜度的抓取策略：
  - `maxAgeSeconds` — 缓存内容的最大存活秒数（设为 0 表示始终获取最新）

该工具同时适用于 `generateText` 与 `streamText`：

```ts
import { gateway, streamText } from 'ai';

const result = streamText({
  model: 'openai/gpt-5.4-nano',
  prompt: 'Research the latest AI safety guidelines.',
  tools: {
    parallel_search: gateway.tools.parallelSearch(),
  },
});

for await (const part of result.fullStream) {
  switch (part.type) {
    case 'text-delta':
      process.stdout.write(part.text);
      break;
    case 'tool-call':
      console.log('\nTool call:', JSON.stringify(part, null, 2));
      break;
    case 'tool-result':
      console.log('\nTool result:', JSON.stringify(part, null, 2));
      break;
  }
}
```

### 自定义报表

按终端用户跟踪使用量并使用标签归类请求，然后通过报表 API 查询数据。

#### 使用 user 与 tags 跟踪使用量

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'openai/gpt-5.4',
  prompt: 'Summarize this document...',
  providerOptions: {
    gateway: {
      user: 'user-abc-123', // Track usage for this specific end-user
      tags: ['document-summary', 'premium-feature'], // Categorize for reporting
    } satisfies GatewayProviderOptions,
  },
});
```

这样你可以：

- 在分析中按终端用户查看使用量与费用拆分
- 使用标签按功能或用例筛选与分析支出
- 跟踪哪些用户或功能带来了最多的 AI 使用量

#### 查询支出报表

使用 `getSpendReport()` 方法以编程方式查询使用数据。报表 API 仅适用于 Vercel Pro 与 Enterprise 套餐。定价请参阅 [自定义报表文档](https://vercel.com/docs/ai-gateway/capabilities/custom-reporting)。

```ts
import { gateway } from 'ai';

const report = await gateway.getSpendReport({
  startDate: '2026-03-01',
  endDate: '2026-03-25',
  groupBy: 'model',
});

for (const row of report.results) {
  console.log(`${row.model}: $${row.totalCost.toFixed(4)}`);
}
```

`getSpendReport()` 方法接受下列参数：

- **startDate** _string_ — 开始日期，`YYYY-MM-DD` 格式（含当日，必填）
- **endDate** _string_ — 结束日期，`YYYY-MM-DD` 格式（含当日，必填）
- **groupBy** _string_ — 聚合维度：`'day'`（默认）、`'user'`、`'model'`、`'tag'`、`'provider'` 或 `'credential_type'`
- **datePart** _string_ — 当 `groupBy` 为 `'day'` 时的时间粒度：`'day'` 或 `'hour'`
- **userId** _string_ — 仅筛选特定用户
- **model** _string_ — 仅筛选特定模型（例如 `'anthropic/claude-sonnet-4.5'`）
- **provider** _string_ — 仅筛选特定提供商（例如 `'anthropic'`）
- **credentialType** _string_ — 按 `'byok'` 或 `'system'` 凭证筛选
- **tags** _string[]_ — 仅筛选匹配这些标签的请求

`results` 中每一行包含与所选 `groupBy` 对应的归类字段及指标：

- **totalCost** _number_ — 总费用（美元）
- **marketCost** _number_ — 市场价格（美元）
- **inputTokens** _number_ — 输入令牌数
- **outputTokens** _number_ — 输出令牌数
- **cachedInputTokens** _number_ — 缓存输入令牌数
- **cacheCreationInputTokens** _number_ — 缓存创建输入令牌数
- **reasoningTokens** _number_ — 推理令牌数
- **requestCount** _number_ — 请求次数

可将跟踪与查询结合，分析你所定义标签下的支出：

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { gateway, streamText } from 'ai';

// 1. Make requests with tags
const result = streamText({
  model: gateway('anthropic/claude-haiku-4.5'),
  prompt: 'Summarize this quarter's results',
  providerOptions: {
    gateway: {
      tags: ['team:finance', 'feature:summaries'],
    } satisfies GatewayProviderOptions,
  },
});

// 2. Later, query spend filtered by those tags
const report = await gateway.getSpendReport({
  startDate: '2026-03-01',
  endDate: '2026-03-31',
  groupBy: 'tag',
  tags: ['team:finance'],
});

for (const row of report.results) {
  console.log(`${row.tag}: $${row.totalCost.toFixed(4)} (${row.requestCount} requests)`);
}
```

## 提供商选项

AI Gateway 提供商接受用于控制路由行为与提供商专属配置的选项。

### Gateway 提供商选项

可在 `providerOptions` 中使用 `gateway` 键控制 AI Gateway 如何路由请求：

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.6',
  prompt: 'Explain quantum computing',
  providerOptions: {
    gateway: {
      order: ['vertex', 'anthropic'], // Try Vertex AI first, then Anthropic
      only: ['vertex', 'anthropic'], // Only use these providers
    } satisfies GatewayProviderOptions,
  },
});
```

可用的 gateway 提供商选项如下：

- **order** _string[]_

  指定路由请求时尝试提供商的顺序。网关按所列顺序依次尝试。若某提供商失败或不可用，将转到列表中的下一个。

  示例：`order: ['bedrock', 'anthropic']` 会先尝试 Amazon Bedrock，再回退到 Anthropic。

- **only** _string[]_

  将路由限制为仅所列提供商。设置后，网关绝不会路由到列表外的提供商，即使它们原本可用。

  示例：`only: ['anthropic', 'vertex']` 仅允许路由到 Anthropic 或 Vertex AI。

- **sort** _'cost' | 'ttft' | 'tps'_

  在路由前按性能或成本指标对可用提供商排序。网关会先尝试得分最优的提供商，再按排序依次回退。若未指定，则使用网关默认的系统排序。
  - `'cost'` — 成本最低优先
  - `'ttft'` — 首 token 时间最短优先
  - `'tps'` — 每秒令牌数最高优先

  与 `order` 联用时，用户指定的提供商会提升到前排，其余提供商仍遵循排序顺序。

  示例：`sort: 'ttft'` 会优先路由到首 token 最快的提供商。

  启用 `sort` 时，响应中的 `providerMetadata.gateway.routing.sort` 对象会包含所用排序选项、最终执行顺序、各提供商指标值以及被降权的提供商。

- **models** _string[]_

  指定主模型失败或不可用时的回退模型。网关会先尝试 `model` 参数中的主模型，再按数组顺序依次尝试，直至成功。

  示例：`models: ['openai/gpt-5.4-nano', 'gemini-3-flash-preview']` 会在主模型失败时按顺序尝试这些回退模型。

- **user** _string_

  可选的终端用户标识，表示请求所代表的用户。用于支出跟踪与归因，便于在应用中按终端用户统计使用量。

  示例：`user: 'user-123'` 会将该请求关联到使用量报表中的终端用户 ID「user-123」。

- **tags** _string[]_

  可选的标签数组，用于在报表中归类与筛选使用量。适用于按功能、提示版本或应用中其他维度跟踪支出。

  示例：`tags: ['chat', 'v2']` 会为该请求打上「chat」与「v2」标签，便于在使用量分析中筛选。

- **byok** _Record&lt;string, Array&lt;Record&lt;string, unknown&gt;&gt;&gt;_

  请求级 BYOK（自带密钥）凭证。提供时，网关在网关中配置的缓存 BYOK 凭证将不会被考虑。若所提供的凭证失败，请求仍可能回退使用系统凭证。

  每个提供商可有多组凭证（按顺序尝试）。结构为：键为提供商 slug，值为凭证对象数组。

  每个凭证可选包含 `modelMappings` 数组，用于将 AI Gateway 模型 slug 映射到你的部署名称（例如自定义 Azure 部署名）。若 BYOK 请求失败，网关会使用默认模型名回退到系统凭证。

  示例：
  - 单一提供商：`byok: { 'anthropic': [{ apiKey: 'sk-ant-...' }] }`
  - 多组凭证：`byok: { 'vertex': [{ project: 'proj-1', googleCredentials: { privateKey: '...', clientEmail: '...' } }, { project: 'proj-2', googleCredentials: { privateKey: '...', clientEmail: '...' } }] }`
  - 多个提供商：`byok: { 'anthropic': [{ apiKey: '...' }], 'bedrock': [{ accessKeyId: '...', secretAccessKey: '...' }] }`
  - 含模型映射：`byok: { 'azure': [{ apiKey: '...', resourceName: '...', modelMappings: [{ gatewayModelSlug: 'openai/gpt-5.4-nano', customModelId: 'my-deployment' }] }] }`

- **zeroDataRetention** _boolean_

  将路由限制为与 Vercel 就 AI Gateway 签订了零数据保留协议的提供商。使用 BYOK 凭证时不应用此筛选。若 BYOK 失败并回退到系统凭证，仅使用签订了零数据保留协议的提供商。若没有符合零数据保留的提供商可用，请求将失败。请求级 ZDR 仅适用于 Vercel Pro 与 Enterprise 套餐。

- **disallowPromptTraining** _boolean_

  将路由限制为与 Vercel 就 AI Gateway 约定不使用提示进行模型训练的提供商。使用 BYOK 凭证时不应用此筛选。若 BYOK 失败并回退到系统凭证，仅使用不对提示数据训练的提供商。若没有符合「禁止提示训练」的提供商可用，请求将失败。

- **hipaaCompliant** _boolean_

  将路由限制为已与 Vercel 签署 BAA、可用于 AI Gateway 的提供商所提供的模型与工具（需要 Vercel HIPAA BAA 附加服务）。当 `hipaaCompliant` 设为 `true` 时会跳过 BYOK 凭证，以确保请求仅路由到支持 HIPAA 合规的提供商。

- **quotaEntityId** _string_

  用于计量与强制配额所针对实体的唯一标识。用于配额管理与强制执行。

- **providerTimeouts** _object_

  BYOK 凭证下各提供商的超时时间（毫秒）。控制等待某提供商开始响应的最长时间，超时后将回退到下一个可用提供商。

  示例：`providerTimeouts: { byok: { openai: 5000, anthropic: 2000 } }`

  完整说明见 [提供商超时](https://vercel.com/docs/ai-gateway/models-and-providers/provider-timeouts)。

可组合这些选项，精细控制路由与跟踪：

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.6',
  prompt: 'Write a haiku about programming',
  providerOptions: {
    gateway: {
      order: ['vertex'], // Prefer Vertex AI
      only: ['anthropic', 'vertex'], // Only allow these providers
    } satisfies GatewayProviderOptions,
  },
});
```

#### 模型回退示例

`models` 选项可在主模型失败时自动回退到备选模型：

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'openai/gpt-5.4', // Primary model
  prompt: 'Write a TypeScript haiku',
  providerOptions: {
    gateway: {
      models: ['openai/gpt-5.4-nano', 'gemini-3-flash-preview'], // Fallback models
    } satisfies GatewayProviderOptions,
  },
});

// This will:
// 1. Try openai/gpt-5.4 first
// 2. If it fails, try openai/gpt-5.4-nano
// 3. If that fails, try gemini-3-flash-preview
// 4. Return the result from the first model that succeeds
```

#### 零数据保留示例

将 `zeroDataRetention` 设为 true，可将请求路由到与 Vercel 就 AI Gateway 签订了零数据保留协议的提供商。使用 BYOK 凭证时不应用此筛选。若 BYOK 失败并回退到系统凭证，仅使用签订了零数据保留协议的提供商。若没有符合零数据保留的提供商可用，请求将失败。当 `zeroDataRetention` 为 `false` 或未指定时，不会强制限制路由。请求级 ZDR 仅适用于 Vercel Pro 与 Enterprise 套餐。

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.6',
  prompt: 'Analyze this sensitive document...',
  providerOptions: {
    gateway: {
      zeroDataRetention: true,
    } satisfies GatewayProviderOptions,
  },
});
```

#### HIPAA 合规示例

将 `hipaaCompliant` 设为 true，可将请求仅路由到已与 Vercel 签署 BAA、可用于 AI Gateway 的提供商所提供的模型或工具。若模型或工具没有符合 HIPAA 的提供商，请求将失败。当 `hipaaCompliant` 为 `false` 或未指定时，不会强制限制路由。当 `hipaaCompliant` 设为 `true` 时会跳过 BYOK 凭证，以确保请求仅路由到支持 HIPAA 合规的提供商。

```ts
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.6',
  prompt: 'Analyze this patient data...',
  providerOptions: {
    gateway: {
      hipaaCompliant: true,
    } satisfies GatewayProviderOptions,
  },
});
```


### 提供商专属选项

通过 AI Gateway 使用提供商专属选项时，请使用实际提供商名称（例如 `anthropic`、`openai`，而非 `gateway`）作为键：

```ts
import type { AnthropicLanguageModelOptions } from '@ai-sdk/anthropic';
import type { GatewayProviderOptions } from '@ai-sdk/gateway';
import { generateText } from 'ai';

const { text } = await generateText({
  model: 'anthropic/claude-sonnet-4.6',
  prompt: 'Explain quantum computing',
  providerOptions: {
    gateway: {
      order: ['vertex', 'anthropic'],
    } satisfies GatewayProviderOptions,
    anthropic: {
      thinking: { type: 'enabled', budgetTokens: 12000 },
    } satisfies AnthropicLanguageModelOptions,
  },
});
```

这与 AI Gateway 支持的任意提供商均可配合使用。各提供商有各自的选项集——提供商专属选项详情请参阅各 [提供商文档页面](/providers/ai-sdk-providers)。
