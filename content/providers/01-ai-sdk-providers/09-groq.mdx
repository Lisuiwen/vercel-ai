# Groq 提供商

[Groq](https://groq.com/) 提供商为 Groq API 提供语言模型支持。

## 安装

Groq provider 通过 `@ai-sdk/groq` 模块提供。
你可以使用以下命令安装：

## 提供商实例

你可以从 `@ai-sdk/groq` 导入默认的 provider 实例 `groq`：

```ts
import { groq } from '@ai-sdk/groq';
```

如需自定义配置，可从 `@ai-sdk/groq` 导入 `createGroq`，并用你的设置创建 provider 实例：

```ts
import { createGroq } from '@ai-sdk/groq';

const groq = createGroq({
  // 自定义设置
});
```

你可以使用以下可选设置来自定义 Groq provider 实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀，例如用于代理服务器。
  默认前缀为 `https://api.groq.com/openai/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API 密钥。
  默认读取环境变量 `GROQ_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  要在请求中包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch 实现。

## 语言模型

你可以使用 provider 实例创建 [Groq 模型](https://console.groq.com/docs/models)。
第一个参数为模型 id，例如 `gemma2-9b-it`。

```ts
const model = groq('gemma2-9b-it');
```

### 推理模型

Groq 提供多种推理模型，例如 `qwen-qwq-32b` 和 `deepseek-r1-distill-llama-70b`。
可通过 `reasoningFormat` 选项配置推理内容在生成文本中的呈现方式。
支持 `parsed`、`hidden` 和 `raw`。

```ts
import { groq, type GroqLanguageModelChatOptions } from '@ai-sdk/groq';
import { generateText } from 'ai';

const result = await generateText({
  model: groq('qwen/qwen3-32b'),
  providerOptions: {
    groq: {
      reasoningFormat: 'parsed',
      reasoningEffort: 'default',
      parallelToolCalls: true, // 启用并行函数调用（默认：true）
      user: 'user-123', // 终端用户唯一标识（可选）
      serviceTier: 'flex', // 使用 flex 层级以获得更高吞吐（可选）
    } satisfies GroqLanguageModelChatOptions,
  },
  prompt: '"strawberry" 这个词里有多少个字母 "r"？',
});
```

Groq 语言模型还支持以下可选 provider 选项：

- **reasoningFormat** _'parsed' | 'raw' | 'hidden'_

  控制推理内容在生成文本中的暴露方式。仅 reasoning 类模型（如 `qwen-qwq-32b` 与 `deepseek-r1-distill-*`）支持。

  推理模型及其能力的完整列表见 [Groq reasoning 模型文档](https://console.groq.com/docs/reasoning)。

- **reasoningEffort** _'low' | 'medium' | 'high' | 'none' | 'default'_

  控制模型投入推理的程度。
  - `qwen/qwen3-32b`
    - 可选值：
      - `none`：关闭推理，模型不使用任何推理 token。
      - `default`：开启推理。
  - `gpt-oss20b/gpt-oss120b`
    - 可选值：
      - `low`：较低推理强度。
      - `medium`：中等推理强度。
      - `high`：较高推理强度。

  对 `qwen/qwen3-32b` 默认为 `default`。

- **structuredOutputs** _boolean_

  是否使用结构化输出。

  默认为 `true`。

  启用后，对象生成会使用 `json_schema` 格式而非 `json_object`，从而获得更可靠的结构化输出。

- **strictJsonSchema** _boolean_

  是否启用严格的 JSON Schema 校验。为 `true` 时，模型使用约束解码以确保符合 schema。

  默认为 `true`。

  仅在启用 `structuredOutputs` 且提供 schema 时使用。严格模式的限制详见 [Groq Structured Outputs 文档](https://console.groq.com/docs/structured-outputs)。

- **parallelToolCalls** _boolean_

  是否在工具使用过程中启用并行函数调用。默认为 `true`。

- **user** _string_

  表示终端用户的唯一标识，有助于监控与滥用检测。

- **serviceTier** _'on_demand' | 'performance' | 'flex' | 'auto'_

  请求所用服务层级。默认为 `'on_demand'`。
  - `'on_demand'`：默认层级，性能稳定且公平调度
  - `'performance'`：面向延迟敏感场景的优先层级
  - `'flex'`：更高吞吐层级（速率限制约 10 倍），适合可容忍偶发失败的工作负载
  - `'auto'`：先按 on_demand 限速，超出后回退到 flex

  更多层级说明见 [Groq 服务层级文档](https://console.groq.com/docs/service-tiers)。

<Note>仅 Groq reasoning 模型支持 `reasoningFormat` 选项。</Note>

#### 结构化输出

Groq 模型默认启用结构化输出。
可将 `structuredOutputs` 设为 `false` 来关闭。

```ts
import { groq } from '@ai-sdk/groq';
import { generateText, Output } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: groq('moonshotai/kimi-k2-instruct-0905'),
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(z.string()),
        instructions: z.array(z.string()),
      }),
    }),
  }),
  prompt: '生成一份简单的意面食谱。',
});

console.log(JSON.stringify(result.output, null, 2));
```

对不支持该能力的模型，可以关闭结构化输出：

```ts highlight="9"
import { groq, type GroqLanguageModelChatOptions } from '@ai-sdk/groq';
import { generateText, Output } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: groq('gemma2-9b-it'),
  providerOptions: {
    groq: {
      structuredOutputs: false,
    } satisfies GroqLanguageModelChatOptions,
  },
  output: Output.object({
    schema: z.object({
      recipe: z.object({
        name: z.string(),
        ingredients: z.array(z.string()),
        instructions: z.array(z.string()),
      }),
    }),
  }),
  prompt: '以 JSON 格式生成一份简单的意面食谱。',
});

console.log(JSON.stringify(result.output, null, 2));
```

<Note type="warning">
  结构化输出仅较新的 Groq 模型支持，例如
  `moonshotai/kimi-k2-instruct-0905`。不支持的模型可将
  `structuredOutputs` 设为 `false` 关闭。关闭后 Groq
  使用 `json_object` 格式，此时需要在消息中包含单词
  「JSON」。
</Note>

### 示例

可使用 Groq 语言模型配合 `generateText` 生成文本：

```ts
import { groq } from '@ai-sdk/groq';
import { generateText } from 'ai';

const { text } = await generateText({
  model: groq('gemma2-9b-it'),
  prompt: '写一份供 4 人食用的素食千层面食谱。',
});
```

### 图像输入

`meta-llama/llama-4-scout-17b-16e-instruct` 等多模态模型支持图像输入。可在消息中通过 URL 或 base64 编码数据包含图像：

```ts
import { groq } from '@ai-sdk/groq';
import { generateText } from 'ai';

const { text } = await generateText({
  model: groq('meta-llama/llama-4-scout-17b-16e-instruct'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: '这张图片里有什么？' },
        {
          type: 'file',
          mediaType: 'image',
          data: 'https://example.com/image.jpg',
        },
      ],
    },
  ],
});
```

也可以使用 base64 编码的图像：

```ts
import { groq } from '@ai-sdk/groq';
import { generateText } from 'ai';
import { readFileSync } from 'fs';

const imageData = readFileSync('path/to/image.jpg', 'base64');

const { text } = await generateText({
  model: groq('meta-llama/llama-4-scout-17b-16e-instruct'),
  messages: [
    {
      role: 'user',
      content: [
        { type: 'text', text: '详细描述这张图像。' },
        {
          type: 'file',
          mediaType: 'image',
          data: `data:image/jpeg;base64,${imageData}`,
        },
      ],
    },
  ],
});
```

## 浏览器搜索工具

Groq 提供浏览器搜索工具，具备交互式网页浏览能力。与传统网页搜索不同，浏览器搜索会交互式访问网站，从而提供更详尽、更全面的结果。

### 支持的模型

浏览器搜索仅适用于以下模型：

- `openai/gpt-oss-20b`
- `openai/gpt-oss-120b`

<Note type="warning">
  浏览器搜索仅适用于上述支持的模型。与其他模型一起使用时会发出警告，且该工具会被忽略。
</Note>

### 基本用法

```ts
import { groq } from '@ai-sdk/groq';
import { generateText } from 'ai';

const result = await generateText({
  model: groq('openai/gpt-oss-120b'), // 必须使用支持的模型
  prompt:
    '人工智能领域有哪些最新进展？请搜索近期新闻。',
  tools: {
    browser_search: groq.tools.browserSearch({}),
  },
  toolChoice: 'required', // 确保会调用该工具
});

console.log(result.text);
```

### 流式示例

```ts
import { groq } from '@ai-sdk/groq';
import { streamText } from 'ai';

const result = streamText({
  model: groq('openai/gpt-oss-120b'),
  prompt: '搜索最新科技新闻并总结。',
  tools: {
    browser_search: groq.tools.browserSearch({}),
  },
  toolChoice: 'required',
});

for await (const delta of result.fullStream) {
  if (delta.type === 'text-delta') {
    process.stdout.write(delta.text);
  }
}
```

### 主要特性

- **交互式浏览**：像真人用户一样访问网站
- **结果更全面**：比传统搜索摘要更详细
- **服务端执行**：在 Groq 基础设施上运行，无需额外搭建
- **由 Exa 驱动**：使用 Exa 搜索引擎以获得更好结果
- **当前免费**：测试期间不另行收费

### 最佳实践

- 使用 `toolChoice: 'required'` 以确保启用浏览器搜索
- 仅 `openai/gpt-oss-20b` 与 `openai/gpt-oss-120b` 支持
- 工具开箱即用，无需配置参数
- 服务端执行意味着不需要额外的 API 密钥或环境搭建

### 模型校验

Provider 会自动校验模型兼容性：

```ts
// ✅ 支持 —— 可用
const result = await generateText({
  model: groq('openai/gpt-oss-120b'),
  tools: { browser_search: groq.tools.browserSearch({}) },
});

// ❌ 不支持 —— 将显示警告并忽略工具
const result = await generateText({
  model: groq('gemma2-9b-it'),
  tools: { browser_search: groq.tools.browserSearch({}) },
});
// 警告："Browser search is only supported on models: openai/gpt-oss-20b, openai/gpt-oss-120b"
```

<Note>
  浏览器搜索能力与限制的更多说明见 [Groq Browser Search
  文档](https://console.groq.com/docs/browser-search)。
</Note>

## 语音转写模型

可使用 `.transcription()` 工厂方法创建调用 [Groq 语音转写 API](https://console.groq.com/docs/speech-to-text) 的模型。

第一个参数为模型 id，例如 `whisper-large-v3`。

```ts
const model = groq.transcription('whisper-large-v3');
```

还可以通过 `providerOptions` 传入更多 provider 特有选项。例如按 ISO-639-1（如 `en`）指定输入语言可提升准确度并降低延迟。

```ts highlight="6"
import { experimental_transcribe as transcribe } from 'ai';
import { groq, type GroqTranscriptionModelOptions } from '@ai-sdk/groq';
import { readFile } from 'fs/promises';

const result = await transcribe({
  model: groq.transcription('whisper-large-v3'),
  audio: await readFile('audio.mp3'),
  providerOptions: {
    groq: { language: 'en' } satisfies GroqTranscriptionModelOptions,
  },
});
```

可用选项如下：

- **timestampGranularities** _string[]_
  转写中时间戳的粒度。
  默认为 `['segment']`。
  可取 `['word']`、`['segment']`、`['word', 'segment']`。
  说明：分段时间戳不会带来额外延迟，但生成词级时间戳会增加延迟。
  **重要：**需要同时将 `responseFormat` 设为 `'verbose_json'`。

- **responseFormat** _string_
  响应格式。设为 `'verbose_json'` 可接收音频分段的时间戳并启用 `timestampGranularities`。
  设为 `'text'` 则仅返回转写文本。
  可选。

- **language** _string_
  输入音频的语言。按 ISO-639-1（如 `'en'`）指定可提升准确度并降低延迟。
  可选。

- **prompt** _string_
  可选文本，用于引导模型风格或接续上一段音频。prompt 应与音频语言一致。
  可选。

- **temperature** _number_
  采样温度，介于 0 与 1。较高值（如 0.8）使输出更随机，较低值（如 0.2）更聚焦、确定。若为 0，模型会使用对数概率自动调高温度直至达到特定阈值。
  默认为 0。
  可选。
