# Fireworks 提供商

[Fireworks](https://fireworks.ai/) 是通过其 [API](https://readme.fireworks.ai/) 运行与测试 LLM 的平台。

## 安装

Fireworks 提供商位于 `@ai-sdk/fireworks` 模块中，可使用以下命令安装：

## Provider 实例

可从 `@ai-sdk/fireworks` 导入默认的 `fireworks` 提供商实例：

```ts
import { fireworks } from '@ai-sdk/fireworks';
```

如需自定义配置，可导入 `createFireworks` 并以你的设置创建提供商实例：

```ts
import { createFireworks } from '@ai-sdk/fireworks';

const fireworks = createFireworks({
  apiKey: process.env.FIREWORKS_API_KEY ?? '',
});
```

可使用以下可选设置自定义 Fireworks 提供商实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀（例如经由代理）。
  默认前缀为 `https://api.fireworks.ai/inference/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API Key。
  默认为环境变量 `FIREWORKS_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。

## 语言模型

可使用提供商实例创建 [Fireworks 模型](https://fireworks.ai/models)。
首个参数为模型 ID，例如 `accounts/fireworks/models/firefunction-v1`：

```ts
const model = fireworks('accounts/fireworks/models/firefunction-v1');
```

### 推理模型

Fireworks 在 `deepseek-r1` 的生成文本中用 `<think>` 标签暴露推理过程。
可使用 `extractReasoningMiddleware` 抽出该推理，并在结果的 `reasoning` 字段中提供给调用方：

```ts
import { fireworks } from '@ai-sdk/fireworks';
import { wrapLanguageModel, extractReasoningMiddleware } from 'ai';

const enhancedModel = wrapLanguageModel({
  model: fireworks('accounts/fireworks/models/deepseek-r1'),
  middleware: extractReasoningMiddleware({ tagName: 'think' }),
});
```

随后可在 `generateText`、`streamText` 等函数中使用该包装后的模型。

### 示例

可使用 Fireworks 语言模型与 `generateText` 生成文本：

```ts
import { fireworks } from '@ai-sdk/fireworks';
import { generateText } from 'ai';

const { text } = await generateText({
  model: fireworks('accounts/fireworks/models/firefunction-v1'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Fireworks 语言模型也可用于 `streamText`
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

### 提供商选项

Fireworks Chat 模型支持不属于
[标准调用设置](/docs/ai-sdk-core/settings) 的附加提供商选项。可在 `providerOptions` 参数中传入：

```ts
import {
  fireworks,
  type FireworksLanguageModelOptions,
} from '@ai-sdk/fireworks';
import { generateText } from 'ai';

const { text, reasoningText } = await generateText({
  model: fireworks('accounts/fireworks/models/kimi-k2p5'),
  providerOptions: {
    fireworks: {
      thinking: { type: 'enabled', budgetTokens: 4096 },
      reasoningHistory: 'interleaved',
    } satisfies FireworksLanguageModelOptions,
  },
  prompt: 'How many "r"s are in the word "strawberry"?',
});
```

面向 Fireworks Chat 模型，可使用以下可选提供商选项：

- **thinking** _object_

  Kimi K2.5 等推理/思考模型的配置。
  - **type** _'enabled' | 'disabled'_

    是否启用思考模式。

  - **budgetTokens** _number_

    思考过程可用 token 上限（最少 1024）。

- **reasoningHistory** _'disabled' | 'interleaved' | 'preserved'_

  多轮对话中推理历史的处理方式：
  - `'disabled'`：从历史中移除推理内容
  - `'interleaved'`：在同一轮内保留工具调用之间的推理
  - `'preserved'`：在历史中保留全部推理内容

### 补全模型

可使用 `.completionModel()` 工厂方法创建调用 Fireworks Completions API 的模型：

```ts
const model = fireworks.completionModel(
  'accounts/fireworks/models/firefunction-v1',
);
```

## 嵌入模型

可使用 `.embeddingModel()` 工厂方法创建调用 Fireworks Embeddings API 的模型：

```ts
const model = fireworks.embeddingModel('nomic-ai/nomic-embed-text-v1.5');
```

可使用 Fireworks 嵌入模型与 `embed` 生成向量：

```ts
import { fireworks } from '@ai-sdk/fireworks';
import { embed } from 'ai';

const { embedding } = await embed({
  model: fireworks.embeddingModel('nomic-ai/nomic-embed-text-v1.5'),
  value: 'sunny day at the beach',
});
```

### 模型能力

| 模型                             | 维度 | 最大 Token |
| -------------------------------- | ---------- | ---------- |
| `nomic-ai/nomic-embed-text-v1.5` | 768        | 8192       |

<Note>
  更多嵌入模型见 [Fireworks Models](https://fireworks.ai/models)。
</Note>

## 图像模型

可使用 `.image()` 工厂方法创建 Fireworks 图像模型。
详见 AI SDK 的 [generateImage()](/docs/reference/ai-sdk-core/generate-image)。

```ts
import { fireworks } from '@ai-sdk/fireworks';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: fireworks.image('accounts/fireworks/models/flux-1-dev-fp8'),
  prompt: 'A futuristic cityscape at sunset',
  aspectRatio: '16:9',
});
```

<Note>
  不同模型对 `size` 与 `aspectRatio` 的支持不同。请参阅下方「[模型能力](#model-capabilities-1)」，或查阅 [Fireworks Models](https://fireworks.ai/models) 上的模型文档。
</Note>

### 图像编辑

Fireworks 通过 FLUX Kontext 模型（`flux-kontext-pro`、`flux-kontext-max`）支持图像编辑。通过 `prompt.images` 传入输入图像即可变换或编辑。

<Note>
  Fireworks 的 Kontext 模型不支持显式遮罩，编辑依赖提示词驱动——在文本提示中描述希望修改的内容。
</Note>

#### 基本图像编辑

使用文本提示变换现有图像：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: fireworks.image('accounts/fireworks/models/flux-kontext-pro'),
  prompt: {
    text: 'Turn the cat into a golden retriever dog',
    images: [imageBuffer],
  },
  providerOptions: {
    fireworks: {
      output_format: 'jpeg',
    },
  },
});
```

#### 风格迁移

对图像套用艺术风格：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: fireworks.image('accounts/fireworks/models/flux-kontext-pro'),
  prompt: {
    text: 'Transform this into a watercolor painting style',
    images: [imageBuffer],
  },
  aspectRatio: '1:1',
});
```

<Note>
  输入图像可为 `Buffer`、`ArrayBuffer`、`Uint8Array` 或 base64 字符串。Fireworks 每个请求仅支持单张输入图像。
</Note>
