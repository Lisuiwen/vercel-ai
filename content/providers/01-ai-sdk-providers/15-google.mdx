# Google 提供商

[Google](https://ai.google.dev) 提供商为 [Google](https://ai.google.dev/api/rest) API 提供语言模型与嵌入模型支持。

## 安装

Google 提供商位于 `@ai-sdk/google` 模块中。可通过以下方式安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/google" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install @ai-sdk/google" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/google" dark />
  </Tab>

  <Tab>
    <Snippet text="bun add @ai-sdk/google" dark />
  </Tab>
</Tabs>

## 提供商实例

你可以从 `@ai-sdk/google` 导入默认提供商实例 `google`：

```ts
import { google } from '@ai-sdk/google';
```

如需自定义配置，可从 `@ai-sdk/google` 导入 `createGoogle`，并用你的设置创建提供商实例：

```ts
import { createGoogle } from '@ai-sdk/google';

const google = createGoogle({
  // custom settings
});
```

可使用以下可选设置自定义 Google 提供商实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀，例如用于代理服务器。
  默认前缀为 `https://generativelanguage.googleapis.com/v1beta`。

- **apiKey** _string_

  通过 `x-goog-api-key` 请求头发送的 API 密钥。
  默认读取环境变量 `GOOGLE_GENERATIVE_AI_API_KEY`。

- **headers** _Record&lt;string,string&gt;_

  要包含在请求中的自定义标头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。
  可用作中间件拦截请求，
  或在例如测试时提供自定义 fetch 实现。

- **generateId** _() => string_

  为每个请求生成唯一 ID 的可选函数。
  默认为 SDK 内置 ID 生成器。

- **name** _string_

  自定义提供商名称。
  默认为 `'google.generative-ai'`。

## 语言模型

可使用提供商实例创建调用 [Google Generative AI API](https://ai.google.dev/api/rest) 的模型。
第一个参数为模型 ID，例如 `gemini-2.5-flash`。
这些模型支持工具调用，部分具备多模态能力。

```ts
const model = google('gemini-2.5-flash');
```

可使用 Google 语言模型配合 `generateText` 函数生成文本：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text } = await generateText({
  model: google('gemini-2.5-flash'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

Google 语言模型也可用于 `streamText` 函数，
并支持配合 [`Output`](/docs/reference/ai-sdk-core/output) 进行结构化数据生成
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

Google 还支持一些不属于 [标准调用设置](/docs/ai-sdk-core/settings) 的模型专属设置。
可通过 options 参数传入：

```ts
import { google, type GoogleLanguageModelOptions } from '@ai-sdk/google';

const model = google('gemini-2.5-flash');

await generateText({
  model,
  providerOptions: {
    google: {
      safetySettings: [
        {
          category: 'HARM_CATEGORY_UNSPECIFIED',
          threshold: 'BLOCK_LOW_AND_ABOVE',
        },
      ],
    } satisfies GoogleLanguageModelOptions,
  },
});
```

Google 模型可使用以下可选提供商选项：

- **cachedContent** _string_

  可选。用作预测上下文的缓存内容名称。
  格式：cachedContents/\{cachedContent\}

- **structuredOutputs** _boolean_

  可选。启用结构化输出。默认为 true。

  当 JSON Schema 包含 Google 所用 OpenAPI schema 版本
  不支持的元素时，此项很有用。可在需要时
  通过此项关闭结构化输出。

  更多说明见 [故障排查：Schema 限制](#schema-limitations)。

- **safetySettings** _Array\<\{ category: string; threshold: string \}\>_

  可选。模型的安全设置。
  - **category** _string_

    安全设置的类别。可为以下之一：
    - `HARM_CATEGORY_UNSPECIFIED`
    - `HARM_CATEGORY_HATE_SPEECH`
    - `HARM_CATEGORY_DANGEROUS_CONTENT`
    - `HARM_CATEGORY_HARASSMENT`
    - `HARM_CATEGORY_SEXUALLY_EXPLICIT`
    - `HARM_CATEGORY_CIVIC_INTEGRITY`

  - **threshold** _string_

    安全设置的阈值。可为以下之一：
    - `HARM_BLOCK_THRESHOLD_UNSPECIFIED`
    - `BLOCK_LOW_AND_ABOVE`
    - `BLOCK_MEDIUM_AND_ABOVE`
    - `BLOCK_ONLY_HIGH`
    - `BLOCK_NONE`
    - `OFF`

- **responseModalities** _string[]_
  响应使用的模态。支持以下模态：`TEXT`、`IMAGE`。未定义或为空时，模型默认仅返回文本。

- **thinkingConfig** _\{ thinkingLevel?: 'minimal' | 'low' | 'medium' | 'high'; thinkingBudget?: number; includeThoughts?: boolean \}_

  可选。模型思考过程的配置。仅部分 [Google 模型](https://ai.google.dev/gemini-api/docs/thinking) 支持。
  - **thinkingLevel** _'minimal' | 'low' | 'medium' | 'high'_

    可选。控制 Gemini 3 模型的思考深度。Gemini 3.1 Pro 支持 `'low'`、`'medium'` 和 `'high'`，Gemini 3 Pro 支持 `'low'` 和 `'high'`，而 Gemini 3 Flash 支持全部四级：`'minimal'`、`'low'`、`'medium'` 和 `'high'`。仅 Gemini 3 模型支持。

  - **thinkingBudget** _number_

    可选。为模型在生成响应时可使用的思考 token 数量提供指引。若模型支持，设为 0 可关闭思考。
    各模型可取值的范围见 [Google 思考文档](https://ai.google.dev/gemini-api/docs/thinking#set-budget)。

    <Note>
      此选项适用于 Gemini 2.5 模型。Gemini 3 模型应改用
      `thinkingLevel`。
    </Note>

  - **includeThoughts** _boolean_

    可选。若为 true，会返回思考摘要，即对模型原始思考的合成版本，可洞察其内部推理过程。

- **imageConfig** _\{ aspectRatio?: string, imageSize?: string \}_

  可选。图像生成模型的配置。仅部分 [Google 模型](https://ai.google.dev/gemini-api/docs/image-generation) 支持。
  - **aspectRatio** _string_

    模型默认生成 1:1 方形，或使输出图像尺寸与输入图像一致。可为以下之一：
    - 1:1
    - 2:3
    - 3:2
    - 3:4
    - 4:3
    - 4:5
    - 5:4
    - 9:16
    - 16:9
    - 21:9

  - **imageSize** _string_

    控制输出图像分辨率。默认为 1K。可为以下之一：
    - 1K
    - 2K
    - 4K

- **audioTimestamp** _boolean_

  可选。为纯音频文件启用时间戳理解。
  见 [Google Cloud 音频理解文档](https://cloud.google.com/vertex-ai/generative-ai/docs/multimodal/audio-understanding)。

- **mediaResolution** _string_

  可选。若指定，将使用所给媒体分辨率。可为以下之一：
  - `MEDIA_RESOLUTION_UNSPECIFIED`
  - `MEDIA_RESOLUTION_LOW`
  - `MEDIA_RESOLUTION_MEDIUM`
  - `MEDIA_RESOLUTION_HIGH`

  见 [Google API MediaResolution 文档](https://ai.google.dev/api/generate-content#MediaResolution)。

- **labels** _Record&lt;string, string&gt;_

  可选。定义计费报告中使用的标签。仅限 Vertex AI。
  见 [Google Cloud 标签文档](https://cloud.google.com/vertex-ai/generative-ai/docs/multimodal/add-labels-to-api-calls)。

- **serviceTier** _'standard' | 'flex' | 'priority'_

  可选。请求使用的服务层级。
  设为 `'flex'` 可降低约 50% 费用，但延迟会增加。
  设为 `'priority'` 可获得超低延迟，价格较 `'standard'` 高约 75–100%。

- **threshold** _string_

  可选。可独立于 `safetySettings` 使用的单独阈值设置。
  取值与 `safetySettings` 的 threshold 相同。

### 思考（Thinking）

Gemini 2.5 与 Gemini 3 系列模型使用内部「思考过程」，显著提升推理与多步规划能力，在编码、高等数学和数据分析等复杂任务上效果突出。更多说明见 [Google 思考文档](https://ai.google.dev/gemini-api/docs/thinking)。

#### Gemini 3 模型

对 Gemini 3 模型，使用 `thinkingLevel` 参数控制推理深度：

```ts
import { google, GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText } from 'ai';

const model = google('gemini-3.1-pro-preview');

const { text, reasoning } = await generateText({
  model: model,
  prompt: 'What is the sum of the first 10 prime numbers?',
  providerOptions: {
    google: {
      thinkingConfig: {
        thinkingLevel: 'high',
        includeThoughts: true,
      },
    } satisfies GoogleLanguageModelOptions,
  },
});

console.log(text);

console.log(reasoning); // Reasoning summary
```

#### Gemini 2.5 模型

对 Gemini 2.5 模型，使用 `thinkingBudget` 参数控制思考 token 数量：

```ts
import { google, GoogleLanguageModelOptions } from '@ai-sdk/google';
import { generateText } from 'ai';

const model = google('gemini-2.5-flash');

const { text, reasoning } = await generateText({
  model: model,
  prompt: 'What is the sum of the first 10 prime numbers?',
  providerOptions: {
    google: {
      thinkingConfig: {
        thinkingBudget: 8192,
        includeThoughts: true,
      },
    } satisfies GoogleLanguageModelOptions,
  },
});

console.log(text);

console.log(reasoning); // Reasoning summary
```

### 文件输入

Google 提供商支持文件输入，例如 PDF 文件。

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google('gemini-2.5-flash'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model according to this document?',
        },
        {
          type: 'file',
          data: fs.readFileSync('./data/ai.pdf'),
          mediaType: 'application/pdf',
        },
      ],
    },
  ],
});
```

也可以直接使用 YouTube URL：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google('gemini-2.5-flash'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'Summarize this video',
        },
        {
          type: 'file',
          data: 'https://www.youtube.com/watch?v=dQw4w9WgXcQ',
          mediaType: 'video/mp4',
        },
      ],
    },
  ],
});
```

<Note>
  若将 URL 作为 `data` 传入，AI SDK 会自动下载，但
  `https://generativelanguage.googleapis.com/v1beta/files/` 与 YouTube
  URL 除外。可使用 Google Generative AI Files API 将较大文件上传到
  该地址。YouTube URL（公开或不公开列出）可直接支持——每次请求可指定一个 YouTube 视频 URL。
</Note>

有关如何在提示中使用文件，见 [File Parts](/docs/foundations/prompts#file-parts)。

### 缓存内容

Google 支持显式与隐式缓存，有助于降低重复内容的成本。

#### 隐式缓存

Gemini 2.5 模型会自动带来缓存成本节省，无需创建显式缓存。当请求与先前请求共享相同前缀时，缓存内容可享受 75% 的 token 折扣。

为尽量提高隐式缓存命中：

- 保持请求开头内容一致
- 将可变内容（如用户问题）放在提示末尾
- 确保请求满足最低 token 要求：
  - Gemini 2.5 Flash：至少 1024 token
  - Gemini 2.5 Pro：至少 2048 token

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

// Structure prompts with consistent content at the beginning
const baseContext =
  'You are a cooking assistant with expertise in Italian cuisine. Here are 1000 lasagna recipes for reference...';

const { text: veggieLasagna } = await generateText({
  model: google('gemini-2.5-pro'),
  prompt: `${baseContext}\n\nWrite a vegetarian lasagna recipe for 4 people.`,
});

// Second request with same prefix - eligible for cache hit
const { text: meatLasagna, providerMetadata } = await generateText({
  model: google('gemini-2.5-pro'),
  prompt: `${baseContext}\n\nWrite a meat lasagna recipe for 12 people.`,
});

// Check cached token count in usage metadata
console.log('Cached tokens:', providerMetadata.google);
// e.g.
// {
//   groundingMetadata: null,
//   safetyRatings: null,
//   usageMetadata: {
//     cachedContentTokenCount: 2027,
//     thoughtsTokenCount: 702,
//     promptTokenCount: 2152,
//     candidatesTokenCount: 710,
//     totalTokenCount: 3564
//   }
// }
```

<Note>
  `@ai-sdk/google@1.2.23` 起，用量元数据已加入 `providerMetadata`。若使用更旧版本，用量元数据可在 `generateText` 返回值中原始 HTTP
  `response` 正文里获取。
</Note>

#### 显式缓存

若要保证成本节省，仍可对 Gemini 2.5 与 2.0 模型使用显式缓存。见 [模型页面](https://ai.google.dev/gemini-api/docs/models) 确认所用模型是否支持缓存：

```ts
import { google, type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { GoogleGenAI } from '@google/genai';
import { generateText } from 'ai';

const ai = new GoogleGenAI({
  apiKey: process.env.GOOGLE_GENERATIVE_AI_API_KEY,
});

const model = 'gemini-2.5-pro';

// Create a cache with the content you want to reuse
const cache = await ai.caches.create({
  model,
  config: {
    contents: [
      {
        role: 'user',
        parts: [{ text: '1000 Lasagna Recipes...' }],
      },
    ],
    ttl: '300s', // Cache expires after 5 minutes
  },
});

const { text: veggieLasagnaRecipe } = await generateText({
  model: google(model),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
  providerOptions: {
    google: {
      cachedContent: cache.name,
    } satisfies GoogleLanguageModelOptions,
  },
});

const { text: meatLasagnaRecipe } = await generateText({
  model: google(model),
  prompt: 'Write a meat lasagna recipe for 12 people.',
  providerOptions: {
    google: {
      cachedContent: cache.name,
    } satisfies GoogleLanguageModelOptions,
  },
});
```

### 代码执行

借助 [代码执行（Code Execution）](https://ai.google.dev/gemini-api/docs/code-execution)，部分模型可生成并执行 Python 代码以进行计算、解题或提供更准确信息。

可在请求中加入 `code_execution` 工具以启用代码执行。

```ts
import { google } from '@ai-sdk/google';
import { googleTools } from '@ai-sdk/google/internal';
import { generateText } from 'ai';

const { text, toolCalls, toolResults } = await generateText({
  model: google('gemini-2.5-pro'),
  tools: { code_execution: google.tools.codeExecution({}) },
  prompt: 'Use python to calculate the 20th fibonacci number.',
});
```

响应中会包含代码执行产生的工具调用与结果。

<h3 id="google-search">Google 搜索</h3>

借助 [Google 搜索 grounding](https://ai.google.dev/gemini-api/docs/google-search)，
模型可通过 Google 搜索获取最新信息。

```ts highlight="8,17-20"
import { google } from '@ai-sdk/google';
import { GoogleProviderMetadata } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text, sources, providerMetadata } = await generateText({
  model: google('gemini-2.5-flash'),
  tools: {
    google_search: google.tools.googleSearch({}),
  },
  prompt:
    'List the top 5 San Francisco news from the past week.' +
    'You must include the date of each article.',
});

// access the grounding metadata. Casting to the provider metadata type
// is optional but provides autocomplete and type safety.
const metadata = providerMetadata?.google as GoogleProviderMetadata | undefined;
const groundingMetadata = metadata?.groundingMetadata;
const safetyRatings = metadata?.safetyRatings;
```

`googleSearch` 工具支持以下可选配置：

- **searchTypes** _object_

  启用特定搜索类型。可同时启用两者。
  - `webSearch`：启用网页搜索 grounding（传入 `{}` 即可）。此为默认。
  - `imageSearch`：启用[图像搜索 grounding](https://ai.google.dev/gemini-api/docs/image-generation#image-search)（传入 `{}` 即可）。

- **timeRangeFilter** _object_

  将搜索结果限制在特定时间范围。`startTime` 与 `endTime` 均必填。
  - `startTime`：ISO 8601 格式的开始时间（例如 `'2025-01-01T00:00:00Z'`）。
  - `endTime`：ISO 8601 格式的结束时间（例如 `'2025-12-31T23:59:59Z'`）。

```ts
google.tools.googleSearch({
  searchTypes: { webSearch: {} },
  timeRangeFilter: {
    startTime: '2025-01-01T00:00:00Z',
    endTime: '2025-12-31T23:59:59Z',
  },
});
```

启用 Google 搜索 grounding 时，模型会在响应中包含来源。

此外，grounding 元数据包含搜索结果如何用于 grounding 模型响应的详细信息。可用字段如下：

- **`webSearchQueries`** (`string[] | null`)
  - 用于检索信息的搜索查询数组
  - 示例：`["What's the weather in Chicago this weekend?"]`

- **`searchEntryPoint`** (`{ renderedContent: string } | null`)
  - 用作入口点的主要搜索结果内容
  - `renderedContent` 字段为格式化后的内容

- **`groundingSupports`**（支持对象数组 | null）
  - 描述响应片段如何由搜索结果支持
  - 每个支持对象包括：
    - **`segment`**：有关被 grounding 文本片段的信息
      - `text`：实际文本片段
      - `startIndex`：在响应中的起始位置
      - `endIndex`：在响应中的结束位置
    - **`groundingChunkIndices`**：指向支持性搜索结果的 chunk 索引
    - **`confidenceScores`**：各支持 chunk 的置信度分数（0–1）

示例响应：

```json
{
  "groundingMetadata": {
    "webSearchQueries": ["What's the weather in Chicago this weekend?"],
    "searchEntryPoint": {
      "renderedContent": "..."
    },
    "groundingSupports": [
      {
        "segment": {
          "startIndex": 0,
          "endIndex": 65,
          "text": "Chicago weather changes rapidly, so layers let you adjust easily."
        },
        "groundingChunkIndices": [0],
        "confidenceScores": [0.99]
      }
    ]
  }
}
```

### 企业网页搜索

借助[企业网页搜索（Enterprise Web Search）](https://cloud.google.com/vertex-ai/generative-ai/docs/grounding/web-grounding-enterprise)，
模型可访问面向强监管行业（如金融、医疗、公共部门）的合规向网页索引。

<Note>
  企业网页搜索仅在 Vertex AI 上可用。必须使用 Google
  Vertex 提供商（`@ai-sdk/google-vertex`），而非标准 Google
  提供商（`@ai-sdk/google`）。需要 Gemini 2.0 或更新
  模型。
</Note>

```ts
import { createGoogleVertex } from '@ai-sdk/google-vertex';
import { generateText } from 'ai';

const vertex = createGoogleVertex({
  project: 'my-project',
  location: 'us-central1',
});

const { text, sources, providerMetadata } = await generateText({
  model: vertex('gemini-2.5-flash'),
  tools: {
    enterprise_web_search: vertex.tools.enterpriseWebSearch({}),
  },
  prompt: 'What are the latest regulatory updates for financial services?',
});
```

企业网页搜索提供以下优势：

- 不记录客户数据
- 支持 VPC 服务控件
- 面向监管行业的合规网页索引

### 文件搜索

[文件搜索工具](https://ai.google.dev/gemini-api/docs/file-search) 让 Gemini 从你已在 File Search 存储中建立索引的自有文档中检索上下文。仅 Gemini 2.5 与 Gemini 3 模型支持此功能。

```ts highlight="9-13"
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text, sources } = await generateText({
  model: google('gemini-2.5-pro'),
  tools: {
    file_search: google.tools.fileSearch({
      fileSearchStoreNames: [
        'projects/my-project/locations/us/fileSearchStores/my-store',
      ],
      metadataFilter: 'author = "Robert Graves"',
      topK: 8,
    }),
  },
  prompt: "Summarise the key themes of 'I, Claudius'.",
});
```

文件搜索响应通过常规 `sources` 字段提供引用，并在 `providerMetadata.google.groundingMetadata` 中暴露原始 [grounding 元数据](#google-search)。

### URL 上下文

Google 提供由提供商定义的 URL 上下文工具。

URL 上下文工具让你指定希望模型在提示中直接分析的 URL。

```ts highlight="9,13-17"
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text, sources, providerMetadata } = await generateText({
  model: google('gemini-2.5-flash'),
  prompt: `Based on the document: https://ai.google.dev/gemini-api/docs/url-context.
          Answer this question: How many links we can consume in one request?`,
  tools: {
    url_context: google.tools.urlContext({}),
  },
});

const metadata = providerMetadata?.google as GoogleProviderMetadata | undefined;
const groundingMetadata = metadata?.groundingMetadata;
const urlContextMetadata = metadata?.urlContextMetadata;
```

URL 上下文元数据包含模型如何使用 URL 上下文生成响应的详细信息。可用字段如下：

- **`urlMetadata`** (`{ retrievedUrl: string; urlRetrievalStatus: string; }[] | null`)
  - URL 上下文元数据数组
  - 每个对象包含：
    - **`retrievedUrl`**：上下文对应的 URL
    - **`urlRetrievalStatus`**：URL 检索状态

示例响应：

```json
{
  "urlMetadata": [
    {
      "retrievedUrl": "https://ai-sdk.dev/providers/ai-sdk-providers/google",
      "urlRetrievalStatus": "URL_RETRIEVAL_STATUS_SUCCESS"
    }
  ]
}
```

### Google 地图 grounding

借助 [Google 地图 grounding](https://ai.google.dev/gemini-api/docs/maps-grounding)，
模型可访问 Google 地图数据以生成与位置相关的回复，例如查找附近餐厅。

```ts highlight="7-16"
import { google, type GoogleLanguageModelOptions } from '@ai-sdk/google';
import { GoogleProviderMetadata } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text, sources, providerMetadata } = await generateText({
  model: google('gemini-2.5-flash'),
  tools: {
    google_maps: google.tools.googleMaps({}),
  },
  providerOptions: {
    google: {
      retrievalConfig: {
        latLng: { latitude: 34.090199, longitude: -117.881081 },
      },
    } satisfies GoogleLanguageModelOptions,
  },
  prompt:
    'What are the best Italian restaurants within a 15-minute walk from here?',
});

const metadata = providerMetadata?.google as GoogleProviderMetadata | undefined;
const groundingMetadata = metadata?.groundingMetadata;
```

可选的提供商选项 `retrievalConfig.latLng` 为「附近地点」类查询提供位置上下文。该配置适用于所有支持位置上下文的 grounding 工具，包括 Google 地图与 Google 搜索。

启用 Google 地图 grounding 时，模型响应中的来源会指向 Google 地图 URL。grounding 元数据包含含地点信息的 `maps` chunk：

```json
{

    "groundingChunks": [
      {
        "maps": {
          "uri": "https://maps.google.com/?cid=12345",
          "title": "Restaurant Name",
          "placeId": "places/ChIJ..."
        }
      }
    ]
  }
}
```

<Note>Google 地图 grounding 支持 Gemini 2.0 及更新模型。</Note>

### RAG 引擎 grounding

借助 [RAG 引擎 grounding](https://cloud.google.com/vertex-ai/generative-ai/docs/rag-engine/use-vertexai-search#generate-content-using-gemini-api)，
模型可通过 Vertex RAG 引擎访问你的自定义知识库，
从而基于你的具体数据源与文档作答。

<Note>
  RAG 引擎 grounding 仅支持 Vertex Gemini 模型。必须使用
  Google Vertex 提供商（`@ai-sdk/google-vertex`），而非标准
  Google 提供商（`@ai-sdk/google`）。
</Note>

```ts highlight="8,17-20"
import { createGoogleVertex } from '@ai-sdk/google-vertex';
import { GoogleProviderMetadata } from '@ai-sdk/google';
import { generateText } from 'ai';

const vertex = createGoogleVertex({
  project: 'my-project',
  location: 'us-central1',
});

const { text, sources, providerMetadata } = await generateText({
  model: vertex('gemini-2.5-flash'),
  tools: {
    vertex_rag_store: vertex.tools.vertexRagStore({
      ragCorpus:
        'projects/my-project/locations/us-central1/ragCorpora/my-rag-corpus',
      topK: 5,
    }),
  },
  prompt:
    'What are the key features of our product according to our documentation?',
});

// access the grounding metadata. Casting to the provider metadata type
// is optional but provides autocomplete and type safety.
const metadata = providerMetadata?.google as GoogleProviderMetadata | undefined;
const groundingMetadata = metadata?.groundingMetadata;
const safetyRatings = metadata?.safetyRatings;
```

启用 RAG 引擎 grounding 时，模型会在响应中包含来自 RAG 语料的来源。

此外，grounding 元数据包含 RAG 结果如何用于 grounding 模型响应的详细信息。可用字段如下：

- **`groundingChunks`**（chunk 对象数组 | null）
  - 来自 RAG 语料的检索上下文 chunk
  - 每个 chunk 包含：
    - **`retrievedContext`**：检索到的上下文信息
      - `uri`：来源文档的 URI 或标识符
      - `title`：来源文档标题（可选）
      - `text`：chunk 的实际文本内容

- **`groundingSupports`**（支持对象数组 | null）
  - 描述响应片段如何由 RAG 结果支持
  - 每个支持对象包括：
    - **`segment`**：有关被 grounding 文本片段的信息
      - `text`：实际文本片段
      - `startIndex`：在响应中的起始位置
      - `endIndex`：在响应中的结束位置
    - **`groundingChunkIndices`**：指向支持性 RAG chunk 的索引
    - **`confidenceScores`**：各支持 chunk 的置信度分数（0–1）

示例响应：

```json
{
  "groundingMetadata": {
    "groundingChunks": [
      {
        "retrievedContext": {
          "uri": "gs://my-bucket/docs/product-guide.pdf",
          "title": "Product User Guide",
          "text": "Our product includes advanced AI capabilities, real-time processing, and enterprise-grade security features."
        }
      }
    ],
    "groundingSupports": [
      {
        "segment": {
          "startIndex": 0,
          "endIndex": 45,
          "text": "Our product includes advanced AI capabilities and real-time processing."
        },
        "groundingChunkIndices": [0],
        "confidenceScores": [0.95]
      }
    ]
  }
}
```

#### 配置选项

`vertexRagStore` 工具接受以下配置：

- **`ragCorpus`**（`string`，必填）
  - RagCorpus 资源名称，格式：`projects/{project}/locations/{location}/ragCorpora/{rag_corpus}`
  - 用于指定要检索的具体 RAG 语料

- **`topK`**（`number`，可选）
  - 从 RAG 语料检索的上下文条数上限
  - 未指定时默认使用语料配置

### 故障排查

<h4 id="schema-limitations">Schema 限制</h4>

Google Generative AI API 使用 OpenAPI 3.0 schema 的子集，
不支持联合类型（unions）等特性。
此情况下错误类似：

`GenerateContentRequest.generation_config.response_schema.properties[occupation].type: must be specified`

默认启用结构化输出（工具调用则必须启用）。
可在生成对象时关闭结构化输出作为变通：

```ts highlight="3,8"
const { output } = await generateText({
  model: google('gemini-2.5-flash'),
  providerOptions: {
    google: {
      structuredOutputs: false,
    } satisfies GoogleLanguageModelOptions,
  },
  output: Output.object({
    schema: z.object({
      name: z.string(),
      age: z.number(),
      contact: z.union([
        z.object({
          type: z.literal('email'),
          value: z.string(),
        }),
        z.object({
          type: z.literal('phone'),
          value: z.string(),
        }),
      ]),
    }),
  }),
  prompt: 'Generate an example person for testing.',
});
```

已知以下 Zod 特性与 Google 不兼容：

- `z.union`
- `z.record`

## Interactions API

[Gemini Interactions API](https://ai.google.dev/gemini-api/docs/interactions)
（`POST /v1beta/interactions`）是独立的 Google 端点，具备服务端
状态、统一内容块、一等内置工具、agent 预设，
以及原生多模态图像输出。通过 `google.interactions(...)`
工厂访问：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text } = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  prompt: 'Hello, how are you?',
});
```

`google.interactions(...)` 可接受模型 ID 字符串（例如
`'gemini-2.5-flash'`、`'gemini-3-pro-preview'`），或 `{ agent: <name> }` 以使用
Gemini [agent 预设](#agent-presets)。返回的模型可像其他任何 AI SDK 语言模型一样传给
`generateText` 与 `streamText`。

<Note>
  标准 `:generateContent` /
  `:streamGenerateContent` 端点请用 `google(...)`；新的
  Interactions 端点请用 `google.interactions(...)`。每个模型实例二选一——二者针对不同的
  请求体与 SSE 事件词汇。
</Note>

### 提供商选项

Interactions 模型从共享的
`providerOptions.google.*` 命名空间读取选项。可用
`GoogleLanguageModelInteractionsOptions` 类型校验：

```ts
import {
  google,
  type GoogleLanguageModelInteractionsOptions,
} from '@ai-sdk/google';
import { generateText } from 'ai';

await generateText({
  model: google.interactions('gemini-2.5-flash'),
  prompt: 'What color is the sky in one word?',
  providerOptions: {
    google: {
      serviceTier: 'priority',
    } satisfies GoogleLanguageModelInteractionsOptions,
  },
});
```

可用可选提供商选项如下：

- **previousInteractionId** _string_

  上一轮的服务端 interaction id。设置后，服务端从自身状态拉取
  先前上下文，线路上仅发送新的用户消息。与默认 `store: true` 配合可链式维护
  有状态对话。见 [有状态链接](#stateful-chaining)。

- **store** _boolean_

  服务端是否持久化该 interaction。默认为 `true`。
  设为 `false` 适用于每轮都重新发送完整
  消息历史的无状态多轮对话（不使用 `previousInteractionId`）。

- **agent** _string_

  Gemini agent 预设名称（例如 `'deep-research-pro-preview-12-2025'`）。

      <Note>

  建议在工厂中使用 `google.interactions({ agent: '...' })`，而非在提供商选项里设置 `agent`——工厂形式会针对支持的 agent 名称做类型检查。

  </Note>

- **agentConfig** _object_

  按 agent 的配置。当前支持 `{ type: 'dynamic' }` 与
  `{ type: 'deep-research', thinkingSummaries?, visualization?, collaborativePlanning? }`。

- **thinkingLevel** _'minimal' | 'low' | 'medium' | 'high'_

  控制启用思考的模型的推理深度。映射到
  Interactions 请求中的 `thinking_level`。

- **thinkingSummaries** _'auto' | 'none'_

  模型是否在推理部分返回合成的思考摘要。
  默认遵循 API 默认行为。

- **responseFormat** _Array\<\{ type: 'text' | 'image' | 'audio'; mimeType?: string; schema?: unknown; aspectRatio?: string; imageSize?: '1K' \| '2K' \| '4K' \| '512' \}\>_

  直接映射到 API `response_format`
  数组的输出格式项。用于精细控制图像、音频或非 JSON
  文本输出（例如图像生成的 `aspectRatio` 与 `imageSize`）。
  AI SDK 调用级 `responseFormat: { type: 'json', schema }` 仍会
  自动驱动 JSON 模式并前置匹配的 text 项；
  此处列出的项会追加其后。

  `aspectRatio` 接受 `1:1`、`2:3`、`3:2`、`3:4`、`4:3`、`4:5`、`5:4`、`9:16`、`16:9`、`21:9`、`1:8`、`8:1`、`1:4`、`4:1`。

- **imageConfig** _\{ aspectRatio?: string; imageSize?: '1K' | '2K' | '4K' | '512' \}_（已弃用）

  请改用带 `{ type: 'image', ... }` 项的 **responseFormat**。
  为向后兼容保留；SDK 会将 `imageConfig`
  转换为对应的 `response_format` 图像项并在设置时发出警告。
  若 `responseFormat` 已提供图像项则忽略 `imageConfig`。

- **mediaResolution** _'low' | 'medium' | 'high' | 'ultra_high'_

  应用于图像输入/输出的媒体分辨率。

- **serviceTier** _'flex' | 'standard' | 'priority'_

  请求的服务层级。会在
  `result.providerMetadata.google.serviceTier` 中回显以便观测。

- **systemInstruction** _string_

  AI SDK `system` 消息的替代方案。若两者都设置，以 AI SDK
  `system` 为准并会发出警告。

- **pollingTimeoutMs** _number_

  轮询后台 interaction（agent
  调用）的最长时间（毫秒），超时则放弃。默认 30 分钟（1,800,000 ms）。耗时较长的
  agent（如深度研究）可能需要更长。

### 提供商元数据

`result.providerMetadata.google`（类型为 `GoogleInteractionsProviderMetadata`）
暴露：

- **interactionId** _string_

  服务端 interaction id。下一轮作为 `previousInteractionId` 传回以链接会话。

- **serviceTier** _string_

  请求实际运行的服务层级。

- **signature** _string_

  按块签名哈希，由 SDK 在输出 reasoning 与
  tool-call 部分设置。下一轮自动往返。

<h3 id="stateful-chaining">有状态链接</h3>

默认 `store: true` 时，服务端保留上一轮，因此
下一轮请求只需发送新的用户消息与
`previousInteractionId`：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const turn1 = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  prompt: 'What are the three largest cities in Spain?',
});

const interactionId = turn1.providerMetadata?.google?.interactionId as
  | string
  | undefined;

const turn2 = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  prompt: 'What is the most famous landmark in the second one?',
  providerOptions: {
    google: { previousInteractionId: interactionId },
  },
});
```

无状态多轮对话可设 `store: false`，每轮重新发送完整
消息历史（不使用 `previousInteractionId`）：

```ts
import { google } from '@ai-sdk/google';
import { generateText, type ModelMessage } from 'ai';

const messages: Array<ModelMessage> = [
  { role: 'user', content: 'What are the three largest cities in Spain?' },
];

const turn1 = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  messages,
  providerOptions: { google: { store: false } },
});

messages.push(...turn1.responseMessages);
messages.push({
  role: 'user',
  content: 'What is the most famous landmark in the second one?',
});

const turn2 = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  messages,
  providerOptions: { google: { store: false } },
});
```

### 内置工具

Interactions API 附带内置工具目录。`google.tools.*` 下的提供商定义
工具映射到 Interactions 工具描述符：

| AI SDK 工具                           | Interactions 工具类型 | 说明                                     |
| ------------------------------------- | ---------------------- | ----------------------------------------- |
| `google.tools.googleSearch`           | `google_search`        | 网页/图像搜索 grounding。             |
| `google.tools.codeExecution`          | `code_execution`       | 服务端 Python 执行。             |
| `google.tools.urlContext`             | `url_context`          | 抓取提示中引用的 URL。      |
| `google.tools.fileSearch`             | `file_search`          | 从 File Search 存储检索。        |
| `google.tools.googleMaps`             | `google_maps`          | 附近地点查询的地图 grounding。 |
| _提供商工具_ `google.computer_use` | `computer_use`         | 计算机使用（浏览器环境）。       |
| _提供商工具_ `google.mcp_server`   | `mcp_server`           | 远程 MCP 服务器透传。            |
| _提供商工具_ `google.retrieval`    | `retrieval`            | Vertex AI Search 检索。               |

用 AI SDK `tool(...)` 辅助函数定义的函数工具（`type: 'function'`）
会翻译为 Interactions `function` 工具描述符。其他
工具种类会触发警告并被丢弃。

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text, sources } = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  tools: {
    google_search: google.tools.googleSearch({}),
  },
  prompt:
    "What's a notable AI development from this past week? " +
    'Include the date for each item you mention.',
});
```

函数工具用法与标准提供商相同：

```ts
import { google } from '@ai-sdk/google';
import { generateText, stepCountIs, tool } from 'ai';
import { z } from 'zod';

const weatherTool = tool({
  description: 'Get the weather for a city.',
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ city }) => `It is sunny in ${city}.`,
});

const { text, toolCalls } = await generateText({
  model: google.interactions('gemini-2.5-flash'),
  tools: { getWeather: weatherTool },
  stopWhen: stepCountIs(5),
  prompt: 'What is the weather in San Francisco right now?',
});
```

### 通过 Interactions 输出图像

在支持图像的 Gemini 模型的 `responseFormat` 中加入 `{ type: 'image' }` 项，
即可在响应中以 `LanguageModelV4FilePart` 文件形式获得图像。无需工具包装，该条目也可设置 `aspectRatio`、`imageSize` 与 `mimeType`。

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google.interactions('gemini-3-pro-image-preview'),
  prompt: 'Generate an image of a comic cat in a spaceship.',
  providerOptions: {
    google: {
      responseFormat: [{ type: 'image' }],
    },
  },
});

for (const file of result.files) {
  if (file.mediaType.startsWith('image/')) {
    // file.uint8Array | file.base64 | file.mediaType
  }
}
```

要控制宽高比、图像尺寸或输出 MIME 类型，可在同一图像条目中添加这些
字段：

```ts
const result = await generateText({
  model: google.interactions('gemini-3-pro-image-preview'),
  prompt: 'Generate a high-quality landscape photo of mountains at sunset.',
  providerOptions: {
    google: {
      responseFormat: [
        {
          type: 'image',
          aspectRatio: '16:9',
          imageSize: '4K',
        },
      ],
    },
  },
});
```

多模态输出可为每种模态各列一项。模型返回的文本在 `result.text`，配图在 `result.files`：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google.interactions('gemini-2.5-flash-image'),
  prompt:
    'Tell me a three sentence bedtime story about a unicorn, accompanied by a suitable illustration.',
  providerOptions: {
    google: {
      responseFormat: [
        { type: 'text' },
        { type: 'image', aspectRatio: '16:9' },
      ],
    },
  },
});

console.log(result.text);

const images = result.files.filter(file => file.mediaType.startsWith('image/'));
// images[0].uint8Array | images[0].base64 | images[0].mediaType
```

迭代式图像编辑天然适合有状态链接——多轮保持
`previousInteractionId`，模型可编辑上一轮
输出：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const model = google.interactions('gemini-3-pro-image-preview');

const turn1 = await generateText({
  model,
  prompt: 'Generate an image of a comic cat in a spaceship.',
  providerOptions: { google: { responseFormat: [{ type: 'image' }] } },
});

const interactionId = turn1.providerMetadata?.google?.interactionId as
  | string
  | undefined;

const turn2 = await generateText({
  model,
  prompt: 'now make the cat red',
  providerOptions: {
    google: {
      responseFormat: [{ type: 'image' }],
      previousInteractionId: interactionId,
    },
  },
});
```

<h3 id="agent-presets">Agent 预设</h3>

传入 `{ agent: <name> }` 以指定 Gemini agent 预设。工厂
会对 agent 名称做类型检查：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const result = await generateText({
  model: google.interactions({
    agent: 'deep-research-pro-preview-12-2025',
  }),
  prompt:
    'Briefly summarize the most-cited papers on retrieval-augmented generation since 2024 (2-3 sentences).',
});
```

Agent 调用在线路上以 `background: true` 运行，SDK 内部轮询
`GET /interactions/{id}` 直至 interaction
完成。默认轮询超时 30 分钟；较慢的 agent 可通过
`pollingTimeoutMs` 提高：

```ts
await generateText({
  model: google.interactions({ agent: 'deep-research-max-preview-04-2026' }),
  prompt: 'Produce a long-form research brief on ...',
  providerOptions: {
    google: {
      pollingTimeoutMs: 60 * 60 * 1000, // 1 hour
    },
  },
});
```

Agent 也可像按模型 ID 调用一样通过 `previousInteractionId` 链式接续。

### 流式

支持 `streamText`，消费七种 Interactions SSE 事件
类型（`interaction.start`、`content.start`、`content.delta`、
`content.stop`、`interaction.status_update`、`interaction.complete`、
`error`）。流的 `finish` 部分在
`providerMetadata.google` 上暴露 `interactionId`，供调用方链接。

```ts
import { google } from '@ai-sdk/google';
import { streamText } from 'ai';

const result = streamText({
  model: google.interactions('gemini-2.5-flash'),
  prompt: 'Hello, how are you?',
});

for await (const textPart of result.textStream) {
  process.stdout.write(textPart);
}

const googleMetadata = (await result.providerMetadata)?.google;
console.log('Interaction id:', googleMetadata?.interactionId);
```

## Gemma 模型

可通过 Google Generative AI API 使用 [Gemma 模型](https://deepmind.google/models/gemma/)。
可用 Gemma 模型如下：

- `gemma-3-27b-it`
- `gemma-3-12b-it`

Gemma 模型原生不支持 `systemInstruction` 参数，但提供商会自动将系统指令拼接到首条用户消息前，从而可无感使用系统指令：

```ts
import { google } from '@ai-sdk/google';
import { generateText } from 'ai';

const { text } = await generateText({
  model: google('gemma-3-27b-it'),
  system: 'You are a helpful assistant that responds concisely.',
  prompt: 'What is machine learning?',
});
```

系统指令会自动格式化并纳入对话，Gemma 模型无需额外配置即可遵循该指引。

## 嵌入模型

可使用 `.embedding()` 工厂方法创建调用 [Google Generative AI 嵌入 API](https://ai.google.dev/gemini-api/docs/embeddings) 的模型。

```ts
const model = google.embedding('gemini-embedding-001');
```

Google 提供商根据嵌入类型将 API 请求发往对应端点：

- **单条嵌入**：对单个值调用 `embed()` 时使用单条 `:embedContent` 端点，相比批量端点通常有更高速率限制。
- **批量嵌入**：对多个值使用 `embedMany()`，或在 `embed()` 中传入多个值时，使用 `:batchEmbedContents` 端点。

Google 嵌入模型支持额外设置，可通过 options 参数传入：

```ts
import { google, type GoogleEmbeddingModelOptions } from '@ai-sdk/google';
import { embed } from 'ai';

const model = google.embedding('gemini-embedding-001');

const { embedding } = await embed({
  model,
  value: 'sunny day at the beach',
  providerOptions: {
    google: {
      outputDimensionality: 512, // optional, number of dimensions for the embedding
      taskType: 'SEMANTIC_SIMILARITY', // optional, specifies the task type for generating embeddings
      content: [[{ text: 'additional context' }]], // optional, per-value multimodal content (only 1 here, since `value` is only a single one)
    } satisfies GoogleEmbeddingModelOptions,
  },
});
```

使用 `embedMany` 时，通过 `content` 选项提供按值多模态内容。每项与相同索引的值对应；纯文本项使用 `null`：

```ts
import { google, type GoogleEmbeddingModelOptions } from '@ai-sdk/google';
import { embedMany } from 'ai';

const { embeddings } = await embedMany({
  model: google.embedding('gemini-embedding-2-preview'),
  values: ['sunny day at the beach', 'rainy afternoon in the city'],
  providerOptions: {
    google: {
      // content array must have the same length as values
      content: [
        [{ inlineData: { mimeType: 'image/png', data: '<base64>' } }], // pairs with values[0]
        null, // text-only, pairs with values[1]
      ],
    } satisfies GoogleEmbeddingModelOptions,
  },
});
```

Google 嵌入模型可使用以下可选提供商选项：

- **outputDimensionality**： _number_

  可选。输出嵌入的降维后维度。若设置，输出向量末尾多余维度会被截断。

- **taskType**： _string_

  可选。指定生成嵌入的任务类型。支持的任务类型包括：
  - `SEMANTIC_SIMILARITY`：针对文本相似度优化。
  - `CLASSIFICATION`：针对文本分类优化。
  - `CLUSTERING`：针对基于相似度的文本聚类优化。
  - `RETRIEVAL_DOCUMENT`：针对文档检索优化。
  - `RETRIEVAL_QUERY`：针对查询式检索优化。
  - `QUESTION_ANSWERING`：针对问答优化。
  - `FACT_VERIFICATION`：针对事实核查优化。
  - `CODE_RETRIEVAL_QUERY`：针对根据自然语言查询检索代码块优化。

- **content**： _array_

  可选。用于嵌入非文本内容（图像、视频、PDF、音频）的按值多模态内容片段。每项与相同索引的嵌入值对应——其片段会与请求中的文本值合并。纯文本项使用 `null`。数组长度须与待嵌入值数量一致。每个非 `null` 项为片段数组，片段可为 `{ text: string }`、用于内联 base64 的 `{ inlineData: { mimeType: string, data: string } }`，或引用远程内容的 `{ fileData: { fileUri: string, mimeType: string } }`（HTTP URL 或 Google Cloud Storage URI `gs://...`）。`gemini-embedding-2-preview` 支持。

### 模型能力

| 模型                        | 默认维度 | 自定义维度   | 多模态          |
| ---------------------------- | ------------------ | ------------------- | ------------------- |
| `gemini-embedding-001`       | 3072               | <Check size={18} /> | <Cross size={18} /> |
| `gemini-embedding-2-preview` | 3072               | <Check size={18} /> | <Check size={18} /> |

## 图像模型

可使用 `.image()` 工厂方法创建调用 Google Generative AI API 的图像模型。
关于在 AI SDK 中生成图像，见 [`generateImage()`](/docs/reference/ai-sdk-core/generate-image)。

Google 提供商支持两类图像模型：

- **Imagen 模型**：使用 `:predict` API 的专用图像生成模型
- **Gemini 图像模型**：使用 `:generateContent` API、具备图像输出能力的多模态语言模型

### Imagen 模型

[Imagen](https://ai.google.dev/gemini-api/docs/imagen) 模型为专用图像生成模型。

```ts
import { google } from '@ai-sdk/google';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: google.image('imagen-4.0-generate-001'),
  prompt: 'A futuristic cityscape at sunset',
  aspectRatio: '16:9',
});
```

可通过 Google 提供商选项进一步配置。可使用 `GoogleImageModelOptions` 类型校验提供商选项。

```ts
import { google } from '@ai-sdk/google';
import { GoogleImageModelOptions } from '@ai-sdk/google';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: google.image('imagen-4.0-generate-001'),
  providerOptions: {
    google: {
      personGeneration: 'dont_allow',
    } satisfies GoogleImageModelOptions,
  },
  // ...
});
```

Imagen 模型可使用以下提供商选项：

- **personGeneration** `allow_adult` | `allow_all` | `dont_allow`
  是否允许生成人物。默认为 `allow_adult`。

<Note>
  Imagen 模型不支持 `size` 参数。请改用
  `aspectRatio` 参数。
</Note>

#### Imagen 模型能力

| 模型                           | 宽高比             |
| ------------------------------- | ------------------------- |
| `imagen-4.0-generate-001`       | 1:1, 3:4, 4:3, 9:16, 16:9 |
| `imagen-4.0-ultra-generate-001` | 1:1, 3:4, 4:3, 9:16, 16:9 |
| `imagen-4.0-fast-generate-001`  | 1:1, 3:4, 4:3, 9:16, 16:9 |

<h3 id="gemini-image-models">Gemini 图像模型</h3>

[Gemini 图像模型](https://ai.google.dev/gemini-api/docs/image-generation)（例如 `gemini-2.5-flash-image`）在技术上是多模态输出语言模型，但可与 `generateImage()` 配合以简化图像生成体验。在内部，提供商会以 `responseModalities: ['IMAGE']` 调用语言模型 API。

```ts
import { google } from '@ai-sdk/google';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: google.image('gemini-2.5-flash-image'),
  prompt: 'A photorealistic image of a cat wearing a wizard hat',
  aspectRatio: '1:1',
});
```

Gemini 图像模型也可通过提供输入图像进行图像编辑：

```ts
import { google } from '@ai-sdk/google';
import { generateImage } from 'ai';
import fs from 'node:fs';

const sourceImage = fs.readFileSync('./cat.png');

const { image } = await generateImage({
  model: google.image('gemini-2.5-flash-image'),
  prompt: {
    text: 'Add a small wizard hat to this cat',
    images: [sourceImage],
  },
});
```

输入图像也可使用 URL：

```ts
import { google } from '@ai-sdk/google';
import { generateImage } from 'ai';

const { image } = await generateImage({
  model: google.image('gemini-2.5-flash-image'),
  prompt: {
    text: 'Add a small wizard hat to this cat',
    images: ['https://example.com/cat.png'],
  },
});
```

<Note>
  Gemini 图像模型不支持 `size` 或 `n` 参数。请用
  `aspectRatio` 代替 `size`。也不支持基于遮罩的 inpainting。
</Note>

<Note>
  若需要同时输出文本与图像，或
  希望对生成流程有更多控制，可直接将 Gemini 图像模型与
  `generateText()` 配合使用。详见[图像输出](#image-outputs)。
</Note>

#### Gemini 图像模型能力

| 模型                            | 图像生成    | 图像编辑    | 宽高比                                       |
| -------------------------------- | ------------------- | ------------------- | --------------------------------------------------- |
| `gemini-2.5-flash-image`         | <Check size={18} /> | <Check size={18} /> | 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, 21:9 |
| `gemini-3-pro-image-preview`     | <Check size={18} /> | <Check size={18} /> | 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, 21:9 |
| `gemini-3.1-flash-image-preview` | <Check size={18} /> | <Check size={18} /> | 1:1, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, 21:9 |

<Note>
  `gemini-3-pro-image-preview` 还支持更多能力，包括编辑时最多 14
  张参考图（6 个物体、5 个人物）、通过 `providerOptions.google.imageConfig.imageSize` 设置分辨率（1K、
  2K、4K），以及 Google Search
  grounding。
</Note>
