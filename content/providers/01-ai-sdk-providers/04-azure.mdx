# Azure OpenAI 提供方

[Azure OpenAI](https://azure.microsoft.com/en-us/products/ai-services/openai-service) 提供方包含对 Azure OpenAI 聊天 API 的语言模型支持。

## 安装

Azure OpenAI 提供方位于 `@ai-sdk/azure` 模块。可用以下方式安装：

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add @ai-sdk/azure" dark />
  </Tab>
  <Tab>

  </Tab>
  <Tab>
    <Snippet text="yarn add @ai-sdk/azure" dark />
  </Tab>

  <Tab>

  </Tab>
</Tabs>

## 提供方实例

可从 `@ai-sdk/azure` 导入默认提供方实例 `azure`：

```ts
import { azure } from '@ai-sdk/azure';
```

如需自定义配置，可从 `@ai-sdk/azure` 导入 `createAzure`，并用你的设置创建提供方实例：

```ts
import { createAzure } from '@ai-sdk/azure';

const azure = createAzure({
  resourceName: 'your-resource-name', // Azure resource name
  apiKey: 'your-api-key',
});
```

可使用以下可选设置自定义 OpenAI 提供方实例：

- **resourceName** _string_

  Azure 资源名称。
  默认使用环境变量 `AZURE_RESOURCE_NAME`。

  资源名称会参与拼接 URL：`https://{resourceName}.openai.azure.com/openai/v1{path}`。
  也可改用 `baseURL` 指定 URL 前缀。

- **apiKey** _string_

  通过 `api-key` 请求头发送的 API 密钥。
  默认使用环境变量 `AZURE_API_KEY`。

- **apiVersion** _string_

  设置自定义 [API 版本](https://learn.microsoft.com/en-us/azure/ai-services/openai/api-version-deprecation)。
  默认为 `v1`。

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀，例如用于代理服务器。

  `baseURL` 与 `resourceName` 二选一。
  提供 `baseURL` 时会忽略 `resourceName`。

  使用 `baseURL` 时，解析后的 URL 为 `{baseURL}/v1{path}`。

- **headers** _Record&lt;string,string&gt;_

  请求中要包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch`。
  可用作中间件拦截请求，
  或在测试等场景提供自定义 fetch 实现。

- **useDeploymentBasedUrls** _boolean_

  对 API 调用使用基于部署（deployment）的 URL。设为 `true` 可使用旧版部署格式：
  `{baseURL}/deployments/{deploymentId}{path}?api-version={apiVersion}`，而非
  `{baseURL}/v1{path}?api-version={apiVersion}`。
  默认为 `false`。

  该选项有助于与仍需旧版终结点格式的某些 Azure OpenAI 模型或部署保持兼容。

## 语言模型

Azure OpenAI 提供方实例是一个函数，调用它可创建语言模型：

```ts
const model = azure('your-deployment-name');
```

第一个参数需要传入你的部署名称。

### 推理模型

Azure 会在生成文本中用 `<think>` 标签暴露 `DeepSeek-R1` 的「思考」过程。
可使用 `extractReasoningMiddleware` 提取该推理，并在结果的 `reasoning` 字段中暴露：

```ts
import { azure } from '@ai-sdk/azure';
import { wrapLanguageModel, extractReasoningMiddleware } from 'ai';

const enhancedModel = wrapLanguageModel({
  model: azure('your-deepseek-r1-deployment-name'),
  middleware: extractReasoningMiddleware({ tagName: 'think' }),
});
```

随后可在 `generateText`、`streamText` 等函数中使用该增强模型。

<Note>
  Azure 提供方默认调用 Responses API（除非你显式指定例如
  `azure.chat`）。
</Note>

### 示例

可使用 OpenAI 语言模型配合 `generateText` 生成文本：

```ts
import { azure } from '@ai-sdk/azure';
import { generateText } from 'ai';

const { text } = await generateText({
  model: azure('your-deployment-name'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

OpenAI 语言模型也可用于 `streamText`，
并支持配合 [`Output`](/docs/reference/ai-sdk-core/output) 进行结构化数据生成
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

<Note>
  Azure OpenAI 发送的数据块比 OpenAI 更大，可能让人感觉响应更慢。参见
  [故障排查：Azure OpenAI 流式输出偏慢](/docs/troubleshooting/common-issues/azure-stream-slow)。
</Note>

### 提供方选项

在 Azure 上使用 OpenAI 语言模型时，可通过 `providerOptions.openai` 配置提供方专属选项。更多可用配置见 [OpenAI 提供方页面](/providers/ai-sdk-providers/openai#language-models)。

```ts highlight="12-14,24-26"
import { azure, type OpenAILanguageModelResponsesOptions } from '@ai-sdk/azure';

const messages = [
  {
    role: 'user',
    content: [
      {
        type: 'text',
        text: 'What is the capital of the moon?',
      },
      {
        type: 'file',
        mediaType: 'image',
        data: 'https://example.com/image.png',
        providerOptions: {
          openai: { imageDetail: 'low' },
        },
      },
    ],
  },
];

const { text } = await generateText({
  model: azure('your-deployment-name'),
  providerOptions: {
    openai: {
      reasoningEffort: 'low',
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

### 聊天模型

<Note>
  调用 Azure 聊天模型的 URL 将按如下方式构造：
  `https://RESOURCE_NAME.openai.azure.com/openai/v1/chat/completions?api-version=v1`
</Note>

可使用 `.chat()` 工厂方法创建调用 Azure OpenAI 聊天补全 API 的模型：

```ts
const model = azure.chat('your-deployment-name');
```

Azure OpenAI 聊天模型还支持一些不属于[标准调用设置](/docs/ai-sdk-core/settings)的模型专属设置。
可作为 options 参数传入：

```ts
import { azure, type OpenAILanguageModelChatOptions } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure.chat('your-deployment-name'),
  prompt: 'Write a short story about a robot.',
  providerOptions: {
    openai: {
      logitBias: {
        // optional likelihood for specific tokens
        '50256': -100,
      },
      user: 'test-user', // optional unique user identifier
    } satisfies OpenAILanguageModelChatOptions,
  },
});
```

OpenAI 聊天模型可用的可选提供方选项如下：

- **logitBias** _Record&lt;number, number&gt;_

  调整指定 token 出现在补全中的相对概率。

  接受将 token（由其 GPT 分词器中的 token ID 指定）映射到 -100 到 100 之间偏置值的 JSON 对象。可使用相应分词器工具将文本转换为 token ID。数学上，该偏置会在采样前加到模型输出的 logits 上。
  具体效果因模型而异，但 -1 到 1 之间的值通常会降低或提高被选中的概率；类似 -100 或 100 的值通常会导致禁止或仅选中相关 token。

  例如可传入 `{"50256": -100}` 以防止生成该 token。

- **logprobs** _boolean | number_

  返回 token 的对数概率。包含 logprobs 会增大响应体积并可能拖慢响应，但有助于更好理解模型行为。

  设为 `true` 将返回已生成 token 的对数概率。

  设为数字将返回排名前 n 的已生成 token 的对数概率。

- **parallelToolCalls** _boolean_

  是否在工具使用时启用并行函数调用。默认为 true。

- **user** _string_

  表示终端用户的唯一标识符，有助于 OpenAI 监控与滥用检测。详见官方说明。

### Responses 模型

Azure OpenAI 默认通过 `azure(deploymentName)` 工厂方法使用 responses API。

```ts
const model = azure('your-deployment-name');
```

更多配置可通过 OpenAI 提供方选项完成。
可使用 `OpenAILanguageModelResponsesOptions` 类型校验提供方选项。

<Note>
  在 Responses API 中，`providerOptions` 里应使用 `azure` 作为提供方名称，
  而不是 `openai`。`openai` 键在 `providerOptions` 输入中仍受支持。
</Note>

```ts
import { azure, OpenAILanguageModelResponsesOptions } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('your-deployment-name'),
  providerOptions: {
    azure: {
      parallelToolCalls: false,
      store: false,
      user: 'user_123',
      // ...
    } satisfies OpenAILanguageModelResponsesOptions,
  },
  // ...
});
```

可用提供方选项如下：

- **parallelToolCalls** _boolean_
  是否使用并行工具调用。默认为 `true`。

- **store** _boolean_
  是否存储生成结果。默认为 `true`。

- **metadata** _Record&lt;string, string&gt;_
  与生成结果一并存储的额外元数据。

- **previousResponseId** _string_
  上一轮响应的 ID。可用于延续对话。默认为 `undefined`。

- **instructions** _string_
  给模型的指令。
  在使用 `previousResponseId` 延续对话时，可用于修改系统或开发者消息。
  默认为 `undefined`。

- **user** _string_
  表示终端用户的唯一标识符，有助于 OpenAI 监控与滥用检测。默认为 `undefined`。

- **reasoningEffort** _'low' | 'medium' | 'high'_
  推理模型的推理强度。默认为 `medium`。若通过 `providerOptions` 设置 `reasoningEffort`，该模型设置将被忽略。

- **strictJsonSchema** _boolean_
  是否使用严格的 JSON Schema 校验。默认为 `false`。

Azure OpenAI 提供方还会返回提供方专属元数据：

对于 Responses 模型（`azure(deploymentName)`），可使用 `AzureResponsesProviderMetadata` 为该元数据标注类型：

```ts
import { azure, type AzureResponsesProviderMetadata } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('your-deployment-name'),
});

const providerMetadata = result.providerMetadata as
  | AzureResponsesProviderMetadata
  | undefined;

const { responseId, logprobs, serviceTier } = providerMetadata?.azure ?? {};

// responseId can be used to continue a conversation (previousResponseId).
console.log(responseId);
```

可能返回的 Azure 专属元数据包括：

- **responseId** _string | null | undefined_
  响应 ID。可用于延续对话。
- **logprobs** _(optional)_
  输出 token 的对数概率（启用时）。
- **serviceTier** _(optional)_
  API 返回的服务层级信息。

<Note>
  `providerMetadata` 仅在使用默认 responses API 时返回；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

#### 联网搜索工具

Azure OpenAI responses API 通过 `azure.tools.webSearchPreview` 工具支持联网搜索（预览）。

```ts
const result = await generateText({
  model: azure('gpt-4.1-mini'),
  prompt: 'What happened in San Francisco last week?',
  tools: {
    web_search_preview: azure.tools.webSearchPreview({
      // optional configuration:
      searchContextSize: 'low',
      userLocation: {
        type: 'approximate',
        city: 'San Francisco',
        region: 'California',
      },
    }),
  },
  // Force web search tool (optional):
  toolChoice: { type: 'tool', toolName: 'web_search_preview' },
});

console.log(result.text);

// URL sources directly from `results`
const sources = result.sources;
for (const source of sources) {
  console.log('source:', source);
}
```

<Note>
  使用 Azure OpenAI 的联网搜索（预览）功能时，工具必须命名为 `web_search_preview`。该名称由 Azure OpenAI 的 API 规范要求，不可自定义。
</Note>

<Note>
  `web_search_preview` 工具仅在使用默认 responses API 时支持；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

#### 文件搜索工具

Azure OpenAI 提供方通过 `azure.tools.fileSearch` 工具支持文件搜索。

可将 `toolChoice` 设为 `{ type: 'tool', toolName: 'file_search' }` 以强制使用文件搜索工具。

```ts
const result = await generateText({

  prompt: 'What does the document say about user authentication?',
  tools: {
    file_search: azure.tools.fileSearch({
      // optional configuration:
      vectorStoreIds: ['vs_123', 'vs_456'],
      maxNumResults: 10,
      ranking: {
        ranker: 'auto',
      },
    }),
  },
  // Force file search tool:
  toolChoice: { type: 'tool', toolName: 'file_search' },
});
```

<Note>
  使用 Azure OpenAI 的文件搜索功能时，工具必须命名为 `file_search`。该名称由 Azure OpenAI 的 API 规范要求，不可自定义。
</Note>

<Note>
  `file_search` 工具仅在使用默认 responses API 时支持；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

#### 图像生成工具

Azure OpenAI 的 Responses API 将多模态图像生成作为提供方定义的工具支持。
可用性限于特定模型（例如 `gpt-5` 系列）。

```ts
import { createAzure } from '@ai-sdk/azure';
import { generateText } from 'ai';

const azure = createAzure({
  headers: {
    'x-ms-oai-image-generation-deployment': 'gpt-image-1', // use your own image model deployment
  },
});

const result = await generateText({
  model: azure('gpt-5'),
  prompt:
    'Generate an image of an echidna swimming across the Mozambique channel.',
  tools: {
    image_generation: azure.tools.imageGeneration({ outputFormat: 'png' }),
  },
});

for (const toolResult of result.staticToolResults) {
  if (toolResult.toolName === 'image_generation') {
    const base64Image = toolResult.output.result;
  }
}
```

<Note>
  使用 Azure OpenAI 的图像生成功能时，工具必须命名为 `image_generation`。该名称由 Azure OpenAI 的 API 规范要求，不可自定义。
</Note>

<Note>
  `image_generation` 工具仅在使用默认 responses API 时支持；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

<Note>
  使用 `image_generation` 前须先创建图像生成模型。必须在请求头
  `x-ms-oai-image-generation-deployment` 中添加部署说明。请注意 Responses API
  所用的模型与图像生成模型须在同一资源下。
</Note>

<Note>
  若设置 `store: false`，此前生成的图像将无法被模型访问。建议在使用图像生成工具时不要设置 `store: false`。
</Note>

#### 代码解释器工具

Azure OpenAI 提供方通过 `azure.tools.codeInterpreter` 工具支持代码解释器，允许模型编写并执行 Python 代码。

```ts
import { azure } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('gpt-5'),
  prompt: 'Write and run Python code to calculate the factorial of 10',
  tools: {
    code_interpreter: azure.tools.codeInterpreter({
      // optional configuration:
      container: {
        fileIds: ['assistant-123', 'assistant-456'], // optional file IDs to make available
      },
    }),
  },
});
```

代码解释器工具可配置项：

- **container**：容器 ID 字符串，或包含 `fileIds` 的对象，用于指定应提供给代码解释器的已上传文件

<Note>
  使用 Azure OpenAI 的代码解释器功能时，工具必须命名为 `code_interpreter`。该名称由 Azure OpenAI 的 API 规范要求，不可自定义。
</Note>

<Note>
  `code_interpreter` 工具仅在使用默认 responses API 时支持；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

<Note>
  处理代码解释器生成的文件时，可参考信息既可来自[文本片段中的注解](#typed-providermetadata-in-text-parts)，也可来自[来源文档片段中的 `providerMetadata`](#typed-providermetadata-in-source-document-parts)。
</Note>

#### PDF 支持

Azure OpenAI 提供方支持读取 PDF 文件。
可通过 `file` 类型将 PDF 作为消息内容的一部分传入：

```ts
const result = await generateText({
  model: azure('your-deployment-name'),
  messages: [
    {
      role: 'user',
      content: [
        {
          type: 'text',
          text: 'What is an embedding model?',
        },
        {
          type: 'file',
          data: fs.readFileSync('./data/ai.pdf'),
          mediaType: 'application/pdf',
          filename: 'ai.pdf', // optional
        },
      ],
    },
  ],
});
```

模型可访问 PDF 内容并回答相关问题。
PDF 应通过 `data` 字段传入，
并将 `mediaType` 设为 `'application/pdf'`。

<Note>
  读取 PDF 仅在使用默认 responses API 时支持；使用 `azure.chat` 或 `azure.completion` 时不支持。
</Note>

<h4 id="typed-providermetadata-in-text-parts">文本片段中的类型化 providerMetadata</h4>

使用 Azure OpenAI Responses API 时，SDK 会通过 `providerMetadata` 将 Azure OpenAI 专属元数据附加到输出片段。

客户端可用该元数据完成引用渲染、下载代码解释器生成文件等任务。
为类型安全处理这些元数据，AI SDK 导出了专门的 TypeScript 类型。

对文本片段，当 `part.type === 'text'` 时，`providerMetadata` 的形式为 `AzureResponsesTextProviderMetadata`。

该元数据包含以下字段：

- `itemId`  
  Responses API 中输出项的 ID。
- `annotations`（可选）
  模型生成的注解对象数组。
  若无注解，该属性本身可能省略（`undefined`）。

  `annotations` 中每一项均为带必填 `type` 字段的可区分联合。支持的类型包括例如：
  - `url_citation`
  - `file_citation`
  - `container_file_citation`
  - `file_path`

  这些注解与 Responses API 定义的注解对象一一对应，可用于行内引用展示或输出分析。
  详见 OpenAI 官方文档：
  [Responses API – 输出文本注解](https://platform.openai.com/docs/api-reference/responses/object?lang=javascript#responses-object-output-output_message-content-output_text-annotations)。

```ts
import { azure, type AzureResponsesTextProviderMetadata } from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('gpt-4.1-mini'),
  prompt:
    'Create a program that generates five random numbers between 1 and 100 with two decimal places, and show me the execution results. Also save the result to a file.',
  tools: {
    code_interpreter: azure.tools.codeInterpreter(),
    web_search_preview: azure.tools.webSearchPreview({}),
    file_search: azure.tools.fileSearch({ vectorStoreIds: ['vs_1234'] }), // requires a configured vector store
  },
});

for (const part of result.content) {
  if (part.type === 'text') {
    const providerMetadata = part.providerMetadata as
      | AzureResponsesTextProviderMetadata
      | undefined;
    if (!providerMetadata) continue;
    const { itemId: _itemId, annotations } = providerMetadata.azure;

    if (!annotations) continue;
    for (const annotation of annotations) {
      switch (annotation.type) {
        case 'url_citation':
          // url_citation is returned from web_search and provides:
          // properties: type, url, title, start_index and end_index
          break;
        case 'file_citation':
          // file_citation is returned from file_search and provides:
          // properties: type, file_id, filename and index
          break;
        case 'container_file_citation':
          // container_file_citation is returned from code_interpreter and provides:
          // properties: type, container_id, file_id, filename, start_index and end_index
          break;
        case 'file_path':
          // file_path provides:
          // properties: type, file_id and index
          break;
        default: {
          const _exhaustiveCheck: never = annotation;
          throw new Error(
            `Unhandled annotation: ${JSON.stringify(_exhaustiveCheck)}`,
          );
        }
      }
    }
  }
}
```

<Note>
  为实现代码解释器生成文件的下载，可使用 `providerMetadata` 中的 `container_id` 与 `file_id` 获取文件内容。详见 [Retrieve container file
  content](https://platform.openai.com/docs/api-reference/container-files/retrieveContainerFileContent) API。
</Note>

#### 推理片段中的类型化 providerMetadata

使用 Azure OpenAI Responses API 时，推理输出片段可包含提供方元数据。
要以类型安全方式处理，请使用 `AzureResponsesReasoningProviderMetadata`。

对推理片段，当 `part.type === 'reasoning'` 时，`providerMetadata` 的形式为 `AzureResponsesReasoningProviderMetadata`。

该元数据包含以下字段：

- `itemId`  
  Responses API 中推理项的 ID。
- `reasoningEncryptedContent`（可选）  
  加密的推理内容（仅在通过 `include: ['reasoning.encrypted_content']` 请求时返回）。

```ts
import {
  azure,
  type AzureResponsesReasoningProviderMetadata,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('your-deployment-name'),
  prompt: 'How many "r"s are in the word "strawberry"?',
  providerOptions: {
    azure: {
      store: false,
      include: ['reasoning.encrypted_content'],
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

for (const part of result.content) {
  if (part.type === 'reasoning') {
    const providerMetadata = part.providerMetadata as
      | AzureResponsesReasoningProviderMetadata
      | undefined;

    const { itemId, reasoningEncryptedContent } = providerMetadata?.azure ?? {};
    console.log(itemId, reasoningEncryptedContent);
  }
}
```

<h4 id="typed-providermetadata-in-source-document-parts">来源文档片段中的类型化 providerMetadata</h4>

对来源文档片段，当 `part.type === 'source'` 且 `sourceType === 'document'` 时，`providerMetadata` 的形式为 `AzureResponsesSourceDocumentProviderMetadata`。

该元数据同样是带必填 `type` 字段的可区分联合。支持的类型包括：

- `file_citation`
- `container_file_citation`
- `file_path`

每种类型包含处理所引用资源所需的标识符，例如 `fileId` 与 `containerId`。

```ts
import {
  azure,
  type AzureResponsesSourceDocumentProviderMetadata,
} from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure('gpt-4.1-mini'),
  prompt:
    'Create a program that generates five random numbers between 1 and 100 with two decimal places, and show me the execution results. Also save the result to a file.',
  tools: {
    code_interpreter: azure.tools.codeInterpreter(),
    web_search_preview: azure.tools.webSearchPreview({}),
    file_search: azure.tools.fileSearch({ vectorStoreIds: ['vs_1234'] }), // requires a configured vector store
  },
});

for (const part of result.content) {
  if (part.type === 'source') {
    if (part.sourceType === 'document') {
      const providerMetadata = part.providerMetadata as
        | AzureResponsesSourceDocumentProviderMetadata
        | undefined;
      if (!providerMetadata) continue;
      const annotation = providerMetadata.azure;
      switch (annotation.type) {
        case 'file_citation':
          // file_citation is returned from file_search and provides:
          // properties: type, fileId and index
          // The filename can be accessed via part.filename.
          break;
        case 'container_file_citation':
          // container_file_citation is returned from code_interpreter and provides:
          // properties: type, containerId and fileId
          // The filename can be accessed via part.filename.
          break;
        case 'file_path':
          // file_path provides:
          // properties: type, fileId and index
          break;
        default: {
          const _exhaustiveCheck: never = annotation;
          throw new Error(
            `Unhandled annotation: ${JSON.stringify(_exhaustiveCheck)}`,
          );
        }
      }
    }
  }
}
```

<Note>
  文本片段中的注解遵循 OpenAI Responses API 规范，因此使用 snake_case 属性（如 `file_id`、`container_id`）。相较之下，来源文档片段的 `providerMetadata` 由 SDK 规范化为 camelCase（如 `fileId`、`containerId`）。依赖原始文本内容的字段（如 `start_index`、`end_index`）会省略，在来源对象上可直接获取的字段（如 `filename`）亦会省略。
</Note>

### 补全模型

可使用 `.completion()` 工厂方法创建调用补全 API 的模型。
第一个参数为模型 id。
目前仅支持 `gpt-35-turbo-instruct`。

```ts
const model = azure.completion('your-gpt-35-turbo-instruct-deployment');
```

OpenAI 补全模型还支持一些不属于[标准调用设置](/docs/ai-sdk-core/settings)的模型专属设置。
可作为 options 参数传入：

```ts
import {
  azure,
  type OpenAILanguageModelCompletionOptions,
} from '@ai-sdk/azure';
import { generateText } from 'ai';

const result = await generateText({
  model: azure.completion('your-gpt-35-turbo-instruct-deployment'),
  prompt: 'Write a haiku about coding.',
  providerOptions: {
    openai: {
      echo: true, // optional, echo the prompt in addition to the completion
      logitBias: {
        // optional likelihood for specific tokens
        '50256': -100,
      },
      suffix: 'some text', // optional suffix that comes after a completion of inserted text
      user: 'test-user', // optional unique user identifier
    } satisfies OpenAILanguageModelCompletionOptions,
  },
});
```

Azure OpenAI 补全模型可用的可选提供方选项如下：

- **echo**: _boolean_

  除补全外是否回显提示（prompt）。

- **logitBias** _Record&lt;number, number&gt;_

  调整指定 token 出现在补全中的相对概率。

  接受将 token（由其 GPT 分词器中的 token ID 指定）映射到 -100 到 100 之间偏置值的 JSON 对象。可使用相应分词器工具将文本转换为 token ID。数学上，该偏置会在采样前加到模型输出的 logits 上。
  具体效果因模型而异，但 -1 到 1 之间的值通常会降低或提高被选中的概率；类似 -100 或 100 的值通常会导致禁止或仅选中相关 token。

  例如可传入 `{"50256": -100}` 以防止生成 &lt;|endoftext|&gt; token。

- **logprobs** _boolean | number_

  返回 token 的对数概率。包含 logprobs 会增大响应体积并可能拖慢响应，但有助于更好理解模型行为。

  设为 `true` 将返回已生成 token 的对数概率。

  设为数字将返回排名前 n 的已生成 token 的对数概率。

- **suffix** _string_

  插入文本补全之后的后缀文本。

- **user** _string_

  表示终端用户的唯一标识符，有助于 OpenAI 监控与滥用检测。详见官方说明。

## 语音转写模型

可使用 `.transcription()` 工厂方法创建调用 Azure OpenAI 语音转写 API 的模型。

第一个参数为模型 id，例如 `whisper-1`。

```ts
const model = azure.transcription('whisper-1');
```

<Note>
  若使用转写模型时出现 “DeploymentNotFound” 错误，
  可尝试启用基于部署的 URL：

```ts
const azure = createAzure({
  useDeploymentBasedUrls: true,
  apiVersion: '2025-04-01-preview',
});
```

这会使用旧版终结点格式，某些 Azure OpenAI 部署可能需要。
使用 `useDeploymentBasedUrls` 时，默认的 api-version 无效，须设为 `2025-04-01-preview` 或更早版本。

</Note>

还可通过 `providerOptions` 传入更多提供方专属选项。例如以 ISO-639-1 格式提供输入语言（如 `en`）可提升准确度并降低延迟。

```ts highlight="6"
import { experimental_transcribe as transcribe } from 'ai';
import { azure, type OpenAITranscriptionModelOptions } from '@ai-sdk/azure';
import { readFile } from 'fs/promises';

const result = await transcribe({
  model: azure.transcription('whisper-1'),
  audio: await readFile('audio.mp3'),
  providerOptions: {
    openai: {
      language: 'en',
    } satisfies OpenAITranscriptionModelOptions,
  },
});
```

可用提供方选项如下：

- **timestampGranularities** _string[]_
  转写中时间戳的粒度。
  默认为 `['segment']`。
  可选值为 `['word']`、`['segment']` 以及 `['word', 'segment']`。
  说明：分段时间戳不会带来额外延迟，但生成词级时间戳会产生额外延迟。

- **language** _string_
  输入音频的语言。以 ISO-639-1 格式（如 `'en'`）提供可提升准确度与延迟表现。
  可选。

- **prompt** _string_
  可选文本，用于引导模型风格或延续上一段音频。提示语言应与音频语言一致。
  可选。

- **temperature** _number_
  采样温度，介于 0 与 1 之间。较高值（如 0.8）使输出更随机，较低值（如 0.2）更集中、确定性更强。设为 0 时，模型将使用对数概率自动调高温度直至达到某些阈值。
  默认为 0。
  可选。

- **include** _string[]_
  转写响应中要包含的额外信息。

## 语音合成模型

可使用 `.speech()` 工厂方法创建调用 Azure OpenAI 语音合成 API 的模型。

第一个参数为文本转语音模型的部署名称（例如 `tts-1`）。

```ts
const model = azure.speech('your-tts-deployment-name');
```

### 示例

```ts
import { azure } from '@ai-sdk/azure';
import { experimental_generateSpeech as generateSpeech } from 'ai';

const result = await generateSpeech({
  model: azure.speech('your-tts-deployment-name'),
  text: 'Hello, world!',
  voice: 'alloy', // OpenAI voice ID
});
```

还可通过 `providerOptions` 传入更多提供方专属选项：

```ts
import { azure, type OpenAISpeechModelOptions } from '@ai-sdk/azure';
import { experimental_generateSpeech as generateSpeech } from 'ai';

const result = await generateSpeech({
  model: azure.speech('your-tts-deployment-name'),
  text: 'Hello, world!',
  voice: 'alloy',
  providerOptions: {
    openai: {
      speed: 1.2,
    } satisfies OpenAISpeechModelOptions,
  },
});
```

可用提供方选项如下：

- **instructions** _string_
  通过额外说明控制合成音频的音色，例如：“用缓慢平稳的语调朗读”。
  不适用于 `tts-1` 或 `tts-1-hd`。
  可选。

- **speed** _number_
  合成音频的速度。
  取值范围为 0.25 至 4.0。
  默认为 1.0。
  可选。
