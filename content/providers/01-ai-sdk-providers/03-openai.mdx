# OpenAI 提供商

[OpenAI](https://openai.com/) 提供商为 OpenAI 的 responses、chat 与 completion API 提供语言模型支持，并为 OpenAI embeddings API 提供嵌入模型支持。

## 安装

OpenAI 提供商位于 `@ai-sdk/openai` 模块中。可使用以下命令安装：

## 提供商实例

可从 `@ai-sdk/openai` 导入默认提供商实例 `openai`：

```ts
import { openai } from '@ai-sdk/openai';
```

如需自定义配置，可从 `@ai-sdk/openai` 导入 `createOpenAI`，并按你的设置创建提供商实例：

```ts
import { createOpenAI } from '@ai-sdk/openai';

const openai = createOpenAI({
  // custom settings, e.g.
  headers: {
    'header-name': 'header-value',
  },
});
```

可使用以下可选设置自定义 OpenAI 提供商实例：

- **baseURL** _string_

  为 API 调用使用不同的 URL 前缀，例如用于代理服务器。
  默认前缀为 `https://api.openai.com/v1`。

- **apiKey** _string_

  通过 `Authorization` 请求头发送的 API 密钥。
  默认读取环境变量 `OPENAI_API_KEY`。

- **name** _string_

  提供商名称。在使用 OpenAI 兼容提供商时可设置此项，
  以更改模型提供商属性。默认为 `openai`。

- **organization** _string_

  OpenAI 组织。

- **project** _string_

  OpenAI 项目。

- **headers** _Record&lt;string,string&gt;_

  请求中包含的自定义请求头。

- **fetch** _(input: RequestInfo, init?: RequestInit) => Promise&lt;Response&gt;_

  自定义 [fetch](https://developer.mozilla.org/en-US/docs/Web/API/fetch) 实现。
  默认为全局 `fetch` 函数。
  可作为中间件拦截请求，
  或在测试等场景提供自定义 fetch 实现。

## 语言模型

OpenAI 提供商实例是一个函数，调用它即可创建语言模型：

```ts
const model = openai('gpt-5');
```

它会根据模型 ID 自动选择正确的 API。
你也可以在第二个参数中传入额外设置：

```ts
const model = openai('gpt-5', {
  // additional settings
});
```

可用选项取决于为模型自动选择的 API（见下文）。
若要显式选择某一模型 API，可使用 `.responses`、`.chat` 或 `.completion`。

<Note>
  自 AI SDK 5 起，默认调用 OpenAI responses API（除非你
  指定例如 `openai.chat`）
</Note>

### 示例

可使用 OpenAI 语言模型配合 `generateText` 函数生成文本：

```ts
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: openai('gpt-5'),
  prompt: 'Write a vegetarian lasagna recipe for 4 people.',
});
```

OpenAI 语言模型也可用于 `streamText` 函数，
并支持使用 [`Output`](/docs/reference/ai-sdk-core/output) 生成结构化数据
（参见 [AI SDK Core](/docs/ai-sdk-core)）。

### Responses 模型

可通过工厂方法 `openai(modelId)` 或 `openai.responses(modelId)` 使用 OpenAI responses API。这是 OpenAI 提供商默认使用的 API（自 AI SDK 5 起）。

```ts
const model = openai('gpt-5');
```

可使用 OpenAI 提供商选项进行进一步配置。
可使用类型 `OpenAILanguageModelResponsesOptions` 校验提供商选项。

```ts
import { openai, OpenAILanguageModelResponsesOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'), // or openai.responses('gpt-5')
  providerOptions: {
    openai: {
      parallelToolCalls: false,
      store: false,
      user: 'user_123',
      // ...
    } satisfies OpenAILanguageModelResponsesOptions,
  },
  // ...
});
```

提供以下提供商选项：

- **parallelToolCalls** _boolean_
  是否使用并行工具调用。默认为 `true`。

- **store** _boolean_

  是否存储生成结果。默认为 `true`。

- **passThroughUnsupportedFiles** _boolean_

  是否将非图像文件类型作为通用输入文件透传。默认为 `false`，会将内联文件输入限制为图像与 PDF。当目标 OpenAI Responses 模型支持更多文件媒体类型时可启用此项。

- **maxToolCalls** _integer_
  单次响应中可处理的内置工具调用总次数上限。
  该上限适用于所有内置工具调用之和，而非每个工具单独计数。
  模型此后尝试调用工具将被忽略。

- **metadata** _Record&lt;string, string&gt;_
  与生成结果一并存储的额外元数据。

- **conversation** _string_
  要继续的 OpenAI Conversation ID。
  必须先通过 [OpenAI API](https://platform.openai.com/docs/api-reference/conversations/create) 创建会话。
  不能与 `previousResponseId` 同时使用。
  默认为 `undefined`。

- **previousResponseId** _string_
  上一次响应的 ID，可用于延续对话。默认为 `undefined`。

- **instructions** _string_
  模型指令。
  在使用 `previousResponseId` 选项延续对话时，可用于更改系统或开发者消息。
  默认为 `undefined`。

- **logprobs** _boolean | number_
  返回 token 的对数概率。启用 logprobs 会增加响应体积并可能拖慢响应时间，但有助于理解模型行为。设为 `true` 返回已生成 token 的对数概率；设为数字（1–20）返回已生成 token 中 top n 的对数概率。

- **user** _string_
  表示终端用户的唯一标识符，可帮助 OpenAI 监控并检测滥用。默认为 `undefined`。

- **reasoningEffort** _'none' | 'minimal' | 'low' | 'medium' | 'high' | 'xhigh'_
  推理模型的推理强度。默认为 `medium`。若通过 `providerOptions` 设置 `reasoningEffort`，则忽略此模型级设置。

<Note>
  `reasoningEffort` 的 `'none'` 仅适用于 OpenAI 的 GPT-5.1
  系列模型。`'xhigh'` 仅适用于
  OpenAI 的 GPT-5.1-Codex-Max 模型。在不支持的模型上将 `reasoningEffort` 设为 `'none'` 或
  `'xhigh'` 会导致错误。
</Note>

- **reasoningSummary** _'auto' | 'detailed'_
  控制模型是否返回推理过程。`'auto'` 为精简摘要，`'detailed'` 为更完整的推理。默认为 `undefined`（无推理摘要）。启用后，流式输出中会以类型为 `'reasoning'` 的事件出现推理摘要；非流式响应中则在 `reasoning` 字段中返回。

- **strictJsonSchema** _boolean_
  是否使用严格的 JSON schema 校验。默认为 `true`。

<Note type="warning">
  OpenAI 结构化输出存在多项
  [限制](https://openai.com/index/introducing-structured-outputs-in-the-api)，
  尤其在 [支持的
  schema](https://platform.openai.com/docs/guides/structured-outputs/supported-schemas) 方面，
  因此属于可选开启能力。例如，不支持可选 schema 属性。
  需要将 Zod 的 `.nullish()` 和 `.optional()` 改为
  `.nullable()`。
</Note>

- **serviceTier** _'auto' | 'flex' | 'priority' | 'default'_
  请求的服务层级。设为 `'flex'` 可降低约 50% 费用，
  但延迟更高（适用于 o3、o4-mini 与 gpt-5 等模型）。
  设为 `'priority'` 可在企业访问下获得更快处理（适用于 gpt-4、gpt-5、gpt-5-mini、o3、o4-mini；不支持 gpt-5-nano）。

  默认为 `'auto'`。

- **textVerbosity** _'low' | 'medium' | 'high'_
  控制模型回复的详略。较低值更简洁，
  较高值更冗长。默认为 `'medium'`。

- **include** _Array&lt;string&gt;_
  指定响应中包含的额外内容。支持的值：
  `['file_search_call.results']` 用于在响应中包含文件搜索结果。
  `['message.output_text.logprobs']` 用于 logprobs。
  默认为 `undefined`。

- **truncation** _string_
  模型响应采用的截断策略。
  - Auto：若本 Response 的输入超过模型上下文窗口，模型会通过丢弃对话开头的条目来截断响应以适配上下文窗口。
  - disabled（默认）：若输入将超过模型的上下文窗口，请求会以 400 错误失败。

- **promptCacheKey** _string_
  用于手动控制提示缓存的缓存键。OpenAI 用它缓存相似请求的响应以提升缓存命中率。

- **promptCacheRetention** _'in_memory' | '24h'_
  提示缓存的保留策略。设为 `'24h'` 可启用延长提示缓存，使缓存前缀最长保留 24 小时。标准提示缓存默认为 `'in_memory'`。注意：`'24h'` 目前仅适用于 5.1 系列模型。

- **safetyIdentifier** _string_
  用于协助识别可能违反 OpenAI 使用政策的用户的稳定标识符。ID 应为可唯一标识每个用户的字符串。

- **systemMessageMode** _'system' | 'developer' | 'remove'_
  控制发起请求时系统消息的角色。默认（省略时），对支持推理的模型，`system` 消息会自动转换为 `developer` 消息。`systemMessageMode` 为 `system` 时以系统级指令传递系统消息；`developer` 时作为开发者消息传递；`remove` 则从请求中省略系统消息。

- **forceReasoning** _boolean_
  强制将本模型视为推理模型。适用于「隐匿」推理模型（例如通过自定义 baseURL），其模型 ID 未被 SDK 白名单识别的情况。启用后，SDK 会应用推理模型的参数兼容规则，并在未覆盖时将 `systemMessageMode` 默认为 `developer`。

- **contextManagement** _Array&lt;object&gt;_
  启用服务端上下文管理（压缩 compaction）。配置后，当 token 使用量超过指定阈值时，服务器会自动压缩对话上下文。数组中每个对象应包含：
  - `type`：`'compaction'`
  - `compactThreshold`：_number_ — 触发压缩时的 token 数量

OpenAI responses 提供商还会返回提供商特定的元数据：

对于 Responses 模型，可使用 `OpenaiResponsesProviderMetadata` 为该元数据标注类型：

```ts
import { openai, type OpenaiResponsesProviderMetadata } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'),
});

const providerMetadata = result.providerMetadata as
  | OpenaiResponsesProviderMetadata
  | undefined;

const { responseId, logprobs, serviceTier } = providerMetadata?.openai ?? {};

// responseId can be used to continue a conversation (previousResponseId).
console.log(responseId);
```

可能返回以下 OpenAI 特定元数据：

- **responseId** _string | null | undefined_
  响应 ID，可用于延续对话。
- **logprobs** _（可选）_
  输出 token 的对数概率（启用时）。
- **serviceTier** _（可选）_
  API 返回的服务层级信息。

#### 推理输出

对于 `gpt-5` 等推理模型，可启用推理摘要以查看模型的思考过程。不同模型支持的摘要器不同——例如 `o4-mini` 支持详细摘要。将 `reasoningSummary` 设为 `"auto"` 可自动获得当前可用的最高丰富度。

```ts highlight="8-9,16"
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { streamText } from 'ai';

const result = streamText({
  model: openai('gpt-5'),
  prompt: 'Tell me about the Mission burrito debate in San Francisco.',
  providerOptions: {
    openai: {
      reasoningSummary: 'detailed', // 'auto' for condensed or 'detailed' for comprehensive
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

for await (const part of result.fullStream) {
  if (part.type === 'reasoning') {
    console.log(`Reasoning: ${part.textDelta}`);
  } else if (part.type === 'text-delta') {
    process.stdout.write(part.textDelta);
  }
}
```

对于使用 `generateText` 的非流式调用，推理摘要可在响应的 `reasoning` 字段中获取：

```ts highlight="8-9,13"
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'Tell me about the Mission burrito debate in San Francisco.',
  providerOptions: {
    openai: {
      reasoningSummary: 'auto',
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
console.log('Reasoning:', result.reasoning);
```

有关推理摘要的更多说明见 [OpenAI 文档](https://platform.openai.com/docs/guides/reasoning?api-mode=responses#reasoning-summaries)。

#### WebSocket 传输

OpenAI 的 [WebSocket API](https://developers.openai.com/api/docs/guides/websocket-mode) 维持持久连接，可显著
降低包含大量工具调用的智能体工作流中的首字节时间（TTFB）。
建立初始连接后，后续请求可完全跳过 TCP/TLS/HTTP 协商。

[`ai-sdk-openai-websocket-fetch`](https://www.npmjs.com/package/ai-sdk-openai-websocket-fetch)
包提供可替换的 `fetch` 实现，通过持久 WebSocket 连接路由流式请求。

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab>
    <Snippet text="pnpm add ai-sdk-openai-websocket-fetch" dark />
  </Tab>
  <Tab>
    <Snippet text="npm install ai-sdk-openai-websocket-fetch" dark />
  </Tab>
  <Tab>
    <Snippet text="yarn add ai-sdk-openai-websocket-fetch" dark />
  </Tab>
  <Tab>
    <Snippet text="bun add ai-sdk-openai-websocket-fetch" dark />
  </Tab>
</Tabs>

通过 `fetch` 选项将 WebSocket fetch 传给 `createOpenAI`：

```ts highlight="2,6-7,15"
import { createOpenAI } from '@ai-sdk/openai';
import { createWebSocketFetch } from 'ai-sdk-openai-websocket-fetch';
import { streamText } from 'ai';

// Create a WebSocket-backed fetch instance
const wsFetch = createWebSocketFetch();
const openai = createOpenAI({ fetch: wsFetch });

const result = streamText({
  model: openai('gpt-4.1-mini'),
  prompt: 'Hello!',
  tools: {
    // ...
  },
  onFinish: () => wsFetch.close(), // close the WebSocket when done
});
```

首个请求会较慢，因为需要建立 WebSocket 连接
（DNS + TCP + TLS + WebSocket 升级）。之后，多步工具调用循环中的后续步骤可复用已打开的连接，从而降低每步的 TTFB。

<Note>
  WebSocket 传输仅将通过 WebSocket 路由发往 OpenAI Responses
  API 的流式请求（`POST /responses` 且 `stream: true`）。
  其余请求（非流式、嵌入等）会回退到标准
  `fetch` 实现。
</Note>

可在 [演示应用](https://github.com/vercel-labs/ai-sdk-openai-websocket) 中查看 HTTP 与 WebSocket 流式性能的并排对比。

#### 详细程度控制

可使用 `textVerbosity` 参数控制模型回复的长度与细节：

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Write a poem about a boy and his first pet dog.',
  providerOptions: {
    openai: {
      textVerbosity: 'low', // 'low' for concise, 'medium' (default), or 'high' for verbose
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

`textVerbosity` 参数在不改变底层提示的前提下调节输出长度：

- `'low'`：生成简短、极简回复
- `'medium'`：细节适中（默认）
- `'high'`：冗长回复，细节更全面

#### 网页搜索工具

OpenAI responses API 通过 `openai.tools.webSearch` 工具支持网页搜索。

```ts
const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'What happened in San Francisco last week?',
  tools: {
    web_search: openai.tools.webSearch({
      // optional configuration:
      externalWebAccess: true,
      searchContextSize: 'high',
      userLocation: {
        type: 'approximate',
        city: 'San Francisco',
        region: 'California',
      },
      filters: {
        allowedDomains: ['sfchronicle.com', 'sfgate.com'],
      },
    }),
  },
  // Force web search tool (optional):
  toolChoice: { type: 'tool', toolName: 'web_search' },
});

// URL sources directly from `results`
const sources = result.sources;

// Or access sources from tool results
for (const toolResult of result.toolResults) {
  if (toolResult.toolName === 'web_search') {
    console.log('Query:', toolResult.output.action.query);
    console.log('Sources:', toolResult.output.sources);
    // `sources` is an array of object: { type: 'url', url: string }
  }
}
```

网页搜索工具支持以下配置选项：

- **externalWebAccess** _boolean_ — 是否使用外部网页访问获取实时内容。默认为 `true`。
- **searchContextSize** _'low' | 'medium' | 'high'_ — 控制搜索所用上下文量。更高值通常结果更全面，但延迟与成本可能更高。
- **userLocation** — 可选的位置信息，用于地理位置相关结果。包含 `type`（始终为 `'approximate'`）、`country`、`city`、`region` 与 `timezone`。
- **filters** — 可选过滤配置以限制搜索结果。
  - **allowedDomains** _string[]_ — 允许的域名数组。所提供域名的子域名会自动包含在内。

配置选项的详细说明见 [OpenAI 网页搜索工具文档](https://platform.openai.com/docs/guides/tools-web-search?api-mode=responses)。

#### 文件搜索工具

OpenAI responses API 通过 `openai.tools.fileSearch` 工具支持文件搜索。

可将 `toolChoice` 设为 `{ type: 'tool', toolName: 'file_search' }` 以强制使用文件搜索工具。

```ts
const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'What does the document say about user authentication?',
  tools: {
    file_search: openai.tools.fileSearch({
      vectorStoreIds: ['vs_123'],
      // configuration below is optional:
      maxNumResults: 5,
      filters: {
        key: 'author',
        type: 'eq',
        value: 'Jane Smith',
      },
      ranking: {
        ranker: 'auto',
        scoreThreshold: 0.5,
      },
    }),
  },
  providerOptions: {
    openai: {
      // optional: include results
      include: ['file_search_call.results'],
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

文件搜索工具支持比较过滤器与复合过滤器：

**比较过滤器** — 按单一属性过滤：

- `eq` — 等于
- `ne` — 不等于
- `gt` — 大于
- `gte` — 大于等于
- `lt` — 小于
- `lte` — 小于等于
- `in` — 值在数组中
- `nin` — 值不在数组中

```ts
// Single comparison filter
filters: { key: 'year', type: 'gte', value: 2023 }

// Filter with array values
filters: { key: 'status', type: 'in', value: ['published', 'reviewed'] }
```

**复合过滤器** — 使用 `and` 或 `or` 组合多个过滤器：

```ts
// Compound filter with AND
filters: {
  type: 'and',
  filters: [
    { key: 'author', type: 'eq', value: 'Jane Smith' },
    { key: 'year', type: 'gte', value: 2023 },
  ],
}

// Compound filter with OR
filters: {
  type: 'or',
  filters: [
    { key: 'department', type: 'eq', value: 'Engineering' },
    { key: 'department', type: 'eq', value: 'Research' },
  ],
}
```

#### 图像生成工具

OpenAI Responses API 将多模态图像生成作为提供商定义的工具支持。
可用性限于特定模型（例如 `gpt-5` 系列）。

可使用 `generateText` 或 `streamText` 配合图像工具：

```ts
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'),
  prompt:
    'Generate an image of an echidna swimming across the Mozambique channel.',
  tools: {
    image_generation: openai.tools.imageGeneration({ outputFormat: 'webp' }),
  },
});

for (const toolResult of result.staticToolResults) {
  if (toolResult.toolName === 'image_generation') {
    const base64Image = toolResult.output.result;
  }
}
```

```ts
import { openai } from '@ai-sdk/openai';
import { streamText } from 'ai';

const result = streamText({
  model: openai('gpt-5'),
  prompt:
    'Generate an image of an echidna swimming across the Mozambique channel.',
  tools: {
    image_generation: openai.tools.imageGeneration({
      outputFormat: 'webp',
      quality: 'low',
    }),
  },
});

for await (const part of result.fullStream) {
  if (part.type == 'tool-result' && !part.dynamic) {
    const base64Image = part.output.result;
  }
}
```

<Note>
  若设置 `store: false`，此前生成的图像将无法被
  模型访问。建议在使用图像生成工具时不要
  设置 `store: false`。
</Note>

关于模型可用性、画质控制、支持尺寸及工具专属参数的完整说明，
请参阅 OpenAI 文档：

- 图像生成概览与模型：[OpenAI 图像生成](https://platform.openai.com/docs/guides/image-generation)
- 图像生成工具参数（背景、尺寸、画质、格式等）：[图像生成工具选项](https://platform.openai.com/docs/guides/tools-image-generation#tool-options)

#### MCP 工具

OpenAI responses API 支持通过 `openai.tools.mcp` 工具连接 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 服务器。模型由此可调用远程 MCP 服务器或服务连接器暴露的工具。

```ts
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'Search the web for the latest news about AI developments',
  tools: {
    mcp: openai.tools.mcp({
      serverLabel: 'web-search',
      serverUrl: 'https://mcp.exa.ai/mcp',
      serverDescription: 'A web-search API for AI agents',
    }),
  },
});
```

MCP 工具可配置项：

- **serverLabel** _string_（必填）

  用于标识 MCP 服务器的标签。该标签在工具调用中用于区分多个 MCP 服务器。

- **serverUrl** _string_（若未提供 `connectorId` 则必填）

  MCP 服务器 URL。必须提供 `serverUrl` 或 `connectorId` 之一。

- **connectorId** _string_（若未提供 `serverUrl` 则必填）

  服务连接器标识符。必须提供 `serverUrl` 或 `connectorId` 之一。

- **serverDescription** _string_（可选）

  MCP 服务器的可选描述，帮助模型理解其用途。

- **allowedTools** _string[] | object_（可选）

  控制 MCP 服务器上哪些工具可用。可为：
  - 工具名称数组：`['tool1', 'tool2']`
  - 带过滤条件的对象：
    ```ts
    {
      readOnly: true, // Only allow read-only tools
      toolNames: ['tool1', 'tool2'] // Specific tool names
    }
    ```

- **authorization** _string_（可选）

  用于向 MCP 服务器或连接器认证的 OAuth 访问令牌。

- **headers** _Record&lt;string, string&gt;_（可选）

  发往 MCP 服务器的请求中可选的 HTTP 请求头。

- **requireApproval** _'always' | 'never' | object_（可选）

  控制哪些 MCP 工具调用在执行前需要用户批准。可为：
  - `'always'`：所有 MCP 工具调用都需要批准
  - `'never'`：所有 MCP 工具调用都不需要批准（默认）
  - 带过滤条件的对象：
    ```ts
    {
      never: {
        toolNames: ['safe_tool', 'another_safe_tool']; // Skip approval for these tools
      }
    }
    ```

  需要批准时，模型会返回 `tool-approval-request` 内容片段，可用于提示用户批准。实现审批流程的更多说明见 [Human in the Loop](/cookbook/next/human-in-the-loop)。

<Note>
  未设置 `requireApproval` 时，工具调用默认会被批准。请确保
  仅连接你信任的 MCP 服务器，并放心与之共享数据。
</Note>

<Note>
  OpenAI MCP 工具与 [MCP Tools](/docs/ai-sdk-core/mcp-tools) 中描述的通用 MCP 客户端方式不同。OpenAI MCP 工具是
  内置的提供商定义工具，允许 OpenAI 模型直接连接
  MCP 服务器；而通用 MCP 客户端需要先将 MCP 工具
  转换为 AI SDK 工具。
</Note>

const result = await generateText({

#### Shell 工具

OpenAI Responses API 通过 `openai.tools.shell` 工具支持 shell。
shell 工具可运行 bash 命令并与命令行交互。
模型提议 shell 命令；你的集成负责执行并返回输出。

<Note type="warning">
  运行任意 shell 命令可能有风险。在将命令交给系统 shell 执行前，请务必沙箱化执行或
  添加严格的允许/拒绝列表。
</Note>

shell 工具支持三种环境模式，决定命令在何处执行：

##### 本地执行（默认）

未指定 `environment`（或使用 `type: 'local'`）时，通过你的 `execute` 回调在本地执行命令：

```ts
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      execute: async ({ action }) => {
        // ... your implementation, e.g. sandbox access ...
        return { output: results };
      },
    }),
  },
  prompt: 'List the files in the current directory and show disk usage.',
});
```

##### 托管容器（auto）

将 `environment.type` 设为 `'containerAuto'` 可在 OpenAI 托管容器中运行命令。无需 `execute` 回调——由 OpenAI 在服务端执行：

```ts
const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      environment: {
        type: 'containerAuto',
        // optional configuration:
        memoryLimit: '4g',
        fileIds: ['file-abc123'],
        networkPolicy: {
          type: 'allowlist',
          allowedDomains: ['example.com'],
        },
      },
    }),
  },
  prompt: 'Install numpy and compute the eigenvalues of a 3x3 matrix.',
});
```

`containerAuto` 环境支持：

- **fileIds** _string[]_ — 要在容器中可用的文件 ID
- **memoryLimit** _'1g' | '4g' | '16g' | '64g'_ — 容器内存上限
- **networkPolicy** — 网络访问策略：
  - `{ type: 'disabled' }` — 无网络访问
  - `{ type: 'allowlist', allowedDomains: string[], domainSecrets?: Array<{ domain, name, value }> }` — 允许特定域名，并可附带密钥

##### 引用已有容器

将 `environment.type` 设为 `'containerReference'` 可按 ID 使用已有容器：

```ts
const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      environment: {
        type: 'containerReference',
        containerId: 'cntr_abc123',
      },
    }),
  },
  prompt: 'Check the status of running processes.',
});
```

##### execute 回调

本地执行（默认或 `type: 'local'`）时，execute 函数必须为每条命令返回包含结果的输出数组：

- **stdout** _string_ — 命令的标准输出
- **stderr** _string_ — 命令的标准错误
- **outcome** — `{ type: 'timeout' }` 或 `{ type: 'exit', exitCode: number }`

##### Skills

[Skills](https://platform.openai.com/docs/guides/tools-skills) 是带 `SKILL.md` 清单的版本化文件包，用于扩展 shell 工具能力。可附加到 `containerAuto` 与 `local` 环境。

**容器 skills** 支持两种格式——引用（适用于已上传到 OpenAI 的 skill）或内联（base64 编码的 zip）：

```ts
const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      environment: {
        type: 'containerAuto',
        skills: [
          // By reference:
          { type: 'skillReference', skillId: 'skill_abc123' },
          // Or inline:
          {
            type: 'inline',
            name: 'my-skill',
            description: 'What this skill does',
            source: {
              type: 'base64',
              mediaType: 'application/zip',
              data: readFileSync('./my-skill.zip').toString('base64'),
            },
          },
        ],
      },
    }),
  },
  prompt: 'Use the skill to solve this problem.',
});
```

**本地 skills** 指向磁盘上包含 `SKILL.md` 的目录：

```ts
const result = await generateText({
  model: openai('gpt-5.2'),
  tools: {
    shell: openai.tools.shell({
      execute: async ({ action }) => {
        // ... your local execution implementation ...
        return { output: results };
      },
      environment: {
        type: 'local',
        skills: [
          {
            name: 'my-skill',
            description: 'What this skill does',
            path: resolve('path/to/skill-directory'),
          },
        ],
      },
    }),
  },
  prompt: 'Use the skill to solve this problem.',
  stopWhen: isStepCount(5),
});
```

创建 skills 的更多细节见 [OpenAI Skills 文档](https://platform.openai.com/docs/guides/tools-skills)。

#### Apply Patch 工具

OpenAI Responses API 通过 `openai.tools.applyPatch` 工具为 GPT-5.1 模型提供 apply patch 支持。
apply patch 工具让模型使用结构化 diff 在代码库中创建、更新和删除文件。
模型不再仅建议修改，而是发出补丁操作，由你的应用应用并回报结果，
从而支持迭代、多步的代码编辑流程。

```ts
import { openai } from '@ai-sdk/openai';
import { generateText, isStepCount } from 'ai';

const result = await generateText({
  model: openai('gpt-5.1'),
  tools: {
    apply_patch: openai.tools.applyPatch({
      execute: async ({ callId, operation }) => {
        // ... your implementation for applying the diffs.
      },
    }),
  },
  prompt: 'Create a python file that calculates the factorial of a number',
  stopWhen: isStepCount(5),
});
```

execute 函数必须返回：

- **status** _'completed' | 'failed'_ — 补丁是否应用成功
- **output** _string_（可选）— 人类可读的日志文本（例如结果或错误信息）

#### 工具搜索

工具搜索允许模型按需动态查找并将工具加载到上下文中，
而不是一开始就加载全部工具定义。在工具很多时可降低 token 用量、成本与延迟。
在工具的 `providerOptions` 中使用 `deferLoading: true` 标记需要可被搜索的工具。

有两种执行模式：

- **服务端执行（托管）：** OpenAI 在请求声明的延迟加载工具中搜索，并在同一响应中返回已加载子集。无需额外往返。
- **客户端执行：** 模型发出 `tool_search_call`，你的应用执行查找，并通过 `execute` 回调返回匹配的工具。

##### 服务端执行（托管）工具搜索

当候选工具在请求时已已知时使用托管工具搜索。
添加无参数的 `openai.tools.toolSearch()`，并为工具标记 `deferLoading: true`：

```ts
import { openai } from '@ai-sdk/openai';
import { generateText, tool, isStepCount } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: openai.responses('gpt-5.4'),
  prompt: 'What is the weather in San Francisco?',
  stopWhen: isStepCount(10),
  tools: {
    toolSearch: openai.tools.toolSearch(),

    get_weather: tool({
      description: 'Get the current weather at a specific location',
      inputSchema: z.object({
        location: z.string(),
        unit: z.enum(['celsius', 'fahrenheit']),
      }),
      execute: async ({ location, unit }) => ({
        location,
        temperature: unit === 'celsius' ? 18 : 64,
      }),
      providerOptions: {
        openai: { deferLoading: true },
      },
    }),

    search_files: tool({
      description: 'Search through files in the workspace',
      inputSchema: z.object({ query: z.string() }),
      execute: async ({ query }) => ({
        results: [`Found 3 files matching "${query}"`],
      }),
      providerOptions: {
        openai: { deferLoading: true },
      },
    }),
  },
});
```

在托管模式下，模型在内部搜索延迟加载的工具，加载相关工具并在
同一响应内继续调用它们。`tool_search_call` 与
`tool_search_output` 条目会以 `execution: 'server'` 且 `call_id: null` 出现在响应中。

##### 客户端执行工具搜索

当工具发现依赖运行时状态时使用客户端执行——例如
按租户、项目或外部系统变化的工具。传入 `execution: 'client'`，并提供
`description`、`parameters` schema 与 `execute` 回调：

```ts
import { openai } from '@ai-sdk/openai';
import { generateText, tool, isStepCount } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: openai.responses('gpt-5.4'),
  prompt: 'What is the weather in San Francisco?',
  stopWhen: isStepCount(10),
  tools: {
    toolSearch: openai.tools.toolSearch({
      execution: 'client',
      description: 'Search for available tools based on what the user needs.',
      parameters: {
        type: 'object',
        properties: {
          goal: {
            type: 'string',
            description: 'What the user is trying to accomplish',
          },
        },
        required: ['goal'],
        additionalProperties: false,
      },
      execute: async ({ arguments: args }) => {
        // Your custom tool discovery logic here.
        // Return the tools that match the search goal.
        return {
          tools: [
            {
              type: 'function',
              name: 'get_weather',
              description: 'Get the current weather at a specific location',
              deferLoading: true,
              parameters: {
                type: 'object',
                properties: {
                  location: { type: 'string' },
                },
                required: ['location'],
                additionalProperties: false,
              },
            },
          ],
        };
      },
    }),

    get_weather: tool({
      description: 'Get the current weather at a specific location',
      inputSchema: z.object({ location: z.string() }),
      execute: async ({ location }) => ({
        location,
        temperature: 64,
        condition: 'Partly cloudy',
      }),
      providerOptions: {
        openai: { deferLoading: true },
      },
    }),
  },
});
```

客户端模式下流程分为两步：

1. **步骤 1：** 模型发出带 `execution: 'client'` 和非空 `call_id` 的 `tool_search_call`。SDK 用你的搜索参数调用 `execute` 回调，回调返回发现的工具。
2. **步骤 2：** SDK 将 `tool_search_output`（匹配的 `call_id`）发回模型。模型随后可将已加载的工具作为普通函数调用。

更多说明见 [OpenAI 工具搜索文档](https://platform.openai.com/docs/guides/tools-tool-search)。

#### 自定义工具

OpenAI Responses API 通过 `openai.tools.customTool` 支持
[自定义工具](https://developers.openai.com/api/docs/guides/function-calling/#custom-tools)。
自定义工具返回原始字符串而非 JSON，并可选用语法（
regex 或 Lark）约束。适用于生成 SQL 查询、代码片段等须匹配特定模式的结构化文本。

```ts
import { openai } from '@ai-sdk/openai';
import { generateText, isStepCount } from 'ai';

const result = await generateText({
  model: openai.responses('gpt-5.2-codex'),
  tools: {
    write_sql: openai.tools.customTool({
      description: 'Write a SQL SELECT query to answer the user question.',
      format: {
        type: 'grammar',
        syntax: 'regex',
        definition: 'SELECT .+',
      },
      execute: async input => {
        // input is a raw string matching the grammar, e.g. "SELECT * FROM users WHERE age > 25"
        const rows = await db.query(input);
        return JSON.stringify(rows);
      },
    }),
  },
  toolChoice: 'required',
  prompt: 'Write a SQL query to get all users older than 25.',
  stopWhen: isStepCount(3),
});
```

自定义工具也适用于 `streamText`：

```ts
import { openai } from '@ai-sdk/openai';
import { streamText } from 'ai';

const result = streamText({
  model: openai.responses('gpt-5.2-codex'),
  tools: {
    write_sql: openai.tools.customTool({
      description: 'Write a SQL SELECT query to answer the user question.',
      format: {
        type: 'grammar',
        syntax: 'regex',
        definition: 'SELECT .+',
      },
    }),
  },
  toolChoice: 'required',
  prompt: 'Write a SQL query to get all users older than 25.',
});

for await (const chunk of result.fullStream) {
  if (chunk.type === 'tool-call') {
    console.log(`Tool: ${chunk.toolName}`);
    console.log(`Input: ${chunk.input}`);
  }
}
```

自定义工具可配置项：

- **description** _string_（可选）— 工具用途描述，帮助模型理解何时使用。
- **format** _object_（可选）— 输出格式约束。省略则无约束文本输出。
  - **type** _'grammar' | 'text'_ — 格式类型。`'grammar'` 为约束输出，`'text'` 为显式无约束文本。
  - **syntax** _'regex' | 'lark'_ —（仅 grammar）语法类型。`'regex'` 为正则，`'lark'` 为 [Lark 解析器语法](https://lark-parser.readthedocs.io/)。
  - **definition** _string_ —（仅 grammar）语法定义字符串（正则模式或 Lark 文法）。
- **execute** _function_（可选）— 异步函数，接收原始字符串输入并返回字符串结果，用于多轮工具调用。

#### 文本片段中的类型化 providerMetadata

使用 OpenAI Responses API 时，SDK 会通过 `providerMetadata` 将 OpenAI 专属元数据附加到输出片段。

该元数据可用于客户端渲染引用、下载代码解释器生成的文件等场景。
为类型安全处理这些元数据，AI SDK 导出了专用 TypeScript 类型。

对于文本片段，当 `part.type === 'text'` 时，`providerMetadata` 的形态为 `OpenaiResponsesTextProviderMetadata`。

该元数据包含以下字段：

- `itemId`
  Responses API 中输出条目的 ID。
- `annotations`（可选）
  模型生成的注释对象数组。
  若无注释，该属性本身可省略（`undefined`）。

  `annotations` 中每个元素均为带必填 `type` 字段的可辨识联合。支持的类型例如：
  - `url_citation`
  - `file_citation`
  - `container_file_citation`
  - `file_path`

  这些注释与 Responses API 定义的注释对象一一对应，可用于行内引用渲染或输出分析。
  详见 OpenAI 官方文档：
  [Responses API – 输出文本注释](https://platform.openai.com/docs/api-reference/responses/object?lang=javascript#responses-object-output-output_message-content-output_text-annotations)。

```ts
import {
  openai,
  type OpenaiResponsesTextProviderMetadata,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-4.1-mini'),
  prompt:
    'Create a program that generates five random numbers between 1 and 100 with two decimal places, and show me the execution results. Also save the result to a file.',
  tools: {
    code_interpreter: openai.tools.codeInterpreter(),
    web_search: openai.tools.webSearch(),
    file_search: openai.tools.fileSearch({ vectorStoreIds: ['vs_1234'] }), // requires a configured vector store
  },
});

for (const part of result.content) {
  if (part.type === 'text') {
    const providerMetadata = part.providerMetadata as
      | OpenaiResponsesTextProviderMetadata
      | undefined;
    if (!providerMetadata) continue;
    const { itemId: _itemId, annotations } = providerMetadata.openai;

    if (!annotations) continue;
    for (const annotation of annotations) {
      switch (annotation.type) {
        case 'url_citation':
          // url_citation is returned from web_search and provides:
          // properties: type, url, title, start_index and end_index
          break;
        case 'file_citation':
          // file_citation is returned from file_search and provides:
          // properties: type, file_id, filename and index
          break;
        case 'container_file_citation':
          // container_file_citation is returned from code_interpreter and provides:
          // properties: type, container_id, file_id, filename, start_index and end_index
          break;
        case 'file_path':
          // file_path provides:
          // properties: type, file_id and index
          break;
        default: {
          const _exhaustiveCheck: never = annotation;
          throw new Error(
            `Unhandled annotation: ${JSON.stringify(_exhaustiveCheck)}`,
          );
        }
      }
    }
  }
}
```

<Note>
  实现代码解释器生成文件的下载时，可使用 `providerMetadata` 中的
  `container_id` 与 `file_id`
  获取文件内容。详见 [Retrieve container file
  content](https://platform.openai.com/docs/api-reference/container-files/retrieveContainerFileContent)
  API。
</Note>

#### 推理片段中的类型化 providerMetadata

使用 OpenAI Responses API 时，推理输出片段可包含提供商元数据。
要类型安全地处理这些元数据，请使用 `OpenaiResponsesReasoningProviderMetadata`。

对于推理片段，当 `part.type === 'reasoning'` 时，`providerMetadata` 的形态为 `OpenaiResponsesReasoningProviderMetadata`。

该元数据包含以下字段：

- `itemId`  
  Responses API 中推理条目的 ID。
- `reasoningEncryptedContent`（可选）  
  加密的推理内容（仅在通过 `include: ['reasoning.encrypted_content']` 请求时返回）。

```ts
import {
  openai,
  type OpenaiResponsesReasoningProviderMetadata,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'How many "r"s are in the word "strawberry"?',
  providerOptions: {
    openai: {
      store: false,
      include: ['reasoning.encrypted_content'],
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

for (const part of result.content) {
  if (part.type === 'reasoning') {
    const providerMetadata = part.providerMetadata as
      | OpenaiResponsesReasoningProviderMetadata
      | undefined;

    const { itemId, reasoningEncryptedContent } =
      providerMetadata?.openai ?? {};
    console.log(itemId, reasoningEncryptedContent);
  }
}
```

#### 来源文档片段中的类型化 providerMetadata

对于来源文档片段，当 `part.type === 'source'` 且 `sourceType === 'document'` 时，`providerMetadata` 的形态为 `OpenaiResponsesSourceDocumentProviderMetadata`。

该元数据同样是带必填 `type` 的可辨识联合。支持的类型包括：

- `file_citation`
- `container_file_citation`
- `file_path`

每种类型包含操作引用资源所需的标识符，例如 `fileId` 与 `containerId`。

```ts
import {
  openai,
  type OpenaiResponsesSourceDocumentProviderMetadata,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai('gpt-4.1-mini'),
  prompt:
    'Create a program that generates five random numbers between 1 and 100 with two decimal places, and show me the execution results. Also save the result to a file.',
  tools: {
    code_interpreter: openai.tools.codeInterpreter(),
    web_search: openai.tools.webSearch(),
    file_search: openai.tools.fileSearch({ vectorStoreIds: ['vs_1234'] }), // requires a configured vector store
  },
});

for (const part of result.content) {
  if (part.type === 'source') {
    if (part.sourceType === 'document') {
      const providerMetadata = part.providerMetadata as
        | OpenaiResponsesSourceDocumentProviderMetadata
        | undefined;
      if (!providerMetadata) continue;
      const annotation = providerMetadata.openai;
      switch (annotation.type) {
        case 'file_citation':
          // file_citation is returned from file_search and provides:
          // properties: type, fileId and index
          // The filename can be accessed via part.filename.
          break;
        case 'container_file_citation':
          // container_file_citation is returned from code_interpreter and provides:
          // properties: type, containerId and fileId
          // The filename can be accessed via part.filename.
          break;
        case 'file_path':
          // file_path provides:
          // properties: type, fileId and index
          break;
        default: {
          const _exhaustiveCheck: never = annotation;
          throw new Error(
            `Unhandled annotation: ${JSON.stringify(_exhaustiveCheck)}`,
          );
        }
      }
    }
  }
}
```

<Note>
  文本片段中的注释遵循 OpenAI Responses API 规范，因此使用 snake_case 属性（例如 `file_id`、`container_id`）。
  相比之下，来源文档片段的 `providerMetadata` 会被 SDK
  规范为 camelCase（例如 `fileId`、`containerId`）。依赖原始文本内容的字段（如 `start_index`、`end_index`）会被省略；来源对象上可直接获取的字段（如 `filename`）同样会被省略。
</Note>

#### 压缩（Compaction）

OpenAI Responses API 支持服务端上下文压缩（compaction）。启用后，当 token 使用量超过配置的阈值时，服务器会自动压缩对话上下文。适用于长时间对话或希望在 token 限制内运行且无需手动管理上下文的智能体循环。

服务器返回的 compaction 条目不透明且已加密——以更少的 token 将关键先前状态与推理带入下一轮。AI SDK 会自动处理：compaction 条目以带特殊 `providerMetadata` 的文本片段返回，在后续请求中传回时会作为 compaction 输入条目发送。

```ts highlight="7-11"
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai.responses('gpt-5.2'),
  messages: conversationHistory,
  providerOptions: {
    openai: {
      store: false,
      contextManagement: [{ type: 'compaction', compactThreshold: 50000 }],
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});
```

**配置：**

- **type** — 必须为 `'compaction'`
- **compactThreshold** — 触发压缩时的 token 数量。当渲染后的输入 token 数超过该阈值时，服务器会在继续推理前执行压缩。

<Note>
  在请求上设置 `store: false` 时，服务端压缩对零数据驻留（ZDR）更友好。
</Note>

##### 在流中检测压缩

使用 `streamText` 时，可通过检查 `text-start` 与 `text-end` 事件上的 `providerMetadata` 检测压缩：

```ts
import {
  openai,
  type OpenAILanguageModelResponsesOptions,
} from '@ai-sdk/openai';
import { streamText } from 'ai';

const result = streamText({
  model: openai.responses('gpt-5.2'),
  messages: conversationHistory,
  providerOptions: {
    openai: {
      store: false,
      contextManagement: [{ type: 'compaction', compactThreshold: 50000 }],
    } satisfies OpenAILanguageModelResponsesOptions,
  },
});

for await (const part of result.fullStream) {
  switch (part.type) {
    case 'text-start': {
      const isCompaction = part.providerMetadata?.openai?.type === 'compaction';
      if (isCompaction) {
        // ... your logic
      }
      break;
    }
    case 'text-end': {
      const isCompaction = part.providerMetadata?.openai?.type === 'compaction';
      if (isCompaction) {
        // ... your logic
      }
      break;
    }
    case 'text-delta': {
      process.stdout.write(part.text);
      break;
    }
  }
}
```

##### UI 应用中的压缩

使用 `useChat` 或其他 UI hook 时，压缩条目会以带 `providerMetadata` 的文本片段出现。可在 UI 中检测并单独样式化：

```tsx
{
  message.parts.map((part, index) => {
    if (part.type === 'text') {
      const isCompaction =
        (part.providerMetadata?.openai as { type?: string } | undefined)
          ?.type === 'compaction';

      if (isCompaction) {
        return (
          <div
            key={index}
            className="bg-yellow-100 border-l-4 border-yellow-500 p-2"
          >
            <span className="font-bold">[Context Compacted]</span>
            <p className="text-sm text-yellow-700">
              The server compressed the conversation context to reduce token
              usage.
            </p>
          </div>
        );
      }
      return <div key={index}>{part.text}</div>;
    }
  });
}
```

元数据包含以下字段：

- **type** — 始终为 `'compaction'`
- **itemId** _string_ — Responses API 中压缩条目的 ID
- **encryptedContent** _string_（可选）— 加密的压缩状态。当消息包含在后续请求中时会自动发回 API。

### Chat 模型

可使用 `.chat()` 工厂方法创建调用 [OpenAI Chat API](https://platform.openai.com/docs/api-reference/chat) 的模型。
第一个参数为模型 ID，例如 `gpt-4`。
OpenAI Chat 模型支持工具调用，部分模型具备多模态能力。

```ts
const model = openai.chat('gpt-5');
```

OpenAI Chat 模型还支持部分不属于 [标准调用设置](/docs/ai-sdk-core/settings) 的模型专属提供商选项。
可通过 `providerOptions` 参数传入：

```ts
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';

const model = openai.chat('gpt-5');

await generateText({
  model,
  providerOptions: {
    openai: {
      logitBias: {
        // optional likelihood for specific tokens
        '50256': -100,
      },
      user: 'test-user', // optional unique user identifier
    } satisfies OpenAILanguageModelChatOptions,
  },
});
```

OpenAI Chat 模型可使用以下可选提供商选项：

- **logitBias** _Record&lt;number, number&gt;_

  调整指定 token 出现在补全中的相对概率。

  接受 JSON 对象，将 token（由其 GPT 分词器中的 token ID 指定）映射到 -100 至 100 的偏置值。可使用分词器工具将文本转为 token ID。数学上，偏置会在采样前加到模型输出的 logits 上。
  具体效果因模型而异，但 -1 至 1 之间的值通常会降低或提高被选概率；-100 或 100 等值会导致禁止或强制选择相关 token。

  例如可传入 `{"50256": -100}` 以防止生成该 token。

- **logprobs** _boolean | number_

  返回 token 的对数概率。启用 logprobs 会增加响应体积并可能拖慢响应，但有助于理解模型行为。

  设为 true 返回已生成 token 的对数概率。

  设为数字返回已生成 token 中 top n 的对数概率。

- **parallelToolCalls** _boolean_

  是否在工具使用时启用并行函数调用。默认为 `true`。

- **user** _string_

  表示终端用户的唯一标识符，可帮助 OpenAI 监控并检测滥用。[了解更多](https://platform.openai.com/docs/guides/safety-best-practices/end-user-ids)。

- **reasoningEffort** _'minimal' | 'low' | 'medium' | 'high' | 'xhigh'_

  推理模型的推理强度。默认为 `medium`。若通过
  `providerOptions` 设置 `reasoningEffort`，则忽略此
  模型级设置。

- **maxCompletionTokens** _number_

  生成的补全 token 上限。对推理模型很有用。

- **store** _boolean_

  是否在 Responses API 中启用持久化。

- **metadata** _Record&lt;string, string&gt;_

  与请求关联的元数据。

- **prediction** _Record&lt;string, any&gt;_

  预测模式参数。

- **serviceTier** _'auto' | 'flex' | 'priority' | 'default'_

  请求的服务层级。`'flex'` 可降低约 50% 费用、延迟更高（适用于 o3、o4-mini、gpt-5 等）。
  `'priority'` 在企业访问下更快（适用于 gpt-4、gpt-5、gpt-5-mini、o3、o4-mini；不支持 gpt-5-nano）。

  默认为 `'auto'`。

- **strictJsonSchema** _boolean_

  是否使用严格 JSON schema 校验。
  默认为 `true`。

- **textVerbosity** _'low' | 'medium' | 'high'_

  控制模型回复详略。较低值更简洁，较高值更冗长。

- **promptCacheKey** _string_

  用于手动控制提示缓存的缓存键。OpenAI 用它缓存相似请求以提升命中率。

- **promptCacheRetention** _'in_memory' | '24h'_

  提示缓存保留策略。`'24h'` 启用延长缓存，前缀最长保留 24 小时。标准缓存默认为 `'in_memory'`。注意：`'24h'` 目前仅适用于 5.1 系列模型。

- **safetyIdentifier** _string_

  用于协助识别可能违反 OpenAI 使用政策的用户的稳定标识符。应为可唯一标识每个用户的字符串。

- **systemMessageMode** _'system' | 'developer' | 'remove'_

  覆盖此模型的系统消息模式。未指定时按模型自动判定。`system` 对系统消息使用 system 角色（多数模型默认）；`developer` 使用 developer 角色（推理模型）；`remove` 完全移除系统消息。

- **forceReasoning** _boolean_

  强制将本模型视为推理模型。适用于通过自定义 baseURL 等方式使用的「隐匿」推理模型，其 ID 未被 SDK 白名单识别。启用后 SDK 应用推理模型参数兼容规则，并在未覆盖时将 `systemMessageMode` 默认为 `developer`。

#### 推理

OpenAI 推出了 `o1`、`o3` 与 `o4` 系列的 [推理模型](https://platform.openai.com/docs/guides/reasoning)。
目前 `o4-mini`、`o3`、`o3-mini` 与 `o1` 可通过 Chat 与 Responses API 使用。模型
`gpt-5.1-codex-mini` 仅可通过 [Responses API](#responses-models) 使用。

推理模型目前仅生成文本，存在若干限制，且仅支持 `generateText` 与 `streamText`。

还支持额外设置与响应元数据：

- 可使用 `providerOptions` 设置
  - `reasoningEffort` 选项（或使用 `reasoningEffort` 模型设置），控制推理量。

- 可使用响应的 `providerMetadata` 读取模型生成的推理 token 数量。

```ts highlight="4,7-11,17"
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text, usage, providerMetadata } = await generateText({
  model: openai.chat('gpt-5'),
  prompt: 'Invent a new holiday and describe its traditions.',
  providerOptions: {
    openai: {
      reasoningEffort: 'low',
    } satisfies OpenAILanguageModelChatOptions,
  },
});

console.log(text);
console.log('Usage:', {
  ...usage,
  reasoningTokens: providerMetadata?.openai?.reasoningTokens,
});
```

<Note>
  在支持的情况下，系统消息会自动转换为 OpenAI 的开发者消息（developer），适用于
  推理模型。
</Note>

- 可通过 `providerOptions` 的 `systemMessageMode` 控制系统消息处理方式：
  - `developer`：将提示视为开发者消息（推理模型默认）。
  - `system`：保留系统消息作为系统级指令。
  - `remove`：从消息中移除系统消息。

```ts highlight="12"
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const result = await generateText({
  model: openai.chat('gpt-5'),
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'Tell me a joke.' },
  ],
  providerOptions: {
    openai: {
      systemMessageMode: 'system',
    } satisfies OpenAILanguageModelChatOptions,
  },
});
```

<Note>
  推理模型需要额外运行时推理来完成
  推理阶段再生成回复，因此相比其他模型延迟更长。
</Note>

<Note>
  对推理模型，`maxOutputTokens` 会自动映射到 `max_completion_tokens`。
</Note>

#### 严格结构化输出

默认启用严格结构化输出。
可将 `strictJsonSchema` 设为 `false` 以关闭。

```ts highlight="7"
import { openai, OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText, Output } from 'ai';
import { z } from 'zod';

const result = await generateText({
  model: openai.chat('gpt-4o-2024-08-06'),
  providerOptions: {
    openai: {
      strictJsonSchema: false,
    } satisfies OpenAILanguageModelChatOptions,
  },
  output: Output.object({
    schema: z.object({
      name: z.string(),
      ingredients: z.array(
        z.object({
          name: z.string(),
          amount: z.string(),
        }),
      ),
      steps: z.array(z.string()),
    }),
    schemaName: 'recipe',
    schemaDescription: 'A recipe for lasagna.',
  }),
  prompt: 'Generate a lasagna recipe.',
});

console.log(JSON.stringify(result.output, null, 2));
```

<Note type="warning">
  OpenAI 结构化输出存在多项
  [限制](https://openai.com/index/introducing-structured-outputs-in-the-api)，
  尤其在 [支持的 schema](https://platform.openai.com/docs/guides/structured-outputs/supported-schemas) 方面，
  因此属于可选开启能力。

例如，不支持可选 schema 属性。
需要将 Zod `.nullish()` 和 `.optional()` 改为 `.nullable()`。

</Note>

#### 预测输出（Predicted Outputs）

OpenAI 为 `gpt-4o` 与 `gpt-4o-mini` 支持 [预测输出](https://platform.openai.com/docs/guides/latency-optimization#use-predicted-outputs)。
预测输出允许你指定模型应在其基础上修改的基准文本，从而降低延迟。
可在 `providerOptions.openai` 中添加 `prediction` 选项以启用：

```ts highlight="15-18"
const result = streamText({
  model: openai.chat('gpt-5'),
  messages: [
    {
      role: 'user',
      content: 'Replace the Username property with an Email property.',
    },
    {
      role: 'user',
      content: existingCode,
    },
  ],
  providerOptions: {
    openai: {
      prediction: {
        type: 'content',
        content: existingCode,
      },
    } satisfies OpenAILanguageModelChatOptions,
  },
});
```

OpenAI 提供预测输出的用量信息（`acceptedPredictionTokens` 与 `rejectedPredictionTokens`）。
可在 `providerMetadata` 对象中访问。

```ts highlight="11"
const openaiMetadata = (await result.providerMetadata)?.openai;

const acceptedPredictionTokens = openaiMetadata?.acceptedPredictionTokens;
const rejectedPredictionTokens = openaiMetadata?.rejectedPredictionTokens;
```

<Note type="warning">
  OpenAI 预测输出存在多项
  [限制](https://platform.openai.com/docs/guides/predicted-outputs#limitations)，
  例如不支持部分 API 参数且无工具调用支持。
</Note>

```ts highlight="13-16"
const result = await generateText({

#### 蒸馏（Distillation）

OpenAI 对部分模型支持模型蒸馏。
若要将某次生成保存以供蒸馏流程使用，可在 `providerOptions.openai` 中添加 `store` 选项。
生成结果会保存到 OpenAI 平台供后续蒸馏使用。

```typescript highlight="9-16"
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

  const { text, usage } = await generateText({
    model: openai.chat('gpt-4o-mini'),

      openai: {
        store: true,
        metadata: {
          custom: 'value',
        },
      } satisfies OpenAILanguageModelChatOptions,
    },
  });

  console.log(text);
  console.log();
  console.log('Usage:', usage);
}

main().catch(console.error);
```

#### 提示缓存（Prompt Caching）

OpenAI 为支持的模型推出了 [提示缓存](https://platform.openai.com/docs/guides/prompt-caching)，
包括 `gpt-4o` 与 `gpt-4o-mini`。

- 当提示达到 1024 token 或更长时，这些模型会自动启用提示缓存，无需单独开启。
- 可使用响应的 `providerMetadata` 读取命中缓存的提示 token 数量。
- 缓存行为取决于 OpenAI 基础设施负载。提示前缀通常在闲置 5–10 分钟后被淘汰，低谷期可能保留长达约一小时。

```ts highlight="11"
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text, usage, providerMetadata } = await generateText({
  model: openai.chat('gpt-4o-mini'),
  prompt: `A 1024-token or longer prompt...`,
});

console.log(`usage:`, {
  ...usage,
  cachedPromptTokens: providerMetadata?.openai?.cachedPromptTokens,
});
```

为提高缓存命中率，可使用 `promptCacheKey` 选项手动控制缓存：

```ts highlight="7-11"
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text, usage, providerMetadata } = await generateText({
  model: openai.chat('gpt-5'),
  prompt: `A 1024-token or longer prompt...`,
  providerOptions: {
    openai: {
      promptCacheKey: 'my-custom-cache-key-123',
    } satisfies OpenAILanguageModelChatOptions,
  },
});

console.log(`usage:`, {
  ...usage,
  cachedPromptTokens: providerMetadata?.openai?.cachedPromptTokens,
});
```

对于 GPT-5.1 模型，可启用最长保留 24 小时的延长提示缓存：
import { openai, type OpenAILanguageModelChatOptions } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text, usage, providerMetadata } = await generateText({
  model: openai.chat('gpt-5.1'),
  prompt: `A 1024-token or longer prompt...`,
  providerOptions: {
    openai: {
      promptCacheKey: 'my-custom-cache-key-123',
      promptCacheRetention: '24h', // Extended caching for GPT-5.1
    } satisfies OpenAILanguageModelChatOptions,
  },
});

console.log(`usage:`, {
  ...usage,
  cachedPromptTokens: providerMetadata?.openai?.cachedPromptTokens,
});
```

### Completion 模型

可使用 `.completion()` 工厂方法创建调用 [OpenAI Completions API](https://platform.openai.com/docs/api-reference/completions) 的模型。
第一个参数为模型 ID。
目前仅支持 `gpt-3.5-turbo-instruct`。

```ts
const model = openai.completion('gpt-3.5-turbo-instruct');
```

OpenAI Completion 模型还支持部分不属于 [标准调用设置](/docs/ai-sdk-core/settings) 的模型专属设置。
可作为选项参数传入：

```ts
const model = openai.completion('gpt-3.5-turbo-instruct');

await model.doGenerate({
  providerOptions: {
    openai: {
      echo: true, // optional, echo the prompt in addition to the completion
      logitBias: {
        // optional likelihood for specific tokens
        '50256': -100,
      },
      suffix: 'some text', // optional suffix that comes after a completion of inserted text
      user: 'test-user', // optional unique user identifier
    } satisfies OpenAILanguageModelCompletionOptions,
  },
});
```

OpenAI Completion 模型可使用以下可选提供商选项：

- **echo**：_boolean_

  除补全外是否回显提示。

- **logitBias** _Record&lt;number, number&gt;_

  调整指定 token 出现在补全中的相对概率。

  接受 JSON 对象，将 token（由其 GPT 分词器中的 token ID 指定）映射到 -100 至 100 的偏置值。可使用分词器工具将文本转为 token ID。数学上，偏置会在采样前加到模型输出的 logits 上。
  具体效果因模型而异，但 -1 至 1 之间的值通常会降低或提高被选概率；-100 或 100 等值会导致禁止或强制选择相关 token。

  例如可传入 `{"50256": -100}` 以防止生成 &lt;|endoftext|&gt; token。

- **logprobs** _boolean | number_

  返回 token 的对数概率。启用 logprobs 会增加响应体积并可能拖慢响应，但有助于理解模型行为。

  设为 true 返回已生成 token 的对数概率。

  设为数字返回已生成 token 中 top n 的对数概率。

- **suffix** _string_

  插入文本补全之后的后缀。

- **user** _string_

  表示终端用户的唯一标识符，可帮助 OpenAI 监控并检测滥用。[了解更多](https://platform.openai.com/docs/guides/safety-best-practices/end-user-ids)。

### 图像编辑

OpenAI 的 `gpt-image-1` 模型支持强大的图像编辑能力。通过 `prompt.images` 传入输入图像，即可变换、合成或编辑现有图像。

#### 基础图像编辑

使用文本提示变换现有图像：

```ts
const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: openai.image('gpt-image-1'),
  prompt: {
    text: 'Turn the cat into a dog but retain the style of the original image',
    images: [imageBuffer],
  },
});
```

#### 蒙版局部重绘（Inpainting）

使用蒙版编辑图像的指定区域。蒙版中的透明区域表示需要编辑的位置：

```ts
const image = readFileSync('./input-image.png');
const mask = readFileSync('./mask.png'); // Transparent areas = edit regions

const { images } = await generateImage({
  model: openai.image('gpt-image-1'),
  prompt: {
    text: 'A sunlit indoor lounge area with a pool containing a flamingo',
    images: [image],
    mask: mask,
  },
});
```

#### 背景移除

将 `background` 设为 `transparent` 可移除图像背景：

```ts
import { openai, type OpenAIImageModelEditOptions } from '@ai-sdk/openai';
import { generateImage } from 'ai';

const imageBuffer = readFileSync('./input-image.png');

const { images } = await generateImage({
  model: openai.image('gpt-image-1'),
  prompt: {
    text: 'do not change anything',
    images: [imageBuffer],
  },
  providerOptions: {
    openai: {
      background: 'transparent',
      outputFormat: 'png',
    } satisfies OpenAIImageModelEditOptions,
  },
});
```

#### 多图合成

将多张参考图合成为单张输出。`gpt-image-1` 最多支持 16 张输入图像：

```ts
const cat = readFileSync('./cat.png');
const dog = readFileSync('./dog.png');
const owl = readFileSync('./owl.png');
const bear = readFileSync('./bear.png');

const { images } = await generateImage({
  model: openai.image('gpt-image-1'),
  prompt: {
    text: 'Combine these animals into a group photo, retaining the original style',
    images: [cat, dog, owl, bear],
  },
});
```

<Note>
  输入图像可为 `Buffer`、`ArrayBuffer`、`Uint8Array` 或
  base64 字符串。对 `gpt-image-1`，每张图像应为小于 50MB 的
  `png`、`webp` 或 `jpg` 文件。
</Note>

## 语音转写模型

可使用 `.transcription()` 工厂方法创建调用 [OpenAI 语音转写 API](https://platform.openai.com/docs/api-reference/audio/transcribe) 的模型。

第一个参数为模型 ID，例如 `whisper-1`。

```ts
const model = openai.transcription('whisper-1');
```

还可通过 `providerOptions` 传入提供商专属选项。例如以 ISO-639-1（如 `en`）提供输入语言可提升准确度与延迟表现。

```ts highlight="6"
import { experimental_transcribe as transcribe } from 'ai';
import { openai, type OpenAITranscriptionModelOptions } from '@ai-sdk/openai';

const result = await transcribe({
  model: openai.transcription('whisper-1'),
  audio: new Uint8Array([1, 2, 3, 4]),
  providerOptions: {
    openai: { language: 'en' } satisfies OpenAITranscriptionModelOptions,
  },
});
```

若要词级时间戳，请指定粒度：

```ts highlight="8-9"
import { experimental_transcribe as transcribe } from 'ai';
import { openai, type OpenAITranscriptionModelOptions } from '@ai-sdk/openai';

const result = await transcribe({
  model: openai.transcription('whisper-1'),
  audio: new Uint8Array([1, 2, 3, 4]),
  providerOptions: {
    openai: {
      //timestampGranularities: ['word'],
      timestampGranularities: ['segment'],
    } satisfies OpenAITranscriptionModelOptions,
  },
});

// Access word-level timestamps
console.log(result.segments); // Array of segments with startSecond/endSecond
```

提供以下提供商选项：

- **timestampGranularities** _string[]_
  转写中时间戳的粒度。
  默认为 `['segment']`。
  可取 `['word']`、`['segment']`、`['word', 'segment']`。
  注意：分段时间戳无额外延迟；生成词级时间戳会增加延迟。

- **language** _string_
  输入音频的语言。以 ISO-639-1（如 `'en'`）提供可提升准确度与延迟。
  可选。

- **prompt** _string_
  可选文本，用于引导模型风格或接续上一段音频。提示应与音频语言一致。
  可选。

- **temperature** _number_
  采样温度，介于 0 与 1。较高值（如 0.8）输出更随机，较低值（如 0.2）更聚焦、确定性更强。设为 0 时，模型会使用对数概率自动调高温度直至达到阈值。
  默认为 0。
  可选。

- **include** _string[]_
  转写响应中包含的额外信息。

第一个参数为模型 ID，例如 `tts-1`。

`voice` 参数可设为 OpenAI 提供的音色之一：`alloy`、`ash`、`coral`、`echo`、`fable`、`onyx`、`nova`、`sage` 或 `shimmer`。

import { experimental_generateSpeech as generateSpeech } from 'ai';

import { experimental_generateSpeech as generateSpeech } from 'ai';

const result = await generateSpeech({

  可选。
