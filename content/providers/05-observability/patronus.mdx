# Patronus 可观测性

[Patronus AI](https://patronus.ai) 提供端到端方案，用于评估、监控并改进 LLM 系统表现，帮助开发者安全可靠地交付 AI 产品。更多说明见 [此处](https://docs.patronus.ai/docs)。

使用 AI SDK 构建时，可将 OpenTelemetry（OTEL）traces 直接流入 Patronus，并为每次生成配对丰富的自动评估。

## 设置

### 1. OpenTelemetry

Patronus 暴露全托管 OTEL 端点。配置 **OTLP exporter** 指向该端点，传入 API key 即可——Patronus 会自动将 LLM span 转换为可浏览与评估的 prompt/response 记录。

#### 环境变量（推荐）

```bash filename=".env.local"
OTEL_EXPORTER_OTLP_ENDPOINT=https://otel.patronus.ai/v1/traces
OTEL_EXPORTER_OTLP_HEADERS="x-api-key:<PATRONUS_API_KEY>"
```

#### 使用 `@vercel/otel`

```ts filename="instrumentation.ts"
import { registerOTel } from '@vercel/otel';
import { OTLPTraceExporter } from '@opentelemetry/exporter-trace-otlp-http';
import { BatchSpanProcessor } from '@opentelemetry/sdk-trace-node';
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';

export function register() {
  registerOTel({
    serviceName: 'next-app',
    additionalSpanProcessors: [
      new BatchSpanProcessor(
        new OTLPTraceExporter({
          url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT,
          headers: {
            'x-api-key': process.env.PATRONUS_API_KEY!,
          },
        }),
      ),
    ],
  });
}

registerTelemetry(new LegacyOpenTelemetry());
```

<Note>
  若需 gRPC 而非 HTTP，请将 exporter 换为
  `@opentelemetry/exporter-trace-otlp-grpc`，并使用
  `https://otel.patronus.ai:4317`。
</Note>

### 2. 注册集成并启用 telemetry

安装 `@ai-sdk/otel` 并在 `instrumentation.ts` 中加入 `registerTelemetry` 后，遥测会自动采集。传入 `telemetry` 可附加元数据：

```ts
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: openai('gpt-4o'),
  prompt: 'Write a haiku about spring.',
  telemetry: {
    functionId: 'spring-haiku', // span name
    metadata: {
      userId: 'user-123', // custom attrs surface in Patronus UI
    },
  },
});
```

`metadata` 中的每个属性都会成为 OTEL 属性，并由 Patronus 索引以供筛选。

## 示例——tracing 与自动评估

```ts filename="app/api/chat/route.ts"
import { trace } from '@opentelemetry/api';
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  const body = await req.json();
  const tracer = trace.getTracer('next-app');

  return await tracer.startActiveSpan('chat-evaluate', async span => {
    try {
      /* 1️⃣ generate answer */
      const answer = await generateText({
        model: openai('gpt-4o'),
        prompt: body.prompt,
        telemetry: { functionId: 'chat' },
      });

      /* 2️⃣ run Patronus evaluation inside the same trace */
      await fetch('https://api.patronus.ai/v1/evaluate', {
        method: 'POST',
        headers: {
          'X-API-Key': process.env.PATRONUS_API_KEY!,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          evaluators: [
            { evaluator: 'lynx', criteria: 'patronus:hallucination' },
          ],
          evaluated_model_input: body.prompt,
          evaluated_model_output: answer.text,
          trace_id: span.spanContext().traceId,
          span_id: span.spanContext().spanId,
        }),
      });

      return new Response(answer.text);
    } finally {
      span.end();
    }
  });
}
```

结果：单一 trace 包含根 HTTP 请求、LLM generation span，以及你的评估 span——**全部可在 Patronus 中查看**，并附带 hallucination 分数。

## 完成 tracing 之后

- 若在追踪智能体，Patronus 的 AI 助手 Percival 可协助错误分析与提示优化。详见 [此处](https://docs.patronus.ai/docs/percival/percival)。
- 在 Patronus 上查看日志与 traces，并配置 webhook 告警，搭建生产监控。详见 [此处](https://docs.patronus.ai/docs/real_time_monitoring/webhooks)。
