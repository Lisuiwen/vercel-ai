# MLflow 可观测性

![MLflow Tracing 与 Vercel AI SDK](https://mlflow.org/docs/latest/images/llms/tracing/vercel-ai-tracing.mp4)

[MLflow Tracing](https://mlflow.org/docs/latest/genai/tracing) 通过 OpenTelemetry 为基于 [Vercel AI SDK](https://ai-sdk.dev/)（`ai` 包）构建的应用提供自动 tracing，为 TypeScript 与 JavaScript 应用解锁可观测能力。

启用后，MLflow 会记录：

- 提示词/消息与生成回复
- 延迟与调用层级
- Token 用量（在 provider 返回时）
- 异常

## 快速上手（NextJS）

若使用 NextJS，为 Vercel AI SDK 启用 MLflow tracing 相对直接。

<Note>
  还没有示例应用？可试用 Vercel 的演示聊天机器人：
  https://vercel.com/templates/next.js/ai-chatbot-telemetry
</Note>

### 1. 启动 MLflow Tracking Server

```bash
mlflow server --backend-store-uri sqlite:///mlruns.db --port 5000
```

也可用 Docker Compose 启动服务；见 [MLflow Setup Guide](https://mlflow.org/docs/latest/genai/getting-started/connect-environment/)。

### 2. 配置环境变量

将其添加到 `.env.local`：

```bash filename=".env.local"
OTEL_EXPORTER_OTLP_ENDPOINT=<your-mlflow-tracking-server-endpoint>
OTEL_EXPORTER_OTLP_TRACES_HEADERS=x-mlflow-experiment-id=<your-experiment-id>
OTEL_EXPORTER_OTLP_TRACES_PROTOCOL=http/protobuf
```

本地测试：`OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:5000`。

### 3. 启用 OpenTelemetry

安装 Vercel OpenTelemetry 集成：

```bash
pnpm i @opentelemetry/api @vercel/otel @ai-sdk/otel
```

在项目根目录创建 `instrumentation.ts`，注册 `LegacyOpenTelemetry` 并配合 OTel provider 设置：

```ts filename="instrumentation.ts"
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { registerOTel } from '@vercel/otel';

registerTelemetry(new LegacyOpenTelemetry());

export async function register() {
  registerOTel({ serviceName: 'next-app' });
}
```

注册集成后，AI SDK 调用会自动发出遥测：

```ts filename="route.ts"
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

export async function POST(req: Request) {
  const { prompt } = await req.json();

  const { text } = await generateText({
    model: openai('gpt-5'),
    prompt,
  });

  return new Response(JSON.stringify({ text }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

上下文传播等高级选项见 [Vercel OpenTelemetry docs](https://vercel.com/docs/tracing/instrumentation)。

### 4. 运行应用并查看 traces

启动 NextJS 应用，在 tracking server 地址打开 MLflow UI（例如 `http://localhost:5000`）。AI SDK 调用的 traces 会出现在配置的实验中。

## 其它 Node.js 应用

对其它 Node.js 框架，需手动接入 OpenTelemetry Node SDK 与 OTLP exporter。

```ts filename="main.ts"
import { init } from 'mlflow-tracing';
import { registerTelemetry, generateText } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { openai } from '@ai-sdk/openai';
import { NodeSDK } from '@opentelemetry/sdk-node';
import { SimpleSpanProcessor } from '@opentelemetry/sdk-trace-node';
import { OTLPTraceExporter } from '@opentelemetry/exporter-trace-otlp-proto';

const sdk = new NodeSDK({
  spanProcessors: [
    new SimpleSpanProcessor(
      new OTLPTraceExporter({
        url: '<your-mlflow-tracking-server-endpoint>/v1/traces',
        headers: { 'x-mlflow-experiment-id': '<your-experiment-id>' },
      }),
    ),
  ],
});

sdk.start();
init();
registerTelemetry(new LegacyOpenTelemetry());

const result = await generateText({
  model: openai('gpt-5'),
  prompt: 'What is MLflow?',
});

console.log(result.text);
sdk.shutdown();
```

```bash
npx tsx main.ts
```

## 流式

支持流式。与 `generateText` 类似，注册集成后会自动捕获 traces。

```ts
import { streamText } from 'ai';
import { openai } from '@ai-sdk/openai';

const stream = await streamText({
  model: openai('gpt-5'),
  prompt: 'Explain vector databases in one paragraph.',
});

for await (const part of stream.textStream) {
  process.stdout.write(part);
}
```

## 关闭自动 tracing

要关闭 Vercel AI SDK 的 tracing，在调用上设置 `telemetry: { isEnabled: false }`。

## 延伸阅读

- 完成 AI SDK 的 MLflow Tracing 后，还可使用更广的 MLflow GenAI 能力：
  - [Evaluation](https://mlflow.org/docs/latest/genai/eval-monitor/)：使用内置 LLM judge 与数据集管理，系统化度量质量并从开发到生产监控 GenAI 应用。
  - [Prompt Management](https://mlflow.org/docs/latest/genai/prompt-registry/)：集中管理提示词模板，支持版本、别名、血缘与协作，便于团队安全复用与对比。
  - [MCP Server](https://mlflow.org/docs/latest/genai/mcp/)：将编码智能体连接到 MLflow MCP Server，以编程方式交互 MLflow traces 并改进 LLM 应用。
- AI SDK tracing 更多信息见 [telemetry documentation](/docs/ai-sdk-core/telemetry)。
