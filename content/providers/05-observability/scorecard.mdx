# Scorecard

[Scorecard](https://www.scorecard.io/) 是用于监控与评估 LLM 应用的可观测平台。
接入 AI SDK 后，可使用 Scorecard 追踪、监控与分析 LLM 提供商、prompts 与应用流程。

## 设置

Scorecard 支持 [AI SDK telemetry data](/docs/ai-sdk-core/telemetry)。
请在 https://app.scorecard.io 注册，并在 [设置页](https://app.scorecard.io/settings) 获取 API Key。

### Next.js

要通过 AI SDK 向 Scorecard 发送 telemetry，先在项目中设置这些环境变量：

```bash
OTEL_EXPORTER_OTLP_ENDPOINT=https://tracing.scorecard.io/otel/v1/traces
OTEL_EXPORTER_OTLP_HEADERS="Authorization=Bearer <Your Scorecard API Key>"
```

接下来安装 `@ai-sdk/otel`，并在项目根目录创建 `instrumentation.ts`，初始化 OpenTelemetry 并注册 AI SDK 遥测集成：

```typescript filename="instrumentation.ts"
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { registerOTel } from '@vercel/otel';

registerTelemetry(new LegacyOpenTelemetry());

export function register() {
  registerOTel({
    serviceName: 'my-service-name',
  });
}
```

注册集成后，所有 AI SDK 调用会自动采集遥测：

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: 'Tell me a joke',
});
```

## 在 Scorecard 中看到的内容

接入后，你可在 Scorecard 控制台查看：

- **LLM call traces**：每次 AI SDK 调用的详细信息
- **Performance metrics**：延迟、token 用量与成本追踪
- **Model information**：使用的模型与提供商
- **Custom context**：通过 `context` 选项传入的额外上下文
- **Error tracking**：失败请求与调试信息
- **Usage analytics**：LLM 用量的模式与趋势
