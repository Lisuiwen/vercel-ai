# Langfuse 可观测性

[Langfuse](https://langfuse.com/)（[GitHub](https://github.com/langfuse/langfuse)）是开源 LLM 工程平台，帮助团队协作开发、监控与调试 AI 应用。Langfuse 与 AI SDK 集成可提供：

- [应用 traces（追踪）](https://langfuse.com/docs/tracing)
- 使用模式
- 按用户与模型的成本数据
- 回放会话以排查问题
- [Evaluations（评估）](https://langfuse.com/docs/scores/overview)

## 设置

AI SDK 通过 OpenTelemetry 支持 tracing。使用 `LangfuseSpanProcessor` 可将这些 traces 采集到 Langfuse。

安装 `@ai-sdk/otel`，并在 `instrumentation.ts` 注册 `LegacyOpenTelemetry`。注册后，每次 AI SDK 调用会自动发出遥测：

```ts
const result = await generateText({
  model: openai('gpt-4o'),
  prompt: 'Write a short story about a cat.',
});
```

要将 traces 采集到 Langfuse，需要在应用中添加 `LangfuseSpanProcessor`。

可通过环境变量或直接向 `LangfuseSpanProcessor` 构造函数传入 Langfuse 凭据。

获取 Langfuse API keys：可 [自托管 Langfuse](https://langfuse.com/docs/deployment/self-host) 或 [在此](https://cloud.langfuse.com) 注册 Langfuse Cloud。在 Langfuse 控制台创建项目以获取 `secretKey` 与 `publicKey.`

接下来需通过 OpenTelemetry SDK 注册该 span processor。

<Tabs items={["Next.js","Node.js"]}>
<Tab>

Next.js 在框架层面支持 OpenTelemetry instrumentation，详见 [Next.js OpenTelemetry guide](https://nextjs.org/docs/app/building-your-application/optimizing/open-telemetry)。

安装依赖：

```bash
npm install @langfuse/otel @langfuse/tracing @opentelemetry/sdk-trace-node @ai-sdk/otel
```

使用 `NodeTracerProvider` 进行手动 OTel 设置，并向 instrumentation 添加 `LangfuseSpanProcessor`：

```ts filename="instrumentation.ts"
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { LangfuseSpanProcessor, ShouldExportSpan } from '@langfuse/otel';
import { NodeTracerProvider } from '@opentelemetry/sdk-trace-node';

registerTelemetry(new LegacyOpenTelemetry());

// Optional: filter out Next.js infra spans
const shouldExportSpan: ShouldExportSpan = span => {
  return span.otelSpan.instrumentationScope.name !== 'next.js';
};

export const langfuseSpanProcessor = new LangfuseSpanProcessor({
  shouldExportSpan,
});

const tracerProvider = new NodeTracerProvider({
  spanProcessors: [langfuseSpanProcessor],
});

tracerProvider.register();
```

</Tab>
<Tab>

安装依赖：

```bash
npm install ai @ai-sdk/openai @langfuse/otel @opentelemetry/sdk-node @ai-sdk/otel
```

在你的 OpenTelemetry 设置中加入 `LangfuseSpanProcessor`：

```ts highlight="3-6, 8-10, 13"
import { openai } from '@ai-sdk/openai';
import { registerTelemetry, generateText } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { LangfuseSpanProcessor } from '@langfuse/otel';
import { NodeSDK } from '@opentelemetry/sdk-node';

const sdk = new NodeSDK({
  spanProcessors: [new LangfuseSpanProcessor()],
});

sdk.start();
registerTelemetry(new LegacyOpenTelemetry());

async function main() {
  const result = await generateText({
    model: openai('gpt-4o'),
    maxOutputTokens: 50,
    prompt: 'Invent a new holiday and describe its traditions.',
    telemetry: {
      functionId: 'my-awesome-function',
      metadata: {
        something: 'custom',
        someOtherThing: 'other-value',
      },
    },
  });

  console.log(result.text);

  await sdk.shutdown(); // Flushes the trace to Langfuse
}

main().catch(console.error);
```

</Tab>
</Tabs>

完成！包含 AI SDK span 的所有 traces 会自动采集到 Langfuse。

## 配置

### 将多次执行归入同一 trace

可开启 Langfuse trace，并将 trace ID 传入 AI SDK 调用，把多次执行的 span 归到同一 trace 下。`functionId` 中传入的名称会成为对应执行的根 span 名称。

```ts
import { randomUUID } from 'crypto';
import { Langfuse } from 'langfuse';

const langfuse = new Langfuse();
const parentTraceId = randomUUID();

langfuse.trace({
  id: parentTraceId,
  name: 'holiday-traditions',
});

for (let i = 0; i < 3; i++) {
  const result = await generateText({
    model: openai('gpt-3.5-turbo'),
    maxOutputTokens: 50,
    prompt: 'Invent a new holiday and describe its traditions.',
    telemetry: {
      functionId: `holiday-tradition-${i}`,
      metadata: {
        langfuseTraceId: parentTraceId,
        langfuseUpdateParent: false, // Do not update the parent trace with execution results
      },
    },
  });

  console.log(result.text);
}

await langfuse.flushAsync();
await sdk.shutdown();
```

### 关闭输入/输出记录

默认情况下，exporter 会记录每次请求的输入与输出。可将 `recordInputs` 与 `recordOutputs` 设为 `false` 以关闭。

### 将 Langfuse prompts 关联到 traces

可在 `metadata` 中设置 `langfusePrompt`，将 Langfuse prompt 关联到 AI SDK 的 generation：

```typescript
import { generateText } from 'ai';
import { Langfuse } from 'langfuse';

const langfuse = new Langfuse();

const fetchedPrompt = await langfuse.getPrompt('my-prompt');

const result = await generateText({
  model: openai('gpt-4o'),
  prompt: fetchedPrompt.prompt,
  telemetry: {
    metadata: {
      langfusePrompt: fetchedPrompt.toJSON(),
    },
  },
});
```

对应的 generation 会在 Langfuse 的 trace 中关联该 prompt。关于 Langfuse prompts 详见 [此处](https://langfuse.com/docs/prompts/get-started)。

### 传入自定义属性

exporter 会自动采集所有 `metadata` 字段。也可传入自定义 trace 属性（例如追踪用户或会话）。

```ts highlight="5-11"
const result = await generateText({
  model: openai('gpt-4o'),
  prompt: 'Write a short story about a cat.',
  telemetry: {
    functionId: 'my-awesome-function', // Trace name
    metadata: {
      langfuseTraceId: 'trace-123', // Langfuse trace
      tags: ['story', 'cat'], // Custom tags
      userId: 'user-123', // Langfuse user
      sessionId: 'session-456', // Langfuse session
      foo: 'bar', // Any custom attribute recorded in metadata
    },
  },
});
```

## 故障排除

- 若部署在 Vercel，Vercel 的 OpenTelemetry Collector 仅在 Pro 与企业套餐可用（[文档](https://vercel.com/docs/observability/otel-overview)）。
- 需使用 `"ai": "^7.0.0"` 并安装 `@ai-sdk/otel` 才能使用遥测。若有问题请升级到最新版本。
- Next.js 上请确保仅有一个 instrumentation 文件。
- 若使用 Sentry，请确保：
  - 在 `Sentry.init` 中设置 `skipOpenTelemetrySetup: true`，或
  - 按 Sentry 文档手动将 Sentry 与 OTEL 配置在一起
