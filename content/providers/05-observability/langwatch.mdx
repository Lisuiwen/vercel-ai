# LangWatch 可观测性

[LangWatch](https://langwatch.ai/)（[GitHub](https://github.com/langwatch/langwatch)）是 LLM Ops 平台，用于监控、实验、度量与改进 LLM 流水线，采用 fair-code 分发模式。

## 设置

从 [LangWatch dashboard](https://app.langwatch.com/) 获取 `LANGWATCH_API_KEY`。

## 基本概念

- 触发整条 LLM 流水线的每条消息会作为 [Trace](https://docs.langwatch.ai/concepts#traces) 记录。
- [Trace](https://docs.langwatch.ai/concepts#traces) 包含多个 [Spans](https://docs.langwatch.ai/concepts#spans)，即流水线中的步骤。
  - Span 可以是 LLM 调用、RAG 检索的数据库查询，或简单的函数变换。
  - 不同类型的 [Spans](https://docs.langwatch.ai/concepts#spans) 会捕获不同参数。
  - [Spans](https://docs.langwatch.ai/concepts#spans) 可嵌套以反映流水线结构。
- 若 [Traces](https://docs.langwatch.ai/concepts#traces) 的 metadata 中有相同的 [`thread_id`](https://docs.langwatch.ai/concepts#threads)，LangWatch Dashboard 会将它们归为同一会话，使单条消息成为对话的一部分。
  - 也建议在 metadata 中提供 [`user_id`](https://docs.langwatch.ai/concepts#user-id) 以便做用户侧分析。

## 配置

AI SDK 支持通过 Next.js 的 OpenTelemetry 集成进行 tracing。使用 `LangWatchExporter` 可自动将这些 traces 采集到 LangWatch。

首先安装必要依赖：

```bash
npm install @vercel/otel langwatch @opentelemetry/api-logs @opentelemetry/instrumentation @opentelemetry/sdk-logs @ai-sdk/otel
```

接着为应用配置 OpenTelemetry，根据你使用的是 Next.js 上的 AI SDK 还是 Node.js，选择下方对应页签：

<Tabs items={['Next.js', 'Node.js']}>
<Tab title="Next.js">

若尚未启用，需在 `next.config.js` 中启用 `instrumentationHook`：

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    instrumentationHook: true,
  },
};

module.exports = nextConfig;
```

然后在项目**根目录**创建 `instrumentation.ts`（或 `.js`，若使用 `src` 文件夹则放在 `src` 内），并以 `LangWatchExporter` 作为 traceExporter：

```typescript
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { registerOTel } from '@vercel/otel';
import { LangWatchExporter } from 'langwatch';

registerTelemetry(new LegacyOpenTelemetry());

export function register() {
  registerOTel({
    serviceName: 'next-app',
    traceExporter: new LangWatchExporter(),
  });
}
```

（Next.js OpenTelemetry 配置的更多说明见 [官方指南](https://nextjs.org/docs/app/building-your-application/optimizing/open-telemetry#manual-opentelemetry-configuration)）

最后，在需要追踪的 AI SDK 调用上启用 `telemetry`：

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: openai('gpt-4o-mini'),
  prompt:
    'Explain why a chicken would make a terrible astronaut, be creative and humorous about it.',
  telemetry: {
    // optional metadata
    metadata: {
      userId: 'myuser-123',
      threadId: 'mythread-123',
    },
  },
});
```

</Tab>
<Tab title="Node.js">
在 Node.js 上，请先遵循官方 OpenTelemetry 入门文档：

- [OpenTelemetry Node.js Getting Started](https://opentelemetry.io/docs/languages/js/getting-started/nodejs/)

完成 OpenTelemetry 设置后，可使用 `LangWatchExporter` 自动将 traces 发送到 LangWatch：

```typescript
import { LangWatchExporter } from 'langwatch';

const sdk = new NodeSDK({
  traceExporter: new LangWatchExporter({
    apiKey: process.env.LANGWATCH_API_KEY,
  }),
  // ...
});
```

</Tab>
</Tabs>

### 手动集成

若未使用 AI SDK 的 OpenTelemetry 集成，以下文档适用于手动集成：可手动开启 trace 以捕获消息：

```typescript
import { LangWatch } from 'langwatch';

const langwatch = new LangWatch();

const trace = langwatch.getTrace({
  metadata: { threadId: 'mythread-123', userId: 'myuser-123' },
});
```

然后在 trace 内启动 LLM span，记录即将发往 LLM 的输入：

```typescript
const span = trace.startLLMSpan({
  name: 'llm',
  model: model,
  input: {
    type: 'chat_messages',
    value: messages,
  },
});
```

这会记录 LLM 输入并注册调用开始时间。LLM 调用结束后结束 span，以记录结束时间戳并捕获输出与 token 指标（用于成本计算等），例如：

```typescript
span.end({
  output: {
    type: 'chat_messages',
    value: [chatCompletion.choices[0]!.message],
  },
  metrics: {
    promptTokens: chatCompletion.usage?.prompt_tokens,
    completionTokens: chatCompletion.usage?.completion_tokens,
  },
});
```
