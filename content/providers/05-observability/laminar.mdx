# Laminar 可观测性

[Laminar](https://laminar.sh) 是用于追踪与评估 AI 应用的开源平台。

Laminar 提供：

- [与 AI SDK 等兼容的 Tracing](https://docs.lmnr.ai/tracing/introduction)，
- [Evaluations（评估）](https://docs.lmnr.ai/evaluations/introduction)，
- [浏览器智能体可观测性](https://docs.lmnr.ai/tracing/browser-agent-observability)

<Note>
  [Laminar 文档](https://docs.lmnr.ai/tracing/integrations/vercel-ai-sdk) 中亦有本指南版本。
</Note>

## 设置

Laminar 的 tracing 基于 OpenTelemetry，支持 AI SDK [telemetry](/docs/ai-sdk-core/telemetry)。

### 安装

开始使用 Laminar tracing 时，请先 [安装](https://docs.lmnr.ai/installation) `@lmnr-ai/lmnr` 包。

### 获取项目 API key 并配置环境变量

在 [Laminar](https://laminar.sh) 注册，或自建实例（[github](https://github.com/lmnr-ai/lmnr)）并创建新项目。

在项目设置中创建并复制 API key。

在 `.env` 中：

```bash
LMNR_PROJECT_API_KEY=...
```

## Next.js

### 初始化 tracing

在 Next.js 中，Laminar 初始化与 AI SDK 遥测集成均应在 `instrumentation.{ts,js}` 中完成：

```javascript
export async function register() {
  // prevent this from running in the edge runtime
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    const { registerTelemetry } = await import('ai');
    const { LegacyOpenTelemetry } = await import('@ai-sdk/otel');
    const { Laminar, getTracer } = await import('@lmnr-ai/lmnr');

    Laminar.initialize({
      projectApiKey: process.env.LMNR_PROJECT_API_KEY,
    });
    registerTelemetry(new LegacyOpenTelemetry({ tracer: getTracer() }));
  }
}
```

### 将 @lmnr-ai/lmnr 加入 next.config

在 `next.config.js`（`.ts` / `.mjs`）中加入：

```javascript
const nextConfig = {
  serverExternalPackages: ['@lmnr-ai/lmnr'],
};

export default nextConfig;
```

Laminar 依赖 OpenTelemetry，其中使用部分 Node.js 专有能力，需告知 Next.js。详见 [Next.js 文档](https://nextjs.org/docs/app/api-reference/config/next-config-js/serverExternalPackages)。

### 追踪 AI SDK 调用

注册集成后，每次 AI SDK 调用会自动采集遥测：

```javascript
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: 'What is Laminar flow?',
});
```

会为 `ai.generateText` 创建 span。Laminar 收集并展示：

- LLM 调用输入与输出
- 开始与结束时间
- 耗时 / 延迟
- 提供方与所用模型
- 输入与输出 token
- 输入与输出价格
- 其他元数据与 span 属性

### 较早版本的 Next.js

若使用 13.4 ≤ Next.js &lt; 15，还需开启实验性 instrumentation hook。在 `next.config.js` 中加入：

```javascript
module.exports = {
  experimental: {
    instrumentationHook: true,
  },
};
```

更多信息见 Laminar 的 [Next.js guide](https://docs.lmnr.ai/tracing/nextjs) 与 Next.js [instrumentation 文档](https://nextjs.org/docs/app/api-reference/file-conventions/instrumentation)。也可在文档中了解如何为 Next.js 启用全部 traces。

### 与 `@vercel/otel` 共用

Laminar 可与 `@vercel/otel` 共存并追踪 AI SDK 调用。默认 Laminar 配置会确保：

- 常规 Next.js traces 经由 `@vercel/otel` 发往你在 Vercel 配置的 Telemetry 后端，
- AI SDK 及其他 LLM 或浏览器智能体 traces 经由 Laminar 发送。

```javascript
import { registerOTel } from '@vercel/otel';

export async function register() {
  registerOTel('my-service-name');
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    const { registerTelemetry } = await import('ai');
    const { LegacyOpenTelemetry } = await import('@ai-sdk/otel');
    const { Laminar } = await import('@lmnr-ai/lmnr');
    // Make sure to initialize Laminar **after** `@registerOTel`
    Laminar.initialize({
      projectApiKey: process.env.LMNR_PROJECT_API_KEY,
    });
    registerTelemetry(new LegacyOpenTelemetry());
  }
}
```

若需将 Next.js 全部 traces 经 Laminar 导出，参见示例 [仓库](https://github.com/lmnr-ai/lmnr-ts/tree/main/examples/nextjs)。

### 与 `@sentry/node` 共用

Laminar 可与 `@sentry/node` 共存并追踪 AI SDK 调用。请在 `Sentry.init` **之后** 初始化 Laminar。

这样会确保：

- 由 Sentry 埋点的内容发往你的 Sentry 后端，
- AI SDK 及其他 LLM 或浏览器智能体 traces 经由 Laminar 发送。

```javascript
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    const { registerTelemetry } = await import('ai');
    const { LegacyOpenTelemetry } = await import('@ai-sdk/otel');
    const Sentry = await import('@sentry/node');
    const { Laminar } = await import('@lmnr-ai/lmnr');

    Sentry.init({
      dsn: process.env.SENTRY_DSN,
    });

    // Make sure to initialize Laminar **after** `Sentry.init`
    Laminar.initialize({
      projectApiKey: process.env.LMNR_PROJECT_API_KEY,
    });
    registerTelemetry(new LegacyOpenTelemetry());
  }
}
```

## Node.js

### 初始化 tracing

在应用中初始化 tracing：

```javascript
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { Laminar, getTracer } from '@lmnr-ai/lmnr';

Laminar.initialize();
registerTelemetry(new LegacyOpenTelemetry({ tracer: getTracer() }));
```

应用中只需执行一次，且应尽早执行，但须在其它 tracing 库（如 `@sentry/node`）初始化**之后**。

详见 Laminar [文档](https://docs.lmnr.ai/tracing/introduction)。

### 追踪 AI SDK 调用

注册集成后，每次 AI SDK 调用会自动采集遥测：

```javascript
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';

const { text } = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: 'What is Laminar flow?',
});
```

会为 `ai.generateText` 创建 span。Laminar 收集并展示：

- LLM 调用输入与输出
- 开始与结束时间
- 耗时 / 延迟
- 提供方与所用模型
- 输入与输出 token
- 输入与输出价格
- 其他元数据与 span 属性

### 与 `@sentry/node` 共用

Laminar 可与 `@sentry/node` 配合追踪 AI SDK 调用。请在 `Sentry.init` **之后** 初始化 Laminar：

```javascript
const Sentry = await import('@sentry/node');
const { Laminar } = await import('@lmnr-ai/lmnr');

Sentry.init({
  dsn: process.env.SENTRY_DSN,
});

Laminar.initialize({
  projectApiKey: process.env.LMNR_PROJECT_API_KEY,
});
```

这样会确保：

- 由 Sentry 埋点的内容发往你的 Sentry 后端，
- AI SDK 及其他 LLM 或浏览器智能体 traces 经由 Laminar 发送。

两库还支持更多高级配置，上述默认接入方式通常已足够。

## 额外配置

### Span 名称

若要覆盖默认 span 名称，可在 `telemetry` 选项中设置 `functionId`。

```javascript
const { text } = await generateText({
  model: openai('gpt-4.1-nano'),
  prompt: `Write a poem about Laminar flow.`,
  telemetry: {
    functionId: 'poem-writer',
  },
});
```

### 嵌套 span

若要追踪 AI SDK 调用之外的应用内其它函数，可使用 Laminar 的 `observe` 包装器。

```javascript highlight="3"
import { observe } from '@lmnr-ai/lmnr';

const result = await observe({ name: 'my-function' }, async () => {
  // ... some work
  await generateText({
    //...
  });
  // ... some work
});
```

会创建名为 "my-function" 的 span 并追踪该函数调用；其内可看到嵌套的 `ai.generateText` span。

若要追踪 `observe` 所包装函数的入参，请将其作为额外参数传入包装器；函数返回值会从包装器原样返回，并作为 span 的输出被记录。

```javascript
const result = await observe(
  { name: 'poem writer' },
  async (topic: string, mood: string) => {
    const { text } = await generateText({
      model: openai('gpt-4.1-nano'),
      prompt: `Write a poem about ${topic} in ${mood} mood.`,
    });
    return text;
  },
  'Laminar flow',
  'happy',
);
```

### 元数据（Metadata）

在 Laminar 中，metadata 设置在 trace 级别，为键值对，可用于筛选 traces。

```javascript
const { text } = await generateText({
  model: openai('gpt-4.1-nano'),
  prompt: `Write a poem about Laminar flow.`,
  telemetry: {
    metadata: {
      'my-key': 'my-value',
      'another-key': 'another-value',
    },
  },
});
```

会转换为 Laminar 的 [metadata](https://docs.lmnr.ai/tracing/structure/metadata) 并存入 trace。

### 标签（Tags）

预留的 metadata 键之一是 `tags`，可用于向 span 添加 [tags](https://docs.lmnr.ai/tracing/structure/tags)。

随后可在 Laminar 中用标签筛选 traces。

```javascript
const { text } = await generateText({
  model: openai('gpt-4.1-nano'),
  prompt: `Write a poem about Laminar flow.`,
  telemetry: {
    metadata: {
      tags: ['fallback-model', 'api-handler'],
    },
  },
});
```

### Session ID 与 User ID

Laminar 中的 traces 可按 [sessions](https://docs.lmnr.ai/tracing/structure/session) 或 [user ID](https://docs.lmnr.ai/tracing/structure/user-id) 分组，对应保留的 metadata 键。

```javascript
const { text } = await generateText({
  model: openai('gpt-4.1-nano'),
  prompt: `Write a poem about Laminar flow.`,
  telemetry: {
    metadata: {
      sessionId: 'session-123',
      userId: 'user-123',
    },
  },
});
```
