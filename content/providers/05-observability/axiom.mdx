# Axiom 可观测性

**Axiom** 是具有 **AI 工程工作流** 专项能力的数据平台，帮助你充满信心地构建复杂 AI 系统。

Axiom 与 AI SDK 的集成采用模型包装器，自动捕获每次 LLM 调用的详细追踪，即时呈现应用的性能、成本与行为。

## 设置

### 1. 配置 Axiom

首先你需要 Axiom 组织、用于接收 traces 的数据集以及 API token。

- [创建 Axiom 组织](https://app.axiom.co/register)。
- [创建新数据集](https://app.axiom.co/datasets)（例如 `my-ai-app`）。
- [创建 API token](https://app.axiom.co/settings/api-tokens)，并赋予对应数据集的写入权限。

### 3. 设置环境变量

在 `.env` 中配置环境变量。以下为标准 OpenTelemetry 配置，将 traces 直接发送到 Axiom 数据集。

```bash filename=".env"
# Axiom Configuration
AXIOM_TOKEN="YOUR_AXIOM_API_TOKEN"
AXIOM_DATASET="your-axiom-dataset-name"

# Vercel and OpenTelemetry Configuration
OTEL_SERVICE_NAME="my-ai-app"
OTEL_EXPORTER_OTLP_ENDPOINT="https://api.axiom.co/v1/traces"
OTEL_EXPORTER_OTLP_HEADERS="Authorization=Bearer YOUR_AXIOM_API_TOKEN,X-Axiom-Dataset=your-axiom-dataset-name"

# Your AI Provider Key
OPENAI_API_KEY="YOUR_OPENAI_API_KEY"
```

将占位符替换为你的 Axiom token 与数据集名称。

2. 创建 instrumentation 文件：

```typescript filename="src/instrumentation.ts"
import { trace } from '@opentelemetry/api';
import { OTLPTraceExporter } from '@opentelemetry/exporter-trace-otlp-http';
import type { Resource } from '@opentelemetry/resources';
import { resourceFromAttributes } from '@opentelemetry/resources';
import { NodeSDK } from '@opentelemetry/sdk-node';
import { SimpleSpanProcessor } from '@opentelemetry/sdk-trace-node';
import { ATTR_SERVICE_NAME } from '@opentelemetry/semantic-conventions';
import { initAxiomAI, RedactionPolicy } from 'axiom/ai';

const tracer = trace.getTracer('my-tracer');

const sdk = new NodeSDK({
  resource: resourceFromAttributes({
    [ATTR_SERVICE_NAME]: 'my-ai-app',
  }) as Resource,
  spanProcessor: new SimpleSpanProcessor(
    new OTLPTraceExporter({
      url: `https://api.axiom.co/v1/traces`,
      headers: {
        Authorization: `Bearer ${process.env.AXIOM_TOKEN}`,
        'X-Axiom-Dataset': process.env.AXIOM_DATASET,
      },
    }),
  ),
});

sdk.start();

initAxiomAI({ tracer, redactionPolicy: RedactionPolicy.AxiomDefault });
```

### 5. 包装并使用 AI 模型

在应用代码中，从 Axiom 导入 `wrapAISDKModel`，用它来包装已有的 AI SDK 模型客户端。

```typescript
import { createOpenAI } from '@ai-sdk/openai';
import { generateText } from 'ai';
import { wrapAISDKModel } from 'axiom/ai';

// 1. Create your standard AI model provider
const openaiProvider = createOpenAI({
  apiKey: process.env.OPENAI_API_KEY,
});

// 2. Wrap the model to enable automatic tracing
const tracedGpt4o = wrapAISDKModel(openaiProvider('gpt-4o'));

// 3. Use the wrapped model as you normally would
const { text } = await generateText({
  model: tracedGpt4o,
  prompt: 'What is the capital of Spain?',
});

console.log(text);
```

使用 `tracedGpt4o` 模型发起的调用会自动将详细 traces 发送到你的 Axiom 数据集。

## 在 Axiom 中你将看到的内容

接入后，Axiom 数据集中将包含：

- **AI Trace Waterfall：** 用于可视化单步与多步 LLM 工作流的专用视图。
- **Gen AI Dashboard：** 预制面板，监控成本、延迟、token 用量与错误率。
- **Detailed Spans：** 每次调用的丰富遥测，含完整 prompt 与 completion、token 计数与模型信息。

## 高级用法

Axiom 的 AI SDK 集成还提供更深度的 instrumentation：

- **业务上下文：** 使用 `withSpan`，将若干 LLM 调用归组到某项业务能力下（例如 `customer_support_agent`）。
- **工具追踪：** 使用 `wrapTool`，自动追踪模型调用的工具执行过程。

进一步了解请参阅 [Axiom AI SDK Instrumentation guide](https://axiom.co/docs/ai-engineering/observe/axiom-ai-sdk-instrumentation)。
