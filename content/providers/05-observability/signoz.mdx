# SigNoz 可观测性

[SigNoz](https://signoz.io/) 是一套覆盖多种监控需求的统一工具。选择 SigNoz 的几个理由：

- 单一工具完成可观测性（logs、metrics 与 traces）
- 建立在 [OpenTelemetry](https://opentelemetry.io/) 之上，开源标准可避免厂商锁定
- 关联 logs、metrics 与 traces，排障上下文更丰富
- 使用 ClickHouse（Uber、Cloudflare 等也在用）作为存储，专为可观测数据高速优化
- 支持 DIY Query builder、PromQL 与 ClickHouse 查询，满足各类分析需求

# 设置

- 创建 [SigNoz Cloud Account](https://signoz.io/teams/)
- 生成 SigNoz Ingestion Key

## 为 Next.js 应用添加 instrumentation

如何在 Next.js 应用中设置 OpenTelemetry instrumentation，并在 SigNoz 查看应用 traces，详见 [此处](https://signoz.io/docs/instrumentation/opentelemetry-nextjs/)。

## 直接向 SigNoz Cloud 发送 traces

**步骤 1。** 安装 OpenTelemetry 相关包

```bash
npm install @vercel/otel @opentelemetry/api @ai-sdk/otel
```

**步骤 2。** 更新 **`next.config.mjs`**，启用 instrumentationHook

> 仅在 NextJs 14 及以下版本需要此步骤

```jsx
/** @type {import('next').NextConfig} */
const nextConfig = {
  // include instrumentationHook experimental feature
  experimental: {
    instrumentationHook: true,
  },
};
export default nextConfig;
```

**步骤 3。** 在项目根目录创建 **`instrumentation.ts`**

```jsx
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';
import { registerOTel, OTLPHttpJsonTraceExporter } from '@vercel/otel';
// Add otel logging
import { diag, DiagConsoleLogger, DiagLogLevel } from '@opentelemetry/api';
diag.setLogger(new DiagConsoleLogger(), DiagLogLevel.ERROR); // set diag log level to DEBUG when debugging

registerTelemetry(new LegacyOpenTelemetry());

export function register() {
  registerOTel({
    serviceName: '<service_name>',
    traceExporter: new OTLPHttpJsonTraceExporter({
      url: 'https://ingest.<region>.signoz.cloud:443/v1/traces',
      headers: { 'signoz-ingestion-key': '<your-ingestion-key>' },
    }),
  });
}
```

- **`<service_name>`** 为你的服务名称
- 将 **`<region>`** 设为与 SigNoz Cloud [**区域**](https://signoz.io/docs/ingestion/signoz-cloud/overview/#endpoint) 一致
- 将 **`<your-ingestion-key>`** 替换为你的 SigNoz [**ingestion key**](https://signoz.io/docs/ingestion/signoz-cloud/keys/)

> instrumentation 文件应位于项目根目录，不要放在 app 或 pages 目录内。若使用 src 文件夹，请将文件放在与 pages、app 同级的 src 下。

你的 Next.js 应用现已正确完成 instrumentation。

## 为 Vercel AI SDK 启用 Telemetry

Vercel AI SDK 使用 [OpenTelemetry](https://signoz.io/blog/what-is-opentelemetry/) 采集遥测数据。OpenTelemetry 是开源可观测框架，提供标准化的 instrumentation 以采集遥测。

## 启用 Telemetry

关于 Vercel AI SDK 的遥测选项，更多说明见 [此处](https://ai-sdk.dev/docs/ai-sdk-core/telemetry#telemetry)。

安装 `@ai-sdk/otel` 并在 `instrumentation.ts` 中加入 `registerTelemetry`（见上文步骤 3）后，每次 AI SDK 调用会自动采集遥测：

```jsx
const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Write a short story about a cat.',
});
```

你也可以控制是否记录函数的输入值与输出值。默认二者均启用。可将 `recordInputs` 与 `recordOutputs` 设为 `false` 以关闭：

```jsx
telemetry: { recordInputs: false, recordOutputs: false }
```

关闭输入输出记录可能对隐私、数据传输与性能有益。例如，输入包含敏感信息时，你可能希望关闭输入记录。

## Telemetry Metadata

可提供 `functionId` 标明遥测所属函数，并通过 `context` 在遥测中加入额外信息。

```jsx
const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Write a short story about a cat.',
  context: {
    something: 'custom',
    someOtherThing: 'other-value',
  },
  telemetry: {
    functionId: 'my-awesome-function',
  },
});
```

## Custom Tracer

若希望 traces 使用非 `@opentelemetry/api` 单例提供的 `TracerProvider`，可向 `LegacyOpenTelemetry` 构造函数传入自定义 `Tracer`：

```jsx
import { registerTelemetry, generateText } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';

const tracerProvider = new NodeTracerProvider();
registerTelemetry(
  new LegacyOpenTelemetry({
    tracer: tracerProvider.getTracer('ai'),
  }),
);

const result = await generateText({
  model: openai('gpt-5-mini'),
  prompt: 'Write a short story about a cat.',
});
```

你的 Vercel AI SDK 调用现在会自动发出 traces、spans 与 events。生成的 span 与 event 类型见 [此处](https://ai-sdk.dev/docs/ai-sdk-core/telemetry#collected-data)。

最后，你应能在 SigNoz Cloud 的 traces 标签页中查看这些数据。
