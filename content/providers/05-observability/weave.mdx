# Weave 可观测性

[Weave](https://wandb.ai/site/weave) 由 [Weights & Biases](https://wandb.ai/site/) 打造，用于跟踪、实验、评估、部署并改进基于 LLM 的应用。

接入 AI SDK 后，可使用 Weave 查看并与 AI SDK 应用的 trace 信息交互，包括 prompts、回复、流程、成本等。

## 设置

要将 Weave 配置为 [OpenTelemetry](https://opentelemetry.io/docs/) 后端，需将 traces 路由到 Weave 的 OpenTelemetry 端点、设置 API key，并指定团队与项目。要将 traces 记录到 Weave，你需要拥有 [Weights & Biases 账户](https://wandb.ai/site/weave)。

### 认证

首先访问 [wandb.ai/authorize](https://wandb.ai/authorize)，复制 API key，并运行以下命令生成 base64 授权串：

```bash
echo -n "api:<YOUR_API_KEY>" | base64
```

记下输出，后续环境配置会使用。

### 项目配置

W&B project ID 决定遥测写入位置，格式为 `<YOUR_TEAM_NAME>/<YOUR_PROJECT_NAME>`。

1. 打开 [Weights & Biases dashboard](https://wandb.ai/home)。
2. 在 **Teams** 区域选择或创建团队。
3. 选择已有项目或新建项目。
4. 记下 `<YOUR_TEAM_NAME>/<YOUR_PROJECT_NAME>` 供下一步使用。

### Next.js

在 Next.js 应用的 `.env` 中设置 OTEL 环境变量。将 `<BASE64_AUTH_STRING>` 与 `<YOUR_TEAM_NAME>/<YOUR_PROJECT_NAME>` 替换为上一步的值：

```bash
OTEL_EXPORTER_OTLP_ENDPOINT="https://trace.wandb.ai/otel/v1/traces"
OTEL_EXPORTER_OTLP_HEADERS="Authorization=Basic <BASE64_AUTH_STRING>,project_id=<YOUR_TEAM_NAME>/<YOUR_PROJECT_NAME>"
```

安装 `@ai-sdk/otel`，并在 `instrumentation.ts` 注册 `LegacyOpenTelemetry`：

```typescript filename="instrumentation.ts"
import { registerTelemetry } from 'ai';
import { LegacyOpenTelemetry } from '@ai-sdk/otel';

registerTelemetry(new LegacyOpenTelemetry());
```

随后可在支持的 AI SDK 函数调用上使用 `telemetry` 选项启用遥测：

```typescript
import { generateText } from 'ai';
import { openai } from '@ai-sdk/openai';

const result = await generateText({
  model: openai('gpt-4o-mini'),
  prompt: 'What is 2 + 2?',
  telemetry: {
    metadata: {
      query: 'math',
      difficulty: 'easy',
    },
  },
});
```
