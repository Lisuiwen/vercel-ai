# 动态 Prompt 缓存

构建 agent 时，对话变长会迅速推高 API 成本。许多提供商提供 prompt 缓存，可缓存对话前缀，显著降低重复上下文的费用。

本示例展示可复制到项目中并按提供商与策略定制的模式。示例默认实现 Anthropic 推荐做法，也可按需扩展其他提供商。

尤其适用于：

1. **长对话 agent** — 多轮交互会累积上下文，每次请求都会重发。
2. **大量使用工具** — 工具调用与结果带来大量 token，适合缓存。

非 Anthropic 模型时消息原样透传，可在与提供商无关的代码中安全使用。

## 实现

该工具向消息添加 Anthropic 的 `cacheControl` 指令，在最终消息上标记 `{ type: "ephemeral" }`，告知 Anthropic 缓存此前全部内容，后续请求仅对新内容按全价计费。

### 工作原理

函数检测模型提供商并应用相应缓存策略。本实现通过提供商名与模型 ID 判断 Anthropic 模型，在消息数组最后一条上添加 `providerOptions` 与 `cacheControl: { type: "ephemeral" }`。Anthropic 文档要求：「在最终消息的最终块上标记 cache_control，以便对话可增量缓存。」

非 Anthropic 模型时原样返回消息；可通过检测逻辑与提供商专属选项扩展其他提供商。

### 消息级与块级缓存控制

本实现在**消息级**设置 `providerOptions`，而 Anthropic API 在**内容块级**期望 `cache_control`。AI SDK 会自动完成转换。

在消息上设置 `providerOptions` 时，SDK 在构造 API 请求时将其应用到最后一个内容块。例如：

```ts
// What you write (message-level)
{
  role: 'user',
  content: [
    { type: 'text', text: 'First part' },
    { type: 'text', text: 'Second part' },
  ],
  providerOptions: {
    anthropic: { cacheControl: { type: 'ephemeral' } },
  },
}

// What the SDK sends to Anthropic (block-level)
{
  "role": "user",
  "content": [
    { "type": "text", "text": "First part" },
    { "type": "text", "text": "Second part", "cache_control": { "type": "ephemeral" } }
  ]
}
```

该行为是有意设计，对用户消息、助手消息与工具结果一致。若需更细粒度控制，也可直接在单个内容部分设置 `providerOptions`，其优先级高于消息级设置。

### 工具函数

```ts
import type { ModelMessage, JSONValue, LanguageModel } from 'ai';

function isAnthropicModel(model: LanguageModel): boolean {
  if (typeof model === 'string') {
    return model.includes('anthropic') || model.includes('claude');
  }
  return (
    model.provider === 'anthropic' ||
    model.provider.includes('anthropic') ||
    model.modelId.includes('anthropic') ||
    model.modelId.includes('claude')
  );
}

export function addCacheControlToMessages({
  messages,
  model,
  providerOptions = {
    anthropic: { cacheControl: { type: 'ephemeral' } },
  },
}: {
  messages: ModelMessage[];
  model: LanguageModel;
  providerOptions?: Record<string, Record<string, JSONValue>>;
}): ModelMessage[] {
  if (messages.length === 0) return messages;
  if (!isAnthropicModel(model)) return messages;

  return messages.map((message, index) => {
    if (index === messages.length - 1) {
      return {
        ...message,
        providerOptions: {
          ...message.providerOptions,
          ...providerOptions,
        },
      };
    }
    return message;
  });
}
```

## 使用工具函数

通过 `generateText` 的 `prepareStep` 回调与 `stopWhen` 将工具集成到 agent：

```ts
import { anthropic } from '@ai-sdk/anthropic';
import { generateText, tool, isStepCount } from 'ai';
import { z } from 'zod';
import { addCacheControlToMessages } from './add-cache-control-to-messages';

async function main() {
  const result = await generateText({
    model: anthropic('claude-sonnet-4-5'),
    prompt: 'Help me analyze this codebase and suggest improvements.',
    stopWhen: isStepCount(10),
    tools: {
      // your tools here
      analyzeFile: tool({
        description: 'Analyze a file in the codebase',
        inputSchema: z.object({
          path: z.string().describe('Path to the file'),
        }),
        execute: async ({ path }) => {
          // implementation
          return { analysis: `Analysis of ${path}` };
        },
      }),
    },
    prepareStep: ({ messages, model }) => ({
      messages: addCacheControlToMessages({ messages, model }),
    }),
  });

  console.log(result.text);
}

main().catch(console.error);
```

也可按需自定义 cache control 选项：

```ts
prepareStep: ({ messages, model }) => ({
  messages: addCacheControlToMessages({
    messages,
    model,
    providerOptions: {
      anthropic: { cacheControl: { type: "ephemeral" } },
    },
  }),
}),
```

## 注意事项

使用时请注意：

1. **提供商专属行为** — 本实现面向 Anthropic；其他提供商消息不变，可扩展模式以支持更多提供商。
2. **最低 token 阈值** — Anthropic 需达到最少 token 才启用缓存，短对话可能无法受益；其他提供商或有类似要求。
3. **缓存寿命** — Anthropic 临时缓存 TTL 为 5 分钟，不活跃对话会失效；请参阅各提供商文档。
4. **成本结构** — Anthropic 缓存 token 按输入 token 的 10% 计费，写入缓存多 25%；命中多于未命中才省钱；各提供商成本结构不同。
