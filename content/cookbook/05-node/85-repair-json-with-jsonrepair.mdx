# 使用 jsonrepair 修复畸形 JSON

生成结构化输出时，语言模型有时会产出畸形 JSON，常见原因包括：

- 响应被截断（达到 token 上限）
- 语法错误（如用单引号代替双引号）
- 缺少闭合括号或花括号
- 尾随逗号

在 AI SDK v6 中，实用模式是：生成文本 → 修复 → 解析并校验。结合 [`jsonrepair`](https://github.com/josdejong/jsonrepair) 库，可自动修复许多常见 JSON 问题。

## 安装

首先安装 `jsonrepair` 库：

```bash
pnpm add jsonrepair
```

## 与 generateText 配合使用

以下展示如何将 `jsonrepair` 与 `generateText` 结合：

```ts
import { generateText, Output } from 'ai';
import { safeParseJSON } from '@ai-sdk/provider-utils';
__PROVIDER_IMPORT__;
import { jsonrepair } from 'jsonrepair';
import { z } from 'zod';

const recipeSchema = z.object({
  recipe: z.object({
    name: z.string(),
    ingredients: z.array(
      z.object({
        name: z.string(),
        amount: z.string(),
      }),
    ),
    steps: z.array(z.string()),
  }),
});

const result = await generateText({
  model: __MODEL__,
  output: Output.text(),
  prompt: 'Generate a lasagna recipe.',
});

const repairedText = jsonrepair(result.text);
const parseResult = safeParseJSON({ text: repairedText });

if (!parseResult.success) {
  throw parseResult.error;
}

const output = recipeSchema.parse(parseResult.value);
console.log(output.recipe);
```

## 工作原理

流程分三步：

1. 从模型生成纯文本（`Output.text()`）。
2. 用 `jsonrepair` 修复畸形 JSON。
3. 安全解析并用 schema 校验。

## jsonrepair 能修复什么

`jsonrepair` 可自动修复许多常见问题：

- Missing closing brackets: `{"name": "test"` -> `{"name": "test"}`
- Single quotes: `{'name': 'test'}` -> `{"name": "test"}`
- Missing quotes around keys: `{name: "test"}` -> `{"name": "test"}`
- Trailing commas: `{"items": [1, 2, 3,]}` -> `{"items": [1, 2, 3]}`
- Comments in JSON
- Unescaped special characters

## 注意事项

1. **尽力修复** — `jsonrepair` 能处理很多情况，但无法修复所有畸形 JSON（例如语义错误但语法合法的 JSON）
2. **Schema 校验** — 修复后对象仍须符合 schema；若模型产出结构错误的数据，修复也无济于事
3. **严重截断** — 响应被严重截断时，可提高 `maxOutputTokens` 或简化 schema
