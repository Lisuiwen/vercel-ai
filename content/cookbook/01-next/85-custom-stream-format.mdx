# 自定义格式流式输出

创建自定义流，控制流式格式与工具调用结构，而非使用 AI SDK 内置数据流格式（`toUIMessageStream()`）。

`StreamTextResult` 上的 `fullStream` 可直接访问所有模型事件，你可将其转换、过滤并构造成自己的流式格式，在享受 AI SDK 统一提供商接口的同时自由消费流。

你可以：

- 定义自己的流块格式
- 控制步骤与工具调用的结构
- 在客户端手动解析流
- 根据流数据构建自定义 UI

若需同时完全控制流式格式与执行循环，可将本模式与[手动 agent 循环](/cookbook/node/manual-agent-loop)结合。

## 实现

### 服务端

创建路由处理器：调用模型并以自定义格式流式返回响应：

```tsx filename="app/api/stream/route.ts"
import { tools } from '@/ai/tools'; // your tools
import { isStepCount, streamText } from 'ai';
__PROVIDER_IMPORT__;

export type StreamEvent =
  | { type: 'text'; text: string }
  | { type: 'tool-call'; toolName: string; input: unknown }
  | { type: 'tool-result'; toolName: string; result: unknown };

const encoder = new TextEncoder();

function formatEvent(event: StreamEvent): Uint8Array {
  return encoder.encode('data: ' + JSON.stringify(event) + '\n\n');
}

export async function POST(request: Request) {
  const { prompt } = await request.json();

  const result = streamText({
    prompt,
    model: __MODEL__,
    tools,
    stopWhen: isStepCount(5),
  });

  const transformStream = new TransformStream({
    transform(chunk, controller) {
      switch (chunk.type) {
        case 'text-delta':
          controller.enqueue(formatEvent({ type: 'text', text: chunk.text }));
          break;
        case 'tool-call':
          controller.enqueue(
            formatEvent({
              type: 'tool-call',
              toolName: chunk.toolName,
              input: chunk.input,
            }),
          );
          break;
        case 'tool-result':
          controller.enqueue(
            formatEvent({
              type: 'tool-result',
              toolName: chunk.toolName,
              result: chunk.output,
            }),
          );
          break;
      }
    },
  });

  return new Response(result.fullStream.pipeThrough(transformStream), {
    headers: { 'Content-Type': 'text/event-stream' },
  });
}
```

路由使用 `streamText` 处理带工具的 prompt。每个事件（文本、工具调用、工具结果）编码为带 `data: ` 前缀的 Server-Sent Event 并发送给客户端。

### 客户端

创建简单界面以解析并展示流：

```tsx filename="app/page.tsx"
'use client';

import { useState } from 'react';
import { StreamEvent } from './api/stream/route';

export default function Home() {
  const [prompt, setPrompt] = useState('');
  const [events, setEvents] = useState<StreamEvent[]>([]);
  const [isStreaming, setIsStreaming] = useState(false);

  const handleSubmit = async () => {
    setEvents([]);
    setIsStreaming(true);
    setPrompt('');

    const response = await fetch('/api/stream', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ prompt }),
    });

    const reader = response.body?.getReader();
    const decoder = new TextDecoder();

    if (reader) {
      let buffer = '';
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';

        for (const line of lines) {
          if (line.trim()) {
            const dataStr = line.replace(/^data: /, '');
            const event = JSON.parse(dataStr) as StreamEvent;
            setEvents(prev => [...prev, event]);
          }
        }
      }
    }

    setIsStreaming(false);
  };

  return (
    <div>
      <input
        value={prompt}
        onChange={e => setPrompt(e.target.value)}
        placeholder="Enter a prompt..."
      />
      <button onClick={handleSubmit} disabled={isStreaming}>
        {isStreaming ? 'Streaming...' : 'Send'}
      </button>

      <pre>{JSON.stringify(events, null, 2)}</pre>
    </div>
  );
}
```

## 工作原理

客户端用 Fetch API 从服务端流式接收响应。服务端发送 Server-Sent Events（以换行分隔、带 `data: ` 前缀），客户端：

1. 用 `getReader()` 读取流块
2. 将二进制块解码为文本
3. 按换行拆分以识别完整事件
4. 去掉 `data: ` 前缀并解析 JSON，追加到事件列表

事件按到达顺序渲染，线性展示 AI 的响应。
