# 跟踪 Agent 的 Token 用量

<Note>
  关于构建 agent 的更多信息，请参阅 [ToolLoopAgent
  文档](/docs/reference/ai-sdk-core/tool-loop-agent)。
</Note>

在 agent 应用中跟踪 token 消耗有助于监控成本并实施上下文管理策略。
本示例展示如何跨步骤跟踪用量，并在 agent 整个生命周期内使其可用。

## 从基础 Agent 开始

首先搭建带工具的 `ToolLoopAgent`。使用 `InferAgentUIMessage` 定义 `AgentUIMessage` 类型，在前端获得类型安全的消息（含类型化的工具调用与结果）。

```typescript filename='ai/agent.ts'
import { type InferAgentUIMessage, ToolLoopAgent, tool } from 'ai';
import { z } from 'zod';

export const agent = new ToolLoopAgent({
  model: 'anthropic/claude-haiku-4.5',
  tools: {
    greet: tool({
      description: 'Greets a person by their name.',
      inputSchema: z.object({ name: z.string() }),
      execute: async ({ name }) => `Greeted ${name}`,
    }),
  },
});

export type AgentUIMessage = InferAgentUIMessage<typeof agent>;
```

创建路由处理器以流式返回 agent 回复。使用 `AgentUIMessage` 为来自客户端的消息提供类型。

```tsx filename='app/api/chat/route.ts'
import { convertToModelMessages } from 'ai';
import { type AgentUIMessage, agent } from '@/ai/agent';

export async function POST(req: Request) {
  const { messages }: { messages: AgentUIMessage[] } = await req.json();

  const result = await agent.stream({
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse();
}
```

再搭建基于 `useChat` 的基础聊天界面。将 `AgentUIMessage` 作为泛型传入，以类型安全地访问消息（含类型化的工具调用与结果）。

```tsx filename='app/page.tsx'
'use client';

import { type AgentUIMessage } from '@/ai/agent';
import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const [input, setInput] = useState('');
  const { messages, sendMessage } = useChat<AgentUIMessage>();

  return (
    <div>
      {messages.map(m => (
        <div key={m.id}>
          <strong>{m.role}:</strong>
          {m.parts.map(
            (p, i) => p.type === 'text' && <span key={i}>{p.text}</span>,
          )}
        </div>
      ))}
      <form
        onSubmit={e => {
          e.preventDefault();
          sendMessage({ text: input });
          setInput('');
        }}
      >
        <input value={input} onChange={e => setInput(e.target.value)} />
      </form>
    </div>
  );
}
```

## 通过消息元数据访问步骤间用量

要跟踪 token 用量，可通过 `messageMetadata` 回调将其附加到每条消息。先定义元数据类型，并作为第二个泛型传给 `InferAgentUIMessage`。

```typescript filename='ai/agent.ts' highlight="1,15-16"
import {
  type InferAgentUIMessage,
  type LanguageModelUsage,
  ToolLoopAgent,
  tool,
} from 'ai';
import { z } from 'zod';

export const agent = new ToolLoopAgent({
  model: 'anthropic/claude-haiku-4.5',
  tools: {
    greet: tool({
      description: 'Greets a person by their name.',
      inputSchema: z.object({ name: z.string() }),
      execute: async ({ name }) => `Greeted ${name}`,
    }),
  },
});

type AgentMetadata = { usage: LanguageModelUsage };
export type AgentUIMessage = InferAgentUIMessage<typeof agent, AgentMetadata>;
```

在路由处理器中添加 `messageMetadata` 回调。向 `toUIMessageStreamResponse` 传入 `AgentUIMessage` 泛型以约束回调类型。步骤结束时，`finish-step` 部分包含用量数据，可写入消息元数据。

```tsx filename='app/api/chat/route.ts' highlight="11-17"
import { convertToModelMessages } from 'ai';
import { type AgentUIMessage, agent } from '@/ai/agent';

export async function POST(req: Request) {
  const { messages }: { messages: AgentUIMessage[] } = await req.json();

  const result = await agent.stream({
    messages: await convertToModelMessages(messages),
  });

  return result.toUIMessageStreamResponse<AgentUIMessage>({
    messageMetadata: ({ part }) => {
      if (part.type === 'finish-step') {
        return { usage: part.usage };
      }
    },
  });
}
```

现在可在客户端访问元数据。`AgentUIMessage` 已包含元数据结构，可类型安全地访问 `m.metadata.usage`。

```typescript filename='app/page.tsx' highlight="17-19"
'use client';

import { type AgentUIMessage } from '@/ai/agent';
import { useChat } from '@ai-sdk/react';
import { useState } from 'react';

export default function Chat() {
  const [input, setInput] = useState('');
  const { messages, sendMessage } = useChat<AgentUIMessage>();

  return (
    <div>
      {messages.map((m) => (
        <div key={m.id}>
          <strong>{m.role}:</strong>
          {m.parts.map((p, i) => p.type === 'text' && <span key={i}>{p.text}</span>)}
          {m.metadata?.usage && (
            <div>Input tokens: {m.metadata.usage.inputTokens}</div>
          )}
        </div>
      ))}
      <form onSubmit={(e) => {
        e.preventDefault();
        sendMessage({ text: input });
        setInput('');
      }}>
        <input value={input} onChange={(e) => setInput(e.target.value)} />
      </form>
    </div>
  );
}
```

## 通过 Call Options 将用量传回 Agent

UI 已能展示用量数据。若要根据这些数据采取行动呢？例如接近 token 上限时实施上下文压缩。

操纵消息或应用上下文管理策略时，会使用 `prepareStep` 回调。但 `prepareStep` 只能访问当前运行中的步骤；新请求的第一步 `steps` 为空，无法得知此前请求中对话已累计多少 token。

解决方法是将先前消息的用量传回 agent：用 `callOptionsSchema` 定义数据结构，用 `prepareCall` 将其放到 `context` 上，供 `prepareStep` 访问。

```typescript filename='ai/agent.ts' highlight="11-13,21-26"
import {
  type InferAgentUIMessage,
  type LanguageModelUsage,
  ToolLoopAgent,
  tool,
} from 'ai';
import { z } from 'zod';

export const agent = new ToolLoopAgent({
  model: 'anthropic/claude-haiku-4.5',
  callOptionsSchema: z.object({
    lastInputTokens: z.number(),
  }),
  tools: {
    greet: tool({
      description: 'Greets a person by their name.',
      inputSchema: z.object({ name: z.string() }),
      execute: async ({ name }) => `Greeted ${name}`,
    }),
  },
  prepareCall: ({ options, ...settings }) => {
    return {
      ...settings,
      context: { lastInputTokens: options.lastInputTokens },
    };
  },
});

type AgentMetadata = { usage: LanguageModelUsage };
export type AgentUIMessage = InferAgentUIMessage<typeof agent, AgentMetadata>;
```

从先前消息中提取最后一次的 input token 数量并传给 agent。

```tsx filename='app/api/chat/route.ts' highlight="7-9,13-15"
import { convertToModelMessages } from 'ai';
import { type AgentUIMessage, agent } from '@/ai/agent';

export async function POST(req: Request) {
  const { messages }: { messages: AgentUIMessage[] } = await req.json();

  const lastInputTokens =
    messages.filter(m => m.role === 'assistant').at(-1)?.metadata?.usage
      ?.inputTokens ?? 0;

  const result = await agent.stream({
    messages: await convertToModelMessages(messages),
    options: {
      lastInputTokens,
    },
  });

  return result.toUIMessageStreamResponse<AgentUIMessage>({
    messageMetadata: ({ part }) => {
      if (part.type === 'finish-step') {
        return { usage: part.usage };
      }
    },
  });
}
```

## 在 prepareStep 与工具中访问用量

用量在 `context` 上后，可在 `prepareStep` 中用于上下文管理决策，或传给工具。

```typescript filename='ai/agent.ts' highlight="9-11,31-40"
import {
  type InferAgentUIMessage,
  type LanguageModelUsage,
  ToolLoopAgent,
  tool,
} from 'ai';
import { z } from 'zod';

type TContext = {
  lastInputTokens: number;
};

export const agent = new ToolLoopAgent({
  model: 'anthropic/claude-haiku-4.5',
  callOptionsSchema: z.object({
    lastInputTokens: z.number(),
  }),
  tools: {
    greet: tool({
      description: 'Greets a person by their name.',
      inputSchema: z.object({ name: z.string() }),
      execute: async ({ name }) => `Greeted ${name}`,
    }),
  },
  prepareCall: ({ options, ...settings }) => {
    return {
      ...settings,
      context: { lastInputTokens: options.lastInputTokens },
    };
  },
  prepareStep: ({ steps, context }) => {
    const lastStep = steps.at(-1);
    const lastStepUsage =
      lastStep?.usage?.inputTokens ??
      (context as TContext)?.lastInputTokens ??
      0;
    console.log('Last step input tokens:', lastStepUsage);
    // You can use this to implement context compaction strategies
    return {
      context: {
        ...context,
        lastStepUsage,
      },
    };
  },
});

type AgentMetadata = { usage: LanguageModelUsage };
export type AgentUIMessage = InferAgentUIMessage<typeof agent, AgentMetadata>;
```

`prepareStep` 在每一步之前运行，可访问：

- `steps`：此前所有步骤及其用量数据
- `context`：由 `prepareCall` 设置的上下文（来自上一次请求的用量）

这样即可在整个对话生命周期内跟踪 token 消耗，并在接近上限时实施上下文压缩等策略。
