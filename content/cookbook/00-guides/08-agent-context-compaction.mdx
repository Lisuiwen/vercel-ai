# 压缩 Agent 上下文

本指南介绍如何通过从 `prepareStep` 返回新的 `messages` 数组来压缩 agent 上下文。

示例使用内置辅助函数 `pruneMessages` 移除选定的消息与消息部分。你可使用任意压缩逻辑；核心行为是 `prepareStep` 可变更后续步骤收到的消息状态。

## 从不断增长的 Agent 循环开始

会调用工具的 agent 会累积大量消息历史。每次工具调用与工具结果都会成为下一步的上下文。

本示例使用返回较长结果的工具：

```ts
import { ToolLoopAgent, isStepCount, tool } from 'ai';
import { z } from 'zod';
__PROVIDER_IMPORT__;

const readDocument = tool({
  description: 'Read a document by name',
  inputSchema: z.object({
    name: z.string(),
  }),
  execute: async ({ name }) => {
    return {
      name,
      text: await loadLargeDocument(name),
    };
  },
});

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    readDocument,
  },
  stopWhen: isStepCount(10),
});

const result = await agent.generate({
  prompt: 'Read the project documents and summarize the migration plan.',
});
```

这样可行，但每次工具调用后消息列表都会变长。若 agent 读取多份大文档，后续步骤可能仍发送模型已不再需要完整内容的旧工具结果。

## 添加压缩触发条件

由你决定何时压缩。

本示例使用简单的 token 估算：

```ts
import type { ModelMessage } from 'ai';

const COMPACT_AFTER_TOKENS = 100_000;

const estimateTokens = (messages: ModelMessage[]) => {
  return JSON.stringify(messages).length / 4;
};
```

若需更精确统计，可使用真实分词器或提供商用量数据。具体触发条件不影响该模式本身。

## 在 prepareStep 中压缩消息

`prepareStep` 在每次模型步骤之前运行，接收该步将发给模型的 `messages`。

返回新的 `messages` 数组时，SDK 会将其用于当前步骤，并作为后续步骤的基础。

```ts
import {
  ToolLoopAgent,
  isStepCount,
  pruneMessages,
  tool,
  type ModelMessage,
} from 'ai';
import { z } from 'zod';
__PROVIDER_IMPORT__;

const COMPACT_AFTER_TOKENS = 100_000;

const estimateTokens = (messages: ModelMessage[]) => {
  return JSON.stringify(messages).length / 4;
};

const readDocument = tool({
  description: 'Read a document by name',
  inputSchema: z.object({
    name: z.string(),
  }),
  execute: async ({ name }) => {
    return {
      name,
      text: await loadLargeDocument(name),
    };
  },
});

const agent = new ToolLoopAgent({
  model: __MODEL__,
  tools: {
    readDocument,
  },
  stopWhen: isStepCount(10),
  prepareStep: ({ messages }) => {
    if (estimateTokens(messages) > COMPACT_AFTER_TOKENS) {
      return {
        messages: pruneMessages({
          messages,
          reasoning: 'all',
          toolCalls: 'before-last-3-messages',
          emptyMessages: 'remove',
        }),
      };
    }
  },
});

const result = await agent.generate({
  prompt: 'Read the project documents and summarize the migration plan.',
});
```

`pruneMessages` 只是压缩方式之一；需要不同消息形状时可替换为自定义逻辑。

## 理解持久化行为

`messages` 参数是循环当前的消息状态。若 `prepareStep` 返回 `messages`，该变更列表会延续到后续步骤；新的助手与工具响应消息会随循环继续追加。

若希望基于原始输入与迄今的模型响应构建某一步，可使用 `initialMessages` 与 `responseMessages`：

```ts
prepareStep: ({ initialMessages, responseMessages, stepNumber }) => {
  if (stepNumber > 0) {
    return {
      messages: [...initialMessages, ...responseMessages.slice(-10)],
    };
  }
};
```

当你不希望上一次的 `messages` 覆盖成为下一步起点时，这很有用。

## 在核心函数中使用相同模式

`generateText` 与 `streamText` 同样支持上述 `prepareStep` 行为：

```ts
import { generateText, isStepCount, pruneMessages } from 'ai';
__PROVIDER_IMPORT__;

const result = await generateText({
  model: __MODEL__,
  prompt: 'Read the project documents and summarize the migration plan.',
  tools: {
    readDocument,
  },
  stopWhen: isStepCount(10),
  prepareStep: ({ messages }) => {
    if (estimateTokens(messages) > COMPACT_AFTER_TOKENS) {
      return {
        messages: pruneMessages({
          messages,
          reasoning: 'all',
          toolCalls: 'before-last-3-messages',
          emptyMessages: 'remove',
        }),
      };
    }
  },
});
```
